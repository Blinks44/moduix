## API Reference

Ark UI does not ship a dedicated `bleed` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Basic

Start with the default `inline="full"` behavior when media, banners, or section accents should
break out of a constrained container while staying in normal document flow.

## Anatomy

`Bleed` has a single Ark-aligned root element. It applies negative inline and/or block margins on
that root.

```text
Bleed.Root
└─ root[data-scope="bleed"][data-part="root"][data-slot="bleed-root"]
   └─ children
````

| Part                   | Role                                            |
| ---------------------- | ----------------------------------------------- |
| `Bleed` / `Bleed.Root` | Root layout wrapper for inline and block bleed. |
| `Bleed`                | Callable alias of `Bleed.Root`.                 |

## Composition

### Recommended composition

Use the shorter `Bleed` form when the layout only needs controlled negative margins on one wrapper.
`Bleed.Root` is the equivalent Ark-aligned namespace form.
This is the intended path for full-bleed media, inset section backgrounds, dividers, and banners
inside centered or padded shells.

### When to go custom

Use `className` and local `--moduix-bleed-*` overrides when a drawer, nested scroll region, or app shell
needs different full-bleed math than the default viewport-based values.

### Default props

| Entry       | Default | Values / Notes                               |
| ----------- | ------- | -------------------------------------------- |
| `inline`    | `full`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `block`     | `none`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`         |
| `asChild`   | `false` | Ark factory composition                      |
| `className` | -       | Applied to the root                          |

### Behavioral notes

- `Bleed.Root` renders exactly one root and forwards Ark factory div props to it.
- `ref` targets the rendered root element. With `asChild`, it targets the single child element.
- `asChild` requires one semantic child that can accept DOM props and a ref.
- Use `inline` for horizontal escape, and add `block` when a surface should also escape vertical
  padding.
- The old `as` prop was removed during the Ark migration. Use `asChild` when a semantic element
  should own the DOM node.
- Use one of the fixed scale values when content only needs to escape container padding instead of
  reaching the viewport edge.
- `inline="full"` uses viewport math, so nested shells may need local overrides for
  `--moduix-bleed-inline-full` and `--moduix-bleed-inline-full-size`.
- Ancestors with `overflow: hidden` or `overflow: clip` can crop the bleed.
- Unlike Chakra's layout primitive, moduix intentionally exposes token-based `inline` and `block`
  modes rather than one-sided `inlineStart`, `inlineEnd`, `blockStart`, and `blockEnd` props.

## Styling

`Bleed` exposes a small CSS variable contract and stable root data attributes. Use variables when
the layout math needs to change, and use `className` for the visual surface inside the bleed.

### CSS Properties

<CssPropertiesSection properties={BleedDocs.bleedCssProperties} />

### Styling hooks

- `data-scope="bleed"` and `data-part="root"` provide Ark-aligned hooks on the exported root.
- `data-slot="bleed-root"` is the stable moduix selector on the exported root.
- `data-inline` reflects the current inline bleed mode: `none`, `xs`, `sm`, `md`, `lg`, `xl`, or
  `full`.
- `data-block` reflects the current block bleed mode: `none`, `xs`, `sm`, `md`, `lg`, or `xl`.
- The public `--moduix-bleed-*` variables in `theme.css` cover both scale-based bleed amounts and the
  viewport math used by `inline="full"`.
