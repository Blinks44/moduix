## Choosing the right component

Use `Select` when users choose from a fixed collection and the trigger should stay read-only.

- Choose **`Combobox`** when typing is part of finding or creating the value.
- Choose **`Select`** when the user picks from known values, including grouped or multiple values.

## Basic

Start with `Select`, pass a `collection`, render the visible field through `Label` and
`Select.Field`, then render the popup structure explicitly. Pass `clearLabel` when the clear action
is needed. The root renders its native select automatically for native form submission and browser
autofill.

```text

├─ Select.Label
├─ Select.Control / Select.Field
│  ├─ Select.Trigger
│  │  ├─ Select.ValueText
│  │  └─ Select.Indicator
│  └─ Select.ClearTrigger
├─ Overlay subtree (automatically portalled)
│  └─ Select.Positioner
│     └─ Select.Content
│        ├─ Select.List
│        └─ Select.ItemGroup
│           ├─ Select.ItemGroupLabel
│           └─ Select.Item[item]
│              ├─ Select.ItemText
│              └─ Select.ItemIndicator
└─ native select (automatic)
```

| Part                     | Role                                                                   |
| ------------------------ | ---------------------------------------------------------------------- |
| `Select` / `Select.Root` | Root state, collection, selection, popup lifecycle, and form data.     |
| `Select.RootProvider`    | Renders a select tree from state returned by `useSelect`.              |
| `Select.Label`           | Accessible label inherited by the trigger and native select.           |
| `Select.Control`         | Visual field wrapper that receives focus, invalid, and open state.     |
| `Select.Field`           | Moduix helper that renders the standard control, trigger, and actions. |
| `Select.Trigger`         | Button that opens the popup and owns keyboard entry.                   |
| `Select.ValueText`       | Placeholder or selected item labels.                                   |
| `Select.ClearTrigger`    | Button that clears the current value.                                  |
| `Select.Indicator`       | Trigger indicator with the moduix default chevron icon.                |
| `Select.Positioner`      | Floating layer that exposes Ark positioning CSS variables.             |
| `Select.Content`         | Popup surface and listbox container.                                   |
| `Select.List`            | Optional list wrapper for virtualized or custom list layouts.          |
| `Select.ItemGroup`       | Group wrapper for related collection items.                            |
| `Select.ItemGroupLabel`  | Label for a collection group.                                          |
| `Select.Item`            | Selectable row. Pass the collection item through `item`.               |
| `Select.ItemText`        | Text used for display and accessibility.                               |
| `Select.ItemIndicator`   | Selected-state indicator with the moduix default check icon.           |

## Composition

The root portals `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Use `Select.Field` for the standard read-only field. It renders `Control`, `Trigger`, `ValueText`,
and the indicator; pass `clearLabel` to include a labeled clear action or `indicator={<Icon />}`
to replace the default chevron. Keep the popup structure explicit with `Positioner` and `Content`.

Use the explicit Ark part tree: root, label, control, trigger, value text, optional clear and
indicator buttons, automatically portalled positioner/content, and items. The root renders its
native select automatically.

`Select` is the short root form and is equivalent to `Select.Root`.

### When to go custom

Use `asChild` when a part must render a custom host element, but keep a single semantic child that
can receive Ark props. Use `Select.List` for custom list wrappers and virtualization paths; regular
selects can render items directly in `Select.Content`.

### Default props

| Prop                    | Default | Notes                                                          |
| ----------------------- | ------- | -------------------------------------------------------------- |
| `closeOnSelect`         | `true`  | Closes the popup after item selection.                         |
| `composite`             | `true`  | Allows composition with nested composite widgets.              |
| `defaultValue`          | `[]`    | Values are string arrays, including single selection.          |
| `lazyMount`             | `false` | Defers popup content mounting until opened.                    |
| `loopFocus`             | `false` | Keeps arrow navigation from wrapping by default.               |
| `multiple`              | `false` | Enables multiple values when set.                              |
| `positioning`           | -       | Ark floating positioning options.                              |
| `unmountOnExit`         | `false` | Removes popup content after close when set with lazy mounting. |
| `ValueText.placeholder` | -       | Text shown when the value array is empty.                      |

### Behavioral notes

For external state and context reads, use `Select.useSelect`, `Select.Context`,
`Select.ItemContext`, `Select.useSelectContext`, and `Select.useSelectItemContext`.

`value`, `defaultValue`, and `onValueChange(details)` use string arrays. Read
`details.value` for selected keys and `details.items` for the selected collection items.

`onOpenChange(details)`, `onHighlightChange(details)`, and `onSelect(details)` keep Ark's detail
object signatures. `Field.Root` and `Fieldset.Root` can provide disabled, invalid, required, and
read-only state through Ark context.

The root renders its native select automatically for native submission, autofill, and reset behavior.
For very large virtualized collections, set `nativeFormControl="input"` to submit lightweight
hidden inputs instead of rendering one native option per item. This mode deliberately omits native
select autofill and progressive enhancement.

### Max Selection

<ExampleFrame clientOnly>
  <SelectDocs.MaxSelectionSelectExample />

```tsx
import { createListCollection } from '@ark-ui/react/collection';
import { Select } from '@moduix/react';

const value = ['javascript'];

const collection = createListCollection({
  items: [
    {
      label: 'C#',
      value: 'csharp',
      disabled: false,
    },
    {
      label: 'Go',
      value: 'go',
      disabled: false,
    },
    {
      label: 'JavaScript',
      value: 'javascript',
      disabled: false,
    },
    {
      label: 'Python',
      value: 'python',
      disabled: false,
    },
    {
      label: 'Rust',
      value: 'rust',
      disabled: false,
    },
    {
      label: 'TypeScript',
      value: 'typescript',
      disabled: false,
    },
  ],
});

export default function SelectMaxSelectionDemo() {
  return (
    <Select
      collection={collection}
      multiple
      value={value}
      onValueChange={(details) => {
        if (details.value.length <= 3) setValue(details.value);
      }}
    >
      <Select.Label>Languages</Select.Label>
      <Select.Control>
        <Select.Trigger>
          <Select.ValueText placeholder="Select up to 3" />
          <Select.Indicator />
        </Select.Trigger>
        <Select.ClearTrigger aria-label="Clear selection" />
      </Select.Control>
      <Select.Positioner>
        <Select.Content>
          {collection.items.map((item) => (
            <Select.Item key={item.value} item={item}>
              <Select.ItemText>{item.label}</Select.ItemText>
              <Select.ItemIndicator />
            </Select.Item>
          ))}
        </Select.Content>
      </Select.Positioner>
    </Select>
  );
}
```

### Styling hooks

`Select.Field` renders the existing `select-control`, `select-trigger`, `select-value-text`,
`select-indicator`, and optional `select-clear-trigger` slots, so it uses the same styling
contract as explicit composition. Its `indicator` prop supplies the child of the existing
`select-indicator` slot. In explicit markup, keep `Select.Indicator` inside `Select.Trigger` and
render `Select.ClearTrigger` directly under `Select.Control`.

- Root and root provider expose `data-slot="select-root"` and `data-slot="select-root-provider"`.
- Ark parts keep `data-scope="select"` and `data-part` for anatomy selectors.
- State styling uses `data-state="open" | "closed"`, `data-focus`, `data-invalid`,
  `data-disabled`, `data-readonly`, `data-placeholder-shown`, `data-highlighted`, and
  `data-state="checked" | "unchecked"` on items.
- The trigger shows its focus ring while open and for keyboard `:focus-visible` focus; dismissing
  a popup with a pointer does not leave a ring on the closed field.
- `Select.Positioner` exposes `--reference-width`, `--reference-height`, `--available-width`,
  `--available-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`.
