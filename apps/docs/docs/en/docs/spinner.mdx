## Anatomy

The normal consumer path is short:

```text
Spinner / Spinner.Root
└─ spinner-indicator
   └─ spinner-ring (default indicator only)
```

| Part                       | Role                                                                                  |
| -------------------------- | ------------------------------------------------------------------------------------- |
| `Spinner` / `Spinner.Root` | Root status element. Supports `size`, `decorative`, `asChild`, and native span props. |
| `spinner-indicator`        | Inner rotating wrapper that keeps sizing and spin behavior.                           |
| `spinner-ring`             | Default ring indicator. Renders only when custom `children` are absent.               |

## Composition

### Recommended composition

Stay with the default ring when a compact loading indicator is enough.

- Use `size` for density changes.
- Use `decorative` when adjacent visible text already communicates the loading state.
- Use `aria-label` or `aria-labelledby` when the status name should be more specific than
  `"Loading"`.

### When to go custom

`Spinner` is a root-only component. Use `children` to replace the default ring while keeping the
default root, sizing, and rotating indicator wrapper.

Use `asChild` when the root element itself must be provided by the consumer. In that mode, Ark merges
root props into the single child, so the child owns its visual contents.

### Default props

| Prop         | Default     | Notes                                                    |
| ------------ | ----------- | -------------------------------------------------------- |
| `size`       | `'md'`      | Accepts `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'`.          |
| `decorative` | `false`     | Switches the root from status semantics to presentation. |
| `asChild`    | `false`     | Merges root props into the single child element.         |
| `aria-label` | `'Loading'` | Applied by default when the spinner is not decorative.   |

### Behavioral notes

- `Spinner` and `Spinner.Root` are the same root component.
- The default root renders as a `span` with `data-scope="spinner"`, `data-part="root"`,
  `data-slot="spinner-root"`, and `data-size`.
- Non-decorative spinners render with `role="status"`.
- `decorative={true}` switches the root to `role="presentation"` and `aria-hidden="true"`.
- `Spinner` owns these semantics and its `data-scope`, `data-part`, `data-slot`, and `data-size`
  hooks, so native props cannot override them.
- The rotating wrapper is `data-scope="spinner"`, `data-part="indicator"`, and
  `data-slot="spinner-indicator"`.
- Custom `children` replace only the default ring. The wrapper still owns sizing and rotation.
- With `asChild`, the consumer-provided child becomes the root and must provide any visual indicator
  content it needs.
- Spinner motion is disabled automatically under `prefers-reduced-motion`.

### Styling hooks

- `[data-scope="spinner"][data-part="root"]` targets the Ark-style root.
- `[data-scope="spinner"][data-part="indicator"]` targets the rotating inner wrapper.
- `[data-scope="spinner"][data-part="ring"]` targets the default ring only.
- `data-slot="spinner-root"` is the stable moduix selector for the exported root.
- `data-slot="spinner-indicator"` is the stable moduix selector for the rotating inner wrapper.
- `data-slot="spinner-ring"` targets the default ring only.
- `Spinner` writes `data-size="xs" | "sm" | "md" | "lg" | "xl"` on the root.
- `className` applies only to the root. Target the indicator or ring with descendant selectors from
  that root when needed.
