## Anatomy

```text
Clipboard.Root | Clipboard.RootProvider
├─ Clipboard.Label (optional)
├─ Clipboard.Control
│  ├─ Clipboard.Input or Clipboard.ValueText
│  └─ Clipboard.Trigger
│     ├─ Clipboard.Indicator (optional)
│     └─ Clipboard.CopyText (optional)
```

| Part                           | Role                                                                  |
| ------------------------------ | --------------------------------------------------------------------- |
| `Clipboard` / `Clipboard.Root` | Owns clipboard state, copied lifecycle, and controlled value.         |
| `Clipboard.RootProvider`       | Injects `Clipboard.useClipboard()` state created outside the subtree. |
| `Clipboard.Label`              | Accessible label for the clipboard value.                             |
| `Clipboard.Control`            | Layout row for value surface and trigger.                             |
| `Clipboard.Input`              | Editable or read-only text field for the current value.               |
| `Clipboard.Trigger`            | Button that copies the current value.                                 |
| `Clipboard.Indicator`          | Copy-state icon slot. moduix defaults to copy and check icons.        |
| `Clipboard.CopyText`           | Copy-state text slot. moduix defaults to `Copy` and `Copied`.         |
| `Clipboard.ValueText`          | Text-only value surface when an input is not the right control.       |

## Composition

### Recommended composition

Stay with `Clipboard`, `Clipboard.Label`, `Clipboard.Control`, `Clipboard.Input`,
`Clipboard.Trigger`, and `Clipboard.Indicator` for the normal Ark path.

`Clipboard` is the short root form and is equivalent to `Clipboard.Root`.

Use `value`, `defaultValue`, `onValueChange(details)`, `onStatusChange(details)`, `timeout`, and
`ids` exactly as Ark defines them. moduix does not remap callback payloads or rename props on this
surface.

`Clipboard.Indicator` and `Clipboard.CopyText` are the two narrow sugar layers: they render moduix
copy/check icons and `Copy`/`Copied` text by default while keeping the Ark trigger structure
explicit.

### When to go custom

Use `Clipboard.ValueText` when the value should be displayed as text instead of inside an input.

Use `Clipboard.useClipboard()` with `Clipboard.RootProvider` when clipboard state must be created
outside the rendered subtree. Use `Clipboard.Context` or `useClipboardContext()` from moduix to read
that state within a descendant.

Use `asChild` on `Clipboard.Input` and `Clipboard.Trigger` when another element must own the
rendered node, such as moduix `Input` or `Button` during a migration from a shadcn-style field plus
button layout. Keep the Ark part tree visible instead of hiding it behind a convenience wrapper.

### Default props

| Prop                 | Default  | Values / Notes                                       |
| -------------------- | -------- | ---------------------------------------------------- |
| `timeout`            | `3000`   | Ark copied-state timeout in milliseconds             |
| `Indicator.copied`   | —        | `Clipboard.Indicator` defaults to moduix `CheckIcon` |
| `Indicator.children` | —        | `Clipboard.Indicator` defaults to moduix `CopyIcon`  |
| `CopyText.copied`    | `Copied` | `Clipboard.CopyText` copied-state label              |
| `CopyText.children`  | `Copy`   | `Clipboard.CopyText` idle label                      |

### Behavioral notes

- `onValueChange(details)` exposes the next clipboard value as `details.value`.
- `onStatusChange(details)` follows Ark copy-status details and is the right hook for analytics,
  counters, or adjacent feedback.
- Ark applies `data-copied` to the relevant parts while copied state is active. By default moduix
  keeps the input and trigger visually stable and only swaps `Indicator` or `CopyText`, while
  `data-copied` remains available for opt-in custom styling.
- `Clipboard` no longer caps field width by default. Set `--moduix-clipboard-max-width` when a specific
  maximum width belongs to the layout.
- The old `CopyButton` API was removed. There is no `copiedDuration`, `copyLabels`, `onCopy`, or
  `onCopyError` alias layer on `Clipboard`.

### CSS Properties

### Styling hooks

- `Clipboard.Root` -> `data-slot="clipboard-root"`
- `Clipboard.RootProvider` -> `data-slot="clipboard-root-provider"`
- `Clipboard.Label` -> `data-slot="clipboard-label"`
- `Clipboard.Control` -> `data-slot="clipboard-control"`
- `Clipboard.Input` -> `data-slot="clipboard-input"`
- `Clipboard.Trigger` -> `data-slot="clipboard-trigger"`
- `Clipboard.Indicator` -> `data-slot="clipboard-indicator"`
- `Clipboard.CopyText` -> `data-slot="clipboard-copy-text"`
- `Clipboard.ValueText` -> `data-slot="clipboard-value-text"`
- Ark adds `data-copied` to `Root`, `Label`, `Control`, `Input`, and `Trigger` while copied state is active.
- Ark adds `data-readonly` to `Clipboard.Input` when the input is read-only.
- `Clipboard.Indicator` adds `data-slot="clipboard-indicator-idle-icon"` and
  `data-slot="clipboard-indicator-copied-icon"` for the default icon nodes.
