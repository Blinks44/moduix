## Anatomy

```text
ProgressLinear.Root
├─ ProgressLinear.Label
├─ ProgressLinear.ValueText
└─ ProgressLinear.Track
   └─ ProgressLinear.Range

ProgressLinear.RootProvider
└─ same part tree connected to ProgressLinear.useProgress()
```

| Part                                     | Role                                                                 |
| ---------------------------------------- | -------------------------------------------------------------------- |
| `ProgressLinear` / `ProgressLinear.Root` | Owns progress state, formatting, and ids.                            |
| `ProgressLinear.RootProvider`            | Connects parts to state created with `ProgressLinear.useProgress()`. |
| `ProgressLinear.Context`                 | Render-prop access to the current progress state.                    |
| `ProgressLinear.Label`                   | Visible label for the progressbar.                                   |
| `ProgressLinear.ValueText`               | Visible value text generated from Ark translations.                  |
| `ProgressLinear.Track`                   | Linear rail with progressbar role and ARIA value attributes.         |
| `ProgressLinear.Range`                   | Filled range that reflects the current percent.                      |
| `ProgressLinear.View`                    | Conditional view for `indeterminate`, `loading`, or `complete`.      |

## Composition

### Recommended composition

`ProgressLinear` is the short root form. It is equivalent to `ProgressLinear.Root`, and the child
parts stay explicit so the rendered anatomy matches Ark UI.

Use `defaultValue` for uncontrolled progress, `value` with `onValueChange(details)` for controlled
progress, and `defaultValue={null}` or `value={null}` for indeterminate progress.

### When to go custom

Use `ProgressLinear.RootProvider` with `ProgressLinear.useProgress()` when progress state must be
created outside the rendered subtree.

Use `translations.value(details)` for custom visible and accessible value text. This replaces the
old legacy `getAriaValueText` and child render function pattern.

Use `ids` when the label, track, or surrounding layout need stable explicit IDs across a more
custom tree. Use `ProgressLinear.View` when supporting UI should switch with the Ark progress state
instead of duplicating state checks outside the component. Use `ProgressLinear.Context` or
`ProgressLinear.useProgressContext()` for state access below a root/provider. Direct Ark imports
remain an escape hatch for surfaces intentionally outside the moduix API.

### Default props

| Prop            | Default                | Notes                                                  |
| --------------- | ---------------------- | ------------------------------------------------------ |
| `defaultValue`  | `50`                   | Initial uncontrolled value.                            |
| `min`           | `0`                    | Lower bound for percent and ARIA value.                |
| `max`           | `100`                  | Upper bound for percent and ARIA value.                |
| `formatOptions` | `{ style: 'percent' }` | Controls `ValueText` formatting.                       |
| `locale`        | `'en-US'`              | Locale used by `Intl.NumberFormat`.                    |
| `orientation`   | `'horizontal'`         | Ark orientation. Use `vertical` with a defined height. |

### Behavioral notes

`ProgressLinear.Track` receives the progressbar role and ARIA value attributes from Ark. The root
stores progress state and exposes `data-state="indeterminate|loading|complete"`. Ark writes
`data-orientation` on the root, label, track, and range so horizontal and vertical styles can share
the same part tree.

All exported parts support `asChild` through Ark polymorphic composition. Keep replacement children
semantic and preserve the progressbar semantics on the track.

`ProgressLinear.ValueText` announces changes with `aria-live="polite"`. Keep the visible label in
the tree, or provide `aria-label`/`aria-labelledby` when composing without `ProgressLinear.Label`.
