## API Reference

Ark UI does not ship a dedicated kbd primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Composition

### Recommended composition

Use `Kbd` as the root shorthand and `Kbd.Group` for shortcut hints, command menus, and inline help.

Both parts accept `HTMLArkProps<'kbd'>`, forward refs to the rendered element, and support `asChild`.
There is no variant, size, separator, callback, or state API.

### When to go custom

Use `asChild` when an existing semantic `<kbd>` element must receive the moduix styles and Ark
composition behavior. The child must be one element and must preserve keyboard-input semantics.

### Behavioral notes

- `Kbd`, `Kbd.Root`, and `Kbd.Group` are static shortcut labels, not interactive controls.
- Separators such as `+`, `/`, or `then` stay in plain composition as normal text.
- Add `aria-label` to `Kbd.Group` when abbreviations or symbols need a clearer spoken form.
- The parts expose `data-scope="kbd"`, `data-part`, and stable `data-slot` attributes.

### Styling hooks

- `data-slot="kbd-root"` targets the individual keycap element
- `data-slot="kbd-group"` targets the inline semantic wrapper for grouped shortcuts
- `[data-scope="kbd"][data-part="root"]` and `[data-part="group"]` expose Ark-style anatomy selectors
- `Kbd.Group` inherits text styles for separators, and `--kbd-group-separator-color` controls their color
