## Basic

Start with `TreeView`, pass a `createTreeCollection` result, and render each recursive node through
`NodeProvider`. Use branches for nodes with children and items for leaves.

<style>{TreeViewDocs.treeViewExampleCss}</style>

<ExampleFrame>
  <TreeViewDocs.TreeViewExample />
</ExampleFrame>

```tsx
const collection = createTreeCollection({
  nodeToValue: (node) => node.id,
  nodeToString: (node) => node.name,
  rootNode: {
    id: 'ROOT',
    name: '',
    children: [
      { id: 'src', name: 'src', children: [{ id: 'src/App.tsx', name: 'App.tsx' }] },
      { id: 'package.json', name: 'package.json' },
    ],
  },
});

createTreeViewCode({
  demo: `export function TreeViewDemo() {
  return (
    <TreeView collection={collection} defaultExpandedValue={["src"]}>
      <TreeView.Label>Project files</TreeView.Label>
      <TreeView.Tree>
        ${createTreeNodesCode()}
      </TreeView.Tree>
    </TreeView>
  );
}`,
});
```

## Anatomy

```tsx
<TreeView>
  <TreeView.Label />
  <TreeView.Tree>
    <TreeView.NodeProvider>
      <TreeView.Branch>
        <TreeView.BranchControl>
          <TreeView.BranchIndicator />
          <TreeView.BranchText />
        </TreeView.BranchControl>
        <TreeView.BranchContent>
          <TreeView.BranchIndentGuide />
          <TreeView.Item>
            <TreeView.ItemText />
          </TreeView.Item>
        </TreeView.BranchContent>
      </TreeView.Branch>
    </TreeView.NodeProvider>
  </TreeView.Tree>
</TreeView>
```

| Part                             | Role                                                                                        |
| -------------------------------- | ------------------------------------------------------------------------------------------- |
| `TreeView` / `TreeView.Root`     | Owns collection state, selection, expansion, focus, checking, loading, and rename behavior. |
| `TreeView.RootProvider`          | Renders a tree from a `useTreeView` instance owned outside the JSX tree.                    |
| `TreeView.Label`                 | Labels the tree for assistive technology.                                                   |
| `TreeView.Tree`                  | Contains the recursive node tree.                                                           |
| `TreeView.NodeProvider`          | Provides the current node and `indexPath` to branch and item parts.                         |
| `TreeView.Branch`                | Wraps an expandable node.                                                                   |
| `TreeView.BranchControl`         | Interactive row for a branch.                                                               |
| `TreeView.BranchTrigger`         | Optional dedicated expand/collapse trigger.                                                 |
| `TreeView.BranchIndicator`       | Visual branch state indicator.                                                              |
| `TreeView.BranchText`            | Branch label content.                                                                       |
| `TreeView.BranchContent`         | Collapsible child region.                                                                   |
| `TreeView.BranchIndentGuide`     | Optional visual guide for nested levels.                                                    |
| `TreeView.Item`                  | Leaf node row.                                                                              |
| `TreeView.ItemText`              | Leaf label content.                                                                         |
| `TreeView.ItemIndicator`         | Optional selected item indicator.                                                           |
| `TreeView.NodeCheckbox`          | Node checkbox for checked trees.                                                            |
| `TreeView.NodeCheckboxIndicator` | Checked and indeterminate visual states.                                                    |
| `TreeView.NodeRenameInput`       | Inline input shown while a node is being renamed.                                           |

## Composition

### Recommended composition

Keep the recursive `NodeProvider` visible. It is how Ark connects each node to the collection,
keyboard focus, selection, expansion, loading, checking, and rename state.

### When to go custom

Use `asChild` on `TreeView.Item` when a leaf should be a link or another semantic element. The child
must be a single element that can receive Ark props and preserve the tree interaction semantics.

### Default props

| Prop            | Default    | Notes                                               |
| --------------- | ---------- | --------------------------------------------------- |
| `selectionMode` | `"single"` | Use `"multiple"` for multiple selected node values. |
| `expandOnClick` | `true`     | Clicking a branch row expands or collapses it.      |
| `typeahead`     | `true`     | Typeahead uses `nodeToString` from the collection.  |
| `lazyMount`     | `false`    | Passed through to Ark render strategy.              |
| `unmountOnExit` | `false`    | Passed through to Ark render strategy.              |

### Behavioral notes

`expandedValue`, `selectedValue`, `checkedValue`, and `focusedValue` are controlled arrays or strings
using Ark detail callbacks. `loadChildren` returns child nodes asynchronously and
`onLoadChildrenComplete` receives the updated collection. `canRename`, `onRenameStart`,
`onBeforeRename`, and `onRenameComplete` preserve Ark rename detail objects.

Keyboard navigation, roving focus, typeahead, branch expansion, checkbox state, disabled nodes, and
inline rename handling come from Ark UI. Import advanced hooks and generic types such as
`useTreeView`, `useTreeViewNodeContext`, `TreeViewNodeProviderProps`, and
`TreeViewLoadChildrenDetails` from `@moduix/react` when building typed local wrappers around the
recursive node renderer.

Mutation and virtualized rendering stay regular Ark collection patterns: replace the collection after
editing node data, or render the visible recursive nodes inside `TreeView.Tree` without adding a
moduix-specific prop.

## Examples

### Async Loading

Use `loadChildren` for branches whose children are not present yet, then replace local collection
state with the collection from `onLoadChildrenComplete`.

<style>{TreeViewDocs.treeViewExampleCss}</style>

<ExampleFrame>
  <TreeViewDocs.AsyncTreeViewExample />
</ExampleFrame>

```tsx
const asyncResponse = {
  src: [
    { id: 'src/App.tsx', name: 'App.tsx' },
    { id: 'src/main.tsx', name: 'main.tsx' },
    { id: 'src/components', name: 'components', childrenCount: 2 },
  ],
  'src/components': [
    { id: 'src/components/Button.tsx', name: 'Button.tsx' },
    { id: 'src/components/Dialog.tsx', name: 'Dialog.tsx' },
  ],
};

createTreeViewCode({
  imports: `import { useState } from "react";
import { TreeView, type TreeViewLoadChildrenDetails, type TreeViewNodeProviderProps, createTreeCollection, useTreeViewNodeContext } from "@moduix/react";
import { File as FileIcon, Folder as FolderIcon, FolderOpen as FolderOpenIcon } from "lucide-react";`,
  extraSetup: `const asyncCollection = createTreeCollection<FileNode>({
  nodeToValue: (node) => node.id,
  nodeToString: (node) => node.name,
  rootNode: {
    id: "ROOT",
    name: "",
    children: [
      { id: "src", name: "src", childrenCount: 3 },
      { id: "package.json", name: "package.json" },
      { id: "README.md", name: "README.md" },
    ],
  },
});

const asyncResponse: Record<string, FileNode[]> = {
  src: [
    { id: "src/App.tsx", name: "App.tsx" },
    { id: "src/main.tsx", name: "main.tsx" },
    { id: "src/components", name: "components", childrenCount: 2 },
  ],
  "src/components": [
    { id: "src/components/Button.tsx", name: "Button.tsx" },
    { id: "src/components/Dialog.tsx", name: "Dialog.tsx" },
  ],
};

function loadChildren(details: TreeViewLoadChildrenDetails<FileNode>) {
  const value = details.valuePath.join("/");
  return new Promise<FileNode[]>((resolve) => {
    window.setTimeout(() => resolve(asyncResponse[value] ?? []), 450);
  });
}`,
  demo: `export function AsyncTreeDemo() {
  const [source, setSource] = useState(asyncCollection);

  return (
    <TreeView
      collection={source}
      loadChildren={loadChildren}
      onLoadChildrenComplete={(details) => setSource(details.collection)}
    >
      <TreeView.Label>Lazy folders</TreeView.Label>
      <TreeView.Tree>
        ${createTreeNodesCode({
          source: 'source',
        })}
      </TreeView.Tree>
    </TreeView>
  );
}`,
});
```

```tsx
const collection = createTreeCollection({
  nodeToValue: (node) => node.id,
  nodeToString: (node) => node.name,
  rootNode: {
    id: 'ROOT',
    name: '',
    children: [
      { id: 'src', name: 'src', children: [{ id: 'src/App.tsx', name: 'App.tsx' }] },
      { id: 'package.json', name: 'package.json' },
    ],
  },
});

createTreeViewCode({
  imports: `import { useMemo, useState } from "react";
${treeViewCoreImportsCode}`,
  demo: `export function FilteringTreeDemo() {
  const [query, setQuery] = useState("");
  const filtered = useMemo(() => {
    const value = query.trim().toLowerCase();
    if (!value) return collection;
    return collection.filter((node) => node.name.toLowerCase().includes(value));
  }, [query]);

  return (
    <>
      <input value={query} onChange={(event) => setQuery(event.currentTarget.value)} />
      <TreeView collection={filtered}>
        <TreeView.Label>Filtered files</TreeView.Label>
        <TreeView.Tree>
          ${createTreeNodesCode({
            source: 'filtered',
          })}
        </TreeView.Tree>
      </TreeView>
    </>
  );
}`,
});
```

<ExampleFrame>
  <TreeViewDocs.RenameTreeViewExample />
</ExampleFrame>

```tsx
const collection = createTreeCollection({
  nodeToValue: (node) => node.id,
  nodeToString: (node) => node.name,
  rootNode: {
    id: 'ROOT',
    name: '',
    children: [
      { id: 'src', name: 'src', children: [{ id: 'src/App.tsx', name: 'App.tsx' }] },
      { id: 'package.json', name: 'package.json' },
    ],
  },
});

createTreeViewCode({
  imports: `import { useState } from "react";
${treeViewCoreImportsCode}`,
  setup: treeViewRenameSetupCode,
  demo: `export function RenameTreeDemo() {
  const [source, setSource] = useState(collection);

  return (
    <TreeView
      collection={source}
      canRename={() => true}
      onRenameComplete={(details) => {
        setSource((current) => {
          const node = current.at(details.indexPath);
          if (!node) return current;
          return current.replace(details.indexPath, { ...node, name: details.label });
        });
      }}
    >
      <TreeView.Label>Rename with F2</TreeView.Label>
      <TreeView.Tree>
        ${createTreeNodesCode({
          source: 'source',
          component: 'RenamableFileTreeNode',
        })}
      </TreeView.Tree>
    </TreeView>
  );
}`,
});
```
