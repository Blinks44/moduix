# React Hook Form

Use `register` for parts that render a native input. Use `Controller` when an Ark component exposes
controlled values and detail callbacks. Ark compound controls render their own native form control
when they need one, so never add or register a separate hidden input.

```bash
npm install react-hook-form
```

## Mapping

| Control                                         | React Hook Form wiring                                                    |
| ----------------------------------------------- | ------------------------------------------------------------------------- |
| `Field.Input`, `Field.Textarea`, `Field.Select` | Spread `register(name, rules)` onto the native control.                   |
| `Select`                                        | Use `Controller`; map a scalar string to Ark's string-array `value`.      |
| `Combobox`                                      | Use `Controller`; map `value` and keep filtering in `onInputValueChange`. |
| `Checkbox`                                      | Use `Controller`; map `details.checked === true` to a boolean.            |
| Automatic native controls                       | Pass semantic root props such as `name` or `form`; moduix owns the input. |

Always provide `defaultValues` for controlled fields. React Hook Form uses them for initial state,
dirty checks, and `reset()`.

## Validation and reset

Use `rules` for small field-local checks. For shared, cross-field, or schema-defined validation,
add a resolver; the moduix component mapping stays the same. When React Hook Form owns visible
errors, add `noValidate` to the native form and derive `Field invalid` and `Field.ErrorText` from
`fieldState`. Omit `noValidate` when browser constraint validation should run first.

Call `reset()` to restore the same `defaultValues` that initialized the form. Calling only the DOM
form's `reset()` is not enough for controlled `Controller` fields: React Hook Form will otherwise
render its previous value back into the moduix control.

## Repeated fields

Use `useFieldArray` for a list the user can edit. It supplies a stable `id` for every row; use it
as the React key, never the current index. The index is only for the registered field path. Keep
the group's label in `Fieldset` when the list has a shared purpose. This is a fragment for the
component that owns `useForm`; render the rows and buttons inside its form.

```tsx
import { Button, Field } from '@moduix/react';
import { useFieldArray, useForm } from 'react-hook-form';

type EmailFormValues = {
  emails: { address: string }[];
};

const { control, register } = useForm<EmailFormValues>({
  defaultValues: { emails: [{ address: '' }] },
});
const { fields, append, remove } = useFieldArray({ control, name: 'emails' });

{
  fields.map((item, index) => (
    <Field key={item.id}>
      <Field.Label>Email {index + 1}</Field.Label>
      <Field.Input type="email" {...register(`emails.${index}.address` as const)} />
      <Button type="button" onClick={() => remove(index)}>
        Remove email
      </Button>
    </Field>
  ));
}

<Button type="button" onClick={() => append({ address: '' })}>
  Add email
</Button>;
```

## Errors returned by the server

After a request, use `setError` for errors that belong to a field and the `root` namespace for a
form-wide error. The existing `errors.name` mapping still drives `Field invalid` and
`Field.ErrorText`; render the root message once near the form heading or submit button.

```tsx
const {
  handleSubmit,
  setError,
  formState: { errors },
} = useForm<{ name: string }>({
  defaultValues: { name: '' },
});

const onSubmit = handleSubmit(async (values) => {
  const result = await createProject(values);

  if (result.nameError) {
    setError('name', {
      type: 'server',
      message: result.nameError,
    });
  }

  if (result.message) {
    setError('root.server', {
      type: 'server',
      message: result.message,
    });
  }
});

{
  errors.root?.server?.message ? <p role="alert">{errors.root.server.message}</p> : null;
}
```

## Behavioral notes

- Do not spread a `Controller` field object onto `Select` or `Combobox`; their callback and value
  shapes are not native input events.
- For single selection, pass `field.value ? [field.value] : []` and read
  `details.value[0] ?? ""`.
- Call `field.onBlur` from `Select.onInteractOutside` or `Combobox.Input.onBlur` so touched state
  and blur-based validation follow the user's interaction.
- Forward `field.ref` to `Select.Trigger` or `Combobox.Input`. This lets React Hook Form focus the
  first invalid field.
- `Select` renders its native select automatically for native semantics, autofill, and reset
  synchronization.
- Map `Checkbox` checked state to a boolean field value; its root renders the native input
  automatically.
- `name={field.name}` enables native `FormData` semantics but does not replace `Controller` as the
  source of the field value. Keep the controlled value and callback mapping shown above.
- For the other automatic controls, keep exceptional form configuration on their roots: `DateInput`
  uses `names` for distinct range keys, `Select` uses `nativeFormControl="input"` for virtualized
  collections, and `SignaturePad` uses `getFormValue` for custom serialization.
