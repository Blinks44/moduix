| --------------------------------- | ------------------------------------------------------------------ |
| `Pagination` / `Pagination.Root`  | Navigation root that owns page and page-size state.                |

| `Pagination.Items`                | Renders the standard computed page items and ellipses.             |
| `Pagination.Context`              | Reads root state and helpers through an Ark render prop.           |
| `Pagination.usePagination`        | Creates Ark pagination state outside the rendered root.            |
| `Pagination.usePaginationContext` | Reads Ark pagination state from a descendant.                      |
| `Pagination.Item`                 | Interactive page item. Receives `{ type: 'page', value }`.         |
| `Pagination.Ellipsis`             | Non-interactive collapsed-range marker with a required `index`.    |
| `Pagination.PrevTrigger`          | Moves to the previous page and disables itself at the start.       |
| `Pagination.NextTrigger`          | Moves to the next page and disables itself at the end.             |
| `Pagination.FirstTrigger`         | Moves to the first page and disables itself at the start.          |
| `Pagination.LastTrigger`          | Moves to the last page and disables itself at the end.             |

## Composition

### Recommended composition

`Pagination.Root` owns keyboard behavior, current page, page size, disabled trigger state, page range
calculation, and accessible labels. Use `count` for the total number of data items, not the number
of pages. Pair it with `pageSize` or `defaultPageSize` so Ark can calculate `totalPages`.

Use `Pagination.Items` for the standard page list. It renders the current Ark `pages` array with
`Pagination.Item` and `Pagination.Ellipsis`, preserving their selected state, labels, and handlers.

### When to go custom

Use `Pagination.Context` when page items or adjacent controls need custom rendering. When state
must be created outside the rendered pagination tree, use `Pagination.usePagination()` with
`Pagination.RootProvider`; use `Pagination.usePaginationContext()` from a descendant that needs the
same state. Set `type="link"` and provide `getPageUrl(details)` when pagination should render
navigational links.

### Default props

| Prop              | Default      | Notes                                                       |
| ----------------- | ------------ | ----------------------------------------------------------- |
| `aria-label`      | `Pagination` | Moduix root default for the navigation landmark.            |
| `defaultPage`     | `1`          | Ark initial active page when uncontrolled.                  |
| `defaultPageSize` | `10`         | Ark initial items per page when uncontrolled.               |
| `siblingCount`    | `1`          | Ark pages shown beside the active page.                     |
| `boundaryCount`   | `1`          | Ark pages shown at the beginning and end of the page range. |
| `type`            | `button`     | Use `link` with `getPageUrl(details)` for anchors.          |

### Behavioral notes

Use `defaultPage` and `defaultPageSize` for uncontrolled state. Use `page`, `pageSize`,
`onPageChange(details)`, and `onPageSizeChange(details)` for controlled state. Callback detail
objects are the Ark objects; read `details.page` or `details.pageSize` without remapping. Ark
preserves keyboard behavior, selected item semantics, disabled trigger state, `ids`, `asChild`,
translations, and link URL generation.

### Styling hooks

- `data-scope="pagination"` and `data-part="item"` on `Pagination.Item`
- `data-scope="pagination"` and `data-part="prev-trigger"` on `Pagination.PrevTrigger`
- `data-scope="pagination"` and `data-part="next-trigger"` on `Pagination.NextTrigger`
- `data-scope="pagination"` and edge trigger parts on first and last triggers
- `data-selected` on the active page item
- `data-disabled` on unavailable triggers
- `data-slot="pagination-root"`, `pagination-root-provider`, `pagination-item`,
  `pagination-ellipsis`, and trigger-specific slots for Moduix overrides
