## Basic

Use `ScrollArea` as the Ark root and compose the viewport, content, scrollbar, thumb, and corner
parts explicitly. The viewport keeps native wheel, touch, trackpad, and keyboard scrolling while the
scrollbar parts provide the styled controls. Moduix adds one small extra: `fade`, which applies a
top and bottom mask to the viewport using Ark overflow measurements.

## Anatomy

```text
ScrollArea
├─ ScrollArea.Viewport
│  └─ ScrollArea.Content
│     └─ children
├─ ScrollArea.Scrollbar
│  └─ ScrollArea.Thumb
└─ ScrollArea.Corner
```

| Part                             | Role                                                            |
| -------------------------------- | --------------------------------------------------------------- |
| `ScrollArea` / `ScrollArea.Root` | State owner and outer positioning box.                          |
| `ScrollArea.RootProvider`        | Root that receives a `ScrollArea.useScrollArea()` state object. |
| `ScrollArea.Viewport`            | Native scroll viewport and focus target.                        |
| `ScrollArea.Content`             | Inner content wrapper measured by Ark UI.                       |
| `ScrollArea.Scrollbar`           | Custom scrollbar track for one axis.                            |
| `ScrollArea.Thumb`               | Draggable thumb inside a scrollbar.                             |
| `ScrollArea.Corner`              | Bottom-end intersection filler when both axes can overflow.     |

## Composition

### Recommended composition

`ScrollArea` is the styled root. It does not render viewport, content, or scrollbar parts for you;
compose the Ark parts explicitly so each structural node remains configurable.

### When to go custom

Use `ScrollArea.useScrollArea()` with `ScrollArea.RootProvider` when controls outside the root need
to call the scroll area API. Advanced Ark context reads stay available directly from
`@ark-ui/react/scroll-area`. Do not render `ScrollArea` and `ScrollArea.RootProvider` for the same
state object.

### Default props

| Prop                                                    | Default    | Notes                                                                                  |
| ------------------------------------------------------- | ---------- | -------------------------------------------------------------------------------------- |
| `fade` on `ScrollArea` and `ScrollArea.RootProvider`    | `false`    | Adds a top and bottom viewport mask driven by Ark `--scroll-area-overflow-y-*` values. |
| `variant` on `ScrollArea` and `ScrollArea.RootProvider` | `hover`    | Use `always` to keep overflowing scrollbar tracks visible and interactive.             |
| `orientation` on `ScrollArea.Scrollbar`                 | `vertical` | Set `orientation="horizontal"` for a horizontal track.                                 |
| `asChild`                                               | `false`    | Available on each Ark part for single-child host composition.                          |
| `ids` on `ScrollArea`                                   | generated  | Use when separate code needs stable root, viewport, content, scrollbar, or thumb IDs.  |

### Behavioral notes

- Render one `ScrollArea.Scrollbar` per axis you want to expose.
- Ark adds `data-overflow-x` and `data-overflow-y`; moduix hides scrollbar tracks until the matching
  axis actually overflows.
- Set `variant="always"` when the matching overflowing scrollbar should remain visible and
  interactive at rest.
- The viewport CSS hides native browser scrollbars with `scrollbar-width: none` and the WebKit
  scrollbar rule required by Ark UI.
- The ref on `ScrollArea` points to the root element; the ref on `ScrollArea.Viewport` points to the
  native scroll container.

## Styling

Use CSS variables on the root when scrollbar thickness, focus ring treatment, or surface styling
needs to change without replacing the structure. The default thumb keeps the border token color and
grows by `2px` across its track on hover or drag; customize the amount with
`--moduix-scroll-area-thumb-hover-increase` and the timing with
`--moduix-scroll-area-thumb-hover-transition`.

### Styling hooks

- `data-slot="scroll-area-root"`, `scroll-area-root-provider`, `scroll-area-viewport`,
  `scroll-area-content`, `scroll-area-scrollbar`, `scroll-area-thumb`, and `scroll-area-corner` are
  the stable Moduix hooks.
- `data-fade` is added by the `fade` prop and `data-variant="hover" | "always"` by the `variant`
  prop on `ScrollArea` and `ScrollArea.RootProvider`.
- Ark adds `data-scope="scroll-area"` and `data-part` for every part.
- `data-overflow-x` and `data-overflow-y` are present when the measured content overflows on the
  corresponding axis.
- `data-at-top`, `data-at-bottom`, `data-at-left`, and `data-at-right` are available on the viewport.
- `data-orientation="vertical"` and `data-orientation="horizontal"` are exposed on scrollbars and
  thumbs.
- `data-hover` and `data-dragging` are available on scrollbar and thumb parts; `data-scrolling` is
  available on scrollbar parts.
- Ark root variables `--corner-width`, `--corner-height`, `--thumb-width`, and `--thumb-height` drive
  the corner offset and thumb dimensions. Ark viewport variables
  `--scroll-area-overflow-x-start`, `--scroll-area-overflow-x-end`,
  `--scroll-area-overflow-y-start`, and `--scroll-area-overflow-y-end` expose edge distances.
