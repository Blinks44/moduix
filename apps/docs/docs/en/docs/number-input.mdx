## Composition

### Recommended composition

`NumberInput.Root` owns parsing, range checks, keyboard increment/decrement, spin-on-press, mouse
wheel changes, scrubber interaction, and formatting. Use `NumberInput` as the short root form and
use `NumberInput.Field` for the default visible parts.

Use string values for `value` and `defaultValue`. Ark reports changes through
`onValueChange(details)`, where `details.value` is the string value and `details.valueAsNumber` is
the parsed number.

### When to go custom

Use `NumberInput.Control` with its individual parts when the trigger order, icons, or input styling
must differ from the default `NumberInput.Field`. Use `NumberInput.Scrubber` alongside the field for
drag-to-change input.

### Default props

| Prop                               | Default | Notes                                                        |
| ---------------------------------- | ------- | ------------------------------------------------------------ |
| `NumberInput` / `NumberInput.Root` | ŌĆö       | No additional behavioral defaults; Ark defaults apply.       |
| `NumberInput.Field`                | ŌĆö       | Renders the default control, input, and trigger group.       |
| `NumberInput.DecrementTrigger`     | ŌĆö       | Renders the moduix minus icon when no children are provided. |
| `NumberInput.IncrementTrigger`     | ŌĆö       | Renders the moduix plus icon when no children are provided.  |

### Behavioral notes

- Use `formatOptions`, not `format`, for localized number formatting.
- Use `translations.incrementLabel`, `translations.decrementLabel`, and `translations.valueText` for accessible localized labels.
- `min`, `max`, `step`, `allowOverflow`, `clampValueOnBlur`, `spinOnPress`, `inputMode`, and `allowMouseWheel` pass directly to Ark.
- `onValueInvalid(details)` reports `rangeUnderflow` or `rangeOverflow` when the value exceeds configured bounds.
- Use `ids` when external labels or descriptions need stable element IDs.
- Ark-backed parts support `asChild` for custom host composition; `NumberInput.Field` intentionally
  renders its fixed three-part structure and forwards its ref to `NumberInput.Control`.
- `useNumberInput`, `useNumberInputContext`, and `NumberInput.Context` are available from `@moduix/react`.
