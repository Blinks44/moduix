## Choosing The Right Component

- Choose **`Input`** for plain date strings without calendar behavior.
- Choose **`DateInput`** for keyboard-first segmented date entry.
- Choose **`DatePicker`** when users need an editable input plus a calendar popup or inline calendar.

## Basic

`DatePicker` keeps Ark's date state and calendar behavior while adding small convenience parts for
the common path. Use `DatePicker.Field` for the input actions and `DatePicker.DayTable` for the
standard day grid.

│     └─ DatePicker.Content

│        │  │  ├─ DatePicker.PrevTrigger
│        │  │  ├─ DatePicker.ViewTrigger / DatePicker.RangeText
│        │  │  └─ DatePicker.NextTrigger
│        │  ├─ DatePicker.DayTable (convenience day grid)
│        │  └─ DatePicker.Table
│        │     ├─ DatePicker.TableHead / TableBody / TableRow
│        │     ├─ DatePicker.TableHeader
│        │     ├─ DatePicker.TableCell
│        │     └─ DatePicker.TableCellTrigger

│        └─ DatePicker.PresetTrigger

| -------------------------------- | ---------------------------------------------------------------- |
| `DatePicker` / `DatePicker.Root` | Root value, view, open state, locale, validation, form context.  |
| `DatePicker.RootProvider`        | Renders parts from state returned by `useDatePicker()`.          |
| `DatePicker.Label`               | Accessible label.                                                |
| `DatePicker.Control`             | Input field wrapper that receives Ark state attributes.          |
| `DatePicker.Field`               | Convenience control with input, clear trigger, and open trigger. |
| `DatePicker.RangeField`          | Convenience range control with indexed start and end inputs.     |
| `DatePicker.Input`               | Editable date text and native form input.                        |
| `DatePicker.Trigger`             | Opens the calendar, defaults to a calendar icon.                 |
| `DatePicker.ClearTrigger`        | Clears the current value, defaults to a close icon.              |
| `DatePicker.Positioner`          | Floating layer using Ark positioning CSS variables.              |
| `DatePicker.Content`             | Calendar surface for popup or inline mode.                       |
| `DatePicker.View`                | Day, month, or year panel.                                       |
| `DatePicker.DayTable`            | Standard day-view table rendered from Ark context.               |
| `DatePicker.Table*`              | Low-level calendar table composition for custom layouts.         |
| `DatePicker.MonthSelect`         | Native month select connected to focused date.                   |
| `DatePicker.YearSelect`          | Native year select connected to focused date.                    |
| `DatePicker.PresetTrigger`       | Preset range button.                                             |

## Composition

### Recommended composition

Use `DatePicker.Field` for the standard single-date field, `DatePicker.RangeField` for range
fields, and `DatePicker.DayTable` inside the day view. The root portals `Positioner`
automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container;
structural popup parts remain explicit and styleable.

### When to go custom

Use `DatePicker.Control`, `DatePicker.Input`, `DatePicker.Trigger`, `DatePicker.ClearTrigger`, and
the low-level `Table*` parts directly when the field or calendar grid needs custom structure. Import
`DatePicker.Context` from `@ark-ui/react/date-picker` when custom rendering needs Ark state such as
`weekDays`, `weeks`, `getMonthsGrid()`, `getYearsGrid()`, `getOffset()`, or action helpers.

### Default props

| Prop                                     | Default         | Notes                                              |
| ---------------------------------------- | --------------- | -------------------------------------------------- |
| `DatePicker.Field.placeholder`           | `Select date`   | Passed to the generated single `DatePicker.Input`. |
| `DatePicker.Field.clearLabel`            | `Clear date`    | Used as the generated clear trigger label.         |
| `DatePicker.Field.triggerLabel`          | `Open calendar` | Used as the generated trigger label.               |
| `DatePicker.RangeField.startPlaceholder` | `Start date`    | Passed to the generated start input.               |
| `DatePicker.RangeField.endPlaceholder`   | `End date`      | Passed to the generated end input.                 |
| `DatePicker.DayTable.showHeader`         | `true`          | Renders prev/view/next controls before the table.  |
| `DatePicker.DayTable.showWeekNumbers`    | `false`         | Renders week-number header and cells.              |

### Behavioral notes

`value`, `defaultValue`, and `onValueChange(details)` use Ark `DateValue[]`. Import `parseDate`,
`DatePicker.Context`, `useDatePicker`, and event/detail types directly from
`@ark-ui/react/date-picker` when you need them. The default root and popup width is `18.75rem`
(300px); override `--date-picker-width` or `--date-picker-content-width` when a wider composition
is needed.
Because popup content is portalled, put popup sizing variables on `DatePicker.Content` or another
element inside the overlay subtree, not only on `DatePicker.Root`.

`open`, `defaultOpen`, and `onOpenChange(details)` control popup state. `view`, `defaultView`,
`minView`, `maxView`, and `onViewChange(details)` control the active calendar panel.

Use `selectionMode="single"`, `"range"`, or `"multiple"`. `maxSelectedDates` applies only to
multiple selection. `numOfMonths`, `fixedWeeks`, and `showWeekNumbers` change the calendar table.

Use `min`, `max`, and `isDateUnavailable(date, locale)` for unavailable dates. Use `locale`,
`timeZone`, `format`, `parse`, `translations`, and `createCalendar` for localization.

### Month And Year Select

Ark ships native `DatePicker.MonthSelect` and `DatePicker.YearSelect` parts. This example uses
moduix `Select` with `DatePicker.Context` instead, so the header matches the rest of the library
while still updating Ark's focused date.

<style>{DatePickerDocs.datePickerExampleCss}</style>

<ExampleFrame clientOnly>
  <DatePickerDocs.MonthYearSelectDatePickerExample />
</ExampleFrame>

```tsx
import { createListCollection } from '@ark-ui/react/collection';
import { parseDate } from '@ark-ui/react/date-picker';
import { DatePicker } from '@moduix/react';

const monthItems = [
  {
    label: 'January',
    value: '1',
  },
  {
    label: 'February',
    value: '2',
  },
  {
    label: 'March',
    value: '3',
  },
  {
    label: 'April',
    value: '4',
  },
  {
    label: 'May',
    value: '5',
  },
  {
    label: 'June',
    value: '6',
  },
  {
    label: 'July',
    value: '7',
  },
  {
    label: 'August',
    value: '8',
  },
  {
    label: 'September',
    value: '9',
  },
  {
    label: 'October',
    value: '10',
  },
  {
    label: 'November',
    value: '11',
  },
  {
    label: 'December',
    value: '12',
  },
];
const months = createListCollection({
  items: monthItems,
});
export default function MonthYearSelectDatePickerDemo() {
  return (
    <DatePicker defaultValue={[parseDate('2026-06-22')]}>
      <DatePicker.Label>Report date</DatePicker.Label>
      <DatePicker.Field />
      <DatePicker.Positioner>
        <DatePicker.Content>
          <DatePicker.ViewControl className="date-picker-month-year-control">
            <DatePicker.Context>
              {(datePicker) => {
                const focusedYear = datePicker.focusedValue.year;
                const years = createListCollection({
                  items: Array.from(
                    {
                      length: 12,
                    },
                    (_, index) => {
                      const year = focusedYear - 5 + index;
                      return {
                        label: String(year),
                        value: String(year),
                      };
                    },
                  ),
                });
                return (
                  <div className="date-picker-month-year-selects">
                    <Select
                      className="date-picker-month-select"
                      collection={months}
                      value={[String(datePicker.focusedValue.month)]}
                      onValueChange={(details) => {
                        const month = Number(details.value[0]);
                        if (month) {
                          datePicker.setFocusedValue(
                            datePicker.focusedValue.set({
                              month,
                            }),
                          );
                        }
                      }}
                    >
                      <Select.Control>
                        <Select.Trigger>
                          <Select.ValueText />
                        </Select.Trigger>
                        <Select.Indicators>
                          <Select.Indicator />
                        </Select.Indicators>
                      </Select.Control>
                      <Select.Positioner>
                        <Select.Content>
                          <Select.List>
                            {monthItems.map((item) => (
                              <Select.Item key={item.value} item={item}>
                                <Select.ItemText>{item.label}</Select.ItemText>
                                <Select.ItemIndicator />
                              </Select.Item>
                            ))}
                          </Select.List>
                        </Select.Content>
                      </Select.Positioner>
                    </Select>
                    <Select
                      className="date-picker-year-select"
                      collection={years}
                      value={[String(datePicker.focusedValue.year)]}
                      onValueChange={(details) => {
                        const year = Number(details.value[0]);
                        if (year) {
                          datePicker.setFocusedValue(
                            datePicker.focusedValue.set({
                              year,
                            }),
                          );
                        }
                      }}
                    >
                      <Select.Control>
                        <Select.Trigger>
                          <Select.ValueText />
                        </Select.Trigger>
                        <Select.Indicators>
                          <Select.Indicator />
                        </Select.Indicators>
                      </Select.Control>
                      <Select.Positioner>
                        <Select.Content>
                          <Select.List>
                            {years.items.map((item) => (
                              <Select.Item key={item.value} item={item}>
                                <Select.ItemText>{item.label}</Select.ItemText>
                                <Select.ItemIndicator />
                              </Select.Item>
                            ))}
                          </Select.List>
                        </Select.Content>
                      </Select.Positioner>
                    </Select>
                  </div>
                );
              }}
            </DatePicker.Context>
            <div className="date-picker-month-year-nav">
              <DatePicker.PrevTrigger />
              <DatePicker.NextTrigger />
            </div>
          </DatePicker.ViewControl>
          <DatePicker.View view="day">
            <DatePicker.DayTable showHeader={false} />
          </DatePicker.View>
        </DatePicker.Content>
      </DatePicker.Positioner>
    </DatePicker>
  );
}
```

### Styling hooks

- `data-slot="date-picker-root"` on the root.
- `data-slot="date-picker-root-provider"` on the provider root.
- `data-slot="date-picker-label"` on the label.
- `data-slot="date-picker-control"` on the input wrapper.
- `data-slot="date-picker-control"` on `DatePicker.Field` and `DatePicker.RangeField`, because
  both render `DatePicker.Control`.
- `data-slot="date-picker-input"` on the editable input.
- `data-slot="date-picker-trigger"` and `date-picker-clear-trigger` on input actions.
- `data-slot="date-picker-positioner"` and `date-picker-content` on the floating layer.
- `data-slot="date-picker-view"`, `view-control`, `view-trigger`, `prev-trigger`, and
  `next-trigger` on calendar navigation.
- `data-slot="date-picker-table*"` on table parts and `date-picker-table-cell-trigger` on cells.
- `data-slot="date-picker-week-number-*"` on week-number cells.
- `data-slot="date-picker-month-select"` and `date-picker-year-select` on native selects.
- `data-slot="date-picker-preset-trigger"` on range preset buttons.

Ark state attributes include `data-state`, `data-disabled`, `data-readonly`, `data-invalid`,
`data-focus`, `data-selected`, `data-today`, `data-unavailable`, `data-outside-range`,
`data-in-range`, `data-range-start`, `data-range-end`, `data-in-hover-range`,
`data-hover-range-start`, `data-hover-range-end`, and `data-view`.
