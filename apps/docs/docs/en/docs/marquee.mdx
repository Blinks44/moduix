## Basic

Use `Marquee` as the Ark root, then compose `Viewport`, `Content`, and `Item` explicitly. Provide
a descriptive label with `aria-label` or `translations.root`, especially when the content is
informational. The root is visually neutral by default, so the basic example only sets width.

## Anatomy

```text
Marquee / Marquee.Root
├─ Marquee.Edge
├─ Marquee.Viewport
│  └─ Marquee.Content
│     └─ Marquee.Item
└─ Marquee.Edge

Marquee.RootProvider
└─ same part tree connected to moduix useMarquee()
```

| Part                       | Role                                                                  |
| -------------------------- | --------------------------------------------------------------------- |
| `Marquee` / `Marquee.Root` | Root state, labels, ids, orientation, pause state, and CSS variables. |
| `Marquee.RootProvider`     | Renders a marquee tree from state returned by `useMarquee`.           |
| `Marquee.Viewport`         | Clipping viewport for the scrolling content.                          |
| `Marquee.Content`          | Animated content wrapper. Ark clones it when needed.                  |
| `Marquee.Item`             | Individual repeated item inside the content.                          |
| `Marquee.Edge`             | Optional fade overlay. Pass `side` to choose the edge.                |

## Composition

### Recommended composition

Use the short root form, `Marquee`, and keep the Ark part tree explicit. The library supplies the
animation keyframes and state hooks, while item visuals and root framing stay in consumer CSS.

### When to go custom

Use `Marquee.Edge` when the content should fade into the page background. Use moduix `useMarquee()` with
`Marquee.RootProvider` when controls outside the marquee need the API: `pause()`, `resume()`,
`togglePause()`, and `restart()`. Do not render `Marquee` and `Marquee.RootProvider` for the same
state object.

Use `asChild` on Ark parts when replacing the host element. Keep the replacement child semantic and
single. Use `ids` when a custom tree needs stable root, viewport, or content ids.

### Props and callbacks

| Prop                 | Default | Notes                                                            |
| -------------------- | ------- | ---------------------------------------------------------------- |
| `side`               | `start` | Use `start`, `end`, `top`, or `bottom`. Top/bottom are vertical. |
| `speed`              | `50`    | Pixels per second. Ark calculates duration from content size.    |
| `spacing`            | `1rem`  | Gap between content instances and items.                         |
| `delay`              | `0`     | Delay before animation starts, in seconds.                       |
| `loopCount`          | `0`     | `0` means infinite loops.                                        |
| `autoFill`           | `false` | Duplicates content until the viewport is filled.                 |
| `pauseOnInteraction` | `false` | Pauses on hover and focus. Recommended for readable content.     |
| `reverse`            | `false` | Reverses animation direction without changing `side`.            |
| `defaultPaused`      | `false` | Initial uncontrolled pause state.                                |
| `paused`             | -       | Controlled pause state. Pair with `onPauseChange`.               |
| `onPauseChange`      | -       | Receives Ark pause status details.                               |
| `onLoopComplete`     | -       | Fires after each loop iteration.                                 |
| `onComplete`         | -       | Fires after the final loop when `loopCount` is finite.           |
| `translations`       | -       | Localized root label. Use this or `aria-label`.                  |
| `ids`                | -       | Stable ids for root, viewport, and content instances.            |
| `asChild`            | -       | Preserved on every Ark part for host element replacement.        |
| `Marquee.Edge side`  | -       | Required edge side: `start`, `end`, `top`, or `bottom`.          |
| `RootProvider.value` | -       | Required `UseMarqueeReturn` from `useMarquee()`.                 |

### Behavioral notes

Use `paused` with `onPauseChange(details)` for controlled playback. Use `loopCount`,
`onLoopComplete`, and `onComplete` when the marquee should stop after a finite number of loops.
The callback signatures are Ark's original signatures.

Ark renders marquee semantics as a labelled region and hides cloned content from assistive
technology. Pass a useful `aria-label` or `translations.root`. Prefer `pauseOnInteraction` when
content is interactive or needs time to read, and avoid marquees for critical information.

There is no roving focus or arrow-key navigation. When `pauseOnInteraction` is enabled, focus pauses
the marquee and blur resumes it.

## Styling

Marquee needs the bundled content keyframes to move. Override layout and visual treatment through
CSS variables and part classes; keep Ark state attributes for behavioral styling.

### Styling hooks

- `data-slot="marquee-root"`, `marquee-root-provider`, `marquee-viewport`, `marquee-content`,
  `marquee-item`, and `marquee-edge`.
- Ark hooks: `data-scope="marquee"`, `data-part="root|viewport|content|item|edge"`,
  `data-state="paused|idle"`, `data-orientation`, `data-side`, `data-index`, `data-paused`,
  `data-reverse`, and `data-clone`.
- Ark CSS variables: `--marquee-duration`, `--marquee-spacing`, `--marquee-delay`,
  `--marquee-loop-count`, and `--marquee-translate`.
