## API Reference

Ark UI does not ship a dedicated typesetting primitive. Moduix builds `Typeset` with the
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), so the root keeps
standard HTML props, `asChild`, and a forwarded ref while its CSS styles semantic descendants.

## Basic

Use the short `<Typeset>` root for rendered HTML. Add `asChild` when the container itself should
be semantic, such as an `article` or `section`. Use plain `<ul>` and `<ol>` elements: Typeset adds
the bullets, numbering, nested markers, and reading rhythm without utility classes.

<style>{`
      .article {
        width: min(42rem, 100%);
      }
    `}</style>

## Composition

### Recommended composition

Keep Markdown parsing, sanitization, rendering, and reading measure in the application. `Typeset`
only supplies the scoped visual rhythm and deliberately does not set a reading width. Use
`Typeset.Scroll` around a wide table or another block that should scroll horizontally.

### Behavioral notes

`Typeset` has no state machine, controlled state, keyboard behavior, form integration, or hidden
inputs. Its styles use `margin-block-start` and avoid `:last-child`, `:has()`, and `:empty`, so
appending a streaming Markdown block does not restyle earlier blocks.

## Examples

### Reading rhythm

Use CSS variables to create roomier or denser presets without adding component props.

<style>{`
      .typeset-reading {
        --moduix-typeset-size: 1.125rem;
        --moduix-typeset-leading: 1.9;
        --moduix-typeset-flow: 1.75em;
      }
    `}</style>

### Streaming content

Use a tighter rhythm for chat messages. New blocks contribute only their own spacing as the stream
grows.

<style>{`
      .typeset-chat {
        --moduix-typeset-leading: 1.6;
        --moduix-typeset-flow: 0.875em;
      }
    `}</style>

### Scrollable table

Wrap a table generated by the renderer in `Typeset.Scroll` when it should scroll instead of
squeezing into a narrow container.

<style>{`
      .wide-table {
        min-width: 36rem;
      }
    `}</style>

### Excluding application surfaces

Add `.not-typeset` or `data-not-typeset` to a nested subtree with its own visual contract. Both
exclude the marked element and everything inside it, including a nested `Typeset` root.

<style>{`
      .app-surface {
        padding: var(--moduix-spacing-4);
        border: var(--moduix-border-width-sm) solid var(--moduix-color-border);
        border-radius: var(--moduix-radius-md);
        background: var(--moduix-color-muted);
      }
    `}</style>

The three rhythm controls are `--moduix-typeset-size`, `--moduix-typeset-leading`, and `--moduix-typeset-flow`. Font
variables are optional; colors, borders, and radius come straight from the active moduix theme.
Element rules live in the `moduix.components` layer and use low-specificity `:where()` selectors, so
normal application CSS can override individual elements without `!important`.

### Styling hooks

- `data-slot="typeset"` targets the root container.
- `data-slot="typeset-scroll"` targets the optional overflow wrapper.
- `[data-scope="typeset"][data-part="root"]` and `[data-part="scroll"]` expose stable anatomy
  hooks.
