## Anatomy

```text
Splitter.Root
├─ Splitter.Panel[id]
├─ Splitter.ResizeTrigger[id="panel-a:panel-b"]
│  └─ Splitter.ResizeTriggerIndicator (default, customizable)
└─ Splitter.Panel[id]
```

| Part                              | Role                                                                                   |
| --------------------------------- | -------------------------------------------------------------------------------------- |
| `Splitter` / `Splitter.Root`      | Owns panel constraints, sizes, orientation, callbacks, and the splitter state machine. |
| `Splitter.Panel`                  | Resizable region. Its `id` must match one item in the `panels` prop.                   |
| `Splitter.ResizeTrigger`          | Accessible handle button between adjacent panels.                                      |
| `Splitter.ResizeTriggerIndicator` | Default visual indicator; pass custom trigger children to replace it.                  |
| `Splitter.RootProvider`           | Renders the same part tree from state created with `useSplitter()`.                    |

## Composition

### Recommended composition

Render panels and handles explicitly in the same order they appear visually. Use trigger ids in the
`"left:right"` form so Ark can connect the two adjacent panel ids. Omit trigger children for the
default resize indicator; pass children only when its content needs customization.

### When to go custom

Use controlled `size` with `onResize(details)` when panel sizes must be mirrored into app state.
Import `useSplitter`, `useSplitterContext`, and `createSplitterRegistry` from `@moduix/react` for
imperative controls, externally-owned state, and nested multi-drag layouts. Pair `useSplitter()`
with `Splitter.RootProvider` when state must be created outside the rendered part tree.

Use `asChild` only when another element must own the rendered node. Custom resize triggers must keep
button semantics, focus behavior, and keyboard interaction intact.

### Default props

| Prop               | Default        | Notes                                                     |
| ------------------ | -------------- | --------------------------------------------------------- |
| `orientation`      | `"horizontal"` | Use `"vertical"` for stacked panels.                      |
| `defaultSize`      | Ark-calculated | Use percentages for stable server-rendered sizing.        |
| `keyboardResizeBy` | Ark default    | Controls arrow-key resize distance.                       |
| `size`             | uncontrolled   | Pass with `onResize(details)` for controlled panel sizes. |
| `disabled`         | `false`        | Set on `ResizeTrigger` to disable a handle.               |

### Behavioral notes

`panels` is required and defines constraints such as `minSize`, `maxSize`, `collapsible`, and
`collapsedSize`. Numeric size values are percentages; Ark also accepts CSS lengths for initial and
constraint sizes, which are measured on the client and can shift after hydration.

Zag 1.41.2 currently renders a collapsed panel at `minSize` when `collapsedSize` is smaller. The
upstream [issue](https://github.com/chakra-ui/zag/issues/3179) and
[fix](https://github.com/chakra-ui/zag/pull/3180) cover this regression; moduix does not add a local
layout override while the fix is pending.

Callbacks keep Ark detail objects: `onResize(details)`, `onResizeEnd(details)`, `onCollapse(details)`,
and `onExpand(details)`. `onResizeStart` is called when a resize begins.

## Styling

Style `Splitter` through `className`, Ark attributes, stable `data-slot="splitter-*"` hooks, and the
public Moduix CSS variables below. `Splitter.Root` and `Splitter.RootProvider` apply
`--splitter-width` and `--splitter-height` through their default inline style so those variables can
override Ark's root layout style. The default surface lives on the root, while panels stay flat and
flush inside it.

The resize trigger contributes a `1px` separator to layout with a visually thinner `0.5px` line, a
transparent hit area, and a raised rounded indicator on top. The indicator is centered over the
separator for both orientations. Hover strengthens its border and shadow, while dragging adds a
slight scale and stronger shadow. Releasing the pointer returns the trigger to its idle appearance.
Override the corresponding `--splitter-resize-trigger-*` variables when the surrounding theme needs
a different treatment.

### Styling hooks

Ark writes `data-scope="splitter"` and `data-part` to rendered parts. Moduix also writes
`data-slot` values: `splitter-root`, `splitter-root-provider`, `splitter-panel`,
`splitter-resize-trigger`, and `splitter-resize-trigger-indicator`.

Important Ark state hooks include `data-orientation`, `data-dragging`, trigger `data-focus`, trigger
`data-disabled`, panel `data-id`, and panel `data-index`.
