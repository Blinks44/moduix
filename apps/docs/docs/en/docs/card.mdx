## API Reference

Ark UI does not ship a dedicated card primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Basic

Use the callable `<Card />` root with namespaced parts for the standard card composition.

If you are moving from shadcn, the mental model stays the same. The main API change is
`CardContent` becoming `Card.Body`, while the other parts stay namespaced under `Card`.

## Anatomy

```text
Card.Root
├─ Card.Media (optional moduix sugar)
├─ Card.Header
│  ├─ Card.Title
│  ├─ Card.Description
│  └─ Card.Action (optional moduix sugar)
├─ Card.Body
└─ Card.Footer (optional)
```

`Card` and `Card.Root` are the same root component.

| Part                 | Role                                                                                    |
| -------------------- | --------------------------------------------------------------------------------------- |
| `Card` / `Card.Root` | Root surface with `size`, `variant`, and `asChild`.                                     |
| `Card.Media`         | Optional edge-to-edge media wrapper with predictable top-radius clipping.               |
| `Card.Header`        | Header layout for title, description, and an optional trailing action.                  |
| `Card.Body`          | Main content area.                                                                      |
| `Card.Footer`        | Wrapping action or metadata row.                                                        |
| `Card.Title`         | Heading part. Defaults to `h3`; use `asChild` for another heading level.                |
| `Card.Description`   | Supporting text.                                                                        |
| `Card.Action`        | moduix header-side layout slot.                                                         |
| `Card.Link`          | moduix stretched overlay link for a navigable card that also contains separate actions. |

## Composition

### Recommended composition

Use `<Card />`, `Card.Header`, `Card.Title`, `Card.Description`, `Card.Body`, and `Card.Footer`.
Wrap leading media in `Card.Media` when it should bleed to the card edges without custom clipping.
The root defaults to `size="md"` and `variant="outline"`.

### When to go custom

Use `asChild` when one semantic child should own a part's DOM node. Ark composition requires exactly
one child. A linked root must not contain nested buttons, inputs, or links.

Use `Card.Link` inside the title when the card needs an overlay navigation target and also contains
independent controls. `Card.Link` is the stretched-link pattern for cards: it keeps the visible link
in the title, then extends the hit area across the card while separate controls stay interactive.
`Card.Action` only provides layout and does not create ARIA relationships.

| Situation                                                                                 | Recommended API                                                |
| ----------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| The whole card is one link and there are no nested controls.                              | `Card asChild` with an `<a>` root                              |
| The card should navigate, but it also contains buttons, menus, or other separate actions. | `Card.Link` inside `Card.Title`, optionally with `Card.Action` |
| The card is only a grouped surface and does not navigate.                                 | Plain `Card` composition without link sugar                    |

### Default props

| Part        | Prop      | Default     | Values                                |
| ----------- | --------- | ----------- | ------------------------------------- |
| `Card.Root` | `size`    | `'md'`      | `'sm' \| 'md' \| 'lg'`                |
| `Card.Root` | `variant` | `'outline'` | `'elevated' \| 'outline' \| 'subtle'` |

### Behavioral notes

- The root renders a presentational `div` by default and forwards its ref to the rendered element.
- All parts accept native props, `className`, `style`, and Ark factory `asChild`.
- `Card.Media` is optional sugar. Omit it when you need fully custom media framing or non-leading placement.
- `Card.Title` renders `h3` by default.
- Card has no managed state, callbacks, keyboard behavior, provider, context, or hidden input.
- `Card.Body` resets first and last direct-child margins.
- `--moduix-card-spacing`, `--moduix-card-spacing-sm`, and `--moduix-card-spacing-lg` control shared inset and section spacing without replacing the narrower `--moduix-card-*` overrides.

        height: 100%;

### Styling hooks

- Every part exposes `data-scope="card"` and a matching `data-part`.
- Stable moduix hooks are `data-slot="card-root"`, `card-media`, `card-header`, `card-body`,
  `card-footer`, `card-title`, `card-description`, `card-action`, and `card-link`.
- The root exposes `data-size="sm | md | lg"` and
  `data-variant="elevated | outline | subtle"`.
- Card has no interactive state attributes because it does not own state or behavior.
