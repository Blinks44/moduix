## API Reference

Ark UI does not ship a dedicated `container` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance

## Basic

Use `Container` when the main concern is readable inline width. It centers a content column,
applies responsive inline gutters, and stays out of vertical spacing decisions.

<style>{`
      .container {
        padding-block: var(--spacing-4);
        border-block: var(--border-width-sm) dashed var(--color-border);
        background-color: var(--color-muted);
      }

      .container > * {
        max-width: 42rem;
      }
    `}</style>

## Anatomy

`Container` is a single root layout primitive built on Ark factory composition.

```text
Container / Container.Root
└─ children
```

| Part                           | Role                                                     |
| ------------------------------ | -------------------------------------------------------- |
| `Container` / `Container.Root` | Root layout wrapper. Controls content width and gutters. |

## Composition

### Recommended composition

Use `Container` for page shells, article bodies, docs sections, settings screens, and other layouts
where the main requirement is a centered content column with predictable gutters.

`Container.Root` is the same component exposed as an Ark-aligned namespace alias.

Use `size` to choose the content width and `gutter` to control the edge padding. Use `asChild` when
the wrapper itself should also carry semantic meaning such as `main`, `section`, or `article`.

### When to go custom

`Container` is intentionally root-only. There are no extra subparts, slot props, or nested
layout helpers.

The custom path here is small:

- pass `className` for local layout or surface styling;
- override `--container-*` variables when a page needs different width or gutter math;
- use `asChild` when another element should own the DOM node;
- compose with `Bleed` when a child should escape the constrained text column.

### Default props

| Prop      | Default | Values / Notes                       |
| --------- | ------- | ------------------------------------ |
| `size`    | `lg`    | `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `gutter`  | `md`    | `none`, `sm`, `md`, `lg`             |
| `asChild` | `false` | Ark factory composition              |

### Behavioral notes

- `Container` does not manage vertical rhythm. Add section spacing outside it or on its children.
- `size="full"` removes only the max-width cap. The selected gutter still applies unless
  `gutter="none"` is used.
- The root exposes `data-scope="container"`, `data-part="root"`, `data-slot="container-root"`,
  `data-size`, and `data-gutter` for styling and tests.

## Styling

Use CSS variables on the root when a page shell needs different gutter math or a different width
scale. Use `className` for local block styling such as padding, borders, or backgrounds.

### CSS Properties

<CssPropertiesSection properties={ContainerDocs.containerOverrideCssProperties} />

### Styling hooks

- `Container` and `Container.Root` expose `data-slot="container-root"` on the root.
- `data-scope="container"` and `data-part="root"` provide Ark-aligned root hooks.
- `data-size` selects the width preset: `xs`, `sm`, `md`, `lg`, `xl`, `full`.
- `data-gutter` selects the gutter preset: `none`, `sm`, `md`, `lg`.
- `--container-*` variables control only inline gutter and max-width math; vertical spacing and
  child layout remain the responsibility of surrounding layout CSS.
