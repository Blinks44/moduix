## Anatomy

```text
FileUpload / FileUpload.Root
├─ FileUpload.Label
├─ FileUpload.Dropzone
│  ├─ FileUpload.DropzoneIcon
│  └─ FileUpload.Trigger
├─ FileUpload.ItemGroup[type]
│  ├─ FileUpload.Items
│  └─ FileUpload.Context
│     └─ FileUpload.Item[file]
│        ├─ FileUpload.ItemPreview[type]
│        │  ├─ FileUpload.ItemPreviewImage
│        │  └─ FileUpload.ItemPreviewIcon
│        ├─ FileUpload.ItemName
│        ├─ FileUpload.ItemMetadata[file]
│        ├─ FileUpload.ItemSizeText
│        └─ FileUpload.ItemDeleteTrigger
├─ FileUpload.ClearTrigger
└─ native input (automatic)
```

| Part                             | Role                                                                   |
| -------------------------------- | ---------------------------------------------------------------------- |
| `FileUpload` / `FileUpload.Root` | Owns file state, validation, drag/drop, paste, and form integration.   |
| `FileUpload.RootProvider`        | Renders a file upload tree from state returned by Ark `useFileUpload`. |
| `FileUpload.Label`               | Accessible label for the hidden input and upload controls.             |
| `FileUpload.Dropzone`            | Drag/drop target. Use `disableClick` with a nested trigger.            |
| `FileUpload.DropzoneIcon`        | Decorative upload icon helper for dropzone surfaces.                   |
| `FileUpload.Trigger`             | Button that opens the native file picker.                              |
| `FileUpload.ItemGroup`           | List wrapper for accepted or rejected file items.                      |
| `FileUpload.Items`               | Image cards or file rows with previews, metadata, and delete controls. |
| `FileUpload.Context`             | Ark-compatible render-prop context for custom file rows.               |
| `FileUpload.Item`                | File row. Pass the current `file` object.                              |
| `FileUpload.ItemPreview`         | Conditional preview slot matched by MIME type.                         |
| `FileUpload.ItemPreviewImage`    | Image preview rendered inside `ItemPreview`.                           |
| `FileUpload.ItemPreviewIcon`     | Generic file icon for an `ItemPreview` fallback.                       |
| `FileUpload.ItemName`            | File name text.                                                        |
| `FileUpload.ItemMetadata`        | File type and localized size; pass the current `file`.                 |
| `FileUpload.ItemSizeText`        | Localized file size text.                                              |
| `FileUpload.ItemDeleteTrigger`   | Removes one file. Defaults to the moduix trash icon.                   |
| `FileUpload.ClearTrigger`        | Clears selected files. Defaults to the moduix close icon.              |

## Composition

### Recommended composition

Keep the Ark part tree visible. Use `Trigger` for compact upload controls and `Dropzone` for
drag-and-drop surfaces. If `Trigger` is nested inside `Dropzone`, pass `disableClick` to `Dropzone`
so the native file picker is not opened twice.

The root renders the native form input whenever the upload participates in a form. Use `Items` for compact accepted-file rows. Read
accepted files from `FileUpload.Context.acceptedFiles` and rejected files from `FileUpload.Context.rejectedFiles`
when custom rows are needed.

Import `FileUpload.Context`, `useFileUpload()`, and `useFileUploadContext()` from `@moduix/react` when advanced
state access is needed. These are direct Ark re-exports, so callback details and state contracts stay unchanged.

### When to go custom

`Items` renders image files as compact vertical cards and other files as horizontal metadata rows. Use explicit
`ItemPreview.type` is a regular-expression predicate, not a priority list: render exactly one preview for the current
file. For example, use `file.type.startsWith("image/")` to choose `ItemPreviewImage`; otherwise render
`ItemPreviewIcon`, then compose `ItemName`, `ItemMetadata file={file}`, and `ItemDeleteTrigger`. Use `asChild` only
when the child is a single semantic element that can preserve the required button, label, or input behavior.

### Default props

| Prop                  | Default | Notes                                                        |
| --------------------- | ------- | ------------------------------------------------------------ |
| `allowDrop`           | `true`  | Enables drag/drop on `Dropzone`.                             |
| `maxFiles`            | `1`     | Increase or remove it for multi-file and directory uploads.  |
| `minFileSize`         | `0`     | Minimum accepted file size in bytes.                         |
| `maxFileSize`         | `∞`     | Maximum accepted file size in bytes.                         |
| `preventDocumentDrop` | `true`  | Prevents accidental navigation when dropping outside target. |
| `locale`              | `en-US` | Used for file size formatting.                               |

### Behavioral notes

`acceptedFiles`, `defaultAcceptedFiles`, and `onFileChange(details)` preserve Ark controlled and
uncontrolled state. `onFileAccept(details)` and `onFileReject(details)` keep Ark detail objects.
`Field.Root` and `Fieldset.Root` can provide disabled, invalid, required, and read-only state.
Use `ids` when separate labels or descriptions need stable accessibility links.

### Styling hooks

Target stable moduix slots such as `[data-slot="file-upload-dropzone"]`,
`[data-slot="file-upload-item"]`, and `[data-slot="file-upload-item-delete-trigger"]`.
Ark also exposes `data-scope="file-upload"`, part-specific `data-part`, and state attributes:
`data-disabled`, `data-readonly`, `data-invalid`, `data-required`, `data-dragging`, and `data-type`.
