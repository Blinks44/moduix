## Choosing the right component

Use `Menu` for trigger-anchored command lists, context menus, lightweight selections, and nested
actions.

- Choose **`Menu`** for overflow actions, row actions, and command lists opened from a button or
  context trigger.

## Anatomy

```text
Menu
├─ Menu.Trigger
│  └─ Menu.Indicator
├─ Menu.ContextTrigger
├─ Overlay subtree (automatically portalled)
│  └─ Menu.Positioner
│     └─ Menu.Content
│        ├─ Menu.Arrow
│        │  └─ Menu.ArrowTip
│        ├─ Menu.Item
│        ├─ Menu.CheckboxItem
│        │  ├─ Menu.ItemIndicator
│        │  └─ Menu.ItemText
│        ├─ Menu.RadioItemGroup
│        │  └─ Menu.RadioItem
│        ├─ Menu.ItemGroup
│        │  └─ Menu.ItemGroupLabel
│        └─ Menu
│           └─ Menu.TriggerItem
```

| Part                  | Role                                                        |
| --------------------- | ----------------------------------------------------------- |
| `Menu` / `Menu.Root`  | Ark root for open state, positioning, typeahead, dismissal. |
| `Menu.RootProvider`   | Renders a menu from a `useMenu` state instance.             |
| `Menu.Trigger`        | Button trigger; use `asChild` for a custom host.            |
| `Menu.Indicator`      | Ark state-aware trigger indicator.                          |
| `Menu.ContextTrigger` | Right-click and long-press trigger for context menus.       |
| `Menu.Positioner`     | Floating layer positioner.                                  |
| `Menu.Content`        | Popup surface with menu role and item collection.           |
| `Menu.Item`           | Command row.                                                |
| `Menu.CheckboxItem`   | Boolean option item.                                        |
| `Menu.RadioItemGroup` | Exclusive value group for radio items.                      |
| `Menu.RadioItem`      | Exclusive option item.                                      |
| `Menu.TriggerItem`    | Row that opens a nested `Menu`.                             |
| `Menu.ItemIndicator`  | Check indicator for checkbox and radio rows.                |

## Composition

The root portals `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Keep popup structure explicit. `Menu.Content` is the actual Ark content part; it no longer hides
`Menu.Positioner`.

```tsx
<Menu positioning={{ placement: 'bottom-start', gutter: 8 }}>
  <Menu.Trigger asChild>
    <Button>
      Actions
      <Menu.Indicator />
    </Button>
  </Menu.Trigger>
  <Menu.Positioner>
    <Menu.Content className="menu-content">
      <Menu.Item value="edit">Edit</Menu.Item>
    </Menu.Content>
  </Menu.Positioner>
</Menu>
```

### When to go custom

Use `asChild` when the trigger or item must render as another semantic element. For links, put
`asChild` on `Menu.Item` and make the anchor the single child so Ark can apply the correct ARIA,
focus, and keyboard behavior.

Nested menus are regular `Menu` roots inside the parent content. The nested root uses
`Menu.TriggerItem`, then its own `Menu.Positioner` and `Menu.Content`.

### Default props

| Surface                           | Default | Notes                                                  |
| --------------------------------- | ------- | ------------------------------------------------------ |
| `Menu` / `Menu.Root`              | —       | No additional behavioral defaults; Ark defaults apply. |
| `Menu.CheckboxItem` / `RadioItem` | `start` | `indicator` controls the indicator column position.    |

### Behavioral notes

- `open`, `defaultOpen`, `onOpenChange(details)`, `highlightedValue`,
  `onHighlightChange(details)`, `onSelect(details)`, `closeOnSelect`, `ids`, `lazyMount`,
  `unmountOnExit`, and `present` follow Ark directly.
- Use `value`, not a custom `id`, for menu item identity.
- `Menu.CheckboxItem.onCheckedChange` receives the next boolean checked state.
- `Menu.RadioItemGroup.onValueChange` receives `details.value`.
- `Menu.RootProvider` must be used with a `useMenu` instance instead of rendering a second `Menu`
  root for the same state.
- Roots portal `Menu.Positioner` automatically. Use `portalled={false}` to keep a menu within a
  dialog or scroll container, and pair it with `positioning={{ strategy: 'fixed', hideWhenDetached: true }}`
  when the container scrolls.
- `Menu.Indicator` mirrors Ark's indicator part. `Menu.TriggerIcon`, `Menu.TriggerItemIcon`,
  `Menu.ItemShortcut`, and item text helpers are moduix leaf sugar only.

  `[data-part='content']`, `[data-part='item']`, `[data-part='option-item']`,
  `[data-type='checkbox' | 'radio']`, `[data-state='open' | 'closed' | 'checked' | 'unchecked']`,
  `[data-highlighted]`, `[data-disabled]`, `[data-placement]`, and `[data-side]`.
- Ark variables: `--reference-width`, `--available-width`, `--available-height`,
  `--transform-origin`, `--layer-index`, `--arrow-size`, and `--arrow-background`.
- moduix hooks: `data-slot="menu-trigger"`, `menu-content`, `menu-item`, `menu-checkbox-item`,
  `menu-radio-item`, `menu-trigger-item`, `menu-item-indicator`, and `menu-item-shortcut`.
