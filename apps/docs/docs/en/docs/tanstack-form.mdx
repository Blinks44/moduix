# TanStack Form

TanStack Form fields are controlled. Pass `field.state.value` to the component, then map the
component callback back to `field.handleChange`. Ark compound controls render their own native form
control when they need one, so never add a separate hidden input.

```bash
npm install @tanstack/react-form
```

## Mapping

| Control                                         | TanStack Form wiring                                                      |
| ----------------------------------------------- | ------------------------------------------------------------------------- |
| `Field.Input`, `Field.Textarea`, `Field.Select` | Pass `field.state.value` and map the native change event.                 |
| `Select`                                        | Convert the scalar field value to and from Ark's string-array `value`.    |
| `Combobox`                                      | Use the same value mapping and keep filtering in `onInputValueChange`.    |
| `Checkbox`                                      | Map `details.checked === true` to the boolean field value.                |
| Automatic native controls                       | Pass semantic root props such as `name` or `form`; moduix owns the input. |

Define every field in `defaultValues`. Call `field.handleBlur` from the actual interaction
boundary so touched state and blur validation remain accurate.

## Validation and reset

Use a field validator for local rules and a form-level Standard Schema validator for shared or
cross-field rules. The component mapping below stays the same. The example validates on submit;
when you validate on blur or input, show `Field.ErrorText` and set `Field invalid` after the field
has been touched, rather than marking every initially empty required field as invalid.

`noValidate` is appropriate only when TanStack Form owns the visible error flow. Otherwise leave
native constraint validation enabled. Use `form.reset()` for controlled fields so the library state
and moduix controls reset together; a DOM reset alone cannot replace the controlled value stored by
the form.

## Repeated fields

Use an array-mode field as the owner of add and remove operations, and render a nested field for
each value. A user-created item should also have its own persistent `id`; use it as the React key
while the index remains part of the nested field name. Wrap the list in `Fieldset` when it has a
shared label. This fragment is rendered in the component where `form` was created.

```tsx
const newEmail = () => ({ id: crypto.randomUUID(), address: '' });

<form.Field name="emails" mode="array">
  {(arrayField) => (
    <>
      {arrayField.state.value.map((item, index) => (
        <form.Field key={item.id} name={`emails[${index}].address`}>
          {(field) => (
            <Field invalid={!field.state.meta.isValid}>
              <Field.Label>Email {index + 1}</Field.Label>
              <Field.Input
                name={field.name}
                type="email"
                value={field.state.value}
                onBlur={field.handleBlur}
                onChange={(event) => field.handleChange(event.currentTarget.value)}
              />
              <Field.ErrorText>{field.state.meta.errors.join(', ')}</Field.ErrorText>
              <Button type="button" onClick={() => arrayField.removeValue(index)}>
                Remove email
              </Button>
            </Field>
          )}
        </form.Field>
      ))}

      <Button type="button" onClick={() => arrayField.pushValue(newEmail())}>
        Add email
      </Button>
    </>
  )}
</form.Field>;
```

Use an ID already returned by your API when editing persisted items; `newEmail()` is only for
creating new client-side rows.

## Errors returned by the server

Put an asynchronous submit validator on the form. Return `fields` with the exact field names that
the server rejected and an optional `form` message for errors that do not belong to one field. The
normal `field.state.meta.errors` rendering then shows field errors without a separate error store.

```tsx
type ProjectFormValues = {
  name: string;
  emails: { address: string; id: string }[];
};

const form = useForm<ProjectFormValues>({
  defaultValues: { name: '', emails: [] },
  validators: {
    onSubmitAsync: async ({ value }) => {
      const result = await validateProjectOnServer(value);

      if (!result.ok) {
        return {
          form: result.message,
          // For example: { name: 'Already in use.', 'emails[0].address': 'Cannot be used.' }
          fields: result.fieldErrors,
        };
      }

      return null;
    },
  },
  onSubmit: ({ value }) => saveProject(value),
});

<form.Subscribe selector={(state) => [state.errorMap.onSubmit] as const}>
  {([error]) => (error ? <p role="alert">{error}</p> : null)}
</form.Subscribe>;
```

## Behavioral notes

- Wrap `form.handleSubmit()` in the native form submit handler, after calling `preventDefault()`;
  the complete form shows this boundary explicitly.
- For single selection, pass `field.state.value ? [field.state.value] : []` and read
  `details.value[0] ?? ""`.
- Call `field.handleBlur` from `Select.onInteractOutside` or `Combobox.Input.onBlur` so TanStack
  Form can update touched state.
- `Select` renders its native select automatically for native semantics and reset synchronization.
- Map `Checkbox` checked state to a boolean field value; its root renders the native input
  automatically.
- `name={field.name}` enables native `FormData` semantics but does not replace the field's controlled
  `state.value` and `handleChange` mapping.
- For the other automatic controls, keep exceptional form configuration on their roots: `DateInput`
  uses `names` for distinct range keys, `Select` uses `nativeFormControl="input"` for virtualized
  collections, and `SignaturePad` uses `getFormValue` for custom serialization.
- TanStack Form intentionally does not manage DOM focus. If you want to focus the first invalid
  field, implement it in `onSubmitInvalid` using the rendered `aria-invalid` controls.
- Use `form.Subscribe` for derived form state instead of reading `form.state` during render.
