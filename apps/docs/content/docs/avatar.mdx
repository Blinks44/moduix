Compose `Avatar.Root`, `Avatar.Fallback`, and `Avatar.Image`. Keep the fallback mounted so Ark UI
can switch between it and the image as loading state changes.

## Anatomy

```text
Avatar.Root | Avatar.RootProvider
├─ Avatar.Fallback
├─ Avatar.Image
└─ Avatar.Context (optional state reader)
```

| Part                     | Role                                                              |
| ------------------------ | ----------------------------------------------------------------- |
| `Avatar` / `Avatar.Root` | Creates and owns the Ark UI avatar state.                         |
| `Avatar.RootProvider`    | Renders an avatar from state created with `useAvatar`.            |
| `Avatar.Fallback`        | Shows initials, an icon, or custom content until the image loads. |
| `Avatar.Image`           | Shows the native image after a successful load.                   |
| `Avatar.Context`         | Exposes `loaded` and image state methods through a render prop.   |

## Composition

### Recommended composition

Use `Avatar.Root` for the normal path. `onStatusChange(details)` reports the Ark loading state,
while `ids` provides stable IDs for the root, image, and fallback. All DOM parts forward their refs
and support Ark `asChild`.

`Avatar.Image` renders an `img`, so provide meaningful `alt` text when the image conveys identity.
Use `alt=""` when the surrounding link or button already has the accessible name.

### When to go custom

Use `Avatar.Context` for an inline state read and `useAvatarContext` inside reusable custom child
components. Use `useAvatar` with `Avatar.RootProvider` when state must be created outside the
rendered avatar. Do not render `Avatar.Root` around the same `RootProvider` state instance.

For framework image components, merge the props from `useAvatarContext().getImageProps()` into the
rendered image and preserve Ark's hidden/visible behavior.

Use `asChild` with one semantic child when a link, button, or framework component must own a part's
DOM node.

### Default props

| Surface                              | Prop   | Default | Values                       |
| ------------------------------------ | ------ | ------- | ---------------------------- |
| `Avatar.Root`, `Avatar.RootProvider` | `size` | `md`    | `xs`, `sm`, `md`, `lg`, `xl` |

### Behavioral notes

- Ark controls `data-state="visible" | "hidden"` on `Avatar.Image` and `Avatar.Fallback`.
- `onStatusChange` receives a detail object; read the current value from `details.status`.
- A missing or failed image keeps the fallback visible.
- `Avatar.RootProvider` requires the `value` returned by `useAvatar`.
- `Avatar.Context` and `useAvatarContext` expose `loaded`, `setSrc`, `setLoaded`, and `setError`.
- moduix adds only visual defaults, stable `data-slot` hooks, CSS variables, and `size`.

## Examples

### Events

Use `onStatusChange(details)` when surrounding UI needs to react to the image lifecycle.

<Preview>
  <AvatarStatusExample />

  <Preview.Code>
    {`
      import { Avatar } from "@moduix/react";
      import { useState } from "react";

      export function AvatarStatusDemo() {
        const [status, setStatus] = useState("idle");

        return (
          <div className="docs-avatar-status">
            <Avatar onStatusChange={(details) => setStatus(details.status)}>
              <Avatar.Fallback>LT</Avatar.Fallback>
              <Avatar.Image src={avatarImage} alt="Alex T." />
            </Avatar>
            <output>Status: {status}</output>

        );

### Root Provider

Create state with `useAvatar` when controls outside the avatar need to coordinate with it, then
render `Avatar.RootProvider` instead of `Avatar.Root`.

<Preview>
  <AvatarRootProviderExample />

  <Preview.Code>
    {`
      import { Avatar, useAvatar } from "@moduix/react";
      import { useState } from "react";

      export function AvatarRootProviderDemo() {
        const [count, setCount] = useState(0);
        const avatar = useAvatar();

        return (
          <div className="docs-avatar-provider">
            <button
              type="button"
              className="docs-avatar-provider-button"
              onClick={() => setCount(count + 1)}
            >
              Change avatar
            </button>
            <Avatar.RootProvider value={avatar}>
              <Avatar.Fallback>LT</Avatar.Fallback>
              <Avatar.Image src={\`\${avatarImage}&seed=\${count}\`} alt="Alex T." />
            </Avatar.RootProvider>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{avatarExampleCss}</Preview.CSS>
  <Preview.Data>{avatarImageData}</Preview.Data>
</Preview>

### Provider

Pass options to `useAvatar` when the state owner needs the same Ark callbacks as `Avatar.Root`.

<Preview>
  <AvatarProviderExample />

  <Preview.Code>
    {`
      import { Avatar, useAvatar } from "@moduix/react";
      import { useState } from "react";

      export function AvatarProviderDemo() {
        const [status, setStatus] = useState("idle");
        const avatar = useAvatar({
          onStatusChange: (details) => setStatus(details.status),
        });

        return (
          <div className="docs-avatar-status">
            <Avatar.RootProvider value={avatar}>
              <Avatar.Fallback>LT</Avatar.Fallback>
              <Avatar.Image src={avatarImage} alt="Alex T." />
            </Avatar.RootProvider>
            <output>Hook status: {status}</output>
          </div>
        );
      }
    `}

  </Preview.Code>

### Styling hooks

- `data-slot="avatar-root"`, `avatar-root-provider`, `avatar-image`, and `avatar-fallback`.
- Ark anatomy hooks: `data-scope="avatar"` and
  `data-part="root" | "image" | "fallback"`.
- Ark image state: `data-state="visible" | "hidden"` on `Avatar.Image` and
  `Avatar.Fallback`.
- moduix size state: `data-size="xs" | "sm" | "md" | "lg" | "xl"` on
  `Avatar.Root` and `Avatar.RootProvider`.
