## Basic

Use `AvatarImage` and `AvatarFallback` together inside `Avatar`. This is the recommended default
path and keeps image loading, error handling, and fallback timing on the primitive.

## Anatomy

`Avatar` keeps the public surface short: one root and two visible slots.

```text
Avatar
├─ AvatarImage
└─ AvatarFallback
   └─ initials | icon | custom content
```

| Part             | Role                                                                 |
| ---------------- | -------------------------------------------------------------------- |
| `Avatar`         | Root slot. Controls size, shape, shared color, and `render` support. |
| `AvatarImage`    | Visible image when the source loads successfully.                    |
| `AvatarFallback` | Fallback content during loading, missing image, or image error.      |

## Composition

### When to stay high-level

Stay with `Avatar`, `AvatarImage`, and `AvatarFallback` for most user pictures and identity chips.

Use `AvatarFallback delay` when you want to suppress a brief fallback flash during normal image
loading.

Use `size` for the standard token sizes, or root-level CSS variables such as `--avatar-size`,
`--avatar-font-size`, and `--avatar-line-height` when dimensions need to differ from the presets.

### When to go custom

Switch to custom composition only when the root must render as another element through `render`, or
when the root, image, and fallback need independent visual treatment.

### Default props

| Surface          | Prop                    | Default | Values / Notes               |
| ---------------- | ----------------------- | ------- | ---------------------------- |
| `Avatar`         | `render`                | `span`  | Any valid React element      |
| `Avatar`         | `size`                  | `md`    | `xs`, `sm`, `md`, `lg`, `xl` |
| `AvatarFallback` | `delay`                 | -       | `number` in milliseconds     |
| `AvatarImage`    | `onLoadingStatusChange` | -       | `(status) => void`           |

### Behavioral notes

- `AvatarImage` is shown only when the source loads successfully.
- `AvatarFallback` remains the identity fallback while the image is absent, delayed, loading, or failed.
- The wrappers do not create their own loading state, focus behavior, or accessibility labels.
- If the avatar is interactive through `render`, the rendered element must carry the correct accessible semantics.

## Styling

Use CSS variables on the root when avatar size, radius, image framing, or fallback treatment needs
to change. `size` is intentionally limited to the common token presets.

### Styling hooks

- `data-slot="avatar-root"`, `avatar-image`, and `avatar-fallback` are stable selectors on the exported parts.
- `Avatar` writes `data-size` when the `size` prop is provided.
- `AvatarImage` keeps Base UI transition attributes such as `data-starting-style` and `data-ending-style`.
