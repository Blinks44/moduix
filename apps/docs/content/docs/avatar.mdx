## Anatomy

```text
Avatar.Root
├─ Avatar.Image
└─ Avatar.Fallback
   └─ initials | icon | custom content
```

| Part              | Role                                                     |
| ----------------- | -------------------------------------------------------- |
| `Avatar.Root`     | Shared size, shape, color, and Ark loading state owner.  |
| `Avatar.Image`    | Visible image when the source loads successfully.        |
| `Avatar.Fallback` | Identity fallback while the image is absent or rejected. |

## Composition

### When to stay high-level

Stay with `Avatar.Root`, `Avatar.Image`, and `Avatar.Fallback` for most user pictures and compact
identity surfaces.

Use `size` on `Avatar.Root` for the standard token sizes, or override `--avatar-size`,
`--avatar-font-size`, and `--avatar-line-height` when dimensions need a custom value.

### When to go custom

Use `asChild` when the root or one of its parts must compose with another element. This replaces
the old Base UI `render` path.

### Default props

| Surface       | Prop             | Default | Values / Notes               |
| ------------- | ---------------- | ------- | ---------------------------- |
| `Avatar.Root` | `size`           | `md`    | `xs`, `sm`, `md`, `lg`, `xl` |
| `Avatar.Root` | `asChild`        | `false` | Ark composition escape hatch |
| `Avatar.Root` | `onStatusChange` | -       | `(details) => void`          |

### Behavioral notes

- `Avatar.Image` is visible only after a successful image load.
- `Avatar.Fallback` remains visible while the image is missing or has failed.
- `onStatusChange` follows Ark shape and receives `details` (`details.status` contains the current
  loading state).
- If `Avatar.Root` is interactive through `asChild`, the child element must provide the correct
  accessible semantics and name.

### Styling hooks

- `data-slot="avatar-root"`, `avatar-image`, and `avatar-fallback` are stable selectors on the
  exported parts.
- Ark also exposes `data-scope="avatar"` and `data-part="root" | "image" | "fallback"` on the
  rendered nodes.
- `Avatar.Image` and `Avatar.Fallback` expose `data-state="visible" | "hidden"` for state-aware
  styling.
- `data-size` is added on `Avatar.Root` when the moduix `size` prop is used.
