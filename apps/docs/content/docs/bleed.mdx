## API Reference

`Bleed` is a standalone moduix layout primitive and does not wrap a Base UI primitive.

## Basic

Start with the default `inline="full"` behavior when media, banners, or section accents should
break out of a constrained container while staying in normal document flow.

<Preview>
  <BleedExample />

  <Preview.Code>
    {`
      import { Bleed, Text } from "moduix";

      export function BleedDemo() {
        return (
          <div className={styles.container}>
            <Text tone="muted">Container content stays constrained.</Text>
            <Bleed className={styles.bleed}>
              <Text weight="semibold">
                This block bleeds to the viewport edges.
              </Text>
            </Bleed>
            <Text tone="muted">
              Following content returns to the container width.
            </Text>

      }

## Anatomy

`Bleed` has a single root element. It applies negative inline and/or block margins on that root.

```text
Bleed
└─ root[data-slot="bleed-root"]
   └─ children
```

| Part    | Role                                            |
| ------- | ----------------------------------------------- |
| `Bleed` | Root layout wrapper for inline and block bleed. |

## Composition

### When to stay high-level

Stay with `Bleed` when the layout only needs controlled negative margins on one wrapper.
This is the intended path for full-bleed media, inset section backgrounds, dividers, and banners
inside centered or padded shells.

### When to go custom

Use `className` and local `--bleed-*` overrides when a drawer, nested scroll region, or app shell
needs different full-bleed math than the default viewport-based values.

### Default props

| Prop     | Default | Values                                       |
| -------- | ------- | -------------------------------------------- |
| `inline` | `full`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `block`  | `none`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`         |
| `as`     | `div`   | Any React element type                       |

### Behavioral notes

- `Bleed` renders exactly one root and forwards regular DOM props to it.
- Use `inline` for horizontal escape, and add `block` when a surface should also escape vertical
  padding.
- Use one of the fixed scale values when content only needs to escape container padding instead of
  reaching the viewport edge.
- `inline="full"` uses viewport math, so nested shells may need local overrides for
  `--bleed-inline-full` and `--bleed-inline-full-size`.
- Ancestors with `overflow: hidden` or `overflow: clip` can crop the bleed.

### Custom Composition

Pass `className` when styling the root or overriding bleed variables in local CSS.
This keeps the public API small while still allowing local full-bleed math for shells, drawers,
and inset layouts that do not align with the viewport.

<Preview>
  <CustomCompositionBleedExample />

  <Preview.Code>
    {`
      import { Bleed, Text } from "moduix";

      export function CustomCompositionBleedDemo() {
        return (
          <div className={styles.container}>
            <Bleed className={styles.customBleed}>
              <Text weight="semibold">Customized bleed amount.</Text>
            </Bleed>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`

## Styling

`Bleed` exposes a small CSS variable contract and stable root data attributes. Use variables when
the layout math needs to change, and use `className` for the visual surface inside the bleed.

### CSS Properties

### Styling hooks

- `data-slot="bleed-root"` is the stable selector on the exported root.
- `data-inline` reflects the current inline bleed mode: `none`, `xs`, `sm`, `md`, `lg`, `xl`, or
  `full`.
- `data-block` reflects the current block bleed mode: `none`, `xs`, `sm`, `md`, `lg`, or `xl`.
- The public `--bleed-*` variables in `theme.css` cover both scale-based bleed amounts and the
  viewport math used by `inline="full"`.
