## API Reference

`Bleed` is a small layout wrapper for content that should escape a constrained parent without leaving normal document flow.
Use `inline` and `block` to control each axis, and `as` when the root should render as a different element.
`inline="full"` is viewport-based, so it is best suited for full-bleed sections rather than container-matching layout.

## Anatomy

`Bleed` has a single root element. It applies negative inline and/or block margins so content can
escape a constrained parent while staying in normal document flow.

```text
Bleed
└─ children
```

```tsx
<Bleed inline="md" block="none">
  <div className={styles.panel}>Escaped content</div>
</Bleed>
```

| Part    | Role                                            |
| ------- | ----------------------------------------------- |
| `Bleed` | Root layout wrapper for inline and block bleed. |

## Composition

Use `inline` and `block` to control bleed per axis. Use `as` when the root should render as a
different element.

| Prop     | Default | Values                                       |
| -------- | ------- | -------------------------------------------- |
| `inline` | `full`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `block`  | `none`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`         |
| `as`     | `div`   | Any React element type                       |

`Bleed` has no internal slots. Customize via `className` and root `--bleed-*` variables.

`inline="full"` uses viewport math, so ancestor clipping still applies:

- `overflow: hidden` or `overflow: clip` on an ancestor can crop the bleed.
- Nested scroll areas, drawers, and app shells may need local overrides for `--bleed-inline-full`
  and `--bleed-inline-full-size`.

## Examples

### Inline Amounts

Use `inline` to choose how far content escapes the container on the inline axis.

### Semantic Element

Use `as` when the escaping content has a more specific semantic role.
