## Overview

`Bleed` is a small layout primitive for content that should intentionally escape a constrained
parent while staying in normal document flow.

Use `inline` for horizontal bleed, add `block` when a surface should also escape vertical padding,
and use `as` when the root needs a semantic element.
The default path is `inline="full"` for full-bleed media, banners, and section accents.

## Anatomy

`Bleed` has a single root element. It applies negative inline and/or block margins so content can
escape a constrained parent while staying in normal document flow.

```text
Bleed
└─ children
```

```tsx
<Bleed inline="md" block="none">
  <div className={styles.panel}>Escaped content</div>
</Bleed>
```

| Part    | Role                                            |
| ------- | ----------------------------------------------- |
| `Bleed` | Root layout wrapper for inline and block bleed. |

## Composition

`Bleed` has one job: apply negative margins on the root element.
It does not add inner wrappers, layout state, or slot APIs.

### Defaults

| Prop     | Default | Values                                       |
| -------- | ------- | -------------------------------------------- |
| `inline` | `full`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `block`  | `none`  | `none`, `xs`, `sm`, `md`, `lg`, `xl`         |
| `as`     | `div`   | Any React element type                       |

Use `className` for root styling and local `--bleed-*` overrides when layout math needs to change.

`inline="full"` uses viewport math. Keep these constraints in mind:

- `overflow: hidden` or `overflow: clip` on an ancestor can crop the bleed.
- Nested scroll regions, drawers, and app shells may need local overrides for
  `--bleed-inline-full` and `--bleed-inline-full-size`.
- Use one of the fixed scale values when content only needs to escape container padding, not reach
  the viewport edge.

## Custom Composition

Pass `className` when styling the root or overriding bleed variables in local CSS.
This keeps the public API small while still allowing local full-bleed math for shells, drawers,
and inset layouts that do not align with the viewport.

<Preview>
  <CustomCompositionBleedExample />

  <Preview.Code>
    {`
      import { Bleed, Text } from "moduix";

      export function CustomCompositionBleedDemo() {
        return (
          <div className={styles.container}>
            <Bleed className={styles.customBleed}>
              <Text weight="semibold">Customized bleed amount.</Text>
            </Bleed>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
