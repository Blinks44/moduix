## Choosing The Right Component

Use `SegmentGroup` when the user chooses one value from a small visible set.

- Choose **`Tabs`** when selection changes the visible content panel.
- Choose **`ToggleGroup`** when items behave like toolbar buttons and can support multiple active
  values.
- Choose **`RadioGroup`** when the form-control shape should stay visibly radio-like.
- Choose **`SegmentGroup`** when the value is still radio-like, but the visual should be a compact
  segmented control.

## Anatomy

```text
SegmentGroup / SegmentGroup.Root
├─ SegmentGroup.Label (optional)
├─ SegmentGroup.Indicator
└─ SegmentGroup.Item[value]
   ├─ SegmentGroup.ItemText
   ├─ SegmentGroup.ItemControl
   └─ SegmentGroup.ItemHiddenInput

SegmentGroup.RootProvider
└─ same item tree connected to an Ark useSegmentGroup() store
```

| Part                                 | Role                                                          |
| ------------------------------------ | ------------------------------------------------------------- |
| `SegmentGroup` / `SegmentGroup.Root` | Owns single-value state, keyboard navigation, and forms.      |
| `SegmentGroup.RootProvider`          | Connects parts to state created with Ark `useSegmentGroup()`. |
| `SegmentGroup.Label`                 | Optional accessible label for the group.                      |
| `SegmentGroup.Indicator`             | Measured active-item highlight using Ark CSS variables.       |
| `SegmentGroup.Item`                  | One selectable segment; renders a `label` by default.         |
| `SegmentGroup.ItemText`              | Visible segment text.                                         |
| `SegmentGroup.ItemControl`           | Ark control part; visually hidden by moduix default styles.   |
| `SegmentGroup.ItemHiddenInput`       | Native input for form submission, validation, and reset.      |

## Composition

### Recommended composition

Use the short root form `<SegmentGroup>` for normal usage. Render `Indicator` before items so the
measured highlight sits behind the checked segment, and keep `ItemHiddenInput` inside each item
when the control participates in native forms or reset behavior.

### When to go custom

Use `SegmentGroup.Item asChild` when a custom item node should own the label element. The child
must still be a single semantic `label`. Use `SegmentGroup.RootProvider` with Ark
`useSegmentGroup()` when surrounding controls need to call methods such as `setValue`,
`clearValue`, or `focus`.

### Default props

| Prop           | Default      | Notes                                                     |
| -------------- | ------------ | --------------------------------------------------------- |
| `defaultValue` | -            | Initial uncontrolled value.                               |
| `value`        | -            | Controlled selected value.                                |
| `orientation`  | `horizontal` | Moduix defaults to horizontal; set `vertical` explicitly. |
| `disabled`     | `false`      | Disables the whole group when set on the root.            |
| `readOnly`     | `false`      | Prevents changes while keeping the value readable.        |
| `required`     | `false`      | Marks hidden inputs as required for native forms.         |
| `invalid`      | `false`      | Adds Ark invalid state attributes for styling/ARIA.       |

### Behavioral notes

`onValueChange(details)` keeps Ark's callback shape. Read `details.value` for the selected value.
Ark owns roving focus, arrow-key navigation, focus management, disabled/read-only/invalid/required
state, ids, form submission, and form reset synchronization.

## Styling

Style `SegmentGroup` through `className`, stable `data-slot="segment-group-*"` hooks, Ark
`data-scope`, `data-part`, `data-state`, and the public `--segment-group-*` CSS variables below.

### Styling hooks

- Ark anatomy hooks: `data-scope="segment-group"` and `data-part` on root, label, indicator,
  item, item text, item control, and hidden input.
- Ark state hooks: `data-state="checked" | "unchecked"`, `data-orientation`,
  `data-disabled`, `data-readonly`, `data-invalid`, `data-required`, `data-active`, and
  `data-focus-visible`.
- Moduix hooks: `data-slot="segment-group-root"`, `segment-group-root-provider`,
  `segment-group-label`, `segment-group-indicator`, `segment-group-item`,
  `segment-group-item-text`, `segment-group-item-control`, and
  `segment-group-item-hidden-input`.
- Indicator sizing uses Ark runtime variables `--left`, `--top`, `--width`, and `--height`.
  Indicator motion uses Ark `--transition-duration` and `--transition-timing-function`, with
  moduix defaults mapped from `--segment-group-indicator-transition-duration` and
  `--segment-group-indicator-transition-timing-function`.
