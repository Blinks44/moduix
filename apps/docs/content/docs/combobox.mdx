## Choosing the right component

Use `Combobox` when users select from known values and typing helps filter the available options.
Use `Select` when the same predefined choice does not need an editable input.

## Basic

Create an Ark `ListCollection`, pass it to `Combobox.Root`, and keep the popup structure explicit.
Use `Combobox.Option` for ordinary rows; drop to `Combobox.Item` when a row needs custom structure.
`onInputValueChange` receives an Ark details object.

## Anatomy

```text
Combobox.Root[collection]
├─ Combobox.Label
├─ Combobox.Control
│  ├─ Combobox.Input
│  ├─ Combobox.ClearTrigger
│  └─ Combobox.Trigger
└─ Overlay subtree (automatically portalled)
   └─ Combobox.Positioner
      └─ Combobox.Content
         ├─ Combobox.Empty
         ├─ Combobox.List
         │  ├─ Combobox.Option[item]
         │  └─ Combobox.Item[item]
         │     ├─ Combobox.ItemText
         │     └─ Combobox.ItemIndicator
         └─ Combobox.ItemGroup
            ├─ Combobox.ItemGroupLabel
            └─ Combobox.Item
```

| Part                         | Role                                                               |
| ---------------------------- | ------------------------------------------------------------------ |
| `Combobox` / `Combobox.Root` | Owns combobox state, filtering, highlighting, and selected values. |
| `Combobox.Control`           | Wrapper for input and trigger controls.                            |
| `Combobox.Input`             | Text input that drives search and selection.                       |
| `Combobox.Trigger`           | Opens and closes the popup list.                                   |
| `Combobox.ClearTrigger`      | Clears current value and/or input.                                 |
| `Combobox.Positioner`        | Floating positioner for popup content.                             |
| `Combobox.Content`           | Popup content container.                                           |
| `Combobox.List`              | Option list wrapper.                                               |
| `Combobox.Option`            | Convenience row for simple options with text and an indicator.     |
| `Combobox.Item`              | Selectable option row.                                             |
| `Combobox.ItemText`          | Text label slot inside an item.                                    |
| `Combobox.ItemIndicator`     | Selected/highlight indicator slot for an item.                     |

`Combobox.RootProvider` renders the same styled root around state created with Ark `useCombobox`.
Context parts and hooks remain direct Ark imports.

## Composition

The root portals `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Use `createListCollection` for a static collection or `useListCollection` when filtering changes
the rendered items. Object collections should define `itemToString` and `itemToValue`.

Keep `Combobox.Positioner` and `Combobox.Content` visible. `Content` is the actual Ark content part;
the root only automates the portal boundary.

When that popup structure repeats, prefer a small local helper. Use `Combobox.Option` for simple
rows and keep `Combobox.Item` for links, rich rows, or custom indicator placement:

```tsx
function FruitComboboxPopup({ items }: { items: Array<{ label: string; value: string }> }) {
  return (
    <Combobox.Positioner>
      <Combobox.Content>
        <Combobox.Empty>No fruits found.</Combobox.Empty>
        <Combobox.List>
          {items.map((item) => (
            <Combobox.Option key={item.value} item={item}>
              {item.label}
            </Combobox.Option>
          ))}
        </Combobox.List>
      </Combobox.Content>
    </Combobox.Positioner>
  );
}
```

### When to go custom

Use controlled `value` and Ark detail callbacks when adjacent UI needs selected values. When state
must be created outside the rendered subtree, create it with Ark `useCombobox` and render
`Combobox.RootProvider`. Context and state hooks remain available directly from
`@ark-ui/react/combobox`.

Multiple selection does not add a combobox-specific chip API. Render tags or another value summary
from controlled state, then keep selection behavior on `Combobox.Root multiple`.

Use ordinary elements inside `Combobox.Content` for loading and error status. Ark does not expose a
combobox `Status` part.

### Default props

| Surface                            | Default               | Notes                                                                 |
| ---------------------------------- | --------------------- | --------------------------------------------------------------------- |
| `Combobox.Trigger`                 | moduix chevron icon   | Used when children are omitted.                                       |
| `Combobox.ClearTrigger`            | moduix close icon     | Used when children are omitted.                                       |
| `Combobox.ItemIndicator`           | moduix check icon     | Used when children are omitted.                                       |
| `Combobox.Option.indicator`        | check icon            | Pass `false` to hide it or a React node to replace the icon.          |
| `Combobox` / `Combobox.Root` width | `16rem` (max `100%`)  | Styled root width defaults.                                           |
| `Combobox.Positioner` variables    | Ark runtime variables | Uses `--reference-width`, `--available-height`, `--transform-origin`. |

### Behavioral notes

- `collection` is required.
- `value` and `defaultValue` are string arrays, including single selection.
- `onValueChange(details)` reads the current values from `details.value`.
- `onInputValueChange(details)` reads input text from `details.inputValue` and its source from
  `details.reason`.
- `onOpenChange`, `onHighlightChange`, and `onSelect` keep their Ark detail shapes.
- All Ark DOM parts preserve `asChild`.
- The current Ark Combobox package does not expose a `HiddenInput` part. Use the root `name` and
  `form` props for Ark's built-in form submission behavior.
- `Combobox.RootProvider` preserves moduix root styling and portal configuration for externally
  created Ark state. Hooks, context parts, and Ark types are imported directly from
  `@ark-ui/react/combobox`.
- Ark `Field.Root` state inheritance remains available when composing with `@ark-ui/react/field`.
  The current moduix `Field` component has not migrated to the Ark field contract yet and does not provide Ark field
  context.

### Migrating from shadcn

Keep the moduix API Ark-shaped, then translate the common shadcn conveniences explicitly:

| shadcn pattern          | moduix path                                                          |
| ----------------------- | -------------------------------------------------------------------- |
| `items` on the root     | `createListCollection()` or `useListCollection()`, then `collection` |
| `itemToStringValue`     | `itemToString` and usually `itemToValue` on the collection           |
| `showClear`             | Render `Combobox.ClearTrigger` when needed                           |
| Plain `ComboboxItem`    | Use `Combobox.Option`; use `Combobox.Item` for custom row structure  |
| Built-in chips surface  | Controlled `value` plus custom tags, or `TagsInput` composition      |
| `ComboboxContent` sugar | Explicit `Combobox.Positioner` and `Combobox.Content`                |

### Styling hooks

- Stable slots include `combobox-root`, `combobox-label`, `combobox-control`, `combobox-input`,
  `combobox-clear-trigger`, `combobox-trigger`, `combobox-positioner`, `combobox-content`,
  `combobox-empty`, `combobox-list`, `combobox-item-group`, `combobox-item-group-label`,
  `combobox-item`, `combobox-item-text`, and `combobox-item-indicator`. `Combobox.Option` reuses the
  item slots instead of adding its own styling hook.
- Ark state hooks include `data-state="open|closed"`, `data-focus`, `data-invalid`,
  `data-disabled`, `data-highlighted`, `data-empty`, `data-placement`, and `data-side`.
- Ark positioning variables include `--reference-width`, `--available-width`,
  `--available-height`, `--transform-origin`, `--z-index`, and `--layer-index`.
