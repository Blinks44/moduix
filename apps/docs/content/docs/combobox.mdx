## Choosing the right component

Use `Combobox` when users must choose from a known dataset, but typing is useful for searching
or filtering the available options.

- Choose **`Autocomplete`** when users can enter free-form text and suggestions only help complete the value.
- Choose **`Combobox`** when users must choose predefined options, but typing helps search or filter the list.
- Choose **`Select`** when users must choose predefined options and typing in the trigger is not needed.

## Anatomy

`Combobox` keeps popup infrastructure behind `ComboboxContent`, so the common composition stays
short and focused on the actual field and list parts.

```text
Combobox
├─ ComboboxField
│  ├─ label (native)
│  └─ ComboboxInputGroup
│     ├─ ComboboxInput
│     └─ ComboboxControlActions
│        ├─ ComboboxClear
│        └─ ComboboxTrigger
└─ ComboboxContent
   ├─ ComboboxInlineInputContainer (optional)
   ├─ ComboboxStatus / ComboboxEmpty
   └─ ComboboxList
      ├─ ComboboxGroup / ComboboxCollection
      ├─ ComboboxRow (for grid lists)
      ├─ ComboboxSeparator
      └─ ComboboxItem[value]
         ├─ ComboboxItemIndicator
         └─ ComboboxItemText
```

| Part                           | Role                                                                                                                    |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------- |
| `Combobox`                     | Root state machine. Controls selected value, input text, popup open state, filtering, and keyboard navigation.          |
| `ComboboxField`                | Field wrapper for label and input area. Keep visible form-control pieces here for layout and accessibility semantics.   |
| `ComboboxInput`                | Search/input control. Drives filtering and exposes input events, placeholder, and focus behavior.                       |
| `ComboboxInlineInputContainer` | Optional inline search wrapper for the input-inside-popup composition. It adds the divider and nested input treatment.  |
| `ComboboxControlActions`       | Slot for inline controls such as clear and trigger buttons.                                                             |
| `ComboboxContent`              | Popup surface. It also owns the default portal and positioner used by the common composition.                           |
| `ComboboxList`                 | Collection renderer for options or grouped options. Contains `ComboboxItem`, `ComboboxGroup`, and `ComboboxCollection`. |
| `ComboboxItem`                 | One selectable option. Renders selected state and text/indicator subparts.                                              |
| `ComboboxChips`                | Multiple-selection chip container used with `multiple` and `ComboboxValue`.                                             |

Most customizations should stay on visible parts such as `ComboboxInput`, `ComboboxItem`,
`ComboboxItemIndicator`, `ComboboxItemText`, and `ComboboxContent`.

For the standard input path, pair a native `<label htmlFor={id}>` with `ComboboxInput id={id}`.
Use `ComboboxFieldLabel` only in the trigger-first layout, where Base UI labels the trigger.

## Composition

### When to stay high-level

Stay with `Combobox`, `ComboboxField`, `ComboboxInputGroup`, `ComboboxInput`, and
`ComboboxContent` for most searchable single-select and grouped-list fields.

Use `Combobox` for root state and behavior props such as `value`, `defaultValue`,
`onValueChange`, `inputValue`, `onInputValueChange`, `open`, `onOpenChange`, `multiple`,
`filter`, `filteredItems`, `limit`, `grid`, `virtualized`, `modal`, `openOnInputClick`,
`autoHighlight`, `loopFocus`, `itemToStringLabel`, `itemToStringValue`, and
`isItemEqualToValue`.

`ComboboxContent` is the high-level popup wrapper. It accepts popup `className`, supports shared
positioning props (`side`, `sideOffset`, `align`, `alignOffset`, `arrowPadding`,
`collisionAvoidance`, `collisionBoundary`, `collisionPadding`), and can opt into the built-in
arrow with `showArrow`.

### When to go custom

Switch to `ComboboxPortal`, `ComboboxBackdrop`, `ComboboxPositioner`, `ComboboxPopup`, and
`ComboboxArrow` when the popup infrastructure itself needs to change, such as for a custom
backdrop, custom arrow treatment, or explicit stacking and layering control.

Use `ComboboxFieldTrigger`, `ComboboxValue`, and `ComboboxInlineInputContainer` when the visible
field opens a popup that contains the searchable input.

### Default props

| Prop               | Default | Values / Notes                            |
| ------------------ | ------- | ----------------------------------------- |
| `showArrow`        | `false` | Enables the built-in popup arrow          |
| `sideOffset`       | `5`     | Any number accepted by Base UI positioner |
| `indicator` (item) | `start` | `start`, `end`, `none`                    |

### Behavioral notes

- Prefer a native `<label htmlFor={id}>` with `ComboboxInput id={id}` for the standard input path.
- Use `ComboboxFieldLabel` only in the trigger-first pattern where Base UI labels the trigger.
- Use `filter={null}` when results are already filtered by application state or a remote request.
- `ComboboxStatus` is the intended place for async loading or error messaging.
- `ComboboxItemText` content does not control the selected label. Use root item string helpers such
  as `itemToStringLabel`.
- Default icons are only fallbacks: pass children to `ComboboxTrigger`, `ComboboxIcon`,
  `ComboboxClear`, `ComboboxItemIndicator`, and `ComboboxChipRemove` to use your own icon set.

### Async Search

Disable the built-in filter with `filter={null}` when results are loaded from your application state or a remote API.

<Preview>
  <AsyncSearchComboboxExample />

  <Preview.Code>
    {`
      import {
        Combobox,
        ComboboxField,
        ComboboxFieldLabel,
        ComboboxInputGroup,
        ComboboxInput,
        ComboboxControlActions,
        ComboboxClear,
        ComboboxTrigger,
        ComboboxContent,
        ComboboxStatus,
        ComboboxEmpty,
        ComboboxList,
        ComboboxItem,
        ComboboxItemIndicator,
        ComboboxItemText,
        useComboboxFilter,
      } from "moduix";
      import styles from "./combobox.module.css";
      import { useId, useMemo, useRef, useState, useTransition } from "react";

      export function AsyncSearchComboboxDemo() {
        const id = useId();
        const { contains } = useComboboxFilter();
        const [searchResults, setSearchResults] = useState([]);
        const [selectedValue, setSelectedValue] = useState(null);
        const [searchValue, setSearchValue] = useState("");
        const [error, setError] = useState(null);
        const [isPending, startTransition] = useTransition();
        const abortControllerRef = useRef(null);
        const trimmedSearchValue = searchValue.trim();

        const items = useMemo(() => {
          if (!selectedValue || searchResults.some((user) => user.id === selectedValue.id)) {
            return searchResults;
          }

          return [...searchResults, selectedValue];
        }, [searchResults, selectedValue]);

        async function searchUsers(query) {
          await new Promise((resolve) => setTimeout(resolve, 250));

          if (query === "error") {
            return { users: [], error: "Failed to fetch people. Try again." };
          }

          return {
            users: directoryUsers.filter(
              (user) =>
                contains(user.name, query) ||
                contains(user.username, query) ||
                contains(user.email, query) ||
                contains(user.title, query),
            ),
            error: null,
          };
        }

        const status = isPending
          ? "Searching..."
          : error
            ? error
            : trimmedSearchValue === ""
              ? selectedValue ? null : "Start typing to search people..."
              : searchResults.length === 0
                ? \`No matches for "\${trimmedSearchValue}".\`
                : null;

        return (
          <Combobox
            items={items}
            itemToStringLabel={(user) => user.name}
            filter={null}
            onOpenChangeComplete={(open) => {
              if (!open && selectedValue) {
                setSearchResults([selectedValue]);
              }
            }}
            onValueChange={(nextSelectedValue) => {
              setSelectedValue(nextSelectedValue);
              setSearchValue("");
              setError(null);
            }}
            onInputValueChange={(nextSearchValue, { reason }) => {
              setSearchValue(nextSearchValue);

              if (nextSearchValue === "") {
                abortControllerRef.current?.abort();
                setSearchResults([]);
                setError(null);
                return;
              }

              if (reason === "item-press") {
                return;
              }

              const controller = new AbortController();
              abortControllerRef.current?.abort();
              abortControllerRef.current = controller;

              startTransition(async () => {
                setError(null);
                const result = await searchUsers(nextSearchValue);

                if (controller.signal.aborted) {
                  return;
                }

                startTransition(() => {
                  setSearchResults(result.users);
                  setError(result.error);
                });
              });
            }}
          >
            <ComboboxField>
              <label htmlFor={id}>Assign reviewer</label>
              <ComboboxInputGroup>
                <ComboboxInput id={id} placeholder="e.g. Michael" />
                <ComboboxControlActions>
                  <ComboboxClear aria-label="Clear selection" />
                  <ComboboxTrigger aria-label="Open options" />
                </ComboboxControlActions>
              </ComboboxInputGroup>
            </ComboboxField>

            <ComboboxContent sideOffset={4}>
              <ComboboxStatus>{status}</ComboboxStatus>
              <ComboboxEmpty>
                {trimmedSearchValue !== "" && !isPending && searchResults.length === 0
                  ? "Try a different search term."
                  : null}
              </ComboboxEmpty>
              <ComboboxList>
                {(user) => (
                  <ComboboxItem key={user.id} value={user}>
                    <ComboboxItemIndicator />
                    <ComboboxItemText className={styles.multilineItemText}>
                      <span className={styles.itemTitle}>{user.name}</span>
                      <span className={styles.itemMeta}>
                        <span>@{user.username}</span>
                        <span>{user.title}</span>
                      </span>
                      <span className={styles.itemMeta}>{user.email}</span>
                    </ComboboxItemText>
                  </ComboboxItem>
                )}
              </ComboboxList>
            </ComboboxContent>
          </Combobox>
        );
      }
    `}

  </Preview.Code>

## Styling

Style `Combobox` through `className`, `data-slot="combobox-*"` hooks, and the public
`--combobox-*` CSS variables shown below.

### Styling hooks

Use the part that owns the visual concern:

- `ComboboxInput` and `ComboboxInputGroup` for the standard field chrome
- `ComboboxContent` or `ComboboxPopup` for popup surface overrides
- `ComboboxItem`, `ComboboxItemIndicator`, and `ComboboxItemText` for list row layout
- `ComboboxGroupLabel`, `ComboboxStatus`, and `ComboboxEmpty` for support rows and grouping
- `ComboboxChip`, `ComboboxChipRemove`, and `ComboboxChipsInput` for multiple-selection chip styling

The main public hooks include `data-slot="combobox-input-group"`, `data-slot="combobox-input"`,
`data-slot="combobox-popup"`, `data-slot="combobox-item"`, `data-slot="combobox-group-label"`,
`data-slot="combobox-status"`, and `data-slot="combobox-chip"`.
