## API Reference

`PasswordInput` does not wrap a dedicated Base UI primitive. It is a moduix composition component
built from the local `InputGroup`, `InputGroupInput`, and `InputGroupButton` wrappers, then narrowed
to the common password-field workflow.

## Overview

`PasswordInput` keeps the default path short:

- it renders one text-like input with password semantics
- it includes a built-in visibility toggle button
- it stays compatible with `Field` labels, descriptions, errors, and validation state

Use it when the field value should be masked by default but still revealable on demand. For plain
text-like fields, use `Input`. For mixed inline affordances beyond password visibility, use
`InputGroup` directly.

## Anatomy

`PasswordInput` is a focused composition wrapper around the grouped field shell.

```text
Field (optional wrapper)
└─ PasswordInput
   ├─ input
   └─ visibility toggle button
```

| Part                     | Role                                                               |
| ------------------------ | ------------------------------------------------------------------ |
| `PasswordInput`          | Grouped field shell plus password-specific show/hide behavior.     |
| inner grouped input      | The real input element used for form value, validation, and focus. |
| visibility toggle button | Icon-only button that reveals or masks the current password value. |

## Composition

`PasswordInput` keeps the input contract intact for value and validation while owning only one extra
behavior: visibility toggling.

- The ref points to the real input element, not the toggle button.
- The component manages `type="password"` and `type="text"` internally, so there is no public `type`
  prop to configure.
- Use `defaultVisible` to set the initial reveal state. Use `onVisibleChange` when the parent should
  react to reveal/hide events.
- Use `size` on `PasswordInput` to resize the whole control. The grouped shell, input, and toggle
  stay aligned automatically.
- `className` styles the grouped shell, not the nested input.

`PasswordInput` is designed to work naturally inside `Field`:

```tsx
<Field validationMode="onBlur">
  <FieldLabel>Password</FieldLabel>
  <PasswordInput required autoComplete="current-password" />
  <FieldError match="valueMissing">Please enter your password.</FieldError>
</Field>
```

### Defaults

| Prop                     | Default                                            | Notes                                                       |
| ------------------------ | -------------------------------------------------- | ----------------------------------------------------------- |
| `size`                   | `md`                                               | Supports `xs`, `sm`, `md`, `lg`, and `xl`.                  |
| `defaultVisible`         | `false`                                            | Starts the field masked unless you opt into reveal-on-load. |
| `visibilityToggleLabels` | `{ show: "Show password", hide: "Hide password" }` | Accessible toggle labels for the two states.                |
| toggle button `type`     | `button`                                           | Prevents accidental form submission.                        |

## Examples

### Controlled Value

Use `value` and `onValueChange` when the password value should stay in React state.

<Preview>
  <ControlledPasswordInputExample />

  <Preview.Code>
    {`
      import { Field, FieldLabel, PasswordInput } from "moduix";
      import { useState } from "react";

      export function ControlledPasswordInputDemo() {
        const [value, setValue] = useState("");

        return (
          <Field className={styles.field}>
            <FieldLabel>Workspace password</FieldLabel>
            <PasswordInput
              autoComplete="new-password"
              value={value}
              onValueChange={setValue}
              placeholder="Type to control value"
            />
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Styling

`PasswordInput` does not introduce a separate `--password-input-*` token family. Style it with the
inherited grouped-field variables:

- `--input-group-*` for shell, spacing, icon, separator, and toggle-adjacent styling
- `className` on `PasswordInput` when the whole grouped control should change
- targeted selectors for `data-slot="password-input-toggle"` or `data-slot="input-group-input"` when
  one inner part needs additional local treatment

The most common customization path is to change border, focus ring, radius, button color, and icon
size at the grouped shell level instead of styling the inner input separately.
