## API Reference

Ark UI does not ship a dedicated `table` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Basic

Start with `Table.ScrollArea`, `Table`, `Table.Header`, `Table.Body`, and the native row and cell
parts. This is the recommended path for invoices, summaries, and admin-style read-only datasets.

## Anatomy

The normal consumer path stays close to native table markup:

```text
Table.ScrollArea (optional)
└─ Table
   ├─ Table.ColumnGroup (optional)
   │  └─ Table.Column
   ├─ Table.Caption (optional)
   ├─ Table.Header
   │  └─ Table.Row
   │     └─ Table.ColumnHeader
   ├─ Table.Body
   │  ├─ Table.Row
   │  │  └─ Table.Cell
   │  └─ Table.Empty (optional)
   └─ Table.Footer (optional)
```

| Part                   | Role                                                                       |
| ---------------------- | -------------------------------------------------------------------------- |
| `Table` / `Table.Root` | Native `table` root. Owns typography, dividers, and state attributes.      |
| `Table.ScrollArea`     | Optional bordered overflow shell for the common “table in a surface” case. |
| `Table.ColumnGroup`    | Native `colgroup` wrapper for column width distribution.                   |
| `Table.Column`         | Native `col` wrapper. Supports `htmlWidth`.                                |
| `Table.Caption`        | Optional label above or below the table via `side`.                        |
| `Table.Header`         | Native `thead` wrapper for column headers.                                 |
| `Table.Body`           | Native `tbody` wrapper for body rows.                                      |
| `Table.Footer`         | Native `tfoot` wrapper for summary rows.                                   |
| `Table.Row`            | Native `tr` wrapper with divider behavior.                                 |
| `Table.ColumnHeader`   | Native `th` wrapper. Supports `numeric`.                                   |
| `Table.Cell`           | Native `td` wrapper. Supports `numeric`.                                   |
| `Table.Empty`          | Convenience empty-state row inside `Table.Body`.                           |

## Composition

### Recommended composition

Stay with `Table.ScrollArea` plus the semantic table parts for most read-only data views.

Use `Table.ScrollArea` when you need the common bordered horizontal-scroll shell. Use `numeric` on
`Table.ColumnHeader` and `Table.Cell` for amounts, counts, percentages, and latency columns. Use
`Table.Caption` when the table needs a short label or summary, and `Table.Footer` when the last row
represents totals or rollups. Use `striped`, `interactive`, `showColumnBorder`, `stickyHeader`,
and `size` / `variant` when the table needs Chakra-style visual modes without adding data-grid
state.

### When to go custom

Skip `Table.ScrollArea` when the surrounding layout already owns scrolling, padding, border, or
surface styling. In that case, keep `Table` itself and move the wrapper concerns into your own
layout component or shell markup.

`Table` intentionally does not own sorting, filtering, selection, pagination, or row action state.
Treat it as the visual and semantic primitive that higher-level table abstractions can build on.

### Default props

All parts accept `className` and their matching native element props.

| Component            | Prop               | Default    | Notes                                                  |
| -------------------- | ------------------ | ---------- | ------------------------------------------------------ |
| `Table`              | `interactive`      | `false`    | Enables body-row hover highlighting.                   |
| `Table`              | `showColumnBorder` | `false`    | Adds borders between visible columns.                  |
| `Table`              | `size`             | `'md'`     | Supports `'sm'`, `'md'`, and `'lg'`.                   |
| `Table`              | `stickyHeader`     | `false`    | Makes `Table.Header` sticky inside a scroll container. |
| `Table`              | `striped`          | `false`    | Zebra-stripes even body rows, excluding empty rows.    |
| `Table`              | `variant`          | `'line'`   | Supports `'line'` and `'outline'`.                     |
| `Table.Caption`      | `side`             | `'bottom'` | Supports `'top'` and `'bottom'`.                       |
| `Table.Column`       | `htmlWidth`        | —          | Maps to the native `width` attribute on `col`.         |
| `Table.ColumnHeader` | `numeric`          | `false`    | Right-aligns content and uses tabular nums.            |
| `Table.Cell`         | `numeric`          | `false`    | Right-aligns content and uses tabular nums.            |
| `Table.Empty`        | `colSpan`          | required   | Must match the current body column count.              |

### Behavioral notes

- `Table.Empty` renders a full table row for the empty state, so pass the body column count through
  `colSpan`.
- Header and body rows render a shared bottom divider. The last body row drops its border.
- Body rows get a subtle hover background only when `interactive` is set.
- `Table.Caption` inherits muted typography and adds inline padding so it does not sit flush against
  the table edges.

totals or summaries.

## Styling

Style `Table` through `className`, the public `data-slot` hooks, `data-side` and `data-numeric`
attributes, and the `--table-*` CSS variables shown below.

### Styling hooks

Use the part that owns the visual concern:

- `className` on `Table.ScrollArea` for shell-wide border, radius, and surface overrides
- `className` on `Table` for table typography and width constraints
- `data-slot="table-root"`, `table-scroll-area`, `table-column-group`, `table-column`,
  `table-caption`, `table-header`, `table-body`, `table-footer`, `table-row`,
  `table-column-header`, `table-cell`, and `table-empty` for part-level targeting
- `data-side="top" | "bottom"` on `Table.Caption` for caption placement styling
- `data-numeric` on `Table.ColumnHeader` and `Table.Cell` for numeric-column targeting
- `data-size`, `data-variant`, `data-striped`, `data-interactive`, `data-sticky-header`, and
  `data-show-column-border` on `Table` for root-level state styling
- `data-sticky="start" | "end"` on `Table.ColumnHeader` and `Table.Cell` for sticky columns
