## API Reference

`Table` is a native semantic primitive, not a Base UI wrapper. moduix keeps the underlying table
markup intact and only adds a few small high-frequency conveniences:

- `TableContainer` for the common bordered horizontal-scroll shell;
- `numeric` on `TableHead` and `TableCell` for right-aligned numeric columns;
- `TableEmpty` for empty-state rows;
- `TableCaption side="top" | "bottom"` for short table labels.

## Anatomy

```text
TableContainer (optional)
└─ Table
   ├─ TableCaption (optional)
   ├─ TableHeader
   │  └─ TableRow
   │     └─ TableHead
   ├─ TableBody
   │  ├─ TableRow
   │  │  └─ TableCell
   │  └─ TableEmpty (optional)
   └─ TableFooter (optional)
```

| Part             | Role                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------ |
| `TableContainer` | Optional bordered overflow shell for the common “table in a card-like surface” case. |
| `Table`          | Native `table` root. Owns table typography, dividers, and hover styling.             |
| `TableCaption`   | Optional label above or below the table via `side`.                                  |
| `TableHeader`    | Native `thead` wrapper for column headers.                                           |
| `TableBody`      | Native `tbody` wrapper for body rows.                                                |
| `TableFooter`    | Native `tfoot` wrapper for summary rows.                                             |
| `TableRow`       | Native `tr` wrapper with row divider and hover behavior.                             |
| `TableHead`      | Native `th` wrapper. Supports `numeric`.                                             |
| `TableCell`      | Native `td` wrapper. Supports `numeric`.                                             |
| `TableEmpty`     | Convenience empty-state row inside `TableBody`.                                      |

## Public Props

All parts accept `className` and their matching native element props.

Small sugar props:

| Component      | Prop      | Type                | Default    |
| -------------- | --------- | ------------------- | ---------- |
| `TableCaption` | `side`    | `'top' \| 'bottom'` | `'bottom'` |
| `TableHead`    | `numeric` | `boolean`           | `false`    |
| `TableCell`    | `numeric` | `boolean`           | `false`    |
| `TableEmpty`   | `colSpan` | `number`            | required   |

## Composition

Keep the mental model simple:

- use `TableContainer` when you want the common overflow shell;
- skip `TableContainer` when the surrounding layout already owns the wrapper;
- use `numeric` for amounts, counts, percentages, and latency columns;
- use `TableFooter` for totals or rollups;
- use `TableEmpty` when a body would otherwise render no rows.

`Table` intentionally does **not** own sorting, filtering, selection, sticky headers, pagination, or
column sizing. Treat it as the visual/semantic layer that future higher-level tables can build on.
