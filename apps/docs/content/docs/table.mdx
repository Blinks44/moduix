## API Reference

`Table` does not wrap a dedicated Ark UI primitive. It is a native semantic table wrapper with moduix
styling, `data-slot` hooks, and a few small convenience parts for the common read-only table path.

## Basic

Start with `TableContainer`, `Table`, `TableHeader`, `TableBody`, and the native row and cell
parts. This is the recommended path for invoices, summaries, and admin-style read-only datasets.

## Anatomy

The normal consumer path stays close to native table markup:

```text
TableContainer (optional)
└─ Table
   ├─ TableCaption (optional)
   ├─ TableHeader
   │  └─ TableRow
   │     └─ TableHead
   ├─ TableBody
   │  ├─ TableRow
   │  │  └─ TableCell
   │  └─ TableEmpty (optional)
   └─ TableFooter (optional)
```

| Part             | Role                                                                       |
| ---------------- | -------------------------------------------------------------------------- |
| `TableContainer` | Optional bordered overflow shell for the common “table in a surface” case. |
| `Table`          | Native `table` root. Owns typography, dividers, and hover styling.         |
| `TableCaption`   | Optional label above or below the table via `side`.                        |
| `TableHeader`    | Native `thead` wrapper for column headers.                                 |
| `TableBody`      | Native `tbody` wrapper for body rows.                                      |
| `TableFooter`    | Native `tfoot` wrapper for summary rows.                                   |
| `TableRow`       | Native `tr` wrapper with divider and hover behavior.                       |
| `TableHead`      | Native `th` wrapper. Supports `numeric`.                                   |
| `TableCell`      | Native `td` wrapper. Supports `numeric`.                                   |
| `TableEmpty`     | Convenience empty-state row inside `TableBody`.                            |

## Composition

### When to stay high-level

Stay with `TableContainer` plus the semantic table parts for most read-only data views.

Use `TableContainer` when you need the common bordered horizontal-scroll shell. Use `numeric` on
`TableHead` and `TableCell` for amounts, counts, percentages, and latency columns. Use
`TableCaption` when the table needs a short label or summary, and `TableFooter` when the last row
represents totals or rollups. Use `striped` when long read-only bodies need stronger row-by-row
scanability.

### When to go custom

Skip `TableContainer` when the surrounding layout already owns scrolling, padding, border, or
surface styling. In that case, keep `Table` itself and move the wrapper concerns into your own
layout component or shell markup.

`Table` intentionally does not own sorting, filtering, selection, sticky headers, pagination, or
column sizing. Treat it as the visual and semantic primitive that higher-level table abstractions
can build on.

### Default props

All parts accept `className` and their matching native element props.

| Component      | Prop      | Default    | Notes                                               |
| -------------- | --------- | ---------- | --------------------------------------------------- |
| `Table`        | `striped` | `false`    | Zebra-stripes even body rows, excluding empty rows. |
| `TableCaption` | `side`    | `'bottom'` | Supports `'top'` and `'bottom'`.                    |
| `TableHead`    | `numeric` | `false`    | Right-aligns content and uses tabular nums.         |
| `TableCell`    | `numeric` | `false`    | Right-aligns content and uses tabular nums.         |
| `TableEmpty`   | `colSpan` | required   | Must match the current body column count.           |

### Behavioral notes

- `TableEmpty` renders a full table row for the empty state, so pass the body column count through
  `colSpan`.
- Header and body rows render a shared bottom divider. The last body row drops its border.
- Body rows get a subtle hover background by default.
- `TableCaption` inherits muted typography and adds inline padding so it does not sit flush against
  the table edges.

## Styling

Style `Table` through `className`, the public `data-slot` hooks, `data-side` and `data-numeric`
attributes, and the `--table-*` CSS variables shown below.

### CSS Properties

  <Tabs items={['CSS Variables']} className="mb-0 mt-3">
    <Tab className="max-h-150 overflow-auto">
      <TableCssPropertiesPanel />
    </Tab>

### Styling hooks

Use the part that owns the visual concern:

- `className` on `TableContainer` for shell-wide border, radius, and surface overrides
- `className` on `Table` for table typography and width constraints
- `data-slot="table-container"`, `table`, `table-caption`, `table-header`, `table-body`,
  `table-footer`, `table-row`, `table-head`, `table-cell`, and `table-empty` for part-level
  targeting
- `data-side="top" | "bottom"` on `TableCaption` for caption placement styling
- `data-numeric` on `TableHead` and `TableCell` for numeric-column targeting
- `data-striped` on `Table` when zebra striping is enabled
