## API Reference

`InputGroup` is a standalone moduix component and does not wrap a Base UI primitive.
Use it when an input needs a shared border, focus ring, prefix, suffix, inline text, or action.

## Anatomy

`InputGroup` controls the visual shell. Place `InputGroupInput` once inside it, then compose
addons, text, and buttons before or after the input.

```text
InputGroup
├─ InputGroupAddon (optional)
├─ InputGroupInput
├─ InputGroupText (optional)
└─ InputGroupButton (optional)
```

| Part               | Role                                                               |
| ------------------ | ------------------------------------------------------------------ |
| `InputGroup`       | Root slot. Applies size, border, background, focus, and state.     |
| `InputGroupInput`  | Input slot. Renders the moduix `Input` without its own border.     |
| `InputGroupAddon`  | Prefix or suffix surface with separator and optional icon content. |
| `InputGroupText`   | Inline prefix or suffix text without filled addon background.      |
| `InputGroupButton` | Inline action slot based on the moduix `Button`.                   |

## Composition

Use `size` on `InputGroup` to keep all parts aligned. `InputGroupInput` accepts the same props as
`Input`, including `value`, `defaultValue`, `onValueChange`, `type`, native attributes, and Field
validation behavior.

Pass `className` to the root or individual parts for targeted overrides. Use CSS variables for
theme-level changes such as border color, addon background, padding, radius, and focus ring.

`InputGroupButton` defaults to `type="button"` and `variant="ghost"` so it can be used inside forms
without submitting them accidentally. Override `type`, `variant`, or `size` when a form submit or
stronger action is intended.

## Examples

### With Action

Use `InputGroupButton` for compact actions that operate on the current input value.

<Preview>
  <InputGroupWithActionExample />

  <Preview.Code>
    {`
      import { Field, FieldLabel, InputGroup, InputGroupButton, InputGroupInput } from "moduix";
      import { useState } from "react";

      export function InputGroupWithActionDemo() {
        const [value, setValue] = useState("");

        return (
          <Field className={styles.field}>
            <FieldLabel>Invite by email</FieldLabel>
            <InputGroup>
              <InputGroupInput
                value={value}
                onValueChange={setValue}
                type="email"
                placeholder="name@example.com"
              />
              <InputGroupButton disabled={!value}>Send</InputGroupButton>
            </InputGroup>
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Prefix And Suffix

Mix `InputGroupAddon` and `InputGroupText` when one side should feel like a fixed token and the
other should read as quiet helper text.

### Field Validation

`InputGroupInput` keeps the Base UI input state, so Field validation can style the group as invalid.

<Preview>
  <InputGroupFieldValidationExample />

  <Preview.Code>
    {`
      import { Field, FieldError, FieldLabel, InputGroup, InputGroupInput, InputGroupText } from "moduix";

      export function InputGroupFieldValidationDemo() {
        return (
          <Field className={styles.field} validationMode="onBlur">
            <FieldLabel>Domain</FieldLabel>
            <InputGroup>
              <InputGroupInput required placeholder="company" />
              <InputGroupText>.2gis.com</InputGroupText>
            </InputGroup>
            <FieldError match="valueMissing">Please enter a domain.</FieldError>
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>
