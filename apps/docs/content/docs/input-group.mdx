## API Reference

`InputGroup` does not wrap a dedicated Base UI primitive. It is a moduix composition helper built
from the local `Input` and `Button` components, so the public API lives on these exported parts:

- `InputGroup`
- `InputGroupInput`
- `InputGroupAddon`
- `InputGroupText`
- `InputGroupButton`

## Overview

`InputGroup` keeps the public surface short:

- `InputGroup` renders the shared field shell
- `InputGroupInput` renders the actual `Input`
- `InputGroupAddon`, `InputGroupText`, and `InputGroupButton` decorate the input inline

The group is composition-first. You place one `InputGroupInput` inside the shell, then add the
parts you need before or after it.

## Anatomy

```text
InputGroup
├─ InputGroupAddon (optional)
├─ InputGroupInput
├─ InputGroupText (optional)
└─ InputGroupButton (optional)
```

| Part               | Role                                                     |
| ------------------ | -------------------------------------------------------- |
| `InputGroup`       | Shared shell for border, radius, focus ring, and sizing. |
| `InputGroupInput`  | The `Input` rendered inside the shell.                   |
| `InputGroupAddon`  | Filled prefix or suffix surface with a separator.        |
| `InputGroupText`   | Plain inline text without the filled addon background.   |
| `InputGroupButton` | Inline action based on `Button`.                         |

## Composition

Use one `InputGroupInput` per group. The shell click-to-focus behavior targets the first grouped
input, so multiple grouped inputs are not a supported composition.

Addons, helper text, and buttons can sit on either side of the input depending on the flow you
need.

Set `size` on `InputGroup` to keep the whole row aligned. `InputGroupInput` and
`InputGroupButton` inherit that size automatically, so the default path stays short. Pass an
explicit `size` to either part only when one child should intentionally differ from the group.

Clicking the shell focuses the input, which keeps prefix and suffix layouts feeling like one field
instead of separate controls. That focus redirect is skipped for `disabled` and `readOnly` grouped
inputs.

`InputGroupInput` accepts the same props as `Input`, including controlled values, native input
attributes, and Base UI field validation state.

`InputGroupButton` defaults to `variant="ghost"` and `type="button"`, so inline actions work
inside forms without submitting them accidentally.

For standalone widgets that combine an input and action into one logical unit, add `role="group"`
with `aria-label` or `aria-labelledby` on `InputGroup`. When the group lives inside `Field`, prefer
`FieldLabel` and keep `InputGroup` purely visual.

### Defaults

| Part / Prop                | Default              | Notes                                                     |
| -------------------------- | -------------------- | --------------------------------------------------------- |
| `InputGroup.size`          | `md`                 | Supports `xs`, `sm`, `md`, `lg`, and `xl`.                |
| `InputGroupInput.size`     | inherited from group | Override only when one input should intentionally differ. |
| `InputGroupButton.variant` | `ghost`              | Accepts the same variants as `Button`.                    |
| `InputGroupButton.size`    | inherited from group | Override for icon or mixed-density actions.               |
| `InputGroupButton.type`    | `button`             | Use `submit` or `reset` when the action should do that.   |

## Examples

### With Action

Use `InputGroupButton` for compact inline actions tied to the current field value.

<Preview>
  <InputGroupWithActionExample />

  <Preview.Code>
    {`
      import { Field, FieldLabel, InputGroup, InputGroupButton, InputGroupInput } from "moduix";
      import { useState } from "react";

      export function InputGroupWithActionDemo() {
        const [value, setValue] = useState("");

        return (
          <Field className={styles.field}>
            <FieldLabel>Invite by email</FieldLabel>
            <InputGroup>
              <InputGroupInput
                value={value}
                onValueChange={setValue}
                type="email"
                placeholder="name@example.com"
              />
              <InputGroupButton disabled={!value}>Send</InputGroupButton>
            </InputGroup>
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Inline Editing

For single-line inline editing, keep one mounted `InputGroupInput`, switch it between `readOnly` and
editable modes, and swap only the action buttons. This avoids layout shift and keeps focus handling
simple. If you want the read-only state to look flatter or more text-like, override the
`--input-group-readonly-*` shell variables instead of styling only the nested input.

<Preview>
  <InputGroupInlineEditingExample />

  <Preview.Code>
    {`
      import {
        CheckIcon,
        CloseIcon,
        Field,
        FieldLabel,
        InputGroup,
        InputGroupButton,
        InputGroupInput,
        PencilIcon,
      } from "moduix";
      import { useEffect, useRef, useState } from "react";

      export function InlineEditingInputGroupDemo() {
        const [editing, setEditing] = useState(false);
        const [value, setValue] = useState("Workspace display name");
        const [draft, setDraft] = useState(value);
        const inputRef = useRef(null as HTMLInputElement | null);

        useEffect(() => {
          if (editing) {
            inputRef.current?.focus();
          }
        }, [editing]);

        const handleEdit = () => {
          setDraft(value);
          setEditing(true);
        };

        const handleCancel = () => {
          setDraft(value);
          setEditing(false);
        };

        const handleSave = () => {
          setValue(draft);
          setEditing(false);
        };

        return (
          <Field className={styles.field}>
            <FieldLabel>Display name</FieldLabel>
            <InputGroup>
              <InputGroupInput
                ref={inputRef}
                aria-label="Display name"
                readOnly={!editing}
                value={editing ? draft : value}
                onValueChange={setDraft}
              />
              {editing ? (
                <>
                  <InputGroupButton aria-label="Cancel editing" size="icon-md" onClick={handleCancel}>
                    <CloseIcon />
                  </InputGroupButton>
                  <InputGroupButton aria-label="Save changes" size="icon-md" onClick={handleSave}>
                    <CheckIcon />
                  </InputGroupButton>
                </>
              ) : (
                <InputGroupButton aria-label="Edit display name" size="icon-md" onClick={handleEdit}>
                  <PencilIcon />
                </InputGroupButton>
              )}
            </InputGroup>
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Field Validation

`InputGroupInput` keeps the Base UI input state, so `Field` validation can style the whole group as
invalid.

<Preview>
  <InputGroupFieldValidationExample />

  <Preview.Code>
    {`
      import { Field, FieldError, FieldLabel, InputGroup, InputGroupInput, InputGroupText } from "moduix";

      export function InputGroupFieldValidationDemo() {
        return (
          <Field className={styles.field} validationMode="onBlur">
            <FieldLabel>Domain</FieldLabel>
            <InputGroup>
              <InputGroupInput required placeholder="company" />
              <InputGroupText>.test.com</InputGroupText>
            </InputGroup>
            <FieldError match="valueMissing">Please enter a domain.</FieldError>
          </Field>
        );
      }
    `}

  </Preview.Code>
</Preview>
