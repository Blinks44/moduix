## API Reference

Ark UI does not ship a dedicated input-group primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance

## Choosing the right component

- Choose **`Input`** for a plain single-line field.
- Choose **`InputGroup`** for inline prefixes, suffixes, text, or actions.
- Choose **`Editable`** for preview-first text that switches into edit mode.
- Choose **`PasswordInput`** for the existing password visibility composition.
- Choose **`NumberInput`** when numeric behavior matters.

## Anatomy

```text
InputGroup / InputGroup.Root
├─ InputGroupAddon / InputGroup.Addon (optional)
├─ InputGroupInput / InputGroup.Input
├─ InputGroupText / InputGroup.Text (optional)
└─ InputGroupButton / InputGroup.Button (optional)
```

| Part                             | Role                                                         |
| -------------------------------- | ------------------------------------------------------------ |
| `InputGroup` / `InputGroup.Root` | Ark factory root for shared layout, border, focus, and size. |
| `InputGroupInput` / `.Input`     | Ark-backed native input and form value owner.                |
| `InputGroupAddon` / `.Addon`     | Filled presentational prefix or suffix.                      |
| `InputGroupText` / `.Text`       | Unfilled inline presentational text.                         |
| `InputGroupButton` / `.Button`   | Native button action with grouped visual defaults.           |

## Composition

### Recommended composition

Use exactly one `InputGroupInput`. Place addons, text, or buttons before or after it. Put the
composition inside `Field` when it needs a label, helper text, error text, ids, or shared state.

`InputGroupInput` uses native input events. Controlled values use `value` and `onChange(event)`.
The root `size` is inherited by the input and button.

### When to go custom

`InputGroup`, `InputGroupAddon`, and `InputGroupText` support Ark `asChild`. The root child must
remain a container; addon and text children should remain presentational and accept forwarded props
and refs.

Use `InputGroupButton` for actions instead of making an addon interactive. Use `role="group"` with
an accessible name only when the standalone composition itself needs grouped semantics.

### Default props

| Part / prop                | Default             |
| -------------------------- | ------------------- |
| `InputGroup.size`          | `md`                |
| `InputGroupInput.size`     | inherited from root |
| `InputGroupButton.size`    | inherited from root |
| `InputGroupButton.variant` | `ghost`             |
| `InputGroupButton.type`    | `button`            |

### Behavioral notes

- The input ref targets the real `HTMLInputElement`.
- The root does not own value, validation, ARIA ids, context hooks, or a `HiddenInput`.
- Root invalid, disabled, and read-only visuals derive from the nested input state.

## Examples

### With Action

Use a grouped button for an action tied to the native input value.

<Preview>
  <InputGroupWithActionExample />

  <Preview.Code>
    {`
      import { Field, InputGroup, InputGroupButton, InputGroupInput } from "@moduix/react";
      import { useState } from "react";

      const initialInviteEmail = "";

      export function InputGroupWithActionDemo() {
        const [value, setValue] = useState(initialInviteEmail);

        return (
          <Field className="input-group-demo-field">
            <Field.Label>Invite by email</Field.Label>
            <InputGroup>
              <InputGroupInput
                value={value}
                onChange={(event) => setValue(event.currentTarget.value)}
                type="email"
                placeholder="name@example.com"
              />
              <InputGroupButton disabled={!value}>Send</InputGroupButton>
            </InputGroup>
          </Field>
        );
      }
    `}

  </Preview.Code>

<Preview.CSS>
  {`
      .input-group-demo-field {
        width: min(16rem, 100%);
      }
    `}
</Preview.CSS>

  <Preview.Data>
    {`
      const initialInviteEmail = "";
    `}
  </Preview.Data>
</Preview>

### Styling hooks

- `data-slot="input-group-root"` and `data-size`
- `data-scope="input-group"` and `data-part="root" | "addon" | "text"` on factory parts
- `data-slot="input-group-input"` plus Ark Field input state attributes
- `data-slot="input-group-addon"`
- `data-slot="input-group-text"`
- `data-slot="input-group-button"` on the grouped action button
- nested `data-invalid`, `aria-invalid`, `disabled`, and `readOnly` state through root `:has(...)`
