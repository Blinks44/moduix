## Anatomy

Checkbox Group uses one shared root state with optional label and list wrappers.
`CheckboxGroupLabel` auto-labels the nearest `CheckboxGroup` when you place it
inside the same tree. Keep `CheckboxGroupItemControl` and `CheckboxGroupItemLabel`
inside the same default `CheckboxGroupItem` label wrapper for the compact
enclosing-label pattern.

```text
CheckboxGroup
├─ CheckboxGroupLabel
└─ CheckboxGroupList
   └─ CheckboxGroupItem
      ├─ CheckboxGroupItemControl
      │  └─ CheckboxIndicator
      │     └─ CheckboxIndicatorIcon
      └─ CheckboxGroupItemLabel
```

```tsx
<CheckboxGroup defaultValue={['email']}>
  <CheckboxGroupLabel>Notification Channels</CheckboxGroupLabel>
  <CheckboxGroupList>
    <CheckboxGroupItem>
      <CheckboxGroupItemControl value="email" name="notifications" />
      <CheckboxGroupItemLabel>Email updates</CheckboxGroupItemLabel>
    </CheckboxGroupItem>
  </CheckboxGroupList>
</CheckboxGroup>
```

| Part                       | Role                                                                                |
| -------------------------- | ----------------------------------------------------------------------------------- |
| `CheckboxGroup`            | Root state. Manages selected values, `disabled`, `allValues`, and change callbacks. |
| `CheckboxGroupLabel`       | Optional group heading. Auto-connects to the nearest `CheckboxGroup`.               |
| `CheckboxGroupList`        | Optional list wrapper for checkbox rows.                                            |
| `CheckboxGroupItem`        | Optional row wrapper for one control + one text label. Defaults to `<label>`.       |
| `CheckboxGroupItemControl` | Interactive checkbox control for an item.                                           |
| `CheckboxGroupItemLabel`   | Optional text label for the item.                                                   |

`CheckboxGroup` does not use service slots (`portal`, `backdrop`, `viewport`).
Customize visible parts with `className`, `classNames`, and CSS variables.

## Composition

Use `CheckboxGroup` for shared state (`value`, `defaultValue`, `onValueChange`) and
group behavior (`allValues`, `disabled`, `render`, `className`, state-aware `style`).
Use `allValues` with `CheckboxGroupItemControl parent` for select-all behavior.

| Prop                                    | Default     | Values                                                                  |
| --------------------------------------- | ----------- | ----------------------------------------------------------------------- |
| `defaultValue`                          | `[]`        | `string[]`                                                              |
| `disabled`                              | `false`     | `boolean`                                                               |
| `value`                                 | `undefined` | `string[]` (controlled mode, with `onValueChange`)                      |
| `className`                             | `undefined` | `string \| ((state) => string \| undefined)`                            |
| `allValues`                             | `undefined` | `string[]` (required for parent/select-all checkbox patterns)           |
| `CheckboxGroupItem.render`              | `undefined` | `ReactElement` (replaces the default `<label>` wrapper)                 |
| `CheckboxGroupItemControl.size`         | `md`        | `xs`, `sm`, `md`, `lg`, `xl`                                            |
| `CheckboxGroupItemControl.parent`       | `false`     | `boolean` (enables parent checkbox behavior when group has `allValues`) |
| `CheckboxGroupItemControl.nativeButton` | `false`     | `boolean`                                                               |

`CheckboxGroupLabel`, `CheckboxGroupList`, `CheckboxGroupItem`, and
`CheckboxGroupItemLabel` are layout slots and accept `className`.
`CheckboxGroupItemControl` composes `Checkbox`, so it also supports the same
checkbox customization props, refs, and form props described on the Checkbox
page. Inside a group,
prefer letting `CheckboxGroup` own checked state rather than mixing in per-item
`checked` or `defaultChecked`.

Use `className` for the checkbox root, `classNames` for internal indicator slots, and
`slotProps.indicator` for Base UI indicator props like `keepMounted`:

```tsx
<CheckboxGroupItemControl
  className={styles.customControl}
  classNames={{
    indicator: styles.customIndicator,
    indicatorIcon: styles.customIndicatorIcon,
    checkedIcon: styles.customCheckedIcon,
    indeterminateIcon: styles.customIndeterminateIcon,
  }}
  slotProps={{
    indicator: { keepMounted: true },
  }}
/>
```

### Controlled

Control `value` from React state when the selected options need to coordinate with other UI.

<Preview>
  <ControlledCheckboxGroupExample />

  <Preview.Code>

        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCheckboxGroupDemo() {
        const [value, setValue] = useState(["push"] as string[]);

        return (
          <div className={styles.wrapper}>

              <CheckboxGroupLabel>Active Alerts</CheckboxGroupLabel>

              </CheckboxGroupList>
            </CheckboxGroup>

### Parent Checkbox

Use `allValues` with a `parent` checkbox to create a select-all control that enters the
indeterminate state when only some child options are checked.

<Preview>
  <ParentCheckboxGroupExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";

      export function ParentCheckboxGroupDemo() {
        const [value, setValue] = useState([] as string[]);

        return (
          <CheckboxGroup value={value} onValueChange={setValue} allValues={fruitValues}>
            <CheckboxGroupLabel>Fruits</CheckboxGroupLabel>
            <CheckboxGroupList>
              <CheckboxGroupItem>
                <CheckboxGroupItemControl parent />
                <CheckboxGroupItemLabel>Select all</CheckboxGroupItemLabel>
              </CheckboxGroupItem>

              {options.map((option) => (
                <CheckboxGroupItem key={option.value}>
                  <CheckboxGroupItemControl value={option.value} />
                  <CheckboxGroupItemLabel>{option.label}</CheckboxGroupItemLabel>
                </CheckboxGroupItem>
              ))}
            </CheckboxGroupList>
          </CheckboxGroup>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const options = [
        { value: "apple", label: "Apple" },
        { value: "orange", label: "Orange" },
        { value: "pear", label: "Pear" },
      ];

      const fruitValues = options.map((option) => option.value);
    `}

  </Preview.Data>
</Preview>

### Custom Item Wrapper

Use `CheckboxGroupItem render` when the row should not be a `<label>`, such as
button-based or custom grid layouts. In that case, use `nativeButton` and a
sibling `<label htmlFor>` if clicking the text should still toggle the checkbox.

<Preview>
  <CheckboxGroupCustomItemWrapperExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";

      export function CheckboxGroupCustomItemWrapperDemo() {
        return (
          <CheckboxGroup defaultValue={["email"]}>
            <CheckboxGroupLabel>Channels</CheckboxGroupLabel>
            <CheckboxGroupList>
              {options.map((option) => (
                <CheckboxGroupItem key={option.value} render={<div />}>
                  <CheckboxGroupItemControl
                    nativeButton
                    render={<button />}
                    id={\`custom-item-\${option.value}\`}
                    value={option.value}
                    name="custom-item-wrapper"
                  />
                  <label htmlFor={\`custom-item-\${option.value}\`}>{option.label}</label>
                </CheckboxGroupItem>
              ))}
            </CheckboxGroupList>
          </CheckboxGroup>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const options = [
        { value: "email", label: "Email updates" },
        { value: "push", label: "Push notifications" },
        { value: "sms", label: "SMS alerts" },
      ];
    `}

</Preview>
