## Basic

Use `CheckboxGroup` for shared selection state and the `CheckboxGroupItem*` parts for the default
row layout. `CheckboxGroupItem` and `CheckboxGroupItemLabel` are thin convenience wrappers over
`CheckboxField` and `CheckboxLabel`.

<Preview>
  <CheckboxGroupExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useId } from "react";

      export function CheckboxGroupDemo() {
        const labelId = useId();

        return (
          <CheckboxGroup defaultValue={["email"]} aria-labelledby={labelId}>
            <CheckboxGroupLabel id={labelId}>
              Notification Channels
            </CheckboxGroupLabel>
            <CheckboxGroupList>
              {options.map((option) => (
                <CheckboxGroupItem key={option.value}>
                  <CheckboxGroupItemControl value={option.value} name="notifications" />
                  <CheckboxGroupItemLabel>{option.label}</CheckboxGroupItemLabel>
                </CheckboxGroupItem>
              ))}
            </CheckboxGroupList>
          </CheckboxGroup>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>

## Anatomy

The normal consumer path is short:

```text
CheckboxGroup
├─ CheckboxGroupLabel
└─ CheckboxGroupList
   └─ CheckboxGroupItem
      ├─ CheckboxGroupItemControl
      │  └─ CheckboxIndicator
      │     └─ CheckboxIndicatorIcon
      └─ CheckboxGroupItemLabel
```

Use `aria-labelledby` on `CheckboxGroup` when `CheckboxGroupLabel` is the visible group label.
Keep `CheckboxGroupItemControl` and `CheckboxGroupItemLabel` inside `CheckboxGroupItem` for the
default enclosing-label row.

| Part                       | Role                                                                                 |
| -------------------------- | ------------------------------------------------------------------------------------ |
| `CheckboxGroup`            | Root state. Manages selected values, `allValues`, disabled state, and form behavior. |
| `CheckboxGroupLabel`       | Optional visible group label. Pair its `id` with `aria-labelledby`.                  |
| `CheckboxGroupList`        | Optional layout wrapper for checkbox rows.                                           |
| `CheckboxGroupItem`        | Default row wrapper for one control and one text label. Built on `CheckboxField`.    |
| `CheckboxGroupItemControl` | Interactive checkbox control for an item or parent/select-all row.                   |
| `CheckboxGroupItemLabel`   | Default item label. Built on `CheckboxLabel`.                                        |

## Composition

### When to stay high-level

Stay with `CheckboxGroup`, `CheckboxGroupLabel`, `CheckboxGroupList`, and the
`CheckboxGroupItem*` parts for most settings panels, filter groups, and multi-select form fields.

`CheckboxGroup` keeps Base UI group behavior intact. Use it for `value`, `defaultValue`,
`onValueChange`, `allValues`, `disabled`, `required`, `name`, `form`, `aria-labelledby`,
`className`, and `render`.

`CheckboxGroupItemControl` composes `Checkbox`, so it inherits the checkbox indicator,
`size`, `readOnly`, `nativeButton`, `inputRef`, `render`, and explicit indicator composition.

### When to go custom

Drop down to `CheckboxField`, `Checkbox`, and `CheckboxLabel` when the row structure should not use
the default `CheckboxGroupItem` wrapper.

Use `allValues` on `CheckboxGroup` together with `parent` on `CheckboxGroupItemControl` for
select-all behavior.

For sibling-label layouts or custom root markup, use `nativeButton` with `render={<button />}` on
`CheckboxGroupItemControl` instead of replacing the whole group composition.

### Default props

| Prop                                    | Default     | Values / Notes                                                                  |
| --------------------------------------- | ----------- | ------------------------------------------------------------------------------- |
| `defaultValue`                          | `[]`        | `string[]`                                                                      |
| `disabled`                              | `false`     | `boolean`                                                                       |
| `value`                                 | `undefined` | Controlled `string[]` used with `onValueChange`                                 |
| `allValues`                             | `undefined` | `string[]`; required for parent/select-all patterns                             |
| `CheckboxGroupItemControl.size`         | `md`        | `xs`, `sm`, `md`, `lg`, `xl`                                                    |
| `CheckboxGroupItemControl.parent`       | `false`     | Enables the parent/select-all checkbox when the group has `allValues`           |
| `CheckboxGroupItemControl.nativeButton` | `false`     | Use with `render={<button />}` for sibling label layouts                        |
| `CheckboxGroupItemControl.readOnly`     | `false`     | Item-level readonly behavior; there is no group-level `readOnly` prop currently |

### Behavioral notes

- Keep checked state on the group with `defaultValue` or `value` plus `onValueChange`.
- Use `disabled` on the group when the whole set is unavailable.
- Use `readOnly` on `CheckboxGroupItemControl` when a specific item must stay visible and focusable
  but should not change.
- The group needs an accessible name. The default path is `CheckboxGroupLabel` plus
  `aria-labelledby` on `CheckboxGroup`.

### Controlled Value

Control `value` from React state when the selected options need to coordinate with surrounding UI.

<Preview>

        CheckboxGroup,

        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";
      import styles from "./checkbox-group-demo.module.css";

### Parent Checkbox

Use `allValues` with a `parent` checkbox to create a select-all control that becomes indeterminate
when only some child options are checked.

<Preview>
  <ParentCheckboxGroupExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";

      export function ParentCheckboxGroupDemo() {
        const [value, setValue] = useState([] as string[]);

        return (
          <CheckboxGroup
            value={value}
            onValueChange={setValue}
            allValues={fruitValues}
            aria-labelledby="fruits"
          >
            <CheckboxGroupLabel id="fruits">Fruits</CheckboxGroupLabel>
            <CheckboxGroupList>
              <CheckboxGroupItem>
                <CheckboxGroupItemControl parent />
                <CheckboxGroupItemLabel>Select all</CheckboxGroupItemLabel>
              </CheckboxGroupItem>

              {options.map((option) => (
                <CheckboxGroupItem key={option.value}>
                  <CheckboxGroupItemControl value={option.value} />
                  <CheckboxGroupItemLabel>{option.label}</CheckboxGroupItemLabel>
                </CheckboxGroupItem>
              ))}
            </CheckboxGroupList>
          </CheckboxGroup>
        );
      }
    `}

## Styling

Use CSS variables on the group and row parts when spacing, typography, or checkbox treatment needs
to change. Use explicit composition when the row structure or indicator markup needs to change.

### Styling hooks

- `CheckboxGroup` exposes `data-slot="checkbox-group-root"` and supports Base UI state callback
  `className`.
- `CheckboxGroupLabel`, `CheckboxGroupList`, `CheckboxGroupItem`, and `CheckboxGroupItemLabel`
  expose stable `data-slot` selectors for layout and typography overrides.
- `CheckboxGroupItemControl` exposes `data-slot="checkbox-group-item-control"` and also inherits
  the checkbox styling hooks, including `data-size`, `data-checked`, `data-unchecked`,
  `data-indeterminate`, `data-disabled`, and `data-readonly`.
- `--checkbox-group-*` variables style the group shell, while `--checkbox-*` variables still apply
  to the nested checkbox controls.
