## Anatomy

Checkbox Group uses one shared root state with optional label and list wrappers.
Use `aria-labelledby` on `CheckboxGroup` when `CheckboxGroupLabel` is the visible
group label. Keep `CheckboxGroupItemControl` and `CheckboxGroupItemLabel` inside
the default `CheckboxGroupItem` label wrapper for the compact enclosing-label
pattern.

```text
CheckboxGroup
├─ CheckboxGroupLabel
└─ CheckboxGroupList
   └─ CheckboxGroupItem
      ├─ CheckboxGroupItemControl
      │  └─ CheckboxIndicator
      │     └─ CheckboxIndicatorIcon
      └─ CheckboxGroupItemLabel
```

```tsx
<CheckboxGroup defaultValue={['email']} aria-labelledby="notification-channels">
  <CheckboxGroupLabel id="notification-channels">Notification Channels</CheckboxGroupLabel>
  <CheckboxGroupList>
    <CheckboxGroupItem>
      <CheckboxGroupItemControl value="email" name="notifications" />
      <CheckboxGroupItemLabel>Email updates</CheckboxGroupItemLabel>
    </CheckboxGroupItem>
  </CheckboxGroupList>
</CheckboxGroup>
```

| Part                       | Role                                                                                |
| -------------------------- | ----------------------------------------------------------------------------------- |
| `CheckboxGroup`            | Root state. Manages selected values, `disabled`, `allValues`, and change callbacks. |
| `CheckboxGroupLabel`       | Optional group heading. Use its `id` with `aria-labelledby` on `CheckboxGroup`.     |
| `CheckboxGroupList`        | Optional list wrapper for checkbox rows.                                            |
| `CheckboxGroupItem`        | Optional row wrapper for one control + one text label. Defaults to `<label>`.       |
| `CheckboxGroupItemControl` | Interactive checkbox control for an item.                                           |
| `CheckboxGroupItemLabel`   | Optional text label for the item.                                                   |

`CheckboxGroup` does not use service slots (`portal`, `backdrop`, `viewport`).
Customize visible parts with `className`, composition, and CSS variables.

## Composition

Use `CheckboxGroup` for shared state (`value`, `defaultValue`, `onValueChange`) and
group behavior (`allValues`, `disabled`). Use `allValues` with
`CheckboxGroupItemControl parent` for select-all behavior.

| Prop                                    | Default     | Values                                                                  |
| --------------------------------------- | ----------- | ----------------------------------------------------------------------- |
| `defaultValue`                          | `[]`        | `string[]`                                                              |
| `disabled`                              | `false`     | `boolean`                                                               |
| `value`                                 | `undefined` | `string[]` (controlled mode, with `onValueChange`)                      |
| `allValues`                             | `undefined` | `string[]` (required for parent/select-all checkbox patterns)           |
| `CheckboxGroupItemControl.size`         | `md`        | `xs`, `sm`, `md`, `lg`, `xl`                                            |
| `CheckboxGroupItemControl.parent`       | `false`     | `boolean` (enables parent checkbox behavior when group has `allValues`) |
| `CheckboxGroupItemControl.nativeButton` | `false`     | `boolean`                                                               |

`CheckboxGroupLabel`, `CheckboxGroupList`, `CheckboxGroupItem`, and
`CheckboxGroupItemLabel` are layout slots and accept `className`.
`CheckboxGroupItemControl` composes `Checkbox`, so it also supports the same
checkbox composition patterns, refs, and form props described on the Checkbox
page. Inside a group,
prefer letting `CheckboxGroup` own checked state rather than mixing in per-item
`checked` or `defaultChecked`.

For sibling labels or custom row layouts, compose `CheckboxGroupItemControl` with
your own wrapper element instead of replacing `CheckboxGroupItem`.

Use `className` for the checkbox root. Compose `CheckboxIndicator` when you need
indicator-specific styling or behavior:

```tsx
<CheckboxGroupItemControl className={styles.customControl}>
  <CheckboxIndicator className={styles.customIndicator} keepMounted />
</CheckboxGroupItemControl>
```

### Sizes

Use `size` on `CheckboxGroupItemControl` to align the controls with different form densities.

### Controlled

Control `value` from React state when the selected options need to coordinate with other UI.

<Preview>
  <ControlledCheckboxGroupExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCheckboxGroupDemo() {
        const [value, setValue] = useState(["push"] as string[]);

        return (
          <div className={styles.wrapper}>
            <CheckboxGroup
              value={value}
              onValueChange={setValue}
              aria-labelledby="active-alerts"
            >
              <CheckboxGroupLabel id="active-alerts">Active Alerts</CheckboxGroupLabel>
              <CheckboxGroupList>
                {options.map((option) => (
                  <CheckboxGroupItem key={option.value}>
                    <CheckboxGroupItemControl value={option.value} name="alerts" />
                    <CheckboxGroupItemLabel>{option.label}</CheckboxGroupItemLabel>
                  </CheckboxGroupItem>
                ))}
              </CheckboxGroupList>
            </CheckboxGroup>
            <span className={styles.hint}>Current value: {value.join(", ") || "none"}</span>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>

### Parent Checkbox

Use `allValues` with a `parent` checkbox to create a select-all control that enters the
indeterminate state when only some child options are checked.

<Preview>
  <ParentCheckboxGroupExample />

  <Preview.Code>
    {`
      import {
        CheckboxGroup,
        CheckboxGroupItem,
        CheckboxGroupItemControl,
        CheckboxGroupItemLabel,
        CheckboxGroupLabel,
        CheckboxGroupList,
      } from "moduix";
      import { useState } from "react";

      export function ParentCheckboxGroupDemo() {
        const [value, setValue] = useState([] as string[]);

        return (
          <CheckboxGroup
            value={value}
            onValueChange={setValue}
            allValues={fruitValues}
            aria-labelledby="fruits"
          >
            <CheckboxGroupLabel id="fruits">Fruits</CheckboxGroupLabel>
            <CheckboxGroupList>
              <CheckboxGroupItem>
                <CheckboxGroupItemControl parent />
                <CheckboxGroupItemLabel>Select all</CheckboxGroupItemLabel>
              </CheckboxGroupItem>

              {options.map((option) => (
                <CheckboxGroupItem key={option.value}>
                  <CheckboxGroupItemControl value={option.value} />
                  <CheckboxGroupItemLabel>{option.label}</CheckboxGroupItemLabel>
                </CheckboxGroupItem>
              ))}
            </CheckboxGroupList>
          </CheckboxGroup>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const options = [
        { value: "apple", label: "Apple" },
        { value: "orange", label: "Orange" },
        { value: "pear", label: "Pear" },
      ];

      const fruitValues = options.map((option) => option.value);
    `}

  </Preview.Data>
</Preview>

and `id`.

<Preview>

        return (
          <div className={styles.siblingRow}>
