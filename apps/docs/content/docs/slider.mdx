## Choosing the right component

Use `Slider` when users should adjust a bounded numeric value directly with pointer or keyboard
input.

- Choose **`Slider`** for interactive values such as volume, brightness, budgets, and thresholds.
- Choose **`Input`** or **`NumberField`** when exact typed entry matters more than scrubbing.
- Choose **`Meter`** or **`Progress`** when the value is display-only and should not be adjusted by
  the user.

## Basic

Start with `Slider`, `SliderLabel`, `SliderValue`, and `SliderThumb`. This is the recommended path
when the default control, track, and indicator structure already fits.

## Anatomy

The normal consumer path is short:

```text
Slider
├─ SliderLabel
├─ SliderValue
└─ SliderThumb[index]
```

`Slider` then renders the default interactive structure for you:

```text
SliderRoot
├─ SliderLabel
├─ SliderValue
└─ SliderControl
   └─ SliderTrack
      ├─ SliderIndicator
      └─ SliderThumb[index]
```

| Part              | Role                                                                |
| ----------------- | ------------------------------------------------------------------- |
| `Slider`          | High-level wrapper with the default control, track, and indicator.  |
| `SliderRoot`      | Low-level root for explicit composition and custom track placement. |
| `SliderLabel`     | Visible label for the slider group.                                 |
| `SliderValue`     | Optional formatted value output.                                    |
| `SliderControl`   | Interaction area for pointer and keyboard input.                    |
| `SliderTrack`     | Visual track that contains the indicator and thumbs.                |
| `SliderIndicator` | Filled portion of the track.                                        |
| `SliderThumb`     | Interactive handle. Render one thumb per selected value.            |

## Composition

### When to stay high-level

Stay with `Slider` for most settings panels and forms. It preserves the Base UI root behavior and
automatically renders `SliderControl`, `SliderTrack`, and `SliderIndicator` after the visible
content.

Use `Slider` or `SliderRoot` for the public root props:

- `value` and `defaultValue`
- `onValueChange` and `onValueCommitted`
- `min`, `max`, `step`, and `largeStep`
- `minStepsBetweenValues`
- `orientation`
- `thumbAlignment`
- `thumbCollisionBehavior`
- `name`, `form`, `locale`, and `format`

`Slider` does not render a thumb automatically. Render one `SliderThumb` per selected value so each
thumb can keep its own accessible name.

### When to go custom

Switch to `SliderRoot`, `SliderControl`, `SliderTrack`, and `SliderIndicator` when the track needs
its own layout position, separate part-level selectors, or styling that should not depend on the
default wrapper structure.

### Default props

| Prop                     | Default      | Values                               |
| ------------------------ | ------------ | ------------------------------------ |
| `orientation`            | `horizontal` | `horizontal`, `vertical`             |
| `min`                    | `0`          | Any `number`                         |
| `max`                    | `100`        | Any `number`                         |
| `step`                   | `1`          | Any `number`                         |
| `largeStep`              | `10`         | Any `number`                         |
| `minStepsBetweenValues`  | `0`          | Any `number`                         |
| `thumbAlignment`         | `center`     | `center`, `edge`, `edge-client-only` |
| `thumbCollisionBehavior` | `push`       | `push`, `swap`, `none`               |

### Behavioral notes

- For range sliders, pass an array value and render the same number of `SliderThumb` elements.
- For SSR-safe multi-thumb sliders, set `index` on each thumb to match the value array index.
- Prefer `SliderLabel` for the shared visible label. Add `aria-label` on each thumb when there is
  no visible label or when range thumbs need distinct names.
- Pass `name` on `Slider` when the current value should submit with a form.
- For multi-thumb form controls, `Fieldset` can own the shared visible label while each
  `SliderThumb` keeps a distinct `aria-label`.
- `SliderValue` render functions always receive arrays: `(formattedValues, values)`.

## Examples

### Range

Pass an array value and render one `SliderThumb` for each selected value.

<Preview>
  <RangeSliderExample />

  <Preview.Code>
    {`
      import {
        Slider,
        SliderLabel,
        SliderThumb,
        SliderValue,
      } from "moduix";
      import { useState } from "react";

      export function RangeSliderDemo() {
        const [value, setValue] = useState([20, 70] as readonly number[]);

        return (
          <Slider value={value} min={0} max={100} onValueChange={setValue}>
            <SliderLabel>Price range</SliderLabel>
            <SliderValue>{([minValue, maxValue]) => \`\${minValue} - \${maxValue}\`}</SliderValue>
            <SliderThumb index={0} aria-label="Minimum price" />
            <SliderThumb index={1} aria-label="Maximum price" />
          </Slider>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Steps and Constraints

Use `step`, `largeStep`, `minStepsBetweenValues`, and `thumbCollisionBehavior` when the slider
represents a constrained numeric domain.

<Preview>
  <SteppedRangeSliderExample />

  <Preview.Code>
    {`
      import {
        Slider,
        SliderLabel,
        SliderThumb,
        SliderValue,
      } from "moduix";
      import { useState } from "react";

      export function SteppedRangeSliderDemo() {
        const [value, setValue] = useState([250, 750] as readonly number[]);

        return (
          <Slider
            value={value}
            min={0}
            max={1000}
            step={50}
            largeStep={200}
            minStepsBetweenValues={2}
            thumbCollisionBehavior="none"
            format={{
              style: "currency",
              currency: "USD",
              maximumFractionDigits: 0,
            }}
            onValueChange={setValue}
          >
            <SliderLabel>Budget</SliderLabel>
            <SliderValue>
              {(_, [minValue, maxValue]) =>
                \`\${priceFormatter.format(minValue)} - \${priceFormatter.format(maxValue)}\`
              }
            </SliderValue>
            <SliderThumb index={0} aria-label="Minimum budget" />
            <SliderThumb index={1} aria-label="Maximum budget" />
          </Slider>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const priceFormatter = new Intl.NumberFormat("en-US", {
        style: "currency",
        currency: "USD",
        maximumFractionDigits: 0,
      });
    `}

  </Preview.Data>
</Preview>

### Controlled

Control the value from React state when the slider must stay in sync with another input or derived
UI.

<Preview>
  <ControlledSliderExample />

  <Preview.Code>
    {`
      import {
        Slider,
        SliderLabel,
        SliderThumb,
        SliderValue,
      } from "moduix";
      import { useState } from "react";
      import styles from "./slider.module.css";

      export function ControlledSliderDemo() {
        const [value, setValue] = useState(24);

        return (
          <div className={styles.stack}>
            <Slider value={value} onValueChange={setValue}>
              <SliderLabel>Brightness</SliderLabel>
              <SliderValue>{([formattedValue]) => \`\${formattedValue}%\`}</SliderValue>
              <SliderThumb aria-label="Brightness" />
            </Slider>
            <input
              className={styles.range}
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: grid;
        gap: var(--spacing-4);
      }

      .range {
        width: 12rem;
      }
    `}

  </Preview.CSS>
</Preview>

### Styling hooks

Use the part that owns the visual concern:

- `Slider` or `SliderRoot` for width, gap, shared text color, and vertical layout width
- `SliderLabel` and `SliderValue` for typography
- `SliderControl` for the interactive hit area and orientation-specific padding
- `SliderTrack` for rail thickness, radius, background, and border treatment
- `SliderIndicator` for the filled range color
- `SliderThumb` for handle size, border, shadow, and focus-ring treatment

The main public hooks are `data-slot="slider-root"`, `data-slot="slider-label"`,
`data-slot="slider-value"`, `data-slot="slider-control"`, `data-slot="slider-track"`,
`data-slot="slider-indicator"`, and `data-slot="slider-thumb"`.
