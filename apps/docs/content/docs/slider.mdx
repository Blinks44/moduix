## Choosing the right component

Use `Slider` when users should adjust a bounded numeric value directly with pointer or keyboard
input.

- Choose **`Slider`** for volume, brightness, budgets, thresholds, and multi-thumb ranges.
- Choose **`Input`** or **`NumberInput`** when exact typed entry matters more than scrubbing.
- Choose **`ProgressLinear`** or **`ProgressCircular`** when the value is display-only.

## Basic

`Slider.Thumbs` is the recommended way to render one styled thumb and native form input for every
value. Keep `Slider.Thumb` and `Slider.HiddenInput` for custom thumb composition.

## Anatomy

```text
Slider / Slider.Root
├─ Slider.Label
├─ Slider.ValueText
├─ Slider.Control
│  ├─ Slider.Track
│  │  └─ Slider.Range
│  └─ Slider.Thumbs
│     └─ Slider.Thumb[index]
│        ├─ Slider.DraggingIndicator
│        └─ Slider.HiddenInput
└─ Slider.MarkerGroup
   └─ Slider.Marker[value]
```

| Part                                   | Role                                                           |
| -------------------------------------- | -------------------------------------------------------------- |
| `Slider` / `Slider.Root`               | Styled Ark root for controlled and uncontrolled state.         |
| `Slider.RootProvider`                  | Root that receives an externally owned `useSlider` machine.    |
| `Slider.Label`                         | Visible label for the slider group.                            |
| `Slider.ValueText`                     | Current value text generated by Ark.                           |
| `Slider.Control`                       | Pointer interaction area.                                      |
| `Slider.Track`                         | Full visual rail.                                              |
| `Slider.Range`                         | Filled portion from the configured origin to the value.        |
| `Slider.Thumb`                         | Focusable slider handle.                                       |
| `Slider.Thumbs`                        | Moduix helper that renders a thumb and native input per value. |
| `Slider.HiddenInput`                   | Native input for form submission and reset behavior.           |
| `Slider.MarkerGroup` / `Slider.Marker` | Optional tick mark group and individual mark.                  |
| `Slider.DraggingIndicator`             | Optional value bubble while dragging.                          |

## Composition

### Recommended composition

Use the Ark part tree: root, label, optional value text, control, track, range, and `Slider.Thumbs`.
It creates one thumb and `HiddenInput` per value for forms and resets.

`Slider` is the short root form and is equivalent to `Slider.Root`.

### When to go custom

Use `asChild` when a part must render a custom host element, but keep a single semantic child that
can receive Ark props. Use explicit `Slider.Thumb` and `Slider.HiddenInput` for custom thumb
children, labels, or ordering. For advanced state access, import `useSliderContext` or `useSlider`
from `@moduix/react` and pair them with `Slider.RootProvider`.

### Default props

| Prop                     | Default      | Notes                                      |
| ------------------------ | ------------ | ------------------------------------------ |
| `defaultValue`           | `[50]`       | Ark stores slider values as `number[]`.    |
| `orientation`            | `horizontal` | Use `vertical` for vertical keyboard axis. |
| `min`                    | `0`          | Lower bound.                               |
| `max`                    | `100`        | Upper bound.                               |
| `step`                   | `1`          | Pointer and keyboard increment.            |
| `minStepsBetweenThumbs`  | `0`          | Minimum gap between range thumbs.          |
| `origin`                 | `start`      | Fill start for single-value ranges.        |
| `thumbAlignment`         | `contain`    | Use `center` to let the thumb overflow.    |
| `thumbCollisionBehavior` | `none`       | Use `push` or `swap` for range dragging.   |
| `thumbSize`              | measured     | Override Ark's measured thumb dimensions.  |

`value`, `defaultValue`, `onValueChange(details)`, and `onValueChangeEnd(details)` use Ark detail
objects. Use `details.value` to read the array.

### Behavioral notes

`Slider` follows Ark's WAI-ARIA slider behavior. Each thumb needs an accessible name through
`Slider.Label`, `aria-label`, or `aria-labelledby`. Horizontal sliders use left/right arrow keys;
vertical sliders use up/down arrow keys. Home and End move to bounds.

`Slider.Thumbs` includes a `HiddenInput` for every rendered thumb. Keep `Slider.HiddenInput` inside
every explicit `Slider.Thumb` when native form submission, form association, or form reset
synchronization matters. `disabled`, `invalid`, `readOnly`, `name`,
`form`, `ids`, `aria-label`, `aria-labelledby`, and `getAriaValueText(details)` pass through to
Ark.

### Styling hooks

Ark writes `data-scope="slider"` and `data-part` to all rendered parts. Moduix also writes
`data-slot` values: `slider-root`, `slider-root-provider`, `slider-label`, `slider-value-text`,
`slider-control`, `slider-track`, `slider-range`, `slider-thumb`, `slider-hidden-input`,
`slider-marker-group`, `slider-marker`, and `slider-dragging-indicator`.

Important Ark state hooks include `data-orientation`, `data-disabled`, `data-invalid`,
`data-focus`, `data-dragging`, marker `data-state`, and dragging-indicator
`data-state="open | closed"`. Ark runtime CSS variables include `--slider-thumb-width`,
`--slider-thumb-height`, `--slider-thumb-transform`, `--slider-range-start`,
`--slider-range-end`, `--translate-x`, and `--translate-y`.

The default thumb focus ring uses `:focus-visible` for keyboard focus and Ark `data-dragging` while
the thumb is pressed. It is removed after pointer release.
