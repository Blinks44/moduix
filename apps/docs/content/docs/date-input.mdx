## Choosing the right component

- Choose **`Input`** for plain text values.
- Choose **`DateInput`** when users need keyboard-friendly date segments and native form values.
- Choose a date picker composition when users need a calendar popup.

## Basic

Render date segments through Ark `DateInputSegmentContext`. Ark controls segment order, keyboard
editing, focus, placeholders, and form synchronization, while moduix keeps the styled visual parts.

<Preview>
  <DateInputExample />

  <Preview.Code>
    {`
      import { CalendarDate } from "@internationalized/date";
      import { DateInputSegmentContext } from "@ark-ui/react/date-input";
      import { DateInput } from "@moduix/react";

      function DateInputSegments() {
        return (
          <DateInput.SegmentGroup>
            <DateInputSegmentContext>
              {(segment) => <DateInput.Segment segment={segment} />}
            </DateInputSegmentContext>
          </DateInput.SegmentGroup>
        );
      }

      export function DateInputDemo() {
        return (
          <DateInput defaultValue={[new CalendarDate(2026, 6, 22)]} name="release-date">
            <DateInput.Label>Release date</DateInput.Label>
            <DateInput.Control>
              <DateInputSegments />
            </DateInput.Control>
            <DateInput.HiddenInput />
          </DateInput>
        );
      }
    `}

## Anatomy

```text
DateInput.Root
├─ DateInput.Label
├─ DateInput.Control
│  └─ DateInput.SegmentGroup
│     └─ DateInputSegmentContext (from @ark-ui/react/date-input)
│        └─ DateInput.Segment
└─ DateInput.HiddenInput

DateInput.RootProvider
└─ same part tree connected to a useDateInput() store
```

| Part                           | Role                                                               |
| ------------------------------ | ------------------------------------------------------------------ |
| `DateInput` / `DateInput.Root` | Controls date value, locale formatting, segment focus, validation. |
| `DateInput.RootProvider`       | Connects parts to state created with `useDateInput()`.             |
| `DateInput.Label`              | Accessible label for the date input.                               |
| `DateInput.Control`            | Visual frame around one or more segment groups.                    |
| `DateInput.SegmentGroup`       | Groups segments for one date index.                                |
| `DateInput.Segment`            | Editable or literal date segment returned by Ark.                  |
| `DateInput.HiddenInput`        | Native hidden input synchronized with the selected date.           |
| `DateInput.Separator`          | Presentational text for range layouts.                             |
| `DateInputSegmentContext`      | Upstream Ark render helper for locale-aware segment rendering.     |

## Composition

### Recommended composition

Use `DateInput` as the short root form. Render `DateInput.SegmentGroup`, then use
Ark `DateInputSegmentContext` to receive each locale-aware segment and pass it to
`DateInput.Segment`.

Use `DateInput.HiddenInput` when the date should submit with a form or reset with a native form
reset. For ranges, render indexed segment groups and hidden inputs for each side.

### When to go custom

Use `asChild` on Ark parts when you need a custom host element. Keep the child semantic and single.
Use `ids` when an external label, helper text, or another component must share exact IDs.

### Default props

| Prop                              | Default        | Notes                                                 |
| --------------------------------- | -------------- | ----------------------------------------------------- |
| `DateInput` / `DateInput.Root`    | —              | No local behavioral defaults; Ark defaults apply.     |
| `DateInput.HiddenInput`           | —              | Explicit, so single and range form names stay clear.  |
| `DateInput.Separator.aria-hidden` | `true`         | Presentational text is hidden from assistive tech.    |
| `DateInput.Separator.role`        | `presentation` | Keeps range separators out of the accessibility tree. |

### Behavioral notes

- Values are `DateInputDateValue[]` objects from Ark / `@internationalized/date`, not strings.
- `onValueChange(details)` preserves Ark's details object with `details.value` and `details.valueAsString`.
- `focused`, `defaultFocused`, and `onFocusChange(details)` control root focus state.
- `placeholderValue`, `defaultPlaceholderValue`, and `onPlaceholderChange(details)` control the placeholder date.
- `selectionMode="range"` enables indexed segment groups and hidden inputs.
- `min`, `max`, and `isDateUnavailable` validate committed dates.
- `locale`, `timeZone`, `hourCycle`, `granularity`, `formatter`, `format`, `translations`, and `createCalendar` pass through to Ark.
- Import Ark `useDateInput()`, `useDateInputContext()`, `DateInputContext`, and
  `DateInputSegmentContext` directly from `@ark-ui/react/date-input` when you need advanced state
  access or renderless helpers.

## Examples

### Controlled

Control the selected date array and update it from `details.value`.

<Preview>
  <ControlledDateInputExample />

  <Preview.Code>
    {`
      import { CalendarDate } from "@internationalized/date";
      import {
        DateInputSegmentContext,
        type DateInputDateValue,
      } from "@ark-ui/react/date-input";
      import { DateInput } from "@moduix/react";
      import { useState } from "react";

      function DateInputSegments() {
        return (
          <DateInput.SegmentGroup>
            <DateInputSegmentContext>
              {(segment) => <DateInput.Segment segment={segment} />}
            </DateInputSegmentContext>
          </DateInput.SegmentGroup>
        );
      }

      export function ControlledDateInputDemo() {
        const [value, setValue] = useState([
          new CalendarDate(2026, 6, 22),
        ] as DateInputDateValue[]);

        return (
          <div>
            <DateInput value={value} onValueChange={(details) => setValue(details.value)}>
              <DateInput.Label>Controlled date</DateInput.Label>
              <DateInput.Control>
                <DateInputSegments />
              </DateInput.Control>
              <DateInput.HiddenInput />
            </DateInput>
            <div className="date-input-state">Current value: {value[0]?.toString() ?? "empty"}</div>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{dateInputExampleCss}</Preview.CSS>
  <Preview.Data>{dateInputNoData}</Preview.Data>
</Preview>

### Root Provider

Use Ark `useDateInput()` with `DateInput.RootProvider` when state actions need to live outside the
rendered root tree.

<Preview>
  <RootProviderDateInputExample />

  <Preview.Code>
    {`
      import { today } from "@internationalized/date";
      import { DateInputSegmentContext, useDateInput } from "@ark-ui/react/date-input";
      import { DateInput } from "@moduix/react";

      function DateInputSegments() {
        return (
          <DateInput.SegmentGroup>
            <DateInputSegmentContext>
              {(segment) => <DateInput.Segment segment={segment} />}
            </DateInputSegmentContext>
          </DateInput.SegmentGroup>
        );
      }

      export function RootProviderDateInputDemo() {
        const dateInput = useDateInput({ defaultValue: [today("UTC")] });

        return (
          <div>
            <DateInput.RootProvider value={dateInput}>
              <DateInput.Label>Report date</DateInput.Label>
              <DateInput.Control>
                <DateInputSegments />
              </DateInput.Control>
              <DateInput.HiddenInput name="report-date" />
            </DateInput.RootProvider>
            <div className="date-input-root-provider-actions">
              <button type="button" onClick={() => dateInput.clearValue()}>Clear</button>
              <button type="button" onClick={() => dateInput.focus()}>Focus</button>
            </div>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{dateInputExampleCss}</Preview.CSS>

### Styling hooks

- `data-slot="date-input-root"` on the root.
- `data-slot="date-input-label"` on the label.
- `data-slot="date-input-control"` on the visual frame.
- `data-slot="date-input-segment-group"` on each date group.
- `data-slot="date-input-segment"` on editable and literal segments.
- `data-slot="date-input-hidden-input"` on the hidden native input.
- `data-slot="date-input-separator"` on moduix range separator sugar.
- Ark state attributes include `data-disabled`, `data-readonly`, `data-invalid`, `data-focus` on
  control and segment groups, `data-type`, `data-editable`, and `data-placeholder-shown`.
