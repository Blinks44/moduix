
  </Tab>
  <Tab value="bun">

```bash
bunx shadcn@latest add Blinks44/moduix/preview-card
```

  </Tab>
</Tabs>

This install path also pulls in the shared moduix foundation files that `Preview Card` depends on, such
as styles, icons, and small utilities.

`PreviewCardContent` already renders `PreviewCardPortal`, `PreviewCardPositioner`,
`PreviewCardPopup`, and `PreviewCardViewport`. That keeps the default path short while leaving the
low-level parts available when the layout needs to change.

Preview cards are a visual enhancement for sighted hover and focus users. Keep essential
information available on the linked destination or nearby page content, because touch and screen
reader users may not receive the same preview.

## Anatomy

```text
PreviewCard
â”śâ”€ PreviewCardTrigger
â””â”€ PreviewCardContent
   â””â”€ PreviewCardPortal
      â””â”€ PreviewCardPositioner
         â””â”€ PreviewCardPopup
            â””â”€ PreviewCardViewport
```

| Part                    | Role                                                                                                                     |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `PreviewCard`           | Root state machine for open state, payload, and detached handles.                                                        |
| `PreviewCardTrigger`    | Hover and focus trigger, usually rendered as a link.                                                                     |
| `PreviewCardContent`    | Default popup wrapper for the common case.                                                                               |
| `PreviewCardPortal`     | Popup mount target; used internally by `PreviewCardContent` and available for custom portal placement.                   |
| `PreviewCardBackdrop`   | Optional backdrop for custom composition.                                                                                |
| `PreviewCardPositioner` | Low-level positioning part.                                                                                              |
| `PreviewCardPopup`      | Low-level visible popup container.                                                                                       |
| `PreviewCardArrow`      | Optional built-in arrow, with custom children supported.                                                                 |
| `PreviewCardViewport`   | Inner content wrapper; included by `PreviewCardContent` so payload transitions can be added later without restructuring. |

## Composition

Use `PreviewCard` for root behavior such as `open`, `defaultOpen`, `onOpenChange`, and `handle`.
Use `PreviewCardTrigger` for the interactive link itself.

`PreviewCardContent` is intentionally small. It accepts popup props plus the common positioning
surface:

- `side`
- `sideOffset`
- `align`
- `alignOffset`
- `arrowPadding`
- `collisionAvoidance`
- `collisionBoundary`
- `collisionPadding`
- `showArrow`

Defaults:

| Prop         | Default |
| ------------ | ------- |
| `showArrow`  | `false` |
| `sideOffset` | `8`     |

Use `showArrow` only when the popup should visually point back to its trigger. When the popup
needs a custom arrow, backdrop, portal container, or manual viewport structure, switch to explicit
composition with the low-level parts.

### Detached Trigger

tree.

### Payload-Driven Content

Pass `payload` through the trigger when one popup should render different content for different
links.

<Preview>
  <MultipleTriggersPreviewCardExample />

  <Preview.Code>
    {`
      import {
        createPreviewCardHandle,
        PreviewCard,
        PreviewCardContent,
        PreviewCardTrigger,
      } from "moduix";
      import { useMemo } from "react";

      type PreviewItem = (typeof items)[number];

      export function MultipleTriggersPreviewCard() {
        const previewCardHandle = useMemo(() => createPreviewCardHandle<PreviewItem>(), []);

        return (
          <div className={styles.row}>
            {items.map((item) => (
              <PreviewCardTrigger
                key={item.title}
                handle={previewCardHandle}
                href={item.url}
                payload={item}
              >
                {item.title}
              </PreviewCardTrigger>
            ))}

            <PreviewCard handle={previewCardHandle}>
              {({ payload }) => (
                <PreviewCardContent>
                  <div className={styles.popupContent}>
                    <img
                      className={styles.image}
                      alt={"Preview illustration for " + (payload?.title ?? "Typography")}
                      src={payload?.image ?? items[0].image}
                    />
                    <p className={styles.summary}>
                      <strong>{payload?.title ?? "Typography"}</strong>
                      <br />
                      {payload?.summary ?? items[0].summary}
                    </p>
                  </div>
                </PreviewCardContent>
              )}
            </PreviewCard>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`      const items = [
        {
          title: "Typography",
          image: "/images/typography-preview.jpg",
          url: "https://en.wikipedia.org/wiki/Typography",
          summary: "Typography is the art and technique of arranging type.",
        },
        {
          title: "Grid systems",
          image: "/images/grid-preview.jpg",
          url: "https://en.wikipedia.org/wiki/Grid_(graphic_design)",
          summary: "Grid systems help organize content and spacing.",
        },
      ];`}

  </Preview.Data>

## Custom Composition

Use the low-level parts when the default wrapper is no longer enough.

### Custom Arrow Markup

Keep `PreviewCardContent` for the default arrow. Switch to `PreviewCardArrow` directly when the
arrow markup itself needs to change.

<Preview>
  <CustomArrowPreviewCardExample />

  <Preview.Code>
    {`
      import {
        PreviewCard,
        PreviewCardArrow,
        PreviewCardPopup,
        PreviewCardPortal,
        PreviewCardPositioner,
        PreviewCardTrigger,
        PreviewCardViewport,
      } from "moduix";

      export function CustomArrowPreviewCard() {
        return (
          <PreviewCard>
            <PreviewCardTrigger href="https://en.wikipedia.org/wiki/Typography">
              Preview with custom arrow
            </PreviewCardTrigger>
            <PreviewCardPortal>
              <PreviewCardPositioner>
                <PreviewCardPopup className={styles.customArrowPopup}>
                  <PreviewCardArrow>
                    <CustomArrow />
                  </PreviewCardArrow>
                  <PreviewCardViewport>
                     <div className={styles.popupContent}>
                       <img
                         className={styles.image}
                         alt="Preview illustration for Typography"
                         src="/images/typography-preview.jpg"
                       />
                       <p className={styles.summary}>
                         <strong>Typography</strong>
                         <br />
                         Custom arrow content now lives in explicit composition.
                       </p>
                     </div>
                   </PreviewCardViewport>
                </PreviewCardPopup>
              </PreviewCardPositioner>
            </PreviewCardPortal>
          </PreviewCard>
        );
      }

      function CustomArrow() {
        return (
          <svg className={styles.customArrow} viewBox="0 0 16 8" fill="none" aria-hidden="true">
            <path d="M8 0L16 8H0L8 0Z" fill="currentColor" />
          </svg>
        );
      }
    `}

  </Preview.Code>
