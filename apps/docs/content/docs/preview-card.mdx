## Anatomy

The normal consumer path is short:

```text
PreviewCard
├─ PreviewCardTrigger
└─ PreviewCardContent
   └─ PreviewCardPortal
      └─ PreviewCardPositioner
         └─ PreviewCardPopup
            └─ PreviewCardViewport
```

`PreviewCardContent` can also render `PreviewCardArrow` when `showArrow` is enabled.

| Part                    | Role                                                               |
| ----------------------- | ------------------------------------------------------------------ |
| `PreviewCard`           | Root state, payload, and detached handle provider.                 |
| `PreviewCardTrigger`    | Hover and focus trigger, usually rendered as a link.               |
| `PreviewCardContent`    | Default popup wrapper with built-in portal, positioner, and popup. |
| `PreviewCardPortal`     | Popup mount target for custom composition.                         |
| `PreviewCardPositioner` | Floating layout and collision-aware positioning layer.             |
| `PreviewCardPopup`      | Visible popup surface.                                             |
| `PreviewCardViewport`   | Inner content wrapper for preview content.                         |

## Composition

### When to stay high-level

Stay with `PreviewCard`, `PreviewCardTrigger`, and `PreviewCardContent` for most link-preview
workflows.

`PreviewCard` keeps the Base UI root behavior intact. Use it for controlled and uncontrolled open
state, detached handles, trigger IDs, and payload-driven rendering.

`PreviewCardContent` is the high-level popup wrapper. It accepts popup `className`, supports the
shared positioning props (`side`, `sideOffset`, `align`, `alignOffset`, `arrowPadding`,
`collisionAvoidance`, `collisionBoundary`, `collisionPadding`), and adds `showArrow` for the
built-in arrow.

### When to go custom

Switch to `PreviewCardPortal`, `PreviewCardBackdrop`, `PreviewCardPositioner`, `PreviewCardPopup`,
`PreviewCardArrow`, and `PreviewCardViewport` when the popup needs custom arrow markup, backdrop
control, portal props, or manual viewport styling.

`PreviewCardContent` intentionally does not expose portal props such as `container` or
`keepMounted`. Use explicit composition for those cases.

### Default props

| Prop         | Default | Values / Notes                            |
| ------------ | ------- | ----------------------------------------- |
| `showArrow`  | `false` | Enables the built-in arrow                |
| `sideOffset` | `8`     | Any number accepted by Base UI positioner |

### Behavioral notes

- `PreviewCardTrigger` keeps the default moduix link styling only when `render` is omitted.
- `PreviewCardContent className` styles `PreviewCardPopup`, not the portal, positioner, or
  viewport.
- `PreviewCardViewport` clips overflowing content. Add your own inner scroll container when preview
  content must scroll.
- `PreviewCardBackdrop` is visual-only by default because it keeps `pointer-events: none`.
- Preview cards are a progressive enhancement for sighted hover and focus users. Keep essential
  information available in nearby content or on the linked destination.

tree.

### Payload-Driven Content

Pass `payload` through the trigger when one popup should render different content for different
links.

<Preview>
  <MultipleTriggersPreviewCardExample />

  <Preview.Code>
    {`
      import {
        createPreviewCardHandle,
        PreviewCard,
        PreviewCardContent,
        PreviewCardTrigger,
      } from "moduix";
      import { useMemo } from "react";

      export function MultipleTriggersPreviewCard() {
        const previewCardHandle = useMemo(() => createPreviewCardHandle(), []);

        return (
          <div className={styles.row}>
            {items.map((item) => (
              <PreviewCardTrigger
                key={item.title}
                handle={previewCardHandle}
                href={item.url}
                payload={item}
              >
                {item.title}
              </PreviewCardTrigger>
            ))}

            <PreviewCard handle={previewCardHandle}>
              {({ payload }) => (
                <PreviewCardContent>
                  <div className={styles.popupContent}>
                    <img
                      className={styles.image}
                      alt={"Preview illustration for " + (payload?.title ?? "Typography")}
                      src={payload?.image ?? items[0].image}
                    />
                    <p className={styles.summary}>
                      <strong>{payload?.title ?? "Typography"}</strong>
                      <br />
                      {payload?.summary ?? items[0].summary}
                    </p>
                  </div>
                </PreviewCardContent>
              )}
            </PreviewCard>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const items = [
        {
          title: "Typography",
          image: "/images/typography-preview.jpg",
          url: "https://en.wikipedia.org/wiki/Typography",
          summary: "Typography is the art and technique of arranging type.",
        },
        {
          title: "Grid systems",
          image: "/images/grid-preview.jpg",
          url: "https://en.wikipedia.org/wiki/Grid_(graphic_design)",
          summary: "Grid systems help organize content and spacing.",
        },
      ];
    `}

  </Preview.Data>
</Preview>

## Styling

Style `Preview Card` through `className`, `data-slot="preview-card-*"` hooks, and the public
`--preview-card-*` CSS variables shown below.

### Styling hooks

Use the part that owns the visual concern:

- `PreviewCardTrigger` for link color, underline, and focus styling
- `PreviewCardContent` or `PreviewCardPopup` for popup surface overrides
- `PreviewCardArrow` for arrow placement or custom arrow content
- `PreviewCardViewport` for inner clipping and content transitions
- `PreviewCardBackdrop` for optional ambient overlay styling

The main public hooks include `data-slot="preview-card-trigger"`,
`data-slot="preview-card-backdrop"`, `data-slot="preview-card-positioner"`,
`data-slot="preview-card-popup"`, `data-slot="preview-card-arrow"`, and
`data-slot="preview-card-viewport"`.
