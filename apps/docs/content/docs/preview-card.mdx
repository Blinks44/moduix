## Anatomy

`PreviewCard` is a thin wrapper over the Base UI preview-card primitive. The default path keeps the
API small:

```tsx
<PreviewCard>
  <PreviewCardTrigger href="https://en.wikipedia.org/wiki/Typography">
    typography
  </PreviewCardTrigger>
  <PreviewCardContent>
    <div className={styles.popupContent}>Preview content</div>
  </PreviewCardContent>
</PreviewCard>
```

`PreviewCardContent` renders `PreviewCardPortal`, `PreviewCardPositioner`, `PreviewCardPopup`, and
`PreviewCardViewport` for you. Set `showArrow` when you want the built-in arrow.

Preview cards are a visual enhancement for hover and focus. Keep essential information available on
the linked destination or in nearby page content, because touch and screen reader users may not
receive the same popup preview.

| Part                    | Role                                                                                                      |
| ----------------------- | --------------------------------------------------------------------------------------------------------- |
| `PreviewCard`           | Root state machine. Controls open state, payload, and detached trigger handles.                           |
| `PreviewCardTrigger`    | Interactive link or trigger. It opens the preview on hover or focus and receives open state data.         |
| `PreviewCardContent`    | High-level popup wrapper for the common case. It owns portal, positioning, popup, and viewport structure. |
| `PreviewCardPortal`     | Portal escape hatch for custom container or manual structure.                                             |
| `PreviewCardBackdrop`   | Optional backdrop for custom composition.                                                                 |
| `PreviewCardPositioner` | Low-level positioning part.                                                                               |
| `PreviewCardPopup`      | Low-level popup container.                                                                                |
| `PreviewCardArrow`      | Optional arrow part with overridable children.                                                            |
| `PreviewCardViewport`   | Transition-aware content container.                                                                       |

## Composition

Use `PreviewCard` for root behavior props such as `open`, `defaultOpen`, `onOpenChange`, and
`handle`. Use `PreviewCardTrigger` for the link itself.

Use `PreviewCardContent` for the default path. It supports popup props plus the most useful
positioning props from `PreviewCardPositioner`:

- `side`
- `sideOffset`
- `align`
- `alignOffset`
- `arrowPadding`
- `collisionAvoidance`
- `collisionBoundary`
- `collisionPadding`
- `disableAnchorTracking`
- `anchor`
- `sticky`
- `positionMethod`
- `showArrow`

Defaults:

| Prop         | Default |
| ------------ | ------- |
| `showArrow`  | `false` |
| `sideOffset` | `8`     |

When you need a backdrop, custom arrow markup, custom portal behavior, or a different structure,
compose the low-level parts directly.

### Multiple Triggers With Payload

<Preview>
  <MultipleTriggersPreviewCardExample />

  <Preview.Code>
    {`
      import { createPreviewCardHandle, PreviewCard, PreviewCardContent, PreviewCardTrigger } from "moduix";
      import { useMemo } from "react";

      const items = [
        {
          title: "Typography",
          url: "https://en.wikipedia.org/wiki/Typography",
          summary: "Typography is the art and technique of arranging type.",
        },
        {
          title: "Grid systems",
          url: "https://en.wikipedia.org/wiki/Grid_(graphic_design)",
          summary: "Grid systems help organize content and spacing.",
        },
      ];

      export function MultipleTriggersPreviewCard() {
        const previewCardHandle = useMemo(() => createPreviewCardHandle(), []);

        return (
          <div className={styles.row}>
            {items.map((item) => (
              <PreviewCardTrigger
                key={item.title}
                handle={previewCardHandle}
                href={item.url}
                payload={item}
              >
                {item.title}
              </PreviewCardTrigger>
            ))}

            <PreviewCard handle={previewCardHandle}>
              {({ payload }) => (
                <PreviewCardContent>
                  <div className={styles.popupContent}>
                    <p className={styles.summary}>{payload?.summary}</p>
                  </div>
                </PreviewCardContent>
              )}
            </PreviewCard>
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Side Control

<Preview>
  <SideControlPreviewCardExample />

  <Preview.Code>
    {`
      import { PreviewCard, PreviewCardContent, PreviewCardTrigger } from "moduix";
      import { useState } from "react";

      const sides = ["top", "right", "bottom", "left"] as const;

      export function SideControlPreviewCard() {
        const [side, setSide] = useState("bottom" as (typeof sides)[number]);

        return (
          <div className={styles.stack}>
            <div className={styles.sideButtons}>
              {sides.map((item) => (
                <button
                  key={item}
                  type="button"
                  className={styles.sideButton}
                  data-active={item === side || undefined}
                  onClick={() => setSide(item)}
                >
                  {item}
                </button>
              ))}
            </div>

            <PreviewCard>
              <PreviewCardTrigger href="https://en.wikipedia.org/wiki/Typography">
                Placement: {side}
              </PreviewCardTrigger>
              <PreviewCardContent side={side} className={styles.sidePopup}>
                <div className={styles.popupContent}>
                  <p className={styles.summary}>Current side is {side}.</p>
                </div>
              </PreviewCardContent>
            </PreviewCard>
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>
