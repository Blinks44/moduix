## Basic

Use `RadioGroup` with Ark-style item parts and keep `RadioGroup.ItemHiddenInput` in each
form-backed item.

## Anatomy

```text
RadioGroup.Root
├─ RadioGroup.Label
├─ RadioGroup.Indicator (optional)
└─ RadioGroup.Item[value]
   ├─ RadioGroup.ItemControl
   ├─ RadioGroup.ItemText
   └─ RadioGroup.ItemHiddenInput

RadioGroup.RootProvider
└─ same item tree connected to an Ark useRadioGroup() store
```

| Part                         | Role                                                        |
| ---------------------------- | ----------------------------------------------------------- |
| `RadioGroup` / `.Root`       | Owns value state, keyboard navigation, orientation, forms.  |
| `RadioGroup.RootProvider`    | Connects parts to state created with Ark `useRadioGroup()`. |
| `RadioGroup.Label`           | Accessible label for the group.                             |
| `RadioGroup.Item`            | One selectable option; renders a `label` by default.        |
| `RadioGroup.ItemControl`     | Visual radio control; supports the moduix `size` prop.      |
| `RadioGroup.ItemText`        | Visible option text.                                        |
| `RadioGroup.ItemHiddenInput` | Native input for form submission, validation, and reset.    |
| `RadioGroup.Indicator`       | Optional active-item indicator using Ark measurement vars.  |

## Composition

### Recommended composition

Use `RadioGroup` as the short form for `RadioGroup.Root`. Keep the Ark item tree explicit:
`RadioGroup.Label`, `RadioGroup.Item`, `RadioGroup.ItemControl`, `RadioGroup.ItemText`, and
`RadioGroup.ItemHiddenInput`.

### When to go custom

Use `RadioGroup.RootProvider` with Ark `useRadioGroup()` when state must be created outside the
rendered group tree. Use `RadioGroup.Item asChild` when a custom item layout should own the item
DOM node; the direct child must still be a semantic `label`.

### Default props

| Prop                          | Default | Notes                                                 |
| ----------------------------- | ------- | ----------------------------------------------------- |
| `RadioGroup.ItemControl size` | `md`    | Moduix-only control diameter and indicator-dot scale. |

### Behavioral notes

`RadioGroup.Root` accepts Ark props such as `value`, `defaultValue`,
`onValueChange(details)`, `name`, `form`, `orientation`, `disabled`, `invalid`, `readOnly`,
`required`, `ids`, and `asChild`.

`RadioGroup.ItemHiddenInput` should stay in each item that participates in native form submission,
validation, or reset. Ark owns the controlled and uncontrolled value state, roving focus, keyboard
navigation, and `onValueChange(details)` callback shape.

### Controlled

<Preview>
  <ControlledRadioGroupExample />

  <Preview.Code>

      export function ControlledRadioGroupDemo() {

### Root Provider

Use `RootProvider` when an Ark `useRadioGroup()` store needs to be shared with controls outside the
radio tree.

<Preview>
  <RadioGroupRootProviderExample />

  <Preview.Code>
    {`
      import { useRadioGroup } from "@ark-ui/react/radio-group";
      import { RadioGroup } from "@moduix/react";

      export function RadioGroupRootProviderDemo() {
        const radioGroup = useRadioGroup({ defaultValue: "React" });

        return (
          <div className="radio-stack">
            <RadioGroup.RootProvider value={radioGroup}>
              <RadioGroup.Label>Framework</RadioGroup.Label>
              {frameworks.map((framework) => (
                <RadioGroup.Item key={framework} value={framework}>
                  <RadioGroup.ItemControl />
                  <RadioGroup.ItemText>{framework}</RadioGroup.ItemText>
                  <RadioGroup.ItemHiddenInput />
                </RadioGroup.Item>
              ))}
            </RadioGroup.RootProvider>
            <button className="radio-button" type="button" onClick={() => radioGroup.setValue("Solid")}>
              Set to Solid
            </button>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{radioGroupExampleCss}</Preview.CSS>

## Styling

Style `RadioGroup` through `className`, stable `data-slot="radio-group-*"` hooks, Ark
`data-scope`, `data-part`, `data-state`, and the public `--radio-*` CSS variables below.

### CSS Properties

### Styling hooks

- Ark anatomy hooks: `data-scope="radio-group"` and `data-part` on root, label, item,
  item-control, item-text, item-hidden-input, and indicator.
- Ark state hooks: `data-state="checked|unchecked"`, `data-disabled`, `data-readonly`,
  `data-invalid`, `data-required`, `data-focus`, `data-focus-visible`, `data-hover`,
  `data-active`, and `data-orientation`.
- Ark indicator variables: `--transition-property`, `--left`, `--top`, `--width`, and `--height`.
- Moduix hooks: `data-slot="radio-group-root"`, `radio-group-label`,
  `radio-group-item`, `radio-group-item-control`, `radio-group-item-text`,
  `radio-group-item-hidden-input`, and `radio-group-indicator`.
