## Anatomy

```text
RadioGroup.Root
├─ RadioGroup.Label
├─ RadioGroup.Indicator (optional)
└─ RadioGroup.Item[value]
   ├─ RadioGroup.ItemControl
   ├─ RadioGroup.ItemText
   └─ the native form input

RadioGroup.RootProvider
└─ same item tree connected to a useRadioGroup() store

RadioGroup.Option
└─ RadioGroup.Item + ItemControl + ItemText + native input (automatic)
```

| Part                      | Role                                                        |
| ------------------------- | ----------------------------------------------------------- |
| `RadioGroup` / `.Root`    | Owns value state, keyboard navigation, orientation, forms.  |
| `RadioGroup.RootProvider` | Connects parts to state created with Ark `useRadioGroup()`. |
| `RadioGroup.Label`        | Accessible label for the group.                             |
| `RadioGroup.Item`         | One selectable option; renders a `label` by default.        |
| `RadioGroup.Option`       | Convenience item with control, text, and hidden input.      |
| `RadioGroup.ItemControl`  | Visual radio control; supports the moduix `size` prop.      |
| `RadioGroup.ItemText`     | Visible option text.                                        |
| `RadioGroup.Indicator`    | Optional active-item indicator using Ark measurement vars.  |

## Composition

### Recommended composition

Use `RadioGroup` as the short form for `RadioGroup.Root` and `RadioGroup.Option` for labelled
items. The option accepts the Ark item props, `size`, and `className`; its class is applied to the
underlying `Item`, so its nested parts remain targetable through their `data-slot` hooks.

### When to go custom

Use `RadioGroup.RootProvider` with `useRadioGroup()` from `@moduix/react` when state must be
created outside the rendered group tree. Use the explicit `Item` tree when a custom layout or
`RadioGroup.Item asChild` must own the item DOM node; the direct child must still be a semantic
`label`.

### Default props

| Prop                          | Default | Notes                                                 |
| ----------------------------- | ------- | ----------------------------------------------------- |
| `RadioGroup.Option size`      | `md`    | Convenience control diameter and indicator-dot scale. |
| `RadioGroup.ItemControl size` | `md`    | Moduix-only control diameter and indicator-dot scale. |

### Behavioral notes

`RadioGroup.Root` accepts Ark props such as `value`, `defaultValue`,
`onValueChange(details)`, `name`, `form`, `orientation`, `disabled`, `invalid`, `readOnly`,
`required`, `ids`, and `asChild`.

Each `RadioGroup.Item` renders its native form input automatically for native form submission,
validation, and reset. Ark owns the controlled and uncontrolled value state, roving focus, keyboard
navigation, and `onValueChange(details)` callback shape.

`Field.Root` and `Fieldset.Root` state flows through to the group. Use `Field.Label`,
`Field.HelperText`, and `Field.ErrorText` for visible form context; `Item` and `Option` retain
native form submission and reset behavior automatically.

## Styling

Style `RadioGroup` through `className`, stable `data-slot="radio-group-*"` hooks, Ark
`data-scope`, `data-part`, `data-state`, and the public `--radio-*` CSS variables below.

### Styling hooks

- Ark anatomy hooks: `data-scope="radio-group"` and `data-part` on root, label, item,
  item-control, item-text, item-hidden-input, and indicator.
- Ark state hooks: `data-state="checked|unchecked"`, `data-disabled`, `data-readonly`,
  `data-invalid`, `data-required`, `data-focus`, `data-focus-visible`, `data-hover`,
  `data-active`, and `data-orientation`.
- Ark indicator variables: `--transition-property`, `--left`, `--top`, `--width`, and `--height`.
- Moduix hooks: `data-slot="radio-group-root"`, `radio-group-label`,
  `radio-group-item`, `radio-group-item-control`, `radio-group-item-text`,
  `radio-group-item-hidden-input`, and `radio-group-indicator`.
