## API Reference

Ark UI does not ship a dedicated close-button primitive. `CloseButton.Root` is an Ark-aligned
standalone wrapper built with [`@ark-ui/react/factory`](https://ark-ui.com/docs/guides/composition)
and styled through Ark
[`data-scope` and `data-part` hooks](https://ark-ui.com/docs/guides/styling).

## Basic

Use `CloseButton.Root` when the control itself performs a dismiss or close action. It provides
button semantics and a default close glyph, but it does not manage overlay state by itself.

## Anatomy

`CloseButton` exposes one Ark-style root part.

```text
CloseButton.Root
└─ root[data-scope="close-button"][data-part="root"][data-slot="close-button-root"]
   └─ CloseIcon (default) | custom children
```

| Part               | Role                                                                     |
| ------------------ | ------------------------------------------------------------------------ |
| `CloseButton.Root` | Interactive dismiss control built on button semantics and accessibility. |

## Composition

### Recommended composition

Use `CloseButton.Root` directly when a native button should own the close action. The wrapper
forwards Ark factory button props, native attributes, event handlers, and refs.

### When to go custom

Use `asChild` when another button component must own the DOM node. Ark composition requires one
child element, and that child must include its own icon content.

If an overlay component already ships a close trigger, prefer that trigger so it can manage
dismissal state and focus lifecycle.

### Default props

| Prop         | Default                                         | Values / Notes                             |
| ------------ | ----------------------------------------------- | ------------------------------------------ |
| `type`       | `button`                                        | Applied only to the native root            |
| `aria-label` | `Close` only on the unlabeled default-icon path | Prefer a surface-specific label in real UI |
| `children`   | `CloseIcon`                                     | Replaced by custom icon content            |
| `asChild`    | `false`                                         | Ark factory single-child composition       |

### Behavioral notes

- `CloseButton.Root` does not close anything by itself; wire `onClick` or use an overlay close
  trigger.
- Custom icon children need an accessible name through `aria-label` or `aria-labelledby`.
- Native `disabled` and `aria-disabled="true"` expose `data-disabled` for Ark-style state
  targeting.
- The previous Base UI `render`, `nativeButton`, and `focusableWhenDisabled` props are removed.
- `CloseButton` is a namespace object and must be rendered through `CloseButton.Root`.

## Examples

### Custom Children

Replace the default glyph through `children` while keeping the root button and styling contract.

<Preview>
  <CloseButtonCustomChildrenExample />

  <Preview.Code>
    {`
      import { CloseButton } from "moduix";

      function CircleXIcon() {
        return (
          <svg viewBox="0 0 24 24" fill="none" aria-hidden="true" stroke="currentColor">
            <circle cx="12" cy="12" r="10" />
            <path d="m15 9-6 6" />
            <path d="m9 9 6 6" />
          </svg>
        );
      }

      export function CloseButtonCustomChildrenDemo() {
        return (
          <CloseButton.Root className="customChildrenButton" aria-label="Close panel">
            <CircleXIcon />
          </CloseButton.Root>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`

        --close-button-size: 2.25rem;

      }
    `}

  </Preview.CSS>
</Preview>

Use `asChild` when an existing button component must own the rendered element while receiving the

### Styling hooks

- `data-scope="close-button"` identifies the component family.
- `data-part="root"` identifies the only exported part.
- `data-slot="close-button-root"` provides the moduix slot hook.
- `data-disabled`, `:disabled`, and `[aria-disabled="true"]` expose disabled styling.
- Direct nested SVG children inherit sizing from `--close-button-icon-size`.
