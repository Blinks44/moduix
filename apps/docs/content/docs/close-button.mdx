## Basic

Use `CloseButton` when the control itself performs a dismiss or close action. It provides the
button semantics and close glyph, but it does not manage overlay state by itself.

## Anatomy

`CloseButton` is a single icon-button root.

```text
CloseButton
└─ CloseIcon (default) | custom children
```

| Part          | Role                                                                     |
| ------------- | ------------------------------------------------------------------------ |
| `CloseButton` | Interactive dismiss control built on button semantics and accessibility. |

## Composition

### When to stay high-level

Stay with `CloseButton` directly for dialogs, notifications, cards, lightboxes, and similar
surfaces where one icon-only control dismisses the UI.

The wrapper keeps Base UI `Button` behavior intact. Use it for `onClick`, `disabled`,
`focusableWhenDisabled`, `nativeButton`, `render`, refs, and native button attributes.

### When to go custom

There are no exported subparts or slot maps. The custom path here is small:

- pass `children` to replace the default `CloseIcon`;
- pass `className` to style the root;
- use Base UI `render` only when button semantics still remain correct.

If a component already ships its own close primitive, prefer that part over a standalone
`CloseButton`.

### Default props

| Prop         | Default                                                           | Values / Notes                             |
| ------------ | ----------------------------------------------------------------- | ------------------------------------------ |
| `type`       | `button`                                                          | Override only for a real form workflow     |
| `aria-label` | `Close` only when no children, `aria-label`, or `aria-labelledby` | Prefer a surface-specific label in real UI |
| `children`   | `CloseIcon`                                                       | Any icon-sized React node                  |

### Behavioral notes

- `CloseButton` does not close anything by itself; wire the behavior with `onClick` or the
  surrounding overlay primitive.
- When you replace the default icon, provide an accessible name with `aria-label` or
  `aria-labelledby`.
- Do not use `CloseButton` for navigation links. Keep real button semantics on this component.

## Styling

Use CSS variables on the root when size, icon treatment, or interaction colors need to change.
Because the component has one public part, `className` on the root is the main styling hook.

### CSS Properties

### Styling hooks

- `CloseButton` exposes `data-slot="close-button"` on the root.
- The root accepts `className` and Base UI state callback class names through `mergeClassName`.
- Direct nested SVG children inherit sizing from `--close-button-icon-size`.
- Disabled visuals and pointer behavior are exposed through both `:disabled` and `[data-disabled]`.
