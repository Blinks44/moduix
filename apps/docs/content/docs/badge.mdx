## API Reference

`Badge` is a standalone moduix component. Use `variant` for the built-in visual
styles, `className` or CSS variables for local overrides, and `BadgeDot` as an
optional status marker.

## Anatomy

`Badge` can render text only, with `BadgeDot`, or with an inline icon.

```text
Badge
└─ content
   └─ BadgeDot (optional)
   └─ svg icon (optional)
```

```tsx
<Badge variant="default">
  <BadgeDot />
  Online
</Badge>
```

| Part       | Role                                                                                 |
| ---------- | ------------------------------------------------------------------------------------ |
| `Badge`    | Root slot. Applies variant, border, spacing, and text truncation.                    |
| `BadgeDot` | Optional indicator slot. Renders a circular marker that inherits current text color. |

`Badge` does not use service slots such as `portal`, `positioner`, `backdrop`, or `viewport`.

## Composition

Pass `className` to `Badge` and `BadgeDot` when integrating with CSS Modules, Tailwind CSS, or
CSS-in-JS. Place icons directly before or after the label; direct child `svg` icons inherit
`currentColor` and use `--badge-icon-size`. The root accepts standard `span` props so you can add
`title`, `aria-*`, and testing attributes without extra wrappers.

Keep the visible status in text. `BadgeDot` is decorative and is hidden from assistive
technology, so color or the dot alone should not be the only signal. Long labels stay on one line
and are truncated by the root; add `title` when the full value should remain discoverable. For
one-off size changes, prefer `className` or local CSS variables instead of extra badge props.

| Prop      | Default   | Values                                                    |
| --------- | --------- | --------------------------------------------------------- |
| `variant` | `default` | `default`, `secondary`, `destructive`, `outline`, `ghost` |

## Examples

### Variants

Use shadcn-like variants: `default`, `secondary`, `destructive`, `outline`, `ghost`.

<Preview>
  <BadgeVariantsExample />

  <Preview.Code>
    {`
      import { Badge } from "moduix";

      const variants = ["default", "secondary", "destructive", "outline", "ghost"] as const;

      export function BadgeVariantsDemo() {
        return (
          <div>
            {variants.map((variant) => (
              <Badge key={variant} variant={variant}>
                {variant}
              </Badge>
            ))}
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Sizes

Adjust size with local CSS variables when a context needs a smaller or larger badge.

<Preview>
  <BadgeSizesExample />

  <Preview.Code>
    {`
      import { Badge } from "moduix";
      import styles from "./badge-demo.module.css";

      export function BadgeSizesDemo() {
        return (
          <>
            <Badge className={styles.small}>Small</Badge>
            <Badge>Default</Badge>
            <Badge className={styles.large}>Large</Badge>
          </>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .small {
        --badge-height: 1.125rem;
        --badge-font-size: var(--text-xs);
        --badge-icon-size: 0.6875rem;
        --badge-line-height: var(--line-height-text-xs);
        --badge-padding-x: 0.5rem;
      }

      .large {
        --badge-height: 1.5rem;
        --badge-font-size: var(--text-sm);
        --badge-icon-size: 0.875rem;
        --badge-line-height: var(--line-height-text-sm);
        --badge-padding-x: 0.75rem;
      }
    `}

  </Preview.CSS>
</Preview>
