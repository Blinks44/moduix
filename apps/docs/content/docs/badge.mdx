## API Reference

`Badge` is a standalone moduix component for short status labels, categories, and small counters.
Its Ark migration exposes explicit parts: `Badge.Root` and `Badge.Dot`.

## Choosing the right component

Use `Badge` when the UI is showing metadata about something else rather than the selected value
itself.

- Choose **`Badge`** for short metadata such as status, category, version, or count.
- Choose **`Tag`** when the UI is showing selected values, filters, assignments, or labels that belong to the current state of the interface.

## Basic

Start with `Badge.Root` only. Add `variant` when the default tone is not enough, and add `Badge.Dot`
only when the status needs a grouped indicator.

## Anatomy

`Badge` keeps the public surface small and exposes only the visible parts.

```text
Badge.Root
├─ content
├─ Badge.Dot (optional)
└─ svg icon (optional)
```

| Part         | Role                                                                                 |
| ------------ | ------------------------------------------------------------------------------------ |
| `Badge.Root` | Root slot. Applies variant, border, spacing, and text truncation.                    |
| `Badge.Dot`  | Optional indicator slot. Renders a circular marker that inherits current text color. |

## Composition

### Recommended composition

Stay with `Badge.Root` directly for most status labels, categories, version pills, and short
counters.

Use `variant` for the built-in tones: `default`, `secondary`, `destructive`, `outline`, `ghost`.

Place `Badge.Dot` or a direct child icon next to the label when a badge needs an extra visual cue.

### When to go custom

Switch to local CSS variables or `className` overrides when the surface needs custom sizing or a
different tone. `Badge.Root` stays a plain Ark factory `span`, so the styling path should stay
simple.

### Default props

| Prop      | Default   | Values / Notes                                            |
| --------- | --------- | --------------------------------------------------------- |
| `variant` | `default` | `default`, `secondary`, `destructive`, `outline`, `ghost` |

### Behavioral notes

- `Badge.Root` is not a button or link. If the label must be interactive, compose it with `asChild`
  and a real interactive element.
- `Badge.Dot` is decorative and hidden from assistive technology, so the visible text must carry the status meaning.
- Long labels stay on one line and are truncated by the root; add `title` when the full value should remain discoverable.
- Direct child `svg` icons inherit `currentColor` and use `--badge-icon-size`.

### Styling hooks

- `data-slot="badge-root"` and `data-slot="badge-dot"` are stable selectors on the exported parts.
- `Badge.Root` also exposes `data-scope="badge"` and `data-part="root"`. `Badge.Dot` exposes `data-part="dot"`.
- `Badge.Root` writes the selected tone to `data-variant`.
- Direct child `svg` icons inherit `currentColor`, do not receive pointer events, and are sized with `--badge-icon-size`.
