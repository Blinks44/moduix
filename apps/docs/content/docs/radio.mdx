## Choosing the right component

Use `Radio` when users must choose exactly one option and the available options should stay visible.

- Choose **`Radio`** for a short, always-visible single-choice list.
- Choose **`Select`** when the same single-choice input should stay more compact.
- Choose **`Checkbox`** when each option is independent or multiple options can be selected.

## Basic

Start with `RadioGroup`, `RadioGroupLabel`, `RadioGroupList`, and `RadioField`. This is the
recommended path for forms, settings panels, and short option lists.

## Anatomy

The normal consumer path is short:

```text
RadioGroup
├─ RadioGroupLabel
└─ RadioGroupList
   └─ RadioField
      ├─ Radio
      │  └─ RadioIndicator
      │     └─ RadioIndicatorIcon
      └─ RadioLabel
```

When `children` are omitted, `Radio` renders the built-in `RadioIndicator`, and that indicator
renders the default `RadioIndicatorIcon`.

| Part                 | Role                                                                    |
| -------------------- | ----------------------------------------------------------------------- |
| `RadioGroup`         | Shared value state, roving focus, keyboard navigation, and form wiring. |
| `RadioGroupLabel`    | Visible group heading. Pair it with `aria-labelledby` on `RadioGroup`.  |
| `RadioGroupList`     | Layout wrapper for radio rows.                                          |
| `RadioField`         | Optional wrapping `<label>` for the common clickable row layout.        |
| `Radio`              | Interactive control root with hidden input and `size` support.          |
| `RadioIndicator`     | Optional checked-state visual container for custom composition.         |
| `RadioIndicatorIcon` | Built-in dot visual rendered by CSS.                                    |
| `RadioLabel`         | Styled text wrapper for label content.                                  |

## Composition

### When to stay high-level

Stay with `RadioGroup`, `RadioField`, `Radio`, and `RadioLabel` for most forms and settings
panels.

`RadioGroup` keeps the Base UI group behavior intact. Use it for `value`, `defaultValue`,
`onValueChange`, `name`, `required`, `readOnly`, `disabled`, `form`, and `inputRef`.

`RadioField` is the recommended wrapper when the control and text can live in one clickable label.

### When to go custom

Switch to direct `Radio` composition when the checked indicator needs a different shape or markup.
Render the root as a native button with `nativeButton` and `render={<button />}` when layout
requires a separate sibling `<label>`.

`RadioLabel` is only a styled `span`. It does not create labeling on its own outside real label
semantics.

### Default props

| Prop        | Default | Values                       |
| ----------- | ------- | ---------------------------- |
| `size`      | `md`    | `xs`, `sm`, `md`, `lg`, `xl` |
| `className` | -       | `string` or Base UI callback |

### Behavioral notes

- Use `value` with `onValueChange` for controlled groups, or `defaultValue` for uncontrolled ones.
- Put the accessible group name on `RadioGroup`, usually with `aria-labelledby={labelId}` and a
  matching `RadioGroupLabel`.
- Use `disabled` to block interaction for the whole group.
- Use `readOnly` when the current selection should stay visible and focusable but must not change.

## Examples

### Controlled

Use controlled mode when the selected value drives surrounding UI.

<Preview>
  <ControlledRadioExample />

  <Preview.Code>
    {`
      import {
        Radio,
        RadioField,
        RadioGroup,
        RadioGroupLabel,
        RadioGroupList,
        RadioLabel,
      } from "moduix";
      import { useId, useState } from "react";

      export function ControlledRadioDemo() {
        const labelId = useId();
        const [value, setValue] = useState("personal");

        return (
          <RadioGroup aria-labelledby={labelId} value={value} onValueChange={setValue}>
            <RadioGroupLabel id={labelId}>Workspace Visibility</RadioGroupLabel>
            <RadioGroupList>
              <RadioField>
                <Radio value="personal" />
                <RadioLabel>Only me</RadioLabel>
              </RadioField>
              <RadioField>
                <Radio value="team" />
                <RadioLabel>Team</RadioLabel>
              </RadioField>
            </RadioGroupList>
          </RadioGroup>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Styling hooks

Use the part that owns the visual concern:

- `Radio` for control chrome, focus ring, checked state, and size-dependent styling
- `RadioIndicator` and `RadioIndicatorIcon` for custom checked visuals
- `RadioField` and `RadioLabel` for row layout and label typography
- `RadioGroup`, `RadioGroupLabel`, and `RadioGroupList` for group spacing and heading styling

The main public hooks include `data-slot="radio-root"`, `data-slot="radio-indicator"`,
`data-slot="radio-label"`, `data-slot="radio-group-root"`, and `data-slot="radio-group-list"`.
