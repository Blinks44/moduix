`Radio` stays close to the Base UI primitive. The default path keeps the checked indicator internal,
while `RadioGroup` owns selection state and form behavior.

## Anatomy

Use `RadioField` when the radio and label can live inside one wrapping `<label>`.

```text
RadioGroup
├─ RadioGroupLabel
└─ RadioGroupList
   └─ RadioField
      ├─ Radio
      │  └─ RadioIndicator
      │     └─ RadioIndicatorIcon
      └─ RadioLabel
```

```tsx
<RadioGroup>
  <RadioGroupLabel>Account Type</RadioGroupLabel>
  <RadioGroupList>
    <RadioField>
      <Radio value="team" />
      <RadioLabel>Team account</RadioLabel>
    </RadioField>
  </RadioGroupList>
</RadioGroup>
```

| Part                 | Role                                                                               |
| -------------------- | ---------------------------------------------------------------------------------- |
| `RadioGroup`         | Shared value state, form ownership, disabled state, and readonly state.            |
| `RadioGroupLabel`    | Visible group label. Pair it with `aria-labelledby` on `RadioGroup`.               |
| `RadioGroupList`     | Layout wrapper for radio rows.                                                     |
| `RadioField`         | Optional wrapping label for the common clickable-row layout.                       |
| `Radio`              | Interactive root. Renders the default indicator when `children` is omitted.        |
| `RadioIndicator`     | Optional checked-state part for custom composition.                                |
| `RadioIndicatorIcon` | Default dot wrapper for the built-in checked indicator.                            |
| `RadioLabel`         | Optional text wrapper for label content that should inherit the radio text tokens. |

## Composition

Put selection state on `RadioGroup`: use `defaultValue` for uncontrolled mode and `value` with
`onValueChange` for controlled mode. `Radio` forwards the Base UI root props directly, so form
props such as `name`, `required`, `readOnly`, `disabled`, `form`, and `inputRef` continue to work
without extra wrapper API.

`size` is the only component-specific DX sugar. It scales the control and default dot without
changing the composition model.

When the default indicator is enough, render `<Radio />` directly. When the checked visual needs to
change, compose `RadioIndicator` explicitly:

```tsx
<Radio value="team">
  <RadioIndicator>{/* custom icon or shape */}</RadioIndicator>
</Radio>
```

When the label must be a sibling instead of a wrapping `<label>`, switch the root to a native
button:

```tsx
<div>
  <Radio id="delivery-email" nativeButton render={<button />} value="email" />
  <label htmlFor="delivery-email">Email</label>
</div>
```

### Controlled

Use controlled mode when the selected value drives surrounding UI.

<Preview>
  <ControlledRadioExample />

  <Preview.Code>
    {`
      import {
        Radio,
        RadioField,
        RadioGroup,
        RadioGroupLabel,
        RadioGroupList,
        RadioLabel,
      } from "moduix";
      import { useId, useState } from "react";

      export function ControlledRadioDemo() {
        const labelId = useId();
        const [value, setValue] = useState("personal");

        return (
          <RadioGroup aria-labelledby={labelId} value={value} onValueChange={setValue}>
            <RadioGroupLabel id={labelId}>Workspace Visibility</RadioGroupLabel>
            <RadioGroupList>
              <RadioField>
                <Radio value="personal" />
                <RadioLabel>Only me</RadioLabel>
              </RadioField>
              <RadioField>
                <Radio value="team" />
                <RadioLabel>Team</RadioLabel>
              </RadioField>
            </RadioGroupList>
          </RadioGroup>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Custom Composition

### Custom Indicator

Compose `RadioIndicator` inside `Radio` when the built-in dot is not enough.

<Preview>
  <CustomIndicatorRadioExample />

  <Preview.Code>
    {`
      import {
        Radio,
        RadioField,
        RadioGroup,
        RadioGroupLabel,
        RadioGroupList,
        RadioIndicator,
        RadioLabel,
      } from "moduix";
      import type { ComponentProps } from "react";
      import { useId } from "react";

      const options = [
        { value: "personal", label: "Personal account" },
        { value: "team", label: "Team account" },
        { value: "enterprise", label: "Enterprise account" },
      ];

      function DiamondIcon(props: ComponentProps<"svg">) {
        return (
          <svg
            width="12"
            height="12"
            viewBox="0 0 12 12"
            fill="none"
            aria-hidden="true"
            focusable="false"
            {...props}
          >
            <path d="M6 1.5L10.5 6L6 10.5L1.5 6L6 1.5Z" fill="currentColor" />
          </svg>
        );
      }

      export function CustomIndicatorRadioDemo() {
        const labelId = useId();

        return (
          <RadioGroup aria-labelledby={labelId} defaultValue="team">
            <RadioGroupLabel id={labelId}>Account Type</RadioGroupLabel>
            <RadioGroupList>
              {options.map((option) => (
                <RadioField key={option.value}>
                  <Radio value={option.value}>
                    <RadioIndicator>
                      <DiamondIcon />
                    </RadioIndicator>
                  </Radio>
                  <RadioLabel>{option.label}</RadioLabel>
                </RadioField>
              ))}
            </RadioGroupList>
          </RadioGroup>
        );
      }
    `}

  </Preview.Code>
</Preview>
