## Basic

Use Ark-style parts: `Accordion.Root`, `Accordion.Item`, `Accordion.ItemTrigger`,
`Accordion.ItemIndicator`, and `Accordion.ItemContent`.

## Anatomy

```text
Accordion.Root
└─ Accordion.Context (optional render-prop access)
└─ Accordion.Item[value]
   ├─ Accordion.ItemTrigger
   │  ├─ label
   │  ├─ Accordion.ItemContext (optional render-prop access)
   │  └─ Accordion.ItemIndicator (optional)
   └─ Accordion.ItemContent
      └─ content

Accordion.RootProvider
└─ same item tree connected to a useAccordion() store
```

| Part                           | Role                                                                                      |
| ------------------------------ | ----------------------------------------------------------------------------------------- |
| `Accordion` / `Accordion.Root` | Controls accordion state and keyboard behavior.                                           |
| `Accordion.RootProvider`       | Connects parts to state created with `useAccordion()`.                                    |
| `Accordion.Item`               | One collapsible item with a required `value`.                                             |
| `Accordion.ItemTrigger`        | Interactive button that opens or closes the item.                                         |
| `Accordion.ItemIndicator`      | Visual open/close cue. Defaults to the moduix plus icon when no children are provided.    |
| `Accordion.ItemContent`        | Collapsible content region. Keep spacing wrappers inside for clean `--height` animations. |
| `Accordion.Context`            | Reads root state through an Ark render prop.                                              |
| `Accordion.ItemContext`        | Reads item state through an Ark render prop.                                              |

## Composition

### Recommended composition

`Accordion.Root` owns keyboard navigation, focus movement, and open item state. Use it for
`value`, `defaultValue`, `onValueChange(details)`, `onFocusChange(details)`, `multiple`,
`collapsible`, `orientation`, `lazyMount`, `unmountOnExit`, and `ids`.

Stay with `Accordion.Root`, `Accordion.Item`, `Accordion.ItemTrigger`, `Accordion.ItemIndicator`,
and `Accordion.ItemContent` for the standard Ark composition path.

### When to go custom

`value` and `defaultValue` are always arrays, including single-open mode. `onValueChange` follows
the Ark callback shape and exposes the current item values as `details.value`.

Use the Ark `RootProvider` and `useAccordion()` pattern when accordion state must be created
outside the rendered subtree.

Use `Accordion.Context`, `Accordion.ItemContext`, `useAccordionContext()`, or
`useAccordionItemContext()` when surrounding UI needs to react to root or item state without adding
a separate state layer.

`Accordion.Item` requires a stable `value`. Use `disabled` on the item when one section should stay
present but unavailable.

`Accordion.ItemIndicator` is optional. In moduix it renders `PlusIcon` by default when no children
are provided, but you can replace the indicator content entirely.

Use `asChild` only when another element must own the rendered node. Keep the Ark part tree visible
instead of hiding it behind convenience wrappers.

### Default Props

| Prop                   | Default    | Notes                                                 |
| ---------------------- | ---------- | ----------------------------------------------------- |
| `collapsible`          | `false`    | Keeps one expanded item open until another is chosen. |
| `lazyMount`            | `false`    | Mounts item content immediately.                      |
| `multiple`             | `false`    | Allows a single expanded item by default.             |
| `orientation`          | `vertical` | Uses vertical keyboard and layout behavior.           |
| `unmountOnExit`        | `false`    | Keeps collapsed content mounted.                      |
| `value`/`defaultValue` | `[]`       | Values are string arrays, including single-open mode. |

### Behavioral notes

Keep spacing wrappers inside `Accordion.ItemContent` instead of on the content part itself. Ark
measurement variables drive the open/close animation: `--height` for vertical accordions and
`--width` for horizontal accordions.

All exported parts support `asChild` for cases where another element must own the rendered node.

### Controlled

Control `value` from React state when the open items need to stay in sync with routing, filters, or surrounding UI.

<Preview>
  <ControlledAccordionExample />
  <Preview.Code>
    {`
      import { Accordion } from "@moduix/react";
      import { useState } from "react";

      export function ControlledAccordionDemo() {
        const [value, setValue] = useState(["getting-started"] as string[]);

        return (
          <Accordion value={value} onValueChange={(details) => setValue(details.value)}>
            {items.map((item) => (
              <Accordion.Item key={item.value} value={item.value}>
                <Accordion.ItemTrigger>
                  {item.title}

### Root Provider

Use `useAccordion()` with `Accordion.RootProvider` when the state store must be created outside
the rendered accordion tree. Do not render `Accordion.Root` for the same store.

<Preview>
  <RootProviderAccordionExample />
  <Preview.Code>
    {`
      import { Accordion, useAccordion } from "@moduix/react";

      export function RootProviderAccordionDemo() {
        const accordion = useAccordion({ defaultValue: ["what-is-ark-ui"] });

        return (
          <>
            <div className="accordion-provider-stack">
              <div className="accordion-state">Open sections: {accordion.value.join(", ")}</div>
              <Accordion.RootProvider value={accordion}>
                {items.map((item) => (
                  <Accordion.Item key={item.value} value={item.value}>
                    <Accordion.ItemTrigger>
                      {item.title}
                      <Accordion.ItemIndicator />
                    </Accordion.ItemTrigger>
                    <Accordion.ItemContent>
                      <div className="accordion-panel-content">{item.description}</div>
                    </Accordion.ItemContent>
                  </Accordion.Item>
                ))}
              </Accordion.RootProvider>
            </div>
          </>
        );

### Item State

<Preview>
  <ItemStateAccordionExample />

      export function ItemStateAccordionDemo() {
        return (
          <Accordion defaultValue={["what-is-ark-ui"]}>
            {items.map((item) => (
              <Accordion.Item
                key={item.value}
                value={item.value}
                disabled={item.value === "getting-started"}
              >
                <Accordion.ItemTrigger>
                  {item.title}
                  <Accordion.ItemContext>
                    {(itemState) => (
                      <span className="accordion-item-state">
                        {itemState.expanded ? "Open" : "Closed"}
                        {itemState.focused ? " / Focused" : ""}
                        {itemState.disabled ? " / Disabled" : ""}
                      </span>
                    )}
                  </Accordion.ItemContext>
                  <Accordion.ItemIndicator />
                </Accordion.ItemTrigger>
                <Accordion.ItemContent>
                  <div className="accordion-panel-content">{item.description}</div>
                </Accordion.ItemContent>

## Styling

Use `className` on exported parts for structural styling changes. For state styling, prefer Ark
data attributes (`data-state`, `data-disabled`, `data-orientation`) and moduix `data-slot` hooks.

### Styling hooks

- `data-slot="accordion-root"`, `accordion-root-provider`, `accordion-item`,
  `accordion-item-trigger`, `accordion-item-indicator`, and `accordion-item-content`.
- Ark anatomy hooks: `data-scope="accordion"` and `data-part="root|item|item-trigger|item-indicator|item-content"`.
- Ark state hooks: `data-state="open|closed"`, `data-disabled`, `data-focus`,
  `data-controls`, and `data-orientation="horizontal|vertical"`.
- Ark content variables: `--height` for vertical animation and `--width` for horizontal animation.
