## Anatomy

`Accordion` is a stack of independent items. Keep each `AccordionTrigger` and `AccordionPanel`
inside the same `AccordionItem` so state and accessibility bindings stay paired.

```text
Accordion
└─ AccordionItem[value]
   ├─ AccordionHeader
   │  └─ AccordionTrigger
   │     ├─ label
   │     └─ AccordionTriggerIcon
   └─ AccordionPanel
      └─ content
```

```tsx
<Accordion defaultValue={['shipping']}>
  <AccordionItem value="shipping">
    <AccordionHeader>
      <AccordionTrigger>
        Shipping details
        <AccordionTriggerIcon />
      </AccordionTrigger>
    </AccordionHeader>
    <AccordionPanel>
      <div className={styles.panelContent}>Delivery times, prices, and tracking options.</div>
    </AccordionPanel>
  </AccordionItem>
</Accordion>
```

| Part                   | Role                                                                                                             |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `Accordion`            | Root state machine. Controls open values, keyboard navigation, `multiple`, `disabled`, and `orientation`.        |
| `AccordionItem`        | One collapsible row. Its `value` identifies the row for `defaultValue`, controlled `value`, and change handlers. |
| `AccordionHeader`      | Semantic heading wrapper for a trigger. Use `render` when the heading element needs to change.                   |
| `AccordionTrigger`     | Interactive button that opens or closes the matching panel.                                                      |
| `AccordionTriggerIcon` | Optional helper that renders the default plus icon. Replace its children or omit it when you want a custom cue.  |
| `AccordionPanel`       | The collapsible content region. Put padding/content wrappers inside it so panel height animations stay clean.    |

## Composition

Use `Accordion` for state and behavior (`value`, `defaultValue`, `onValueChange`, `multiple`,
`disabled`, `orientation`, `loopFocus`). Use `keepMounted` or `hiddenUntilFound` on the root or
panel when closed content must remain in the DOM.

Style public parts with `className` on the exported parts. Base UI behavior props (for example
`render`, `keepMounted`, `hiddenUntilFound`) pass through to the matching primitive part.

Keep the structure explicit: wrap each trigger in `AccordionHeader`, and render
`AccordionTriggerIcon` only when you want a visual indicator. This keeps the component close to the
Base UI primitive and avoids trigger-specific slot APIs.

```tsx
<AccordionItem value="shipping">
  <AccordionHeader render={<h3 />}>
    <AccordionTrigger>
      Shipping details
      <AccordionTriggerIcon />
    </AccordionTrigger>
  </AccordionHeader>
  <AccordionPanel>
    <div className={styles.panelContent}>Delivery times, prices, and tracking options.</div>
  </AccordionPanel>
</AccordionItem>
```

| Prop          | Default    | Values                   |
| ------------- | ---------- | ------------------------ |
| `multiple`    | `false`    | `true`, `false`          |
| `disabled`    | `false`    | `true`, `false`          |
| `orientation` | `vertical` | `vertical`, `horizontal` |
| `loopFocus`   | `true`     | `true`, `false`          |

## Examples

### Multiple

Use `multiple` to allow multiple open items.

<Preview>
  <AccordionExample multiple defaultValue={["what-is-base-ui", "can-i-use-it"]} />

  <Preview.Code>
    {`
      import {
        Accordion,
        AccordionHeader,
        AccordionItem,
        AccordionPanel,
        AccordionTrigger,
        AccordionTriggerIcon,
      } from "moduix";

      export function MultipleAccordionDemo() {
        return (
          <Accordion multiple defaultValue={["what-is-base-ui", "can-i-use-it"]}>

              <AccordionItem key={item.value} value={item.value}>
                <AccordionHeader>
                  <AccordionTrigger>
                    {item.title}
                    <AccordionTriggerIcon />
                  </AccordionTrigger>
                </AccordionHeader>
                <AccordionPanel>
                  <div className={styles.panelContent}>{item.description}</div>
                </AccordionPanel>
              </AccordionItem>
            ))}
          </Accordion>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Controlled

Control the open panels from React state when the accordion needs to coordinate with other UI.

<Preview>
  <ControlledAccordionExample />

  <Preview.Code>
    {`
      import {
        Accordion,
        AccordionHeader,
        AccordionItem,
        AccordionPanel,
        AccordionTrigger,
        AccordionTriggerIcon,
      } from "moduix";
      import { useState } from "react";

      export function ControlledAccordionDemo() {
        const [value, setValue] = useState(["getting-started"] as string[])

        return (
          <Accordion value={value} onValueChange={setValue}>
            {items.map((item) => (
              <AccordionItem key={item.value} value={item.value}>
                <AccordionHeader>
                  <AccordionTrigger>
                    {item.title}
                    <AccordionTriggerIcon />
                  </AccordionTrigger>
                </AccordionHeader>
                <AccordionPanel>
                  <div className={styles.panelContent}>{item.description}</div>
                </AccordionPanel>
              </AccordionItem>
            ))}
          </Accordion>
        );
      }
    `}

  </Preview.Code>

</Preview>

### Custom Icon

Compose a custom indicator by passing children to `AccordionTriggerIcon`.

<Preview>
  <CustomCompositionAccordionExample />

  <Preview.Code>
    {`

        Accordion,
        AccordionHeader,
        AccordionItem,
        AccordionPanel,
        AccordionTrigger,
        AccordionTriggerIcon,
        ChevronDownIcon,
      } from "moduix";

      export function CustomCompositionAccordionDemo() {
        return (
          <Accordion defaultValue={["what-is-base-ui"]}>

              <AccordionItem key={item.value} value={item.value}>
                <AccordionHeader>

                    {item.title}

                    </AccordionTriggerIcon>
                  </AccordionTrigger>
                </AccordionHeader>
                <AccordionPanel>
                  <div className={styles.panelContent}>{item.description}</div>

      }
