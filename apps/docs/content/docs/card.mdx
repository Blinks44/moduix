## API Reference

`Card` is a local moduix component. There is no upstream Base UI primitive for this API.

## Basic

Start with `Card`, `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, and `CardFooter`.
This is the recommended path for summaries, settings panels, and dashboard surfaces.

## Anatomy

The normal consumer path is short:

```text
Card
├─ CardHeader
│  ├─ CardTitle
│  │  └─ CardLink (optional)
│  ├─ CardDescription
│  └─ CardAction (optional)
├─ CardContent
└─ CardFooter
```

| Part              | Role                                                                                   |
| ----------------- | -------------------------------------------------------------------------------------- |
| `Card`            | Root surface. Supports `className`, `render`, native props, and `size="sm"`.           |
| `CardHeader`      | Header grid for title, description, and an optional trailing action.                   |
| `CardTitle`       | Heading part. Defaults to `h3` and supports `as`.                                      |
| `CardDescription` | Muted supporting text under the title.                                                 |
| `CardAction`      | Optional compact header-side action slot.                                              |
| `CardContent`     | Main body area with content spacing and body text styles.                              |
| `CardFooter`      | Wrapping footer row for actions or secondary metadata.                                 |
| `CardLink`        | Link that stretches its hit area across the card without nesting interactive elements. |

## Composition

### When to stay high-level

Stay with the standard parts when the card is grouping content on a visible surface.

- Use `CardHeader`, `CardTitle`, and `CardDescription` for the primary heading block.
- Use `CardContent` for body copy, metrics, media summaries, or form fields.
- Use `CardFooter` for actions that may need to wrap.
- Use `CardAction` only for a compact badge, menu trigger, or one small control aligned to the
  trailing edge of the header.

### When to go custom

`Card` does not expose a lower-level primitive stack. The custom path is plain composition:

- Render the root as another element with `render`, such as `render={<a href="..." />}`, when the
  whole card is a single navigation target.
- Use `CardLink` inside `CardTitle` when the card should navigate but still contains nested
  interactive controls.
- Add plain children before the header for media blocks or other custom structure.
- Override visuals with `className` and `--card-*` variables without changing the part layout.

### Default props

| Part        | Prop   | Default     | Notes                                   |
| ----------- | ------ | ----------- | --------------------------------------- |
| `Card`      | `size` | `'default'` | Accepts `'default' \| 'sm'`.            |
| `CardTitle` | `as`   | `'h3'`      | Accepts heading tags `h1` through `h6`. |

### Behavioral notes

- `Card` is presentational by default and renders a `div`.
- Use `render={<a href="..." />}` only when the card has no nested buttons, links, inputs, menus,
  or other focusable controls.
- Use `CardLink` for the overlay-link pattern when the card needs both navigation and nested
  actions.
- `size="sm"` reduces padding and title typography for denser dashboards and side panels.
- `CardContent` removes the first and last direct-child margins and only adds top padding after a
  preceding part.

## Styling

Style `Card` through `className`, the stable `data-slot` hooks, `data-size` on the root, and the
public `--card-*` variables from `theme.css`.

### CSS Properties

  <Tabs items={['CSS Variables']} className="mb-0 mt-3">

      <CardCssPropertiesPanel />

### Styling hooks

- `data-slot="card"` owns the root surface, border, radius, background, and size-specific tokens.
- `data-slot="card-header"`, `card-content`, and `card-footer` are the main layout hooks for
  spacing overrides.
- `data-slot="card-title"` and `card-description` own heading typography and text color.
- `data-slot="card-action"` stays above the `CardLink` overlay and is the right hook for compact
  trailing controls.
- `data-slot="card-link"` owns the overlay-link focus ring and clickable surface.
- `Card` writes `data-size="default"` or `data-size="sm"` on the root for size-specific styling.
