## API Reference

`Card` is a standalone presentational component. It does not wrap a Base UI primitive, so the API stays intentionally small: a styled root, a compact size option, and composable parts.

## Composition

Use the following composition to build a `Card`:

```text
Card
├─ CardHeader
│  ├─ CardTitle
│  ├─ CardDescription
│  └─ CardAction (optional)
├─ CardContent
└─ CardFooter
```

```tsx
<Card>
  <CardHeader>
    <CardTitle>Billing plan</CardTitle>
    <CardDescription>Team workspace, monthly billing.</CardDescription>
  </CardHeader>
  <CardContent>...</CardContent>
  <CardFooter>...</CardFooter>
</Card>
```

| Part              | Role                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| `Card`            | Root container. Supports `className`, native props, and `size="sm"`. |
| `CardHeader`      | Header layout for title, description, and action content.            |
| `CardContent`     | Main content area with card body spacing.                            |
| `CardFooter`      | Footer layout for actions or secondary content.                      |
| `CardTitle`       | Card heading. Defaults to `h3` and supports `as`.                    |
| `CardDescription` | Muted supporting text under the title.                               |
| `CardAction`      | Right-aligned header action slot.                                    |

## Usage Notes

- Prefer composition over root-level behavior flags. If the whole card should be interactive, use a link or button inside the card instead of turning the root into a custom interactive wrapper.
- Use `size="sm"` for denser cards. The default size covers standard page and dashboard layouts.
- Use `className` and `--card-*` variables for visual customization.
