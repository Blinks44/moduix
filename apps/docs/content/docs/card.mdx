## API Reference

`Card` is a standalone moduix component and does not wrap a Base UI primitive.
Use its compound slots for consistent content structure.

```tsx

  <CardHeader>
    <CardTitle>Billing plan</CardTitle>
    <CardDescription>Team workspace, monthly billing.</CardDescription>
  </CardHeader>
  <CardBody>...</CardBody>
  <CardFooter withDivider>...</CardFooter>
</Card>
```

| Part              | Role                                                                        |
| ----------------- | --------------------------------------------------------------------------- |
| `Card`            | Root slot. Applies size, variant, border, radius, shadow, and text color.   |
| `CardHeader`      | Header layout for title, description, and optional action content.          |
| `CardBody`        | Main content area. Trims first and last child margins for predictable flow. |
| `CardFooter`      | Footer layout for actions or metadata. Supports alignment and dividers.     |
| `CardTitle`       | Heading slot for the primary card label.                                    |
| `CardDescription` | Muted supporting text for the card title.                                   |
| `CardAction`      | Right-aligned header action slot.                                           |

## Composition

Use `variant` to set visual weight and `size` to control density.
`Card` supports `as` for sectioning semantics, `render` for semantic interactive roots such as links
and buttons, and `interactive` for hover/focus affordance.
Prefer `render={<a />}` or `render={<button />}` over `tabIndex` on non-interactive elements.
`CardTitle` defaults to `h3` and supports `as` for heading-level control.
Pass `className` to slots and customize visuals with `--card-*` variables.

| Prop           | Default   | Values                                             |
| -------------- | --------- | -------------------------------------------------- |
| `render`       | â€”         | Rendered root element for semantic link/button use |
| `variant`      | `default` | `default`, `elevated`, `outline`, `ghost`          |
| `size`         | `md`      | `xs`, `sm`, `md`, `lg`                             |
| `interactive`  | `false`   | `true`, `false`                                    |
| `as`           | `div`     | `div`, `article`, `section`, `aside`               |
| `align`        | `end`     | `start`, `center`, `end`, `between` (`CardFooter`) |
| `withDivider`  | `false`   | `true`, `false` (`CardHeader` and `CardFooter`)    |
| `CardTitle as` | `h3`      | `h1`, `h2`, `h3`, `h4`, `h5`, `h6`                 |

### Interactive

Use `interactive` when app logic makes the card selectable or navigable.
Pair it with `render` so the root element keeps correct semantics.

<Preview>
  <InteractiveCardExample />

  <Preview.Code>
    {`
      import { Badge, Card, CardAction, CardDescription, CardHeader, CardTitle } from "moduix";

      export function InteractiveCardDemo() {
        return (
          <Card
            interactive
            render={<a href="/imports/42" />}
            aria-label="Open import report"
          >
            <CardHeader>
              <CardTitle as="h2">Data import completed</CardTitle>
              <CardDescription>42,816 rows processed without validation errors.</CardDescription>
              <CardAction>
                <Badge>New</Badge>
              </CardAction>
            </CardHeader>
          </Card>
        );
      }
    `}

  </Preview.Code>
