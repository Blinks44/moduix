## API Reference

`Card` is a local moduix component. There is no dedicated Ark UI Card primitive, so moduix models it
with `@ark-ui/react/factory` and Chakra's Card anatomy.

## Basic

Start with `Card.Root`, `Card.Header`, `Card.Title`, `Card.Description`, `Card.Body`, and
`Card.Footer`.

## Anatomy

```text
Card.Root
├─ Card.Header
│  ├─ Card.Title
│  ├─ Card.Description
│  └─ Card.Action (optional)
├─ Card.Body
└─ Card.Footer (optional)
```

| Part               | Role                                                                         |
| ------------------ | ---------------------------------------------------------------------------- |
| `Card.Root`        | Root surface. Supports Ark root props, `size`, and `asChild`.                |
| `Card.Header`      | Header grid for title, description, and optional trailing action.            |
| `Card.Body`        | Main body area with content spacing and body text styling.                   |
| `Card.Footer`      | Wrapping footer row for actions or secondary metadata.                       |
| `Card.Title`       | Heading part. Defaults to `h3` and uses `asChild` for custom heading levels. |
| `Card.Description` | Supporting text under the title.                                             |
| `Card.Action`      | Optional compact header-side action slot.                                    |
| `Card.Link`        | Overlay link for cards that still contain nested interactive controls.       |

### Defaults

| Part        | Prop   | Default | Notes                  |
| ----------- | ------ | ------- | ---------------------- |
| `Card.Root` | `size` | `'md'`  | Accepts `'sm' \| 'md'` |

### Behavioral notes

- `Card.Root` is presentational by default and renders a `div`.
- Use `Card.Root asChild` when the entire card is one link and there are no nested interactive
  controls.
- Use `Card.Link` when the card should navigate and still contains nested actions.
- `Card.Body` removes first and last direct-child margins and only adds top padding after a
  preceding part.
- `Card.Title` uses `asChild` instead of an `as` prop for heading changes.

## Styling

Style `Card` through `className`, `data-slot`, Ark `data-scope` / `data-part` hooks, and the public
`--card-*` variables from `theme.css`.

### CSS Properties

Stable hooks:

- `data-slot="card-root"`, `card-header`, `card-body`, `card-footer`
- `data-slot="card-title"`, `card-description`, `card-action`, `card-link`
- `data-scope="card"`
- `data-part="root" | "header" | "body" | "footer" | "title" | "description" | "action" | "link"`
- `data-size="sm" | "md"`

## Accessibility

- `Card.Root` is a `div` by default. Use `asChild` when another semantic element should own the DOM
  node.
- Avoid a linked root when the card contains buttons, menus, inputs, checkboxes, or nested links.
- `Card.Title` defaults to `h3`; use `asChild` when the outline needs another heading level.
- `Card.Action` is layout only and does not create ARIA relationships.
