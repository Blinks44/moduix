## API Reference

`Card` is a standalone moduix component and does not wrap a Base UI primitive.
Use the compound slots to keep layouts consistent across summaries, dashboards, settings panels,
and object previews.

## Anatomy

`Card` exposes semantic slots for common card structure. Use only the parts that fit the content.

```text
Card
├─ CardHeader
│  ├─ CardTitle
│  ├─ CardDescription
│  └─ CardAction (optional)
├─ CardBody
└─ CardFooter
```

```tsx
<Card>
  <CardHeader>
    <CardTitle>Billing plan</CardTitle>
    <CardDescription>Team workspace, monthly billing.</CardDescription>
  </CardHeader>
  <CardBody>...</CardBody>
  <CardFooter withDivider>...</CardFooter>
</Card>
```

| Part              | Role                                                                        |
| ----------------- | --------------------------------------------------------------------------- |
| `Card`            | Root slot. Applies size, variant, border, radius, shadow, and text color.   |
| `CardHeader`      | Header layout for title, description, and optional action content.          |
| `CardBody`        | Main content area. Trims first and last child margins for predictable flow. |
| `CardFooter`      | Footer layout for actions or metadata. Supports alignment and dividers.     |
| `CardTitle`       | Heading slot for the primary card label.                                    |
| `CardDescription` | Muted supporting text for the card title.                                   |
| `CardAction`      | Right-aligned header action slot.                                           |

## Composition

Use `variant="default"`, `variant="elevated"`, `variant="outline"`, or `variant="ghost"` to match
the surrounding surface. Use `size="xs"`, `size="sm"`, `size="md"`, or `size="lg"` to control density.

`Card` accepts `as="div"`, `as="article"`, `as="section"`, or `as="aside"` for common semantics.
Use `interactive` when the whole card should visually respond to hover, and add the appropriate
keyboard and click behavior in application code. Pass `className` to any slot and customize the
visual contract with `--card-*` variables.

### Variants

Choose the visual weight that matches the surrounding layout.

<Preview>
  <CardVariantsExample />

  <Preview.Code>
    {`
      import { Card, CardDescription, CardHeader, CardTitle } from "moduix";

          <div>
