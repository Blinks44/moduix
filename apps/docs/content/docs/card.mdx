## API Reference

`Card` is a standalone presentational component. It does not wrap a Base UI primitive, so the API stays intentionally small: a styled root, a compact size option, and composable parts.

## Overview

Use `Card` as a thin styled container for grouped content. The default path stays short:

- `Card` for the visible surface
- `CardHeader` for heading content
- `CardTitle` and `CardDescription` for the primary label
- `CardContent` for the body
- `CardFooter` for actions or secondary metadata

`CardAction` is optional and only handles the common header-side layout for a badge or compact action.

## Anatomy

Use the following composition to build a `Card`:

```text
Card
├─ CardHeader
│  ├─ CardTitle
│  ├─ CardDescription
│  └─ CardAction (optional)
├─ CardContent
└─ CardFooter
```

```tsx
<Card>
  <CardHeader>
    <CardTitle>Billing plan</CardTitle>
    <CardDescription>Team workspace, monthly billing.</CardDescription>
  </CardHeader>
  <CardContent>...</CardContent>
  <CardFooter>...</CardFooter>
</Card>
```

| Part              | Role                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| `Card`            | Root container. Supports `className`, native props, and `size="sm"`. |
| `CardHeader`      | Header layout for title, description, and action content.            |
| `CardContent`     | Main content area with card body spacing.                            |
| `CardFooter`      | Footer layout for actions or secondary content.                      |
| `CardTitle`       | Card heading. Defaults to `h3` and supports `as`.                    |
| `CardDescription` | Muted supporting text under the title.                               |
| `CardAction`      | Right-aligned header action slot.                                    |

## Composition

Prefer the default composition when the card is just grouping content. Add `CardAction` only when the header needs a compact status or control aligned to the trailing edge.

If the whole surface should navigate somewhere, keep the root presentational and place a link or button inside the content instead of turning `Card` into an interactive wrapper.

Use `size="sm"` for denser dashboards, menus, and supporting panels. Keep the default size for standard page sections and primary summaries.

Use `className` and `--card-*` CSS variables for visual customization. The layout parts stay the same even when the visuals change.

### Custom Composition
