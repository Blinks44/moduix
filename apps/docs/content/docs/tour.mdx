## Anatomy

```text
Tour / Tour.Root
└─ Overlay subtree (automatically portalled)
   ├─ Tour.Backdrop
   ├─ Tour.Spotlight
   └─ Tour.Positioner
      └─ Tour.Content
         ├─ Tour.Arrow
         │  └─ Tour.ArrowTip
         ├─ Tour.CloseTrigger / Tour.CloseIcon
         ├─ Tour.Title
         ├─ Tour.Description
         ├─ Tour.ProgressText
         └─ Tour.Control
            ├─ Tour.ActionList
            └─ Tour.Actions
               └─ Tour.ActionTrigger[action]
```

| Part                      | Role                                                                 |
| ------------------------- | -------------------------------------------------------------------- |
| `Tour` / `Tour.Root`      | Renders the tour tree from the `UseTourReturn` object.               |
| `Tour.Backdrop`           | Optional overlay layer for dialog and spotlight steps.               |
| `Tour.Spotlight`          | Highlights the current target element.                               |
| `Tour.Positioner`         | Positions dialog, tooltip, or floating content.                      |
| `Tour.Content`            | Alert-dialog surface with Ark title and description wiring.          |
| `Tour.Arrow` / `ArrowTip` | Tooltip arrow using Ark positioning variables.                       |
| `Tour.CloseTrigger`       | Low-level dismiss trigger.                                           |
| `Tour.CloseIcon`          | Shared icon-only close helper built on the Ark close trigger.        |
| `Tour.ProgressText`       | Localized progress text from `translations.progressText`.            |
| `Tour.Title`              | Current step title.                                                  |
| `Tour.Description`        | Current step description.                                            |
| `Tour.Control`            | Action group container.                                              |
| `Tour.ActionList`         | Recommended mapping of the current step's actions to styled buttons. |
| `Tour.Actions`            | Render prop for the current step action objects.                     |
| `Tour.ActionTrigger`      | Button that runs a `StepAction` such as `next`, `prev`, `dismiss`.   |

## Composition

`Tour.Root` portals `Backdrop`, `Spotlight`, and `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; all structural parts remain explicit and styleable.

### Recommended composition

Use `useTour({ steps })` as the state owner and render the structural parts directly under
`Tour.Root`; the root handles their portal boundary.
Moduix adds default styling, `data-slot` hooks, a default `ArrowTip`, and `Tour.ActionList` for
ordinary step actions, but it does not hide the overlay tree behind a content shortcut.

### When to go custom

Use `Tour.Actions` and `Tour.ActionTrigger` when action UI needs custom composition. Import Ark
`useTourContext()` directly when custom child UI needs the current step, progress, or navigation
methods. Use step `effect` callbacks for async work and wait helpers. Use `ids` when separate markup
must share the Ark-generated title, description, content, backdrop, positioner, or arrow IDs.

### Default props

| Prop                     | Default                         | Notes                                                |
| ------------------------ | ------------------------------- | ---------------------------------------------------- |
| `closeOnEscape`          | `true`                          | Escape dismisses the tour.                           |
| `closeOnInteractOutside` | `true`                          | Outside interaction dismisses unless disabled.       |
| `keyboardNavigation`     | `true`                          | Left and Right navigate between available steps.     |
| `preventInteraction`     | `false`                         | Set to `true` to block interaction outside the tour. |
| `spotlightOffset`        | `{ x: 10, y: 10 }`              | Expands the highlighted target region.               |
| `spotlightRadius`        | `4`                             | Corner radius for the spotlight clip path.           |
| `translations`           | Ark defaults                    | Includes progress, next, previous, close, and skip.  |
| `lazyMount`              | `false` on Ark presence wrapper | Supported on `Tour` / `Tour.Root`.                   |
| `unmountOnExit`          | `false` on Ark presence wrapper | Supported on `Tour` / `Tour.Root`.                   |

### Behavioral notes

`Tour` does not expose `RootProvider`; Ark requires a `tour` object from `useTour`. Controlled usage
lives in `useTour` through `steps`, `stepId`, `onStepChange`, `onStepsChange`, and
`onStatusChange`. `Tour.ActionList` passes step actions directly to `Tour.ActionTrigger`; use the
lower-level parts when their rendering needs to vary.
Styled parts forward refs to their Ark DOM elements and support `asChild`; keep a single semantic
child with the required keyboard and focus behavior. `Tour` / `Tour.Root` also preserve Ark
presence props such as `lazyMount`, `unmountOnExit`, `present`, and `onExitComplete`.

### Styling hooks

Target moduix slots with `data-slot="tour-content"` and related hooks. Target Ark state with
`data-scope="tour"`, `data-part`, `data-state`, `data-type`, `data-placement`, `data-side`,
`data-step`, and `data-disabled`. Ark runtime variables include `--tour-layer`, `--available-width`,
`--available-height`, `--reference-width`, `--reference-height`, `--transform-origin`,
`--arrow-size`, and `--arrow-background`. Pass `className` to `Tour.ActionList` to style every
generated `Tour.ActionTrigger`.
