## Anatomy

```text
Carousel / Carousel.Root
├─ Carousel.ItemGroup
│  └─ Carousel.Item[index]
├─ Carousel.Control
│  ├─ Carousel.PrevTrigger
│  └─ Carousel.NextTrigger
├─ Carousel.Indicators
│  └─ Carousel.IndicatorGroup
│     └─ Carousel.Indicator[index]
└─ Carousel.ProgressText
```

| Part                         | Role                                                           |
| ---------------------------- | -------------------------------------------------------------- |
| `Carousel` / `Carousel.Root` | Owns carousel state, navigation behavior, and item collection. |
| `Carousel.Control`           | Layout wrapper for navigation triggers and companion controls. |
| `Carousel.ItemGroup`         | Slide track that owns `Carousel.Item` parts.                   |
| `Carousel.Item`              | Individual slide at a stable `index`.                          |
| `Carousel.PrevTrigger`       | Previous-page trigger.                                         |
| `Carousel.NextTrigger`       | Next-page trigger.                                             |
| `Carousel.Indicators`        | Moduix convenience wrapper for the default page indicators.    |
| `Carousel.IndicatorGroup`    | Group wrapper for page indicators.                             |
| `Carousel.Indicator`         | Page indicator trigger for a specific slide/page index.        |
| `Carousel.ProgressText`      | Current page and total progress text.                          |

Additional advanced surfaces stay available for advanced flows:

- `Carousel.Context`
- `Carousel.AutoplayTrigger`
- `Carousel.AutoplayIndicator`
- `Carousel.RootProvider`

Ark state hooks remain available directly from `@ark-ui/react/carousel`.

## Composition

### Recommended composition

Use `Carousel` as the root, render `Carousel.ItemGroup` as the track, keep `Carousel.Control` for
navigation controls, and reach for `Carousel.Indicators` when the default dot pager is enough.

`Carousel.Control` is usually a toolbar sibling to `Carousel.ItemGroup`.

`Carousel.Root` remains available as an explicit alias of `Carousel` when the namespace itself is
what you want to show.

### When to go custom

Use `Carousel.Context` plus `Carousel.IndicatorGroup` and `Carousel.Indicator` when you need a
custom pager layout, thumbnail indicators, or tighter control over `pageSnapPoints`.

Use moduix `Carousel.RootProvider` with Ark `useCarousel()` when carousel state must be created
outside the rendered subtree.

Use Ark `useCarouselContext()` inside custom child components that need the carousel API without a
render prop.

Use `Carousel.AutoplayTrigger` and `Carousel.AutoplayIndicator` when autoplay controls should be
part of the visible UI instead of only a root prop.

### Default props

| Prop         | Default | Notes                                     |
| ------------ | ------- | ----------------------------------------- |
| `slideCount` | —       | Required on `Carousel` / `Carousel.Root`. |

### Behavioral notes

- `onPageChange(details)` follows Ark shape and exposes the active page as `details.page`.
- `spacing`, `padding`, `slidesPerPage`, `slidesPerMove`, and `autoSize` follow Ark sizing and
  snap-point behavior directly.
- Vertical carousels stay on the same public surface. Default triggers rotate automatically when the
  component uses moduix-provided chevrons.

## Styling

Use `className` on exported parts for structural styling changes. For state styling, prefer Ark
data attributes (`data-part`, `data-orientation`, `data-current`, `data-dragging`) and moduix
`data-slot` hooks.

### CSS Properties

<CssPropertiesSection properties={CarouselDocs.carouselOverrideCssProperties} />

### Styling hooks

- `data-slot="carousel-root"`, `carousel-control`, `carousel-item-group`, `carousel-item`,
  `carousel-prev-trigger`, `carousel-next-trigger`, `carousel-indicator-group`,
  `carousel-indicator`, `carousel-autoplay-trigger`, `carousel-autoplay-indicator`, and
  `carousel-progress-text` are stable selectors on the exported parts.
- Ark also exposes `data-scope="carousel"` and `data-part` names on the rendered nodes.
- `data-orientation="horizontal" | "vertical"` is available on root and structural parts.
- `Carousel.ItemGroup` exposes `data-dragging` while drag interaction is active.
- `Carousel.Indicator` exposes `data-current`, and `Carousel.Item` exposes `data-index` and
  `data-inview`.
- `Carousel.Indicator` exposes `data-readonly`, and `Carousel.AutoplayTrigger` exposes
  `data-pressed`.
