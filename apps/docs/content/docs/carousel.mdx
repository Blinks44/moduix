## Basic

Use the Ark composition directly: `Carousel.Root`, `Carousel.Control`, `Carousel.ItemGroup`,
`Carousel.Item`, `Carousel.PrevTrigger`, `Carousel.NextTrigger`, `Carousel.IndicatorGroup`, and
`Carousel.Indicator`.

<Preview>
  <CarouselExample />

  <Preview.Code>
    {`
      import { Carousel } from "moduix";

      export function TravelCarousel() {
        return (
          <Carousel.Root slideCount={slides.length}>
            <Carousel.Control>
              <Carousel.PrevTrigger />
              <Carousel.ItemGroup aria-label="Travel gallery">
                {slides.map((slide, index) => (
                  <Carousel.Item key={slide.id} index={index}>
                    <img src={slide.src} alt={slide.alt} />
                  </Carousel.Item>
                ))}
              </Carousel.ItemGroup>
              <Carousel.NextTrigger />
            </Carousel.Control>

            <Carousel.IndicatorGroup>
              {slides.map((_, index) => (
                <Carousel.Indicator key={index} index={index} />
              ))}
            </Carousel.IndicatorGroup>

            <Carousel.ProgressText />
          </Carousel.Root>
        );
      }
    `}

  </Preview.Code>

## Anatomy

```text
Carousel.Root
├─ Carousel.Control
│  ├─ Carousel.PrevTrigger
│  ├─ Carousel.ItemGroup
│  │  └─ Carousel.Item[index]
│  └─ Carousel.NextTrigger
├─ Carousel.IndicatorGroup
│  └─ Carousel.Indicator[index]
└─ Carousel.ProgressText
```

Additional Ark parts stay available for advanced flows:

- `Carousel.AutoplayTrigger`
- `Carousel.AutoplayIndicator`
- `Carousel.Context`
- `Carousel.RootProvider`
- `useCarousel()`

## Composition

- `slideCount` is required on `Carousel.Root`.
- `onPageChange(details)` follows Ark shape and exposes the active page as `details.page`.
- `spacing`, `padding`, `slidesPerPage`, `slidesPerMove`, and `autoSize` replace the old
  `SnapCarousel` sizing sugar.
- Use `Carousel.Context` when indicator count depends on runtime `pageSnapPoints`, especially with
  `autoSize`, `slidesPerPage > 1`, and programmatic scrolling.
- Use `Carousel.RootProvider` and `useCarousel()` when carousel state must be created outside the
  rendered subtree.
- Vertical carousels stay on the same public surface. Default triggers rotate automatically when the
  component uses moduix-provided chevrons.

      export function ControlledCarousel() {

### Dynamic Slides

<Preview>
  <DynamicSlidesCarouselExample />

  <Preview.Code>
    {`
      import { Button, Carousel, PlusIcon } from "moduix";
      import { useMemo, useState } from "react";

      export function DynamicSlidesCarousel() {
        const [count, setCount] = useState(4);
        const [page, setPage] = useState(0);
        const visibleSlides = useMemo(() => slides.slice(0, count), [count]);

        return (
          <>
            <Carousel.Root
              slideCount={visibleSlides.length}
              page={page}
              onPageChange={(details) => setPage(details.page)}
            >
              <Carousel.ItemGroup aria-label="Dynamic gallery">
                {visibleSlides.map((slide, index) => (
                  <Carousel.Item key={slide.id} index={index}>
                    <img src={slide.src} alt={slide.alt} />
                  </Carousel.Item>
                ))}
              </Carousel.ItemGroup>
              <Carousel.Control>
                <Carousel.PrevTrigger />
                <Carousel.IndicatorGroup>
                  {visibleSlides.map((_, index) => (
                    <Carousel.Indicator key={index} index={index} />
                  ))}
                </Carousel.IndicatorGroup>
                <Carousel.NextTrigger />
              </Carousel.Control>
            </Carousel.Root>

            <Button
              variant="outline"
              onClick={() => setCount((currentCount) => Math.min(slides.length, currentCount + 1))}
            >
              <PlusIcon />
              Add slide
            </Button>
          </>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Pause On Hover

<Preview>

      import { Carousel } from "moduix";

      export function PauseOnHoverCarousel() {
        return (

            <Carousel.Context>

## Styling

Use `className` on exported parts for structural styling changes. For state styling, prefer Ark
data attributes (`data-part`, `data-orientation`, `data-current`, `data-dragging`) and moduix
`data-slot` hooks.

### Styling hooks

- `data-slot="carousel-root"`, `carousel-control`, `carousel-item-group`, `carousel-item`,
  `carousel-prev-trigger`, `carousel-next-trigger`, `carousel-indicator-group`,
  `carousel-indicator`, `carousel-autoplay-trigger`, `carousel-autoplay-indicator`, and
  `carousel-progress-text` are stable selectors on the exported parts.
- Ark also exposes `data-scope="carousel"` and `data-part` names on the rendered nodes.
- `data-orientation="horizontal" | "vertical"` is available on root and structural parts.
- `Carousel.ItemGroup` exposes `data-dragging` while drag interaction is active.
- `Carousel.Indicator` exposes `data-current`, and `Carousel.Item` exposes `data-index` and
  `data-inview`.
