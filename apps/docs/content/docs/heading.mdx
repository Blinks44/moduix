## Basic

Use `Heading` for semantic page and section titles. Keep `as` tied to the document outline, then
use `size` and `weight` when the visual hierarchy needs to differ from the heading level.

## Anatomy

`Heading` renders a single styled heading element.

```text
Heading
└─ text or inline content
```

| Part      | Role                                                     |
| --------- | -------------------------------------------------------- |
| `Heading` | Semantic heading root with design-system heading styles. |

## Composition

### When to stay high-level

Stay with `Heading` for normal page titles, section titles, and other document-outline headings.
The component already exposes the full public surface on one native `h1`-`h6` element.

Use `as` for semantics, `size` for visual scale, and `weight` for the built-in emphasis presets.

### When to go custom

There is no lower-level composition path here. The escape hatch is direct root styling through
`className` and the documented `--heading-*` CSS variables.

If you need body copy, inline copy, or a more general typography wrapper, use `Text` instead.

### Default props

| Prop     | Default    | Values                                  |
| -------- | ---------- | --------------------------------------- |
| `as`     | `h1`       | `h1`, `h2`, `h3`, `h4`, `h5`, `h6`      |
| `size`   | by `as`    | `xs`, `sm`, `md`, `lg`, `xl`, `2xl`     |
| `weight` | `semibold` | `regular`, `medium`, `semibold`, `bold` |

### Behavioral notes

- Default size mapping follows the semantic level: `h1 -> 2xl`, `h2 -> xl`, `h3 -> lg`, `h4 -> md`,
  `h5 -> sm`, `h6 -> xs`.
- `Heading` keeps native `h1`-`h6` semantics and does not need extra ARIA by default.
- Use `size` for appearance changes instead of changing the semantic level for visual reasons.
- The root uses `text-wrap: balance` and `overflow-wrap: break-word` by default.

### Styling hooks

- `data-slot="heading-root"` is the stable selector on the exported root.
- `data-size` reflects the resolved visual size: `xs`, `sm`, `md`, `lg`, `xl`, or `2xl`.
- `data-weight` reflects the resolved weight preset: `regular`, `medium`, `semibold`, or `bold`.
- Use `className` and `--heading-*` variables for color, typography scale, wrapping, and one-off
  presentation overrides.
