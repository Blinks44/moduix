## Overview

`Heading` is a thin styled wrapper around native `h1`-`h6` elements.

Base UI does not ship a standalone heading primitive, so this component stays close to plain HTML:
`as` controls semantics, `size` controls visual scale, and `weight` provides the built-in emphasis
presets.

Use the default `h1` when the heading is the page or section title. Reach for `size` only when the
visual hierarchy should differ from the semantic outline.

## Anatomy

`Heading` renders a single styled heading element.

```text
Heading
└─ text or inline content
```

| Part      | Role                                                     |
| --------- | -------------------------------------------------------- |
| `Heading` | Semantic heading root with design-system heading styles. |

The root element exposes `data-slot="heading-root"` for targeted styling.

## Composition

Keep semantics and presentation separate:

- use `as` for document outline
- use `size` when visual scale should differ from the heading level
- use `weight` for the built-in typography presets
- use `className` or CSS variables for one-off visual overrides

`Heading` has no internal slots or structural composition. The rendered heading element is already
the full public surface.

### Defaults

| Prop     | Default    | Values                                  |
| -------- | ---------- | --------------------------------------- |
| `as`     | `h1`       | `h1`, `h2`, `h3`, `h4`, `h5`, `h6`      |
| `size`   | by `as`    | `xs`, `sm`, `md`, `lg`, `xl`, `2xl`     |
| `weight` | `semibold` | `regular`, `medium`, `semibold`, `bold` |

Default `size` mapping by `as`: `h1 -> 2xl`, `h2 -> xl`, `h3 -> lg`, `h4 -> md`, `h5 -> sm`,
`h6 -> xs`.

Keep document structure and visual emphasis independent by combining `as` and `size`.

## Custom Composition

`Heading` is already the full structure, so the escape hatch is direct styling on the root element.
Pass `className` when you need one-off presentation changes such as a different color or a scoped
token override.

<Preview>
  <CustomCompositionHeadingExample />

  <Preview.Code>
    {`
      import { Heading } from "moduix";

      export function CustomHeadingDemo() {
        return (
          <Heading as="h2" className={styles.customHeading}>
            Customized heading
          </Heading>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .customHeading {
        --heading-color: var(--color-primary);
        --heading-font-size-xl: var(--text-3xl);
        --heading-line-height-xl: var(--line-height-text-3xl);
        --heading-font-weight-semibold: var(--weight-bold);
      }
    `}

  </Preview.CSS>
</Preview>
