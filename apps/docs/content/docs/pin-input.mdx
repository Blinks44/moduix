## Choosing the right component

Use `PinInput` when a short fixed-length value should be entered across separate one-character
inputs.

- Choose **`PinInput`** for one-time passwords, PINs, backup codes, and invite codes.
- Choose **`Input`** when one plain text input is enough.
- Choose **`PasswordInput`** when the value is one secret string with a visibility toggle.

## Basic

Start with the Ark part tree: root state, label, control, indexed inputs, and `HiddenInput` for
form submission and reset synchronization.

## Anatomy

```text
PinInput / PinInput.Root
├─ PinInput.Label
├─ PinInput.Control
│  ├─ PinInput.Input[index=0]
│  ├─ PinInput.Input[index=1]
│  └─ PinInput.Input[index=2]
└─ PinInput.HiddenInput
```

| Part                           | Role                                                                  |
| ------------------------------ | --------------------------------------------------------------------- |
| `PinInput` / `PinInput.Root`   | Root state machine, count, value array, validation, and form options. |
| `PinInput.RootProvider`        | Renders a pin input from state returned by `usePinInput`.             |
| `PinInput.Label`               | Accessible label for the pin input group.                             |
| `PinInput.Control`             | Wrapper around the visible indexed inputs.                            |
| `PinInput.Input`               | One visible character input. Pass a zero-based `index`.               |
| `PinInput.HiddenInput`         | Hidden native input for forms, autofill, validation, and reset.       |
| `PinInput.Separator`           | Moduix decorative grouping helper. Not an Ark primitive part.         |
| `PinInput.Context`             | Inline access to Ark root state.                                      |
| `usePinInput` / `RootProvider` | External state ownership and imperative methods.                      |
| `usePinInputContext`           | Hook access for reusable custom child parts.                          |

## Composition

### Recommended composition

Use the short `<PinInput>` root form, render a visible `Label`, keep `Control` around the indexed
`Input` parts, and mount `HiddenInput` whenever form submission, reset synchronization, or OTP
autofill should keep working through the native input.

### When to go custom

Reach for `PinInput.RootProvider`, `usePinInput`, `PinInput.Context`, or `usePinInputContext` when
state ownership or custom child composition needs to move outside the default tree. Use
`PinInput.Separator` only as a visual grouping aid and keep the Ark input indexes continuous across
the full code.

### Default props

| Prop             | Default        | Notes                                                               |
| ---------------- | -------------- | ------------------------------------------------------------------- |
| `count`          | -              | Number of inputs. Pass it for SSR and indexed rendering.            |
| `type`           | `numeric`      | `numeric`, `alphabetic`, or `alphanumeric`.                         |
| `placeholder`    | `''` in moduix | Ark defaults to `○`; moduix hides it unless you pass `placeholder`. |
| `otp`            | `false`        | Adds one-time-code autocomplete behavior when true.                 |
| `mask`           | `false`        | Masks visible values like password inputs.                          |
| `autoSubmit`     | `false`        | Submits the owning form when all inputs are filled.                 |
| `blurOnComplete` | `false`        | Blurs the last input after completion.                              |
| `selectOnFocus`  | -              | Selects a focused input's current value when enabled.               |
| `defaultValue`   | `[]`           | Uncontrolled value as a string array.                               |
| `value`          | -              | Controlled value as a string array.                                 |

### Behavioral notes

`value` and `defaultValue` are string arrays. `onValueChange(details)` and
`onValueComplete(details)` expose `details.value` and `details.valueAsString`.
`onValueInvalid(details)` exposes the rejected `details.value` and `details.index`.

Use `type`, `pattern`, and `sanitizeValue` for validation and paste cleanup. `Field.Root` and
`Fieldset.Root` can provide disabled, invalid, required, and read-only state through Ark context.
All exported parts support `asChild`, and `ids` can be used when external labels or descriptions
need stable element IDs.

### Change Events

Control the value array from React state and read the Ark callback detail objects directly.

<Preview>
  <PinInputChangeEventsExample />

  <Preview.Code>
    {`
      import { PinInput, type PinInputValueChangeDetails } from "@moduix/react";
      import { useState } from "react";

      export function ChangeEventsPinInput() {
        const [value, setValue] = useState([] as string[]);
        const [invalidValue, setInvalidValue] = useState("");

        function handleValueChange(details: PinInputValueChangeDetails) {
          setValue(details.value);
        }

        return (
          <PinInput
            count={6}
            type="alphanumeric"
            value={value}
            sanitizeValue={(nextValue) => nextValue.toUpperCase().replace(/[^A-Z0-9]/g, "")}
            onValueChange={handleValueChange}
            onValueInvalid={(details) => {
              setInvalidValue(details.value);
            }}
          >
            <PinInput.Label>Invite code</PinInput.Label>
            <PinInput.Control>
              {Array.from({ length: 6 }, (_, index) => (
                <PinInput.Input key={index} index={index} />
              ))}
            </PinInput.Control>
            <PinInput.HiddenInput />
          </PinInput>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{`.hint { color: var(--color-muted-foreground); }`}</Preview.CSS>
  <Preview.Data>{`const [value, setValue] = useState([] as string[]);`}</Preview.Data>
</Preview>

### Root Provider

Use `usePinInput` with `RootProvider` when state and imperative methods need to live outside the
rendered pin input tree.

        return (

### Auto Submit

Use `autoSubmit` with `name` and `HiddenInput` when a complete code should submit its owning form.

<Preview>
  <PinInputAutoSubmitExample />

  <Preview.Code>
    {`
      import { PinInput } from "@moduix/react";
      import { useState } from "react";

      export function AutoSubmitPinInput() {
        const [submittedValue, setSubmittedValue] = useState("");

        return (
          <form
            onSubmit={(event) => {
              event.preventDefault();
              const formData = new FormData(event.currentTarget);
              setSubmittedValue(String(formData.get("verificationCode") ?? ""));
            }}
          >
            <PinInput count={6} name="verificationCode" autoSubmit>
              <PinInput.Label>Verification code</PinInput.Label>
              <PinInput.Control>
                {Array.from({ length: 6 }, (_, index) => (
                  <PinInput.Input key={index} index={index} />
                ))}
              </PinInput.Control>
              <PinInput.HiddenInput />
            </PinInput>
          </form>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{`.form { display: grid; gap: var(--spacing-3); }`}</Preview.CSS>

### Styling hooks

- Root and root provider expose `data-slot="pin-input-root"` and `data-slot="pin-input-root-provider"`.
- Ark parts keep `data-scope="pin-input"` and `data-part="root" | "label" | "control" | "input"`.
- State styling uses `data-focus`, `data-disabled`, `data-readonly`, `data-invalid`, and
  `data-complete` on relevant parts.
- `PinInput.Separator` exposes `data-slot="pin-input-separator"` and is decorative by default.

## Accessibility

Ark owns focus transfer, paste distribution, keyboard editing, hidden-input form data, and
autocomplete behavior. Keep `HiddenInput` mounted for native forms and reset behavior, and keep
input indexes continuous when inserting visual separators.
