## Choosing the right component

Use `AngleSlider` when users should manipulate a direction or rotation directly on a circular dial.

- Choose **`AngleSlider`** for rotation, heading, spotlight direction, or other angular values.
- Choose **`Slider`** when the value is better understood on a linear range.
- Choose **`NumberInput`** when exact typed degree entry matters more than direct manipulation.

## Anatomy

```text
AngleSlider.Root
├─ AngleSlider.Label
├─ AngleSlider.Control
│  ├─ AngleSlider.MarkerGroup
│  │  └─ AngleSlider.Marker[value]
│  └─ AngleSlider.Thumb
├─ AngleSlider.ValueText
└─ AngleSlider.HiddenInput
```

| Part                               | Role                                                                  |
| ---------------------------------- | --------------------------------------------------------------------- |
| `AngleSlider` / `AngleSlider.Root` | Owns uncontrolled or controlled slider state.                         |
| `AngleSlider.RootProvider`         | Connects externally owned Ark state.                                  |
| `AngleSlider.Label`                | Associates a visible label with the slider thumb.                     |
| `AngleSlider.Control`              | Circular pointer interaction surface.                                 |
| `AngleSlider.MarkerGroup`          | Contains optional reference markers.                                  |
| `AngleSlider.Marker`               | Positions a numeric marker and exposes its relation to current value. |
| `AngleSlider.Thumb`                | Focusable slider handle and rotating angle indicator.                 |
| `AngleSlider.ValueText`            | Renders the current value as degree text.                             |
| `AngleSlider.HiddenInput`          | Participates in native form submission and reset.                     |

## Composition

### Recommended composition

Use the full part tree shown above. `MarkerGroup` is optional, but `Label`, `Thumb`, and
`HiddenInput` should remain when their accessibility and form behavior are needed.

### When to go custom

Use Ark's `useAngleSlider` with `AngleSlider.RootProvider` when state must be controlled outside
the rendered slider subtree. Import context parts and state hooks directly from
`@ark-ui/react/angle-slider`. Do not render `Root` and `RootProvider` for the same state instance.

Every DOM part supports Ark `asChild`. Supply one semantic child that can preserve the part's
keyboard, focus, ARIA, and pointer behavior. Use `ids` on `Root` when composed elements need stable
shared IDs. Refs are forwarded to the DOM element rendered by each wrapped Ark part.

### Default props

| Prop           | Default | Notes                                      |
| -------------- | ------- | ------------------------------------------ |
| `defaultValue` | `0`     | Initial uncontrolled angle                 |
| `step`         | `1`     | Keyboard and pointer value increment       |
| `disabled`     | `false` | Prevents focus and interaction             |
| `readOnly`     | `false` | Keeps the value visible but non-editable   |
| `invalid`      | `false` | Exposes invalid state for ARIA and styling |

### Behavioral notes

- `value` is a single number; `onValueChange(details)` and `onValueChangeEnd(details)` preserve Ark
  callback objects with `details.value` and `details.valueAsDegree`.
- The thumb supports Ark slider keyboard behavior and pointer dragging around the circular control.
- Set `name` and render `HiddenInput` for native form submission and reset synchronization.
- `ValueText` renders Ark's degree string when no custom children are provided.
- `data-disabled`, `data-invalid`, `data-readonly`, marker `data-state`, `--value`, `--angle`,
  `--marker-value`, and `--marker-display-value` remain available for styling.

## Styling

Style the component through `className`, stable `data-slot="angle-slider-*"` hooks, Ark
`data-scope="angle-slider"` / `data-part` attributes, state attributes, and the public CSS
variables below.

### CSS Properties

<CssPropertiesSection properties={AngleSliderDocs.angleSliderCssProperties} />

### Styling hooks

- `data-slot="angle-slider-root"` and `data-slot="angle-slider-root-provider"` own shared layout.
- `data-slot="angle-slider-control"` owns the dial and receives `data-disabled`, `data-invalid`,
  and `data-readonly`.
- `data-slot="angle-slider-thumb"` owns the focusable rotating indicator and can be targeted with
  `:focus-visible`.
- `data-slot="angle-slider-marker"` exposes `data-value` and
  `data-state="under-value | at-value | over-value"`.
- Ark root variables `--value` and `--angle` and marker variables `--marker-value` and
  `--marker-display-value` are preserved.
