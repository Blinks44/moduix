## Choosing the right component

Use `AngleSlider` when users should set a single angular value on a circular dial.

- Choose **`AngleSlider`** for rotation, heading, spotlight direction, or any other 0-360 degree
  input.
- Choose **`Slider`** when the same value should be read as a linear range.
- Choose **`NumberField`** when exact typed degree entry matters more than direct manipulation.

## Basic

Use Ark-style parts: `AngleSlider.Root`, `AngleSlider.Label`, `AngleSlider.Control`,
`AngleSlider.Thumb`, `AngleSlider.ValueText`, and `AngleSlider.HiddenInput`.

## Anatomy

```text
AngleSlider.Root
├─ AngleSlider.Label
├─ AngleSlider.Control
│  └─ AngleSlider.Thumb
├─ AngleSlider.ValueText
└─ AngleSlider.HiddenInput
```

Optional dial markers:

```text
AngleSlider.Control
├─ AngleSlider.MarkerGroup
│  └─ AngleSlider.Marker[value]
└─ AngleSlider.Thumb
```

| Part                      | Role                                                                  |
| ------------------------- | --------------------------------------------------------------------- |
| `AngleSlider.Root`        | Owns the current value, Ark interaction state, and form integration.  |
| `AngleSlider.Label`       | Visible label associated with the slider thumb.                       |
| `AngleSlider.Control`     | Circular interaction surface for pointer and keyboard input.          |
| `AngleSlider.Thumb`       | Rotating active indicator and handle.                                 |
| `AngleSlider.MarkerGroup` | Optional container for static value markers around the dial.          |
| `AngleSlider.Marker`      | Optional marker positioned by its numeric `value`.                    |
| `AngleSlider.ValueText`   | Visible formatted value output. Defaults to Ark `valueAsDegree`.      |
| `AngleSlider.HiddenInput` | Hidden form field for submission. Keep it when the value must submit. |

## Composition

### Recommended defaults

- `defaultValue`: `0`
- `step`: `1`
- `disabled`: `false`
- `readOnly`: `false`
- `invalid`: `false`

`onValueChange` and `onValueChangeEnd` follow Ark shape and receive `details`
(`details.value` and `details.valueAsDegree`).

### Behavioral notes

- `AngleSlider` manages a single numeric value only.
- `AngleSlider.ValueText` renders the current angle text automatically when no children are passed.
- `AngleSlider.HiddenInput` is opt-in, so form participation stays explicit in composition.
- Marker ticks are optional. The default moduix recommendation is a clean dial without them.

## Examples

### Controlled

Use controlled state when the angle must stay in sync with other UI or derived application state.

<Preview>
  <ControlledAngleSliderExample />

  <Preview.Code>
    {`
      import { AngleSlider } from "moduix";
      import { useState } from "react";

      export function ControlledAngleSliderDemo() {
        const [value, setValue] = useState(210);

        return (
          <AngleSlider.Root
            value={value}
            aria-label="Heading"
            onValueChange={(details) => setValue(details.value)}
          >
            <AngleSlider.Label>Heading</AngleSlider.Label>
            <AngleSlider.Control>
              <AngleSlider.Thumb />
            </AngleSlider.Control>
            <AngleSlider.ValueText />
            <AngleSlider.HiddenInput />
          </AngleSlider.Root>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Styling

Style `AngleSlider` through `className`, stable `data-slot="angle-slider-*"` hooks, and the public
`--angle-slider-*` CSS variables shown below.

### Styling hooks

- `data-slot="angle-slider-root"` owns shared dial layout and text color.
- `data-slot="angle-slider-label"` and `data-slot="angle-slider-value-text"` own typography.
- `data-slot="angle-slider-control"` owns the outer ring, inner disc, and center dot.
- `data-slot="angle-slider-thumb"` owns the rotating thumb and active line.
- `data-slot="angle-slider-marker-group"` and `data-slot="angle-slider-marker"` are stable hooks
  for optional dial ticks.
- Ark also exposes `data-scope="angle-slider"` with `data-part` on every rendered part.
- `data-disabled`, `data-invalid`, and `data-readonly` are available on the interactive parts.
- Root CSS variables `--value` and `--angle` remain available for custom visual treatments.
