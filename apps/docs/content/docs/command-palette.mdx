## API Reference

Ark UI does not ship a dedicated `command-palette` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance, and composes Ark [`Dialog`](https://ark-ui.com/docs/components/dialog)
with Ark [`Combobox`](https://ark-ui.com/docs/components/combobox) for behavior.

## Basic

Use `CommandPalette.Panel` for the standard dialog shell and place `CommandPalette.Combobox` inside it. `CommandPalette.Search` renders the input-like search row with a clear button that appears after typing and clears the search query, while the collection stays outside the rendered tree to match Ark Combobox examples.

## Composition

The root portals `Backdrop` and `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Use `useListCollection` with `itemToString`, `itemToValue`, and `groupBy`, then call `filter(details.inputValue)` from `CommandPalette.Combobox` `onInputValueChange`. Use `CommandPalette.Panel` for the repeated overlay chrome and `CommandPalette.Search` for the standard input-like search row, including its clear action.

### When to go custom

Render `Backdrop`, `Positioner`, `Content`, `Header`, `Body`, `Control`, `Input`, and `ClearTrigger` directly when you need custom overlay structure or search controls. Set `closeOnSelect={false}` on `CommandPalette.Combobox` when selecting a command should update palette state instead of closing the dialog. Use `aria-label` on the root for the recommended palette label, or render `CommandPalette.Title` in a custom composition when you need visible dialog chrome. `CommandPalette.ItemIndicator` renders a default check icon and can share the trailing row layout with `ItemMeta`. Keep `CommandPalette.RootProvider` for externally owned dialog state, and import advanced combobox provider/context APIs directly from `@ark-ui/react/combobox` when you need them.

### Default props

| Prop                                        | Default         | Notes                                                                            |
| ------------------------------------------- | --------------- | -------------------------------------------------------------------------------- |
| `CommandPalette.shortcut`                   | `false`         | Pass `alt+k` or another primary modifier plus key to enable the toggle listener. |
| `CommandPalette.shortcutTarget`             | `document`      | Used only when `shortcut` is enabled; `null` skips listener registration.        |
| `CommandPalette.lazyMount`                  | `true`          | Defers dialog DOM until the palette is opened.                                   |
| `CommandPalette.unmountOnExit`              | `true`          | Keeps exit animation mounted, then unmounts after it completes.                  |
| `CommandPalette.Combobox.open`              | `true`          | Keeps the listbox visible while the dialog is mounted.                           |
| `CommandPalette.Combobox.inputBehavior`     | `autohighlight` | Highlights the first matching command as the user types.                         |
| `CommandPalette.Combobox.selectionBehavior` | `preserve`      | Keeps typed search text instead of replacing it with the selected item.          |
| `CommandPalette.Combobox.disableLayer`      | `true`          | Avoids adding an extra dismissable layer inside the dialog.                      |
| `CommandPalette.Combobox.closeOnSelect`     | `true`          | Also closes the dialog after Ark Combobox selection.                             |

### Behavioral notes

The shortcut listener is opt-in because documentation pages and complex applications can mount multiple command palettes at once. It accepts one modifier and one key, for example `mod+k`, `ctrl+1`, `meta+k`, or `alt+k`; supported modifier aliases are `mod`, `ctrl` / `control`, `meta` / `cmd` / `command`, and `alt` / `option`. When enabled, it ignores `input`, `textarea`, `select`, and `contenteditable` targets while the palette is closed, and closes the palette when the same shortcut is pressed while it is open. `alt+k` is the recommended docs shortcut because Fumadocs reserves `Cmd+K`. `mod` accepts either Meta or Control, which keeps the component platform-agnostic. `CommandPalette.ClearTrigger` clears `inputValue`, rather than the Combobox selection value, and stays hidden while the search is empty. `CommandPalette` preserves Ark `onOpenChange(details)`, and `CommandPalette.Combobox` preserves Ark Combobox callback detail objects such as `onInputValueChange(details)`, `onSelect(details)`, and `onValueChange(details)`.

## Styling

Use CSS variables on the trigger, backdrop, content, list, and item parts when the palette needs to match a denser tool UI or a branded overlay style. Use the structural parts when you need to change layout or layering.

### Styling hooks

- Exported visual parts expose stable `data-slot` hooks such as `command-palette-trigger`, `command-palette-backdrop`, `command-palette-positioner`, `command-palette-content`, `command-palette-list`, `command-palette-scroll-area`, `command-palette-scrollbar`, and `command-palette-item`.
- Ark Dialog state hooks include `data-scope="dialog"`, `data-part`, and `data-state="open" | "closed"` on backdrop and content.
- Ark Combobox state hooks include `data-scope="combobox"`, `data-part`, `data-state`, `data-highlighted`, `data-disabled`, `data-invalid`, and `data-empty`.
- Motion is tied to Ark `data-state`; legacy `data-starting-style`, `data-ending-style`, and `data-open` are not part of the command-palette contract.
