## Anatomy

`CommandPaletteContent` is the default composition:

```text
CommandPalette
├─ CommandPaletteTrigger (optional)
└─ CommandPaletteContent
   ├─ CommandPaletteBackdrop
   ├─ CommandPaletteViewport
   │  └─ CommandPalettePopup
   │     └─ autocomplete root
   │        ├─ CommandPaletteInputWrap
   │        │  ├─ CommandPaletteInput
   │        │  └─ CommandPaletteClear (optional)
   │        ├─ CommandPaletteEmpty
   │        ├─ CommandPaletteList
   │        └─ CommandPaletteFooter (optional)
```

| Part                            | Role                                                |
| ------------------------------- | --------------------------------------------------- |
| `CommandPalette`                | Dialog state and optional global shortcut listener. |
| `CommandPaletteTrigger`         | Optional visible trigger.                           |
| `CommandPaletteContent`         | Ready-to-use modal shell with inline autocomplete.  |
| `CommandPalettePortal`          | Portal part for custom composition.                 |
| `CommandPaletteBackdrop`        | Backdrop part for custom composition.               |
| `CommandPaletteViewport`        | Viewport part for custom composition.               |
| `CommandPalettePopup`           | Styled popup surface for custom composition.        |
| `CommandPaletteClose`           | Explicit dismiss control.                           |
| `CommandPaletteCollection`      | Group collection renderer for Base UI grouped data. |
| `CommandPaletteItem`            | Selectable row that closes the palette by default.  |
| `CommandPaletteItemIcon`        | Leading icon container.                             |
| `CommandPaletteItemText`        | Text column wrapper.                                |
| `CommandPaletteItemLabel`       | Primary label.                                      |
| `CommandPaletteItemDescription` | Secondary label.                                    |
| `CommandPaletteItemMeta`        | Trailing metadata, usually a shortcut.              |
| `CommandPaletteFooter`          | Optional footer row.                                |
| `CommandPaletteKbd`             | Compact shortcut badge.                             |

## Composition

`CommandPalette` opens on `mod+k` by default. In these docs demos we use `shortcut="alt+k"` to
avoid common browser conflicts. Set another combo or use `shortcut={false}` / `shortcutTarget={null}`
to disable the listener.

`CommandPaletteContent` keeps the default palette thin and opinionated:

- always rendered through a portal with backdrop and viewport;
- always uses inline open autocomplete;
- keeps `autoHighlight="always"` and `keepHighlight={true}`;
- accepts the data and filtering props most command palettes need: `items`, `itemToStringValue`,
  `value`, `defaultValue`, `onValueChange`, `filter`, `filteredItems`, and `limit`.

When you need a different container or layering, compose the public parts directly instead of
passing slot configuration through `CommandPaletteContent`.

| Prop                                   | Default    | Values                                        |
| -------------------------------------- | ---------- | --------------------------------------------- |
| `shortcut`                             | `mod+k`    | `false` or keyboard combo string like `alt+k` |
| `shortcutTarget`                       | `document` | `document`, `HTMLElement`, or `null`          |
| `autoHighlight`                        | `always`   | Fixed on `CommandPaletteContent`              |
| `keepHighlight`                        | `true`     | Fixed on `CommandPaletteContent`              |
| `closeOnSelect` (`CommandPaletteItem`) | `true`     | `true`, `false`                               |

### Controlled Open State

```tsx
import { Button, CommandPalette, CommandPaletteContent } from 'moduix';
import { useState } from 'react';

export function ControlledCommandPalette() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button type="button" onClick={() => setOpen(true)}>
        Search actions
      </Button>
      <CommandPalette open={open} onOpenChange={setOpen}>
        <CommandPaletteContent aria-label="Command palette" items={commands}>
          {/* input and results */}
        </CommandPaletteContent>
      </CommandPalette>
    </>
  );
}
```

### Custom Shortcut

```tsx
<CommandPalette shortcut="alt+k">
  <CommandPaletteContent aria-label="Command palette" items={commands}>
    {/* input and results */}
  </CommandPaletteContent>
</CommandPalette>
```

### Disable Global Shortcut

```tsx
<CommandPalette shortcutTarget={null}>
  <CommandPaletteContent aria-label="Command palette" items={commands}>
    {/* input and results */}
  </CommandPaletteContent>
</CommandPalette>
```

### Custom Composition

When the default `CommandPaletteContent` is too opinionated, compose the palette manually from
`CommandPalettePortal`, `CommandPaletteBackdrop`, `CommandPaletteViewport`, and
`CommandPalettePopup`. Keep the autocomplete state inside the popup and style the visible parts
directly.
