## Anatomy

`CommandPaletteContent` owns dialog service slots and renders an always-open autocomplete root inside the popup.

```text
CommandPalette
├─ CommandPaletteTrigger (optional)
└─ CommandPaletteContent
   ├─ portal (service slot)
   │  ├─ backdrop (service slot, optional)
   │  └─ viewport (service slot)
   │     └─ popup
   │        └─ autocomplete root
   │           ├─ CommandPaletteInputWrap
   │           │  ├─ CommandPaletteInput
   │           │  └─ CommandPaletteClear (optional)
   │           ├─ CommandPaletteEmpty
   │           ├─ CommandPaletteList
   │           │  └─ CommandPaletteGroup
   │           │     ├─ CommandPaletteGroupLabel
   │           │     └─ CommandPaletteItem
   │           └─ CommandPaletteFooter (optional)
```

| Part                            | Role                                                                     |
| ------------------------------- | ------------------------------------------------------------------------ |
| `CommandPalette`                | Dialog root state, global shortcut listener, and command handle context. |
| `CommandPaletteTrigger`         | Optional visible trigger.                                                |
| `CommandPaletteContent`         | Modal surface that configures dialog and autocomplete behavior.          |
| `CommandPaletteInputWrap`       | Search row layout.                                                       |
| `CommandPaletteInput`           | Autocomplete input that filters commands.                                |
| `CommandPaletteClear`           | Optional clear action for the current search value.                      |
| `CommandPaletteEmpty`           | Empty state shown when no commands match.                                |
| `CommandPaletteList`            | Scrollable result list.                                                  |
| `CommandPaletteGroup`           | Optional grouped command section.                                        |
| `CommandPaletteGroupLabel`      | Section label.                                                           |
| `CommandPaletteCollection`      | Optional collection renderer when using Base UI grouped data.            |
| `CommandPaletteItem`            | Selectable command row; closes the palette by default on click.          |
| `CommandPaletteItemIcon`        | Leading icon container.                                                  |
| `CommandPaletteItemText`        | Text column wrapper.                                                     |
| `CommandPaletteItemLabel`       | Primary command label.                                                   |
| `CommandPaletteItemDescription` | Secondary command description.                                           |
| `CommandPaletteItemMeta`        | Trailing metadata, usually a shortcut.                                   |
| `CommandPaletteFooter`          | Optional keyboard help/status row.                                       |
| `CommandPaletteKbd`             | Compact shortcut key badge.                                              |

## Composition

`CommandPalette` opens on `mod+k` by default (`mod` is Command on macOS and Control on Windows/Linux).
Set `shortcut="shift+mod+p"` for another combo or use `shortcut={false}` / `shortcutTarget={null}` to
disable the global listener. The shortcut listener ignores editable targets such as inputs,
textareas, selects, and contenteditable regions.

`CommandPaletteContent` accepts common autocomplete props (`items`, `itemToStringValue`, `value`,
`onValueChange`, `filter`, `filteredItems`, `limit`, `autoHighlight`, `keepHighlight`,
`highlightItemOnHover`, `loopFocus`, `locale`, `mode`, `onItemHighlighted`, `openOnInputClick`,
`submitOnItemClick`, `virtualized`, `disabled`, `readOnly`, `required`, `inputRef`, `form`, and
`id`. Results are rendered in place with `CommandPaletteList` or `useCommandPaletteFilteredItems`.

Use `className` to style the popup and `classNames` for dialog service slots (`portal`,
`backdrop`, `viewport`). Use `container`, `slotProps`, and `withBackdrop` for the same layering
controls as other dialog-like components.

| Prop                                   | Default         | Values                                                |
| -------------------------------------- | --------------- | ----------------------------------------------------- |
| `shortcut`                             | `mod+k`         | `false` or keyboard combo string like `shift+mod+p`   |
| `shortcutTarget`                       | `document`      | `document`, `HTMLElement`, or `null`                  |
| `withBackdrop`                         | `true`          | `true`, `false`                                       |
| `container`                            | `document.body` | Any valid dialog portal container                     |
| `classNames.portal`                    | —               | `className` for portal slot                           |
| `classNames.backdrop`                  | —               | `className` for backdrop slot                         |
| `classNames.viewport`                  | —               | `className` for viewport slot                         |
| `slotProps.portal`                     | —               | Base UI `Dialog.Portal` props (without `className`)   |
| `slotProps.backdrop`                   | —               | Base UI `Dialog.Backdrop` props (without `className`) |
| `slotProps.viewport`                   | —               | Base UI `Dialog.Viewport` props (without `className`) |
| `autoHighlight`                        | `always`        | Base UI autocomplete `autoHighlight` values           |
| `filteredItems`                        | —               | Externally computed filtered item list                |
| `highlightItemOnHover`                 | `true`          | `true`, `false`                                       |
| `keepHighlight`                        | `true`          | `true`, `false`                                       |
| `loopFocus`                            | `true`          | `true`, `false`                                       |
| `mode`                                 | `list`          | Base UI autocomplete `mode` values                    |
| `virtualized`                          | `false`         | `true`, `false`                                       |
| `disabled`                             | `false`         | `true`, `false`                                       |
| `readOnly`                             | `false`         | `true`, `false`                                       |
| `closeOnSelect` (`CommandPaletteItem`) | `true`          | `true`, `false`                                       |

### Controlled Open State

Use controlled state when another part of the app should open the palette or react to close events.

```tsx
import { Button, CommandPalette, CommandPaletteContent } from 'moduix';
import { useState } from 'react';

export function ControlledCommandPalette() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button type="button" onClick={() => setOpen(true)}>
        Search actions
      </Button>
      <CommandPalette open={open} onOpenChange={setOpen}>
        <CommandPaletteContent aria-label="Command palette" items={commands}>
          {/* input and results */}
        </CommandPaletteContent>
      </CommandPalette>
    </>
  );
}
```

### Custom Shortcut

```tsx
<CommandPalette shortcut="shift+mod+p">
  <CommandPaletteContent aria-label="Command palette" items={commands}>
    {/* input and results */}
```
