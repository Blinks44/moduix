## Basic

Use `CommandPaletteContent` for the default path. It gives you the dialog shell, search input,
filtered result list, grouped rendering, and keyboard-first interaction in one composition.

In these demos we use `shortcut="alt+k"` to avoid common browser conflicts. The default shortcut is
`mod+k`, where `mod` resolves to `Meta` on macOS/iOS and `Control` elsewhere.

## Composition

### When to stay high-level

Stay with `CommandPalette` plus `CommandPaletteContent` for most command surfaces. This is the
intended default path for grouped commands, search-as-you-type filtering, and modal keyboard launch.

`CommandPaletteContent` is intentionally narrow:

- it always renders through `CommandPalettePortal`;
- it keeps the autocomplete root `inline` and `open`;
- it fixes `autoHighlight="always"` and `keepHighlight={true}`;
- it forwards only the common data and filtering props: `items`, `itemToStringValue`, `value`,
  `defaultValue`, `onValueChange`, `filter`, `filteredItems`, and `limit`.

### When to go custom

Switch to `CommandPalettePortal`, `CommandPaletteBackdrop`, `CommandPaletteViewport`, and
`CommandPalettePopup` when the default shell is too opinionated and you need different layering,
layout, or a different autocomplete root setup.

`CommandPaletteItem` adds one local DX helper: `closeOnSelect`, which defaults to `true`. Keep it
enabled for action-style items. Turn it off only when a command updates palette state and should
leave the surface open.

### Default props

| Surface                 | Prop             | Default    | Values / Notes                       |
| ----------------------- | ---------------- | ---------- | ------------------------------------ |
| `CommandPalette`        | `modal`          | `true`     | `true`, `false`                      |
| `CommandPalette`        | `shortcut`       | `mod+k`    | `false` or a combo like `alt+k`      |
| `CommandPalette`        | `shortcutTarget` | `document` | `document`, `HTMLElement`, or `null` |
| `CommandPaletteContent` | `autoHighlight`  | `always`   | fixed by `CommandPaletteContent`     |
| `CommandPaletteContent` | `keepHighlight`  | `true`     | fixed by `CommandPaletteContent`     |
| `CommandPaletteItem`    | `closeOnSelect`  | `true`     | `true`, `false`                      |

### Behavioral notes

- The shortcut listener ignores editable targets such as `input`, `textarea`, `select`, and
  `contenteditable`.
- Set `shortcut={false}` to disable the global listener entirely.
- Set `modal={false}` when the palette should stay interactive without a backdrop or focus trap.
- Use `itemToStringValue` to define the real search surface for each item.

## Examples

### Item Actions

Keep the action on the item itself, then group the data before passing it to
`CommandPaletteContent`. This keeps filtering, rendering, and execution in one predictable data
flow.

  <CommandPaletteActionsExample />

  <Preview.Code>
    {`
      import {
        Button,
        CommandPalette,
        CommandPaletteClear,
        CommandPaletteCollection,
        CommandPaletteContent,
        CommandPaletteEmpty,
        CommandPaletteFooter,
        CommandPaletteGroup,
        CommandPaletteGroupLabel,
        CommandPaletteInput,
        CommandPaletteInputWrap,

        CommandPaletteItemDescription,
        CommandPaletteItemIcon,

        CommandPaletteItemText,
        CommandPaletteKbd,
        CommandPaletteList,
        CommandPaletteTrigger,
      } from "moduix";
      import { useState } from "react";

      export function CommandPaletteActionsDemo() {
        const [lastAction, setLastAction] = useState("No command executed yet.");
        const actionGroups = commandGroups.map((group) => ({
          value: group.value,
          items: group.items.map((item) => ({
            ...item,
            onSelect: () => {
              if (item.id === "release-notes") {
                window.open("https://github.com/Blinks44/moduix/releases", "_blank", "noopener,noreferrer");
                setLastAction("Navigated: Release notes");
                return;
              }

              setLastAction(\`Executed: \${item.label}\`);
            },
          })),
        }));

        return (
          <CommandPalette shortcut="alt+k">
            <CommandPaletteTrigger render={<Button />}>Open actions palette</CommandPaletteTrigger>
            <CommandPaletteContent
              aria-label="Command palette with actions"
              items={actionGroups}
              itemToStringValue={(item) => \`\${item.label} \${item.description} \${item.section}\`}
            >
              <CommandPaletteInputWrap>
                <CommandPaletteInput
                  aria-label="Search commands"
                  placeholder="Search and run commands..."
                />
                <CommandPaletteClear aria-label="Clear search" />
              </CommandPaletteInputWrap>
              <CommandPaletteEmpty>No commands found.</CommandPaletteEmpty>
              <CommandPaletteList>
                {(group) => (
                  <CommandPaletteGroup key={group.value} items={group.items}>
                    <CommandPaletteGroupLabel>{group.value}</CommandPaletteGroupLabel>
                    <CommandPaletteCollection>
                      {(item) => (
                        <CommandPaletteItem key={item.id} value={item} onClick={item.onSelect}>
                          <CommandPaletteItemIcon>{item.icon}</CommandPaletteItemIcon>
                          <CommandPaletteItemText>
                            <CommandPaletteItemLabel>{item.label}</CommandPaletteItemLabel>
                            <CommandPaletteItemDescription>{item.description}</CommandPaletteItemDescription>

                      )}
                    </CommandPaletteCollection>

                )}
              </CommandPaletteList>
              <CommandPaletteFooter>
                <span>

                </span>
                <span>{lastAction}</span>
              </CommandPaletteFooter>
            </CommandPaletteContent>

        );
      }
    `}

### Controlled Open State

Control `open` from React state when the palette must coordinate with surrounding UI.

```tsx
import { Button, CommandPalette, CommandPaletteContent } from 'moduix';
import { useState } from 'react';

export function ControlledCommandPalette() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <Button type="button" onClick={() => setOpen(true)}>
        Search actions
      </Button>
      <CommandPalette open={open} onOpenChange={setOpen}>
        <CommandPaletteContent aria-label="Command palette" items={commands}>
          {/* input and results */}
        </CommandPaletteContent>
      </CommandPalette>
    </>
  );
}
```

### Disable the Global Shortcut

Disable the listener when the palette should open only from explicit UI or external state.

```tsx
<CommandPalette shortcut={false}>
  <CommandPaletteContent aria-label="Command palette" items={commands}>
    {/* input and results */}
  </CommandPaletteContent>
</CommandPalette>
```

## Styling

Use CSS variables on the trigger, backdrop, popup, list, and item parts when the palette needs to
match a denser tool UI or a more branded overlay style. Use the low-level shell parts when you need
to change layout or layering, not just colors and spacing.

### Styling hooks

- Exported visual parts expose stable `data-slot` hooks such as `command-palette-trigger`,
  `command-palette-backdrop`, `command-palette-popup`, `command-palette-list`, and
  `command-palette-item`.
- `CommandPaletteTrigger render={...}` opts out of the default trigger class, so custom trigger
  hosts own their full layout and surface styling.
- Base UI state hooks used by the shipped styles include `data-open`, `data-starting-style`, and
  `data-ending-style` on backdrop and popup, plus `data-highlighted`, `data-selected`,
  `data-disabled`, and `data-empty` on list items and lists.
- `--command-palette-backdrop-starting-*`, `--command-palette-backdrop-ending-*`,
  `--command-palette-popup-starting-*`, and `--command-palette-popup-ending-*` control overlay
  motion without changing the component structure.
