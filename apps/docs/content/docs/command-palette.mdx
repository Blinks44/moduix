## API Reference

Ark UI has no dedicated command-palette primitive. `CommandPalette` is a moduix composite that uses Ark [`Dialog`](https://ark-ui.com/docs/components/dialog) for overlay, focus, and dismissal behavior plus Ark [`Combobox`](https://ark-ui.com/docs/components/combobox) for input, collection, filtering, groups, item state, and keyboard navigation; Chakra UI has no dedicated command-palette component, so Chakra only informs the command metadata visual convention.

## Basic

Use the explicit dialog structure and place `CommandPalette.Combobox` inside `CommandPalette.Content`. The collection stays outside the rendered tree, matching Ark Combobox examples.

## Anatomy

```text
CommandPalette / CommandPalette.Root
├─ CommandPalette.Trigger
└─ CommandPalette.Portal
   ├─ CommandPalette.Backdrop
   └─ CommandPalette.Positioner
      └─ CommandPalette.Content
         └─ CommandPalette.Combobox
            ├─ CommandPalette.Control
            │  ├─ CommandPalette.Input
            │  └─ CommandPalette.ClearTrigger
            ├─ CommandPalette.List
            │  ├─ CommandPalette.Empty
            │  └─ CommandPalette.ItemGroup
            │     ├─ CommandPalette.ItemGroupLabel
            │     └─ CommandPalette.Item
            │        ├─ CommandPalette.ItemIcon
            │        ├─ CommandPalette.ItemText
            │        └─ CommandPalette.ItemMeta
            └─ CommandPalette.Footer
```

| Part                                                                  | Role                                                                 |
| --------------------------------------------------------------------- | -------------------------------------------------------------------- |
| `CommandPalette` / `CommandPalette.Root`                              | Ark Dialog root plus optional global shortcut listener.              |
| `CommandPalette.Trigger`                                              | Ark Dialog trigger; use `asChild` for custom trigger hosts.          |
| `CommandPalette.Portal`                                               | Ark Portal for the overlay subtree.                                  |
| `CommandPalette.Backdrop`                                             | Ark Dialog backdrop with `data-state` animations.                    |
| `CommandPalette.Positioner`                                           | Ark Dialog positioner for viewport alignment and scroll containment. |
| `CommandPalette.Content`                                              | Ark Dialog content and visual palette surface.                       |
| `CommandPalette.Combobox`                                             | Ark Combobox root for command search state and selection.            |
| `CommandPalette.Control`                                              | Ark Combobox control around the search input and clear trigger.      |
| `CommandPalette.Input`                                                | Ark Combobox input.                                                  |
| `CommandPalette.ClearTrigger`                                         | Ark Combobox clear trigger with a default icon.                      |
| `CommandPalette.List`                                                 | Ark Combobox content/listbox with an internal moduix `ScrollArea`.   |
| `CommandPalette.Empty`                                                | Ark Combobox empty state.                                            |
| `CommandPalette.ItemGroup` / `ItemGroupLabel`                         | Ark Combobox grouped result sections.                                |
| `CommandPalette.Item`                                                 | Ark Combobox item with highlighted, checked, and disabled state.     |
| `CommandPalette.ItemText` / `ItemIndicator`                           | Ark Combobox item subparts.                                          |
| `CommandPalette.ItemIcon`, `ItemLabel`, `ItemDescription`, `ItemMeta` | Moduix visual leaf parts for command rows.                           |
| `CommandPalette.Footer` / `Kbd`                                       | Footer/status area and a command-scoped wrapper around Moduix `Kbd`. |

## Composition

### Recommended composition

Use `useListCollection` with `itemToString`, `itemToValue`, and `groupBy`, then call `filter(details.inputValue)` from `CommandPalette.Combobox` `onInputValueChange`. Keep `CommandPalette.Portal`, `Backdrop`, `Positioner`, and `Content` explicit so overlay structure stays overrideable.

### When to go custom

Set `closeOnSelect={false}` on `CommandPalette.Combobox` when selecting a command should update palette state instead of closing the dialog. Use `CommandPalette.Title` or `aria-label` on the root to provide the dialog name, and use `CommandPalette.RootProvider` / `CommandPalette.ComboboxRootProvider` only when external code owns those Ark state machines.

### Default props

| Prop                                        | Default         | Notes                                                                     |
| ------------------------------------------- | --------------- | ------------------------------------------------------------------------- |
| `CommandPalette.shortcut`                   | `false`         | Pass `alt+k` or another primary modifier plus key to enable the listener. |
| `CommandPalette.shortcutTarget`             | `document`      | Used only when `shortcut` is enabled; `null` skips listener registration. |
| `CommandPalette.lazyMount`                  | `true`          | Defers dialog DOM until the palette is opened.                            |
| `CommandPalette.unmountOnExit`              | `true`          | Keeps exit animation mounted, then unmounts after it completes.           |
| `CommandPalette.Combobox.open`              | `true`          | Keeps the listbox visible while the dialog is mounted.                    |
| `CommandPalette.Combobox.inputBehavior`     | `autohighlight` | Highlights the first matching command as the user types.                  |
| `CommandPalette.Combobox.selectionBehavior` | `preserve`      | Keeps typed search text instead of replacing it with the selected item.   |
| `CommandPalette.Combobox.disableLayer`      | `true`          | Avoids adding an extra dismissable layer inside the dialog.               |
| `CommandPalette.Combobox.closeOnSelect`     | `true`          | Also closes the dialog after Ark Combobox selection.                      |

### Behavioral notes

The shortcut listener is opt-in because documentation pages and complex applications can mount multiple command palettes at once. When enabled, it ignores `input`, `textarea`, `select`, and `contenteditable` targets. `alt+k` is the recommended docs shortcut because Fumadocs reserves `Cmd+K`. `mod` accepts either Meta or Control, which keeps the component platform-agnostic. `CommandPalette` preserves Ark `onOpenChange(details)`, and `CommandPalette.Combobox` preserves Ark Combobox callback detail objects such as `onInputValueChange(details)`, `onSelect(details)`, and `onValueChange(details)`.

## Examples

### Item Actions

Attach execution behavior to items and let `CommandPalette.Combobox` close the dialog on selection.

<Preview>
  <CommandPaletteActionsExample />

  <Preview.Code>
    {`
      import {
        Button,
        CommandPalette,
        useFilter,
        useListCollection,
      } from "moduix";
      import { useState } from "react";

      export function CommandPaletteActionsDemo() {
        const [lastAction, setLastAction] = useState("No command executed yet.");
        const actionItems = commandItems.map((item) => ({
          ...item,
          onSelect: () => setLastAction(\`Executed: \${item.label}\`),
        }));
        const { contains } = useFilter({ sensitivity: "base" });
        const { collection, filter } = useListCollection({
          initialItems: actionItems,
          itemToString: (item) => \`\${item.label} \${item.description} \${item.section}\`,
          itemToValue: (item) => item.id,
          filter: contains,
          groupBy: (item) => item.section,
        });

        return (
          <CommandPalette
            aria-label="Command palette with actions"
            onOpenChange={(details) => {
              if (!details.open) {
                filter("");
              }
            }}
          >
            <CommandPalette.Trigger asChild>
              <Button>Open actions palette</Button>
            </CommandPalette.Trigger>
            <CommandPalette.Portal>
              <CommandPalette.Backdrop />
              <CommandPalette.Positioner>
                <CommandPalette.Content>
                  <CommandPalette.Combobox
                    collection={collection}
                    onInputValueChange={(details) => filter(details.inputValue)}
                    onSelect={(details) => {
                      const selectedItem = collection.items.find((item) => item.id === details.value[0]);
                      selectedItem?.onSelect();
                    }}
                  >
                    <CommandPalette.Control>
                      <CommandPalette.Input
                        aria-label="Search commands"
                        placeholder="Search and run commands..."
                      />
                      <CommandPalette.ClearTrigger aria-label="Clear search" />
                    </CommandPalette.Control>
                    <CommandPalette.List>
                      <CommandPalette.Empty>No commands found.</CommandPalette.Empty>
                      {collection.group().map(([section, items]) => (
                        <CommandPalette.ItemGroup key={section}>
                          <CommandPalette.ItemGroupLabel>{section}</CommandPalette.ItemGroupLabel>
                          {items.map((item) => (
                            <CommandPalette.Item key={item.id} item={item}>
                              <CommandPalette.ItemText>
                                <CommandPalette.ItemLabel>{item.label}</CommandPalette.ItemLabel>
                                <CommandPalette.ItemDescription>
                                  {item.description}
                                </CommandPalette.ItemDescription>
                              </CommandPalette.ItemText>
                              {item.shortcut ? (
                                <CommandPalette.ItemMeta>{item.shortcut}</CommandPalette.ItemMeta>

                  </CommandPalette.Combobox>

### Controlled Open State

Control `open` from React state when the palette must coordinate with surrounding UI.

<Preview>

    {`
      import {
        Button,
        CommandPalette,
        useFilter,
        useListCollection,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCommandPalette() {
        const [open, setOpen] = useState(false);
        const { contains } = useFilter({ sensitivity: "base" });
        const { collection, filter } = useListCollection({
          initialItems: commandItems,
          itemToString: (item) => \`\${item.label} \${item.description} \${item.section}\`,
          itemToValue: (item) => item.id,
          filter: contains,
          groupBy: (item) => item.section,
        });

        return (
          <>
            <Button type="button" onClick={() => setOpen(true)}>
              Search actions
            </Button>
            <CommandPalette
              aria-label="Controlled command palette"
              open={open}
              onOpenChange={(details) => {
                setOpen(details.open);

                if (!details.open) {

              <CommandPalette.Portal>
                <CommandPalette.Backdrop />
                <CommandPalette.Positioner>
                  <CommandPalette.Content className="commandPalette">

### Styling hooks

- Exported visual parts expose stable `data-slot` hooks such as `command-palette-trigger`, `command-palette-backdrop`, `command-palette-positioner`, `command-palette-content`, `command-palette-list`, `command-palette-scroll-area`, `command-palette-scrollbar`, and `command-palette-item`.
- Ark Dialog state hooks include `data-scope="dialog"`, `data-part`, and `data-state="open" | "closed"` on backdrop and content.
- Ark Combobox state hooks include `data-scope="combobox"`, `data-part`, `data-state`, `data-highlighted`, `data-disabled`, `data-invalid`, and `data-empty`.
- Motion is tied to Ark `data-state`; Base UI `data-starting-style`, `data-ending-style`, and `data-open` are not part of the command-palette contract.
