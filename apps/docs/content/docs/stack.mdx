## API Reference

`Stack` is a standalone moduix layout primitive and does not wrap a dedicated Ark UI primitive.

## Basic

Start with `Stack` when one flex container is enough and the layout only needs direction, gap, and
common alignment.

## Anatomy

`Stack` renders one root flex wrapper. It owns direction, gap, common alignment, and a small amount
of responsive sugar, while broader layout rules stay in local CSS.

```text
Stack
└─ root[data-slot="stack-root"]
   └─ children
```

| Part    | Role                                                 |
| ------- | ---------------------------------------------------- |
| `Stack` | Root flex wrapper for direction, gap, and alignment. |

## Composition

### When to stay high-level

Stay with `Stack` directly for most rows, columns, media-object layouts, skeleton groups, and
simple settings panels.

Use `direction`, `gap`, `align`, `justify`, and `wrap` for common flex layout. Use `fill` when a
nested stack should grow as a flex item. Use `as` when the wrapper should also carry document
semantics.

### When to go custom

Switch to local CSS when the layout needs more than one flex container, more than the mobile/desktop
direction switch, or styling that reads clearer in a stylesheet than in component props.

Use `style` when you intentionally need to override the inline `--stack-*` instance variables or
the computed flex properties on one specific stack.

### Default props

| Prop        | Default         | Values                                                |
| ----------- | --------------- | ----------------------------------------------------- |
| `direction` | `column`        | `column`, `row`, or `{ mobile?: ..., desktop?: ... }` |
| `gap`       | -               | CSS length (`number` = `px`)                          |
| `align`     | browser default | Any valid `align-items` value                         |
| `justify`   | browser default | Any valid `justify-content` value                     |
| `wrap`      | browser default | Any valid `flex-wrap` value                           |
| `fill`      | `false`         | `true`, `false` (`true` = `flex: 1 1 0`)              |
| `as`        | `div`           | Any React element type                                |

### Behavioral notes

- `Stack` renders exactly one root and forwards regular DOM props to that root.
- `direction={{ mobile, desktop }}` switches at `640px` and cross-falls back when only one side is
  provided.
- `fill` affects the stack as a flex item, not its children.
- `gap`, `align`, `justify`, and `wrap` are written inline only when you provide them.
- `Stack` does not add wrappers, slot maps, keyboard behavior, or ARIA semantics on its own.

## Styling

### CSS Properties

`Stack` exposes a small CSS variable contract in `theme.css`, but these variables are still written
inline by the component at render time. Use the `style` prop for per-instance overrides when the
computed `direction` or `fill` value needs to change.

<div className="not-prose">
  <Tabs items={['CSS Variables']} className="mb-0 mt-3">
    <Tab className="max-h-150 overflow-auto">
      <StackCssPropertiesPanel />
    </Tab>
  </Tabs>
</div>

### Styling hooks

- `className` styles the single root for width, padding, borders, backgrounds, and other surface
  concerns.
- `data-slot="stack-root"` is the stable selector for targeting the root in local CSS.
- `style` is applied last, so it can override computed `gap`, `alignItems`, `justifyContent`,
  `flexWrap`, and the inline `--stack-*` variables.
- There are no additional slots, state attributes, or `slotProps` maps in the public contract.
