## Overview

`Stack` is a small flex layout primitive for vertical and horizontal composition. It does not wrap
a Base UI primitive.

Use it when the layout intent is just direction, spacing, and common flex alignment. Use `fill`
when a stack should take the remaining flex space. Use responsive `direction` when the layout only
needs a mobile/desktop switch without a dedicated CSS file.

## Anatomy

`Stack` is a single root flex wrapper. It owns direction, gap, common alignment, and a small amount
of opt-in layout sugar, while leaving broader layout rules to local CSS.

| Part    | Role                                                 |
| ------- | ---------------------------------------------------- |
| `Stack` | Root flex wrapper for direction, gap, and alignment. |

## Composition

Use `direction`, `gap`, `align`, `justify`, and `wrap` for common flex layout.
Use `fill` when the stack should grow as a flex item. Use `as` when the wrapper should render a
different element.

### Defaults

| Prop        | Default      | Values                                                |
| ----------- | ------------ | ----------------------------------------------------- |
| `direction` | `column`     | `column`, `row`, or `{ mobile?: ..., desktop?: ... }` |
| `gap`       | -            | CSS length (`number` = `px`)                          |
| `align`     | `stretch`    | Any valid `align-items` value                         |
| `justify`   | `flex-start` | Any valid `justify-content` value                     |
| `wrap`      | `nowrap`     | Any valid `flex-wrap` value                           |
| `fill`      | `false`      | `true`, `false` (`true` = `flex: 1 1 0`)              |
| `as`        | `div`        | Any React element type                                |

Responsive `direction` is intentionally narrow: it only switches between mobile and desktop
directions at `640px`. Use `className` or `style` when you need broader responsive layout rules.

### Responsive Direction

Use responsive `direction` when the layout only needs a column-on-mobile and row-on-desktop switch.
