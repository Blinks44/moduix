## Anatomy

`Button` is a single styled Base UI primitive root.

```text
Button
└─ root[data-slot="button-root"]
   ├─ icon, spinner, or other visual child (optional)
   └─ text label
```

| Part     | Role                                                       |
| -------- | ---------------------------------------------------------- |
| `Button` | Interactive root with moduix styles plus Base UI behavior. |

The root exposes `data-slot="button-root"`, `data-variant`, `data-size`, and Base UI
`data-disabled`. Icons, spinners, and labels are plain children, not separate Button slots.

## Composition

### When to stay high-level

Stay with `Button` when Base UI button semantics are correct and the control only needs moduix
variants, sizes, and child composition.

moduix adds two visual props on top of the Base UI primitive:

| Prop      | Default   | Values                                                                                   |
| --------- | --------- | ---------------------------------------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `secondary`, `destructive`, `destructive-outline`, `ghost`, `link` |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `xl`, `icon-sm`, `icon-md`, `icon-lg`                            |

### When to go custom

Use Base UI passthrough props such as `render`, `nativeButton`, `disabled`,
`focusableWhenDisabled`, and event handlers when the styled control needs different rendered
elements or custom interaction logic.

### Behavioral notes

Use `variant` and `size` for the common visual path. Keep everything else in composition:

- pass `type="submit"` for form actions
- pass `disabled` and `focusableWhenDisabled` for disabled states
- compose icons and spinners through `children`
- use `render` when the styled control should render another element

For navigation-first actions, render an anchor or router link instead of keeping button semantics.

- `Button` forwards Base UI button props and keeps Base UI focus/press semantics.
- The root always writes `data-slot="button-root"`, `data-variant`, and `data-size`.
- Direct child `svg` elements inherit `currentColor` and are sized with `--button-icon-size`.
- Disabled state styling targets both Base UI `data-disabled` and the native `[disabled]`
  attribute.

### Pending State

Compose pending UI with `disabled`, `focusableWhenDisabled`, `aria-busy`, and a child spinner.

<Preview>
  <ButtonLoadingExample />

  <Preview.Code>
    {`
      import { Button, Spinner } from "moduix";
      import { useState } from "react";

      export function PendingButton() {
        const [pending, setPending] = useState(false);

        return (
          <Button
            disabled={pending}
            focusableWhenDisabled
            aria-busy={pending || undefined}
            onClick={() => {
              setPending(true);
              setTimeout(() => setPending(false), 1800);
            }}
          >
            {pending ? (
              <>
                <Spinner decorative size="sm" />
                Saving
              </>
            ) : (
              "Save Changes"
            )}
          </Button>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Styling

`Button` exposes one stable root plus the full `--button-*` token surface used for variants, sizes,
focus ring, spacing, and icon sizing.

### Styling hooks

- `data-slot="button-root"` is the stable selector on the exported root.
- `data-variant` reflects the current visual variant.
- `data-size` reflects the current size, including `icon-sm`, `icon-md`, and `icon-lg`.
- Base UI `data-disabled` and the native `[disabled]` attribute both drive the disabled styles.
