## API Reference

`Button` is an Ark-aligned standalone wrapper built with `@ark-ui/react/factory`. Ark UI does not
currently ship a dedicated button primitive, so moduix exposes `Button.Root` and keeps the wrapper
limited to styling, `data-slot`, and `asChild` support.

## Basic

Start with `Button.Root` for primary actions, form submits, and action-like controls that only need
the local `variant`, `size`, and child composition path.

## Anatomy

`Button` has one Ark-aligned root element.

```text
Button.Root
└─ root[data-scope="button"][data-part="root"][data-slot="button-root"]
   └─ children
```

| Part          | Role                                          |
| ------------- | --------------------------------------------- |
| `Button.Root` | Root interactive surface with moduix styling. |
| `Button`      | Callable alias of `Button.Root`.              |

## Composition

### Recommended composition

Stay with `Button.Root` when the control should keep button semantics and only needs visual intent,
density, and child composition.

moduix adds two visual props on top of Ark factory root props:

| Prop      | Default   | Values                                                                                   |
| --------- | --------- | ---------------------------------------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `secondary`, `destructive`, `destructive-outline`, `ghost`, `link` |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `xl`, `icon-sm`, `icon-md`, `icon-lg`                            |

### When to go custom

Use `asChild` when another element should own the DOM node while keeping the button visual
contract.

For example, render an anchor when the action is navigation-first:

```tsx
<Button.Root asChild variant="outline">
  <a href="/docs/button">Open Button Docs</a>
</Button.Root>
```

### Default props

| Entry       | Default   | Values / Notes          |
| ----------- | --------- | ----------------------- |
| `variant`   | `default` | Visual intent           |
| `size`      | `md`      | Control density         |
| `asChild`   | `false`   | Ark factory composition |
| `className` | -         | Applied to the root     |

### Behavioral notes

- `Button.Root` forwards Ark factory button props such as `className`, `style`, event handlers,
  and native button attributes.
- The old Base UI `render`, `nativeButton`, and `focusableWhenDisabled` surface is gone.
- Use native `disabled` for real buttons. When `asChild` renders a non-button element, use
  `aria-disabled="true"` and prevent navigation or activation in app code.
- Use `type="submit"` explicitly for form submit buttons.
- Direct child `svg` elements inherit `currentColor` and are sized with `--button-icon-size`.
- Icon-only buttons still need an accessible name through `aria-label` or equivalent labeling.

### Pending State

Compose pending UI with `disabled`, `aria-busy`, and a child spinner.

<Preview>
  <ButtonLoadingExample />

  <Preview.Code>
    {`
      import { Button, Spinner } from "moduix";
      import { useState } from "react";

      export function PendingButton() {
        const [pending, setPending] = useState(false);

        return (
          <Button.Root
            disabled={pending}
            aria-busy={pending || undefined}
            onClick={() => {
              setPending(true);
              setTimeout(() => setPending(false), 1800);
            }}
          >
            {pending ? (
              <>
                <Spinner decorative size="sm" />
                Saving
              </>
            ) : (
              "Save Changes"
            )}
          </Button.Root>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Styling

`Button` exposes one stable root plus the full `--button-*` token surface used for variants, sizes,
focus ring, spacing, and icon sizing.

### Styling hooks

- `data-slot="button-root"` is the stable selector on the exported root.
- `data-scope="button"` and `data-part="root"` stay available for Ark-style targeting.
- `data-variant` reflects the current visual variant.
- `data-size` reflects the current size, including `icon-sm`, `icon-md`, and `icon-lg`.
- Native `[disabled]` and `[aria-disabled='true']` drive the disabled visuals.
