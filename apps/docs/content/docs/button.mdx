## Overview

`Button` is a thin styled wrapper over Base UI `Button`. The library keeps the default path short:
pick a `variant`, pick a `size` when needed, and compose content through `children`.

Base UI behavior stays available on the same component, including `render`, `nativeButton`,
`disabled`, and `focusableWhenDisabled`.

## Anatomy

`Button` is a single styled Base UI primitive root.

```text
Button
├─ icon, spinner, or other visual child (optional)
└─ text label
```

| Part     | Role                                                       |
| -------- | ---------------------------------------------------------- |
| `Button` | Interactive root with moduix styles plus Base UI behavior. |

The root exposes `data-slot="button-root"`, `data-variant`, `data-size`, and Base UI
`data-disabled`. Icons, spinners, and labels are plain children, not separate Button slots.

## Composition

moduix only adds two visual props:

| Prop      | Default   | Values                                                                                   |
| --------- | --------- | ---------------------------------------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `secondary`, `destructive`, `destructive-outline`, `ghost`, `link` |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `xl`, `icon-sm`, `icon-md`, `icon-lg`                            |

Use `variant` and `size` for the common visual path. Keep everything else in composition:

- pass `type="submit"` for form actions
- pass `disabled` and `focusableWhenDisabled` for disabled states
- compose icons and spinners through `children`
- use `render` when the styled control should render another element

For navigation-first actions, render an anchor or router link instead of keeping button semantics.

### Pending State

Compose pending UI with `disabled`, `focusableWhenDisabled`, `aria-busy`, and a child spinner.

<Preview>
  <ButtonLoadingExample />

  <Preview.Code>
    {`
      import { Button, Spinner } from "moduix";
      import { useState } from "react";

      export function PendingButton() {
        const [pending, setPending] = useState(false);

        return (
          <Button
            disabled={pending}
            focusableWhenDisabled
            aria-busy={pending || undefined}
            onClick={() => {
              setPending(true);
              setTimeout(() => setPending(false), 1800);
            }}
          >
            {pending ? (
              <>
                <Spinner decorative size="sm" />
                Saving
              </>
            ) : (
              "Save Changes"
            )}
          </Button>
        );
      }
    `}

  </Preview.Code>
</Preview>
