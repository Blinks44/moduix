## Choosing the right component

Use `Select` when users choose from a fixed set of options and do not need to type into the
trigger to search or enter a value.

- Choose **`Autocomplete`** when users can enter free-form text and suggestions only help complete the value.
- Choose **`Combobox`** when users must choose predefined options, but typing helps search or filter the list.
- Choose **`Select`** when users must choose predefined options and typing in the trigger is not needed.

## Default Path

Start with `Select`, `SelectField`, `SelectTrigger`, and `SelectContent`. This is the recommended
path for standard forms and settings panels.

`SelectContent` internally handles `SelectPortal`, `SelectPositioner`, and `SelectPopup`. Those
low-level parts stay exported for custom composition when popup structure or layering needs to
change.

| Part            | Role                                                               |
| --------------- | ------------------------------------------------------------------ |
| `Select`        | Root state, selection, and form behavior.                          |
| `SelectField`   | Groups the visible field parts.                                    |
| `SelectTrigger` | Button that opens and closes the popup.                            |
| `SelectValue`   | Renders placeholder text or the selected value.                    |
| `SelectContent` | Default popup wrapper with built-in portal, positioner, and popup. |
| `SelectItem`    | Selectable option row.                                             |

## Composition

`Select` keeps Base UI root behavior intact. Use it for `value`, `defaultValue`, `multiple`,
`onValueChange`, `items`, `itemToStringLabel`, and `itemToStringValue`.

Use `onValueChange` when the selected value should immediately update React state. When the value
only needs to be read during form submission, pass `name` on `Select` and read it from
`new FormData(event.currentTarget)`.

`SelectContent` is the high-level popup wrapper. It accepts popup `className`, supports the shared
positioning props (`side`, `sideOffset`, `align`, `alignOffset`, `arrowPadding`,
`collisionAvoidance`, `collisionBoundary`, `collisionPadding`), and keeps two select-specific
conveniences:

- `showArrow` for the built-in popup arrow
- `alignItemWithTrigger` for the overlapping select layout behavior from Base UI

When you need backdrop control, sticky positioning, or custom popup structure, switch to
`SelectPortal`, `SelectBackdrop`, `SelectPositioner`, `SelectPopup`, and `SelectArrow` directly.

### Defaults

| Prop               | Default | Values / Notes                            |
| ------------------ | ------- | ----------------------------------------- |
| `showArrow`        | `false` | Enables the built-in arrow                |
| `sideOffset`       | `8`     | Any number accepted by Base UI positioner |
| `indicator` (item) | `start` | `start`, `end`                            |

### Multiple Values

Use `multiple` when the field can contain several selected values. Render `SelectValue` with a
function to format the array.

<Preview>
  <MultipleSelectExample />

  <Preview.Code>
    {`
      import {
        Select,
        SelectContent,
        SelectField,
        SelectIcon,
        SelectItem,
        SelectItemIndicator,
        SelectItemText,
        SelectLabel,
        SelectList,
        SelectTrigger,
        SelectValue,
      } from "moduix";

      function renderMultipleValue(value) {
        if (value.length === 0) {
          return "Select languages";
        }

        const first = languages[value[0]];
        const suffix = value.length > 1 ? \` (+\${value.length - 1})\` : "";

        return \`\${first}\${suffix}\`;
      }

      export function MultipleSelectDemo() {
        return (
          <Select multiple defaultValue={["javascript", "typescript"]}>
            <SelectField>
              <SelectLabel>Languages</SelectLabel>
              <SelectTrigger>
                <SelectValue>{renderMultipleValue}</SelectValue>
                <SelectIcon />
              </SelectTrigger>
            </SelectField>

            <SelectContent alignItemWithTrigger={false}>
              <SelectList>
                {languageValues.map((value) => (
                  <SelectItem key={value} value={value}>
                    <SelectItemIndicator />
                    <SelectItemText>{languages[value]}</SelectItemText>
                  </SelectItem>
                ))}
              </SelectList>
            </SelectContent>
          </Select>
        );
      }
    `}

  </Preview.Code>
  <Preview.Data>
    {`
      const languages = {
        csharp: "C#",
        go: "Go",
        javascript: "JavaScript",
        python: "Python",
        rust: "Rust",
        typescript: "TypeScript",
      } as const;

      type Language = keyof typeof languages;

      const languageValues = Object.keys(languages) as Language[];
    `}

  </Preview.Data>
</Preview>

### Controlled Value

Control `value` from React state when the selection coordinates with other application state. This
is the direct path when you need to read the current value outside native form submission.

<Preview>
  <ControlledSelectExample />

  <Preview.Code>
    {`
      import { useState } from "react";

      export function ControlledSelectDemo() {
        const [value, setValue] = useState("light");

        return (
          <div>
            <Select value={value} onValueChange={setValue} items={themeOptions}>
              <SelectField>
                <SelectLabel>Theme</SelectLabel>
                <SelectTrigger>
                  <SelectValue placeholder="Select theme" />
                  <SelectIcon />
                </SelectTrigger>
              </SelectField>

              <SelectContent>
                <SelectList>
                  {themeOptions.map((item) => (
                    <SelectItem key={item.value} value={item.value}>
                      <SelectItemIndicator />
                      <SelectItemText>{item.label}</SelectItemText>
                    </SelectItem>
                  ))}
                </SelectList>
              </SelectContent>
            </Select>

            <p>Current value: {value ?? "none"}</p>
          </div>
        );
      }
    `}

  </Preview.Code>

## Custom Composition

When the default wrapper is too opinionated, compose the popup parts directly. This is the escape
hatch for custom layering, sticky positioning, backdrops, or popup structure.

<Preview>
  <CustomCompositionSelectExample />

  <Preview.Code>
    {`
      import {
        Select,
        SelectArrow,
        SelectBackdrop,
        SelectField,
        SelectIcon,
        SelectItem,
        SelectItemIndicator,
        SelectItemText,
        SelectLabel,
        SelectList,
        SelectPopup,
        SelectPortal,
        SelectPositioner,
        SelectTrigger,
        SelectValue,
      } from "moduix";

      export function CustomCompositionSelectDemo() {
        return (
          <Select items={fruits}>
            <SelectField>
              <SelectLabel>Choose fruit</SelectLabel>
              <SelectTrigger className={styles.customTrigger}>
                <SelectValue placeholder="Select an option" />
                <SelectIcon />
              </SelectTrigger>
            </SelectField>

            <SelectPortal>
              <SelectBackdrop className={styles.customBackdrop} />
              <SelectPositioner
                alignItemWithTrigger={false}
                sideOffset={8}
                sticky
                className={styles.customPositioner}
              >
                <SelectPopup className={styles.customPopup}>
                  <SelectArrow className={styles.customArrow} />
                  <SelectList>
                    {fruits.map((item) => (
                      <SelectItem key={item.value} value={item.value}>
                        <SelectItemIndicator />
                        <SelectItemText>{item.label}</SelectItemText>
                      </SelectItem>
                    ))}
                  </SelectList>
                </SelectPopup>
              </SelectPositioner>
            </SelectPortal>
          </Select>
        );
      }
    `}

  </Preview.Code>
</Preview>
