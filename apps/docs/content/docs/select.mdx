## Choosing the right component

Use `Select` for strict, predefined choices where users pick from a popup list and do not type into
the field.

- Choose **`Select`** for canonical form pickers with fixed options.
- Choose **`Combobox`** when users should search/filter predefined options via text input.
- Choose **`Autocomplete`** when users may enter arbitrary values beyond suggested options.

## Anatomy

Keep field parts inside `SelectField` and list parts inside `SelectContent`. That preserves labeling,
selection state, and keyboard behavior.

```text
Select
├─ SelectField
│  ├─ SelectLabel
│  └─ SelectTrigger
│     ├─ SelectValue
│     └─ SelectIcon
└─ SelectContent
   └─ SelectPortal
      └─ SelectPositioner
         └─ SelectPopup
            ├─ SelectArrow (optional)
            ├─ SelectScrollUpArrow / SelectScrollDownArrow (optional)
            └─ SelectList
               └─ SelectItem[value]
                  ├─ SelectItemIndicator
                  └─ SelectItemText
```

```tsx
<Select items={fruits}>
  <SelectField>
    <SelectLabel>Choose fruit</SelectLabel>
    <SelectTrigger>
      <SelectValue placeholder="Select an option" />
      <SelectIcon />
    </SelectTrigger>
  </SelectField>

  <SelectContent>
    <SelectScrollUpArrow />
    <SelectList>
      {fruits.map((item) => (
        <SelectItem key={item.value} value={item.value}>
          <SelectItemIndicator />
          <SelectItemText>{item.label}</SelectItemText>
        </SelectItem>
      ))}
    </SelectList>
    <SelectScrollDownArrow />
  </SelectContent>
</Select>
```

| Part            | Role                                                                               |
| --------------- | ---------------------------------------------------------------------------------- |
| `Select`        | Root state and selection logic.                                                    |
| `SelectField`   | Groups the visible field parts.                                                    |
| `SelectTrigger` | Button that opens and closes the popup.                                            |
| `SelectValue`   | Renders placeholder or selected value.                                             |
| `SelectContent` | Default popup composition: portal, positioner, popup, and optional built-in arrow. |
| `SelectItem`    | Selectable option row.                                                             |

Start with `SelectContent` for the normal path. When popup structure, layering, or styling needs to
change more deeply, compose `SelectPortal`, `SelectBackdrop`, `SelectPositioner`, `SelectPopup`,
and `SelectArrow` directly.

## Composition

Use `Select` for root state and behavior props such as `value`, `defaultValue`, `multiple`,
`onValueChange`, `items`, `itemToStringLabel`, and `itemToStringValue`. Base UI root props for
form integration, open state, and item comparison also pass through.

`SelectContent` is the simple path. It wraps `SelectPortal`, `SelectPositioner`, and `SelectPopup`
for you, accepts popup `className`, forwards Base UI popup props, and keeps a small set of
positioning props on the surface:

```tsx
<SelectContent className="popup" showArrow sideOffset={8} alignItemWithTrigger={false} />
```

### Defaults

| Prop               | Default | Values / Notes                            |
| ------------------ | ------- | ----------------------------------------- |
| `showArrow`        | `false` | Enables the built-in arrow                |
| `sideOffset`       | `8`     | Any number accepted by Base UI positioner |
| `animation`        | `—`     | `scale`                                   |
| `indicator` (item) | `start` | `start`, `end`                            |

### Grouped

Use `SelectGroup` and `SelectGroupLabel` to organize related items inside the popup.

<Preview>
  <GroupedSelectExample />

  <Preview.Code>
    {`
      import {
        Select,
        SelectField,
        SelectLabel,
        SelectTrigger,
        SelectValue,
        SelectIcon,
        SelectContent,
        SelectList,
        SelectGroup,
        SelectSeparator,
        SelectGroupLabel,
        SelectItem,
        SelectItemIndicator,
        SelectItemText,
      } from "moduix";

      export function GroupedSelectDemo() {
        return (
          <Select>
            <SelectField>
              <SelectLabel>Choose produce</SelectLabel>
              <SelectTrigger>
                <SelectValue placeholder="Select item">
                  {(value) =>
                    typeof value === "string" ? groupedLabelByValue[value] ?? value : "Select item"
                  }
                </SelectValue>
                <SelectIcon />
              </SelectTrigger>
            </SelectField>

            <SelectContent>
              <SelectList>
                {groupedOptions.map((group, index) => (
                  <SelectGroup key={group.label}>
                    {index > 0 ? <SelectSeparator /> : null}
                    <SelectGroupLabel>{group.label}</SelectGroupLabel>
                    {group.items.map((item) => (
                      <SelectItem key={item.value} value={item.value}>
                        <SelectItemIndicator />
                        <SelectItemText>{item.label}</SelectItemText>
                      </SelectItem>
                    ))}
                  </SelectGroup>
                ))}
              </SelectList>
            </SelectContent>
          </Select>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const groupedOptions = [
        {
          label: "Fruits",
          items: [
            { label: "Apple", value: "apple" },
            { label: "Mango", value: "mango" },
          ],
        },
        {
          label: "Vegetables",
          items: [
            { label: "Carrot", value: "carrot" },
            { label: "Spinach", value: "spinach" },
          ],
        },
      ];

      const groupedLabelByValue = Object.fromEntries(
        groupedOptions.flatMap((group) => group.items.map((item) => [item.value, item.label])),
      );
    `}

  </Preview.Data>
</Preview>

### Multiple

Use `multiple` when the field can contain several selected values. Render `SelectValue` with a
function to format the array.

<Preview>
  <MultipleSelectExample />

  <Preview.Code>
    {`
      import {
        Select,
        SelectField,
        SelectLabel,
        SelectTrigger,
        SelectValue,
        SelectIcon,
        SelectContent,
        SelectList,
        SelectItem,
        SelectItemIndicator,
        SelectItemText,
      } from "moduix";

      function renderMultipleValue(value) {
        if (value.length === 0) {
          return "Select languages";
        }

        const first = languages[value[0]];
        const suffix = value.length > 1 ? \` (+\${value.length - 1})\` : "";

        return \`\${first}\${suffix}\`;
      }

      export function MultipleSelectDemo() {
        return (
          <Select multiple defaultValue={["javascript", "typescript"]}>
            <SelectField>
              <SelectLabel>Languages</SelectLabel>
              <SelectTrigger className={styles.customTrigger}>
                <SelectValue>{renderMultipleValue}</SelectValue>
                <SelectIcon />
              </SelectTrigger>
            </SelectField>

            <SelectContent alignItemWithTrigger={false}>
              <SelectList>
                {languageValues.map((value) => (
                  <SelectItem key={value} value={value}>
                    <SelectItemIndicator />
                    <SelectItemText>{languages[value]}</SelectItemText>
                  </SelectItem>
                ))}
              </SelectList>
            </SelectContent>
          </Select>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const languages = {
        javascript: "JavaScript",
        python: "Python",
        rust: "Rust",
        typescript: "TypeScript",
      };

      const languageValues = Object.keys(languages);
    `}

  </Preview.Data>
</Preview>

### Custom Composition

When the default wrapper is too opinionated, compose the popup parts directly. This is the escape
hatch for custom layering, structure, arrow placement, or backdrop styling.

<Preview>
  <CustomCompositionSelectExample />

  <Preview.Code>
    {`
      import {
        Select,
        SelectArrow,
        SelectBackdrop,
        SelectField,
        SelectLabel,
        SelectTrigger,
        SelectValue,
        SelectIcon,
        SelectList,
        SelectItem,
        SelectItemIndicator,
        SelectItemText,
        SelectPopup,
        SelectPortal,
        SelectPositioner,
      } from "moduix";

      export function CustomCompositionSelectDemo() {
        return (
          <Select items={fruits}>
            <SelectField>
              <SelectLabel>Choose fruit</SelectLabel>
              <SelectTrigger>
                <SelectValue placeholder="Select an option" />
                <SelectIcon />
              </SelectTrigger>
            </SelectField>

            <SelectPortal>
              <SelectBackdrop className={styles.customBackdrop} />
              <SelectPositioner
                alignItemWithTrigger={false}
                sideOffset={8}
                sticky
                className={styles.customPositioner}
              >
                <SelectPopup className={styles.customPopup}>
                  <SelectArrow className={styles.customArrow} />
                  <SelectList>
                    {fruits.map((item) => (
                      <SelectItem key={item.value} value={item.value}>
                        <SelectItemIndicator />
                        <SelectItemText>{item.label}</SelectItemText>
                      </SelectItem>
                    ))}
                  </SelectList>
                </SelectPopup>
              </SelectPositioner>
            </SelectPortal>
          </Select>
        );
      }
    `}
