## Choosing The Right Component

Use `Select` when users choose from a fixed collection and the trigger should stay read-only.

- Choose **`Combobox`** when typing is part of finding or creating the value.
- Choose **`Select`** when the user picks from known values, including grouped or multiple values.

## Basic

Start with `Select`, pass a `collection`, render the visible field through `Label`, `Control`,
`Trigger`, `ValueText`, `ClearTrigger`, and `Indicator`, then portal the popup structure explicitly.
Keep `HiddenSelect` in the tree for native form submission and browser autofill.

<Preview>
  <SelectExample />

  <Preview.Code>
    {`
      import { Portal, Select, createListCollection } from "moduix";

      const fruits = createListCollection({
        items: [
          { label: "Apple", value: "apple" },
          { label: "Banana", value: "banana" },
          { label: "Blueberry", value: "blueberry" },
        ],
      });

      export function SelectDemo() {
        return (
          <Select collection={fruits}>
            <Select.Label>Choose fruit</Select.Label>
            <Select.Control>
              <Select.Trigger>
                <Select.ValueText placeholder="Select an option" />
              </Select.Trigger>
              <Select.Indicators>
                <Select.ClearTrigger aria-label="Clear selection" />
                <Select.Indicator />
              </Select.Indicators>
            </Select.Control>
            <Portal>
              <Select.Positioner>
                <Select.Content>
                  <Select.ItemGroup>
                    <Select.ItemGroupLabel>Fruits</Select.ItemGroupLabel>
                    {fruits.items.map((item) => (
                      <Select.Item key={item.value} item={item}>
                        <Select.ItemText>{item.label}</Select.ItemText>
                        <Select.ItemIndicator />
                      </Select.Item>
                    ))}
                  </Select.ItemGroup>
                </Select.Content>
              </Select.Positioner>
            </Portal>
            <Select.HiddenSelect />
          </Select>
        );
      }
    `}

## Anatomy

```text
Select / Select.Root
├─ Select.Label
├─ Select.Control
│  ├─ Select.Trigger
│  │  └─ Select.ValueText
│  └─ Select.Indicators
│     ├─ Select.ClearTrigger
│     └─ Select.Indicator
├─ Portal
│  └─ Select.Positioner
│     └─ Select.Content
│        ├─ Select.List
│        └─ Select.ItemGroup
│           ├─ Select.ItemGroupLabel
│           └─ Select.Item[item]
│              ├─ Select.ItemText
│              └─ Select.ItemIndicator
└─ Select.HiddenSelect
```

| Part                             | Role                                                               |
| -------------------------------- | ------------------------------------------------------------------ |
| `Select` / `Select.Root`         | Root state, collection, selection, popup lifecycle, and form data. |
| `Select.RootProvider`            | Renders a select tree from state returned by `useSelect`.          |
| `Select.Label`                   | Accessible label inherited by the trigger and hidden select.       |
| `Select.Control`                 | Visual field wrapper that receives focus, invalid, and open state. |
| `Select.Trigger`                 | Button that opens the popup and owns keyboard entry.               |
| `Select.ValueText`               | Placeholder or selected item labels.                               |
| `Select.ClearTrigger`            | Button that clears the current value.                              |
| `Select.Indicator`               | Trigger indicator with the moduix default chevron icon.            |
| `Select.Indicators`              | Moduix wrapper that aligns clear and indicator buttons.            |
| `Select.Positioner`              | Floating layer that exposes Ark positioning CSS variables.         |
| `Select.Content`                 | Popup surface and listbox container.                               |
| `Select.List`                    | Optional list wrapper for virtualized or custom list layouts.      |
| `Select.ItemGroup`               | Group wrapper for related collection items.                        |
| `Select.ItemGroupLabel`          | Label for a collection group.                                      |
| `Select.Item`                    | Selectable row. Pass the collection item through `item`.           |
| `Select.ItemText`                | Text used for display and accessibility.                           |
| `Select.ItemIndicator`           | Selected-state indicator with the moduix default check icon.       |
| `Select.HiddenSelect`            | Native hidden `<select>` for forms, autofill, and reset behavior.  |
| `Select.Context` / `ItemContext` | Inline access to root and item state.                              |
| `Portal`                         | Ark portal exported by moduix for popup mounting.                  |

## Composition

### Default Props

| Prop                    | Default | Notes                                                          |
| ----------------------- | ------- | -------------------------------------------------------------- |
| `closeOnSelect`         | `true`  | Closes the popup after item selection.                         |
| `composite`             | `true`  | Allows composition with nested composite widgets.              |
| `defaultValue`          | `[]`    | Values are string arrays, including single selection.          |
| `lazyMount`             | `false` | Defers popup content mounting until opened.                    |
| `loopFocus`             | `false` | Keeps arrow navigation from wrapping by default.               |
| `multiple`              | `false` | Enables multiple values when set.                              |
| `positioning`           | -       | Ark floating positioning options.                              |
| `unmountOnExit`         | `false` | Removes popup content after close when set with lazy mounting. |
| `ValueText.placeholder` | -       | Text shown when the value array is empty.                      |

### State And Callbacks

`value`, `defaultValue`, and `onValueChange(details)` use string arrays. Read
`details.value` for selected keys and `details.items` for the selected collection items.

`onOpenChange(details)`, `onHighlightChange(details)`, and `onSelect(details)` keep Ark's detail
object signatures. `Field.Root` and `Fieldset.Root` can provide disabled, invalid, required, and
read-only state through Ark context.

### Styling Hooks

- Root and root provider expose `data-slot="select-root"` and `data-slot="select-root-provider"`.
- Ark parts keep `data-scope="select"` and `data-part` for anatomy selectors.
- State styling uses `data-state="open" | "closed"`, `data-focus`, `data-invalid`,
  `data-disabled`, `data-readonly`, `data-placeholder-shown`, `data-highlighted`, and
  `data-state="checked" | "unchecked"` on items.
- `Select.Positioner` exposes `--reference-width`, `--reference-height`, `--available-width`,
  `--available-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`.
