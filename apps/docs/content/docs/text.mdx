## API Reference

There is no dedicated Ark UI `Text` primitive behind this component. `Text` is the shipped moduix
typography wrapper for body copy, helper text, inline emphasis, and simple semantic root swaps.

## Choosing the right component

Use `Text` for body copy, helper copy, inline emphasis, and supporting UI text that should stay on
the moduix text scale.

- Choose **`Heading`** for document structure and heading hierarchy.
- Choose **`Text`** when the content should keep paragraph or inline semantics with body-text
  styling.
- Choose local CSS or layout primitives when the concern is spacing, truncation, or rich-text
  layout rather than typography tokens.

## Basic

Start with the default `p` root. Use `as` only when the semantic HTML tag should change while the
same text styling contract stays in place.

## Anatomy

`Text` renders a single root element.

```text
Text
└─ text or inline content
```

| Part   | Role                                                                |
| ------ | ------------------------------------------------------------------- |
| `Text` | Semantic text root with design-system typography, tone, and sizing. |

## Composition

### When to stay high-level

Stay with `Text` and, when needed, the intrinsic `as` prop for regular body copy, metadata, inline
emphasis, and descriptive text.

Use `as` when a native semantic tag is enough:

- default `p` for paragraphs and body copy
- `span` for inline copy inside other text flows
- `small` for supporting metadata
- `strong` and `em` when semantic emphasis is actually correct
- `div` when the content must stay block-level without paragraph semantics

### When to go custom

Use `render` only when the root must be a custom component such as a router link or app-level
primitive that owns its own element contract.

### Default props

| Prop     | Default         | Values                                                 |
| -------- | --------------- | ------------------------------------------------------ |
| `as`     | `p`             | `p`, `span`, `small`, `strong`, `em`, `div`            |
| `render` | Unset           | React element or render function                       |
| `size`   | Render-based    | `xs`, `sm`, `md`, `lg`, `xl`                           |
| `weight` | Render-based    | `regular`, `medium`, `semibold`, `bold`                |
| `tone`   | `default`       | `default`, `muted`, `subtle`, `primary`, `destructive` |
| `align`  | Unset (`start`) | `left`, `center`, `right`                              |

Resolved defaults for `size` and `weight`:

- default root (`p`): `size="md"`, `weight="regular"`
- `as="small"` or `render={<small />}`: `size="sm"`, `weight="regular"`
- `as="strong"` or `render={<strong />}`: `size="md"`, `weight="semibold"`
- other intrinsic elements and custom components: `size="md"`, `weight="regular"`

### Behavioral notes

- `Text` always renders one root element and applies `data-slot="text-root"`.
- It also exposes `data-size`, `data-weight`, and `data-tone` on every render, plus `data-align`
  only when `align` is provided.
- The base styles reset paragraph margin to `0`, keep `text-align: start` by default, and wrap long
  words with `overflow-wrap: break-word`.
- `Text` does not add interaction behavior, truncation helpers, rich-text parsing, or layout
  spacing.

### Styling hooks

- `data-slot="text-root"` is the stable selector on the exported root.
- `data-size` reflects the resolved visual size: `xs`, `sm`, `md`, `lg`, or `xl`.
- `data-weight` reflects the resolved weight preset: `regular`, `medium`, `semibold`, or `bold`.
- `data-tone` reflects the active tone preset: `default`, `muted`, `subtle`, `primary`, or
  `destructive`.
- `data-align` appears only when `align` is explicitly set to `left`, `center`, or `right`.
