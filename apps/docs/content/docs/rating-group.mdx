## Anatomy

```text
RatingGroup.Root
├─ RatingGroup.Label
└─ RatingGroup.Control
   ├─ RatingGroup.Items (recommended)
   │  └─ RatingGroup.Item[index]
   │     └─ RatingGroup.ItemIndicator
   ├─ ArkRatingGroup.Context (advanced, optional)
   │  └─ RatingGroup.Item[index]
   │     └─ RatingGroup.ItemIndicator
   └─ RatingGroup.HiddenInput (advanced custom item tree)

RatingGroup.RootProvider
└─ same part tree connected to an Ark useRatingGroup() store
```

| Part                               | Role                                                         |
| ---------------------------------- | ------------------------------------------------------------ |
| `RatingGroup` / `RatingGroup.Root` | Controls rating value, keyboard behavior, and form state.    |
| `RatingGroup.RootProvider`         | Connects parts to state created with Ark `useRatingGroup()`. |
| `RatingGroup.Label`                | Accessible label for the rating group.                       |
| `RatingGroup.Control`              | Groups the interactive rating items.                         |
| `RatingGroup.Items`                | Renders Ark items, default stars, and the hidden form input. |
| `RatingGroup.Item`                 | One selectable rating item; requires an `index`.             |
| `RatingGroup.ItemIndicator`        | Moduix star visual that follows Ark item context.            |
| `RatingGroup.HiddenInput`          | Ark hidden input synchronized with the rating value.         |

## Composition

### Recommended composition

`RatingGroup` is the short root form. It owns Ark value state, hover state, focus movement, labels,
form integration, and keyboard behavior. Use `RatingGroup.Items` for standard generated star
items and the synchronized hidden form input. Pass a custom `RatingGroup.ItemIndicator` as its
child to change the icon while retaining the same item behavior.

### When to go custom

Use Ark `RatingGroup.Context`, `RatingGroup.ItemContext`, or `useRatingGroupItemContext()` when a
custom item tree needs `highlighted`, `checked`, or `half` state. Render `RatingGroup.HiddenInput`
explicitly with that tree when it needs native form submission or reset. Use
`RatingGroup.RootProvider` with Ark `useRatingGroup()` when rating state must be created outside
the rendered subtree.

All Ark parts support `asChild`. Keep the child a single semantic element that can preserve the
part's focus and interaction behavior.

### Default props

| Prop                        | Default | Notes                                                 |
| --------------------------- | ------- | ----------------------------------------------------- |
| `count`                     | `5`     | Ark item count.                                       |
| `size`                      | `md`    | Moduix styling size: `xs`, `sm`, `md`, `lg`, or `xl`. |
| `RatingGroup.Items`         | —       | Renders default items and the hidden form input.      |
| `RatingGroup.ItemIndicator` | star    | Renders moduix star foreground/background by default. |

### Behavioral notes

- `value` and `defaultValue` are numbers.
- `onValueChange` receives Ark details; read the value from `details.value`.
- `onHoverChange` receives Ark hover details.
- `allowHalf` enables half-step pointer selection and half-star rendering.
- `name`, `form`, and `required` work with the `HiddenInput` rendered by `RatingGroup.Items`.
- `disabled` and `readOnly` are handled by Ark and reflected through data attributes.

## Styling

Style `RatingGroup` through `className`, Ark `data-scope="rating-group"` / `data-part`
attributes, Ark state attributes, moduix `data-slot` hooks, and the registered
`--rating-group-*` CSS variables.

### Styling hooks

- `data-slot="rating-group-root"` and `data-slot="rating-group-root-provider"`
- `data-slot="rating-group-label"`
- `data-slot="rating-group-control"`
- `data-slot="rating-group-item"`
- `data-slot="rating-group-item-indicator"`
- `data-slot="rating-group-hidden-input"`
- `data-size="xs" | "sm" | "md" | "lg" | "xl"` on the root or root provider
- Ark label attributes: `data-disabled` and `data-required`
- Ark control attributes: `data-disabled` and `data-readonly`
- Ark item attributes: `data-checked`, `data-highlighted`, `data-half`, `data-disabled`, and `data-readonly`
