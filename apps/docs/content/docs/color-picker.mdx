## Choosing the right component

Use `ColorPicker` when users need to choose or fine-tune a color. Use `Input` when users only need
to paste a known color value and direct color manipulation is not useful.

## Basic

Start with Ark's explicit color picker anatomy: a root, visible control, trigger swatch, portal,
positioner, content panel, color area, channel sliders, optional swatches, and a hidden input for
forms.

## Anatomy

```text
ColorPicker.Root
├─ ColorPicker.Label
├─ ColorPicker.Control
│  ├─ ColorPicker.ChannelInput[channel]
│  └─ ColorPicker.Trigger
│     ├─ ColorPicker.TransparencyGrid
│     └─ ColorPicker.ValueSwatch
├─ Portal
│  └─ ColorPicker.Positioner
│     └─ ColorPicker.Content
│        ├─ ColorPicker.Area
│        │  ├─ ColorPicker.AreaBackground
│        │  └─ ColorPicker.AreaThumb
│        ├─ ColorPicker.EyeDropperTrigger
│        ├─ ColorPicker.ChannelSlider[channel]
│        │  ├─ ColorPicker.TransparencyGrid
│        │  ├─ ColorPicker.ChannelSliderTrack
│        │  └─ ColorPicker.ChannelSliderThumb
│        ├─ ColorPicker.SwatchGroup
│        │  └─ ColorPicker.SwatchTrigger[value]
│        │     └─ ColorPicker.Swatch[value]
│        │        └─ ColorPicker.SwatchIndicator
│        └─ ColorPicker.View[format]
│           └─ ColorPicker.ChannelInput[channel]
└─ ColorPicker.HiddenInput
```

| Part                        | Role                                                               |
| --------------------------- | ------------------------------------------------------------------ |
| `ColorPicker.Root`          | Owns color value, format, popup state, and form integration.       |
| `ColorPicker.Control`       | Groups visible inputs and trigger.                                 |
| `ColorPicker.Trigger`       | Button that opens the popup and commonly renders the value swatch. |
| `ColorPicker.Content`       | Actual Ark popup content part. It does not create a portal.        |
| `ColorPicker.Area`          | Two-dimensional color interaction surface.                         |
| `ColorPicker.ChannelSlider` | One-dimensional channel control for hue, alpha, or other channels. |
| `ColorPicker.ChannelInput`  | Text or numeric input for a specific color channel.                |
| `ColorPicker.SwatchTrigger` | Selectable preset color button.                                    |
| `ColorPicker.ValueSwatch`   | Visual swatch for the current value.                               |
| `ColorPicker.HiddenInput`   | Hidden form field for native submission and reset synchronization. |

`ColorPicker.RootProvider`, `ColorPicker.Context`, `parseColor`, `useColorPicker`, and
`useColorPickerContext` expose the corresponding Ark state paths.

## Composition

### Recommended composition

Keep `Portal`, `ColorPicker.Positioner`, and `ColorPicker.Content` visible for popup pickers.
`Content` is the actual Ark part; it does not render the portal or positioner internally.

Use `ColorPicker.HiddenInput` when the color should participate in native form submission or form
reset behavior. Put it inside the same root, and pass `name` on `ColorPicker.Root` or register the
hidden input with a form library.

### When to go custom

Use `inline` on `ColorPicker.Root` when the area, sliders, inputs, and swatches should render as a
permanent panel instead of a popup. Use `ColorPicker.View` to show inputs for one format at a time.

Use `ColorPicker.Context` for local state reads and `useColorPicker` with `ColorPicker.RootProvider`
when the state must be owned outside the rendered subtree. Do not render `Root` and `RootProvider`
for the same state instance.

Use `asChild` on exported Ark parts when another semantic element must own the DOM node. The child
must stay a single element and preserve the required focus and interaction behavior.

### Default props

- Ark defaults to `defaultValue={parseColor("#000000")}` and `defaultFormat="rgba"`.
- `ColorPicker.Content` defaults to `16rem` wide so it does not collapse to the trigger swatch width.
- `ColorPicker.EyeDropperTrigger` renders a moduix pipette icon when children are omitted.
- `ColorPicker.SwatchIndicator` renders a moduix check icon when children are omitted.
- The styled root defaults to `16rem` wide and remains constrained by `100%`.

### Behavioral notes

- `value` and `defaultValue` use Ark `Color` objects. Use `parseColor()` for string defaults.
- `onValueChange(details)` and `onValueChangeEnd(details)` preserve Ark detail objects.
- `onFormatChange(details)` and `onOpenChange(details)` preserve Ark detail objects.
- `ids` is forwarded to Ark for shared accessibility IDs across custom compositions.
- Field and fieldset state can flow through Ark-compatible `disabled`, `invalid`, `required`, and
  `readOnly` root props.
- All Ark DOM parts preserve `asChild`.

### Styling hooks

Use the part that owns the visual concern:

- `ColorPicker.Root` for width, gap, shared color, and disabled opacity
- `ColorPicker.Control`, `ChannelInput`, `Trigger`, `FormatSelect`, and `FormatTrigger` for the
  visible field controls
- `ColorPicker.Positioner` and `ColorPicker.Content` for popup layering, width, padding, radius,
  shadow, and state animations
- `ColorPicker.Area`, `AreaBackground`, and `AreaThumb` for the two-dimensional color surface
- `ColorPicker.ChannelSlider`, `ChannelSliderTrack`, and `ChannelSliderThumb` for hue, alpha, and
  other channel sliders
- `ColorPicker.SwatchGroup`, `SwatchTrigger`, `Swatch`, and `SwatchIndicator` for preset color
  grids
- `ColorPicker.ValueSwatch`, `ValueText`, `View`, and `TransparencyGrid` for value display and
  format-specific layouts

The main state hooks are `data-state`, `data-focus`, `data-invalid`, `data-disabled`,
`data-readonly`, `data-required`, `data-channel`, `data-orientation`, `data-value`,
`data-placement`, and `data-side`. Ark runtime variables include `--color`, `--value`,
`--available-width`, `--available-height`, and `--transform-origin`.
