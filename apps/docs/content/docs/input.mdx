## API Reference

Ark UI does not ship a dedicated input primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance, and delegates field behavior to
[Ark Field.Input](https://ark-ui.com/docs/components/field).

## Choosing the right component

- Choose **`Input`** for a plain single-line native input.
- Choose **`InputGroup`** for inline addons, text, or actions.
- Choose **`PasswordInput`** for the existing visibility-toggle wrapper.
- Choose **`NumberInput`** for numeric parsing and stepping.
- Choose **`Textarea`** for multi-line content.
- Choose **`Editable`** for preview-first text that switches into edit mode.

## Anatomy

```text
Field.Root (optional)
└─ Input
```

| Part                   | Role                                                                  |
| ---------------------- | --------------------------------------------------------------------- |
| `Input` / `Input.Root` | Styled Ark `Field.Input` that renders the native single-line control. |

## Composition

### Recommended composition

Render `Input` inside `Field` for labels, helper/error text, ids, and shared `disabled`, `invalid`,
`required`, or `readOnly` state. `Fieldset` can provide disabled state to nested fields.

Use native input props and events. Controlled inputs use `value` and `onChange(event)`;
uncontrolled inputs use `defaultValue`.

### When to go custom

Use `asChild` when one semantic input element must own the rendered node. Ark merges the wrapper
props, field props, class, and ref into that single child.

For a control that is not input-compatible, do not force it through `Input`; compose it with a
component that owns the correct Ark control primitive.

### Default props

| Prop       | Default | Notes                                                  |
| ---------- | ------- | ------------------------------------------------------ |
| `size`     | `md`    | Visual size: `xs`, `sm`, `md`, `lg`, or `xl`.          |
| `htmlSize` | —       | Writes the native character-based `input[size]` value. |
| `asChild`  | `false` | Merges props into one semantic child through Ark Slot. |

### Behavioral notes

- The ref points to the real `HTMLInputElement`.
- No `HiddenInput` is needed because `Input` is the native form control.
- Ark `Field` supplies `aria-describedby`, `aria-invalid`, ids, and state props.
- `Input` does not manage value, validity, focus, or keyboard navigation.

### Styling hooks

- `data-slot="input-root"`
- `data-size="xs" | "sm" | "md" | "lg" | "xl"`
- `data-scope="field"` and `data-part="input"`
- Ark `data-invalid`, `data-required`, and `data-readonly` inside `Field`
- native `:disabled`, `:read-only`, `:focus-visible`, and `[aria-invalid="true"]`
