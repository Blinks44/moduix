## Choosing the right component

Use `ToggleGroup` when several pressed buttons belong to one shared value set.

- Choose **`ToggleGroup`** for segmented controls, compact filter sets, and formatting toolbars.
- Choose **`Toggle`** for one standalone two-state action.
- Choose **`Tabs`** when selection also swaps associated panels.
- Choose **`RadioGroup`** when the choices should read like form options.

## Anatomy

```text
ToggleGroup / ToggleGroup.Root
â””â”€ ToggleGroup.Item[value]

ToggleGroup.RootProvider
â””â”€ ToggleGroup.Item[value]
```

| Part                               | Role                                                                                |
| ---------------------------------- | ----------------------------------------------------------------------------------- |
| `ToggleGroup` / `ToggleGroup.Root` | Root state machine for selected values, roving focus, orientation, and disabled.    |
| `ToggleGroup.Item`                 | Button item with a required `value` and Ark `data-state` on/off state.              |
| `ToggleGroup.RootProvider`         | Root driven by a `useToggleGroup()` state instance owned outside the rendered tree. |
| `ToggleGroup.Context`              | Render-prop access to the current Ark toggle group state.                           |

## Composition

### Recommended composition

Use `ToggleGroup` for the root and `ToggleGroup.Item` for each button. `value` and
`defaultValue` are string arrays in both single and multiple-selection modes.

moduix adds two visual props on `ToggleGroup`, `ToggleGroup.Root`, `ToggleGroup.RootProvider`, and
`ToggleGroup.Item`:

| Prop      | Default   | Values                                                  |
| --------- | --------- | ------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `ghost`                           |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `icon-sm`, `icon-md`, `icon-lg` |

### When to go custom

Use `asChild` on `ToggleGroup.Item` when another semantic button must own the rendered element.
Keep the child single and button-like so Ark can merge interaction props and ARIA state.

Use `useToggleGroup()` with `ToggleGroup.RootProvider` when state must be created outside the
rendered root. Do not render `ToggleGroup.Root` and `ToggleGroup.RootProvider` for the same state
instance.

Use `ToggleGroup.Context` for inline render-prop access and `useToggleGroupContext()` inside
reusable descendants. Both must render below a root or root provider.

### Default props

| Prop           | Default      | Notes                                                            |
| -------------- | ------------ | ---------------------------------------------------------------- |
| `deselectable` | `true`       | Allows empty selection when `multiple` is not enabled.           |
| `loopFocus`    | `true`       | Wraps arrow-key focus from last to first item and back.          |
| `orientation`  | `horizontal` | Controls layout direction and matching keyboard navigation.      |
| `rovingFocus`  | `true`       | Uses roving tab index so one item participates in the tab order. |
| `multiple`     | `false`      | Allows several items to stay pressed at the same time.           |
| `variant`      | `default`    | moduix visual treatment.                                         |
| `size`         | `md`         | moduix item sizing.                                              |

### Behavioral notes

- `onValueChange` receives Ark details: use `details.value` for the selected string array.
- `deselectable` is ignored when `multiple` is enabled.
- The group needs an accessible name from `aria-label` or `aria-labelledby`.
- Icon-only items need their own accessible name such as `aria-label="Align left"`.
- Use `disabled` on the group for the whole set or on one item for a single unavailable choice.
- Use `ids` to provide stable root and item IDs when composing the group with other accessible UI.
- Single-selection groups use `radiogroup`/`radio` semantics; multiple groups use `group` and
  `aria-pressed`.
- Horizontal groups use Left/Right arrows, vertical groups use Up/Down, and Home/End move to the
  first/last enabled item. Space/Enter activate the focused button, and `loopFocus` controls
  whether arrow navigation wraps.
- Ark writes `data-scope="toggle-group"`, `data-part`, `data-orientation`, `data-disabled`,
  `data-focus`, and item `data-state="on" | "off"` for styling.

## Examples

### Controlled

Control `value` from React state when the group needs to stay in sync with surrounding UI.

<Preview>
  <ControlledToggleGroupExample />

  <Preview.Code>
    {`
      import { ToggleGroup } from "@moduix/react";
      import { useState } from "react";
      import styles from "./toggle-group.module.css";

      export function ControlledToggleGroupDemo() {
        const [value, setValue] = useState(["left"] as string[]);

        return (
          <div className={styles.stack}>
            <ToggleGroup
              value={value}
              onValueChange={(details) => setValue(details.value)}
              aria-label="Text alignment"
            >
              {alignmentItems.map((item) => (
                <ToggleGroup.Item key={item.value} value={item.value}>
                  {item.label}
                </ToggleGroup.Item>
              ))}
            </ToggleGroup>
            <span className={styles.hint}>Current value: {value.join(", ") || "empty"}</span>
          </div>
        );
      }
    `}

  </Preview.Code>

### Root Provider

Use `useToggleGroup()` and `ToggleGroup.RootProvider` when the state instance must be shared before
the root is rendered.

<Preview>

      import styles from "./toggle-group.module.css";

      export function RootProviderToggleGroupDemo() {

        return (
          <div className={styles.stack}>

### Context

Use `ToggleGroup.Context` when item content needs the current selected values without duplicating
the group state.

<Preview>
  <ToggleGroupContextExample />

  <Preview.Code>
    {`
      import { ToggleGroup } from "@moduix/react";
      import styles from "./toggle-group.module.css";

      export function ContextToggleGroupDemo() {
        return (
          <ToggleGroup
            defaultValue={["left"]}
            aria-label="Text alignment"
            className={styles.contextGroup}
          >
            <ToggleGroup.Context>
              {(context) =>
                alignmentItems.map((item) => (
                  <ToggleGroup.Item key={item.value} value={item.value}>
                    {context.value.includes(item.value) ? \`\${item.label} selected\` : item.label}
                  </ToggleGroup.Item>

            </ToggleGroup.Context>

## Styling

`ToggleGroup` exposes stable `data-slot` hooks for Moduix and Ark anatomy/state attributes for
behavioral styling.

### CSS Properties

### Styling hooks

- `data-slot="toggle-group-root"` and `data-slot="toggle-group-root-provider"` are stable root selectors.
- `data-slot="toggle-group-item"` is the stable item selector.
- `data-scope="toggle-group"` and `data-part="root" | "item"` come from Ark.
- `data-state="on" | "off"` is written on each item.
- `data-orientation="horizontal" | "vertical"` is written on root and item parts.
- `data-disabled` and `data-focus` are Ark state attributes.
- `data-variant` and `data-size` reflect the Moduix visual props.
