## Anatomy

`ToggleGroup` coordinates a set of `ToggleGroupItem` buttons. Keep related choices in one group so
keyboard navigation and selection rules stay consistent.

```text
ToggleGroup
â””â”€ ToggleGroupItem[value]
   â””â”€ label or icon content
```

```tsx
<ToggleGroup defaultValue={['left']} aria-label="Text alignment">
  <ToggleGroupItem value="left">Left</ToggleGroupItem>
  <ToggleGroupItem value="center">Center</ToggleGroupItem>
  <ToggleGroupItem value="right">Right</ToggleGroupItem>
</ToggleGroup>
```

| Part              | Role                                                                                                  |
| ----------------- | ----------------------------------------------------------------------------------------------------- |
| `ToggleGroup`     | Root state machine. Controls selected values, arrow-key navigation, and orientation.                  |
| `ToggleGroupItem` | Interactive toggle button with a unique `value` used by `defaultValue`, `value`, and `onValueChange`. |

`ToggleGroupItem` renders `Toggle`, so item styling supports `--toggle-*`. Use
`--toggle-group-*` for the container and `--toggle-*` for item states.

```css
.filterGroup {
  --toggle-group-bg: transparent;
  --toggle-group-border-color: transparent;
  --toggle-group-padding: 0;
}

.filterItem {
  --toggle-ghost-bg-hover: var(--color-accent);
  --toggle-ghost-bg-pressed: var(--color-primary);
  --toggle-ghost-color-pressed: var(--color-primary-foreground);
}
```

## Composition

Use `ToggleGroup` for shared state and behavior (`value`, `defaultValue`, `onValueChange`,
`multiple`, `orientation`, `disabled`, `loopFocus`). Use group-level `variant`, `size`, and
`className` when all items should match; override them on a specific `ToggleGroupItem` only when
one option needs different presentation.

### Defaults

| Prop          | Default      | Values                                                  |
| ------------- | ------------ | ------------------------------------------------------- |
| `variant`     | `default`    | `default`, `outline`, `ghost`                           |
| `size`        | `md`         | `xs`, `sm`, `md`, `lg`, `icon-sm`, `icon-md`, `icon-lg` |
| `multiple`    | `false`      | `true`, `false`                                         |
| `loopFocus`   | `true`       | `true`, `false`                                         |
| `orientation` | `horizontal` | `horizontal`, `vertical`                                |

`ToggleGroupItem` inherits `variant` and `size` from `ToggleGroup` by default and can override
them per item.

### Controlled

Control `value` from React state when the group needs to coordinate with other UI.

<Preview>
  <ControlledToggleGroupExample />

  <Preview.Code>
    {`
      import { BellIcon, CheckSmallIcon, StarIcon, ToggleGroup, ToggleGroupItem } from "moduix";
      import { useState } from "react";
      import styles from "./toggle-group.module.css";

      export function ControlledToggleGroupDemo() {
        const [value, setValue] = useState(["favorites"] as string[]);

        return (
          <div className={styles.stack}>
            <ToggleGroup
              value={value}
              onValueChange={setValue}
              aria-label="Controlled options"
              multiple
            >
              <ToggleGroupItem value="favorites">
                {value.includes("favorites") ? <CheckSmallIcon /> : <StarIcon />}
                Favorites
              </ToggleGroupItem>
              <ToggleGroupItem value="alerts">
                <BellIcon />
                Alerts
              </ToggleGroupItem>
            </ToggleGroup>
            <span className={styles.hint}>Current value: {value.join(", ") || "empty"}</span>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: flex;
        flex-direction: column;
        align-items: flex-start;
        gap: var(--spacing-3);
      }

      .hint {
        color: var(--color-muted-foreground);
        font-size: var(--text-xs);
        line-height: var(--line-height-text-xs);
      }
    `}

  </Preview.CSS>
</Preview>
