## Choosing the right component

Use `ToggleGroup` when several pressed buttons belong to one shared choice set.

- Choose **`ToggleGroup`** for segmented controls, compact filter sets, and formatting toolbars.
- Choose **`Toggle`** for one standalone two-state action.
- Choose **`Tabs`** when the selection also swaps associated panels.
- Choose **`RadioGroup`** when the choices should read like form options instead of pressed buttons.

## Basic

Start with `ToggleGroup` and `ToggleGroupItem`. This is the recommended path for segmented controls
and shared selection state.

## Anatomy

The normal consumer path is short:

```text
ToggleGroup
└─ ToggleGroupItem[value]
   └─ label and/or icon content
```

| Part              | Role                                                                                            |
| ----------------- | ----------------------------------------------------------------------------------------------- |
| `ToggleGroup`     | Root state machine for selected values, arrow-key navigation, orientation, and disabled state.  |
| `ToggleGroupItem` | Styled toggle button. Its `value` participates in `defaultValue`, `value`, and `onValueChange`. |

`ToggleGroup` writes `data-slot="toggle-group-root"`, `data-variant`, and `data-size` on the
root. `ToggleGroupItem` renders the moduix `Toggle` and writes `data-slot="toggle-group-item"`.

## Composition

### When to stay high-level

Stay with `ToggleGroup` and `ToggleGroupItem` for most segmented controls and compact multi-select
toolbars.

`ToggleGroupItem` inherits the group `variant` and `size` by default, so the common path stays
short and consistent with standalone `Toggle`.

moduix adds two visual props on the root:

| Prop      | Default   | Values                                                  |
| --------- | --------- | ------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `ghost`                           |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `icon-sm`, `icon-md`, `icon-lg` |

### When to go custom

Use raw `Toggle` composition inside `ToggleGroup` only when another composed control needs to own
the item surface or layout.

That advanced path still participates in group selection through `value`, but it does not inherit
the group `variant`, `size`, or `toggle-group-item` styling hooks automatically.

### Default props

| Prop          | Default      | Values / Notes           |
| ------------- | ------------ | ------------------------ |
| `multiple`    | `false`      | `true`, `false`          |
| `orientation` | `horizontal` | `horizontal`, `vertical` |
| `loopFocus`   | `true`       | `true`, `false`          |

### Behavioral notes

- `value` and `defaultValue` are always string arrays, even in single-selection mode.
- `onValueChange` always receives the next string array.
- The group needs an accessible name from `aria-label` or `aria-labelledby`.
- Icon-only items need their own accessible name such as `aria-label="Align left"`.
- Use `disabled` on the group when the whole choice set is unavailable, or on one item when only a
  single choice should be unavailable.

### Controlled

Control `value` from React state when the group needs to stay in sync with surrounding UI.

<Preview>
  <ControlledToggleGroupExample />

  <Preview.Code>
    {`
      import { BellIcon, CheckIcon, StarIcon, ToggleGroup, ToggleGroupItem } from "moduix";
      import { useState } from "react";
      import styles from "./toggle-group.module.css";

      export function ControlledToggleGroupDemo() {
        const [value, setValue] = useState(["favorites"] as string[]);

        return (
          <div className={styles.stack}>
            <ToggleGroup
              value={value}
              onValueChange={setValue}
              aria-label="Controlled options"
              multiple
            >
              <ToggleGroupItem value="favorites">
                {value.includes("favorites") ? <CheckIcon /> : <StarIcon />}
                Favorites
              </ToggleGroupItem>
              <ToggleGroupItem value="alerts">
                <BellIcon />
                Alerts
              </ToggleGroupItem>
            </ToggleGroup>
            <span className={styles.hint}>Current value: {value.join(", ") || "empty"}</span>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: flex;
        flex-direction: column;
        align-items: flex-start;
        gap: var(--spacing-3);
      }

        color: var(--color-muted-foreground);

## Styling

`ToggleGroup` exposes one stable root, stable item slot hooks, and the full `--toggle-group-*`
token surface used for group chrome, spacing, radius, and variant surfaces.

### Styling hooks

- `data-slot="toggle-group-root"` is the stable selector on the group root.
- `data-slot="toggle-group-item"` is the stable selector on the default item wrapper.
- `data-variant` reflects the current group variant.
- `data-size` reflects the inherited item size.
- `data-orientation` reflects `horizontal` or `vertical`.
- `data-multiple` is present when `multiple` is `true`.
- `ToggleGroupItem` also keeps the `Toggle` state hooks such as `data-pressed`, `data-disabled`,
  and the public `--toggle-*` item variables.
