Use `ToggleGroup` for shared selection state and `ToggleGroupItem` for each choice. Set `variant`
and `size` on the group when the whole set should share the same look.

This install path also pulls in the shared moduix foundation files that `Toggle Group` depends on, such
as styles, icons, and small utilities.

`ToggleGroupItem` renders the library `Toggle`, so the default path stays short while items still
inherit the same size and variant scale as standalone toggles. Raw `Toggle` composition inside a
group is still possible, but only `ToggleGroupItem` inherits the group `variant`, `size`, and
`toggle-group-item` styling hooks automatically.

## Anatomy

```text
ToggleGroup
└─ ToggleGroupItem[value]
   └─ label and/or icon content
```

| Part              | Role                                                                                            |
| ----------------- | ----------------------------------------------------------------------------------------------- |
| `ToggleGroup`     | Root state machine for selected values, arrow-key navigation, orientation, and disabled state.  |
| `ToggleGroupItem` | Styled toggle button. Its `value` participates in `defaultValue`, `value`, and `onValueChange`. |

## Composition

`ToggleGroup` stays intentionally thin:

- Base UI owns group state, roving focus, and keyboard behavior
- `ToggleGroupItem` inherits `variant` and `size` from the group by default
- `value`, `defaultValue`, and `onValueChange` always work with string arrays, even when `multiple`
  is not set
- raw `Toggle` or `render={<Toggle ... />}` items can join the group through `value`, but they must
  receive their own `variant` / `size` when they should not fall back to the `Toggle` defaults
- item-level `variant`, `size`, `className`, and `--toggle-*` overrides remain available when one choice needs different styling

Use these paths:

| Pattern                                       | Best fit                                               |
| --------------------------------------------- | ------------------------------------------------------ |
| text items + `defaultValue`                   | common segmented controls and simple filters           |
| `multiple`                                    | formatting toolbars and multi-select filters           |
| `value` + `onValueChange`                     | controlled state coordinated with surrounding UI       |
| `size="icon-md"` + `aria-label`               | compact icon-only groups                               |
| `className`, `--toggle-group-*`, `--toggle-*` | custom visuals without replacing the composition model |

### Defaults

| Prop          | Default      | Values                                                  |
| ------------- | ------------ | ------------------------------------------------------- |
| `variant`     | `default`    | `default`, `outline`, `ghost`                           |
| `size`        | `md`         | `xs`, `sm`, `md`, `lg`, `icon-sm`, `icon-md`, `icon-lg` |
| `multiple`    | `false`      | `true`, `false`                                         |
| `orientation` | `horizontal` | `horizontal`, `vertical`                                |
| `loopFocus`   | `true`       | `true`, `false`                                         |

Set `loopFocus={false}` when arrow-key navigation should stop at the first or last item instead of
wrapping.

### Controlled

Control `value` from React state when the group needs to stay in sync with other UI.

<Preview>
  <ControlledToggleGroupExample />

  <Preview.Code>
    {`
      import { BellIcon, CheckIcon, StarIcon, ToggleGroup, ToggleGroupItem } from "moduix";
      import { useState } from "react";
      import styles from "./toggle-group.module.css";

      export function ControlledToggleGroupDemo() {
        const [value, setValue] = useState(["favorites"] as string[]);

        return (
          <div className={styles.stack}>
            <ToggleGroup
              value={value}
              onValueChange={setValue}
              aria-label="Controlled options"
              multiple
            >
              <ToggleGroupItem value="favorites">
                {value.includes("favorites") ? <CheckIcon /> : <StarIcon />}
                Favorites
              </ToggleGroupItem>
              <ToggleGroupItem value="alerts">
                <BellIcon />
                Alerts
              </ToggleGroupItem>
            </ToggleGroup>
            <span className={styles.hint}>Current value: {value.join(", ") || "empty"}</span>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: flex;
        flex-direction: column;
        align-items: flex-start;
        gap: var(--spacing-3);
      }

      .hint {
        color: var(--color-muted-foreground);
        font-size: var(--text-xs);
        line-height: var(--line-height-text-xs);
      }
    `}

  </Preview.CSS>
</Preview>

## Custom Composition

Use `className` on the group and items when the default structure is right but the surface styling
needs to change. Group variables live under `--toggle-group-*`; item states still use `--toggle-*`.

<Preview>

        return (
          <ToggleGroup
            defaultValue={["day"]}
            aria-label="Schedule density"

              Day

              Week
