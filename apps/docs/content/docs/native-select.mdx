## API Reference

`NativeSelect` wraps Ark UI
[Field.Select](https://ark-ui.com/docs/components/field) and renders a real HTML `<select>`.
Chakra UI [Native Select](https://chakra-ui.com/docs/components/native-select) informs its visual
relationship with custom Select, while moduix keeps the public API to one component.

## Choosing the right component

Use `NativeSelect` for straightforward option lists where browser autofill, form reset, native
keyboard behavior, and mobile platform pickers are desirable. Use [`Select`](/docs/select) when the
interface needs a custom popup, collection APIs, rich items, clearing, multiple-value presentation,
or controlled popup behavior.

## Anatomy

`NativeSelect` keeps the real select as its form control and adds a non-interactive moduix
`ChevronDownIcon` for consistent alignment with `Select`.

```text
NativeSelect / NativeSelect.Root
└─ span[data-scope="native-select"][data-part="control"]
   ├─ select[data-scope="field"][data-part="select"][data-slot="native-select-root"]
   │  ├─ option
   │  └─ optgroup
   └─ span[data-scope="native-select"][data-part="indicator"]
      └─ ChevronDownIcon
```

| Part                                 | Role                                          |
| ------------------------------------ | --------------------------------------------- |
| `NativeSelect` / `NativeSelect.Root` | Styled native select with a visual indicator. |

## Composition

### Recommended composition

Use the short `NativeSelect` form with native children and native controlled or uncontrolled props.
Compose it inside `Field.Root` when the control needs a visible label, helper text, error text, or
shared required, invalid, or disabled state.

### When to go custom

Switch to `Select` when option rendering, popup positioning, collection state, or custom indicators
must be controlled by the application. Keep `NativeSelect` for browser-owned selection behavior.

### Default props

| Prop           | Default         | Notes                                               |
| -------------- | --------------- | --------------------------------------------------- |
| `value`        | -               | Native controlled value                             |
| `defaultValue` | -               | Native uncontrolled value                           |
| `onChange`     | -               | Native `ChangeEvent<HTMLSelectElement>`             |
| `multiple`     | `false`         | Enables native multiple selection                   |
| `size`         | browser default | Native visible row count, not a visual size variant |
| `disabled`     | `false`         | Native disabled state                               |
| `asChild`      | `false`         | Ark composition; normally leave disabled for select |

### Behavioral notes

- The ref targets the real `HTMLSelectElement`.
- Native submission, autofill, reset, required validation, options, and keyboard behavior are
  preserved.
- `multiple` and native `size` values greater than one use an intrinsic list height.
- `Field.Root` supplies ids, `aria-describedby`, invalid state, and native state props.
- HTML select does not support a read-only state.
- `ChevronDownIcon` keeps the collapsed control aligned with `Select`; the browser still owns the
  popup appearance.

## Styling

### CSS Properties

`NativeSelect` reuses the control-level `--select-*` variables so native and custom selects can
share dimensions, indicator spacing, borders, colors, radius, focus ring, disabled opacity, and
transitions. Its default padding and indicator geometry follow the shared 4px spacing scale.

### Styling hooks

- `className` styles the real select element.
- `data-scope="field"` and `data-part="select"` preserve the Ark Field styling contract.
- `data-slot="native-select-root"` is the stable moduix selector for the control.
- `data-slot="native-select-control"` and `data-slot="native-select-indicator"` target the internal
  layout and icon.
- Native `:focus-visible`, `:disabled`, `[aria-invalid="true"]`, `[multiple]`, and `[size]` selectors
  drive visual states.
