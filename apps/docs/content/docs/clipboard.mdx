## Composition

### Recommended composition

Stay with `Clipboard.Root`, `Clipboard.Label`, `Clipboard.Control`, `Clipboard.Input`,
`Clipboard.Trigger`, and `Clipboard.Indicator` for the normal Ark path.

Use `value`, `defaultValue`, `onValueChange(details)`, `onStatusChange(details)`, `timeout`, and
`ids` exactly as Ark defines them. moduix does not remap callback payloads or rename props on this
surface.

`Clipboard.Indicator` is the only sugar layer: it renders the moduix copy icon by default and swaps
to the check icon while copied state is active.

### When to go custom

Use `Clipboard.ValueText` when the value should be displayed as text instead of inside an input.

Use Ark `useClipboard()` with `Clipboard.RootProvider` when clipboard state must be created outside
the rendered subtree. Ark context and state hooks remain available directly from
`@ark-ui/react/clipboard`.

Use `asChild` on the Ark parts only when another element must own the rendered node. Keep the Ark
part tree visible instead of hiding it behind a convenience wrapper.

### Default props

| Prop                    | Default | Values / Notes                                       |
| ----------------------- | ------- | ---------------------------------------------------- |
| `timeout`               | `3000`  | Ark copied-state timeout in milliseconds             |
| `copied`                | â€”       | `Clipboard.Indicator` defaults to moduix `CheckIcon` |
| `children` on indicator | â€”       | `Clipboard.Indicator` defaults to moduix `CopyIcon`  |

### Behavioral notes

- `onValueChange(details)` exposes the next clipboard value as `details.value`.
- `onStatusChange(details)` follows Ark copy-status details and is the right hook for analytics,
  counters, or adjacent feedback.
- Ark applies `data-copied` to the relevant parts while copied state is active. By default moduix
  keeps the input and trigger visually stable and only swaps the indicator, while `data-copied`
  remains available for opt-in custom styling.
- The old `CopyButton` API was removed. There is no `copiedDuration`, `copyLabels`, `onCopy`, or
  `onCopyError` alias layer on `Clipboard`.

### Styling hooks

- `Clipboard.Root` -> `data-slot="clipboard-root"`
- `Clipboard.RootProvider` -> `data-slot="clipboard-root-provider"`
- `Clipboard.Label` -> `data-slot="clipboard-label"`
- `Clipboard.Control` -> `data-slot="clipboard-control"`
- `Clipboard.Input` -> `data-slot="clipboard-input"`
- `Clipboard.Trigger` -> `data-slot="clipboard-trigger"`
- `Clipboard.Indicator` -> `data-slot="clipboard-indicator"`
- `Clipboard.ValueText` -> `data-slot="clipboard-value-text"`
- Ark adds `data-copied` to `Root`, `Label`, `Control`, `Input`, and `Trigger` while copied state is active.
- Ark adds `data-readonly` to `Clipboard.Input` when the input is read-only.
- `Clipboard.Indicator` adds `data-slot="clipboard-indicator-idle-icon"` and
  `data-slot="clipboard-indicator-copied-icon"` for the default icon nodes.
