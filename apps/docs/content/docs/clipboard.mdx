## Anatomy

```text
Clipboard.Root | Clipboard.RootProvider
├─ Clipboard.Label (optional)
├─ Clipboard.Control
│  ├─ Clipboard.Input or Clipboard.ValueText
│  └─ Clipboard.Trigger
│     └─ Clipboard.Indicator (optional)
└─ Clipboard.Context (optional render-prop access)
```

| Part                           | Role                                                            |
| ------------------------------ | --------------------------------------------------------------- |
| `Clipboard` / `Clipboard.Root` | Owns clipboard state, copied lifecycle, and controlled value.   |
| `Clipboard.RootProvider`       | Injects a `useClipboard()` store created outside the subtree.   |
| `Clipboard.Label`              | Accessible label for the clipboard value.                       |
| `Clipboard.Control`            | Layout row for value surface and trigger.                       |
| `Clipboard.Input`              | Editable or read-only text field for the current value.         |
| `Clipboard.Trigger`            | Button that copies the current value.                           |
| `Clipboard.Indicator`          | Copy-state icon slot. moduix defaults to copy and check icons.  |
| `Clipboard.ValueText`          | Text-only value surface when an input is not the right control. |

## Composition

### Recommended composition

Stay with `Clipboard.Root`, `Clipboard.Label`, `Clipboard.Control`, `Clipboard.Input`,
`Clipboard.Trigger`, and `Clipboard.Indicator` for the normal Ark path.

Use `value`, `defaultValue`, `onValueChange(details)`, `onStatusChange(details)`, `timeout`, and
`ids` exactly as Ark defines them. moduix does not remap callback payloads or rename props on this
surface.

`Clipboard.Indicator` is the only sugar layer: it renders the moduix copy icon by default and swaps
to the check icon while copied state is active.

### When to go custom

Use `Clipboard.ValueText` when the value should be displayed as text instead of inside an input.

Use `useClipboard()` with `Clipboard.RootProvider` when clipboard state must be created outside the
rendered subtree.

Use `Clipboard.Context` or `useClipboardContext()` when nearby UI needs access to clipboard state
without adding a separate React state layer.

Use `asChild` on the Ark parts only when another element must own the rendered node. Keep the Ark
part tree visible instead of hiding it behind a convenience wrapper.

### Default props

| Prop                    | Default | Values / Notes                                       |
| ----------------------- | ------- | ---------------------------------------------------- |
| `timeout`               | `3000`  | Ark copied-state timeout in milliseconds             |
| `copied`                | —       | `Clipboard.Indicator` defaults to moduix `CheckIcon` |
| `children` on indicator | —       | `Clipboard.Indicator` defaults to moduix `CopyIcon`  |

### Behavioral notes

- `onValueChange(details)` exposes the next clipboard value as `details.value`.
- `onStatusChange(details)` follows Ark copy-status details and is the right hook for analytics,
  counters, or adjacent feedback.
- Ark applies `data-copied` to the relevant parts while copied state is active. By default moduix
  keeps the input and trigger visually stable and only swaps the indicator, while `data-copied`
  remains available for opt-in custom styling.
- The old `CopyButton` API was removed. There is no `copiedDuration`, `copyLabels`, `onCopy`, or
  `onCopyError` alias layer on `Clipboard`.

## Examples

### Controlled

Control the value when the text field should stay in sync with surrounding UI or an external source.

<Preview>
  <ControlledClipboardExample />

  <Preview.Code>
    {`
      import { Clipboard } from "@moduix/react";
      import { useState } from "react";

      export function ControlledClipboardDemo() {
        const [value, setValue] = useState("https://ark-ui.com");

        return (
          <Clipboard
            value={value}
            onValueChange={(details) => setValue(details.value)}
          >
            <Clipboard.Label>Share URL</Clipboard.Label>
            <Clipboard.Control>
              <Clipboard.Input />
              <Clipboard.Trigger aria-label="Copy URL">
                <Clipboard.Indicator />
              </Clipboard.Trigger>
            </Clipboard.Control>
          </Clipboard>
        );
      }
    `}

### Copy Status

Use `onStatusChange(details)` when surrounding UI should react to successful copies without
introducing a separate clipboard abstraction.

<Preview>
  <CopyStatusClipboardExample />

  <Preview.Code>
    {`
      import { Clipboard } from "@moduix/react";
      import { useState } from "react";

      export function ClipboardStatusDemo() {
        const [copyCount, setCopyCount] = useState(0);

        return (
          <Clipboard
            defaultValue="maps-platform-token"
            onStatusChange={(details) => {
              if (details.copied) {
                setCopyCount((value) => value + 1);
              }
            }}
          >
            <Clipboard.Control>
              <Clipboard.Trigger>
                <Clipboard.Indicator />
                <Clipboard.ValueText />
              </Clipboard.Trigger>
            </Clipboard.Control>
            <p>Copied {copyCount} times</p>
          </Clipboard>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{clipboardExampleCss}</Preview.CSS>
  <Preview.Data>{clipboardCopyStatusData}</Preview.Data>
</Preview>

### Root Provider

Use `useClipboard()` with `Clipboard.RootProvider` when clipboard state belongs outside the rendered
tree but the visible parts should stay Ark-shaped.

### Styling hooks

- `Clipboard.Root` -> `data-slot="clipboard-root"`
- `Clipboard.RootProvider` -> `data-slot="clipboard-root-provider"`
- `Clipboard.Label` -> `data-slot="clipboard-label"`
- `Clipboard.Control` -> `data-slot="clipboard-control"`
- `Clipboard.Input` -> `data-slot="clipboard-input"`
- `Clipboard.Trigger` -> `data-slot="clipboard-trigger"`
- `Clipboard.Indicator` -> `data-slot="clipboard-indicator"`
- `Clipboard.ValueText` -> `data-slot="clipboard-value-text"`
- Ark adds `data-copied` to `Root`, `Label`, `Control`, `Input`, and `Trigger` while copied state is active.
- Ark adds `data-readonly` to `Clipboard.Input` when the input is read-only.
- `Clipboard.Indicator` adds `data-slot="clipboard-indicator-idle-icon"` and
  `data-slot="clipboard-indicator-copied-icon"` for the default icon nodes.
