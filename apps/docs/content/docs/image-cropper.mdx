## Anatomy

```tsx
<ImageCropper>
  <ImageCropper.Viewport>
    <ImageCropper.Image />
    <ImageCropper.Selection>
      <ImageCropper.Grid />
      <ImageCropper.Handle />
    </ImageCropper.Selection>
  </ImageCropper.Viewport>
</ImageCropper>
```

| Part                                 | Role                                                                                 |
| ------------------------------------ | ------------------------------------------------------------------------------------ |
| `ImageCropper` / `ImageCropper.Root` | Root group that owns crop state, transforms, ARIA labels, and runtime CSS variables. |
| `ImageCropper.RootProvider`          | Root rendered from a `useImageCropper` instance created outside the tree.            |
| `ImageCropper.Viewport`              | Panning surface and clipped image viewport.                                          |
| `ImageCropper.Image`                 | Transformed image element controlled by Ark.                                         |
| `ImageCropper.Selection`             | Focusable crop selection and keyboard interaction target.                            |
| `ImageCropper.Grid`                  | Horizontal or vertical crop guide lines.                                             |
| `ImageCropper.Handle`                | Resize handle for each `ImageCropper.handles` position.                              |
| `ImageCropper.Context`               | Inline access to Ark cropper state and methods.                                      |

## Composition

### Recommended composition

Render the full cropper anatomy so users can style and rearrange the visible parts without losing
Ark behavior. Use `ImageCropper.handles` to render every handle position.

### When to go custom

Use `ImageCropper.Context` for toolbars inside the root. Use `useImageCropper` with
`ImageCropper.RootProvider` when controls, crop preview, or form integration need the cropper API
outside the root subtree. Keep `RootProvider` separate from `ImageCropper.Root` for the same state
instance.

### Default props

| Prop                     | Default                                  | Notes                                                        |
| ------------------------ | ---------------------------------------- | ------------------------------------------------------------ |
| `cropShape`              | `"rectangle"`                            | Use `"circle"` for avatar-style cropping.                    |
| `defaultZoom`            | `1`                                      | Initial image zoom.                                          |
| `defaultRotation`        | `0`                                      | Initial image rotation in degrees.                           |
| `defaultFlip`            | `{ horizontal: false, vertical: false }` | Initial flip state.                                          |
| `minZoom`                | `1`                                      | Minimum zoom factor.                                         |
| `maxZoom`                | `5`                                      | Maximum zoom factor.                                         |
| `minWidth` / `minHeight` | `40`                                     | Minimum crop selection size.                                 |
| `fixedCropArea`          | `false`                                  | When true, users pan the image underneath a fixed selection. |

### Behavioral notes

Ark manages pointer dragging, panning, wheel zoom, pinch zoom, keyboard movement, generated IDs,
ARIA labels, and crop output helpers. Arrow keys move the selection, `Alt` plus arrow keys resizes,
`Shift` and `Ctrl`/`Cmd` use the nudge-step props, and `+` / `-` zoom. Controlled state preserves
Ark detail objects through `onCropChange`, `onZoomChange`, `onRotationChange`, and `onFlipChange`.

### Controlled Zoom

Control `zoom` with `onZoomChange` and bound the interaction with `minZoom` and `maxZoom`.

<Preview>
  <ControlledZoomImageCropperExample />

  <Preview.Code>
    {`
      import { ImageCropper, ZoomInIcon, ZoomOutIcon } from "@moduix/react";
      import { useState } from "react";

      export function ControlledZoomImageCropperDemo() {
        const [zoom, setZoom] = useState(1);

        return (
          <div className="cropper-stack">
            <div className="cropper-toolbar">
              <button
                className="cropper-button cropper-icon-button"
                type="button"
                aria-label="Zoom out"
                onClick={() => setZoom((value) => Math.max(minZoom, value - 0.1))}
              >
                <ZoomOutIcon />
              </button>
              <output className="cropper-output">{zoom.toFixed(1)}x</output>
              <button
                className="cropper-button cropper-icon-button"
                type="button"
                aria-label="Zoom in"
                onClick={() => setZoom((value) => Math.min(maxZoom, value + 0.1))}
              >
                <ZoomInIcon />
              </button>
            </div>
            <ImageCropper
              zoom={zoom}
              minZoom={minZoom}
              maxZoom={maxZoom}
              onZoomChange={(details) => setZoom(details.zoom)}
            >
              <ImageCropper.Viewport>
                <ImageCropper.Image src={sampleImage} crossOrigin="anonymous" />
                <ImageCropper.Selection>
                  <ImageCropper.Grid axis="horizontal" />
                  <ImageCropper.Grid axis="vertical" />
                  {ImageCropper.handles.map((position) => (
                    <ImageCropper.Handle key={position} position={position} />
                  ))}
                </ImageCropper.Selection>
              </ImageCropper.Viewport>
            </ImageCropper>
          </div>
        );
      }
    `}

### Rotation, Flip, And Reset

Read the Ark context inside the root for toolbar actions such as zoom, rotate, flip, and reset.

<Preview>
  <TransformImageCropperExample />

  <Preview.Code>
    {`
      import {
        FlipHorizontalIcon,
        ImageCropper,
        RestartIcon,
        RotateCcwIcon,
        RotateCwIcon,
        ZoomInIcon,
        ZoomOutIcon,
      } from "@moduix/react";
      import { useState } from "react";

      export function TransformImageCropperDemo() {
        const [rotation, setRotation] = useState(0);
        const [flip, setFlip] = useState({ horizontal: false, vertical: false });

        return (
          <div className="cropper-stack">
            <ImageCropper
              rotation={rotation}
              flip={flip}
              onRotationChange={(details) => setRotation(details.rotation)}
              onFlipChange={(details) => setFlip(details.flip)}
            >
              <ImageCropper.Context>
                {(context) => (
                  <div className="cropper-toolbar">
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Zoom out"
                      onClick={() => context.zoomBy(-0.1)}
                    >
                      <ZoomOutIcon />
                    </button>
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Zoom in"
                      onClick={() => context.zoomBy(0.1)}
                    >
                      <ZoomInIcon />
                    </button>
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Rotate counterclockwise"
                      onClick={() => context.rotateBy(-90)}
                    >
                      <RotateCcwIcon />
                    </button>
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Rotate clockwise"
                      onClick={() => context.rotateBy(90)}
                    >
                      <RotateCwIcon />
                    </button>
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Flip horizontally"
                      onClick={() => context.flipHorizontally()}
                    >
                      <FlipHorizontalIcon />
                    </button>
                    <button
                      className="cropper-button cropper-icon-button"
                      type="button"
                      aria-label="Reset crop"
                      onClick={() => context.reset()}
                    >
                      <RestartIcon />
                    </button>
                  </div>
                )}
              </ImageCropper.Context>
              <ImageCropper.Viewport>
                <ImageCropper.Image src={sampleImage} crossOrigin="anonymous" />
                <ImageCropper.Selection>
                  <ImageCropper.Grid axis="horizontal" />
                  <ImageCropper.Grid axis="vertical" />
                  {ImageCropper.handles.map((position) => (
                    <ImageCropper.Handle key={position} position={position} />
                  ))}
                </ImageCropper.Selection>
              </ImageCropper.Viewport>
            </ImageCropper>
            <output className="cropper-output">
              rotation: {rotation}deg, horizontal flip: {String(flip.horizontal)}
            </output>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{imageCropperExampleCss}</Preview.CSS>
  <Preview.Data>{imageCropperSampleImageData}</Preview.Data>
</Preview>

### Crop Preview

Create the cropper with `useImageCropper` when another control needs to call
`getCroppedImage()`.

<Preview>
  <CropPreviewImageCropperExample />

  <Preview.Code>
    {`
      import { Button, ImageCropper, useImageCropper } from "@moduix/react";
      import { useState } from "react";

      export function CropPreviewImageCropperDemo() {
        const imageCropper = useImageCropper({ cropShape: "circle", aspectRatio: 1 });
        const [preview, setPreview] = useState(null as string | null);

        const handleCrop = async () => {
          const result = await imageCropper.getCroppedImage({ output: "dataUrl" });
          if (typeof result === "string") setPreview(result);
        };

        return (
          <div className="cropper-stack">
            <ImageCropper.RootProvider value={imageCropper}>
              <ImageCropper.Viewport>
                <ImageCropper.Image src={sampleImage} crossOrigin="anonymous" />
                <ImageCropper.Selection>
                  <ImageCropper.Grid axis="horizontal" />
                  <ImageCropper.Grid axis="vertical" />
                  {ImageCropper.handles.map((position) => (
                    <ImageCropper.Handle key={position} position={position} />
                  ))}
                </ImageCropper.Selection>
              </ImageCropper.Viewport>
            </ImageCropper.RootProvider>
            <Button type="button" onClick={handleCrop}>
              Crop image
            </Button>
            <div className="cropper-preview">{preview ? <img src={preview} alt="" /> : null}</div>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{imageCropperExampleCss}</Preview.CSS>

### Styling hooks

Target stable `data-slot` hooks such as `image-cropper-root`, `image-cropper-viewport`,
`image-cropper-image`, `image-cropper-selection`, `image-cropper-grid`, and
`image-cropper-handle`. Ark also exposes `data-scope="image-cropper"`, `data-part` values,
`data-shape`, `data-ready`, `data-disabled`, `data-dragging`, `data-panning`, `data-axis`, and
`data-position`.
