## Anatomy

```tsx
<ImageCropper>
  <ImageCropper.Viewport>
    <ImageCropper.Image />
    <ImageCropper.CropArea />
  </ImageCropper.Viewport>
</ImageCropper>
```

| Part                                 | Role                                                                                 |
| ------------------------------------ | ------------------------------------------------------------------------------------ |
| `ImageCropper` / `ImageCropper.Root` | Root group that owns crop state, transforms, ARIA labels, and runtime CSS variables. |
| `ImageCropper.RootProvider`          | Root rendered from a `useImageCropper` instance created outside the tree.            |
| `ImageCropper.Viewport`              | Panning surface and clipped image viewport.                                          |
| `ImageCropper.Image`                 | Transformed image element controlled by Ark.                                         |
| `ImageCropper.Selection`             | Focusable crop selection and keyboard interaction target.                            |
| `ImageCropper.Grid`                  | Horizontal or vertical crop guide lines.                                             |
| `ImageCropper.Handle`                | Resize handle for each `ImageCropper.handles` position.                              |
| `ImageCropper.CropArea`              | Recommended selection with both grids and every resize handle.                       |

## Composition

### Recommended composition

Use `ImageCropper.CropArea` for the common cropper anatomy. It keeps the Ark part tree intact while
removing repeated selection, grid, and handle markup. Pass `className`, `gridClassName`, or
`handleClassName` to style its rendered parts.

### When to go custom

Use `ImageCropper.Context` for toolbars inside the root. Use moduix `useImageCropper` with
`ImageCropper.RootProvider` when controls, crop preview, or form
integration need the cropper API outside the root subtree. Keep `RootProvider` separate from
`ImageCropper.Root` for the same state instance.

### Default props

| Prop                     | Default                                  | Notes                                                        |
| ------------------------ | ---------------------------------------- | ------------------------------------------------------------ |
| `cropShape`              | `"rectangle"`                            | Use `"circle"` for avatar-style cropping.                    |
| `defaultZoom`            | `1`                                      | Initial image zoom.                                          |
| `defaultRotation`        | `0`                                      | Initial image rotation in degrees.                           |
| `defaultFlip`            | `{ horizontal: false, vertical: false }` | Initial flip state.                                          |
| `minZoom`                | `1`                                      | Minimum zoom factor.                                         |
| `maxZoom`                | `5`                                      | Maximum zoom factor.                                         |
| `minWidth` / `minHeight` | `40`                                     | Minimum crop selection size.                                 |
| `fixedCropArea`          | `false`                                  | When true, users pan the image underneath a fixed selection. |

### Behavioral notes

Ark manages pointer dragging, panning, wheel zoom, pinch zoom, keyboard movement, generated IDs,
ARIA labels, and crop output helpers. Arrow keys move the selection, `Alt` plus arrow keys resizes,
`Shift` and `Ctrl`/`Cmd` use the nudge-step props, and `+` / `-` zoom. Controlled state preserves
Ark detail objects through `onCropChange`, `onZoomChange`, `onRotationChange`, and `onFlipChange`.

### Styling hooks

Target stable `data-slot` hooks such as `image-cropper-root`, `image-cropper-viewport`,
`image-cropper-image`, `image-cropper-selection`, `image-cropper-grid`, and
`image-cropper-handle`. Ark also exposes `data-scope="image-cropper"`, `data-part` values,
`data-shape`, `data-ready`, `data-disabled`, `data-dragging`, `data-panning`, `data-axis`, and
`data-position`.
