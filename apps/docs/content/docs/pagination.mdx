## Basic

Use `Pagination` as the short root form for Ark `Pagination.Root`. Render page buttons from
`Pagination.Context`, which exposes Ark's computed `pages` array.

<Preview>
  <PaginationExample />

  <Preview.Code>
    {`
      import { Pagination } from "@moduix/react";

      export function PaginationDemo() {
        return (
          <Pagination className="pagination-demo" count={200} pageSize={10} siblingCount={2}>
            <Pagination.PrevTrigger />
            <Pagination.Context>
              {(pagination) =>
                pagination.pages.map((page, index) =>
                  page.type === "page" ? (
                    <Pagination.Item key={index} {...page}>
                      {page.value}
                    </Pagination.Item>
                  ) : (
                    <Pagination.Ellipsis key={index} index={index} />
                  ),
                )
              }
            </Pagination.Context>
            <Pagination.NextTrigger />
          </Pagination>
        );
      }
    `}

## Anatomy

```text
Pagination.Root
├─ Pagination.FirstTrigger (optional)
├─ Pagination.PrevTrigger
├─ Pagination.Context (optional render-prop access)
│  ├─ Pagination.Item
│  └─ Pagination.Ellipsis
├─ Pagination.NextTrigger
└─ Pagination.LastTrigger (optional)

Pagination.RootProvider
└─ same trigger, item, ellipsis, and context tree connected to usePagination()
```

| Part                             | Role                                                            |
| -------------------------------- | --------------------------------------------------------------- |
| `Pagination` / `Pagination.Root` | Navigation root that owns page and page-size state.             |
| `Pagination.RootProvider`        | Connects parts to state created with `usePagination()`.         |
| `Pagination.Context`             | Reads root state and helpers through an Ark render prop.        |
| `Pagination.Item`                | Interactive page item. Receives `{ type: 'page', value }`.      |
| `Pagination.Ellipsis`            | Non-interactive collapsed-range marker with a required `index`. |
| `Pagination.PrevTrigger`         | Moves to the previous page and disables itself at the start.    |
| `Pagination.NextTrigger`         | Moves to the next page and disables itself at the end.          |
| `Pagination.FirstTrigger`        | Moves to the first page and disables itself at the start.       |
| `Pagination.LastTrigger`         | Moves to the last page and disables itself at the end.          |

## Composition

### Recommended composition

`Pagination.Root` owns keyboard behavior, current page, page size, disabled trigger state, page range
calculation, and accessible labels. Use `count` for the total number of data items, not the number
of pages. Pair it with `pageSize` or `defaultPageSize` so Ark can calculate `totalPages`.

`Pagination.Item` expects the page object returned by `pagination.pages`. Pass the object through
with `{...page}` so Ark can set the correct selected state, labels, and event handlers.

### When to go custom

Use `Pagination.Context` for inline reads, `usePaginationContext()` inside custom child parts, and
`usePagination()` plus `Pagination.RootProvider` when state must be created outside the rendered
pagination tree. Set `type="link"` and provide `getPageUrl(details)` when pagination should render
navigational links.

### Default props

| Prop              | Default      | Notes                                                       |
| ----------------- | ------------ | ----------------------------------------------------------- |
| `aria-label`      | `Pagination` | Moduix root default for the navigation landmark.            |
| `defaultPage`     | `1`          | Ark initial active page when uncontrolled.                  |
| `defaultPageSize` | `10`         | Ark initial items per page when uncontrolled.               |
| `siblingCount`    | `1`          | Ark pages shown beside the active page.                     |
| `boundaryCount`   | `1`          | Ark pages shown at the beginning and end of the page range. |
| `type`            | `button`     | Use `link` with `getPageUrl(details)` for anchors.          |

### Behavioral notes

Use `defaultPage` and `defaultPageSize` for uncontrolled state. Use `page`, `pageSize`,
`onPageChange(details)`, and `onPageSizeChange(details)` for controlled state. Callback detail
objects are the Ark objects; read `details.page` or `details.pageSize` without remapping. Ark
preserves keyboard behavior, selected item semantics, disabled trigger state, `ids`, `asChild`,
translations, and link URL generation.

## Examples

### Controlled

Control the active page with Ark's `page` and `onPageChange(details)` props.

<Preview>
  <PaginationControlledExample />

  <Preview.Code>
    {`
      import { Pagination } from "@moduix/react";
      import { useState } from "react";

      export function ControlledPaginationDemo() {
        const [page, setPage] = useState(5);

        return (
          <div className="pagination-stack">
            <Pagination
              className="pagination-demo"
              count={200}
              page={page}
              pageSize={10}
              siblingCount={2}
              onPageChange={(details) => setPage(details.page)}
            >
              <Pagination.PrevTrigger />
              <Pagination.Context>
                {(pagination) =>
                  pagination.pages.map((page, index) =>
                    page.type === "page" ? (
                      <Pagination.Item key={index} {...page}>
                        {page.value}
                      </Pagination.Item>
                    ) : (
                      <Pagination.Ellipsis key={index} index={index} />
                    ),
                  )
                }
              </Pagination.Context>
              <Pagination.NextTrigger />
            </Pagination>
            <p className="pagination-muted">Current page: {page}</p>
          </div>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{paginationExampleCss}</Preview.CSS>
  <Preview.Data>{`const initialPage = 5;`}</Preview.Data>
</Preview>

### Context

Use `Pagination.Context` when custom controls need access to page helpers such as
`goToFirstPage()` or `goToLastPage()`.

<Preview>
  <PaginationContextExample />

  <Preview.Code>
    {`
      import { Pagination } from "@moduix/react";

      export function PaginationContextDemo() {
        return (
          <Pagination count={200} pageSize={10}>
            <Pagination.Context>
              {(pagination) => (
                <div className="pagination-row">
                  <button type="button" onClick={() => pagination.goToFirstPage()}>
                    First
                  </button>
                  <button type="button" onClick={() => pagination.goToPrevPage()}>
                    Prev
                  </button>
                  <span className="pagination-muted">
                    Page {pagination.page} of {pagination.totalPages}
                  </span>
                  <button type="button" onClick={() => pagination.goToNextPage()}>
                    Next
                  </button>
                  <button type="button" onClick={() => pagination.goToLastPage()}>
                    Last
                  </button>
                </div>
              )}
            </Pagination.Context>
          </Pagination>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{paginationExampleCss}</Preview.CSS>
  <Preview.Data>{paginationNoData}</Preview.Data>
</Preview>

### Page Size Control

Call `pagination.setPageSize(nextSize)` from context to let a separate control update page size.

<Preview>
  <PaginationPageSizeControlExample />

  <Preview.Code>
    {`
      import { createListCollection } from "@ark-ui/react/collection";
      import { Pagination, Select } from "@moduix/react";

      const pageSizes = createListCollection({
        items: [
          { label: "5", value: "5" },
          { label: "10", value: "10" },
          { label: "20", value: "20" },
          { label: "50", value: "50" },
        ],
      });

      export function PaginationPageSizeControlDemo() {
        return (
          <Pagination count={200} defaultPageSize={10}>
            <Pagination.Context>
              {(pagination) => (
                <div className="pagination-stack">
                  <div className="pagination-row">
                    <Select
                      className="pagination-page-size-select"
                      collection={pageSizes}
                      value={[String(pagination.pageSize)]}
                      positioning={{ sameWidth: true }}
                      onValueChange={(details) => {
                        const nextValue = details.value[0];
                        if (nextValue) pagination.setPageSize(Number(nextValue));
                      }}
                    >
                      <Select.Label>Items per page</Select.Label>
                      <Select.Control>
                        <Select.Trigger>
                          <Select.ValueText placeholder="Page size" />
                        </Select.Trigger>
                        <Select.Indicators>
                          <Select.Indicator />
                        </Select.Indicators>
                      </Select.Control>
                      <Select.Context>
                        {(select) =>
                          select.open ? (
                            <Select.Positioner>
                              <Select.Content>
                                {pageSizes.items.map((item) => (
                                  <Select.Item key={item.value} item={item}>
                                    <Select.ItemText>{item.label}</Select.ItemText>
                                    <Select.ItemIndicator />
                                  </Select.Item>
                                ))}
                              </Select.Content>
                            </Select.Positioner>
                          ) : null
                        }
                      </Select.Context>
                      <Select.HiddenSelect />
                    </Select>
                  </div>
                  <div className="pagination-row">
                    <Pagination.PrevTrigger />
                    {pagination.pages.map((page, index) =>
                      page.type === "page" ? (
                        <Pagination.Item key={index} {...page}>
                          {page.value}
                        </Pagination.Item>
                      ) : (
                        <Pagination.Ellipsis key={index} index={index} />
                      ),
                    )}
                    <Pagination.NextTrigger />
                  </div>
                  <p className="pagination-muted">
                    Page {pagination.page} of {pagination.totalPages}
                  </p>
                </div>
              )}
            </Pagination.Context>
          </Pagination>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{paginationExampleCss}</Preview.CSS>
  <Preview.Data>
    {`

        items: [
          { label: "5", value: "5" },

### Styling hooks

- `data-scope="pagination"` and `data-part="item"` on `Pagination.Item`
- `data-scope="pagination"` and `data-part="prev-trigger"` on `Pagination.PrevTrigger`
- `data-scope="pagination"` and `data-part="next-trigger"` on `Pagination.NextTrigger`
- `data-scope="pagination"` and edge trigger parts on first and last triggers
- `data-selected` on the active page item
- `data-disabled` on unavailable triggers
- `data-slot="pagination-root"`, `pagination-root-provider`, `pagination-item`,
  `pagination-ellipsis`, and trigger-specific slots for Moduix overrides
