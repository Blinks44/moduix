## Default Path

Use `Pagination` with `usePagination` when the page window should be calculated for you.

      import {
        Pagination,
        PaginationContent,
        PaginationEllipsis,

        PaginationNext,
        PaginationPrevious,
        usePagination,
      } from "moduix";

      export function PaginationDemo() {

        return (
          <Pagination>
            <PaginationContent>
              <PaginationItem>
                <PaginationPrevious

                      setPage(pagination.previousPage);

as styles, icons, and small utilities.

When `href` and `render` are omitted, `PaginationLink`, `PaginationPrevious`, and
`PaginationNext` render native `button` elements automatically. Use that path for client-side page
state, tables, and filtered lists.

## Anatomy

```text
Pagination
└─ PaginationContent
   ├─ PaginationItem
   │  └─ PaginationPrevious
   ├─ PaginationItem
   │  └─ PaginationLink
   ├─ PaginationItem
   │  └─ PaginationEllipsis
   └─ PaginationItem
      └─ PaginationNext
```

| Part                 | Role                                                               |
| -------------------- | ------------------------------------------------------------------ |
| `Pagination`         | Navigation landmark.                                               |
| `PaginationContent`  | Base UI toolbar root for keyboard navigation and item arrangement. |
| `PaginationItem`     | Lightweight wrapper for one pagination element.                    |
| `PaginationLink`     | Page control with active styling via `isActive`.                   |
| `PaginationPrevious` | Previous-page shortcut with a default icon-only control.           |
| `PaginationNext`     | Next-page shortcut with a default icon-only control.               |
| `PaginationEllipsis` | Non-interactive collapsed-range marker.                            |

## Composition

`Pagination` stays thin on purpose:

- `usePagination` handles range generation only
- URL mapping stays in consumer code
- routing stays in consumer code
- visible layout stays explicit in JSX

Use these paths:

| Pattern                | Best fit                                              |
| ---------------------- | ----------------------------------------------------- |
| default button path    | Local state, client-side tables, filtered collections |
| `href="/items?page=4"` | Plain links and SSR-first pages                       |
| `render={<Link />}`    | Router-specific link components                       |

`PaginationPrevious` and `PaginationNext` are icon-only by default. Pass `children` when the UI
needs text labels or a custom layout.

### Router Links

Use `href` for plain anchors and `render={<Link />}` when the app standardizes on a router link
component.

```tsx
import {
  Pagination,
  PaginationContent,
  PaginationEllipsis,
  PaginationItem,
  PaginationLink,
  PaginationNext,
  PaginationPrevious,
  usePagination,
} from 'moduix';
import Link from 'next/link';
import { usePathname, useSearchParams } from 'next/navigation';

export function PaginationLinks({ pageCount }: { pageCount: number }) {
  const pathname = usePathname();
  const searchParams = useSearchParams();
  const page = Number(searchParams.get('page') ?? '1');
  const pagination = usePagination({ count: pageCount, page });

  const createHref = (nextPage: number) => {
    const params = new URLSearchParams(searchParams);
    params.set('page', String(nextPage));
    return `${pathname}?${params.toString()}`;
  };

  return (
    <Pagination>
      <PaginationContent>
        <PaginationItem>
          <PaginationPrevious
            render={<Link href={createHref(pagination.previousPage)} />}
            aria-disabled={!pagination.canPreviousPage || undefined}
          />
        </PaginationItem>
        {pagination.items.map((item, index) => (
          <PaginationItem key={String(item) + '-' + index}>
            {typeof item !== 'number' ? (
              <PaginationEllipsis />
            ) : (
              <PaginationLink
                render={<Link href={createHref(item)} />}
                isActive={item === pagination.page}
              >
                {item}
              </PaginationLink>
            )}
          </PaginationItem>
        ))}
        <PaginationItem>
          <PaginationNext
            render={<Link href={createHref(pagination.nextPage)} />}
            aria-disabled={!pagination.canNextPage || undefined}
          />
        </PaginationItem>
      </PaginationContent>
    </Pagination>
  );
}
```

`createHref` is just a small consumer helper that turns a page number into the next URL. In this
example it:

- reads the current query string
- updates only the `page` param
- preserves the rest of the URL state
- returns the final `href` string for the router link

Keep that helper in app code, close to the router you actually use.
