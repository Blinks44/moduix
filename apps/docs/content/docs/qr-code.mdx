## Basic

Use `QrCode` as the Ark root, then render `Frame` and `Pattern` explicitly. The root owns the
encoded value and `Frame` renders the generated SVG.

<Preview>
  <QrCodeExample />

  <Preview.Code>
    {`
      import { QrCode } from "@moduix/react";

      export function QrCodeDemo() {
        return (
          <QrCode defaultValue="https://moduix.dev/docs/qr-code" className="qr-code-root">
            <QrCode.Frame>
              <QrCode.Pattern />
            </QrCode.Frame>
          </QrCode>
        );
      }
    `}

## Anatomy

```text
QrCode / QrCode.Root
├─ QrCode.Frame
│  └─ QrCode.Pattern
├─ QrCode.Overlay
└─ QrCode.DownloadTrigger

QrCode.RootProvider
└─ same part tree connected to useQrCode()
```

| Part                     | Role                                                          |
| ------------------------ | ------------------------------------------------------------- |
| `QrCode` / `QrCode.Root` | Root value state, generation options, ids, and CSS variables. |
| `QrCode.RootProvider`    | Renders a QR code tree from state returned by `useQrCode`.    |
| `QrCode.Frame`           | SVG frame generated by Ark.                                   |
| `QrCode.Pattern`         | SVG path for QR modules.                                      |
| `QrCode.Overlay`         | Centered custom content over the QR code.                     |
| `QrCode.DownloadTrigger` | Button that downloads the current QR frame as an image.       |
| `QrCode.Context`         | Render-prop access to the current Ark QR code API.            |

## Composition

### Recommended composition

Keep the Ark structure visible: `QrCode.Root`, `QrCode.Frame`, and `QrCode.Pattern`. Add
`QrCode.Overlay` only when the QR code has enough error correction for the covered area.

### When to go custom

Use `encoding` for QR generation options, especially `encoding={{ ecc: 'H' }}` when adding an
overlay. Use `QrCode.DownloadTrigger` when users need the generated image as a file. Use
`useQrCode()` with `QrCode.RootProvider` when controls outside the tree need access to the current
QR code API. Use `QrCode.Context` inside the tree when a child needs the current `value`,
`setValue`, or `getDataUrl(type, quality?)`.

Use `asChild` on Ark parts when replacing the host element. Keep the replacement child semantic and
single. Use `ids` when a custom tree needs stable root or frame ids.

### Default props

| Prop           | Default | Notes                                                                      |
| -------------- | ------- | -------------------------------------------------------------------------- |
| `defaultValue` | -       | Initial uncontrolled encoded value.                                        |
| `value`        | -       | Controlled encoded value.                                                  |
| `encoding`     | -       | QR generation options such as `ecc` and `boostEcc`.                        |
| `pixelSize`    | -       | Pixel size passed to the Ark/Zag generator.                                |
| `id`           | -       | Stable machine id forwarded to Ark.                                        |
| `ids`          | -       | Stable ids for `root` and `frame`.                                         |
| `fileName`     | -       | Required on `DownloadTrigger`.                                             |
| `mimeType`     | -       | Required on `DownloadTrigger`, for example `image/png` or `image/svg+xml`. |
| `quality`      | -       | Optional download image quality.                                           |

### Behavioral notes

Use `value` with `onValueChange(details)` when the encoded URL or text should follow surrounding
state. The callback signature is Ark's original detail object.

Use `QrCode.Context` or `useQrCodeContext()` to read the current value, call `setValue`, or create a
data URL with `getDataUrl(type, quality?)`.

There is no QR-specific arrow-key navigation, roving focus, or hidden input. `DownloadTrigger` is a
native button and keeps button keyboard activation and focus behavior.

The SVG is visual output. If the encoded value needs to be available to assistive technology, show
it in adjacent text or label the surrounding UI with meaningful copy.

## Examples

### Controlled

Control `value` when the QR code should stay in sync with external selection or surrounding UI.

<Preview>
  <ControlledQrCodeExample />

  <Preview.Code>
    {`
      import { Button, QrCode } from "@moduix/react";
      import { useState } from "react";

      export function ControlledQrCodeDemo() {
        const [value, setValue] = useState("https://ark-ui.com");

        return (
          <div className="qr-code-stack">
            <QrCode value={value} onValueChange={(details) => setValue(details.value)}>
              <QrCode.Frame>
                <QrCode.Pattern />
              </QrCode.Frame>
            </QrCode>
            <div className="qr-code-actions">
              {destinations.map((destination) => (
                <Button
                  key={destination}
                  size="sm"
                  variant="outline"
                  onClick={() => setValue(destination)}
                >
                  {new URL(destination).hostname}
                </Button>
              ))}
            </div>
          </div>
        );
      }
    `}

  </Preview.Code>

### Error Correction

Set `encoding.ecc` to choose the QR error correction level. Higher levels are useful when the QR
code has an overlay or must tolerate more damage.

<Preview>
  <ErrorCorrectionQrCodeExample />

  <Preview.Code>
    {`
      import { Button, QrCode } from "@moduix/react";
      import { useState } from "react";

      type ErrorLevel = "L" | "M" | "Q" | "H";

      export function ErrorCorrectionQrCodeDemo() {
        const [errorLevel, setErrorLevel] = useState("L" as ErrorLevel);

        return (
          <div className="qr-code-stack">
            <QrCode
              defaultValue="https://moduix.dev/docs/qr-code"
              encoding={{ ecc: errorLevel }}
            >
              <QrCode.Frame>
                <QrCode.Pattern />
              </QrCode.Frame>
            </QrCode>
            <div className="qr-code-actions">
              {errorLevels.map((level) => (
                <Button
                  key={level}
                  size="sm"
                  variant={level === errorLevel ? "default" : "outline"}
                  onClick={() => setErrorLevel(level)}
                >
                  {level}
                </Button>
              ))}
            </div>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{qrCodeExampleCss}</Preview.CSS>
  <Preview.Data>{qrCodeErrorCorrectionData}</Preview.Data>
</Preview>

### Context

Use `QrCode.Context` when a child inside the QR code tree needs the current API, such as generating
a PNG data URL without rendering a separate download trigger.

<Preview>
  <ContextQrCodeExample />

  <Preview.Code>
    {`
      import { Button, QrCode } from "@moduix/react";
      import { useState } from "react";

      export function ContextQrCodeDemo() {
        const [status, setStatus] = useState("Ready to export");

        return (
          <div className="qr-code-stack">
            <QrCode defaultValue="https://moduix.dev/docs/qr-code">
              <QrCode.Frame>
                <QrCode.Pattern />
              </QrCode.Frame>
              <QrCode.Context>
                {(qrCode) => (
                  <Button
                    size="sm"
                    variant="outline"
                    onClick={async () => {
                      const dataUrl = await qrCode.getDataUrl("image/png");
                      setStatus(\`\${Math.round(dataUrl.length / 1000)} KB PNG data URL ready\`);
                    }}
                  >
                    Create PNG data URL
                  </Button>
                )}
              </QrCode.Context>
            </QrCode>
            <output className="qr-code-status">{status}</output>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{qrCodeExampleCss}</Preview.CSS>
  <Preview.Data>{qrCodeContextData}</Preview.Data>
</Preview>

## Styling

Override QR size, fill, overlay, and download trigger styling through CSS variables and part
classes. Ark also sets runtime variables on the root for generated frame dimensions.

### Styling hooks

- `QrCode.Root` -> `data-slot="qr-code-root"`
- `QrCode.RootProvider` -> `data-slot="qr-code-root-provider"`
- `QrCode.Frame` -> `data-slot="qr-code-frame"`
- `QrCode.Pattern` -> `data-slot="qr-code-pattern"`
- `QrCode.Overlay` -> `data-slot="qr-code-overlay"`
- `QrCode.DownloadTrigger` -> `data-slot="qr-code-download-trigger"`
- Ark hooks: `data-scope="qr-code"`, `data-part="root|frame|pattern|overlay|download-trigger"`,
  and runtime CSS variables `--qrcode-pixel-size`, `--qrcode-width`, and `--qrcode-height`.
