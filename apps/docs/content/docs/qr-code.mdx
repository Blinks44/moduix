## Anatomy

```text
QrCode / QrCode.Root
├─ QrCode.Frame
│  └─ QrCode.Pattern
├─ QrCode.Overlay
└─ QrCode.DownloadTrigger

QrCode.RootProvider
└─ same part tree connected to useQrCode() from @ark-ui/react/qr-code
```

| Part                     | Role                                                             |
| ------------------------ | ---------------------------------------------------------------- |
| `QrCode` / `QrCode.Root` | Root value state, generation options, ids, and CSS variables.    |
| `QrCode.RootProvider`    | Renders a QR code tree from state returned by Ark `useQrCode()`. |
| `QrCode.Frame`           | SVG frame generated by Ark.                                      |
| `QrCode.Pattern`         | SVG path for QR modules.                                         |
| `QrCode.Overlay`         | Centered custom content over the QR code.                        |
| `QrCode.DownloadTrigger` | Button that downloads the current QR frame as an image.          |

## Composition

### Recommended composition

Keep the Ark structure visible: `QrCode.Root`, `QrCode.Frame`, and `QrCode.Pattern`. Add
`QrCode.Overlay` only when the QR code has enough error correction for the covered area.

### When to go custom

Use `encoding` for QR generation options, especially `encoding={{ ecc: 'H' }}` when adding an
overlay. Use `QrCode.DownloadTrigger` when users need the generated image as a file. Use
`useQrCode()` from `@ark-ui/react/qr-code` with `QrCode.RootProvider` when controls outside the
tree need access to the current QR code API.

Use `asChild` on Ark parts when replacing the host element. Keep the replacement child semantic and
single. Use `ids` when a custom tree needs stable root or frame ids.

### Default props

| Prop           | Default | Notes                                                                      |
| -------------- | ------- | -------------------------------------------------------------------------- |
| `defaultValue` | -       | Initial uncontrolled encoded value.                                        |
| `value`        | -       | Controlled encoded value.                                                  |
| `encoding`     | -       | QR generation options such as `ecc` and `boostEcc`.                        |
| `pixelSize`    | -       | Pixel size passed to the Ark/Zag generator.                                |
| `id`           | -       | Stable machine id forwarded to Ark.                                        |
| `ids`          | -       | Stable ids for `root` and `frame`.                                         |
| `fileName`     | -       | Required on `DownloadTrigger`.                                             |
| `mimeType`     | -       | Required on `DownloadTrigger`, for example `image/png` or `image/svg+xml`. |
| `quality`      | -       | Optional download image quality.                                           |

### Behavioral notes

Use `value` with `onValueChange(details)` when the encoded URL or text should follow surrounding
state. The callback signature is Ark's original detail object.

If you need Ark context APIs such as reading the current value, updating it, or creating a data
URL with `getDataUrl(type, quality?)`, import them directly from `@ark-ui/react/qr-code`.

There is no QR-specific arrow-key navigation, roving focus, or hidden input. `DownloadTrigger` is a
native button and keeps button keyboard activation and focus behavior.

The SVG is visual output. If the encoded value needs to be available to assistive technology, show
it in adjacent text or label the surrounding UI with meaningful copy.

## Styling

Override QR size, fill, overlay, and download trigger styling through CSS variables and part
classes. Ark also sets runtime variables on the root for generated frame dimensions.

### Styling hooks

- `QrCode.Root` -> `data-slot="qr-code-root"`
- `QrCode.RootProvider` -> `data-slot="qr-code-root-provider"`
- `QrCode.Frame` -> `data-slot="qr-code-frame"`
- `QrCode.Pattern` -> `data-slot="qr-code-pattern"`
- `QrCode.Overlay` -> `data-slot="qr-code-overlay"`
- `QrCode.DownloadTrigger` -> `data-slot="qr-code-download-trigger"`
- Ark hooks: `data-scope="qr-code"`, `data-part="root|frame|pattern|overlay|download-trigger"`,
  and runtime CSS variables `--qrcode-pixel-size`, `--qrcode-width`, and `--qrcode-height`.
