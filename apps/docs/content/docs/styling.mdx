# Styling

moduix ships component styles and a shared CSS foundation, but it does not prescribe an authoring
tool. Use CSS Modules, Panda CSS, Tailwind CSS, or another CSS strategy already used by your
application. The component API stays the same.

Import the foundation stylesheet once in your application entry point:

```tsx
import '@moduix/react/style.css';
```

Panda CSS and Tailwind CSS are not dependencies of moduix or this documentation site. The examples
below style the same `ProjectChecklist` in each tool, so only the authoring syntax changes. They
are consumer snippets for applications where the respective tool is already configured.

## Choose the smallest useful override

| Goal                                             | Preferred surface                                           |
| ------------------------------------------------ | ----------------------------------------------------------- |
| Adopt a complete visual direction                | A curated [Theme preset](/docs/themes)                      |
| Change a system-wide color, size, radius, or gap | A primitive or semantic token from [Tokens](/docs/tokens)   |
| Keep a family of controls or popup rows aligned  | A shared family token such as `--popup-item-min-height`     |
| Make one component intentionally different       | A documented `--<component>-*` alias                        |
| Style one product composition                    | `className` on the rendered part, preferably with local CSS |

This order keeps themes compact and component styling predictable. Prefer a high-level token when
the decision should propagate; move closer to the rendered part only as the scope becomes local.

## Style the parts you render

Start with `className` on the part that owns the visual change. This keeps the component anatomy
visible and the styles local to the product component. Use the documented Ark state attributes on
that same part for stateful styling, and use public CSS variables when a component exposes the value
you need to change.

### When a wrapper hides the part

Use a stable `data-slot` only when a product wrapper owns the composition but does not expose the
part to its callers. It can still tune an internal trigger without depending on DOM order or generated
class names:

```css
.root {
  [data-slot='accordion-item-trigger'] {
    padding-inline: var(--spacing-4);

    &[data-state='open'] {
      color: var(--color-primary);
    }
  }
}
```

This is an escape hatch, not a replacement for `className`. For automatically portalled parts such
as `Dialog.Content` or `Tooltip.Content`, put `className` on the portalled part itself.

## Panda CSS

In an application that already has Panda configured, generate classes for the same
`ProjectChecklist` parts. The token names below belong to your Panda theme, not to moduix.

```tsx
import { Accordion } from '@moduix/react';
import { css } from '../styled-system/css';

const root = css({
  maxW: '2xl',
  p: '3',
  borderWidth: '1px',
  borderColor: 'blue.200',
  borderRadius: 'xl',
  bg: 'blue.50',
});
const accordion = css({ display: 'flex', flexDirection: 'column', gap: '3' });
const item = css({
  overflow: 'hidden',
  borderWidth: '1px',
  borderColor: 'blue.200',
  borderRadius: 'lg',
  bg: 'white',
  transition: 'colors',
  '&[data-state=open]': { borderColor: 'blue.500', boxShadow: 'lg' },
});
const trigger = css({
  display: 'grid',
  gridTemplateColumns: 'minmax(0, 1fr) auto',
  gap: '3',
  alignItems: 'center',
  w: 'full',
  px: '4',
  py: '4',
  bg: 'blue.50',
  color: 'gray.950',
  fontWeight: 'semibold',
  textAlign: 'left',
  transition: 'colors',
  _hover: { bg: 'blue.100' },
  _focusVisible: { outlineWidth: '2px', outlineColor: 'blue.500', outlineOffset: '-1px' },
  '&[data-state=open]': { bg: 'blue.100', color: 'blue.700' },
});
const triggerCopy = css({ display: 'flex', gap: '3', alignItems: 'center', minW: '0' });
const step = css({
  display: 'grid',
  flexShrink: '0',
  placeItems: 'center',
  w: '7',
  h: '7',
  borderWidth: '1px',
  borderColor: 'blue.300',
  borderRadius: 'xl',
  bg: 'blue.100',
  color: 'blue.700',
  fontSize: 'xs',
  fontWeight: 'semibold',
});
const title = css({ minW: '0' });
const indicator = css({ flexShrink: '0', color: 'blue.700' });
const content = css({
  borderTopWidth: '1px',
  borderColor: 'blue.200',
  bg: 'blue.50',
  color: 'gray.950',
});
const body = css({ px: '4', py: '4', color: 'gray.700', lineHeight: '6' });

export function ProjectChecklist() {
  return (
    <section className={root}>
      <Accordion className={accordion} collapsible defaultValue={['outcome']}>
        <Accordion.Item className={item} value="outcome">
          <Accordion.ItemTrigger className={trigger}>
            <span className={triggerCopy}>
              <span className={step}>01</span>
              <span className={title}>Define the project outcome</span>
            </span>
            <Accordion.ItemIndicator className={indicator} />
          </Accordion.ItemTrigger>
          <Accordion.ItemContent className={content}>
            <Accordion.ItemBody className={body}>
              <p>
                Write one sentence that makes the next decision obvious to everyone on the project.
              </p>
            </Accordion.ItemBody>
          </Accordion.ItemContent>
        </Accordion.Item>
      </Accordion>
    </section>
  );
}
```

## Tailwind CSS

In an application that already uses Tailwind, apply utilities and state variants to the same
`ProjectChecklist` parts. No moduix-specific Tailwind plugin or configuration is required.

```tsx
import { Accordion } from '@moduix/react';

export function ProjectChecklist() {
  return (
    <section className="max-w-2xl rounded-xl border border-blue-200 bg-blue-50 p-3">
      <Accordion className="flex flex-col gap-3" collapsible defaultValue={['outcome']}>
        <Accordion.Item
          className="overflow-hidden rounded-lg border border-blue-200 bg-white transition-[border-color,box-shadow] data-[state=open]:border-blue-500 data-[state=open]:shadow-lg"
          value="outcome"
        >
          <Accordion.ItemTrigger className="grid w-full grid-cols-[minmax(0,1fr)_auto] items-center gap-3 bg-blue-50 p-4 text-left font-semibold text-slate-950 transition-colors hover:bg-blue-100 focus-visible:outline-2 focus-visible:outline-blue-500 focus-visible:outline-offset-[-1px] data-[state=open]:bg-blue-100 data-[state=open]:text-blue-700">
```
