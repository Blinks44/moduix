## Choosing the right component

Use `NavigationMenu` for top-level site or application navigation that may reveal richer panels,
grouped links, or a second navigation level.

- Choose **`NavigationMenu`** when navigation items open shared popup content.
- Choose **`Menubar`** when the UI behaves like an application command bar with nested menus.
- Choose **`Menu`** when the content is an action list rather than page navigation.

## Basic

Start with `NavigationMenu`, `NavigationMenuList`, `NavigationMenuItem`,
`NavigationMenuTrigger`, and `NavigationMenuContent`. This is the recommended path for standard

## Anatomy

The normal consumer path stays short:

```text
NavigationMenu
└─ NavigationMenuList
   ├─ NavigationMenuItem
   │  ├─ NavigationMenuTrigger
   │  └─ NavigationMenuContent
   └─ NavigationMenuItem
      └─ NavigationMenuLink
```

When `showPopup` stays enabled, the root also renders the shared popup structure for you:

```text
NavigationMenuPortal
└─ NavigationMenuPositioner
   └─ NavigationMenuPopup
      ├─ NavigationMenuArrow (optional)
      └─ NavigationMenuViewport
```

| Part                     | Role                                                                       |
| ------------------------ | -------------------------------------------------------------------------- |
| `NavigationMenu`         | Root state, keyboard interaction, and optional built-in popup composition. |
| `NavigationMenuList`     | Top-level list container.                                                  |
| `NavigationMenuTrigger`  | Interactive trigger for popup content.                                     |
| `NavigationMenuContent`  | Panel content rendered through the shared viewport.                        |
| `NavigationMenuLink`     | Direct navigation item styled like a trigger.                              |
| `NavigationMenuViewport` | Shared viewport that renders the active content.                           |

## Composition

### When to stay high-level

Stay with `NavigationMenu`, `NavigationMenuList`, `NavigationMenuItem`,
`NavigationMenuTrigger`, `NavigationMenuContent`, and `NavigationMenuLink` for most site headers
and application navigation.

`NavigationMenu` keeps Base UI root behavior intact. Use it for `value`, `defaultValue`,
`onValueChange`, `orientation`, `delay`, `closeDelay`, `actionsRef`, `render`, `className`, and
`style`.

The moduix root also keeps the shared popup path short:

- `showPopup` controls whether the root renders the built-in portal, positioner, popup, and
  viewport.
- `showArrow` enables the built-in popup arrow.
- `side`, `sideOffset`, `align`, `alignOffset`, `arrowPadding`, `anchor`,
  `collisionAvoidance`, `collisionBoundary`, `collisionPadding`, `sticky`, `positionMethod`, and
  `disableAnchorTracking` are forwarded to the built-in `NavigationMenuPositioner` when
  `showPopup` is enabled.

### When to go custom

Switch to `NavigationMenuPortal`, `NavigationMenuBackdrop`, `NavigationMenuPositioner`,
`NavigationMenuPopup`, `NavigationMenuArrow`, and `NavigationMenuViewport` when you need to own
popup structure, add a backdrop, change placement rules, or render the viewport inline.

If you set `showPopup={false}`, render `NavigationMenuViewport` somewhere in the tree yourself or
the active content will not appear.

### Default props

| Prop                 | Default                                      | Values / Notes                                                        |
| -------------------- | -------------------------------------------- | --------------------------------------------------------------------- |
| `showPopup`          | `true`                                       | Enables the built-in popup composition                                |
| `showArrow`          | `false`                                      | Adds `NavigationMenuArrow` inside the built-in popup                  |
| `sideOffset`         | `10`                                         | Default gap for the built-in positioner                               |
| `collisionAvoidance` | `{ side: "none" }`                           | Keeps the shared popup from flipping sides by default                 |
| `collisionPadding`   | `{ top: 5, bottom: 5, left: 20, right: 20 }` | Default viewport-edge padding for the built-in positioner             |
| `icon` (trigger)     | built-in chevron                             | Pass custom content to replace it, or `null` / `false` to suppress it |

### Behavioral notes

- `NavigationMenuTrigger` keeps rendering the built-in icon even when you use `render`; pass
  `icon={null}` if a custom trigger should not show the chevron.
- `NavigationMenuBackdrop` is visual-only by default. The shipped moduix styles set
  `pointer-events: none`.
- Use another `NavigationMenu` inside content when a second level should open in its own popup.
- Use `showPopup={false}` plus an inline `NavigationMenuViewport` when nested content should stay
  inside the current panel.

popup.

### Inline Nested Viewport

Use `showPopup={false}` on a nested menu and render `NavigationMenuViewport` inline when the second
level should stay inside the current panel instead of opening another popup.

        NavigationMenu,
        NavigationMenuContent,
        NavigationMenuItem,
        NavigationMenuList,
        NavigationMenuTrigger,
        NavigationMenuViewport,
      } from "moduix";

        return (
          <NavigationMenu>
            <NavigationMenuList>
              <NavigationMenuItem>
                <NavigationMenuTrigger>Product</NavigationMenuTrigger>

                          <span>Developers</span>
                          <span>API, SDK and integrations</span>

                    </NavigationMenuList>

                    <NavigationMenuViewport />
                  </NavigationMenu>
                </NavigationMenuContent>
              </NavigationMenuItem>
            </NavigationMenuList>
          </NavigationMenu>

      }

### Custom Composition

When the default wrapper is too opinionated, compose the popup parts directly. This is the escape
hatch for custom layering, backdrops, or manual viewport placement.

  <Preview.Code>
    {`
      import {
        NavigationMenu,
        NavigationMenuArrow,
        NavigationMenuBackdrop,
        NavigationMenuContent,
        NavigationMenuItem,
        NavigationMenuList,
        NavigationMenuPopup,
        NavigationMenuPortal,
        NavigationMenuPositioner,
        NavigationMenuTrigger,
        NavigationMenuViewport,
      } from "moduix";

      export function CustomCompositionNavigationMenuDemo() {
        return (
          <NavigationMenu showPopup={false}>
            <NavigationMenuList>
              <NavigationMenuItem>
                <NavigationMenuTrigger>Resources</NavigationMenuTrigger>
                <NavigationMenuContent>{/* panel content */}</NavigationMenuContent>
              </NavigationMenuItem>
            </NavigationMenuList>

            <NavigationMenuPortal>
              <NavigationMenuBackdrop />
              <NavigationMenuPositioner sideOffset={16}>
                <NavigationMenuPopup>
                  <NavigationMenuArrow />
                  <NavigationMenuViewport />
                </NavigationMenuPopup>
              </NavigationMenuPositioner>
            </NavigationMenuPortal>
          </NavigationMenu>
        );
      }
    `}

### Styling hooks

Use the part that owns the visual concern:

- `NavigationMenuTrigger` and `NavigationMenuLink` for top-level item chrome and focus styling
- `NavigationMenuPopup`, `NavigationMenuContent`, and `NavigationMenuViewport` for popup surface
  and panel layout
- `NavigationMenuBackdrop` and `NavigationMenuArrow` for advanced popup presentation
- `NavigationMenuIcon` when the trigger indicator needs custom sizing, color, or motion

The main public hooks include `data-slot="navigation-menu-root"`,
`data-slot="navigation-menu-trigger"`, `data-slot="navigation-menu-content"`,
`data-slot="navigation-menu-popup"`, and `data-slot="navigation-menu-viewport"`.

Useful state and layout attributes include:

- `[data-orientation="vertical"]` on the root for stacked menus
- `[data-active]`, `[data-popup-open]`, `[data-disabled]`, and `:disabled` on triggers and links
- `[data-side]` and `[data-instant]` on the positioner
- `[data-starting-style]` and `[data-ending-style]` on popup, backdrop, and content during motion
- `[data-activation-direction]` on content for directional enter and exit styling
