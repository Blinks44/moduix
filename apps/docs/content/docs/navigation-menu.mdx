## Overview

`NavigationMenu` keeps the common path short: the root renders the shared popup infrastructure for
you, so the default composition is just list, item, trigger, content, and direct links.

Use the low-level popup parts only when you need to own the structure yourself, such as a visual
backdrop or an inline nested viewport.

## Anatomy

The recommended structure stays close to a composition-first API:

```text
NavigationMenu
└─ NavigationMenuList
   ├─ NavigationMenuItem
   │  ├─ NavigationMenuTrigger
   │  └─ NavigationMenuContent
   └─ NavigationMenuItem
      └─ NavigationMenuLink
```

By default, the root also renders:

```text
NavigationMenuPortal
└─ NavigationMenuPositioner
   └─ NavigationMenuPopup
      ├─ NavigationMenuArrow (optional)
      └─ NavigationMenuViewport
```

## Composition

`NavigationMenu` forwards Base UI root props such as `value`, `defaultValue`, `onValueChange`,
`delay`, `closeDelay`, `orientation`, `actionsRef`, `render`, `className`, and `style`.

The root also adds a narrow popup contract for the built-in viewport composition:

| Prop                                                                                                                                                                         | Default                             | Purpose                                                                                                                        |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `showPopup`                                                                                                                                                                  | `true`                              | Disables the built-in popup composition so you can assemble `Portal`, `Positioner`, `Popup`, `Arrow`, and `Viewport` manually. |
| `showArrow`                                                                                                                                                                  | `false`                             | Turns on the built-in popup arrow.                                                                                             |
| `sideOffset`                                                                                                                                                                 | `10`                                | Default offset for the built-in `Positioner`.                                                                                  |
| `side`, `align`, `alignOffset`, `arrowPadding`, `anchor`, `collisionAvoidance`, `collisionBoundary`, `collisionPadding`, `sticky`, `positionMethod`, `disableAnchorTracking` | Base UI defaults unless noted above | Forwarded to the built-in `Positioner` when `showPopup` stays enabled.                                                         |

`NavigationMenuTrigger` keeps one small DX shortcut:

| Prop     | Default             | Purpose                                                                                                                    |
| -------- | ------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `icon`   | built-in chevron    | Replaces the default trigger icon. Pass `null` or `false` to remove it.                                                    |
| `render` | Base UI passthrough | Lets you own the trigger element. moduix skips the default trigger class, but `icon` still applies unless you suppress it. |

Two important advanced-path details:

1. If you set `showPopup={false}`, render `NavigationMenuViewport` somewhere in the tree yourself or
   the active content will not appear.
2. `NavigationMenuBackdrop` is visual-only by default; the moduix class uses `pointer-events: none`.

### Nested Inline Submenu

When second-level content should stay inside the current panel, disable the built-in popup on the
nested menu and render `NavigationMenuViewport` inline.

<Preview>
  <NestedInlineNavigationMenuExample />

  <Preview.Code>
    {`
      import { useMediaQuery } from "@base-ui/react/unstable-use-media-query";
      import {
        NavigationMenu,
        NavigationMenuContent,
        NavigationMenuItem,
        NavigationMenuList,
        NavigationMenuTrigger,
        NavigationMenuViewport,
      } from "moduix";

      export function InlineNavigationMenuDemo() {
        const isDesktop = useMediaQuery("(min-width: 700px)", { defaultMatches: true });

        return (
          <NavigationMenu>
            <NavigationMenuList>
              <NavigationMenuItem>
                <NavigationMenuTrigger>Product</NavigationMenuTrigger>
                <NavigationMenuContent className={styles.inlineProductContent}>
                  <NavigationMenu
                    className={styles.inlineNestedRoot}
                    orientation={isDesktop ? "vertical" : "horizontal"}
                    defaultValue="developers"
                    showPopup={false}
                  >
                    <NavigationMenuList className={styles.inlineList}>
                      <NavigationMenuItem value="developers">
                        <NavigationMenuTrigger className={styles.inlineTrigger} icon={null}>
                          <span className={styles.title}>Developers</span>
                          <span className={styles.description}>API, SDK and integrations</span>
                        </NavigationMenuTrigger>
                        <NavigationMenuContent className={styles.inlineSubmenuContent}>
                          {/* panel content */}
                        </NavigationMenuContent>
                      </NavigationMenuItem>

                      <NavigationMenuItem value="systems">
                        <NavigationMenuTrigger className={styles.inlineTrigger} icon={null}>
                          <span className={styles.title}>Design Systems</span>
                          <span className={styles.description}>Patterns and governance</span>
                        </NavigationMenuTrigger>
                        <NavigationMenuContent className={styles.inlineSubmenuContent}>
                          {/* panel content */}
                        </NavigationMenuContent>
                      </NavigationMenuItem>
                    </NavigationMenuList>

                    <NavigationMenuViewport className={styles.inlineViewport} />
                  </NavigationMenu>
                </NavigationMenuContent>
              </NavigationMenuItem>

              <NavigationMenuItem>
                <NavigationMenuTrigger>Learn</NavigationMenuTrigger>
                <NavigationMenuContent className={styles.inlineGuidesContent}>
                  {/* guide panel */}
                </NavigationMenuContent>
              </NavigationMenuItem>
            </NavigationMenuList>
          </NavigationMenu>
        );
      }
    `}
