## Choosing the right component

Use `Menu` for trigger-anchored command lists, context menus, lightweight selections, and nested
actions.

- Choose **`Menu`** for overflow actions, row actions, and command lists opened from a button or
  context trigger.
- Choose **`Menubar`** for persistent app-level menus in a horizontal or vertical bar.
- Choose **`NavigationMenu`** when the popup content is site navigation, not command-style actions.

## Basic

Start with the Ark part tree: `Menu`, `MenuTrigger`, `Portal`, `MenuPositioner`, and `MenuContent`.
Items use stable `value` identifiers for selection and typeahead.

          </Menu>

## Anatomy

```text
Menu
├─ MenuTrigger / MenuContextTrigger
├─ Portal
│  └─ MenuPositioner
│     └─ MenuContent
│        ├─ MenuArrow
│        │  └─ MenuArrowTip
│        ├─ MenuItem
│        ├─ MenuCheckboxItem
│        │  ├─ MenuItemIndicator
│        │  └─ MenuItemText
│        ├─ MenuRadioItemGroup
│        │  └─ MenuRadioItem
│        ├─ MenuItemGroup
│        │  └─ MenuItemGroupLabel
│        └─ Menu
│           └─ MenuTriggerItem
└─ Menu.Context / Menu.ItemContext
```

| Part                 | Role                                                        |
| -------------------- | ----------------------------------------------------------- |
| `Menu`               | Ark root for open state, positioning, typeahead, dismissal. |
| `MenuRootProvider`   | Renders a menu from a `useMenu` state instance.             |
| `MenuTrigger`        | Button trigger; use `asChild` for a custom host.            |
| `MenuContextTrigger` | Right-click and long-press trigger for context menus.       |
| `MenuPositioner`     | Floating layer positioner.                                  |
| `MenuContent`        | Popup surface with menu role and item collection.           |
| `MenuItem`           | Command row.                                                |
| `MenuCheckboxItem`   | Boolean option item.                                        |
| `MenuRadioItemGroup` | Exclusive value group for radio items.                      |
| `MenuRadioItem`      | Exclusive option item.                                      |
| `MenuTriggerItem`    | Row that opens a nested `Menu`.                             |
| `MenuItemIndicator`  | Check indicator for checkbox and radio rows.                |

## Composition

### Recommended composition

Keep popup structure explicit. `MenuContent` is the actual Ark content part; it no longer hides
`Portal` or `MenuPositioner`.

```tsx
<Menu positioning={{ placement: 'bottom-start', gutter: 8 }}>
  <MenuTrigger asChild>
    <Button>Actions</Button>
  </MenuTrigger>
  <Portal>
    <MenuPositioner>
      <MenuContent>
        <MenuItem value="edit">Edit</MenuItem>
      </MenuContent>
    </MenuPositioner>
  </Portal>
</Menu>
```

### When to go custom

Use `asChild` when the trigger or item must render as another semantic element. For links, put
`asChild` on `MenuItem` and make the anchor the single child so Ark can apply the correct ARIA,
focus, and keyboard behavior.

Nested menus are regular `Menu` roots inside the parent content. The nested root uses
`MenuTriggerItem`, then its own `Portal`, `MenuPositioner`, and `MenuContent`.

### Behavioral notes

- `open`, `defaultOpen`, `onOpenChange(details)`, `highlightedValue`,
  `onHighlightChange(details)`, `onSelect(details)`, `closeOnSelect`, `ids`, `lazyMount`,
  `unmountOnExit`, and `present` follow Ark directly.
- Use `value`, not a custom `id`, for menu item identity.
- `MenuCheckboxItem.onCheckedChange` receives the next boolean checked state.
- `MenuRadioItemGroup.onValueChange` receives `details.value`.
- `Menu.RootProvider` must be used with a `useMenu` instance instead of rendering a second `Menu`
  root for the same state.
- `MenuTriggerIcon`, `MenuTriggerItemIcon`, `MenuItemShortcut`, and item text helpers are moduix
  leaf sugar only; behavior remains Ark-owned.

### Controlled

Control `open` with `onOpenChange(details)` when another control needs to drive the menu.

<Preview>
  <ControlledMenuExample />
  <Preview.Code>
    {`
      import { Button, Menu, MenuContent, MenuItem, MenuPositioner, MenuTrigger, MenuTriggerIcon, Portal } from "moduix";
      import { useState } from "react";

      export function ControlledMenuDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Menu open={open} onOpenChange={(details) => setOpen(details.open)}>
            <Button onClick={() => setOpen((value) => !value)}>Toggle</Button>
            <MenuTrigger asChild>
              <Button>
                Actions
                <MenuTriggerIcon />
              </Button>
            </MenuTrigger>
            <Portal>
              <MenuPositioner>
                <MenuContent>

                  <MenuItem value="archive">Archive</MenuItem>
