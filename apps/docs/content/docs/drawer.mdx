## Choosing the right component

Use `Drawer` when the surface should stay attached to an edge or should feel draggable.

- Choose **`Drawer`** for bottom sheets, side inspectors, edge-attached panels, and flows that need swipe gestures or snap points.
- Choose **`Dialog`** when the overlay does not need edge attachment, drag behavior, or snap points.
- Choose **`AlertDialog`** when the user must explicitly confirm a destructive or irreversible action.

## Anatomy

`DrawerContent` is the default high-level surface. It already renders the portal, optional
backdrop, viewport, popup, decorative handle, and content wrapper for you.

```text
Drawer
├─ DrawerTrigger (optional)
└─ DrawerContent
   ├─ portal (service slot)
   │  ├─ backdrop (modal only)
   │  └─ viewport (service slot)
   │     └─ popup surface
   │        ├─ handle
   │        └─ content
   │           ├─ DrawerHeader
   │           │  ├─ DrawerTitle
   │           │  ├─ DrawerCloseIcon (optional)
   │           │  └─ DrawerDescription (optional)
   │           ├─ DrawerBody (optional)
   │           └─ DrawerFooter (optional)
   │              └─ DrawerClose
```

| Part                     | Role                                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------------------------------ |
| `Drawer`                 | Root state and gesture model. Use `open`, `defaultOpen`, `onOpenChange`, `modal`, and snap props here. |
| `DrawerTrigger`          | Opens the drawer from user interaction. Use `handle` when the trigger lives outside the tree.          |
| `DrawerContent`          | High-level popup wrapper. It renders the internal portal, viewport, popup, and default handle.         |
| `DrawerPortal`           | Optional escape hatch for portal placement and mount behavior.                                         |
| `DrawerBackdrop`         | Optional styled backdrop part for manual composition.                                                  |
| `DrawerViewport`         | Optional positioning and scroll container for manual composition.                                      |
| `DrawerPopup`            | Optional popup surface for manual composition.                                                         |
| `DrawerHandle`           | Optional drag handle for manual composition.                                                           |
| `DrawerContentInner`     | Optional content wrapper inside the popup for manual composition.                                      |
| `DrawerHeader`           | Groups title-level content at the top of the panel.                                                    |
| `DrawerTitle`            | Accessible drawer title announced by assistive technology.                                             |
| `DrawerDescription`      | Optional supporting text announced with the title.                                                     |
| `DrawerBody`             | Optional content region for forms, lists, and scrollable content.                                      |
| `DrawerFooter`           | Optional actions row for close, submit, and secondary actions.                                         |
| `DrawerClose`            | Closes the drawer from action buttons while preserving built-in behavior.                              |
| `DrawerCloseIcon`        | Optional icon-only close helper for side-panel headers.                                                |
| `DrawerSwipeArea`        | Optional edge-open gesture area.                                                                       |
| `DrawerProvider`         | Optional provider for indent effects and nested gesture coordination.                                  |
| `DrawerIndent`           | Optional styled indent wrapper.                                                                        |
| `DrawerIndentBackground` | Optional background layer for the indent effect.                                                       |

## Composition

### When to stay high-level

Stay with `Drawer`, `DrawerTrigger`, and `DrawerContent` for most flows.

Use `Drawer` for state and interaction: `open`, `defaultOpen`, `onOpenChange`, `modal`,
`swipeDirection`, `snapPoints`, `snapPoint`, `disablePointerDismissal`, and `handle`.

Use `DrawerContent` for the visible surface. It is intentionally thin: it accepts Base UI popup
props, applies the library styles, and renders the default portal/backdrop/viewport/popup stack
for you.

### When to go custom

Switch to `DrawerPortal`, `DrawerBackdrop`, `DrawerViewport`, `DrawerPopup`, `DrawerHandle`, and
`DrawerContentInner` when the default overlay structure is not enough.

That is the right path for custom portal placement, `keepMounted`, explicit backdrop control,
viewport-specific layout, or fully custom popup structure.

Use `createDrawerHandle()` when the trigger lives outside the drawer tree or the drawer needs to
open imperatively.

### Default props

| Prop / Part       | Default          | Values / Notes                                     |
| ----------------- | ---------------- | -------------------------------------------------- |
| `modal`           | `true`           | `true`, `false`, or `"trap-focus"`                 |
| `swipeDirection`  | `"down"`         | `"down"`, `"up"`, `"left"`, or `"right"`           |
| `defaultOpen`     | `false`          | Initial uncontrolled open state                    |
| `DrawerContent`   | -                | Renders backdrop only when `modal={true}`          |
| `snapLayout`      | `false`          | Useful for top and bottom drawers with snap points |
| `variant`         | `"default"`      | `"default"` or `"island"`                          |
| `DrawerCloseIcon` | `"Close drawer"` | Default accessible label                           |

### Behavioral notes

- `modal={false}` keeps the page interactive and skips the backdrop entirely.
- `modal="trap-focus"` keeps keyboard focus inside the drawer while allowing outside pointer interaction.
- `className` on `DrawerContent` styles the popup surface, not `DrawerContentInner`.
- `snapLayout` is intended for `down` and `up` drawers with snap points. It is not useful for left and right side panels.
- `variant="island"` removes the hidden bleed tail and adds outer viewport padding.
- `DrawerHandle` is decorative and is hidden automatically for left and right drawers.

### Controlled Persistent State

Keep the drawer open by controlling `open` and ignoring close attempts in your own state logic.

<Preview>
  <ControlledPersistentDrawerExample />

  <Preview.Code>
    {`
      import {
        Button,
        ChevronDownIcon,
        ChevronUpIcon,
        Drawer,
        DrawerBody,
        DrawerContent,
        DrawerDescription,
        DrawerHeader,
        DrawerTitle,
        ScrollArea,
      } from "moduix";
      import { useState } from "react";

      export function ControlledPersistentDrawerDemo() {
        const [open, setOpen] = useState(false);
        const [snapPoint, setSnapPoint] = useState<number | string | null>(snapPoints[0]);
        const expanded = snapPoint === snapPoints[1];

        return (
          <>
            <Button type="button" onClick={() => setOpen(true)}>
              Open persistent drawer
            </Button>

            <Drawer
              open={open}
              onOpenChange={(nextOpen) => {
                if (nextOpen) setOpen(true);
              }}
              modal={false}
              disablePointerDismissal
              snapPoints={[...snapPoints]}
              snapPoint={snapPoint}
              onSnapPointChange={(nextSnapPoint) => {
                if (nextSnapPoint !== null) setSnapPoint(nextSnapPoint);
              }}
            >
              <DrawerContent snapLayout>
                <DrawerHeader className="drawer-header-with-action">
                  <div>
                    <DrawerTitle>Controlled persistent drawer</DrawerTitle>
                    <DrawerDescription>
                      Persistence is controlled from application state; the icon button toggles
                      between collapsed and expanded snap points.
                    </DrawerDescription>
                  </div>
                  <button
                    type="button"
                    className="drawer-snap-toggle"
                    onClick={() => setSnapPoint(expanded ? snapPoints[0] : snapPoints[1])}
                    aria-label={expanded ? "Collapse drawer" : "Expand drawer"}
                  >
                    {expanded ? <ChevronDownIcon /> : <ChevronUpIcon />}
                  </button>
                </DrawerHeader>
                <DrawerBody>
                  <ScrollArea>{/* content */}</ScrollArea>
                </DrawerBody>
              </DrawerContent>
            </Drawer>
          </>
        );
      }
    `}

  </Preview.Code>

## Styling

`DrawerCloseIcon` for most visual overrides. Move to `DrawerPortal`, `DrawerBackdrop`,
`DrawerViewport`, `DrawerPopup`, and `DrawerContentInner` only when the overlay structure itself
needs to change.

### Styling hooks
