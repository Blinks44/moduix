## Choosing the right component

Use `Drawer` for an edge-attached panel that benefits from swipe gestures, snap points, or a
mobile bottom-sheet interaction. Use `Dialog` when the surface should remain centered and does not
need drag behavior.

## Basic

Compose the Ark structural parts explicitly. `Drawer.Content` is the real interactive surface;
`Drawer.Positioner` aligns it and `Drawer.Backdrop` provides the modal layer.

## Anatomy

```text
Drawer.Root
├─ Drawer.Trigger
├─ Drawer.SwipeArea (optional)
├─ Drawer.Backdrop
└─ Drawer.Positioner
   └─ Drawer.Content
      ├─ Drawer.Grabber
      │  └─ Drawer.GrabberIndicator
      ├─ Drawer.Title
      ├─ Drawer.Description
      └─ Drawer.CloseTrigger
```

| Part                      | Role                                                |
| ------------------------- | --------------------------------------------------- |
| `Drawer` / `Drawer.Root`  | Owns drawer state, snap points, and swipe behavior. |
| `Drawer.Trigger`          | Opens the drawer.                                   |
| `Drawer.Backdrop`         | Modal backdrop layer.                               |
| `Drawer.Positioner`       | Floating positioner for drawer content.             |
| `Drawer.Content`          | Drawer surface container.                           |
| `Drawer.Grabber`          | Drag handle container for swipe interactions.       |
| `Drawer.GrabberIndicator` | Visual indicator inside the grabber handle.         |
| `Drawer.CloseTrigger`     | Closes the drawer.                                  |
| `Drawer.RootProvider`     | Connects parts to state created with `useDrawer()`. |
| `Drawer.Context`          | Inline render-prop access to drawer state.          |

`Drawer.Header`, `Drawer.Body`, `Drawer.Footer`, and `Drawer.CloseIcon` are optional moduix layout
and visual helpers. They do not replace or hide Ark structural parts.

## Composition

### Recommended composition

Use `Drawer.Root` for state and behavior. Render `Backdrop`, `Positioner`, and `Content` explicitly,
then place `Grabber` and `GrabberIndicator` inside the content when dragging should have a visible
handle.

Use `asChild` to compose `Drawer.Trigger` or `Drawer.CloseTrigger` with `Button`. The child must be
a single semantic focusable element.

Callbacks preserve Ark detail objects:

- `onOpenChange(details)` → `details.open`
- `onSnapPointChange(details)` → `details.snapPoint`
- `onTriggerValueChange(details)` → `details.value` and `details.triggerElement`

### When to go custom

Apply `className` directly to any exported part. Use `Drawer.Context` or `useDrawerContext` for
state-aware child components. Use `useDrawer` with `Drawer.RootProvider` when the state must be
owned outside the rendered drawer tree; do not combine `Root` and `RootProvider` for the same
instance.

Wrap coordinated indent surfaces in `Drawer.Stack`, then render `Drawer.IndentBackground` and
`Drawer.Indent`.

### Default props

| Prop                     | Default  | Notes                                                  |
| ------------------------ | -------- | ------------------------------------------------------ |
| `modal`                  | `true`   | Hides page content and blocks outside interaction.     |
| `swipeDirection`         | `"down"` | Opens from the bottom by default.                      |
| `snapPoints`             | `[1]`    | Uses a full-size drawer unless snap points are set.    |
| `closeOnEscape`          | `true`   | Keeps Ark Escape dismissal enabled.                    |
| `closeOnInteractOutside` | `true`   | Closes when a modal drawer receives outside input.     |
| `preventDragOnScroll`    | `true`   | Avoids starting a drag from scrollable descendants.    |
| `preventScroll`          | `true`   | Prevents background scroll while open.                 |
| `restoreFocus`           | `true`   | Returns focus to the previous active element.          |
| `trapFocus`              | `true`   | Keeps focus inside modal drawer content.               |
| `lazyMount`              | `true`   | Moduix default, different from Ark's `false`.          |
| `unmountOnExit`          | `true`   | Moduix default, different from Ark's `false`.          |
| `Content draggable`      | `true`   | The grabber still drags when content is not draggable. |

### Behavioral notes

- Public direction values are logical: `up`, `down`, `start`, and `end`. Ark exposes resolved
  physical `left` or `right` values through `data-swipe-direction` for styling.
- `Content draggable={false}` disables dragging from content while keeping the grabber draggable.
- `data-no-drag` prevents a drag gesture from starting inside a specific subtree.
- `ids` customizes accessibility IDs for cross-part composition.
- `present`, `lazyMount`, `unmountOnExit`, and `onExitComplete` control render lifecycle.
  Moduix defaults `lazyMount` and `unmountOnExit` to `true` so exit animations can finish without
  rendering hidden drawers on initial page load.
- Ark preserves dialog focus trapping, Escape handling, title/description associations, and focus
  restoration.

### Multiple Triggers

Assign each trigger a `value` to share one drawer across a list. The callback identifies which
trigger is current.

<Preview>
  <MultipleTriggersDrawerExample />

  <Preview.Code>
    {`
      import { Button, Drawer, type DrawerTriggerValueChangeDetails } from "@moduix/react";
      import { useState } from "react";

      export function MultipleTriggersDrawerDemo() {
        const [activeUser, setActiveUser] = useState(null as (typeof users)[number] | null);
        const handleTriggerValueChange = (details: DrawerTriggerValueChangeDetails) => {
          setActiveUser(users.find((user) => user.id === details.value) ?? null);
        };

        return (
          <Drawer swipeDirection="end" onTriggerValueChange={handleTriggerValueChange}>
            <div className="trigger-group">
              {users.map((user) => (
                <Drawer.Trigger key={user.id} value={user.id} asChild>
                  <Button variant="outline">Edit {user.name}</Button>
                </Drawer.Trigger>
              ))}
            </div>
            <Drawer.Backdrop />
            <Drawer.Positioner>
              <Drawer.Content>
                <Drawer.Header>
                  <Drawer.Title>Edit user</Drawer.Title>
                  <Drawer.CloseIcon />
                  <Drawer.Description>{activeUser?.email}</Drawer.Description>
                </Drawer.Header>
                {activeUser ? <Drawer.Body>Selected: {activeUser.name}</Drawer.Body> : null}
              </Drawer.Content>
            </Drawer.Positioner>
          </Drawer>
        );
      }
    `}

  </Preview.Code>
