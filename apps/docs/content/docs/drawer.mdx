## Overview

Use `Drawer` for edge-attached panels that need gesture support, snap points, or the indent effect.
If a panel only needs edge positioning, prefer `Dialog`.

Common fits:

- bottom sheet for mobile actions, details, or secondary flows
- side panel for filters, inspectors, and contextual editing
- persistent edge surface controlled by application state

## Anatomy

`DrawerContent` is the default high-level path. It renders `DrawerPortal`, `DrawerBackdrop`,
`DrawerViewport`, `DrawerPopup`, `DrawerHandle`, and `DrawerContentInner` for you. Those same
parts stay available when you need to take over the DOM structure manually.

```text
Drawer
├─ DrawerTrigger (optional)
└─ DrawerContent
   ├─ portal (service slot)
   │  ├─ backdrop (modal only)
   │  └─ viewport (service slot)
   │     └─ popup surface
   │        ├─ handle
   │        └─ content
   │           ├─ DrawerHeader
   │           │  ├─ DrawerTitle
   │           │  └─ DrawerDescription (optional)
   │           ├─ DrawerBody (optional)
   │           └─ DrawerFooter (optional)
   │              └─ DrawerClose
```

```tsx
<Drawer>
  <DrawerTrigger render={<Button />}>Open drawer</DrawerTrigger>
  <DrawerContent>
    <DrawerHeader>
      <DrawerTitle>Notifications</DrawerTitle>
      <DrawerDescription>You are all caught up. Good job!</DrawerDescription>
    </DrawerHeader>
    <DrawerBody>Bottom drawers are the default.</DrawerBody>
    <DrawerFooter>
      <DrawerClose render={<Button variant="outline" />}>Close</DrawerClose>
    </DrawerFooter>
  </DrawerContent>
</Drawer>
```

| Part                     | Role                                                                                              |
| ------------------------ | ------------------------------------------------------------------------------------------------- |
| `Drawer`                 | Root state and gesture model. Use `open`, `defaultOpen`, `onOpenChange`, `modal`, and snap props. |
| `DrawerTrigger`          | Opens the drawer from user interaction. Use `handle` when the trigger lives outside the tree.     |
| `DrawerContent`          | Default popup container. It renders the internal portal, backdrop, viewport, handle, and content. |
| `DrawerPortal`           | Optional escape hatch for portal placement and mount behavior.                                    |
| `DrawerBackdrop`         | Optional styled backdrop part for manual composition.                                             |
| `DrawerViewport`         | Optional positioning and scroll container for manual composition.                                 |
| `DrawerPopup`            | Optional popup surface for manual composition.                                                    |
| `DrawerHandle`           | Optional drag handle for manual composition.                                                      |
| `DrawerContentInner`     | Optional wrapper around the Base UI `Content` primitive when composing manually.                  |
| `DrawerHeader`           | Groups title-level content at the top of the panel.                                               |
| `DrawerTitle`            | Accessible drawer title announced by assistive technology.                                        |
| `DrawerDescription`      | Optional supporting text announced with the title.                                                |
| `DrawerBody`             | Optional content region for forms, lists, and scrollable content.                                 |
| `DrawerFooter`           | Optional actions row for close, submit, and secondary actions.                                    |
| `DrawerClose`            | Closes the drawer from action buttons while preserving built-in behavior.                         |
| `DrawerSwipeArea`        | Optional edge-open gesture area.                                                                  |
| `DrawerProvider`         | Optional provider for indent effects and nested gesture coordination.                             |
| `DrawerIndent`           | Optional styled indent wrapper.                                                                   |
| `DrawerIndentBackground` | Optional background layer for the indent effect.                                                  |

For the common case, keep the structure high-level and style the visible parts with `className`
(`DrawerContent`, `DrawerHeader`, `DrawerBody`, `DrawerFooter`). Reach for the low-level parts only
when you need to change portal behavior, backdrop presence, viewport behavior, or popup structure.

## Composition

`Drawer` owns root behavior: `open`, `defaultOpen`, `onOpenChange`, `modal`, `swipeDirection`,
`snapPoints`, `snapPoint`, and the rest of the Base UI root props.

`DrawerContent` stays thin: it accepts the popup props from Base UI, applies the library styles,
and renders the default portal/backdrop/viewport/popup stack. When `modal={false}` or
`modal="trap-focus"`, it skips the backdrop and lets outside pointer interaction through.

`snapLayout` and `variant` are the custom `DrawerContent` props. Use `snapLayout` when a
snap-point drawer should resize its visible content with the active snap point instead of sliding a
full-height panel behind the viewport. Use `variant="island"` when you want the drawer to stay
fully on-screen without the off-screen bleed tail. Use `disableInitialAnimation` only for drawers
that are open on the first page paint and should render immediately in their final position.

When you need deeper control, compose `DrawerPortal`, `DrawerBackdrop`, `DrawerViewport`,
`DrawerPopup`, `DrawerHandle`, and `DrawerContentInner` directly. That is the path for custom portal
behavior (`keepMounted`, `container`), explicit backdrop control, or custom viewport and popup
structure.

### Controlled Persistent State

Keep the drawer open by controlling `open` and ignoring close attempts in your own state logic. This
replaces the removed wrapper-level `persistent` prop.

<Preview>
  <ControlledPersistentDrawerExample />

  <Preview.Code>
    {`
      import {
        Button,
        ChevronDownIcon,
        ChevronUpIcon,
        Drawer,
        DrawerBody,
        DrawerContent,
        DrawerDescription,
        DrawerHeader,
        DrawerTitle,
        ScrollArea,
      } from "moduix";
      import { useState } from "react";
      import styles from "./drawer-demo.module.css";

      export function ControlledPersistentDrawerDemo() {
        const [open, setOpen] = useState(false);
        const [snapPoint, setSnapPoint] = useState<number | string | null>(snapPoints[0]);
        const expanded = snapPoint === snapPoints[1];

        return (
          <>
            <Button type="button" onClick={() => setOpen(true)}>
              Open persistent drawer
            </Button>

            <Drawer
              open={open}
              onOpenChange={(nextOpen) => {
                if (nextOpen) {
                  setOpen(true);
                }
              }}
              modal={false}
              disablePointerDismissal
              snapPoints={[...snapPoints]}
              snapPoint={snapPoint}
              onSnapPointChange={(nextSnapPoint) => {
                if (nextSnapPoint !== null) {
                  setSnapPoint(nextSnapPoint);
                }
              }}
            >
              <DrawerContent snapLayout>
                <DrawerHeader className={styles.headerWithAction}>
                  <div>
                    <DrawerTitle>Controlled persistent drawer</DrawerTitle>
                    <DrawerDescription>
                      Persistence lives in application state; the icon button toggles between
                      collapsed and expanded snap points.
                    </DrawerDescription>
                  </div>
                  <button
                    type="button"
                    className={styles.snapToggle}
                    onClick={() => setSnapPoint(expanded ? snapPoints[0] : snapPoints[1])}
                    aria-label={expanded ? "Collapse drawer" : "Expand drawer"}
                  >
                    {expanded ? <ChevronDownIcon /> : <ChevronUpIcon />}
                  </button>
                </DrawerHeader>
                <DrawerBody>
                  <ScrollArea>{/* content */}</ScrollArea>
                </DrawerBody>
              </DrawerContent>
            </Drawer>
          </>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`      const snapPoints = [0.35, 0.85] as const;`}

  </Preview.Data>
</Preview>
