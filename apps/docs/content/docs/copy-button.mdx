## Basic

Use `CopyButton` when the copied value is already known at render time and the UI only needs one
small action: write that value to the clipboard and expose short-lived copied feedback.

## Anatomy

`CopyButton` stays intentionally small: one local button root plus an internal live region for
transient success feedback.

```text
CopyButton
├─ button root
│  └─ default icon or custom children
└─ visually hidden live region
```

| Part         | Role                                                               |
| ------------ | ------------------------------------------------------------------ |
| `CopyButton` | Action button that writes a known value to the clipboard on click. |
| live region  | Announces transient copied feedback for screen reader users.       |

## Composition

### When to stay high-level

Stay with `CopyButton` directly for API tokens, invite links, commands, IDs, and similar values
that are already known at render time.

The wrapper keeps normal local `Button` behavior intact. Use it for `variant`, `size`, `className`,
`disabled`, `render`, `nativeButton`, refs, native button attributes, and normal click handlers.

The copy-specific surface is intentionally narrow:

- `value` for the copied text
- `copiedDuration` for how long copied state remains active
- `copyLabels` for accessible and visible copied-state wording
- `onCopy` and `onCopyError` for app-level follow-up behavior

### When to go custom

There are no exported subparts or slot maps. The custom path here is small:

- pass `children` for visible button content;
- place `CopyButton` next to `Field`, `Input`, `PasswordInput`, or custom layout when the copied
  value needs surrounding UI;
- use inherited `Button` styling hooks and `[data-copied]` when copied state should change the
  visual treatment.

If the workflow needs async state, toasts, tooltips, or editable field behavior, keep those outside
the component instead of expanding `CopyButton` itself.

### Default props

| Prop             | Default                              | Values / Notes                                                                |
| ---------------- | ------------------------------------ | ----------------------------------------------------------------------------- |
| `type`           | `button`                             | Safe default for action usage inside forms                                    |
| `copiedDuration` | `2000`                               | Duration in milliseconds                                                      |
| `copyLabels`     | `{ copy: 'Copy', copied: 'Copied' }` | `copy` labels the idle icon-only state; `copied` drives success feedback text |
| `size`           | `icon-md` or `md`                    | Defaults to `icon-md` without `children`, otherwise `md`                      |
| `children`       | `CopyIcon` / `CheckIcon`             | Icon-only default swaps to `CheckIcon` while copied                           |

### Behavioral notes

- If `children` is omitted, `CopyButton` becomes an icon-only button and auto-generates an
  accessible name from `copyLabels` unless `aria-label` or `aria-labelledby` is provided.
- If `children` is plain text, the visible label temporarily switches to `copyLabels.copied` after
  success. More complex custom JSX stays untouched.
- The screen-reader live region announces `copyLabels.copied` after success, so copied feedback
  does not rely on `aria-label` changes alone.
- If the consumer `onClick` prevents default, the clipboard write does not run.
- Clipboard failures flow through `onCopyError`. Without that handler, the error is rethrown
  asynchronously instead of being swallowed.

## Examples

### With Text Label

Use visible button text when the copy action should be explicit in the layout. Plain text labels
temporarily switch to the copied label after success.

<Preview>
  <CopyButtonWithTextExample />

  <Preview.Code>
    {`

      export function CopyInviteLinkButton() {
        return (
          <CopyButton
            value="https://moduix.dev/invite/maps"
            copyLabels={{ copy: "Copy invite link", copied: "Invite link copied" }}

## Styling

`CopyButton` intentionally reuses the local `Button` styling surface instead of introducing a
parallel `--copy-button-*` token family.

### CSS Properties

`CopyButton` does not define public `--copy-button-*` variables. Use the inherited `Button`
variables from `theme.css` when size, spacing, focus ring, or variant colors need to change.

### Styling hooks

- `CopyButton` exposes `data-slot="copy-button"` on the root button.
- The root keeps the normal local `Button` selectors, including `data-variant`, `data-size`,
  `[disabled]`, and Base UI disabled state hooks.
- `data-copied` is present only while transient copied state is active.
- `className` applies to the button root, and direct nested SVG icons inherit the normal
  `--button-icon-size` sizing contract.
