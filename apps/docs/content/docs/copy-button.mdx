`CopyButton` is a thin wrapper over the local `Button` that adds one narrow workflow: copying a
known string to the clipboard and exposing a short-lived copied state.

## Anatomy

```text
CopyButton
├─ button root
│  └─ default icon or custom children
└─ visually hidden live region
```

| Part         | Role                                                               |
| ------------ | ------------------------------------------------------------------ |
| `CopyButton` | Action button that writes a known value to the clipboard on click. |
| live region  | Announces transient copied feedback for screen reader users.       |

## Composition

`CopyButton` keeps the normal `Button` composition model. Use `variant`, `size`, `className`, and
`children` the same way you would on `Button`.

The wrapper adds only these workflow-specific props:

- `value` for the copied text
- `copiedDuration` for how long copied state remains active
- `copyLabels` for copy-state feedback text
- `onCopy` and `onCopyError` for app-level follow-up behavior

If `children` is omitted, `CopyButton` becomes an icon-only button by default and uses `CopyIcon`
plus `CheckIcon` during copied state.

If `children` is plain text, that visible label also switches to the copied label during copied
state. More complex custom JSX stays untouched.

`copyLabels` has two jobs:

- `copy` is the default accessible label for icon-only buttons before click
- `copied` is used after success for the icon-only accessible label, the screen-reader live-region announcement, and the temporary visible text on plain-text buttons

## Examples

### With Text Label

Use visible button text when the copy action should be explicit in the layout. Plain text labels
temporarily switch to the copied label after success.

<Preview>
  <CopyButtonWithTextExample />

  <Preview.Code>
    {`
      import { CopyButton } from "moduix";

      export function CopyInviteLinkButton() {
        return (
          <CopyButton
            value="https://moduix.dev/invite/maps"
            copyLabels={{ copy: "Copy invite link", copied: "Invite link copied" }}
          >
            Copy invite link
          </CopyButton>
        );
      }
    `}

## Styling

`CopyButton` intentionally reuses the local `Button` styling surface instead of introducing a
parallel `--copy-button-*` token family.

Use:

- `variant` and `size` for the default visual path
- `className` for local styling
- inherited `--button-*` variables for token-level customization
- `[data-copied]` selectors when copied state should change the visual treatment
