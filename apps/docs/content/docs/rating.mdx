## API Reference

`Rating` does not wrap a dedicated Base UI primitive. It is a moduix composition component built on
radio-group semantics and narrowed to the common star-rating workflow.

## Overview

`Rating` is intentionally small:

- one grouped control
- one discrete numeric value
- one fixed star visual
- no half steps, hover preview, or icon customization

Use it when the user should choose a short integer score such as 1 to 5. For more general
single-choice option sets, use `RadioGroup`.

## Anatomy

`Rating` exports one public component, but internally it behaves like a focused radio-group wrapper.

```text
Rating
└─ star item × max
   └─ star icon
```

| Part      | Role                                                                    |
| --------- | ----------------------------------------------------------------------- |
| `Rating`  | Root grouped control that owns value, form behavior, and size.          |
| star item | Internal discrete option for one score.                                 |
| star icon | Internal visual star that fills when the item is at or below the value. |

## Composition

Use `defaultValue` for uncontrolled mode and `value` with `onValueChange` for controlled mode.
`Rating` maps those numeric props to radio-group semantics internally, so form features like `name`,
`required`, and `form` still work without extra wrapper APIs.

`max` controls how many stars are rendered and defaults to `5`. `size` resizes the whole control,
not individual items.

The component intentionally does **not** provide an in-widget way to clear an existing selection. If
the empty state must be reachable after user selection, control the component and reset `value` from
the parent:

```tsx
<Rating aria-label="Product rating" value={value} onValueChange={setValue} />
```

## Examples

### Controlled

Use controlled mode when the chosen score drives surrounding UI or must be reset by the parent.

<Preview>
  <ControlledRatingExample />

  <Preview.Code>

        return (
          <Field className={styles.field}>
            <FieldLabel>Support quality</FieldLabel>

              aria-label="Support quality"

## Styling

`Rating` has a deliberately small public styling surface:

- `--rating-color` for empty-star color
- `--rating-active-color` for filled-star color
- `--rating-gap` for spacing
- `--rating-icon-size-*` for size scale
- `--rating-focus-ring-*` for keyboard focus treatment

The component uses `data-filled` internally so all stars up to the current score can be styled
consistently, even though the underlying radio primitive only marks one item as checked.
