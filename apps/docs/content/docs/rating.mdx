## API Reference

`Rating` does not wrap a dedicated Ark UI primitive. It is a moduix composition component built on
radio-group semantics and narrowed to the common star-rating workflow.

## Choosing the right component

Use `Rating` when users should choose a short integer score with a familiar star-based affordance.

- Choose **`Rating`** for discrete 1-to-N feedback such as reviews, satisfaction, or quality
  scores.
- Choose **`RadioGroup`** when users still choose exactly one option, but the choices need visible
  labels, richer content, or a non-rating presentation.

Start with `Rating` inside `Field` when the score needs a label, description, and validation.

## Anatomy

The public shape is intentionally small:

```text
Field (optional wrapper)
└─ Rating
   └─ star item × max
      └─ star icon
```

| Part      | Role                                                                    |
| --------- | ----------------------------------------------------------------------- |
| `Rating`  | Root grouped control that owns value, form behavior, and size.          |
| star item | Internal discrete option for one score.                                 |
| star icon | Internal visual star that fills when the item is at or below the value. |

## Composition

### When to stay high-level

Stay with `Rating` for the normal star-rating path. The wrapper keeps radio-group keyboard,
selection, and form behavior intact while exposing a narrower numeric API for ratings.

Use `defaultValue` for uncontrolled mode. Use `value` with `onValueChange` when the selected score
should stay in React state or be reset by the parent.

Pass `name`, `required`, and `form` directly to `Rating` when the score should participate in
native form submission and validation.

Use `size` to scale the whole star row. Use `max` to change how many stars are rendered.

### When to go custom

There is no lower-level moduix `Rating` composition tree to drop into. When the choices should stop
behaving like a rating, switch to the component that better matches the interaction:

- use `RadioGroup` for one-of-many choices with visible labels or custom item content

`Rating` intentionally does not support half steps, hover preview, icon replacement, or an
in-widget clear action.

### Default props

| Prop   | Default | Values / Notes                            |
| ------ | ------- | ----------------------------------------- |
| `max`  | `5`     | Rounded down and normalized to at least 1 |
| `size` | `md`    | Supports `xs`, `sm`, `md`, `lg`, and `xl` |

### Behavioral notes

- `value`, `defaultValue`, and `onValueChange` are numeric wrapper props even though the underlying
  radio-group primitive still uses string values internally.
- `disabled` removes interaction and dims the whole control.
- `readOnly` keeps the current score visible while preventing changes.
- Once the user selects a score, `Rating` does not provide an in-widget way to clear it. When an
  empty state must remain reachable, control the component and reset `value` from the parent.
- Always give the group an accessible name with `aria-label`, `aria-labelledby`, or a surrounding
  labeled field pattern.

## Examples

### Controlled

Use controlled mode when the selected score drives surrounding UI or must be reset by the parent.

<Preview>
  <ControlledRatingExample />

  <Preview.Code>
    {`
      import { Field, Rating } from "moduix";
      import { useState } from "react";

      export function ControlledRatingDemo() {
        const [value, setValue] = useState(3);

        return (
          <div className="stack">
            <Field className="field">
              <Field.Label>Support quality</Field.Label>
              <Rating
                aria-label="Support quality"

### Styling hooks

`Rating` exposes a small public styling surface:

- `data-slot="rating-root"` for the grouped control
- `data-slot="rating-item"` for each internal score item
- `data-slot="rating-icon"` for the star icon inside each item
- `data-size="xs" | "sm" | "md" | "lg" | "xl"` on the root for size-specific styling
- Primitive state attributes such as `data-disabled` and `data-readonly`
- moduix state attributes `data-filled` and `data-empty` on each item so all stars up to the
  current value can be styled consistently
