## Choosing The Right Component

- Choose **`Input`** for plain date strings without calendar behavior.
- Choose **`DateInput`** for keyboard-first segmented date entry.
- Choose **`DatePicker`** when users need an editable input plus a calendar popup or inline calendar.

## Basic

`DatePicker` keeps Ark's explicit composition: render the input field, then render the calendar
views with `ArkDatePicker.Context` from `@ark-ui/react/date-picker`. The wrapper adds styling,
`data-slot` attributes, and default icons for trigger buttons.

## Anatomy

```text
DatePicker / DatePicker.Root
├─ DatePicker.Label
├─ DatePicker.Control
│  ├─ DatePicker.Input
│  ├─ DatePicker.ClearTrigger
│  └─ DatePicker.Trigger
├─ Overlay subtree (automatically portalled)
│  └─ DatePicker.Positioner
│     └─ DatePicker.Content
│        ├─ DatePicker.View[view="day" | "month" | "year"]
│        │  ├─ DatePicker.ViewControl
│        │  │  ├─ DatePicker.PrevTrigger
│        │  │  ├─ DatePicker.ViewTrigger / DatePicker.RangeText
│        │  │  └─ DatePicker.NextTrigger
│        │  └─ DatePicker.Table
│        │     ├─ DatePicker.TableHead / TableBody / TableRow
│        │     ├─ DatePicker.TableHeader
│        │     ├─ DatePicker.TableCell
│        │     └─ DatePicker.TableCellTrigger
│        ├─ DatePicker.MonthSelect / DatePicker.YearSelect
│        └─ DatePicker.PresetTrigger
└─ ArkDatePicker.Context (import from @ark-ui/react/date-picker when you need calendar state)
```

| Part                             | Role                                                            |
| -------------------------------- | --------------------------------------------------------------- |
| `DatePicker` / `DatePicker.Root` | Root value, view, open state, locale, validation, form context. |
| `DatePicker.RootProvider`        | Renders parts from state returned by `useDatePicker()`.         |
| `DatePicker.Label`               | Accessible label.                                               |
| `DatePicker.Control`             | Input field wrapper that receives Ark state attributes.         |
| `DatePicker.Input`               | Editable date text and native form input.                       |
| `DatePicker.Trigger`             | Opens the calendar, defaults to a calendar icon.                |
| `DatePicker.ClearTrigger`        | Clears the current value, defaults to a close icon.             |
| `DatePicker.Positioner`          | Floating layer using Ark positioning CSS variables.             |
| `DatePicker.Content`             | Calendar surface for popup or inline mode.                      |
| `DatePicker.View`                | Day, month, or year panel.                                      |
| `DatePicker.Table*`              | Calendar table composition.                                     |
| `DatePicker.MonthSelect`         | Native month select connected to focused date.                  |
| `DatePicker.YearSelect`          | Native year select connected to focused date.                   |
| `DatePicker.PresetTrigger`       | Preset range button.                                            |

## Composition

The root portals `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

`value`, `defaultValue`, and `onValueChange(details)` use Ark `DateValue[]`. Import `parseDate`,
`DatePicker.Context`, `useDatePicker`, and event/detail types directly from
`@ark-ui/react/date-picker` when you need them. The default root and popup width is `18.75rem`
(300px); override `--date-picker-width` or `--date-picker-content-width` when a wider composition
is needed.
Because popup content is portalled, put popup sizing variables on `DatePicker.Content` or another
element inside the overlay subtree, not only on `DatePicker.Root`.

`open`, `defaultOpen`, and `onOpenChange(details)` control popup state. `view`, `defaultView`,
`minView`, `maxView`, and `onViewChange(details)` control the active calendar panel.

Use `selectionMode="single"`, `"range"`, or `"multiple"`. `maxSelectedDates` applies only to
multiple selection. `numOfMonths`, `fixedWeeks`, and `showWeekNumbers` change the calendar table.

Use `min`, `max`, and `isDateUnavailable(date, locale)` for unavailable dates. Use `locale`,
`timeZone`, `format`, `parse`, `translations`, and `createCalendar` for localization.

### Styling hooks

- `data-slot="date-picker-root"` on the root.
- `data-slot="date-picker-root-provider"` on the provider root.
- `data-slot="date-picker-label"` on the label.
- `data-slot="date-picker-control"` on the input wrapper.
- `data-slot="date-picker-input"` on the editable input.
- `data-slot="date-picker-trigger"` and `date-picker-clear-trigger` on input actions.
- `data-slot="date-picker-positioner"` and `date-picker-content` on the floating layer.
- `data-slot="date-picker-view"`, `view-control`, `view-trigger`, `prev-trigger`, and
  `next-trigger` on calendar navigation.
- `data-slot="date-picker-table*"` on table parts and `date-picker-table-cell-trigger` on cells.
- `data-slot="date-picker-week-number-*"` on week-number cells.
- `data-slot="date-picker-month-select"` and `date-picker-year-select` on native selects.
- `data-slot="date-picker-preset-trigger"` on range preset buttons.

Ark state attributes include `data-state`, `data-disabled`, `data-readonly`, `data-invalid`,
`data-focus`, `data-selected`, `data-today`, `data-unavailable`, `data-outside-range`,
`data-in-range`, `data-range-start`, `data-range-end`, `data-in-hover-range`,
`data-hover-range-start`, `data-hover-range-end`, and `data-view`.
