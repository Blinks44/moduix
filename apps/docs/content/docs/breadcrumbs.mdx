## API Reference

Ark UI does not currently ship a dedicated Breadcrumb primitive. Moduix implements Breadcrumbs with
`@ark-ui/react/factory`, native breadcrumb semantics, and Ark `asChild` composition.

## Basic

Start with explicit semantic parts when the application already knows the visible path. Use links
for navigable ancestors and `Breadcrumbs.Page` for the current page.

## Anatomy

`Breadcrumbs` exposes Ark-aligned namespace parts.

```text
Breadcrumbs / Breadcrumbs.Root
└─ Breadcrumbs.List
   ├─ Breadcrumbs.Item
   │  └─ Breadcrumbs.Link
   ├─ Breadcrumbs.Separator
   └─ Breadcrumbs.Item
      └─ Breadcrumbs.Page
```

| Part                               | Role                                                      |
| ---------------------------------- | --------------------------------------------------------- |
| `Breadcrumbs` / `Breadcrumbs.Root` | Root navigation region with default `aria-label`.         |
| `Breadcrumbs.List`                 | Ordered list of breadcrumb segments.                      |
| `Breadcrumbs.Item`                 | List item wrapper for a segment.                          |
| `Breadcrumbs.Link`                 | Styled anchor for navigable segments; supports `asChild`. |
| `Breadcrumbs.Page`                 | Current page text with `aria-current="page"`.             |
| `Breadcrumbs.Separator`            | Visual delimiter hidden from assistive technology.        |
| `Breadcrumbs.Ellipsis`             | Minimal collapsed marker for menu triggers.               |

## Composition

### Recommended composition

Render the path explicitly. Breadcrumbs does not calculate path items, insert separators, or own
collapsed state.

### When to go custom

Use Ark `asChild` for framework router links. The child must be a single semantic link that keeps
the expected `href`, focus, and keyboard behavior.

Compose with `Menu` when middle segments need to collapse behind an ellipsis trigger. Keep the Menu
structure explicit: `Portal`, `Menu.Positioner`, `Menu.Content`, and `Menu.Item asChild`.

### Behavioral notes

- `Breadcrumbs` renders a `nav` landmark and defaults `aria-label` to `"Breadcrumb"`.
- `Breadcrumbs.Separator` and `Breadcrumbs.Ellipsis` are hidden from assistive technology.
- The accessible label for a collapsed path belongs on the `Menu.Trigger`.
- The last visible item can shrink and truncates with `--breadcrumbs-item-max-width`.
- There are no controlled props, callbacks, context hooks, `RootProvider`, ids, `HiddenInput`, or
  form-state integrations because Breadcrumbs owns no interactive or form state.

### Collapsed Path

Compose with the migrated Ark `Menu` parts when hidden middle segments need disclosure behavior.

<Preview>
  <BreadcrumbsCollapsedExample />

  <Preview.Code>
    {`
      import {
        Breadcrumbs,
        Menu,
        Portal,
      } from "moduix";

      export function BreadcrumbsCollapsedDemo() {
        return (
          <Breadcrumbs>
            <Breadcrumbs.List>
              <Breadcrumbs.Item>
                <Breadcrumbs.Link href="/">Home</Breadcrumbs.Link>
              </Breadcrumbs.Item>
              <Breadcrumbs.Separator />
              <Breadcrumbs.Item>
                <Menu positioning={{ placement: "bottom-start" }}>
                  <Menu.Trigger aria-label="Show hidden path items" className="collapsed-menu-trigger">
                    <Breadcrumbs.Ellipsis />
                  </Menu.Trigger>
                  <Portal>
                    <Menu.Positioner>
                      <Menu.Content className="collapsed-menu-content">
                        {collapsedItems.map((item) => (
                          <Menu.Item key={item.href} value={item.href} asChild>
                            <a href={item.href}>{item.label}</a>
                          </Menu.Item>
                        ))}
                      </Menu.Content>
                    </Menu.Positioner>
                  </Portal>
                </Menu>
              </Breadcrumbs.Item>
              <Breadcrumbs.Separator />
              <Breadcrumbs.Item>
                <Breadcrumbs.Link href="/vacancies">Vacancies</Breadcrumbs.Link>
              </Breadcrumbs.Item>
              <Breadcrumbs.Separator />
              <Breadcrumbs.Item>
                <Breadcrumbs.Page>Go Developer</Breadcrumbs.Page>
              </Breadcrumbs.Item>
            </Breadcrumbs.List>
          </Breadcrumbs>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>
    {`
      .collapsed-menu-trigger {
        min-width: auto;
        min-height: auto;
        padding: 0;
        border: none;
        background: transparent;
        color: inherit;
      }

      .collapsed-menu-content {
        --menu-popup-min-width: 8rem;
        --menu-item-font-size: var(--text-xs);
        --menu-item-padding-x-start: 0.625rem;
        --menu-item-padding-x-end: 0.625rem;
        --menu-item-padding-y: 0.25rem;
      }
    `}

  </Preview.CSS>
  <Preview.Data>
    {`
      const collapsedItems = [
        { href: "/engineering", label: "Engineering" },
        { href: "/engineering/backend", label: "Backend" },
        { href: "/engineering/backend/golang", label: "Golang" },

## Styling

Each exported part has Ark-style `data-scope="breadcrumbs"` and `data-part` attributes plus stable
Moduix `data-slot` hooks. Public `--breadcrumbs-*` variables cover layout, text treatment, focus
ring, separators, and ellipsis sizing.

### Styling hooks

- `data-scope="breadcrumbs"` is present on every exported part.
- `data-part="root"`, `list`, `item`, `link`, `page`, `separator`, and `ellipsis` follow the
  Ark factory contract used by this component.
- `data-slot="breadcrumbs-root"`, `breadcrumbs-list`, `breadcrumbs-item`, `breadcrumbs-link`,
  `breadcrumbs-page`, `breadcrumbs-separator`, and `breadcrumbs-ellipsis` remain stable Moduix
  selectors.
- `Breadcrumbs.Link`, `Breadcrumbs.Page`, and `Breadcrumbs.Ellipsis` share the inline padding from
  `--breadcrumbs-item-padding-x`.
- The last visible `Breadcrumbs.Item` is the truncation target and uses
  `--breadcrumbs-item-max-width`.
