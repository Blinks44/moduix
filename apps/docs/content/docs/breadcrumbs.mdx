## API Reference

`Breadcrumbs` is a standalone moduix component set and does not wrap a Base UI primitive.

## Basic

Start with `Breadcrumbs`, `BreadcrumbsList`, `BreadcrumbsItem`, `BreadcrumbsLink`,
`BreadcrumbsSeparator`, and `BreadcrumbsPage`. This is the recommended path for semantic trail
navigation where the app already knows the visible segments.

## Anatomy

`Breadcrumbs` is a small set of semantic building blocks.

```text
Breadcrumbs
└─ nav[data-slot="breadcrumbs-root"]
   └─ BreadcrumbsList
      ├─ BreadcrumbsItem
      │  └─ BreadcrumbsLink
      ├─ BreadcrumbsSeparator
      └─ BreadcrumbsItem
         └─ BreadcrumbsPage
```

| Part                   | Role                                     |
| ---------------------- | ---------------------------------------- |
| `Breadcrumbs`          | Root navigation region.                  |
| `BreadcrumbsList`      | Ordered list of breadcrumb segments.     |
| `BreadcrumbsItem`      | List item wrapper for a segment.         |
| `BreadcrumbsLink`      | Styled link for navigable segments.      |
| `BreadcrumbsPage`      | Current page text with `aria-current`.   |
| `BreadcrumbsSeparator` | Visual delimiter between visible items.  |
| `BreadcrumbsEllipsis`  | Minimal collapsed marker for menu usage. |

## Composition

### When to stay high-level

Stay with the exported breadcrumb parts when the application already knows the visible path and
just needs semantic navigation markup with library styling.

Use `BreadcrumbsLink` for navigable ancestors and `BreadcrumbsPage` for the current page.

### When to go custom

Use `render={<Link />}` when the app standardizes on a framework router component, and compose
`Breadcrumbs` with `Menu` when collapsed middle segments need disclosure behavior.

### Behavioral notes

- `Breadcrumbs` renders a `nav` landmark and defaults `aria-label` to `"Breadcrumb"`.
- `BreadcrumbsList` owns the ordered-list structure and horizontal layout.
- `BreadcrumbsSeparator` and `BreadcrumbsEllipsis` are explicit JSX parts and are hidden from
  assistive technology.
- `Breadcrumbs` does not calculate paths, inject separators, or manage collapsed state.
- The last visible item can shrink and truncates with ellipsis through
  `--breadcrumbs-item-max-width`.

### Collapsed Path

Compose `Breadcrumbs` with `Menu` when you need a collapsed middle segment.

<Preview>
  <BreadcrumbsCollapsedExample />

  <Preview.Code>
    {`
      import {
        Breadcrumbs,
        BreadcrumbsEllipsis,
        BreadcrumbsItem,
        BreadcrumbsLink,
        BreadcrumbsList,
        BreadcrumbsPage,
        BreadcrumbsSeparator,
        Menu,
        MenuContent,
        MenuLinkItem,
        MenuTrigger,
      } from "moduix";
      import styles from "./breadcrumbs.module.css";

      export function BreadcrumbsCollapsedDemo() {
        return (
          <Breadcrumbs>
            <BreadcrumbsList>
              <BreadcrumbsItem>
                <BreadcrumbsLink href="/">Home</BreadcrumbsLink>
              </BreadcrumbsItem>
              <BreadcrumbsSeparator />
              <BreadcrumbsItem>
                <Menu>
                  <MenuTrigger aria-label="Show hidden path items" className={styles.collapsedMenuTrigger}>
                    <BreadcrumbsEllipsis />
                  </MenuTrigger>
                  <MenuContent align="start" className={styles.collapsedMenuContent}>
                    {collapsedItems.map((item) => (
                      <MenuLinkItem key={item.href} closeOnClick href={item.href}>
                        {item.label}
                      </MenuLinkItem>
                    ))}
                  </MenuContent>
                </Menu>
              </BreadcrumbsItem>
              <BreadcrumbsSeparator />
              <BreadcrumbsItem>
                <BreadcrumbsLink href="/vacancies">Vacancies</BreadcrumbsLink>
              </BreadcrumbsItem>
              <BreadcrumbsSeparator />
              <BreadcrumbsItem>
                <BreadcrumbsPage>Go Developer</BreadcrumbsPage>
              </BreadcrumbsItem>
            </BreadcrumbsList>
          </Breadcrumbs>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>
    {`
      .collapsedMenuTrigger {
        min-width: auto;
        min-height: auto;
        padding: 0;
        border: none;
        background: transparent;
        color: inherit;
      }

      .collapsedMenuContent {
        --menu-popup-min-width: 8rem;
        --menu-item-font-size: var(--text-xs);
        --menu-item-padding-x-start: 0.625rem;
        --menu-item-padding-x-end: 0.625rem;
        --menu-item-padding-y: 0.25rem;
      }
    `}

  </Preview.CSS>
  <Preview.Data>
    {`
      const collapsedItems = [
        { href: "/engineering", label: "Engineering" },
        { href: "/engineering/backend", label: "Backend" },
        { href: "/engineering/backend/golang", label: "Golang" },
      ] as const;
    `}
  </Preview.Data>
</Preview>

## Styling

Each exported part has a stable `data-slot`, and the public `--breadcrumbs-*` variables cover the
layout, text treatment, focus ring, separators, and ellipsis sizing used by the component.

### Styling hooks

- `data-slot="breadcrumbs-root"`, `breadcrumbs-list`, `breadcrumbs-item`, `breadcrumbs-link`,
  `breadcrumbs-page`, `breadcrumbs-separator`, and `breadcrumbs-ellipsis` are stable selectors on
  the exported parts.
- `BreadcrumbsLink`, `BreadcrumbsPage`, and `BreadcrumbsEllipsis` share the inline padding from
  `--breadcrumbs-item-padding-x`.
- The last visible `BreadcrumbsItem` is the truncation target and uses
  `--breadcrumbs-item-max-width`.
