# Blocks

Blocks are complete, editable starting points for common product screens. Install one into your
project, then connect it to your data and routes. The components inside stay in `@moduix/react`;
the block source and its CSS module become yours.

Use blocks when the useful unit is a workflow rather than a primitive. They provide considered
responsive layout and component composition, but deliberately leave routing, mutations,
authentication, and product copy to your application.

## Recommended workflow

1. Complete the one-time hosted registry setup in [Quick start](/docs/quick-start#install-components-into-your-project).
2. Preview the block and its CSS on the page below, then install it with the generated `shadcn`
   command.
3. Replace demo data and links before adding product behavior; keep the block source copy-owned and
   update `@moduix/react` through your package manager.
