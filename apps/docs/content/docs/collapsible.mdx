`Collapsible` is the moduix wrapper for a single trigger/panel pair. It keeps Base UI disclosure
behavior, but the public API here is the flat moduix export surface: `Collapsible`,
`CollapsibleTrigger`, `CollapsiblePanel`, and the optional `CollapsibleTriggerIcon`.

Use it when one section of supporting content needs to expand inline without introducing the
multi-item semantics of `Accordion`.

## Anatomy

Keep one trigger and one panel inside the same root:

```text
Collapsible
├─ CollapsibleTrigger
│  ├─ label
│  └─ CollapsibleTriggerIcon (optional)
└─ CollapsiblePanel
   └─ content
```

```tsx
<Collapsible>
  <CollapsibleTrigger>
    Recovery keys
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

| Part                     | Role                                                                                                        |
| ------------------------ | ----------------------------------------------------------------------------------------------------------- |
| `Collapsible`            | Root state machine for one disclosure pair. Controls `open`, `defaultOpen`, `disabled`, and `onOpenChange`. |
| `CollapsibleTrigger`     | Interactive toggle button. Uses the default moduix trigger styling unless `render` is provided.             |
| `CollapsibleTriggerIcon` | Optional decorative helper. Defaults to `ChevronRightIcon` and rotates from `data-panel-open`.              |
| `CollapsiblePanel`       | Collapsible content region. Put spacing or surface wrappers inside it so height animation stays clean.      |

`CollapsibleTriggerIcon` is narrow sugar for a common production case. Keep it when the default
chevron is enough, replace its children when the UI needs a different cue, or omit it entirely.

## Public props

The wrappers preserve Base UI props instead of renaming them.

| Part                 | Common props                                                                     |
| -------------------- | -------------------------------------------------------------------------------- |
| `Collapsible`        | `open`, `defaultOpen`, `onOpenChange`, `disabled`, `className`, `render`         |
| `CollapsibleTrigger` | `className`, `render`, `nativeButton`, and native button props                   |
| `CollapsiblePanel`   | `className`, `keepMounted`, `hiddenUntilFound`, `render`, and native `div` props |

`className` also accepts Base UI state resolver functions; moduix merges those with the shipped
classes. When the trigger host is not a native `<button>`, pass `nativeButton={false}` so Base UI
can apply ARIA button behavior correctly.

## Styling API

Every exported part exposes `className` and a stable `data-slot`:

| Part                     | `data-slot`                |
| ------------------------ | -------------------------- |
| `Collapsible`            | `collapsible-root`         |
| `CollapsibleTrigger`     | `collapsible-trigger`      |
| `CollapsibleTriggerIcon` | `collapsible-trigger-icon` |
| `CollapsiblePanel`       | `collapsible-panel`        |

Base UI state hooks used by the default styles:

| Attribute             | Where it appears | Meaning                                                                      |
| --------------------- | ---------------- | ---------------------------------------------------------------------------- |
| `data-panel-open`     | trigger          | Matching panel is open.                                                      |
| `data-open`           | panel            | Panel is open.                                                               |
| `data-closed`         | panel            | Panel is closed.                                                             |
| `data-starting-style` | panel            | Panel is entering.                                                           |
| `data-ending-style`   | panel            | Panel is leaving.                                                            |
| `hidden`              | panel            | Closed panels are hidden unless `keepMounted` or `hiddenUntilFound` is used. |

The default trigger is a full-width flex button with hover, active, focus-visible, and disabled
states. `CollapsibleTriggerIcon` rotates automatically. `CollapsiblePanel` animates its height from
the runtime `--collapsible-panel-height` value. By default the root uses
`--collapsible-width: 14rem` with `--collapsible-max-width: 100%`. Use the **CSS Properties** tab
in the preview above for the full variable contract.

## Accessibility and UX notes

- Prefer the default trigger render path. It gives correct button semantics and keyboard behavior with
  no extra work.
- Keep nested interactive controls out of `CollapsibleTrigger`; place them beside the trigger or inside
  the panel.
- `CollapsibleTriggerIcon` is decorative by default because it renders with `aria-hidden`.
- For precise height animation, put padding and borders on inner panel content instead of the panel
  element itself.

```tsx
<Collapsible>
  <CollapsibleTrigger render={<div />} nativeButton={false}>
    <span>Recovery keys</span>
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel keepMounted>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

## Examples

`defaultOpen`, `open`, `disabled`, `keepMounted`, and `hiddenUntilFound` all stay on the same parts
as in Base UI, but the examples below use the shipped moduix wrappers and styles.

### Controlled

Control `open` from React state when the collapsible must coordinate with surrounding UI.

<Preview>
  <ControlledCollapsibleExample />

  <Preview.Code>
    {`
      import {
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCollapsibleDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Collapsible open={open} onOpenChange={setOpen}>
            <CollapsibleTrigger>
              Recovery keys
              <CollapsibleTriggerIcon />
            </CollapsibleTrigger>
            <CollapsiblePanel>
              <ul className={styles.keysList}>
                {recoveryKeys.map((key) => (
                  <li key={key}>{key}</li>
                ))}
              </ul>
            </CollapsiblePanel>
            <span className={styles.status}>Current state: {open ? "open" : "closed"}</span>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Custom Composition

Use `render` when the trigger host must change, and keep `nativeButton={false}` for non-button
elements. `CollapsibleTriggerIcon` can still provide the default rotation behavior while you swap
the icon and surface styles.

<Preview>
  <CustomCompositionCollapsibleExample />

  <Preview.Code>
    {`
      import {
        ChevronDownIcon,
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";

      export function CustomCompositionCollapsibleDemo() {
        return (
          <Collapsible>
            <CollapsibleTrigger
              render={<div />}
              nativeButton={false}
              className={styles.customTrigger}
            >
              <span className={styles.triggerLabel}>Styled recovery keys</span>
              <CollapsibleTriggerIcon className={styles.customTriggerIcon}>
                <ChevronDownIcon />
              </CollapsibleTriggerIcon>
            </CollapsibleTrigger>
            <CollapsiblePanel className={styles.customPanel}>
              <div className={styles.customPanelContent}>
                <ul className={styles.keysList}>
                  {recoveryKeys.map((key) => (
                    <li key={key}>{key}</li>
                  ))}
                </ul>
              </div>
            </CollapsiblePanel>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>
