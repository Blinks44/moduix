## Basic

Use `Collapsible` when one section of supporting content needs to expand inline without the
multi-item semantics of `Accordion`.

## Anatomy

Keep one trigger and one panel inside the same root.

```text
Collapsible
├─ CollapsibleTrigger
│  ├─ label
│  └─ CollapsibleTriggerIcon (optional)
└─ CollapsiblePanel
   └─ content
```

| Part                     | Role                                                                                                        |
| ------------------------ | ----------------------------------------------------------------------------------------------------------- |
| `Collapsible`            | Root state machine for one disclosure pair. Controls `open`, `defaultOpen`, `disabled`, and `onOpenChange`. |
| `CollapsibleTrigger`     | Interactive toggle button. Uses the default moduix trigger styling unless `render` is provided.             |
| `CollapsibleTriggerIcon` | Optional decorative helper. Defaults to `ChevronRightIcon` and rotates from `data-panel-open`.              |
| `CollapsiblePanel`       | Collapsible content region. Put spacing or surface wrappers inside it so height animation stays clean.      |

## Composition

### When to stay high-level

Stay with `Collapsible`, `CollapsibleTrigger`, `CollapsiblePanel`, and the optional
`CollapsibleTriggerIcon` for most inline disclosure rows, recovery-key sections, and supporting
metadata blocks.

The wrappers keep Base UI props intact. Use `open`, `defaultOpen`, `onOpenChange`, `disabled`,
`keepMounted`, and `hiddenUntilFound` on the same parts as the upstream primitive.

### When to go custom

Use `render` on `CollapsibleTrigger` only when the host element must change. When that host is not
a real `<button>`, set `nativeButton={false}` so Base UI can apply ARIA button behavior correctly.

Use `CollapsibleTriggerIcon` when the default chevron is enough, replace its children when the UI
needs a different disclosure cue, or omit it entirely.

### Default props

| Part                     | Default            | Values / Notes                                                           |
| ------------------------ | ------------------ | ------------------------------------------------------------------------ |
| `Collapsible`            | closed             | `defaultOpen`, `open`, `onOpenChange`, `disabled`, `className`, `render` |
| `CollapsibleTrigger`     | button host        | `className`, `render`, `nativeButton`, native button props               |
| `CollapsiblePanel`       | hidden when closed | `className`, `keepMounted`, `hiddenUntilFound`, `render`                 |
| `CollapsibleTriggerIcon` | chevron            | Decorative `span`; `aria-hidden` defaults to `true`                      |

### Behavioral notes

- Prefer the default trigger render path whenever possible. It gives correct button semantics and
  keyboard behavior with no extra work.
- Keep nested interactive controls out of `CollapsibleTrigger`; place them beside the trigger or
  inside the panel.
- Put padding, borders, and surface styling on inner panel content when precise height animation
  matters.

### Controlled

Control `open` from React state when the disclosure must coordinate with surrounding UI.

<Preview>
  <ControlledCollapsibleExample />

  <Preview.Code>
    {`
      import {
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCollapsibleDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Collapsible open={open} onOpenChange={setOpen}>
            <CollapsibleTrigger>
              Recovery keys
              <CollapsibleTriggerIcon />
            </CollapsibleTrigger>
            <CollapsiblePanel>
              <ul className={styles.keysList}>
                {recoveryKeys.map((key) => (
                  <li key={key}>{key}</li>
                ))}
              </ul>
            </CollapsiblePanel>
            <span className={styles.status}>Current state: {open ? "open" : "closed"}</span>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Custom Composition

Use `render` when the trigger host must change and keep `nativeButton={false}` for non-button
elements. `CollapsibleTriggerIcon` can still provide the rotation behavior while you swap the icon
and surface styles.

<Preview>
  <CustomCompositionCollapsibleExample />

  <Preview.Code>
    {`
      import {
        ChevronDownIcon,
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";

      export function CustomCompositionCollapsibleDemo() {
        return (
          <Collapsible>
            <CollapsibleTrigger
              render={<div />}
              nativeButton={false}
              className={styles.customTrigger}
            >
              <span className={styles.triggerLabel}>Styled recovery keys</span>
              <CollapsibleTriggerIcon className={styles.customTriggerIcon}>
                <ChevronDownIcon />
              </CollapsibleTriggerIcon>
            </CollapsibleTrigger>
            <CollapsiblePanel className={styles.customPanel}>
              <div className={styles.customPanelContent}>
                <ul className={styles.keysList}>
                  {recoveryKeys.map((key) => (
                    <li key={key}>{key}</li>
                  ))}
                </ul>
              </div>
            </CollapsiblePanel>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Styling

Use CSS variables on the root, trigger, icon, and panel when spacing, motion, and disclosure
styling need to change. Keep padding and surface treatment on inner panel content when animation
quality matters.

### Styling hooks

- `Collapsible`, `CollapsibleTrigger`, `CollapsibleTriggerIcon`, and `CollapsiblePanel` expose
  stable `data-slot` selectors.
- Base UI state hooks used by the shipped styles include `data-panel-open` on the trigger and
  `data-open`, `data-closed`, `data-starting-style`, and `data-ending-style` on the panel.
- `CollapsibleTrigger render={...}` opts out of the default trigger CSS class, so custom trigger
  hosts own their full layout and surface styling.
- `--collapsible-panel-starting-*` and `--collapsible-panel-ending-*` let you layer fade, scale,
  or slide motion on top of the default measured height transition.
