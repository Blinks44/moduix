## Anatomy

`Collapsible` groups one trigger and one panel that share the same open state. Keep
`CollapsibleTrigger` and `CollapsiblePanel` inside the same root so Base UI can preserve the
keyboard and accessibility wiring.

```text
Collapsible
├─ CollapsibleTrigger
│  ├─ label
│  └─ CollapsibleTriggerIcon (optional)
└─ CollapsiblePanel
   └─ content
```

```tsx
<Collapsible>
  <CollapsibleTrigger>
    Recovery keys
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

| Part                     | Role                                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------------------------------ |
| `Collapsible`            | Root state machine. Controls `open`, `defaultOpen`, `disabled`, and `onOpenChange`.                    |
| `CollapsibleTrigger`     | Interactive toggle button. Opens or closes the panel and receives open/disabled attributes.            |
| `CollapsibleTriggerIcon` | Optional helper for a decorative state marker inside the trigger.                                      |
| `CollapsiblePanel`       | Collapsible content region. Put spacing or surface wrappers inside it so height animation stays clean. |

`CollapsibleTriggerIcon` is narrow sugar for a common production case. Keep it when the default
chevron is enough, replace its children when the UI needs a different cue, or omit it entirely.

## Composition

Use `Collapsible` for root state props such as `open`, `defaultOpen`, `onOpenChange`, and
`disabled`. Style the exported parts directly with `className`. Base UI behavior props such as
`render`, `nativeButton`, `keepMounted`, and `hiddenUntilFound` pass through to the matching part.

If the trigger host is not a native `<button>`, pass `nativeButton={false}` so Base UI can apply
the correct ARIA button behavior.

| Prop          | Default | Values                                               |
| ------------- | ------- | ---------------------------------------------------- |
| `open`        | -       | `boolean` (controlled)                               |
| `defaultOpen` | `false` | `boolean` (uncontrolled initial state)               |
| `disabled`    | `false` | `boolean`                                            |
| `className`   | -       | Part class string or Base UI state resolver function |

```tsx
<Collapsible>
  <CollapsibleTrigger render={<div />} nativeButton={false}>
    <span>Recovery keys</span>
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel keepMounted>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

### Controlled

Control `open` from React state when the collapsible must coordinate with surrounding UI.

<Preview>
  <ControlledCollapsibleExample />

  <Preview.Code>
    {`
      import {
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCollapsibleDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Collapsible open={open} onOpenChange={setOpen} className={styles.root}>
            <CollapsibleTrigger>
              Recovery keys
              <CollapsibleTriggerIcon />
            </CollapsibleTrigger>
            <CollapsiblePanel>
              <ul className={styles.keysList}>
                {recoveryKeys.map((key) => (
                  <li key={key}>{key}</li>
                ))}
              </ul>
            </CollapsiblePanel>
            <span className={styles.status}>Current state: {open ? "open" : "closed"}</span>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Custom Composition

Use `render` when the trigger host must change, and keep `nativeButton={false}` for non-button
elements. `CollapsibleTriggerIcon` can still provide the default rotation behavior while you swap
the icon and surface styles.

<Preview>
  <CustomCompositionCollapsibleExample />

  <Preview.Code>
    {`
      import {
        ChevronDownIcon,
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
      } from "moduix";

      export function CustomCompositionCollapsibleDemo() {
        return (
          <Collapsible className={styles.customRoot}>
            <CollapsibleTrigger
              render={<div />}
              nativeButton={false}
              className={styles.customTrigger}
            >
              <span className={styles.triggerLabel}>Styled recovery keys</span>
              <CollapsibleTriggerIcon className={styles.customTriggerIcon}>
                <ChevronDownIcon />
              </CollapsibleTriggerIcon>
            </CollapsibleTrigger>
            <CollapsiblePanel className={styles.customPanel}>
              <div className={styles.customPanelContent}>
                <ul className={styles.keysList}>
                  {recoveryKeys.map((key) => (
                    <li key={key}>{key}</li>
                  ))}
                </ul>
              </div>
            </CollapsiblePanel>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>
</Preview>
