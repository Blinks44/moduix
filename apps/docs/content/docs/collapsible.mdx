## Anatomy

```text
Collapsible.Root
├─ Collapsible.Trigger
│  ├─ label
│  └─ Collapsible.Indicator (optional)
└─ Collapsible.Content
   └─ content
```

| Part                               | Role                                                   |
| ---------------------------------- | ------------------------------------------------------ |
| `Collapsible` / `Collapsible.Root` | Owns controlled or uncontrolled open state.            |
| `Collapsible.RootProvider`         | Connects parts to state created with `useCollapsible`. |
| `Collapsible.Trigger`              | Accessible button that toggles the content.            |
| `Collapsible.Indicator`            | Optional state cue with a default moduix chevron.      |
| `Collapsible.Content`              | Collapsible region measured through Ark CSS variables. |

## Composition

### Recommended composition

Use `Collapsible`, `Collapsible.Trigger`, optional `Collapsible.Indicator`, and
`Collapsible.Content`. Root props include `open`, `defaultOpen`, `onOpenChange(details)`,
`disabled`, `collapsedHeight`, `collapsedWidth`, `lazyMount`, `unmountOnExit`, `ids`, and
`onExitComplete`.

Keep padding and surface styles on an inner element inside `Collapsible.Content`. Ark measures the
content and exposes `--height`, `--width`, `--collapsed-height`, and `--collapsed-width`.

### When to go custom

Use `asChild` when another semantic element must own the rendered node. In this mode Collapsible
merges Ark behavior and state attributes but leaves the visual styling to that element.

Use Ark `useCollapsible` with `Collapsible.RootProvider` when state must be created outside the part
tree. Advanced state hooks and context access come from `@ark-ui/react/collapsible`, not from the
moduix wrapper.

`Collapsible.Indicator` is optional. It renders the moduix right chevron by default and accepts
custom children.

### Default props

| Part                    | Default          | Notes                                              |
| ----------------------- | ---------------- | -------------------------------------------------- |
| `Collapsible`           | closed           | `defaultOpen={false}`, mounted content             |
| `Collapsible.Trigger`   | `button`         | Supports Ark `asChild`                             |
| `Collapsible.Indicator` | right chevron    | Rotates from `data-state="open"`                   |
| `Collapsible.Content`   | collapsed region | Uses Ark measurement variables and render strategy |

### Behavioral notes

- `onOpenChange` receives an Ark details object. Read the next state from `details.open`.
- `open` is the intended state, while `visible` stays true while closing content remains mounted
  for exit animation.
- Keep nested interactive controls outside `Collapsible.Trigger`.
- Use `Accordion` instead when multiple disclosures need coordinated keyboard or value behavior.

### Controlled

Use `open` with the Ark `onOpenChange(details)` callback when surrounding UI owns the state.

<Preview>
  <CollapsibleDocs.ControlledCollapsibleExample />

  <Preview.Code>
    {`
      import { Collapsible } from "@moduix/react";
      import { useState } from "react";

      export function ControlledCollapsibleDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Collapsible
            open={open}
            onOpenChange={(details) => setOpen(details.open)}
          >
            <Collapsible.Trigger>
              Recovery keys
              <Collapsible.Indicator />
            </Collapsible.Trigger>
            <Collapsible.Content>
              <div className="collapsible-content-body">
                <ul className="collapsible-keys-list">
                  {recoveryKeys.map((key) => (
                    <li key={key}>{key}</li>
                  ))}
                </ul>
              </div>
            </Collapsible.Content>
            <span className="collapsible-state">Current state: {open ? "open" : "closed"}</span>
          </Collapsible>
        );
      }
    `}

## Styling

### CSS Properties

<CssPropertiesSection properties={CollapsibleDocs.collapsibleOverrideCssProperties} />

### Styling hooks

- DOM parts expose `data-slot="collapsible-root"`, `collapsible-root-provider`,
  `collapsible-trigger`, `collapsible-indicator`, and `collapsible-content`.
- Ark exposes `data-state="open" | "closed"` on root, trigger, indicator, and content.
- Ark adds `data-collapsible` to the content part.
- `data-disabled` marks disabled trigger, indicator, and content parts.
- `data-has-collapsed-size` marks content configured with `collapsedHeight` or `collapsedWidth`.
- Content animation uses Ark `--height`, `--width`, `--collapsed-height`, and `--collapsed-width`.
