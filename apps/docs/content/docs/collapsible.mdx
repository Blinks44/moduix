## Anatomy

`Collapsible` groups one toggle control and one panel that share the same open state.
Keep `CollapsibleTrigger` and `CollapsiblePanel` inside the same root so keyboard and
accessibility wiring stay connected.

```text
Collapsible
├─ CollapsibleTrigger
│  ├─ label
│  └─ CollapsibleTriggerIcon (optional)
└─ CollapsiblePanel
   └─ content
```

```tsx
<Collapsible>
  <CollapsibleTrigger>
    Recovery keys
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

| Part                     | Role                                                                                        |
| ------------------------ | ------------------------------------------------------------------------------------------- |
| `Collapsible`            | Root state machine. Controls `open`, `defaultOpen`, `disabled`, and `onOpenChange`.         |
| `CollapsibleTrigger`     | Interactive toggle button. Opens or closes the panel and receives open/disabled attributes. |
| `CollapsibleTriggerIcon` | Optional helper for a decorative state marker inside the trigger.                           |
| `CollapsiblePanel`       | Collapsible content region. Put spacing/surface wrappers inside it for cleaner animations.  |

`CollapsibleTrigger` is a thin styled wrapper over the Base UI trigger. Add
`CollapsibleTriggerIcon` inside it when you want a decorative state marker. The icon helper is
`aria-hidden` by default, so decorative icons stay out of the accessibility tree.

## Composition

Use `Collapsible` for root state props (`open`, `defaultOpen`, `onOpenChange`, `disabled`).
Style root, trigger, panel, and icon directly with `className`. Base UI behavior props such as
`render`, `nativeButton`, `keepMounted`, and `hiddenUntilFound` pass through to the matching part.
If the trigger host is not a native `<button>`, pass `nativeButton={false}` so Base UI applies the
correct ARIA button behavior.

| Prop          | Default | Values                                               |
| ------------- | ------- | ---------------------------------------------------- |
| `open`        | -       | `boolean` (controlled)                               |
| `defaultOpen` | `false` | `boolean` (uncontrolled initial state)               |
| `disabled`    | `false` | `boolean`                                            |
| `className`   | -       | Part class string or Base UI state resolver function |

```tsx
<Collapsible>
  <CollapsibleTrigger render={<div />} nativeButton={false}>
    <span>Recovery keys</span>
    <CollapsibleTriggerIcon />
  </CollapsibleTrigger>
  <CollapsiblePanel keepMounted>
    <div className={styles.panelContent}>Store these keys somewhere safe.</div>
  </CollapsiblePanel>
</Collapsible>
```

### Controlled

Control `open` from React state when the collapsible needs to coordinate with other UI.

<Preview>
  <ControlledCollapsibleExample />

  <Preview.Code>
    {`
      import {
        Collapsible,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
        CollapsiblePanel,
      } from "moduix";
      import { useState } from "react";

      export function ControlledCollapsibleDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Collapsible open={open} onOpenChange={setOpen} className={styles.root}>
            <CollapsibleTrigger>
              Recovery keys
              <CollapsibleTriggerIcon />
            </CollapsibleTrigger>
            <CollapsiblePanel>
              <ul className={styles.keysList}>
                {recoveryKeys.map((key) => (
                  <li key={key}>{key}</li>
                ))}
              </ul>
            </CollapsiblePanel>
            <span className={styles.status}>Current state: {open ? "open" : "closed"}</span>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .root {
        width: 16rem;
      }

      .keysList {
        display: flex;
        flex-direction: column;
        gap: var(--spacing-1);
        margin: 0;
        padding-inline-start: var(--spacing-2);
      }

        margin-top: var(--spacing-2);
        color: var(--color-muted-foreground);

### Manual Trigger Composition

Use `render` to swap the trigger element while keeping the same composition pattern.

<Preview>
  <ManualTriggerCompositionCollapsibleExample />

  <Preview.Code>
    {`
      import {
        Collapsible,
        CollapsiblePanel,
        CollapsibleTrigger,
        CollapsibleTriggerIcon,
        ChevronDownIcon,
      } from "moduix";

      export function ManualTriggerCompositionCollapsibleDemo() {
        return (
          <Collapsible className={styles.customRoot}>
            <CollapsibleTrigger
              render={<div />}
              nativeButton={false}
              className={styles.customTrigger}
            >
              <span className={styles.triggerLabel}>Recovery keys</span>
              <CollapsibleTriggerIcon className={styles.customTriggerIcon}>
                <ChevronDownIcon />
              </CollapsibleTriggerIcon>
            </CollapsibleTrigger>
            <CollapsiblePanel className={styles.customPanel}>
              <div className={styles.customPanelContent}>
                <ul className={styles.keysList}>
                  {recoveryKeys.map((key) => (
                    <li key={key}>{key}</li>
                  ))}
                </ul>
              </div>
            </CollapsiblePanel>
          </Collapsible>
        );
      }
    `}

  </Preview.Code>

    {`
