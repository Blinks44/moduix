## Anatomy

```text
Collapsible.Root
├─ Collapsible.Trigger
│  ├─ label
│  └─ Collapsible.Indicator (optional)
└─ Collapsible.Content
   └─ Collapsible.Body
      └─ content
```

| Part                               | Role                                                   |
| ---------------------------------- | ------------------------------------------------------ |
| `Collapsible` / `Collapsible.Root` | Owns controlled or uncontrolled open state.            |
| `Collapsible.RootProvider`         | Connects parts to state created with `useCollapsible`. |
| `Collapsible.Trigger`              | Accessible button that toggles the content.            |
| `Collapsible.Indicator`            | Optional state cue with a default moduix chevron.      |
| `Collapsible.Content`              | Collapsible region measured through Ark CSS variables. |
| `Collapsible.Body`                 | Moduix inner wrapper for padding and content layout.   |

## Composition

### Recommended composition

Use `Collapsible`, `Collapsible.Trigger`, optional `Collapsible.Indicator`,
`Collapsible.Content`, and `Collapsible.Body`. Root props include `open`, `defaultOpen`,
`onOpenChange(details)`, `disabled`, `collapsedHeight`, `collapsedWidth`, `lazyMount`,
`unmountOnExit`, `ids`, and `onExitComplete`.

Keep padding and surface styles on `Collapsible.Body` instead of `Collapsible.Content`. Ark measures
the content and exposes `--height`, `--width`, `--collapsed-height`, and `--collapsed-width`.

When migrating from shadcn, map `CollapsibleTrigger` to `Collapsible.Trigger` and
`CollapsibleContent` to `Collapsible.Content`. moduix keeps Ark's details object, so controlled
state reads `details.open` instead of receiving the boolean directly.

### When to go custom

Use `asChild` when another semantic element must own the rendered node. In this mode Collapsible
merges Ark behavior and state attributes but leaves the visual styling to that element.

Use moduix `useCollapsible` with `Collapsible.RootProvider` when state must be created outside the
part tree. Advanced context hooks remain available from `@ark-ui/react/collapsible` as escape
hatches.

`Collapsible.Indicator` is optional. It renders the moduix down chevron by default and accepts
custom children.

### Default props

| Part                    | Default          | Notes                                              |
| ----------------------- | ---------------- | -------------------------------------------------- |
| `Collapsible`           | closed           | `defaultOpen={false}`, mounted content             |
| `Collapsible.Trigger`   | `button`         | Supports Ark `asChild`                             |
| `Collapsible.Indicator` | down chevron     | Rotates upward from `data-state="open"`            |
| `Collapsible.Content`   | collapsed region | Uses Ark measurement variables and render strategy |
| `Collapsible.Body`      | `div`            | Inner layout wrapper with padding and gap          |

### Behavioral notes

- `onOpenChange` receives an Ark details object. Read the next state from `details.open`.
- `open` is the intended state, while `visible` stays true while closing content remains mounted
  for exit animation.
- Keep nested interactive controls outside `Collapsible.Trigger`.
- Use `Accordion` instead when multiple disclosures need coordinated keyboard or value behavior.

## Styling

### CSS Properties

<CssPropertiesSection properties={CollapsibleDocs.collapsibleOverrideCssProperties} />

### Styling hooks

- DOM parts expose `data-slot="collapsible-root"`, `collapsible-root-provider`,
  `collapsible-trigger`, `collapsible-indicator`, `collapsible-content`, and `collapsible-body`.
- Ark exposes `data-state="open" | "closed"` on root, trigger, indicator, and content.
- Ark adds `data-collapsible` to the content part.
- `data-disabled` marks disabled trigger, indicator, and content parts.
- `data-has-collapsed-size` marks content configured with `collapsedHeight` or `collapsedWidth`.
- Content animation uses Ark `--height`, `--width`, `--collapsed-height`, and `--collapsed-width`.
