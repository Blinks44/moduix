## Anatomy

```tsx
<TreeView>
  <TreeView.Label />
  <TreeView.Tree>
    <TreeView.NodeProvider>
      <TreeView.Branch>
        <TreeView.BranchControl>
          <TreeView.BranchIndicator />
          <TreeView.BranchText />
        </TreeView.BranchControl>
        <TreeView.BranchContent>
          <TreeView.BranchIndentGuide />
          <TreeView.Item>
            <TreeView.ItemText />
          </TreeView.Item>
        </TreeView.BranchContent>
      </TreeView.Branch>
    </TreeView.NodeProvider>
  </TreeView.Tree>
</TreeView>
```

| Part                             | Role                                                                                        |
| -------------------------------- | ------------------------------------------------------------------------------------------- |
| `TreeView` / `TreeView.Root`     | Owns collection state, selection, expansion, focus, checking, loading, and rename behavior. |
| `TreeView.RootProvider`          | Renders a tree from a `useTreeView` instance owned outside the JSX tree.                    |
| `TreeView.Label`                 | Labels the tree for assistive technology.                                                   |
| `TreeView.Tree`                  | Contains the recursive node tree.                                                           |
| `TreeView.NodeProvider`          | Provides the current node and `indexPath` to branch and item parts.                         |
| `TreeView.Branch`                | Wraps an expandable node.                                                                   |
| `TreeView.BranchControl`         | Interactive row for a branch.                                                               |
| `TreeView.BranchTrigger`         | Optional dedicated expand/collapse trigger.                                                 |
| `TreeView.BranchIndicator`       | Visual branch state indicator.                                                              |
| `TreeView.BranchText`            | Branch label content.                                                                       |
| `TreeView.BranchContent`         | Collapsible child region.                                                                   |
| `TreeView.BranchIndentGuide`     | Optional visual guide for nested levels.                                                    |
| `TreeView.Item`                  | Leaf node row.                                                                              |
| `TreeView.ItemText`              | Leaf label content.                                                                         |
| `TreeView.ItemIndicator`         | Optional selected item indicator.                                                           |
| `TreeView.NodeCheckbox`          | Node checkbox for checked trees.                                                            |
| `TreeView.NodeCheckboxIndicator` | Checked and indeterminate visual states.                                                    |
| `TreeView.NodeRenameInput`       | Inline input shown while a node is being renamed.                                           |

## Composition

### Recommended composition

Keep the recursive `NodeProvider` visible. It is how Ark connects each node to the collection,
keyboard focus, selection, expansion, loading, checking, and rename state.

### When to go custom

Use `asChild` on `TreeView.Item` when a leaf should be a link or another semantic element. The child
must be a single element that can receive Ark props and preserve the tree interaction semantics.

### Default props

| Prop            | Default    | Notes                                               |
| --------------- | ---------- | --------------------------------------------------- |
| `selectionMode` | `"single"` | Use `"multiple"` for multiple selected node values. |
| `expandOnClick` | `true`     | Clicking a branch row expands or collapses it.      |
| `typeahead`     | `true`     | Typeahead uses `nodeToString` from the collection.  |
| `lazyMount`     | `false`    | Passed through to Ark render strategy.              |
| `unmountOnExit` | `false`    | Passed through to Ark render strategy.              |

### Behavioral notes

`expandedValue`, `selectedValue`, `checkedValue`, and `focusedValue` are controlled arrays or strings
using Ark detail callbacks. `loadChildren` returns child nodes asynchronously and
`onLoadChildrenComplete` receives the updated collection. `canRename`, `onRenameStart`,
`onBeforeRename`, and `onRenameComplete` preserve Ark rename detail objects.

Keyboard navigation, roving focus, typeahead, branch expansion, checkbox state, disabled nodes, and
inline rename handling come from Ark UI. Import advanced hooks and generic types such as
`useTreeView`, `useTreeViewNodeContext`, `TreeViewNodeProviderProps`, and
`TreeViewLoadChildrenDetails` from `@moduix/react` when building typed local wrappers around the
recursive node renderer.

Mutation and virtualized rendering stay regular Ark collection patterns: replace the collection after
editing node data, or render the visible recursive nodes inside `TreeView.Tree` without adding a
moduix-specific prop.

## Styling

Use `className` on exported parts for structural styling changes. For state styling, prefer Ark
data attributes and moduix `data-slot` hooks.

### Styling hooks

- `data-slot="tree-view-root"`, `tree-view-root-provider`, `tree-view-label`,
  `tree-view-tree`, `tree-view-branch`, `tree-view-branch-control`,
  `tree-view-branch-trigger`, `tree-view-branch-indicator`, `tree-view-branch-text`,
  `tree-view-branch-content`, `tree-view-branch-indent-guide`, `tree-view-item`,
  `tree-view-item-text`, `tree-view-item-indicator`, `tree-view-node-checkbox`,
  `tree-view-node-checkbox-indicator`, and `tree-view-node-rename-input`.
- Ark state hooks: `data-state="open|closed"`, `data-focus`, `data-selected`,
  `data-disabled`, `data-loading`, `data-renaming`, `data-checked`, `data-indeterminate`,
  `data-depth`, `data-path`, and `data-value`.
- Ark runtime variables: `--depth` for indentation and collapsible `--height` for branch content
  animation.
