## API Reference

`Kbd` is a standalone moduix component and does not wrap a dedicated Ark UI primitive.
Use `Kbd` for a single key and `KbdGroup` when several keys should read as one shortcut.

## Basic

Start with `Kbd` for a single key and `KbdGroup` when the UI needs a semantic shortcut cluster.

## Anatomy

The normal consumer path is short:

```text
KbdGroup
├─ Kbd
├─ +
└─ Kbd
```

| Part       | Element | Data attributes         | Role                                                 |
| ---------- | ------- | ----------------------- | ---------------------------------------------------- |
| `Kbd`      | `kbd`   | `data-slot="kbd-root"`  | Root keycap slot. Applies the shared keycap styling. |
| `KbdGroup` | `kbd`   | `data-slot="kbd-group"` | Inline wrapper for keyboard shortcut combinations.   |

## Composition

### When to stay high-level

Stay with `Kbd` and `KbdGroup` for shortcut hints, command menus, and inline help text.

Both components accept native `kbd` props, including `className`, `title`, and ARIA attributes.
There is no extra variant, size, separator, or state API.

### When to go custom

This component does not have a lower-level composition path. Customization happens through
`className`, the stable `data-slot` hooks, and the public `--kbd-*` CSS variables.

### Behavioral notes

- `Kbd` and `KbdGroup` are static shortcut labels, not interactive controls.
- Separators such as `+`, `/`, or `then` stay in plain composition as normal text.
- Add `aria-label` to `KbdGroup` when abbreviations or symbols need a clearer spoken form.

## Styling

Style `Kbd` and `KbdGroup` through `className`, the stable `data-slot` hooks, and the public
`--kbd-*` CSS variables shown below.

### Styling hooks

- `data-slot="kbd-root"` targets the individual keycap element
- `data-slot="kbd-group"` targets the inline semantic wrapper for grouped shortcuts
- `KbdGroup` inherits text styles for separators, and `--kbd-group-separator-color` controls their color
