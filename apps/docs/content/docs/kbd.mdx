## API Reference

`Kbd` is a standalone moduix component and does not wrap a Base UI primitive.
Because Base UI does not ship a dedicated keyboard-key primitive, moduix keeps this component as a
thin styled wrapper around native `<kbd>` elements. Use `Kbd` for a single key and `KbdGroup`
when several keys should read as one shortcut.

## Anatomy

Use `Kbd` for a single key label. Use `KbdGroup` to keep multi-key shortcuts aligned and preserve
group-level keyboard semantics.

```text
KbdGroup
└─ Kbd
└─ Kbd
```

```tsx
<KbdGroup aria-label="Command K">
  <Kbd>Cmd</Kbd>+<Kbd>K</Kbd>
</KbdGroup>
```

| Part       | Element | Data attributes         | Role                                                 |
| ---------- | ------- | ----------------------- | ---------------------------------------------------- |
| `Kbd`      | `kbd`   | `data-slot="kbd-root"`  | Root keycap slot. Applies the shared keycap styling. |
| `KbdGroup` | `kbd`   | `data-slot="kbd-group"` | Inline wrapper for keyboard shortcut combinations.   |

## Composition

Pass `className` to `Kbd` or `KbdGroup` for targeted overrides. `Kbd` accepts native `kbd` props,
and `KbdGroup` accepts native `kbd` props.

There is no extra variant or size API. For denser layouts or product-specific visuals, use
`className` and the public `--kbd-*` CSS variables.

Separators such as `+` or `/` stay in plain composition. There is no dedicated separator part or
helper prop.

## UX and Accessibility

`Kbd` and `KbdGroup` are static shortcut labels, not interactive controls. Prefer short visible key
labels and add `aria-label` to `KbdGroup` when abbreviations or symbols need a clearer spoken form.
There is no keyboard behavior, focus management, disabled state, or controlled state because the
component only renders native shortcut semantics.
