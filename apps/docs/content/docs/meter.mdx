## Anatomy

`Meter` combines a semantic root, a visible value, and a visual fill track. By default it renders
the track and indicator automatically, so most cases only need `Meter`, `MeterLabel`, and `MeterValue`.

```text
Meter[value]
├─ MeterLabel
├─ MeterValue
└─ MeterTrack
   └─ MeterIndicator
```

```tsx
<Meter value={64}>
  <MeterLabel>Storage used</MeterLabel>
  <MeterValue>{(formattedValue) => `${formattedValue} used`}</MeterValue>
</Meter>
```

| Part             | Role                                                                                           |
| ---------------- | ---------------------------------------------------------------------------------------------- |
| `Meter`          | Root state and accessibility layer. Provides `value`, `min`, `max`, formatting, and ARIA text. |
| `MeterLabel`     | Label associated with the meter.                                                               |
| `MeterValue`     | Formatted value output, with optional custom text rendering.                                   |
| `MeterTrack`     | Visual track behind the fill. Rendered automatically unless `withTrack={false}` is set.        |
| `MeterIndicator` | Fill that reflects the current value within the range.                                         |

## Composition

Keep the built-in track for standard usage. Switch to `withTrack={false}` only when you need to
compose `MeterTrack` and `MeterIndicator` manually; otherwise you'll render a second track.

### Defaults

| Prop        | Default | Values                     |
| ----------- | ------- | -------------------------- |
| `min`       | `0`     | Any `number`               |
| `max`       | `100`   | Any `number`               |
| `withTrack` | `true`  | `true` \| `false`          |
| `value`     | -       | Required `number`          |
| `format`    | -       | `Intl.NumberFormatOptions` |
| `locale`    | runtime | `Intl.LocalesArgument`     |

Use `className` for the root and `classNames` for the built-in `track` and `indicator`.

## Examples

### Min Max Range

Use `min` and `max` for ranges other than `0` to `100`.

<Preview>
  <MinMaxRangeMeterExample />

  <Preview.Code>
    {`

        Meter,

        MeterValue,

          <Meter value={420} min={200} max={800}>
            <MeterLabel>Requests per minute</MeterLabel>
            <MeterValue />
          </Meter>

### Controlled

Keep the value in React state when the meter mirrors another control.

<Preview>
  <ControlledMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";
      import { useState } from "react";

      function ControlledMeter() {
        const [value, setValue] = useState(45);

        return (
          <div className={styles.stack}>
            <Meter value={value}>
              <MeterLabel>Capacity</MeterLabel>
              <MeterValue />
            </Meter>
            <input
              className={styles.range}
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: grid;
        gap: var(--spacing-4);
      }

      .range {
        width: 12rem;
      }
    `}

  </Preview.CSS>
</Preview>

### Percent Format

Use `format` and a custom `MeterValue` child function for user-facing value text.

<Preview>
  <PercentFormatMeterExample />

  <Preview.Code>
    {`

        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function PercentFormatMeterDemo() {
        return (
          <Meter value={0.64} min={0} max={1} format={{ style: "percent", maximumFractionDigits: 0 }}>
            <MeterLabel>Usage</MeterLabel>
            <MeterValue>{(formattedValue) => \`\${formattedValue} used\`}</MeterValue>
          </Meter>
        );
      }
    `}

### Custom Styles

`Meter` renders the track and indicator automatically. Use `className` for the root and `classNames`
for the internal track and indicator slots.

<Preview>
  <CustomStylesMeterExample />

  <Preview.Code>
    {`

        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function CustomStylesMeterDemo() {
        return (
          <Meter
            value={72}
            className={styles.customMeter}
            classNames={{ track: styles.customTrack, indicator: styles.customIndicator }}
          >
            <MeterLabel>Monthly quota</MeterLabel>
            <MeterValue />
          </Meter>
        );
      }
    `}
