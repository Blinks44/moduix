## Anatomy

`Meter` renders the standard visual bar automatically. For the usual case, you only compose the
visible text slots.

```text
Meter[value]
├─ MeterLabel
├─ MeterValue
└─ track
   └─ indicator
```

```tsx
<Meter value={64}>
  <MeterLabel>Storage used</MeterLabel>
  <MeterValue>{(formattedValue) => `${formattedValue} used`}</MeterValue>
</Meter>
```

| Part         | Role                                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------- |
| `Meter`      | Root state and accessibility layer. Provides `value`, `min`, `max`, formatting, and ARIA text. |
| `MeterLabel` | Label associated with the meter.                                                               |
| `MeterValue` | Formatted value output, with optional custom text rendering.                                   |
| `track`      | Default visual rail rendered by `Meter`.                                                       |
| `indicator`  | Default fill rendered inside the track.                                                        |

## Composition

Style the default component with `className` and the public CSS variables. When the meter bar
itself needs separate selectors or a different DOM shape, switch to explicit composition with
`MeterRoot`, `MeterTrack`, and `MeterIndicator`. `MeterLabel` and `MeterValue` keep their own
`className` and `render` props from Base UI.

### Defaults

| Prop     | Default | Values                     |
| -------- | ------- | -------------------------- |
| `min`    | `0`     | Any `number`               |
| `max`    | `100`   | Any `number`               |
| `value`  | -       | Required `number`          |
| `format` | -       | `Intl.NumberFormatOptions` |
| `locale` | runtime | `Intl.LocalesArgument`     |

### Controlled

Keep the value in React state when the meter mirrors another control.

<Preview>
  <ControlledMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";
      import { useState } from "react";

      function ControlledMeter() {
        const [value, setValue] = useState(45);

        return (
          <div className={styles.stack}>
            <Meter value={value}>
              <MeterLabel>Capacity</MeterLabel>
              <MeterValue />
            </Meter>
            <input
              className={styles.range}
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: grid;
        gap: var(--spacing-4);
      }

      .range {
        width: 12rem;
      }
    `}

  </Preview.CSS>
</Preview>

### Percent Format

Use `format` and a custom `MeterValue` child function for user-facing value text.

<Preview>
  <PercentFormatMeterExample />

  <Preview.Code>
    {`

        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function PercentFormatMeterDemo() {
        return (
          <Meter value={0.64} min={0} max={1} format={{ style: "percent", maximumFractionDigits: 0 }}>
            <MeterLabel>Usage</MeterLabel>
            <MeterValue>{(formattedValue) => \`\${formattedValue} used\`}</MeterValue>
          </Meter>
        );
      }
    `}

### Custom Styles

Style the root with `className`, CSS variables, and the built-in `data-slot` hooks when the
default bar needs more specific selectors.

### Custom Composition

Use the same parts when you need to change layout or structure more deeply.
