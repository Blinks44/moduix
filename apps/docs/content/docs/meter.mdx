## Anatomy

`Meter` renders the visual bar automatically, so the default path only needs the visible text slots.

```text
Meter[value]
├─ MeterLabel
├─ MeterValue
└─ track
   └─ indicator
```

```tsx
<Meter value={64}>
  <MeterLabel>Storage used</MeterLabel>
  <MeterValue>{(formattedValue) => `${formattedValue} used`}</MeterValue>
</Meter>
```

| Part         | Role                                                                                           |
| ------------ | ---------------------------------------------------------------------------------------------- |
| `Meter`      | Root state and accessibility layer. Provides `value`, `min`, `max`, formatting, and ARIA text. |
| `MeterLabel` | Label associated with the meter.                                                               |
| `MeterValue` | Formatted value output, with optional custom text rendering.                                   |
| `track`      | Default visual rail rendered by `Meter`.                                                       |
| `indicator`  | Default fill rendered inside the track.                                                        |

For deeper customization, the same parts are also exported as `MeterRoot`, `MeterTrack`, and
`MeterIndicator`.

## Composition

Use `value` with optional `min` and `max` for bounded metrics such as storage usage, quotas, or
health checks. `Meter` always expects a concrete number and does not have the indeterminate/null
state from `Progress`. `locale`, `format`, `aria-valuetext`, and `getAriaValueText` live on the
root and keep the visible value and accessible text aligned.

Style the default component with `className` and the public CSS variables. When the meter bar
itself needs separate selectors or a different DOM shape, switch to explicit composition with
`MeterRoot`, `MeterTrack`, and `MeterIndicator`. `MeterLabel` and `MeterValue` keep their own
`className` and `render` props from Base UI.

### Defaults

| Prop     | Default | Values                     |
| -------- | ------- | -------------------------- |
| `min`    | `0`     | Any `number`               |
| `max`    | `100`   | Any `number`               |
| `value`  | -       | Required `number`          |
| `format` | -       | `Intl.NumberFormatOptions` |
| `locale` | runtime | `Intl.LocalesArgument`     |

### Controlled

Keep the value in React state when the meter mirrors another control.

<Preview>
  <ControlledMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";
      import { useState } from "react";
      import styles from "./meter-demo.module.css";

      function ControlledMeter() {
        const [value, setValue] = useState(45);

        return (
          <div className={styles.stack}>
            <Meter value={value}>
              <MeterLabel>Capacity</MeterLabel>
              <MeterValue />
            </Meter>
            <input
              className={styles.range}
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .stack {
        display: grid;
        gap: var(--spacing-4);
      }

      .range {
        width: 12rem;
      }
    `}

  </Preview.CSS>
</Preview>

### Locale And Format

Use `locale`, `format`, and a custom `MeterValue` child function for localized value text.

<Preview>
  <LocaleAndFormatMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function LocaleAndFormatMeterDemo() {
        return (
          <Meter
            value={0.64}
            min={0}
            max={1}
            locale="de-DE"
            format={{ style: "percent", maximumFractionDigits: 0 }}
          >
            <MeterLabel>Storage usage</MeterLabel>
            <MeterValue>{(formattedValue) => \`\${formattedValue} belegt\`}</MeterValue>
          </Meter>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Aria Value Text

Use `aria-valuetext` when the numeric value needs a clearer accessible label than the formatted
number alone.

<Preview>
  <AriaValueTextMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function AriaValueTextMeterDemo() {
        return (
          <Meter value={3} min={0} max={5} aria-valuetext="3 of 5 service checks are healthy">
            <MeterLabel>Service health</MeterLabel>
            <MeterValue>{() => "3 of 5 healthy"}</MeterValue>
          </Meter>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Custom Value Text

Use `getAriaValueText` when the visible `MeterValue` wording and screen reader wording should stay
aligned but the accessible text needs more context.

<Preview>
  <CustomValueTextMeterExample />

  <Preview.Code>
    {`
      import {
        Meter,
        MeterLabel,
        MeterValue,
      } from "moduix";

      export function CustomValueTextMeterDemo() {
        return (
          <Meter
            value={68}
            getAriaValueText={(formattedValue, value) =>
              \`\${formattedValue} percent of storage used (\${value} of 100)\`
            }
          >
            <MeterLabel>Storage usage</MeterLabel>
            <MeterValue>{(formattedValue) => \`\${formattedValue}% used\`}</MeterValue>
          </Meter>
        );
      }
    `}

  </Preview.Code>
</Preview>
