## Choosing the right component

Use `Progress` when the UI represents ongoing work and the current amount may be unknown.

- Choose **`Progress`** for task completion, uploads, sync jobs, or any workflow that can be indeterminate.
- Choose **`Progress`** when the value is known and you need a display-only bar for usage, health, or quota states.

## Basic

Start with `Progress`, `ProgressLabel`, and `ProgressValue`. This is the recommended path when the
default track and indicator should render after the visible text.

## Anatomy

The normal consumer path is short:

```text
Progress[value]
├─ ProgressLabel
├─ ProgressValue
└─ track
   └─ indicator
```

| Part                | Role                                                                            |
| ------------------- | ------------------------------------------------------------------------------- |
| `Progress`          | High-level root wrapper with formatting, ARIA state, and default bar output.    |
| `ProgressLabel`     | Accessible label associated with the progress root.                             |
| `ProgressValue`     | Formatted value output, with optional custom text rendering.                    |
| `track`             | Default visual rail rendered by `Progress`.                                     |
| `indicator`         | Default fill or indeterminate moving bar rendered inside the track.             |
| `ProgressRoot`      | Low-level root for explicit composition and custom structure.                   |
| `ProgressTrack`     | Explicit track part for custom layouts.                                         |
| `ProgressIndicator` | Explicit indicator part for custom layouts or custom bar-level styling control. |

## Composition

### When to stay high-level

Stay with `Progress` for most status UIs where the default track and indicator belong after the
visible text content.

Use `Progress` or `ProgressRoot` for the public root props: `value`, `min`, `max`, `locale`,
`format`, `aria-valuetext`, and `getAriaValueText`.

### When to go custom

Switch to `ProgressRoot`, `ProgressTrack`, and `ProgressIndicator` when the bar needs a different
DOM position, separate part-level selectors, or custom layout around the default structure.

### Default props

| Prop     | Default | Values                     |
| -------- | ------- | -------------------------- |
| `value`  | `null`  | `number` or `null`         |
| `min`    | `0`     | Any `number`               |
| `max`    | `100`   | Any `number`               |
| `format` | -       | `Intl.NumberFormatOptions` |
| `locale` | runtime | `Intl.LocalesArgument`     |

### Behavioral notes

- `Progress` supports indeterminate work with `value={null}`.
- `Progress` automatically renders `ProgressTrack` and `ProgressIndicator` after `children`.
- Use `ProgressRoot` when the default track placement is wrong for the layout or should be omitted.
- Prefer `ProgressValue` for visible numeric output so formatting stays aligned with the root value.
- Use `aria-valuetext` or `getAriaValueText` when the numeric value needs domain-specific wording
  for assistive technology.

## Examples

### Controlled

Keep the value in React state when progress mirrors another control or task.

<Preview>
  <ControlledProgressExample />

  <Preview.Code>
    {`
      import {
        Progress,
        ProgressLabel,
        ProgressValue,
      } from "moduix";
      import { useState } from "react";

      export function ControlledProgressDemo() {
        const [value, setValue] = useState(45);

        return (
          <div>
            <Progress value={value}>
              <ProgressLabel>Upload status</ProgressLabel>
              <ProgressValue />
            </Progress>
            <input
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Styling hooks

Use the part that owns the visual concern:

- `Progress` or `ProgressRoot` for layout width, gap, and shared text color
- `ProgressLabel` and `ProgressValue` for typography overrides
- `ProgressTrack` for rail background, border, and radius
- `ProgressIndicator` for fill color, transition, and indeterminate animation overrides

The main public hooks are `data-slot="progress-root"`, `data-slot="progress-label"`,
`data-slot="progress-value"`, `data-slot="progress-track"`, and `data-slot="progress-indicator"`.

Base UI also exposes the current status through `data-complete`, `data-progressing`, and
`data-indeterminate` on the relevant parts, which is useful for state-specific styling.
