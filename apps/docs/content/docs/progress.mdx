## Anatomy

`Progress` renders the visual bar automatically, so the default path only needs the visible text slots.

```text
Progress[value]
├─ ProgressLabel
├─ ProgressValue
└─ track
   └─ indicator
```

```tsx
import { Progress, ProgressLabel, ProgressValue } from 'moduix';

export function ProgressDemo() {
  return (
    <Progress value={24}>
      <ProgressLabel>Export data</ProgressLabel>
      <ProgressValue />
    </Progress>
  );
}
```

| Part            | Role                                                                                                     |
| --------------- | -------------------------------------------------------------------------------------------------------- |
| `Progress`      | Root state machine. Controls `value`, `min`, `max`, formatting, and accessible progress attributes.      |
| `ProgressLabel` | Optional text label associated with the progress root.                                                   |
| `ProgressValue` | Optional formatted value text. It can render the default formatted value or use a child render function. |
| `track`         | Default visual rail rendered by `Progress`.                                                              |
| `indicator`     | Default filled bar rendered inside the track.                                                            |

For deeper customization, the same parts are also exported as `ProgressRoot`, `ProgressTrack`, and
`ProgressIndicator`.

## Composition

Use `value={null}` for indeterminate work, or pass a number with optional `min` and `max` for
known progress. `locale`, `format`, `aria-valuetext`, and `getAriaValueText` live on the root and
keep the visible value and accessible text aligned.

Style the default component with `className` and the public CSS variables. When the bar structure
itself needs separate selectors or a different DOM shape, switch to explicit composition with
`ProgressRoot`, `ProgressTrack`, and `ProgressIndicator`. `ProgressLabel` and `ProgressValue` keep
their own `className` and `render` props from Base UI.

### Defaults

| Prop    | Default | Values             |
| ------- | ------- | ------------------ |
| `value` | `null`  | `number` \| `null` |
| `min`   | `0`     | `number`           |
| `max`   | `100`   | `number`           |

## Examples

### Controlled

Keep the value in React state when progress mirrors another control or task.

<Preview>
  <ControlledProgressExample />

  <Preview.Code>
    {`
      import {
        Progress,
        ProgressLabel,
        ProgressValue,
      } from "moduix";
      import { useState } from "react";

      export function ControlledProgressDemo() {
        const [value, setValue] = useState(45);

        return (
          <div>
            <Progress value={value}>
              <ProgressLabel>Upload status</ProgressLabel>
              <ProgressValue />
            </Progress>
            <input
              type="range"
              min={0}
              max={100}
              value={value}
              onChange={(event) => {
                setValue(Number(event.target.value));
              }}
            />
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Locale And Format

Use `locale`, `format`, and a custom `ProgressValue` child function for localized value text.

<Preview>
  <LocaleAndFormatProgressExample />

  <Preview.Code>
    {`
      import {
        Progress,
        ProgressLabel,
        ProgressValue,
      } from "moduix";

      export function LocaleAndFormatProgressDemo() {
        return (
          <Progress value={0.64} min={0} max={1} locale="de-DE" format={{ style: "percent" }}>
            <ProgressLabel>Storage usage</ProgressLabel>
            <ProgressValue>{(formattedValue) => \`\${formattedValue} belegt\`}</ProgressValue>
          </Progress>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Custom Value Text

Use `getAriaValueText` with custom rendered value text when visual and accessible wording should stay aligned.

<Preview>
  <CustomValueTextProgressExample />

### Custom Composition

Use explicit parts when the track or indicator needs its own class names, DOM structure, or custom render behavior.

<Preview>
  <CustomCompositionProgressExample />

  <Preview.Code>
    {`

        ProgressIndicator,
        ProgressLabel,
        ProgressRoot,
        ProgressTrack,
        ProgressValue,
      } from "moduix";
      import styles from "./progress-demo.module.css";

      export function CustomCompositionProgressDemo() {
        return (
          <ProgressRoot value={72} className={styles.customProgress}>
            <ProgressLabel>Monthly export</ProgressLabel>
            <ProgressValue />
            <ProgressTrack className={styles.customTrack}>
              <ProgressIndicator className={styles.customIndicator} />
            </ProgressTrack>
          </ProgressRoot>
        );
      }
    `}
