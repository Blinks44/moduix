## API Reference

<PrimitiveReference href="https://ark-ui.com/docs/components/dialog" />

Ark UI does not ship a dedicated Lightbox primitive. Moduix `Lightbox` is an image-preview recipe
over Ark UI `Dialog`, so Dialog is the behavior source of truth.

## Choosing the right component

Use `Lightbox` when the overlay is primarily about previewing image or media content.

- Choose **`Lightbox`** for zoomed image previews, product image galleries, and media-first modal
  surfaces.
- Choose **`Dialog`** when the surface contains general application UI such as forms, actions, or
  mixed content.
- Compose **`Carousel` inside `Lightbox.Content`** when the preview needs slides or arrows.

## Basic

Compose the explicit Ark UI dialog structure: `Lightbox`, `Lightbox.Trigger`, `Portal`,
`Lightbox.Backdrop`, `Lightbox.Positioner`, and `Lightbox.Content`. Use `asChild` with one
semantic trigger element.

<Preview>
  <LightboxExample />

  <Preview.Code>
    {`
      import { Lightbox, Portal } from "moduix";

      export function LightboxDemo() {
        return (
          <Lightbox>
            <Lightbox.Trigger asChild>
              <button type="button" className="lightbox-trigger">
                <img src={images[0].thumbnail} alt={images[0].alt} />
              </button>
            </Lightbox.Trigger>
            <Portal>
              <Lightbox.Backdrop />
              <Lightbox.Positioner>
                <Lightbox.CloseIcon />
                <Lightbox.Content aria-label={images[0].alt}>
                  <Lightbox.Frame>
                    <img src={images[0].src} alt={images[0].alt} />
                  </Lightbox.Frame>
                </Lightbox.Content>
              </Lightbox.Positioner>
            </Portal>
          </Lightbox>
        );
      }
    `}

## Anatomy

```text
Lightbox.Root
├─ Lightbox.Trigger
└─ Portal
   ├─ Lightbox.Backdrop
   └─ Lightbox.Positioner
      ├─ Lightbox.CloseTrigger or Lightbox.CloseIcon
      └─ Lightbox.Content
         ├─ Lightbox.Title
         ├─ Lightbox.Description
         └─ Lightbox.Frame

Lightbox.RootProvider
└─ the same part tree connected to useLightbox()
```

| Part                         | Role                                                                           |
| ---------------------------- | ------------------------------------------------------------------------------ |
| `Lightbox` / `Lightbox.Root` | Ark Dialog root for open state, focus lifecycle, dismissal, and accessibility. |
| `Lightbox.RootProvider`      | Provider for state created with `useLightbox()`.                               |
| `Lightbox.Trigger`           | Ark trigger. Use `asChild` for custom semantic buttons or links.               |
| `Portal`                     | Renders overlay structure outside the local DOM flow.                          |
| `Lightbox.Backdrop`          | Modal backdrop with Ark `data-state` and layer variables.                      |
| `Lightbox.Positioner`        | Fixed centered overlay container.                                              |
| `Lightbox.Content`           | Actual Ark dialog content node.                                                |
| `Lightbox.Title`             | Optional accessible title.                                                     |
| `Lightbox.Description`       | Optional accessible description.                                               |
| `Lightbox.CloseTrigger`      | Low-level Ark close trigger.                                                   |
| `Lightbox.CloseIcon`         | Moduix close-button sugar over Ark close trigger.                              |
| `Lightbox.Frame`             | Moduix media layout helper.                                                    |
| `Lightbox.Gallery`           | Delegated capture helper for CMS or third-party image markup.                  |
| `Lightbox.Context`           | Inline Ark dialog state reader.                                                |

## Composition

### Recommended composition

Keep the overlay structure explicit. `Lightbox.Content` is the real Ark content part; it does not
render a portal, backdrop, positioner, frame, or close button for you.

Use `Lightbox.Frame` for image/video sizing, or place your own media layout inside
`Lightbox.Content`.

Render `Lightbox.CloseIcon` as a sibling of `Lightbox.Content` inside `Lightbox.Positioner` when
you want it pinned to the viewport corner. Rendering it inside `Lightbox.Content` makes it inherit
content transforms during entry and exit motion.

### Default props

| Prop                     | Default    | Notes                                                               |
| ------------------------ | ---------- | ------------------------------------------------------------------- |
| `closeOnEscape`          | `true`     | Inherited from Ark Dialog.                                          |
| `closeOnInteractOutside` | `true`     | Inherited from Ark Dialog.                                          |
| `defaultOpen`            | `false`    | Uncontrolled initial state.                                         |
| `modal`                  | `true`     | Prevents outside interaction and hides content below the lightbox.  |
| `preventScroll`          | `true`     | Locks page scroll while the lightbox is open.                       |
| `role`                   | `"dialog"` | Use `aria-label`, `Lightbox.Title`, or `ids` for accessible naming. |
| `trapFocus`              | `true`     | Keeps keyboard focus inside the lightbox while open.                |
| `lazyMount`              | `false`    | Use with `unmountOnExit` for heavy media.                           |
| `unmountOnExit`          | `false`    | Removes overlay nodes after exit.                                   |

### State and callbacks

`onOpenChange` and `onTriggerValueChange` keep Ark detail-object callback shapes. Read
`details.open` for controlled open state and `details.value` for multiple-trigger galleries.

Use `useLightbox()` with `Lightbox.RootProvider` when controls live outside the rendered root. Use
`Lightbox.Context` or `useLightboxContext()` to read the current state from child components.

`Lightbox.Frame closeOnClick` is the supported image-preview shortcut when the media itself should
dismiss the overlay. `Lightbox.Gallery` is the delegated capture helper for CMS or third-party
markup you cannot wrap with `Lightbox.Trigger`.

## Examples

### Controlled

Use the Ark `onOpenChange(details)` callback when synchronizing lightbox state with React.

<Preview>
  <ControlledLightboxExample />

  <Preview.Code>
    {`
      import { Lightbox, Portal } from "moduix";
      import { useState } from "react";

      export function ControlledLightboxDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Lightbox open={open} onOpenChange={(details) => setOpen(details.open)}>
            <Lightbox.Trigger className="lightbox-button">
              Open controlled lightbox
            </Lightbox.Trigger>
            <Portal>
              <Lightbox.Backdrop />
              <Lightbox.Positioner>
                <Lightbox.CloseIcon />
                <Lightbox.Content aria-label={images[1].alt}>

                  </Lightbox.Frame>
                </Lightbox.Content>
              </Lightbox.Positioner>
            </Portal>
          </Lightbox>
        );
      }

### Multiple Triggers

Use `Trigger.value` and `onTriggerValueChange(details)` for gallery thumbnails. The component does
not maintain a hidden image registry.

<Preview>
  <GalleryLightboxExample />

  <Preview.Code>
    {`
      import { Lightbox, Portal } from "moduix";
      import { useState } from "react";

      export function GalleryLightboxDemo() {
        const [activeImage, setActiveImage] = useState(images[0]);

        return (
          <Lightbox
            onTriggerValueChange={(details) => {
              setActiveImage(images.find((image) => image.id === details.value) ?? images[0]);
            }}
          >
            <div className="lightbox-gallery">
              {images.map((image) => (
                <Lightbox.Trigger key={image.id} value={image.id} asChild>
                  <button type="button" className="lightbox-gallery-trigger">
                    <img src={image.thumbnail} alt={image.alt} />
                  </button>
                </Lightbox.Trigger>
              ))}
            </div>
            <Portal>
              <Lightbox.Backdrop />
              <Lightbox.Positioner>
                <Lightbox.CloseIcon />
                <Lightbox.Content aria-label={activeImage.alt}>
                  <Lightbox.Frame>
                    <img src={activeImage.src} alt={activeImage.alt} />
                  </Lightbox.Frame>
                </Lightbox.Content>
              </Lightbox.Positioner>
            </Portal>
          </Lightbox>
        );
      }
    `}

  </Preview.Code>

### Delegated CMS Capture

Use `Lightbox.Gallery` when previewable images come from CMS or third-party markup that you do not
render directly with `Lightbox.Trigger`. Point it at a container with `rootRef` or `rootSelector`,
choose a `selector`, and it will read `data-lightbox-src`, then `currentSrc`, then `src`.

<Preview>
  <DelegatedLightboxExample />

  <Preview.Code>
    {`
      import { Lightbox } from "moduix";
      import { useRef } from "react";

      export function CmsLightboxDemo() {
        const rootRef = useRef<HTMLDivElement | null>(null);

        return (
          <>
            <div ref={rootRef} className="lightbox-gallery">
              {images.map((image) => (
                <button key={image.id} type="button" className="lightbox-gallery-trigger">
                  <img src={image.thumbnail} data-lightbox-src={image.src} alt={image.alt} />
                </button>
              ))}
            </div>

            <Lightbox.Gallery rootRef={rootRef} selector="button" />
          </>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{lightboxExampleCss}</Preview.CSS>
  <Preview.Data>{lightboxImagesData}</Preview.Data>
</Preview>

### Root Provider and Context

Create state with `useLightbox()` when the opener is outside the root. Read inline state with
`Lightbox.Context`.

<Preview>
  <RootProviderLightboxExample />

  <Preview.Code>
    {`
      import { Lightbox, Portal, useLightbox } from "moduix";

      export function LightboxStateDemo() {
        const lightbox = useLightbox();

        return (
          <>
            <button type="button" onClick={() => lightbox.setOpen(true)}>
              Lightbox is {lightbox.open ? "open" : "closed"}
            </button>
            <Lightbox.RootProvider value={lightbox}>
              <Portal>
                <Lightbox.Backdrop />
                <Lightbox.Positioner>
                  <Lightbox.CloseIcon />
                  <Lightbox.Content aria-label={images[2].alt}>
                    <Lightbox.Frame>
                      <img src={images[2].src} alt={images[2].alt} />
                    </Lightbox.Frame>
                    <Lightbox.Context>
                      {(state) => <>Preview is {state.open ? "open" : "closed"}</>}
                    </Lightbox.Context>
                  </Lightbox.Content>
                </Lightbox.Positioner>
              </Portal>
            </Lightbox.RootProvider>
          </>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{lightboxExampleCss}</Preview.CSS>

`unmountOnExit` removes the overlay after close animations finish.

### Styling hooks

- Moduix slots: `lightbox-trigger`, `lightbox-backdrop`, `lightbox-positioner`,
  `lightbox-content`, `lightbox-title`, `lightbox-description`, `lightbox-close-trigger`,
  `lightbox-close-icon`, and `lightbox-frame`.
- Ark state and anatomy hooks: `data-scope="dialog"`, `data-part`, `data-state="open|closed"`,
  `data-nested`, `data-has-nested`, `--layer-index`, and `--nested-layer-count`.
- Motion is driven by Ark `data-state`; use `present` only when JavaScript animation control needs
  to keep the content mounted.
