## Anatomy

`LightboxContent` already renders the default `portal`, `backdrop`, `viewport`, `popup`, frame,
and close button, so most usage stays at `Lightbox`, `LightboxImage`, and `LightboxContent`.

| Part              | Role                                                                                                   |
| ----------------- | ------------------------------------------------------------------------------------------------------ |
| `Lightbox`        | Root state and behavior (`open`, `defaultOpen`, `onOpenChange`, `modal`, `handle`).                    |
| `LightboxTrigger` | Opens the preview from any custom trigger composition.                                                 |
| `LightboxImage`   | Trigger helper for the common thumbnail-to-preview flow. `previewSrc` can point to a larger asset.     |
| `LightboxContent` | Default high-level wrapper that owns `portal`, `backdrop`, `viewport`, popup, frame, and close button. |
| `LightboxClose`   | Primitive close part for custom composition inside the popup.                                          |
| `LightboxGallery` | Dynamic image capture mode for CMS-rendered or unknown image lists.                                    |

## Composition

Use the default path when one trigger image opens one preview image.

Use controlled state on `Lightbox` when outside UI needs to open or close the preview.

Use `LightboxTrigger` when the trigger is not an image.

Use `LightboxGallery` when images come from CMS or HTML you do not compose manually. It reads
`data-lightbox-src` first, then falls back to `currentSrc` or `src`.

`LightboxContent` keeps the default path small and supports the common image-preview workflow:

### Defaults

| Part              | Prop                  | Default       | Values / Notes                                   |
| ----------------- | --------------------- | ------------- | ------------------------------------------------ |
| `LightboxContent` | `showCloseButton`     | `true`        | `true`, `false`                                  |
| `LightboxContent` | `closeOnContentClick` | `true`        | `true`, `false`                                  |
| `LightboxContent` | `closeLabel`          | `Close image` | Accessible label for close controls              |
| `LightboxGallery` | `selector`            | `img`         | Any selector supported by `Element.closest(...)` |

When the default wrapper is not enough, assemble the lower-level parts manually.

                src={images.forest}
                data-lightbox-src={images.forestLarge}
                alt="Forest and mountain road"
                className={styles.dynamicImage}
              />
            </div>
            <LightboxGallery rootRef={rootRef} />
          </Fragment>
        );
      }

### Customized Surface

Customize popup size, backdrop tone, and close controls by composing the lower-level parts directly.

<Preview>
  <CustomizedLightboxExample />

  <Preview.Code>
    {`
      import {
        Lightbox,
        LightboxBackdrop,
        LightboxClose,
        LightboxCloseButton,
        LightboxFrame,
        LightboxImage,
        LightboxPopup,
        LightboxPortal,
        LightboxViewport,
      } from "moduix";

      export function CustomizedLightboxDemo() {
        return (
          <Lightbox>
            <LightboxImage src={thumbnail} alt="Road through forest" className={styles.previewImage} />
            <LightboxPortal>
              <LightboxBackdrop className={styles.customBackdrop} />
              <LightboxViewport>
                <LightboxCloseButton className={styles.customClose} aria-label="Close preview" />
                <LightboxPopup className={styles.customPopup}>
                  <LightboxFrame>
                    <LightboxClose aria-label="Close preview" nativeButton={false} render={<div />}>
                      <img src={fullSize} alt="Road through forest" className={styles.contentImage} />
                    </LightboxClose>
                  </LightboxFrame>
                </LightboxPopup>
              </LightboxViewport>
            </LightboxPortal>
          </Lightbox>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Custom Composition

Use the low-level parts when you need a different portal target, no backdrop, custom layout around
the image, or a fully custom close affordance.

```tsx
<Lightbox>
  <LightboxTrigger render={<button type="button">Open image</button>} />
  <LightboxPortal>
    <LightboxBackdrop />
    <LightboxViewport>
      <LightboxCloseButton />
      <LightboxPopup className={styles.popup}>
        <LightboxFrame>
          <LightboxClose aria-label="Close image" nativeButton={false} render={<div />}>
            <img src={fullSize} alt="Preview image" />
          </LightboxClose>
        </LightboxFrame>
      </LightboxPopup>
    </LightboxViewport>
  </LightboxPortal>
</Lightbox>
```
