## Anatomy

`LightboxContent` already renders `portal`, `backdrop`, `viewport`, and the image frame.
Most usage stays at `Lightbox`, `LightboxImage` or `LightboxTrigger`, and `LightboxContent`.

```text
Lightbox
├─ LightboxTrigger (optional)
│  └─ LightboxImage (optional helper)
└─ LightboxContent
   ├─ portal (service slot)
   │  ├─ backdrop (service slot, optional)
   │  └─ viewport (service slot)
   │     └─ popup
   │        └─ frame
   │           ├─ LightboxClose (optional)
   │           └─ image content
```

```tsx
<Lightbox>
  <LightboxImage src={thumbnail} alt="Preview image" />
  <LightboxContent>
    <img src={fullSize} alt="Preview image" />
  </LightboxContent>
</Lightbox>
```

| Part              | Role                                                                                |
| ----------------- | ----------------------------------------------------------------------------------- |
| `Lightbox`        | Root state and behavior (`open`, `defaultOpen`, `onOpenChange`, `modal`, `handle`). |
| `LightboxTrigger` | Opens the preview from any custom trigger composition.                              |
| `LightboxImage`   | Helper that wraps an image as a trigger and supports `previewSrc` mapping.          |
| `LightboxContent` | Modal content slot that owns `portal`, `backdrop`, `viewport`, popup, and frame.    |
| `LightboxClose`   | Optional close control rendered in the frame area.                                  |
| `LightboxGallery` | Dynamic image capture mode for CMS-rendered or unknown image lists.                 |

## Composition

Use controlled state on `Lightbox` when outside UI needs to open or close the preview.
Use `className` for the popup, `classNames` for service slots, and `slotProps` for non-class slot props.
`container` overrides `slotProps.portal.container`.
Use `withCloseButton` for the built-in top-right close control, or pass `closeButton` to replace its rendered button while keeping the same dismissal behavior.

`LightboxGallery` listens for image clicks inside `rootRef` or `rootSelector` and opens the clicked source.
Keep `withBackdrop` enabled for modal previews; disable it only when the page should stay interactive behind the image.

### Defaults

| Part              | Prop                  | Default       | Values / Notes                                                 |
| ----------------- | --------------------- | ------------- | -------------------------------------------------------------- |
| `LightboxContent` | `withBackdrop`        | `true`        | `true`, `false`                                                |
| `LightboxContent` | `withCloseButton`     | `true`        | `true`, `false`                                                |
| `LightboxContent` | `closeButton`         | `undefined`   | Rendered element used for the built-in top-right close control |
| `LightboxContent` | `closeOnContentClick` | `true`        | `true`, `false`                                                |
| `LightboxContent` | `closeLabel`          | `Close image` | Accessible label for close controls                            |
| `LightboxGallery` | `selector`            | `img`         | Any selector supported by `Element.closest(...)`               |
| `LightboxGallery` | `withBackdrop`        | `true`        | Passed through to internal `LightboxContent`                   |

## Examples

### Dynamic Gallery Capture

Use `LightboxGallery` when images come from CMS or HTML you do not compose manually.

<Preview>
  <DynamicLightboxGalleryExample />

  <Preview.Code>
    {`
      import { LightboxGallery } from "moduix";
      import { Fragment, useRef } from "react";

      export function DynamicGalleryDemo() {
        const rootRef = useRef<HTMLDivElement | null>(null);

        return (
          <Fragment>
            <div ref={rootRef} className={styles.dynamicRoot}>
              <img src={images.mountain} alt="Mountain landscape" className={styles.dynamicImage} />
              <img src={images.sea} alt="Sea at sunset" className={styles.dynamicImage} />
              <img src={images.forest} alt="Forest and mountain road" className={styles.dynamicImage} />
            </div>
            <LightboxGallery rootRef={rootRef} />
          </Fragment>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Customized Surface

Customize popup size, backdrop tone, and the built-in close button with `className`, `classNames`, and CSS variables.

<Preview>
  <CustomizedLightboxExample />

  <Preview.Code>
    {`
      import { CloseButton, Lightbox, LightboxImage, LightboxContent } from "moduix";

      export function CustomizedLightboxDemo() {
        return (
          <Lightbox>
            <LightboxImage src={thumbnail} alt="Road through forest" className={styles.previewImage} />
            <LightboxContent
              className={styles.customPopup}
              classNames={{ backdrop: styles.customBackdrop }}
              closeButton={<CloseButton aria-label="Close preview" className={styles.customClose} />}
            >
              <img src={fullSize} alt="Road through forest" className={styles.contentImage} />
            </LightboxContent>
          </Lightbox>
        );
      }
    `}

  </Preview.Code>
</Preview>
