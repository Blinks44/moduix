`LightboxContent` stays short by default: when `children` are omitted, it reuses the image data from
`LightboxImage` and opens `fullSrc` if you provide one. When `fullSrc` exists, `LightboxImage`
preloads it on pointer hover and keyboard focus to reduce empty-overlay waits before open.

Use this path for the common single-image preview flow. If the popup content contains controls,
captions, or anything beyond the image itself, pass explicit `children` to `LightboxContent` or
compose the low-level parts directly.

## Anatomy

`LightboxContent` already renders the default `portal`, `backdrop`, `viewport`, `popup`, frame,
and close button, so most usage stays at `Lightbox`, `LightboxImage`, and `LightboxContent`.

```text
Lightbox
├─ LightboxTrigger
│  └─ LightboxImage (optional helper)
└─ LightboxContent
   ├─ portal
   │  ├─ backdrop
   │  └─ viewport
   │     ├─ LightboxCloseButton (optional)
   │     └─ popup
   │        └─ LightboxFrame
   │           └─ image content
```

```tsx
<Lightbox>
  <LightboxImage src={thumbnail} fullSrc={fullSize} alt="Preview image" />
  <LightboxContent />
</Lightbox>
```

| Part              | Role                                                                                                                            |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `Lightbox`        | Root state and behavior (`open`, `defaultOpen`, `onOpenChange`, `modal`, `handle`).                                             |
| `LightboxTrigger` | Opens the preview from any custom trigger composition.                                                                          |
| `LightboxImage`   | Trigger helper for the common thumbnail-to-preview flow. `fullSrc` can point to a larger asset and is preloaded on hover/focus. |
| `LightboxContent` | Default high-level wrapper that owns `portal`, `backdrop`, `viewport`, popup, frame, and close button.                          |
| `LightboxClose`   | Primitive close part for custom composition inside the popup.                                                                   |
| `LightboxGallery` | Dynamic image capture mode for CMS-rendered or unknown image lists. It also preloads resolved image sources on hover/focus.     |

## Composition

Use the default path when one trigger image opens one preview image.

Use controlled state on `Lightbox` when outside UI needs to open or close the preview.

Use `LightboxTrigger` when the trigger is not an image.

Use `LightboxGallery` when images come from CMS or HTML you do not compose manually. It reads
`data-lightbox-src` first, then falls back to `currentSrc` or `src`, and it can target a focusable
wrapper with `selector` when the image is nested inside a button, card, or link-like control. Those
resolved sources are also preloaded on pointer hover and keyboard focus.

`LightboxContent` keeps the default path small and supports the common image-preview workflow:

### Defaults

| Part              | Prop                  | Default       | Values / Notes                                   |
| ----------------- | --------------------- | ------------- | ------------------------------------------------ |
| `LightboxContent` | `showCloseButton`     | `true`        | `true`, `false`                                  |
| `LightboxContent` | `closeOnContentClick` | `true`        | `true`, `false`                                  |
| `LightboxContent` | `closeLabel`          | `Close image` | Accessible label for close controls              |
| `LightboxGallery` | `selector`            | `img`         | Any selector supported by `Element.closest(...)` |

When `closeOnContentClick` is enabled, clicking anywhere inside the content area closes the
lightbox. Disable it for interactive children such as buttons, links, or media controls.

## Styling

Style the visual parts through `className`, `data-slot="lightbox-*"` hooks, and the documented
`--lightbox-*` CSS variables in the CSS Properties panel.

Most common hooks:

- `LightboxImage` and `LightboxTrigger` for preview triggers.
- `LightboxContent` `className` for the popup surface.
- `LightboxBackdrop`, `LightboxViewport`, `LightboxPopup`, `LightboxFrame`, and
  `LightboxCloseButton` for advanced composition.

When the default wrapper is not enough, assemble the lower-level parts manually.

### Customized Surface

Customize popup size, backdrop tone, and close controls by composing the lower-level parts directly.

<Preview>
  <CustomizedLightboxExample />

  <Preview.Code>
    {`
      import {
        Lightbox,
        LightboxBackdrop,
        LightboxClose,
        LightboxCloseButton,
        LightboxFrame,
        LightboxImage,
        LightboxPopup,
        LightboxPortal,
        LightboxViewport,
      } from "moduix";

      export function CustomizedLightboxDemo() {
        return (
          <Lightbox>
            <LightboxImage src={thumbnail} alt="Road through forest" className={styles.previewImage} />
            <LightboxPortal>
              <LightboxBackdrop className={styles.customBackdrop} />
              <LightboxViewport>
                <LightboxCloseButton className={styles.customClose} aria-label="Close preview" />
                <LightboxPopup className={styles.customPopup}>
                  <LightboxFrame>
                    <LightboxClose nativeButton={false} render={<div />}>
                      <img src={fullSize} alt="Road through forest" className={styles.contentImage} />
                    </LightboxClose>
                  </LightboxFrame>
                </LightboxPopup>
              </LightboxViewport>
            </LightboxPortal>
          </Lightbox>
        );
      }
    `}

## Custom Composition

Use the low-level parts when you need a different portal target, no backdrop, custom layout around
the image, or a fully custom close affordance.

```tsx
<Lightbox>
  <LightboxTrigger render={<button type="button">Open image</button>} />
  <LightboxPortal>
    <LightboxBackdrop />
    <LightboxViewport>
      <LightboxCloseButton />
      <LightboxPopup className={styles.popup}>
        <LightboxFrame>
          <LightboxClose nativeButton={false} render={<div />}>
            <img src={fullSize} alt="Preview image" />
          </LightboxClose>
        </LightboxFrame>
      </LightboxPopup>
    </LightboxViewport>
  </LightboxPortal>
</Lightbox>
```
