## API Reference

<PrimitiveReference href="https://ark-ui.com/docs/components/dialog" />

Ark UI does not ship a dedicated `lightbox` primitive. Moduix builds this component on Ark UI
`Dialog`, follows Ark's [composition](https://ark-ui.com/docs/guides/composition),
[styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance, and keeps Dialog as the behavior source of
truth.

## Choosing the right component

Use `Lightbox` when the overlay is primarily about previewing image or media content.

- Choose **`Lightbox`** for zoomed image previews, product image galleries, and media-first modal
  surfaces.
- Choose **`Dialog`** when the surface contains general application UI such as forms, actions, or
  mixed content.
- Compose **`Carousel` inside `Lightbox.Gallery`** when the preview needs slides or arrows.

## Anatomy

```text
Lightbox.Root
├─ Lightbox.Trigger
├─ Lightbox.Bind
└─ Overlay subtree (automatically portalled)
   ├─ Lightbox.Backdrop
   └─ Lightbox.Positioner
      ├─ Lightbox.CloseTrigger or Lightbox.CloseIcon
      └─ Lightbox.Content
         ├─ Lightbox.Title
         ├─ Lightbox.Description
         └─ Lightbox.Image

Lightbox.RootProvider
└─ the same part tree connected to Ark useDialog()
```

| Part                         | Role                                                                           |
| ---------------------------- | ------------------------------------------------------------------------------ |
| `Lightbox` / `Lightbox.Root` | Ark Dialog root for open state, focus lifecycle, dismissal, and accessibility. |
| `Lightbox.RootProvider`      | Provider for state created with Ark `useDialog()`.                             |
| `Lightbox.Trigger`           | Ark trigger. Use `asChild` for custom semantic buttons or links.               |
| `Lightbox.Backdrop`          | Modal backdrop with Ark `data-state` and layer variables.                      |
| `Lightbox.Positioner`        | Fixed centered overlay container.                                              |
| `Lightbox.Content`           | Actual Ark dialog content node.                                                |
| `Lightbox.Title`             | Optional accessible title.                                                     |
| `Lightbox.Description`       | Optional accessible description.                                               |
| `Lightbox.CloseTrigger`      | Low-level Ark close trigger.                                                   |
| `Lightbox.CloseIcon`         | Moduix `CloseButton.Root` sugar over Ark close trigger.                        |
| `Lightbox.Image`             | Styled native image with an `HTMLImageElement` ref.                            |
| `Lightbox.Gallery`           | Layout and styling boundary for a composed `Carousel`.                         |
| `Lightbox.Bind`              | Zero-render behavior part that binds external image markup to the root.        |

## Composition

The root portals `Backdrop` and `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Keep the overlay structure explicit. `Lightbox.Content` is the real Ark content part; it does not
render a backdrop, positioner, image, gallery, or close button for you.

Use `Lightbox.Image` for a single image. It renders a native `<img>`, accepts native image props,
and forwards its ref to the `HTMLImageElement`. Place custom media layouts directly inside
`Lightbox.Content`.

Use `Lightbox.Gallery` around `Carousel.Root` for a known image collection. It provides the
lightbox-specific viewport, arrow alignment, media sizing, and indicator layout while Carousel
continues to own all slide behavior.

Render `Lightbox.CloseIcon` as a sibling of `Lightbox.Content` inside `Lightbox.Positioner` when
you want it pinned to the viewport corner. Rendering it inside `Lightbox.Content` makes it inherit
content transforms during entry and exit motion.

### When to go custom

Render custom content directly inside `Lightbox.Content` when it is not a single media frame or a
Carousel gallery. Use `Lightbox.Bind` only when the source markup cannot render
`Lightbox.Trigger`; structured arrays should use explicit triggers and `Lightbox.Gallery`.

### Default props

| Prop                     | Default    | Notes                                                               |
| ------------------------ | ---------- | ------------------------------------------------------------------- |
| `closeOnEscape`          | `true`     | Inherited from Ark Dialog.                                          |
| `closeOnInteractOutside` | `true`     | Inherited from Ark Dialog.                                          |
| `defaultOpen`            | `false`    | Uncontrolled initial state.                                         |
| `modal`                  | `true`     | Prevents outside interaction and hides content below the lightbox.  |
| `preventScroll`          | `true`     | Locks page scroll while the lightbox is open.                       |
| `role`                   | `"dialog"` | Use `aria-label`, `Lightbox.Title`, or `ids` for accessible naming. |
| `trapFocus`              | `true`     | Keeps keyboard focus inside the lightbox while open.                |
| `lazyMount`              | `false`    | Use with `unmountOnExit` for heavy media.                           |
| `unmountOnExit`          | `false`    | Removes overlay nodes after exit.                                   |

### Behavioral notes

`onOpenChange` and `onTriggerValueChange` keep Ark detail-object callback shapes. Read
`details.open` for controlled open state and `details.value` for multiple-trigger galleries.

Use Ark `useDialog()` with `Lightbox.RootProvider` when controls live outside the rendered root.
Read dialog state in child components with Ark `useDialogContext()`.

`Lightbox.Image closeOnClick` is the supported image-preview shortcut when the image itself should
dismiss the overlay. `Lightbox.Bind` is the zero-render binding helper for CMS or third-party
markup you cannot wrap with `Lightbox.Trigger`. It must be rendered inside `Lightbox` and reports
the selected image through `onImageSelect(details)`. Details contain `src`, optional `alt`, and
the source `HTMLImageElement` as `element`.

Other inherited Dialog props pass through unchanged, including `id`, `ids`, `initialFocusEl`,
`finalFocusEl`, `persistentElements`, `restoreFocus`, `present`, `immediate`,
`skipAnimationOnMount`, and outside-interaction callbacks.

### Bind CMS Content

Use `Lightbox.Bind` when previewable images come from CMS or third-party markup that you do not
render directly with `Lightbox.Trigger`. Point it at a container with `rootRef` or `rootSelector`,
choose a `selector`, and handle the resolved `{ src, alt, element }` through `onImageSelect`.
`Bind` opens its parent Lightbox but renders no overlay structure, so backdrop, content, close
controls, and image layout remain fully composable. It reads `data-lightbox-src`, then `currentSrc`,
then `src`, and preloads that source on pointer hover or keyboard focus. Use a semantic button or
link as the matched `selector` so native keyboard activation produces the click that opens the
Lightbox. If your server already returns a real image array, prefer the
`Lightbox.Gallery + Carousel` composition above.

### Styling hooks

- Moduix slots: `lightbox-trigger`, `lightbox-backdrop`, `lightbox-positioner`,
  `lightbox-content`, `lightbox-title`, `lightbox-description`, `lightbox-close-trigger`,
  `lightbox-close-icon`, `lightbox-image`, and `lightbox-gallery`.
- Ark state and anatomy hooks: `data-scope="dialog"`, `data-part`, `data-state="open|closed"`,
  `data-nested`, `data-has-nested`, `--layer-index`, and `--nested-layer-count`.
- Local hooks: `Lightbox.Image` exposes `data-slot="lightbox-image"` and optional
  `data-close-on-click`; `Lightbox.Gallery` exposes `data-slot="lightbox-gallery"` and styles
  nested Carousel slots.
- Motion is driven by Ark `data-state`; use `present` only when JavaScript animation control needs
  to keep the content mounted.
