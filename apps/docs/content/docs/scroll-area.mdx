## Basic

Start with `ScrollArea` when content needs a bounded viewport but should still scroll natively. The
wrapper renders the root, viewport, content, and the default vertical scrollbar for you.

## Anatomy

The normal consumer path is short:

```text
ScrollArea
├─ ScrollAreaViewport
│  └─ ScrollAreaContent
│     └─ children
├─ ScrollAreaScrollbar (vertical by default)
│  └─ ScrollAreaThumb
└─ ScrollAreaCorner (only when both scrollbars are rendered)
```

| Part                  | Role                                                         |
| --------------------- | ------------------------------------------------------------ |
| `ScrollArea`          | High-level wrapper with default viewport, content, and bars. |
| `ScrollAreaRoot`      | Root scroll area container.                                  |
| `ScrollAreaViewport`  | Focusable native scroll viewport.                            |
| `ScrollAreaContent`   | Inner content wrapper.                                       |
| `ScrollAreaScrollbar` | One scrollbar track.                                         |
| `ScrollAreaThumb`     | Draggable scrollbar thumb.                                   |
| `ScrollAreaCorner`    | Fills the corner when both scrollbars are visible.           |

## Composition

### When to stay high-level

Stay with `ScrollArea` for most panels, drawers, settings sections, and dialog bodies. It keeps
the underlying primitive root contract intact and adds two narrow conveniences on top:

- `scrollbars` controls which default scrollbars are rendered
- `fade` adds the root `data-fade` contract that drives the bundled edge-mask styles

Use `className` on `ScrollArea` for the outer size, border, and local CSS variables. Root props
such as `overflowEdgeThreshold`, `dir`, `render`, and other primitive root behavior are forwarded
unchanged.

### When to go custom

Switch to `ScrollAreaRoot`, `ScrollAreaViewport`, `ScrollAreaContent`, `ScrollAreaScrollbar`,
`ScrollAreaThumb`, and `ScrollAreaCorner` when you need to own the structure directly:

- different classes or props on viewport and content
- manual scrollbar props such as `keepMounted`
- different styling per scrollbar axis or a custom corner
- manual `data-fade` values or other root-level data hooks in JSX

### Default props

| Prop         | Default    | Values                                            |
| ------------ | ---------- | ------------------------------------------------- |
| `scrollbars` | `vertical` | `vertical`, `horizontal`, `both`, `false`         |
| `fade`       | `false`    | `false`, `true`, `vertical`, `horizontal`, `both` |

`fade={true}` maps to a vertical edge fade, which matches the most common single-column case.

### Behavioral notes

- `scrollbars={false}` removes the default custom scrollbars but keeps the native scroll viewport.
- The wrapper renders `ScrollAreaCorner` only when both default scrollbars are present.
- The ref on `ScrollArea` points to the underlying root element.
- Horizontal fade direction respects RTL automatically through the root `dir` setting.

## Custom Composition

Use the exported parts directly when the default wrapper stops being enough.

<Preview>
  <CustomCompositionScrollAreaExample />

  <Preview.Code>
    {`
      import {
        ScrollAreaContent,
        ScrollAreaCorner,
        ScrollAreaRoot,
        ScrollAreaScrollbar,
        ScrollAreaThumb,
        ScrollAreaViewport,
      } from "moduix";

      export function CustomCompositionScrollAreaDemo() {
        return (
          <ScrollAreaRoot
            className="customRoot"
            data-fade="both"
            overflowEdgeThreshold={28}
          >
            <ScrollAreaViewport className="customViewport">
              <ScrollAreaContent className="customContent">
                {Array.from({ length: 80 }, (_, index) => (
                  <div key={index} className="customCell">
                    {index + 1}
                  </div>
                ))}
              </ScrollAreaContent>
            </ScrollAreaViewport>
            <ScrollAreaScrollbar className="customVerticalScrollbar" keepMounted>
              <ScrollAreaThumb className="customVerticalThumb" />
            </ScrollAreaScrollbar>
            <ScrollAreaScrollbar
              orientation="horizontal"
              className="customHorizontalScrollbar"
            >
              <ScrollAreaThumb className="customHorizontalThumb" />
            </ScrollAreaScrollbar>
            <ScrollAreaCorner className="customCorner" />
          </ScrollAreaRoot>
        );
      }
    `}

  </Preview.Code>

### Styling hooks

- `data-slot="scroll-area-root"`, `scroll-area-viewport`, `scroll-area-content`,
  `scroll-area-scrollbar`, `scroll-area-thumb`, and `scroll-area-corner` are the stable part hooks.
- `data-fade` on the root drives the built-in viewport edge masks. The high-level `fade` prop sets
  this automatically, or you can set it manually on `ScrollAreaRoot`.
- `data-orientation="vertical"` and `data-orientation="horizontal"` are exposed on scrollbars and
  thumbs for per-axis styling.
- Primitive state attributes such as `data-scrolling` and `data-hovering` are available on the
  relevant parts for interaction styling.
