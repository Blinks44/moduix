Use `Separator` for semantic horizontal or vertical division. Do not use it as a spacing helper
when empty layout space already expresses the relationship.

## Anatomy

`Separator` exposes one root part.

```text
Separator
└─ root[data-slot="separator-root"][data-orientation]
```

```tsx
<Separator orientation="horizontal" />
```

| Part        | Role                                                                                                   |
| ----------- | ------------------------------------------------------------------------------------------------------ |
| `Separator` | Root divider element. Controls `orientation`, accessibility attributes, styling, and element override. |

Style the root directly with `className` or the `--separator-*` CSS properties.

## Composition

### When to stay high-level

Stay with `Separator` for most divider cases. The moduix wrapper keeps the Base UI separator
contract intact and adds the shared styling hooks and CSS variables.

Pass Base UI props such as `orientation`, `className`, `style`, `render`, and ARIA attributes
directly to `Separator`.

### When to go custom

There is no lower-level moduix composition tree for this component. When the divider needs custom
semantics or more surrounding structure, compose that structure outside `Separator` and keep the
separator itself focused on visual separation.

### Default props

| Prop          | Default      | Values                                 |
| ------------- | ------------ | -------------------------------------- |
| `orientation` | `horizontal` | `horizontal`, `vertical`               |
| `className`   | `undefined`  | `string` or state-based class function |
| `style`       | `undefined`  | style object or state-based style fn   |

### Behavioral notes

- Horizontal separators fill the available width by default.
- Vertical separators default to `1em` height so they align naturally with inline text and compact
  action rows.
- `Separator` is not interactive and does not expose disabled, pressed, or focus-management
  behavior.
- Use `orientation="vertical"` only inside layouts that already control cross-axis alignment, such
  as `flex` or `inline-flex` rows.

Use `className` or `--separator-*` custom properties when the divider needs stronger visual weight
or a fixed length.

## Styling

`Separator` exposes one stable root selector and a small `--separator-*` CSS contract for color,
thickness, and orientation-specific length.

### CSS Properties

### Styling hooks

- `data-slot="separator-root"` is the stable selector for the exported root.
- `data-orientation="horizontal"` and `data-orientation="vertical"` let you target axis-specific
  styling.
- `className` and `style` are applied directly to the root separator element.
