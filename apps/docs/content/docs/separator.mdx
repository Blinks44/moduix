## API Reference

Ark UI does not ship a dedicated separator primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Anatomy

`Separator` exposes one root part.

```text
Separator / Separator.Root
└─ root[data-scope="separator"][data-part="root"][data-slot="separator-root"]
```

| Part                           | Role                                                                                   |
| ------------------------------ | -------------------------------------------------------------------------------------- |
| `Separator` / `Separator.Root` | Root divider element. Controls orientation, accessibility metadata, and visual recipe. |

Style the root directly with `className`, `asChild`, or the `--separator-*` CSS properties.

## Composition

### Recommended composition

Use the short `<Separator />` form for most dividers. It is equivalent to
`<Separator.Root />` and keeps the public root form consistent with multipart Ark-style
components.

### When to go custom

Use `asChild` when the host must be a semantic element such as `hr`. The child must be one element
that can receive the merged role, ARIA, data attributes, class name, and ref.

### Default props

| Prop          | Default      | Notes                                  |
| ------------- | ------------ | -------------------------------------- |
| `orientation` | `horizontal` | `horizontal`, `vertical`               |
| `variant`     | `solid`      | `solid`, `dashed`, `dotted`            |
| `size`        | `sm`         | `xs`, `sm`, `md`, `lg`                 |
| `asChild`     | `false`      | Ark factory single-child composition   |
| `role`        | `separator`  | Pass `presentation` for decorative use |

### Behavioral notes

- `Separator` writes `aria-orientation` when its role is `separator`.
- Horizontal separators fill the available width by default.
- Vertical separators default to `1em` height so they align naturally with inline text and compact
  action rows.
- `Separator` is not interactive and does not expose state, callbacks, disabled behavior, or focus
  management.
- Use layout primitives or spacing tokens for whitespace-only gaps.

## Examples

### Vertical

Use `orientation="vertical"` for inline groups such as navigation actions.

<Preview>
  <VerticalSeparatorExample />

## Styling

`Separator` exposes Ark-style part selectors and a small `--separator-*` CSS contract for color,
border style, thickness, and orientation-specific length.

### CSS Properties

### Styling hooks

- `data-scope="separator"` and `data-part="root"` identify the Ark-style root part.
- `data-slot="separator-root"` is the stable moduix selector.
- `data-orientation="horizontal"` and `data-orientation="vertical"` target axis-specific styling.
- `data-size="xs|sm|md|lg"` and `data-variant="solid|dashed|dotted"` expose the visual recipe.
- `className` and `style` are applied directly to the root separator element.
