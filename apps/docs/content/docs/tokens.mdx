# Tokens

moduix styles are built from CSS custom properties. Start with semantic color tokens for the
surface, text, borders, rings, and overlays, then tune spacing, radius, size, typography, motion,
shadow, and layering tokens when the product system needs a different rhythm.

When you use moduix as an npm package, import `moduix/style.css` once. When you install source files
with `shadcn`, import `@/lib/moduix/styles/style.css` once. Both entrypoints load the same token
foundation.

The reset stays optional in both paths. Import `moduix/reset.css` or
`@/lib/moduix/styles/reset.css` explicitly before `style.css` only when you want the shared reset.

## Mass Style Overrides

Some variables intentionally sit above individual components. Changing them updates multiple
components at once until a component-specific `--<component>-*` variable overrides the fallback.

| Variable group                                                                           | What it changes                       | Common consumers                                                                                                                                        |
| ---------------------------------------------------------------------------------------- | ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <code>--backdrop-bg</code><br /><code>--backdrop-filter</code>                           | Default backdrop color and blur       | AlertDialog, Autocomplete, Combobox, CommandPalette, ContextMenu, Dialog, Drawer, Lightbox, Menu, Menubar, NavigationMenu, Popover, PreviewCard, Select |
| <code>--duration-\*</code><br /><code>--ease-\*</code><br /><code>--transition-\*</code> | Shared motion timing and easing       | Controls, overlays, tabs, accordions, inputs, toasts                                                                                                    |
| <code>--scale-popup</code><br /><code>--scale-dropdown</code>                            | Default popup open/close scale        | Dialog, Popover, Tooltip, Menu, Toast, NavigationMenu, Select, Combobox, Autocomplete, ContextMenu                                                      |
| <code>--popup-\*</code>                                                                  | Shared popup list density and spacing | Menu, Menubar, Select, Combobox, Autocomplete, ContextMenu, CommandPalette                                                                              |

Use these tokens when you want a broad visual shift across the system. Use component-specific
tokens like `--dialog-backdrop-bg` or `--popover-transition` when a single component should opt out.

The quick table above is the cross-component index. The detailed overview below keeps the same
tokens in their native sections: motion in `Motion`, scale in `Elevation & Layers`, and shared
backdrop/popup defaults in `Shared Component Defaults`.

```css
:root {
  --backdrop-bg: oklch(0.12 0.03 250 / 0.34);
  --transition-default: 240ms cubic-bezier(0.22, 1, 0.36, 1);
  --scale-popup: 0.96;
}

.command-surface {
  --popup-item-min-height: 2.25rem;
}
```

## Override Scope

Set tokens globally in `:root`, inside `.dark`, or on any parent container that wraps moduix
components.

```css
:root {
  --primary: oklch(0.52 0.18 145);
  --radius: 0.75rem;
  --backdrop-filter: blur(4px);
}

.dense-panel {
  --spacing-md: 8px;
  --size-lg: 36px;
  --popup-item-min-height: 1.75rem;
}
```
