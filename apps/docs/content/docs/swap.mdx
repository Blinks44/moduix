## Anatomy

```text
Swap / Swap.Root
├─ Swap.Indicator type="off"
└─ Swap.Indicator type="on"
```

| Part                 | Role                                                             |
| -------------------- | ---------------------------------------------------------------- |
| `Swap` / `Swap.Root` | Owns the Ark `swap` state and layers both visual states.         |
| `Swap.RootProvider`  | Uses state created with `Swap.useSwap()` outside the subtree.    |
| `Swap.Indicator`     | Renders one `on` or `off` visual state with Presence attributes. |

## Composition

### How it works

`Swap` is a presentational primitive, not an interactive control. It renders its `off` and `on`
indicators in the same one-cell grid and `swap` decides which one is visible. Ark keeps the exiting
indicator mounted long enough for its CSS exit animation, exposing `data-state="open"` and
`data-state="closed"` for each phase.

This solves visual replacement without hand-writing the layering and presence lifecycle: an upload
icon can become a checkmark, an empty label can become a text label, or one compact visual state can
become another. Keep semantics outside it: the parent button, link, or status surface owns the event,
focus, and accessible name.

### Recommended composition

Keep `Swap` focused on visual replacement. Put it inside a semantic button, link, or status
surface; that parent owns events, focus handling, and accessible naming.

Use `Swap.RootProvider` with `Swap.useSwap()` when the state needs to be created outside the
rendered subtree. `useSwapContext()` is also available from `moduix` for context-aware children.

### Behavioral notes

- `swap` is a controlled boolean. `Swap` does not react to hover or clicks by itself.
- Each indicator receives Ark `data-state="open" | "closed"` while transitioning.
- Both states can be present during an exit animation. Mark visual indicators `aria-hidden="true"`
  and name the surrounding control explicitly when their content is only decorative.
- `lazyMount` and `unmountOnExit` retain their Ark behavior for controlling inactive DOM nodes.

## Styling

### CSS Properties

<CssPropertiesSection properties={SwapDocs.swapOverrideCssProperties} />

### Styling hooks

- `Swap.Root` -> `data-slot="swap-root"`
- `Swap.RootProvider` -> `data-slot="swap-root-provider"`
- `Swap.Indicator` -> `data-slot="swap-indicator"`
- Ark adds `data-scope="swap"` and `data-part="root" | "indicator"`.
- Ark adds `data-state="open" | "closed"` to indicators; moduix uses those attributes for its
  default enter and exit animations.
