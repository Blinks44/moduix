## Anatomy

Standalone checkbox:

```text
Checkbox.Root
├─ Checkbox.Control
│  └─ Checkbox.Indicator
├─ Checkbox.Label
└─ Checkbox.HiddenInput
```

Grouped checkboxes:

```text
Checkbox.Group
└─ Checkbox.Root[value]
   ├─ Checkbox.Control
   ├─ Checkbox.Label
   └─ Checkbox.HiddenInput
```

| Part                   | Role                                                                  |
| ---------------------- | --------------------------------------------------------------------- |
| `Checkbox.Root`        | Owns one checkbox state and renders the wrapping label element.       |
| `Checkbox.Control`     | Visual control surface. Styles checked, indeterminate, invalid state. |
| `Checkbox.Indicator`   | Checked or indeterminate icon slot. Use `indeterminate` when needed.  |
| `Checkbox.Label`       | Visible label text inside the root.                                   |
| `Checkbox.HiddenInput` | Native input for forms and accessibility.                             |
| `Checkbox.Group`       | Shares value state across multiple checkbox roots.                    |

## Composition

### Recommended path

Keep the Ark part tree visible. Use `Checkbox.Root` for standalone controls and `Checkbox.Group`
for shared value state. `Checkbox.Root` accepts `checked`, `defaultChecked`, `onCheckedChange(details)`,
`disabled`, `readOnly`, `invalid`, `required`, `name`, `value`, `form`, `ids`, and `asChild`.

`Checkbox.Group` accepts `value`, `defaultValue`, `onValueChange(value)`, `disabled`, `readOnly`,
`invalid`, `maxSelectedValues`, and `name`.

`size` is the only moduix DX prop. Apply it on `Checkbox.Root`.

### Indeterminate indicators

Ark keeps checked and indeterminate indicators explicit. For mixed state, render both:

```tsx
<Checkbox.Control>
  <Checkbox.Indicator />
  <Checkbox.Indicator indeterminate />
</Checkbox.Control>
```

### Group labeling

Ark does not ship a dedicated group label part. Use surrounding copy, `aria-labelledby`, or a real
`Fieldset`/`FieldsetLegend` when the checkbox set is one form field.

### Controlled

Control `checked` from React state when the checkbox should coordinate with other UI or derived business logic.

<Preview>
  <ControlledCheckboxExample />

  <Preview.Code>
    {`
      import { Checkbox } from "moduix";
      import { useState } from "react";

      export function ControlledCheckboxDemo() {
        const [checked, setChecked] = useState(true);

        return (
          <Checkbox.Root checked={checked} onCheckedChange={(details) => setChecked(details.checked === true)}>
            <Checkbox.Control>
              <Checkbox.Indicator />
            </Checkbox.Control>
            <Checkbox.Label>{checked ? "Enabled" : "Disabled"}</Checkbox.Label>
            <Checkbox.HiddenInput />
          </Checkbox.Root>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Select All

Compose a parent checkbox and a controlled group when one control should toggle the entire selection set.

<Preview>
  <CheckboxGroupSelectAllExample />

  <Preview.Code>
    {`
      import { Checkbox } from "moduix";
      import { useState } from "react";

      export function CheckboxSelectAllDemo() {
        const [value, setValue] = useState(["react"] as string[]);
        const allValues = ["react", "solid", "vue"];
        const allSelected = value.length === allValues.length;
        const indeterminate = value.length > 0 && value.length < allValues.length;

        return (
          <>
            <Checkbox.Root
              checked={indeterminate ? "indeterminate" : allSelected}
              onCheckedChange={(details) => setValue(details.checked === true ? allValues : [])}
            >
              <Checkbox.Control>
                <Checkbox.Indicator />
                <Checkbox.Indicator indeterminate />
              </Checkbox.Control>
              <Checkbox.Label>Select all</Checkbox.Label>
              <Checkbox.HiddenInput />
            </Checkbox.Root>

            <Checkbox.Group value={value} onValueChange={setValue} name="frameworks">
              {allValues.map((item) => (
                <Checkbox.Root key={item} value={item}>
                  <Checkbox.Control>
                    <Checkbox.Indicator />
                  </Checkbox.Control>
                  <Checkbox.Label>{item}</Checkbox.Label>
                  <Checkbox.HiddenInput />
                </Checkbox.Root>
              ))}
            </Checkbox.Group>
          </>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Styling hooks

- `Checkbox.Root` exposes `data-slot="checkbox-root"` and carries `data-size`.
- `Checkbox.Control` exposes `data-slot="checkbox-control"` and Ark state attributes such as
  `data-state`, `data-disabled`, `data-readonly`, `data-invalid`, and `data-focus-visible`.
- `Checkbox.Indicator` exposes `data-slot="checkbox-indicator"`.
- `Checkbox.Label` exposes `data-slot="checkbox-label"`.
- `Checkbox.HiddenInput` exposes `data-slot="checkbox-hidden-input"`.
- `Checkbox.Group` exposes `data-slot="checkbox-group"`.
