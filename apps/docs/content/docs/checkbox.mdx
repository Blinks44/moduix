`Checkbox` is a thin styled wrapper over the Base UI checkbox root. It keeps the default path
short by rendering the indicator for you, while still exporting the indicator parts for manual
composition.

## Anatomy

Use `CheckboxField` when the checkbox and label can live inside one wrapping `<label>`.

```text
CheckboxField
├─ Checkbox
│  └─ CheckboxIndicator
│     └─ CheckboxIndicatorIcon
└─ CheckboxLabel
```

```tsx
<CheckboxField>
  <Checkbox defaultChecked />
  <CheckboxLabel>Enable notifications</CheckboxLabel>
</CheckboxField>
```

| Part                    | Role                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------- |
| `Checkbox`              | Interactive root. Handles checked, indeterminate, disabled, and readonly states.      |
| `CheckboxIndicator`     | Optional indicator part inside the root.                                              |
| `CheckboxIndicatorIcon` | Default icon wrapper that switches between checked and indeterminate visuals.         |
| `CheckboxField`         | Optional wrapping label for the common clickable-label layout.                        |
| `CheckboxLabel`         | Optional text wrapper for label content that should inherit the checkbox text tokens. |

## Composition

Use `defaultChecked` for uncontrolled mode and `checked` with `onCheckedChange` for controlled mode.
Use `indeterminate` for parent-selection states. `Checkbox` forwards the Base UI root props, so form
props such as `name`, `value`, `uncheckedValue`, `required`, `readOnly`, `form`, and `inputRef`
continue to work without extra wrapper API.

`size` is the only component-specific DX sugar. It scales the control and default icon without
changing the composition model.

When the label must be a sibling instead of a wrapping `<label>`, switch the root to a native
button:

```tsx
<div>
  <Checkbox id="keep-signed-in" nativeButton render={<button />} />
  <label htmlFor="keep-signed-in">Keep me signed in</label>
</div>
```

When the default indicator is enough, render `<Checkbox />` directly. When you need a custom visual
or primitive-level control like `keepMounted`, compose `CheckboxIndicator` and
`CheckboxIndicatorIcon` explicitly:

```tsx
<Checkbox defaultChecked>
  <CheckboxIndicator keepMounted>
    <CheckboxIndicatorIcon />
  </CheckboxIndicator>
</Checkbox>
```

### Controlled

Use controlled mode when the checkbox needs to coordinate with surrounding UI.

<Preview>
  <ControlledCheckboxExample />

  <Preview.Code>
    {`
      import { Checkbox, CheckboxField, CheckboxLabel } from "moduix";
      import { useState } from "react";

      export function ControlledCheckboxDemo() {
        const [checked, setChecked] = useState(false);

        return (
          <CheckboxField>
            <Checkbox checked={checked} onCheckedChange={setChecked} />
            <CheckboxLabel>{checked ? "Enabled" : "Disabled"}</CheckboxLabel>
          </CheckboxField>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Custom Composition

### Custom Icon

Swap the default icon by composing `CheckboxIndicator` inside `Checkbox`.

<Preview>
  <CustomIconCheckboxExample />

  <Preview.Code>
    {`
      import { Checkbox, CheckboxField, CheckboxIndicator, CheckboxLabel } from "moduix";

      function PlusIcon(props) {
        return (
          <svg viewBox="0 0 10 10" fill="none" aria-hidden="true" focusable="false" {...props}>
            <path
              d="M5 1.5V8.5M1.5 5H8.5"
              stroke="currentColor"
              strokeWidth="1.8"
              strokeLinecap="round"
            />
          </svg>
        );
      }

      export function CustomIconCheckboxDemo() {
        return (
          <CheckboxField>
            <Checkbox defaultChecked>
              <CheckboxIndicator>
                <PlusIcon />
              </CheckboxIndicator>
            </Checkbox>
            <CheckboxLabel>Use a custom indicator icon</CheckboxLabel>
          </CheckboxField>
        );
      }
    `}

  </Preview.Code>
</Preview>
