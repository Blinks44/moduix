## Anatomy

The normal consumer path is short:

```text
CheckboxField
├─ Checkbox
│  └─ CheckboxIndicator
│     └─ CheckboxIndicatorIcon
└─ CheckboxLabel
```

Use `CheckboxField` when the checkbox and label can live inside one wrapping `<label>`.

| Part                    | Role                                                                                  |
| ----------------------- | ------------------------------------------------------------------------------------- |
| `Checkbox`              | Interactive root. Handles checked, indeterminate, disabled, readonly, and form state. |
| `CheckboxIndicator`     | Optional indicator part inside the root.                                              |
| `CheckboxIndicatorIcon` | Default icon wrapper that switches between checked and indeterminate visuals.         |
| `CheckboxField`         | Optional wrapping label for the common clickable-label layout.                        |
| `CheckboxLabel`         | Optional text wrapper for label content that should inherit checkbox text tokens.     |

## Composition

### When to stay high-level

Stay with `Checkbox`, `CheckboxField`, and `CheckboxLabel` for most settings panels, forms, and
single-option toggles.

`Checkbox` keeps Base UI root behavior intact. Use it for `checked`, `defaultChecked`,
`onCheckedChange`, `indeterminate`, `name`, `value`, `uncheckedValue`, `required`, `disabled`,
`readOnly`, `inputRef`, `nativeButton`, and `render`.

`size` is the only component-specific DX sugar. It scales the control and the default icon without
changing the composition model.

### When to go custom

Render `<Checkbox />` directly when the default indicator is enough. Compose `CheckboxIndicator`
and `CheckboxIndicatorIcon` when the visual needs to change or when you need primitive props such as
`keepMounted`.

For sibling-label layouts, switch the root to a native button:

```tsx
<div>
  <Checkbox id="keep-signed-in" nativeButton render={<button />} />
  <label htmlFor="keep-signed-in">Keep me signed in</label>
</div>
```

### Default props

| Prop        | Default        | Values / Notes                                       |
| ----------- | -------------- | ---------------------------------------------------- |
| `size`      | `md`           | `xs`, `sm`, `md`, `lg`, `xl`                         |
| `className` | `undefined`    | Root class name or Base UI state callback class name |
| `children`  | auto indicator | Compose `CheckboxIndicator` manually when needed     |

### Behavioral notes

- Use `defaultChecked` for uncontrolled mode and `checked` with `onCheckedChange` for controlled
  mode.
- Use `indeterminate` for mixed parent-selection state. It is not a third submitted form value.
- Every checkbox needs an accessible name. The default recommendation is
  `CheckboxField` + `CheckboxLabel`.
- Use `readOnly` when the value should stay visible but not change. Use `disabled` when the control
  is unavailable.

### Controlled

Use controlled mode when the checkbox needs to coordinate with surrounding UI.

<Preview>
  <ControlledCheckboxExample />

  <Preview.Code>
    {`
      import { Checkbox, CheckboxField, CheckboxLabel } from "moduix";
      import { useState } from "react";
      import styles from "./checkbox-demo.module.css";

      export function ControlledCheckboxDemo() {
        const [checked, setChecked] = useState(false);

        return (
          <div className={styles.stack}>
            <CheckboxField>
              <Checkbox checked={checked} onCheckedChange={setChecked} />
              <CheckboxLabel>{checked ? "Enabled" : "Disabled"}</CheckboxLabel>
            </CheckboxField>
            <span className={styles.hint}>Current value: {String(checked)}</span>
          </div>
        );

### Custom Composition

Swap the default indicator or take direct control over indicator behavior when the built-in visual
is not enough.

<Preview>
  <CustomIconCheckboxExample />

  <Preview.Code>
    {`
      import { Checkbox, CheckboxField, CheckboxIndicator, CheckboxLabel } from "moduix";
      import type { ComponentProps } from "react";
      import styles from "./checkbox-demo.module.css";

      function PlusIcon(props: ComponentProps<"svg">) {
        return (
          <svg viewBox="0 0 10 10" fill="none" aria-hidden="true" focusable="false" {...props}>
            <path
              d="M5 1.5V8.5M1.5 5H8.5"
              stroke="currentColor"
              strokeWidth="1.8"
              strokeLinecap="round"
            />
          </svg>
        );
      }

      export function CustomIconCheckboxDemo() {
        return (
          <CheckboxField>
            <Checkbox defaultChecked>
              <CheckboxIndicator>
                <PlusIcon className={styles.customIndicatorIcon} />
              </CheckboxIndicator>
            </Checkbox>
            <CheckboxLabel>Use a custom indicator icon</CheckboxLabel>
          </CheckboxField>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .customIndicatorIcon {
        transform: rotate(-8deg);
      }
    `}

  </Preview.CSS>
</Preview>

## Styling

Use CSS variables on the root when checkbox size, label typography, or state colors need to change.
Use explicit composition when the indicator structure itself needs to change.

### Styling hooks

- `Checkbox` exposes `data-slot="checkbox-root"` and writes `data-size` for the `size` presets.
- `CheckboxIndicator` exposes `data-slot="checkbox-indicator"` and receives Base UI state
  attributes such as `data-checked`, `data-unchecked`, and `data-indeterminate`.
- `CheckboxIndicatorIcon`, `checkbox-indicator-checked-icon`, and
  `checkbox-indicator-indeterminate-icon` are stable selectors for icon-level styling.
- `CheckboxField` and `CheckboxLabel` expose `data-slot="checkbox-field"` and
  `data-slot="checkbox-label"` for layout and typography overrides.
