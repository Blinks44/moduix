## Anatomy

```text
Checkbox.Root
├─ Checkbox.Control
│  ├─ Checkbox.Indicator
│  └─ Checkbox.Indicator[indeterminate] (optional)
├─ Checkbox.Label
└─ Checkbox.HiddenInput

Checkbox.RootProvider
└─ same part tree connected to useCheckbox()

Checkbox.Group
└─ Checkbox.Root[value]
   ├─ Checkbox.Control
   ├─ Checkbox.Label
   └─ Checkbox.HiddenInput
```

| Part                         | Role                                                                  |
| ---------------------------- | --------------------------------------------------------------------- |
| `Checkbox` / `Checkbox.Root` | Owns one checkbox state and renders the wrapping label element.       |
| `Checkbox.RootProvider`      | Connects parts to Ark state created outside the rendered subtree.     |
| `Checkbox.Control`           | Visual control surface. Styles checked, indeterminate, invalid state. |
| `Checkbox.Indicator`         | Checked or indeterminate icon slot. Use `indeterminate` when needed.  |
| `Checkbox.Label`             | Visible label text inside the root.                                   |
| `Checkbox.HiddenInput`       | Native input for forms, validation, form reset, and submission.       |
| `Checkbox.Group`             | Shares value state across multiple checkbox roots.                    |

## Composition

### Recommended composition

Keep the Ark part tree visible. `Checkbox.Root` accepts Ark root props such as `checked`,
`defaultChecked`, `onCheckedChange(details)`, `disabled`, `readOnly`, `invalid`, `required`,
`name`, `value`, `form`, `ids`, and `asChild`.

`Checkbox.Group` accepts `value`, `defaultValue`, `onValueChange(value)`, `disabled`, `readOnly`,
`invalid`, `maxSelectedValues`, and `name`.

`size` is the only moduix DX prop. Apply it on `Checkbox.Root` or `Checkbox.RootProvider`.

### When to go custom

Use `Checkbox.RootProvider` when checkbox state must be owned outside the rendered subtree. moduix
keeps the provider, but the state hook comes from Ark UI.

### Default props

| Surface                      | Default | Notes                                                   |
| ---------------------------- | ------- | ------------------------------------------------------- |
| `Checkbox` / `Checkbox.Root` | —       | No additional behavioral defaults; Ark defaults apply.  |
| `size`                       | `md`    | Applied on `Checkbox.Root` and `Checkbox.RootProvider`. |

### Behavioral notes

Render `Checkbox.HiddenInput` for native form participation, validation, form reset, and
submission. For a standalone checkbox, pass `name` and `value` to `Checkbox.Root`. For a group,
pass `name` to `Checkbox.Group` and `value` to every `Checkbox.Root`.

Ark `Checkbox.Root` renders a `label`. If you use `asChild`, the direct child must also be a
semantic `label`, otherwise the checkbox no longer has valid form-control labelling.

### Root Provider

Use `Checkbox.RootProvider` when checkbox state needs to live outside the rendered subtree. Import
the state hook from Ark UI directly.

<Preview>
  <CheckboxDocs.CheckboxRootProviderExample />

  <Preview.Code>
    {`
      import { useCheckbox } from "@ark-ui/react/checkbox";
      import { Checkbox } from "@moduix/react";

      export function RootProviderCheckboxDemo() {
        const checkbox = useCheckbox({ defaultChecked: true });

        return (
          <>
            <Checkbox.RootProvider value={checkbox}>
              <Checkbox.Control>
                <Checkbox.Indicator />
              </Checkbox.Control>
              <Checkbox.Label>Managed outside the tree</Checkbox.Label>
              <Checkbox.HiddenInput />
            </Checkbox.RootProvider>
            <button type="button" onClick={() => checkbox.setChecked(!checkbox.checked)}>
              {checkbox.checked ? "Uncheck" : "Check"}
            </button>
          </>
        );
      }
    `}

  </Preview.Code>

### Styling Hooks

- `Checkbox.Root` exposes `data-slot="checkbox-root"`, `data-size`, and Ark state attributes such as `data-state`, `data-disabled`, `data-readonly`, `data-invalid`, `data-required`, `data-hover`, `data-focus`, and `data-focus-visible`.
- `Checkbox.RootProvider` exposes `data-slot="checkbox-root-provider"` and the same root styling contract.
- `Checkbox.Control` exposes `data-slot="checkbox-control"` and Ark state attributes including `data-state="checked" | "indeterminate" | "unchecked"`.
- `Checkbox.Indicator` exposes `data-slot="checkbox-indicator"` and inherits Ark checkbox state attributes.
- `Checkbox.Label` exposes `data-slot="checkbox-label"` and Ark checkbox state attributes.
- `Checkbox.HiddenInput` exposes `data-slot="checkbox-hidden-input"`.
- `Checkbox.Group` exposes `data-slot="checkbox-group"`.
