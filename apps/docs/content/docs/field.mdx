## Choosing the right component

Use `Field` when one control needs a visible label, helper text, error text, or shared
`disabled`, `invalid`, `required`, and `readOnly` state.

- Choose **`Field`** for one control and its label or descriptive text.
- Choose **`Fieldset`** when several related fields need one shared legend.
- Use a standalone control when no field-level ARIA wiring or shared state is needed.

## Basic

Start with Ark parts: `Field`, `Field.Label`, one control part, and optional helper or error text.

## Anatomy

`Field` follows Ark UI's field parts and keeps their names in the public moduix API.

```text
Field
├─ Field.Label
├─ Field.Input | Field.Textarea | Field.Select | custom control
├─ Field.HelperText
├─ Field.ErrorText
└─ Field.RequiredIndicator
```

| Part                      | Role                                                            |
| ------------------------- | --------------------------------------------------------------- |
| `Field` / `Field.Root`    | Root context and default `<div role="group">`.                  |
| `Field.RootProvider`      | Root powered by a `useField()` instance owned outside the tree. |
| `Field.Label`             | Label linked to the current control id.                         |
| `Field.Input`             | Native input wired with field ARIA and state props.             |
| `Field.Textarea`          | Native textarea; supports Ark `autoresize`.                     |
| `Field.Select`            | Native select wired with field ARIA and state props.            |
| `Field.HelperText`        | Helper text included in `aria-describedby`.                     |
| `Field.ErrorText`         | Error text rendered only when `Field` is `invalid`.             |
| `Field.RequiredIndicator` | Visual required marker with optional `fallback`.                |
| `Field.Item`              | Item-scoped field context for grouped controls via `value`.     |
| `Field.Context`           | Render-prop access to the current field context.                |

### Controlled Invalid

Derive `invalid` from local state when the field should update as the user types.

<Preview>
  <FieldControlledInvalidExample />

  <Preview.Code>
    {`
      import { Field } from "moduix";
      import { useState } from "react";

      export function ControlledInvalidFieldDemo() {
        const [value, setValue] = useState("");
        const invalid = value.length > 0 && value.length < 3;

        return (
          <Field invalid={invalid}>
            <Field.Label>Username</Field.Label>
            <Field.Input
              value={value}
              onChange={(event) => setValue(event.currentTarget.value)}
              placeholder="e.g. vinny"
            />
            <Field.HelperText>Use at least 3 characters.</Field.HelperText>
            <Field.ErrorText>Username must be at least 3 characters.</Field.ErrorText>
          </Field>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{`.field { width: fit-content; max-width: min(20rem, 100%); }`}</Preview.CSS>
  <Preview.Data>{`const minLength = 3;`}</Preview.Data>
</Preview>

### Custom Control

Use `useFieldContext` when a custom control must receive Ark's control props directly.

<Preview>
  <FieldCustomControlExample />

  <Preview.Code>
    {`
      import { Field, useFieldContext } from "moduix";

      function CustomControl() {
        const field = useFieldContext();

        return (
          <input
            type="button"
            className="customControl"
            value={field.invalid ? "Resolve status" : "Set status"}
            {...field.getInputProps()}
          />
        );
      }

      export function CustomControlFieldDemo() {
        return (
          <Field invalid>
            <Field.Label>Status</Field.Label>
            <CustomControl />
            <Field.ErrorText>Status needs a resolution.</Field.ErrorText>
          </Field>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`

        border-color: color-mix(in srgb, var(--color-primary) 40%, transparent);
      }
    `}
  </Preview.CSS>

</Preview>

moduix `data-slot="field-*"` hooks, and the public `--field-*` CSS variables shown below.

### State

Ark adds `data-disabled`, `data-invalid`, `data-readonly`, and `data-required` to the relevant
parts. `Field.ErrorText` is only rendered while the current field context is invalid, and helper or
error text ids are included in the control's `aria-describedby` when present.
