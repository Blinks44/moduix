## Choosing the right component

Use `Tooltip` for short, non-essential labels or hints that appear on hover or focus.

- Choose **`Tooltip`** for compact labels and short contextual hints.
- Choose **`Popover`** when the popup needs richer layout, actions, or interactive content.
- Choose **`HoverCard`** when the popup should expose a richer preview surface tied to a trigger.

## Basic

Start with the Ark part tree: `Tooltip`, `Tooltip.Trigger`, `Tooltip.Portal`,
`Tooltip.Positioner`, and `Tooltip.Content`. Keep an accessible name on the trigger itself:
tooltips do not replace `aria-label` or visible text.

## Anatomy

```text
Tooltip / Tooltip.Root
├─ Tooltip.Trigger
└─ Tooltip.Portal
   └─ Tooltip.Positioner
      └─ Tooltip.Content
         └─ content
```

Add `Tooltip.Arrow` and `Tooltip.ArrowTip` inside the content when the popup should point back to
the trigger. `Tooltip.Arrow` renders `Tooltip.ArrowTip` by default.

```text
Tooltip / Tooltip.Root
├─ Tooltip.Trigger
└─ Tooltip.Portal
   └─ Tooltip.Positioner
      └─ Tooltip.Content
         ├─ Tooltip.Arrow
         │  └─ Tooltip.ArrowTip
         └─ content
```

| Part                       | Role                                                                                        |
| -------------------------- | ------------------------------------------------------------------------------------------- |
| `Tooltip` / `Tooltip.Root` | Root state, controlled/uncontrolled open state, delays, positioning options, and lifecycle. |
| `Tooltip.RootProvider`     | Renders a tooltip from a `useTooltip()` state object owned outside the tree.                |
| `Tooltip.Trigger`          | Interactive anchor element that opens the tooltip on hover or focus.                        |
| `Tooltip.Portal`           | Portal boundary for the floating structure.                                                 |
| `Tooltip.Positioner`       | Floating layout layer that consumes Ark positioning options and runtime CSS variables.      |
| `Tooltip.Content`          | Visible tooltip surface.                                                                    |
| `Tooltip.Arrow`            | Optional arrow wrapper.                                                                     |
| `Tooltip.ArrowTip`         | Arrow tip element; rendered by default from `Tooltip.Arrow`.                                |
| `Tooltip.Context`          | Render-prop access to the current tooltip state and methods.                                |

## Composition

### Recommended composition

Use explicit Ark composition for every tooltip. Moduix adds default styling, `data-slot` hooks, and
a default `ArrowTip`, but it does not hide `Portal`, `Positioner`, or `Content` behind a high-level
content wrapper.

### When to go custom

Use `asChild` on `Tooltip.Trigger` when the trigger should be a moduix `Button`, link, or another
semantic control. The child must remain a single focusable element that can support the trigger
interaction.

Use `positioning` on `Tooltip` for placement, strategy, offset, collision behavior, and the rest of
Ark's floating options. Use `ids` when separate tooltip parts need stable shared IDs.

### Default props

| Prop                   | Default | Notes                                                             |
| ---------------------- | ------- | ----------------------------------------------------------------- |
| `openDelay`            | `400`   | Delay before opening on hover or focus.                           |
| `closeDelay`           | `150`   | Delay before closing after hover or focus leaves.                 |
| `closeOnClick`         | `true`  | Closes when the trigger is clicked.                               |
| `closeOnEscape`        | `true`  | Closes when Escape is pressed.                                    |
| `closeOnPointerDown`   | `true`  | Closes on pointer down outside the trigger/content path.          |
| `closeOnScroll`        | `true`  | Closes on scroll.                                                 |
| `interactive`          | `false` | Keeps the tooltip open while hovering content when set to `true`. |
| `lazyMount`            | `false` | Defers mounting until the tooltip first opens.                    |
| `unmountOnExit`        | `false` | Unmounts content after exit when enabled.                         |
| `skipAnimationOnMount` | `false` | Allows the first presence animation unless explicitly skipped.    |

### Behavioral notes

- Controlled state uses Ark's `open` and `onOpenChange(details)` shape. Read `details.open`.
- `Tooltip.Context`, `useTooltipContext`, `useTooltip`, and `Tooltip.RootProvider` are exported
  from `moduix`.
- `Tooltip.Trigger` receives the moduix trigger skin only when `asChild` is not used. With
  `asChild`, styling belongs to the child element.
- `Tooltip.Content` owns the visible surface and z-index. `Tooltip.Positioner` owns floating
  layout and exposes Ark variables such as `--available-width`, `--available-height`, and
  `--transform-origin`.
- Ark emits `data-state="open" | "closed"` on trigger and content, `data-expanded` on trigger,
  and placement attributes such as `data-placement` and `data-side` on content.
- There is no Base UI `Provider`, `render`, `createTooltipHandle`, payload render function,
  `TooltipPopup`, `TooltipViewport`, `side`, `sideOffset`, or `showArrow` API. Use Ark
  composition and `positioning` instead.

### Multiple Triggers

Use trigger `value` and `onTriggerValueChange(details)` to drive one tooltip from several
triggers.

<Preview>
  <MultipleTriggersTooltipExample />

  <Preview.Code>
    {`
      import { Button, InfoIcon, PlusIcon, ShareIcon, Tooltip } from "moduix";
      import { useState } from "react";
      import styles from "./tooltip-demo.module.css";

      export function MultipleTriggersTooltipDemo() {
        const [activeTool, setActiveTool] =
          useState<(typeof tooltipTools)[number] | null>(null);

        return (
          <Tooltip
            onTriggerValueChange={(details) => {
              setActiveTool(
                tooltipTools.find((tool) => tool.id === details.value) ?? null,
              );
            }}
          >
            <div className={styles.toolbar}>
              {tooltipTools.map((tool) => (
                <Tooltip.Trigger
                  key={tool.id}
                  value={tool.id}
                  asChild
                  aria-label={tool.label}
                >
                  <Button variant="ghost" size="icon-md">
                    <tool.icon className={styles.icon} />
                  </Button>
                </Tooltip.Trigger>
              ))}
            </div>
            <Tooltip.Portal>
              <Tooltip.Positioner>
                <Tooltip.Content>
                  {activeTool ? (
                    <>
                      {activeTool.label}{" "}
                      <span className={styles.shortcut}>{activeTool.shortcut}</span>
                    </>
                  ) : null}
                </Tooltip.Content>
              </Tooltip.Positioner>
            </Tooltip.Portal>
          </Tooltip>
        );
      }
    `}

### Styling hooks

Use the part that owns the visual concern:

- `Tooltip.Trigger` for trigger chrome, hover/open feedback, and focus styling.
- `Tooltip.Positioner` for floating layout constraints.
- `Tooltip.Content` for popup surface overrides.
- `Tooltip.Arrow` and `Tooltip.ArrowTip` for arrow size, background, and stroke.

The main public hooks include `data-slot="tooltip-trigger"`, `tooltip-positioner`,
`tooltip-content`, `tooltip-arrow`, and `tooltip-arrow-tip`.

Ark also adds `data-scope="tooltip"`, `data-part`, `data-state="open" | "closed"`,
`data-expanded`, `data-instant`, `data-placement`, and `data-side` on the relevant parts. The
positioner exposes runtime CSS variables including `--available-width`, `--available-height`,
`--reference-width`, `--reference-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`.
