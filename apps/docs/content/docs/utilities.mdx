# Utilities

Ark UI ships several utilities that sit next to the moduix component layer instead of inside it.
Use them directly when your application needs the primitive behavior and the relevant moduix
component page does not already document a wrapper or pass-through prop.

Many of these utilities use dedicated subpath imports:

```tsx
import { Format } from '@ark-ui/react';
import { ClientOnly } from '@ark-ui/react/client-only';
import { EnvironmentProvider } from '@ark-ui/react/environment';
import { LocaleProvider } from '@ark-ui/react/locale';
import { Presence } from '@ark-ui/react/presence';
```

Check the linked Ark page for the exact import and full API.

## Overview

| Utility                                                                        | Reach for it when                                                                                        | moduix context                                                                                    |
| ------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| [Client Only](https://ark-ui.com/docs/utilities/client-only)                   | A subtree needs `window`, layout measurement, or another browser-only API after hydration.               | Useful in SSR apps and docs examples.                                                             |
| [Download Trigger](https://ark-ui.com/docs/utilities/download-trigger)         | A click should download a URL, blob, file, or generated payload.                                         | moduix already wraps this pattern as `QrCode.DownloadTrigger` for QR exports.                     |
| [Environment](https://ark-ui.com/docs/utilities/environment)                   | A moduix or Ark subtree renders inside an iframe, Shadow DOM, Electron, or another custom document root. | Wrap the affected subtree once instead of patching each component.                                |
| [Focus Trap](https://ark-ui.com/docs/utilities/focus-trap)                     | You are building a custom attention-demanding surface that is not a moduix dialog-like component.        | moduix dialog, drawer, popover, menu, and tooltip roots already manage their own focus behavior.  |
| [Format Byte](https://ark-ui.com/docs/utilities/format-byte)                   | You need localized file sizes or storage values.                                                         | Useful around uploads, downloads, and telemetry.                                                  |
| [Format Relative Time](https://ark-ui.com/docs/utilities/format-relative-time) | You need user-facing relative timestamps such as "2 min ago".                                            | Reads locale from `LocaleProvider` when present.                                                  |
| [Format Time](https://ark-ui.com/docs/utilities/format-time)                   | You need localized time output from a `Date` or `HH:mm[:ss]` string.                                     | Useful for scheduling, timers, and activity feeds.                                                |
| [Frame](https://ark-ui.com/docs/utilities/frame)                               | React content should render into an iframe with head injection and mount callbacks.                      | Often paired with `EnvironmentProvider` when the iframe subtree renders Ark or moduix components. |
| [Highlight](https://ark-ui.com/docs/utilities/highlight)                       | Search results or filtered labels should visually mark matching ranges.                                  | Useful with Combobox, Listbox, or command search UIs, but not wrapped directly by moduix.         |
| [JSON Tree View](https://ark-ui.com/docs/utilities/json-tree-view)             | You need a debug or inspection tree for JSON-like data.                                                  | Independent from moduix styling; use Ark directly.                                                |
| [Locale](https://ark-ui.com/docs/utilities/locale)                             | Locale and direction should flow through Ark formatters and locale-aware behavior.                       | Pair with `Format` utilities and RTL interfaces.                                                  |
| [Presence](https://ark-ui.com/docs/utilities/presence)                         | Your app owns visibility and mount lifecycle outside a built-in moduix root.                             | Dialog, Drawer, Popover, Menu, Tooltip, Lightbox, and Tour already expose Ark presence props.     |
| [Swap](https://ark-ui.com/docs/utilities/swap)                                 | Two compact visual states should crossfade or swap in place, often for icons.                            | Independent from moduix styling; useful in custom triggers and indicators.                        |

## Recommended first utilities

### Presence

Use `Presence` when your application owns the visible node and you still want Ark's delayed exit
unmounting behavior.

```tsx
import { Presence } from '@ark-ui/react/presence';
import { useState } from 'react';

export function InlinePresenceNotice() {
  const [open, setOpen] = useState(false);

  return (
    <>
      <button type="button" onClick={() => setOpen((value) => !value)}>
        Toggle notice
      </button>
      <Presence present={open} lazyMount unmountOnExit>
        <div className="notice">Saved filters will apply to the next refresh.</div>
      </Presence>
    </>
  );
}
```

Use the raw utility when the visible node is yours. If the node already belongs to a moduix root,
prefer that component's `lazyMount`, `unmountOnExit`, `present`, and `onExitComplete` props
directly.

See the full API in the [Ark Presence docs](https://ark-ui.com/docs/utilities/presence).

### Client Only

Use `ClientOnly` when a subtree depends on the DOM or browser APIs and should stay out of the
server render.

```tsx
import { ClientOnly } from '@ark-ui/react/client-only';

export function ClientChartShell() {
  return (
    <ClientOnly fallback={<div>Loading chart...</div>}>
      <ChartThatMeasuresContainer />
    </ClientOnly>
  );
}
```

This is especially useful in SSR apps when a chart, editor, or measurement-heavy panel sits next to
moduix layout components.

See the full API in the [Ark Client Only docs](https://ark-ui.com/docs/utilities/client-only).

### EnvironmentProvider

Use `EnvironmentProvider` when a moduix subtree renders in a non-default document or root node.

```tsx
import { EnvironmentProvider } from '@ark-ui/react/environment';
import { Popover } from '@moduix/react';

export function FramePreview({ iframeDocument }: { iframeDocument: Document }) {
  return (
    <EnvironmentProvider value={iframeDocument}>
      <Popover>
        <Popover.Trigger>Open preview tools</Popover.Trigger>
        <Popover.Positioner>
          <Popover.Content>Popover queries now target the iframe document.</Popover.Content>
        </Popover.Positioner>
      </Popover>
    </EnvironmentProvider>
  );
}
```

Wrap the subtree once, close to the alternate environment, and let Ark keep DOM queries scoped
correctly.

See the full API in the [Ark Environment docs](https://ark-ui.com/docs/utilities/environment).

### LocaleProvider and Format

Use `LocaleProvider` when locale and direction should flow through Ark formatters and locale-aware
primitives. Pair it with `Format` when the UI should render localized values without custom wrapper
code.

```tsx
import { Format } from '@ark-ui/react';
import { LocaleProvider } from '@ark-ui/react/locale';

export function ActivityTimestamp() {
  return (
    <LocaleProvider locale="fr-FR">
      <Format.RelativeTime value={-2} unit="day" style="short" />
    </LocaleProvider>
  );
}
```

Use the same provider for RTL interfaces so Ark-aware components and formatters read one locale
source.

See the [Ark Locale docs](https://ark-ui.com/docs/utilities/locale) and
[Ark Format Relative Time docs](https://ark-ui.com/docs/utilities/format-relative-time).

## When to stay in moduix

- Use `QrCode.DownloadTrigger` instead of the raw Ark download trigger when the user is saving a QR
  code image from the moduix QR component.
- Use component-level presence props on Dialog, Drawer, Popover, Menu, Tooltip, Lightbox, Tour,
  Accordion, Tabs, and similar roots before reaching for a standalone `Presence` wrapper.
- Use the built-in focus management in dialog-like and popup-like moduix components before adding a
  separate `FocusTrap`.

## Contract boundary

These utilities come from Ark UI, not from the moduix styled component layer. Ark owns their props,
lifecycle details, accessibility behavior, and release cadence. moduix component pages call out the
places where a utility is already part of a component contract.
