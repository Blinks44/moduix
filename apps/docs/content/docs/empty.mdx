## API Reference

`Empty` does not wrap a Base UI primitive. It is a standalone moduix presentational component built for
empty-state layouts.

## Install with shadcn (optional)

If you want `Empty` to become part of your own project source instead of staying in
`node_modules`, install it from the moduix GitHub registry:

<Tabs items={['npm', 'pnpm', 'bun']}>
  <Tab value="npm">

```bash
npx shadcn@latest add Blinks44/moduix/empty
```

  </Tab>
  <Tab value="pnpm">

```bash
pnpm dlx shadcn@latest add Blinks44/moduix/empty
```

  </Tab>
  <Tab value="bun">

```bash
bunx shadcn@latest add Blinks44/moduix/empty
```

  </Tab>
</Tabs>

This install path also pulls in the shared moduix foundation files that `Empty` depends on, such
as styles, icons, and small utilities.

`Empty` gives missing-data states a consistent surface, optional icon slot, and wrapping actions row
without locking consumers into high-level props like `title`, `description`, or `primaryAction`.

## Anatomy

```text
Empty
├─ EmptyIcon (optional)
├─ EmptyContent
│  ├─ EmptyTitle
│  └─ EmptyDescription
└─ EmptyActions (optional)
```

| Part               | Role                                                               |
| ------------------ | ------------------------------------------------------------------ |
| `Empty`            | Root surface that owns padding, border, background, and alignment. |
| `EmptyIcon`        | Optional icon or small visual affordance container.                |
| `EmptyContent`     | Groups text content with tighter inner spacing.                    |
| `EmptyTitle`       | Main empty-state heading.                                          |
| `EmptyDescription` | Supporting description block for one or more text nodes.           |
| `EmptyActions`     | Optional wrapping row for actions.                                 |

## Composition

Use the component parts directly instead of passing a bag of content props. That keeps the common path
simple while leaving room for richer descriptions, custom buttons, links, or omitted sections.

`EmptyDescription` renders a `div`, not a `p`, so it can safely contain block content when needed.
`EmptyTitle` renders `h3` by default and accepts `as` when the page outline needs another heading
level.

### Custom Styles

Override the `--empty-*` tokens on a single empty state when it should align with a more specific
screen tone.
