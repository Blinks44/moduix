## API Reference

`Empty` does not wrap a Base UI primitive. It is a standalone moduix presentational component built for
empty-state layouts.

## Basic

Start with `Empty`, `EmptyContent`, `EmptyTitle`, and `EmptyDescription`. Add `EmptyIcon` and
`EmptyActions` only when the state needs a visual cue or an explicit next step.

`Empty` gives missing-data states a consistent surface, optional icon slot, and wrapping actions row
without locking consumers into workflow props like `title`, `description`, or `primaryAction`.

## Anatomy

```text
Empty
├─ EmptyIcon (optional)
├─ EmptyContent
│  ├─ EmptyTitle
│  └─ EmptyDescription
└─ EmptyActions (optional)
```

| Part               | Role                                                               |
| ------------------ | ------------------------------------------------------------------ |
| `Empty`            | Root surface that owns padding, border, background, and alignment. |
| `EmptyIcon`        | Optional icon or small visual affordance container.                |
| `EmptyContent`     | Groups text content with tighter inner spacing.                    |
| `EmptyTitle`       | Main empty-state heading.                                          |
| `EmptyDescription` | Supporting description block for one or more text nodes.           |
| `EmptyActions`     | Optional wrapping row for actions.                                 |

## Composition

### When to stay high-level

Stay with the exported parts for most empty states:

- `Empty` for the surface and centered layout
- `EmptyContent` for the text stack
- `EmptyTitle` for the heading
- `EmptyDescription` for supporting copy
- `EmptyIcon` and `EmptyActions` only when they add value

This keeps the common path explicit and leaves room for richer descriptions, links, or custom
buttons without introducing a configuration API.

### When to go custom

`Empty` does not hide a lower-level primitive layer. Customization usually means changing structure
with normal React composition:

- omit `EmptyIcon` or `EmptyActions`
- add richer markup inside `EmptyDescription`
- change the heading level with `EmptyTitle as`
- apply `className` and `--empty-*` variables on the parts that own the visual concern

### Behavioral notes

- `EmptyTitle` renders `h3` by default and accepts `as` for document outline control.
- `EmptyDescription` renders a `div`, not a `p`, so it can safely contain block content.
- `EmptyActions` centers and wraps its children, so paired actions stay readable in narrow layouts.

## Styling

Style `Empty` through `className`, stable `data-slot="empty-*"` hooks, and the public `--empty-*`
CSS variables shown below.

### Styling hooks

Target the part that owns the visual concern:

- `data-slot="empty-root"` for surface layout, border, background, and shadow overrides
- `data-slot="empty-icon"` for icon container sizing, padding, and colors
- `data-slot="empty-content"` for text stack width and spacing
- `data-slot="empty-title"` and `data-slot="empty-description"` for typography
- `data-slot="empty-actions"` for action row spacing and alignment
