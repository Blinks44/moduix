## Choosing the right component

Use `Toast` for transient feedback that should appear without shifting the page layout.

- Choose **`Alert`** when the message should stay visible in the document flow until the user deals
  with it.
- Choose **`ToastRegion`** for page-level notifications such as saved, uploaded, synced, or failed
  actions.
- Choose **`ToastAnchoredRegion`** for short contextual confirmations near the control that
  triggered them.

## Basic

Start with `ToastProvider`, one trigger that uses `useToastManager()`, and `ToastRegion`. This is
the recommended path for ordinary stacked notifications.

<Preview>
  <ToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        ToastProvider,
        ToastRegion,
        useToastManager,
      } from "moduix";

      export function App() {
        return (
          <ToastProvider>
            <SaveButton />
            <ToastRegion />
          </ToastProvider>

      }

              toastManager.add({
                title: "Saved",
                description: "Changes were saved successfully.",
              })
            }
          >
            Save changes
          </Button>
        );
      }
    `}

## Anatomy

The normal consumer path is provider plus one or both regions:

```text
ToastProvider
├─ app content
├─ ToastRegion
│  └─ ToastRoot
│     └─ ToastContent
│        ├─ ToastTitle
│        ├─ ToastDescription
│        ├─ ToastAction
│        └─ ToastClose
└─ ToastAnchoredRegion
   └─ ToastPositioner
      └─ ToastRoot
         ├─ ToastArrow
         └─ ToastContent
            └─ ToastDescription
```

| Part                  | Role                                                             |
| --------------------- | ---------------------------------------------------------------- |
| `ToastProvider`       | Provides the stacked manager and anchored helper manager.        |
| `ToastRegion`         | Default stacked region with built-in portal and viewport.        |
| `ToastAnchoredRegion` | Default anchored region with built-in positioner flow.           |
| `ToastRoot`           | Visible toast surface.                                           |
| `ToastContent`        | Visible body layout wrapper.                                     |
| `ToastAction`         | Action button for stacked toasts when `actionProps` are present. |
| `ToastClose`          | Dismiss button for stacked toasts.                               |
| `ToastArrow`          | Default arrow for anchored toasts.                               |

## Composition

### When to stay high-level

Stay with `ToastProvider`, `ToastRegion`, `ToastAnchoredRegion`, `useToastManager()`, and
`useAnchoredToastManager()` for most applications.

Use `ToastRegion` for stacked page-level notifications and `ToastAnchoredRegion` for brief
contextual feedback near a trigger. `ToastRegion` already owns `ToastPortal` and `ToastViewport`.
`ToastAnchoredRegion` already owns the anchored viewport and `ToastPositioner`.

Use `renderToast` when the region wiring is correct but the visible markup needs to change.

### When to go custom

Drop to `ToastPortal`, `ToastViewport`, `ToastPositioner`, `ToastRoot`, and the other low-level
parts only when consumer code must own the viewport structure or render pipeline itself.

That is mainly useful when:

- stacked toasts need a custom viewport setup
- anchored toasts need richer content than the default description-only layout
- a shared app-level service outside React needs to enqueue toasts with `createToastManager()`

### Default props

| Surface               | Prop            | Default        | Values                                                                                |
| --------------------- | --------------- | -------------- | ------------------------------------------------------------------------------------- |
| `ToastRegion`         | `placement`     | `bottom-right` | `top-left`, `top-center`, `top-right`, `bottom-left`, `bottom-center`, `bottom-right` |
| `ToastRegion`         | `stackBehavior` | `stacked`      | `stacked`, `expanded`                                                                 |
| `ToastAnchoredRegion` | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |

### Behavioral notes

- Mount `ToastProvider` once around the part of the app that needs toast managers.
- `useToastManager()` drives stacked toasts rendered by `ToastRegion`.
- `useAnchoredToastManager().show()` requires an `anchor` element and reuses one toast per anchor.
- The default anchored renderer shows only `description`. Use `renderToast` for titles, icons, or
  additional controls.
- Built-in variant styling is available for `type="info"`, `success`, `warning`, and
  `destructive`. Other `type` values still pass through to `ToastRoot[data-type]`.

## Examples

### Anchored Toast

Use `ToastAnchoredRegion` and `useAnchoredToastManager()` for short contextual confirmations.

<Preview>
  <AnchoredToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        ToastAnchoredRegion,
        ToastProvider,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";

      export function App() {
        return (
          <ToastProvider>
            <CopyButton />
            <ToastAnchoredRegion />
          </ToastProvider>
        );
      }

      function CopyButton() {
        const buttonRef = useRef(null as HTMLButtonElement | null);
        const anchoredToast = useAnchoredToastManager();

        return (
          <Button
            ref={buttonRef}
            onClick={() => {
              if (!buttonRef.current) {
                return;
              }

              anchoredToast.show({
                anchor: buttonRef.current,
                description: "Copied",
                timeout: 1800,
              });
            }}
          >
            Copy
          </Button>
        );
      }
    `}

  </Preview.Code>
</Preview>

Pass `stackBehavior="expanded"` when the stacked viewport should stay fully readable.

<Preview>
  <ExpandedToastExample />

            <ToastRegion placement="bottom-right" stackBehavior="expanded" />

        return (
          <Button

              setCount(next);
              toastManager.add({
                title: \`Stacked toast \${next}\`,
                description: "Create several notifications to compare the expanded stack behavior.",
              });
            }}
          >
            Create expanded toast
          </Button>
        );
      }

### Custom Composition

Use `renderToast` when the region stays the same but the visible layout should change.

<Preview>
  <CustomToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        CloseIcon,
        InfoIcon,
        ToastClose,
        ToastContent,
        ToastDescription,
        ToastProvider,
        ToastRegion,
        ToastRoot,
        ToastTitle,
        useToastManager,
      } from "moduix";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <CustomToastButton />
            <ToastRegion
              renderToast={(toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>

                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast">
                      <CloseIcon className={styles.closeIcon} />
                    </ToastClose>
                  </ToastContent>
                </ToastRoot>
              )}
            />
          </ToastProvider>
        );
      }

      function CustomToastButton() {
        const toastManager = useToastManager();

        return (
          <Button
            onClick={() =>
              toastManager.add({
                title: "Custom composition",
                description: "Every important part stays composable without slot prop APIs.",
              })
            }
          >
            Create custom toast
          </Button>
        );
      }
    `}

### Custom Anchored Composition

Use `renderToast` on `ToastAnchoredRegion` when anchored feedback needs richer content than the
default description-only layout.

<Preview>
  <CustomAnchoredToastExample />

  <Preview.Code>
    {`
      import {
        type AnchoredToastOptions,
        Button,
        InfoIcon,
        ToastAnchoredRegion,
        ToastArrow,
        ToastContent,
        ToastDescription,
        ToastProvider,
        ToastRoot,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <AnchoredToastActions />
            <ToastAnchoredRegion
              renderToast={(toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>
                  <ToastArrow />
                  <ToastContent className={styles.customAnchoredContent}>
                    <InfoIcon className={styles.customAnchoredIcon} />
                    <ToastDescription />
                  </ToastContent>
                </ToastRoot>
              )}
            />
          </ToastProvider>
        );
      }

      function AnchoredToastActions() {
        const copyRef = useRef<HTMLButtonElement | null>(null);
        const saveRef = useRef<HTMLButtonElement | null>(null);
        const shareRef = useRef<HTMLButtonElement | null>(null);
        const anchoredToast = useAnchoredToastManager();

        const showAnchored = (
          anchor: HTMLButtonElement | null,
          description: string,
          positionerProps?: AnchoredToastOptions["positionerProps"],
        ) => {
          if (!anchor) {
            return;
          }

          anchoredToast.show({
            anchor,
            description,
            timeout: 1800,
            positionerProps,
          });
        };

        return (
          <>
            <Button ref={copyRef} onClick={() => showAnchored(copyRef.current, "Copied")}>
              Copy
            </Button>
            <Button
              ref={saveRef}
              variant="outline"
              onClick={() => showAnchored(saveRef.current, "Saved")}
            >
              Save
            </Button>
            <Button
              ref={shareRef}
              variant="secondary"
              onClick={() =>
                showAnchored(shareRef.current, "Shared", {
                  side: "bottom",
                })
              }
            >
              Share (bottom)
            </Button>
          </>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Manual Stacked Region

Most applications should stay on `ToastRegion`, but the stacked viewport can still be assembled
manually when consumer code must own it.

<Preview>
  <ManualToastCompositionExample />

  <Preview.Code>
    {`
      import {
        Button,
        CloseIcon,
        InfoIcon,
        ToastClose,
        ToastContent,
        ToastDescription,
        ToastPortal,
        ToastProvider,
        ToastRoot,
        ToastTitle,
        ToastViewport,
        useToastManager,
      } from "moduix";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <ManualToastButton />
            <ManualToastRegion />
          </ToastProvider>
        );
      }

      function ManualToastButton() {
        const toastManager = useToastManager();

        return (
          <Button
            onClick={() =>
              toastManager.add({
                title: "Manual viewport",
                description: "This stack is assembled from ToastPortal and ToastViewport.",
              })
            }
          >
            Create manual toast
          </Button>
        );
      }

      function ManualToastRegion() {
        const { toasts } = useToastManager();

        return (
          <ToastPortal>
            <ToastViewport placement="bottom-left">
              {toasts.map((toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>
                  <ToastContent className={styles.customContent}>
                    <InfoIcon className={styles.customIcon} />
                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast">
                      <CloseIcon className={styles.closeIcon} />
                    </ToastClose>
                  </ToastContent>
                </ToastRoot>
              ))}
            </ToastViewport>
          </ToastPortal>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Styling hooks

Use the part that owns the visual concern:

- `ToastRegion` and `ToastViewport` for viewport placement, width, and stack layout
- `ToastRoot` for surface, border, motion, and type-based variants
- `ToastContent`, `ToastTitle`, and `ToastDescription` for inner layout and typography
- `ToastAction` and `ToastClose` for interactive controls
- `ToastPositioner` and `ToastArrow` for anchored placement and arrow styling

The main public hooks include `data-slot="toast-portal"`, `toast-viewport`,
`toast-anchored-viewport`, `toast-root`, `toast-anchored-root`, `toast-content`,
`toast-anchored-content`, `toast-title`, `toast-description`, `toast-anchored-description`,
`toast-action`, `toast-close`, `toast-positioner`, and `toast-anchored-arrow`.

Useful state and variant hooks include:

- `data-placement` on `ToastViewport` and `ToastRoot`
- `data-stack-behavior` on `ToastViewport`, `ToastRoot`, and stacked `ToastContent`
- `data-type` on `ToastRoot`
- stacked-state attributes such as `data-expanded`, `data-starting-style`, `data-ending-style`,
  `data-limited`, and `data-swipe-direction`
- `data-behind` and `data-expanded` on stacked `ToastContent`
- `data-side` on `ToastArrow`
