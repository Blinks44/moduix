## Overview

`Toast` in moduix is a two-flow wrapper over Base UI:

- `ToastRegion` renders the normal stacked notification queue
- `ToastAnchoredRegion` renders short, contextual feedback next to an anchor element

The supported moduix API is provider + region composition first. Drop to low-level parts only when
the default region wiring is no longer enough.

## Default Path

Mount `ToastProvider` once near the app root, then render one or both regions depending on the
feedback flows the app needs:

- `ToastRegion` for stacked page-level notifications
- `ToastAnchoredRegion` for short feedback next to the triggering control

```tsx
import { ToastAnchoredRegion, ToastProvider, ToastRegion } from 'moduix';

export function App() {
  return (
    <ToastProvider>
      <AppShell />
      <ToastRegion />
      <ToastAnchoredRegion />
    </ToastProvider>
  );
}
```

Use `useToastManager()` to enqueue stacked toasts and `useAnchoredToastManager()` to enqueue
anchored ones. Most applications never need lower-level setup than this.

## Anatomy

```text
ToastProvider
├─ app content
├─ ToastRegion
│  ├─ ToastPortal
│  └─ ToastViewport
│     └─ ToastRoot
│        └─ ToastContent
│           ├─ ToastTitle
│           ├─ ToastDescription
│           ├─ ToastAction
│           └─ ToastClose
└─ ToastAnchoredRegion
   └─ ToastPositioner
      └─ ToastRoot
         ├─ ToastArrow
         └─ ToastContent
            └─ ToastDescription
```

| Part                  | Role                                                                |
| --------------------- | ------------------------------------------------------------------- |
| `ToastProvider`       | Provides the stacked toast manager and the anchored toast helper.   |
| `ToastRegion`         | Default stacked toast renderer with built-in portal and viewport.   |
| `ToastAnchoredRegion` | Default anchored toast renderer with built-in positioner and arrow. |
| `ToastPortal`         | Low-level portal wrapper for manual stacked composition.            |
| `ToastViewport`       | Low-level viewport wrapper for manual stacked composition.          |
| `ToastPositioner`     | Low-level anchored positioner wrapper.                              |
| `ToastRoot`           | Visible toast surface.                                              |
| `ToastContent`        | Content wrapper for the visible toast body.                         |
| `ToastTitle`          | Renders `toast.title`.                                              |
| `ToastDescription`    | Renders `toast.description`.                                        |
| `ToastAction`         | Renders `toast.actionProps` when present.                           |
| `ToastClose`          | Dismiss button with the default close icon or custom children.      |
| `ToastArrow`          | Default anchored arrow.                                             |

## Composition

`Toast` keeps the high-level API on the region components:

| Surface               | Prop            | Default        | Values                                                                                |
| --------------------- | --------------- | -------------- | ------------------------------------------------------------------------------------- |
| `ToastRegion`         | `placement`     | `bottom-right` | `top-left`, `top-center`, `top-right`, `bottom-left`, `bottom-center`, `bottom-right` |
| `ToastRegion`         | `stackBehavior` | `stacked`      | `stacked`, `expanded`                                                                 |
| `ToastRegion`         | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |
| `ToastAnchoredRegion` | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |

Composition rules:

- use `ToastRegion` for the normal stacked flow
- use `ToastAnchoredRegion` for contextual feedback near a control
- use Base UI `type` for toast variants; moduix styles `info`, `success`, `warning`, and `destructive` out of the box, and custom types can still be targeted through `data-type`
- the default anchored renderer shows only `description`, so use `renderToast` for richer anchored content
- use `renderToast` when the region setup stays the same but the visible markup needs to change
- drop to `ToastPortal` and `ToastViewport` only when the stacked viewport structure itself must be owned by consumer code

Use `createToastManager()` only when a service outside React needs to enqueue stacked toasts.

### Anchored Toast

<Preview>
  <AnchoredToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        ToastAnchoredRegion,
        ToastProvider,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";

      export function App() {
        return (
          <ToastProvider>
            <CopyButton />
            <ToastAnchoredRegion />
          </ToastProvider>
        );
      }

      function CopyButton() {
        const buttonRef = useRef(null as HTMLButtonElement | null);
        const anchoredToast = useAnchoredToastManager();

        return (
          <Button
            ref={buttonRef}
            onClick={() => {
              if (!buttonRef.current) {
                return;
              }

              anchoredToast.show({
                anchor: buttonRef.current,
                description: "Copied",
                timeout: 1800,
              });
            }}
          >
            Copy
          </Button>
        );
      }
    `}

  </Preview.Code>

### Variants

`ToastRegion` stays shared. Pass `type` when creating the toast. moduix styles the same semantic
variants as `Alert` out of the box: `info`, `success`, `warning`, and `destructive`.

For app-specific toast types, keep using the rendered `data-type` hook and override `--toast-*`
variables in your own CSS.

<Preview>
  <ToastVariantsExample />

  <Preview.Code>
    {`
      import { Button, ToastProvider, ToastRegion, useToastManager } from "moduix";

      export function App() {
        return (
          <ToastProvider>
            <ToastVariantButtons />
            <ToastRegion />
          </ToastProvider>
        );
      }

      function ToastVariantButtons() {
        const toastManager = useToastManager();

        return (
          <>
            <Button
              onClick={() =>
                toastManager.add({
                  type: "info",
                  title: "Heads up",
                  description: "New teammates can see the shared workspace now.",
                })
              }
            >
              Info toast
            </Button>
            <Button
              onClick={() =>
                toastManager.add({
                  type: "success",
                  title: "Saved",
                  description: "The document is available to everyone with access.",
                })
              }
            >
              Success toast
            </Button>
            <Button
              variant="outline"
              onClick={() =>
                toastManager.add({
                  type: "warning",
                  title: "Storage almost full",
                  description: "Archive old uploads before the next sync.",
                })
              }
            >
              Warning toast
            </Button>
            <Button
              variant="outline"
              onClick={() =>
                toastManager.add({
                  type: "destructive",
                  title: "Publish failed",
                  description: "Review the required fields and try again.",
                })
              }
            >
              Destructive toast

          </>

### Always Expanded

Pass `stackBehavior="expanded"` when the stack should always stay readable.

<Preview>
  <ExpandedToastExample />

            <ExpandedToastButton />
            <ToastRegion placement="bottom-right" stackBehavior="expanded" />

          <Button

              setCount(next);
              toastManager.add({
                title: \`Stacked toast \${next}\`,
                description: "Create several notifications to compare the expanded stack behavior.",
              });
            }}
          >
            Create expanded toast
          </Button>
        );
      }

## Custom Composition

Use `renderToast` when the region setup stays the same but the visible layout needs to change.

### Custom Stacked Toast

<Preview>
  <CustomToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        CloseIcon,
        InfoIcon,
        ToastClose,
        ToastContent,
        ToastDescription,
        ToastProvider,
        ToastRegion,
        ToastRoot,
        ToastTitle,
        useToastManager,
      } from "moduix";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <CustomToastButton />
            <ToastRegion
              renderToast={(toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>
                  <ToastContent className={styles.customContent}>
                    <InfoIcon className={styles.customIcon} />
                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast">
                      <CloseIcon className={styles.closeIcon} />
                    </ToastClose>
                  </ToastContent>
                </ToastRoot>
              )}
            />
          </ToastProvider>
        );
      }

      function CustomToastButton() {
        const toastManager = useToastManager();

        return (
          <Button
            onClick={() =>
              toastManager.add({
                title: "Custom composition",
                description: "Every important part stays composable without slot prop APIs.",
              })
            }
          >
            Create custom toast
          </Button>
        );
      }
    `}

### Custom Anchored Toast

<Preview>
  <CustomAnchoredToastExample />

  <Preview.Code>
    {`
      import {
        type AnchoredToastOptions,
        Button,
        InfoIcon,
        ToastAnchoredRegion,
        ToastArrow,
        ToastContent,
        ToastDescription,
        ToastProvider,
        ToastRoot,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <AnchoredToastActions />
            <ToastAnchoredRegion
              renderToast={(toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>
                  <ToastArrow />
                  <ToastContent className={styles.customAnchoredContent}>
                    <InfoIcon className={styles.customAnchoredIcon} />
                    <ToastDescription />
                  </ToastContent>
                </ToastRoot>
              )}
            />
          </ToastProvider>
        );
      }

      function AnchoredToastActions() {
        const copyRef = useRef<HTMLButtonElement | null>(null);
        const saveRef = useRef<HTMLButtonElement | null>(null);
        const shareRef = useRef<HTMLButtonElement | null>(null);
        const anchoredToast = useAnchoredToastManager();

        const showAnchored = (
          anchor: HTMLButtonElement | null,
          description: string,
          positionerProps?: AnchoredToastOptions["positionerProps"],
        ) => {
          if (!anchor) {
            return;
          }

          anchoredToast.show({
            anchor,
            description,
            timeout: 1800,
            positionerProps,
          });
        };

        return (
          <>
            <Button ref={copyRef} onClick={() => showAnchored(copyRef.current, "Copied")}>
              Copy
            </Button>
            <Button
              ref={saveRef}
              variant="outline"
              onClick={() => showAnchored(saveRef.current, "Saved")}
            >
              Save
            </Button>
            <Button
              ref={shareRef}
              variant="secondary"
              onClick={() =>
                showAnchored(shareRef.current, "Shared", {
                  side: "bottom",
                })
              }
            >
              Share (bottom)
            </Button>
          </>
        );
      }

    `}

  </Preview.Code>
</Preview>

### Manual Stacked Region

Most applications should stay on `ToastRegion`, but the stacked viewport can still be composed
manually when needed.

<Preview>
  <ManualToastCompositionExample />

  <Preview.Code>
    {`
      import {
        Button,
        CloseIcon,
        InfoIcon,
        ToastClose,
        ToastContent,
        ToastDescription,
        ToastPortal,
        ToastProvider,
        ToastRoot,
        ToastTitle,
        ToastViewport,
        useToastManager,
      } from "moduix";
      import styles from "./toast.module.css";

      export function App() {
        return (
          <ToastProvider>
            <ManualToastButton />
            <ManualToastRegion />
          </ToastProvider>
        );
      }

      function ManualToastButton() {
        const toastManager = useToastManager();

        return (
          <Button
            onClick={() =>
              toastManager.add({
                title: "Manual viewport",
                description: "This stack is assembled from ToastPortal and ToastViewport.",
              })
            }
          >
            Create manual toast
          </Button>
        );
      }

      function ManualToastRegion() {
        const { toasts } = useToastManager();

        return (
          <ToastPortal>
            <ToastViewport placement="bottom-left">
              {toasts.map((toast) => (
                <ToastRoot key={toast.id} toast={toast} className={styles.customToast}>
                  <ToastContent className={styles.customContent}>
                    <InfoIcon className={styles.customIcon} />
                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast">
                      <CloseIcon className={styles.closeIcon} />
                    </ToastClose>
                  </ToastContent>
                </ToastRoot>
              ))}
            </ToastViewport>
          </ToastPortal>
        );
      }
    `}

  </Preview.Code>
</Preview>
