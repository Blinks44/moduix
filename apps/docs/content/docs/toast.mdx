## Overview

`Toast` has two user-facing flows:

- `ToastRegion` for page-level stacked notifications
- `ToastAnchoredRegion` for contextual feedback attached to a specific control

Both flows are mounted under one `ToastProvider`.

## App Integration

Mount `ToastProvider` once near the app root. Render both regions once if the app uses both toast
styles.

```tsx
import { ToastAnchoredRegion, ToastProvider, ToastRegion } from 'moduix';

export function App() {
  return (
    <ToastProvider>
      <AppShell />
      <ToastRegion />
      <ToastAnchoredRegion />
    </ToastProvider>
  );
}
```

How it maps in the app:

- `useToastManager()` sends stacked toasts to `ToastRegion`
- `useAnchoredToastManager()` sends anchored toasts to `ToastAnchoredRegion`
- both regions work independently under the same provider

This is the recommended layout for root providers, app shells, and application wrappers.

## Anchored Toast

Use `ToastAnchoredRegion` when feedback should appear next to the control that triggered it.

<Preview>
  <AnchoredToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        ToastAnchoredRegion,
        ToastProvider,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";

      export function App() {
        return (
          <ToastProvider>
            <CopyButton />
            <ToastAnchoredRegion />
          </ToastProvider>
        );
      }

      function CopyButton() {
        const buttonRef = useRef(null as HTMLButtonElement | null);
        const anchoredToast = useAnchoredToastManager();

        return (
          <Button
            ref={buttonRef}
            onClick={() => {
              if (!buttonRef.current) {
                return;
              }

              anchoredToast.show({
                anchor: buttonRef.current,
                description: "Copied",
                timeout: 1800,
              });
            }}
          >
            Copy
          </Button>
        );
      }
    `}

  </Preview.Code>
</Preview>

## Parts

The library keeps a short default path, but the visible toast parts are still exported for custom
layout.

```text
ToastProvider
├─ app content
├─ ToastRegion
│  ├─ ToastPortal
│  └─ ToastViewport
│     └─ ToastRoot
│        └─ ToastContent
│           ├─ ToastTitle
│           ├─ ToastDescription
│           ├─ ToastAction
│           └─ ToastClose
└─ ToastAnchoredRegion
   └─ ToastPositioner
      └─ ToastRoot
         ├─ ToastArrow
         └─ ToastContent
            └─ ToastDescription
```

| Part                  | Role                                                                |
| --------------------- | ------------------------------------------------------------------- |
| `ToastProvider`       | Provides the stacked toast manager and the anchored toast helper.   |
| `ToastRegion`         | Default stacked toast renderer with built-in portal and viewport.   |
| `ToastAnchoredRegion` | Default anchored toast renderer with built-in positioner and arrow. |
| `ToastPortal`         | Low-level portal wrapper for manual stacked composition.            |
| `ToastViewport`       | Low-level viewport wrapper for manual stacked composition.          |
| `ToastPositioner`     | Low-level anchored positioner wrapper.                              |
| `ToastRoot`           | Visible toast surface.                                              |
| `ToastContent`        | Content wrapper for the visible toast body.                         |
| `ToastTitle`          | Renders `toast.title`.                                              |
| `ToastDescription`    | Renders `toast.description`.                                        |
| `ToastAction`         | Renders `toast.actionProps` when present.                           |
| `ToastClose`          | Dismiss button with the default close icon or custom children.      |
| `ToastArrow`          | Default anchored arrow.                                             |

## Defaults

Small sugar stays on the region components only.

| Surface               | Prop            | Default        | Values                                                                                |
| --------------------- | --------------- | -------------- | ------------------------------------------------------------------------------------- |
| `ToastRegion`         | `placement`     | `bottom-right` | `top-left`, `top-center`, `top-right`, `bottom-left`, `bottom-center`, `bottom-right` |
| `ToastRegion`         | `stackBehavior` | `stacked`      | `stacked`, `expanded`                                                                 |
| `ToastRegion`         | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |
| `ToastAnchoredRegion` | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |

Use `createToastManager()` only when a service outside React needs to enqueue stacked toasts.

## Manual Composition

Most applications should use `ToastRegion` and `ToastAnchoredRegion`. Drop to low-level parts only
when you need to own the stacked viewport structure directly.

### Manual Stacked Region

<Preview>
  <ManualToastCompositionExample />

  <Preview.Code>
    {`
      function ManualToastRegion() {
        const { toasts } = useToastManager();

        return (
          <ToastPortal>
            <ToastViewport placement="bottom-left">
              {toasts.map((toast) => (
                <ToastRoot key={toast.id} toast={toast}>
                  <ToastContent>
                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast" />
                  </ToastContent>
                </ToastRoot>
              ))}
            </ToastViewport>
          </ToastPortal>
        );
      }
    `}

  </Preview.Code>
