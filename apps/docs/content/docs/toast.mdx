## Default Path

Mount `ToastProvider` once near the app root, then render one or both regions depending on the
feedback flows the app needs:

- `ToastRegion` for stacked page-level notifications
- `ToastAnchoredRegion` for short feedback next to the triggering control

```tsx
import { ToastAnchoredRegion, ToastProvider, ToastRegion } from 'moduix';

export function App() {
  return (
    <ToastProvider>
      <AppShell />
      <ToastRegion />
      <ToastAnchoredRegion />
    </ToastProvider>
  );
}
```

Use `useToastManager()` to enqueue stacked toasts and `useAnchoredToastManager()` to enqueue
anchored ones. Most applications never need lower-level setup than this.

## Anatomy

```text
ToastProvider
├─ app content
├─ ToastRegion
│  ├─ ToastPortal
│  └─ ToastViewport
│     └─ ToastRoot
│        └─ ToastContent
│           ├─ ToastTitle
│           ├─ ToastDescription
│           ├─ ToastAction
│           └─ ToastClose
└─ ToastAnchoredRegion
   └─ ToastPositioner
      └─ ToastRoot
         ├─ ToastArrow
         └─ ToastContent
            └─ ToastDescription
```

| Part                  | Role                                                                |
| --------------------- | ------------------------------------------------------------------- |
| `ToastProvider`       | Provides the stacked toast manager and the anchored toast helper.   |
| `ToastRegion`         | Default stacked toast renderer with built-in portal and viewport.   |
| `ToastAnchoredRegion` | Default anchored toast renderer with built-in positioner and arrow. |
| `ToastPortal`         | Low-level portal wrapper for manual stacked composition.            |
| `ToastViewport`       | Low-level viewport wrapper for manual stacked composition.          |
| `ToastPositioner`     | Low-level anchored positioner wrapper.                              |
| `ToastRoot`           | Visible toast surface.                                              |
| `ToastContent`        | Content wrapper for the visible toast body.                         |
| `ToastTitle`          | Renders `toast.title`.                                              |
| `ToastDescription`    | Renders `toast.description`.                                        |
| `ToastAction`         | Renders `toast.actionProps` when present.                           |
| `ToastClose`          | Dismiss button with the default close icon or custom children.      |
| `ToastArrow`          | Default anchored arrow.                                             |

## Composition

`Toast` keeps the high-level API on the region components:

| Surface               | Prop            | Default        | Values                                                                                |
| --------------------- | --------------- | -------------- | ------------------------------------------------------------------------------------- |
| `ToastRegion`         | `placement`     | `bottom-right` | `top-left`, `top-center`, `top-right`, `bottom-left`, `bottom-center`, `bottom-right` |
| `ToastRegion`         | `stackBehavior` | `stacked`      | `stacked`, `expanded`                                                                 |
| `ToastRegion`         | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |
| `ToastAnchoredRegion` | `renderToast`   | default markup | `(toast, index) => ReactNode`                                                         |

Composition rules:

- use `ToastRegion` for the normal stacked flow
- use `ToastAnchoredRegion` for contextual feedback near a control
- use `renderToast` when the region setup stays the same but the visible markup needs to change
- drop to `ToastPortal` and `ToastViewport` only when the stacked viewport structure itself must be owned by consumer code

Use `createToastManager()` only when a service outside React needs to enqueue stacked toasts.

## Examples

### Anchored Toast

<Preview>
  <AnchoredToastExample />

  <Preview.Code>
    {`
      import {
        Button,
        ToastAnchoredRegion,
        ToastProvider,
        useAnchoredToastManager,
      } from "moduix";
      import { useRef } from "react";

      export function App() {
        return (
          <ToastProvider>
            <CopyButton />
            <ToastAnchoredRegion />
          </ToastProvider>
        );
      }

      function CopyButton() {
        const buttonRef = useRef(null as HTMLButtonElement | null);
        const anchoredToast = useAnchoredToastManager();

        return (
          <Button
            ref={buttonRef}
            onClick={() => {
              if (!buttonRef.current) {
                return;
              }

              anchoredToast.show({
                anchor: buttonRef.current,
                description: "Copied",
                timeout: 1800,
              });
            }}
          >
            Copy
          </Button>
        );
      }
    `}

  </Preview.Code>
</Preview>

Use `renderToast` when the region setup stays the same but the visible layout needs to change.

### Manual Stacked Region

Most applications should stay on `ToastRegion`, but the stacked viewport can still be composed
manually when needed.

<Preview>
  <ManualToastCompositionExample />

  <Preview.Code>
    {`
      function ManualToastRegion() {
        const { toasts } = useToastManager();

        return (
          <ToastPortal>
            <ToastViewport placement="bottom-left">
              {toasts.map((toast) => (
                <ToastRoot key={toast.id} toast={toast}>
                  <ToastContent>
                    <ToastTitle />
                    <ToastDescription />
                    <ToastClose aria-label="Close toast" />
                  </ToastContent>
                </ToastRoot>
              ))}
            </ToastViewport>
          </ToastPortal>
        );
      }
    `}

  </Preview.Code>
</Preview>
