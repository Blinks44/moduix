## Basic

Create a toaster store with `createToaster()`, enqueue messages through that store, and render the
store with `Toaster`.

<Preview>
  <ToastExample />

  <Preview.Code>
    {`
      import { Button, Portal, Toast, Toaster, createToaster } from "moduix";

      const toaster = createToaster({ placement: "bottom-end", overlap: true, gap: 24 });

      export function App() {
        return (
          <>
            <Button
              onClick={() =>
                toaster.create({
                  title: "Scheduled for tomorrow",
                  description: "Your meeting has been scheduled for tomorrow at 10am.",
                  type: "info",
                })
              }
            >
              Schedule meeting
            </Button>
            <Portal>
              <Toaster toaster={toaster}>
                {(toast) => (
                  <Toast key={toast.id}>
                    <Toast.Title />
                    <Toast.Description />
                    {toast.closable !== false ? <Toast.CloseTrigger /> : null}
                  </Toast>
                )}
              </Toaster>
            </Portal>
          </>
        );
      }
    `}

  </Preview.Code>

## Anatomy

```text
createToaster()
└─ Portal
   └─ Toaster
      └─ Toast.Root
         ├─ Toast.Context (optional render-prop access)
         ├─ Toast.Title
         ├─ Toast.Description
         ├─ Toast.ActionTrigger
         └─ Toast.CloseTrigger
```

| Part                        | Role                                                                                 |
| --------------------------- | ------------------------------------------------------------------------------------ |
| `Toast` / `Toast.Root`      | Visible toast surface connected to the current Ark toast context.                    |
| `Toaster` / `Toast.Toaster` | Renders a toaster store and owns the Ark toast group.                                |
| `Portal`                    | Shared portal primitive for rendering the toaster group outside local layout.        |
| `Toast.Title`               | Title part. Defaults to the current toast `title` when no children are passed.       |
| `Toast.Description`         | Description part. Defaults to the current toast `description`.                       |
| `Toast.ActionTrigger`       | Action button wired to `toast.action.onClick`. Pass `toast.action.label` as content. |
| `Toast.CloseTrigger`        | Dismiss button. Renders `CloseButton.Root` by default.                               |
| `Toast.Context`             | Reads Ark toast state through a render prop.                                         |

## Composition

### Recommended composition

Create a stable toaster store with `createToaster(options)`, then render it once with `Toaster`.
The store controls placement, overlap, gap, max queue size, duration, remove delay, idle pausing,
and the `Alt+T` focus hotkey.

Import `Portal` separately from `moduix` and wrap `Toaster` with it when the group should render at
the document body level. Toast does not expose a toast-specific portal alias.

Use `Toast` as the short root form in application code. It is equivalent to `Toast.Root` and keeps
the Ark namespace available for the child parts.

### When to go custom

Use the `Toaster` render prop when each toast needs custom layout, icons, or conditional controls.
Keep `Toast.Root` as the outer item so Ark can preserve stacking, height measurement, dismiss
lifecycle, pausing, keyboard focus, and type/state attributes.

Use `Toast.Context` or `useToastContext()` inside custom child parts when the rendered content needs
access to `type`, `placement`, `visible`, `paused`, `dismiss()`, `pause()`, or `resume()`.

All exported Ark parts support `asChild`; custom children must remain semantic and preserve button
behavior for `ActionTrigger` and `CloseTrigger`.

### Default props

| Prop                        | Default              | Notes                                                                                      |
| --------------------------- | -------------------- | ------------------------------------------------------------------------------------------ |
| `createToaster.gap`         | `16`                 | Gap between toast items.                                                                   |
| `createToaster.max`         | `24`                 | New toasts are queued after the visible stack reaches this limit.                          |
| `createToaster.hotkey`      | `["altKey", "KeyT"]` | Moves focus to the toast group.                                                            |
| `createToaster.offsets`     | `"1rem"`             | Safe-edge offset for the toast group.                                                      |
| `createToaster.overlap`     | `false`              | Ark stack mode. Use `true` for overlapped stacks.                                          |
| `createToaster.placement`   | `"bottom"`           | Use Ark placements: `top-start`, `top`, `top-end`, `bottom-start`, `bottom`, `bottom-end`. |
| `createToaster.removeDelay` | `200`                | Keeps closed toasts mounted long enough for exit transitions.                              |

### Behavioral notes

- Use `toaster.create()`, `success()`, `error()`, `warning()`, `info()`, `loading()`,
  `promise()`, `update()`, `dismiss()`, `remove()`, and `dismissByPlacement()` directly from the
  Ark toaster store.
- `toast.action` uses `{ label, onClick }`, rendered by `Toast.ActionTrigger`.
- `toast.duration` controls one toast; `createToaster({ duration })` controls the default.
- `toast.onStatusChange(details)` receives Ark status changes for `visible`, `dismissing`, and
  `unmounted`.
- `Toast.Root` exposes Ark state attributes such as `data-state`, `data-type`, `data-placement`,
  `data-side`, `data-mounted`, `data-paused`, `data-first`, `data-sibling`, `data-stack`, and
  `data-overlap`.

### Placement

Choose one of Ark's six placements when the toast group needs to appear on a specific viewport edge.

<Preview>
  <PlacementToastExample />

  <Preview.Code>
    {`
      import { Button, Portal, Toast, Toaster, createToaster, type ToastPlacement } from "moduix";
      import { useState } from "react";

      const toasters = {
        "top-start": createToaster({ placement: "top-start", overlap: true, gap: 16 }),
        top: createToaster({ placement: "top", overlap: true, gap: 16 }),
        "top-end": createToaster({ placement: "top-end", overlap: true, gap: 16 }),
        "bottom-start": createToaster({ placement: "bottom-start", overlap: true, gap: 16 }),
        bottom: createToaster({ placement: "bottom", overlap: true, gap: 16 }),
        "bottom-end": createToaster({ placement: "bottom-end", overlap: true, gap: 16 }),
      };

      export function App() {
        const [placement, setPlacement] = useState("bottom-end" as ToastPlacement);
        const toaster = toasters[placement];

        return (
          <>
            {placements.map((item) => (
              <Button key={item} onClick={() => setPlacement(item)}>
                {item}
              </Button>
            ))}
            <Button
              onClick={() =>
                toaster.info({
                  title: "Notification",
                  description: \`This toast appears at \${placement}.\`,
                })
              }
            >
              Show {placement}
            </Button>
            {placements.map((item) => (
              <Portal key={item}>
                <Toaster toaster={toasters[item]}>
                  {(toast) => (
                    <Toast key={toast.id}>
                      <Toast.Title />
                      <Toast.Description />
                      {toast.closable !== false ? <Toast.CloseTrigger /> : null}
                    </Toast>
                  )}
                </Toaster>
              </Portal>

### Update

Keep the created toast id and pass it to `toaster.update()` when an existing notification should
change in place.

<Preview>
  <UpdateToastExample />

      import { useRef } from "react";

      const toaster = createToaster({ placement: "bottom-end", overlap: true, gap: 24 });

      export function App() {
        const idRef = useRef<string | undefined>(undefined);

        return (
          <>
            <Button
              onClick={() => {
                idRef.current = toaster.create({
                  title: "Sending message...",
                  description: "Please wait while we deliver your message.",
                  type: "loading",
                });

            >
              Send message
            </Button>
            <Button

              }}
            >
              Mark as sent
            </Button>
            <Portal>
              <Toaster toaster={toaster}>
                {(toast) => (
                  <Toast key={toast.id}>
                    <Toast.Title />
                    <Toast.Description />

### Styling hooks

Use `Toast.Root` for surface, border, motion, type-based variants, and Ark runtime variables.
Use `Toast.Title`, `Toast.Description`, `Toast.ActionTrigger`, and `Toast.CloseTrigger` for inner
typography and controls.

Important Ark hooks:

- `[data-scope='toast'][data-part='group']` on `Toaster`
- `[data-scope='toast'][data-part='root']` on `Toast.Root`
- `[data-scope='toast'][data-part='title']` on `Toast.Title`
- `[data-scope='toast'][data-part='description']` on `Toast.Description`
- `[data-scope='toast'][data-part='actionTrigger']` on `Toast.ActionTrigger`
- `[data-scope='toast'][data-part='closeTrigger']` on `Toast.CloseTrigger`
- `--x`, `--y`, `--scale`, `--z-index`, `--height`, `--opacity`, and `--gap` on `Toast.Root`

moduix also adds stable hooks: `data-slot="toast-toaster"`, `toast-root`, `toast-title`,
`toast-description`, `toast-action-trigger`, and `toast-close-trigger`.
