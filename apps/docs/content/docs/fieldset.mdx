## Anatomy

`Fieldset` groups related controls under one semantic legend. Keep the visible group label in
`FieldsetLegend`, then place related `Field` blocks (or composed rows like radio options) inside the
same root so native fieldset semantics and disabled behavior stay consistent.

```text
Fieldset
├─ FieldsetLegend
└─ grouped content
   ├─ Field
   │  ├─ FieldLabel
   │  ├─ FieldControl
   │  └─ FieldError
   └─ Field / FieldItem rows (optional)
```

```tsx
<Fieldset>
  <FieldsetLegend>Billing details</FieldsetLegend>

  <Field validationMode="onBlur">
    <FieldLabel>Company</FieldLabel>
    <FieldControl required placeholder="Enter company name" />
    <FieldError match="valueMissing">Please enter company name.</FieldError>
  </Field>
</Fieldset>
```

| Part             | Role                                                                                                  |
| ---------------- | ----------------------------------------------------------------------------------------------------- |
| `Fieldset`       | Root semantic group. Renders native `fieldset`, carries shared disabled state, and supports `render`. |
| `FieldsetLegend` | Group label announced for the whole fieldset.                                                         |

`Fieldset` has only two public parts: `Fieldset` and `FieldsetLegend`. There are no extra slot APIs
or exported helper types to learn. In most cases, keep the native structure and style the visible
parts with `className` and CSS variables.

## Composition

`Fieldset` is a thin wrapper over the Base UI primitive. By default it renders a native
`fieldset`, forwards Base UI props directly, and exposes only the visible parts that consumers
actually style.

`Fieldset` and `FieldsetLegend` are the whole component surface. Pass `className` directly to the
part you want to customize.

### Defaults

| Prop        | Default    | Values                                       |
| ----------- | ---------- | -------------------------------------------- |
| `disabled`  | `false`    | `true`, `false`                              |
| `render`    | `fieldset` | Any valid React element or render callback   |
| `className` | _not set_  | `string` or `(state) => string \| undefined` |

```tsx
<Fieldset className={styles.fieldset}>
  <FieldsetLegend className={styles.legend}>Billing details</FieldsetLegend>
</Fieldset>
```

Use `render` only when the fieldset semantics need to be composed with another root, such as
`RadioGroup` or `CheckboxGroup`. That keeps the normal path simple while preserving the Base UI
escape hatch for grouped selection controls.

## Examples

### Disabled

Use `disabled` on `Fieldset` to prevent interaction with every native control inside the group.

<Preview>
  <DisabledFieldsetExample />

        FieldControl,

          </Fieldset>

      }
    `}

## Custom Composition

Use `render` when the semantic fieldset also needs to be another Base UI root. This is the main
advanced composition path for radio groups and checkbox groups inside forms.

<Preview>
  <CustomCompositionFieldsetExample />

  <Preview.Code>
    {`
      import {
        Field,
        FieldItem,
        FieldLabel,
        Fieldset,
        FieldsetLegend,
        Radio,
        RadioGroup,
        RadioLabel,
      } from "moduix";

      export function StorageTypeFieldset() {
        return (
          <Field name="storageType">
            <Fieldset render={<RadioGroup defaultValue="ssd" className={styles.choiceGroup} />}>
              <FieldsetLegend>Storage type</FieldsetLegend>

              {storageTypes.map((item) => (
                <FieldItem key={item.value}>
                  <FieldLabel className={styles.choiceRow}>
                    <Radio value={item.value} />
                    <RadioLabel>{item.label}</RadioLabel>
                  </FieldLabel>
                </FieldItem>
              ))}
            </Fieldset>
          </Field>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const storageTypes = [
        { value: "ssd", label: "SSD" },
        { value: "hdd", label: "HDD" },
      ];
    `}

  </Preview.Data>

  <Preview.CSS>
    {`
      .choiceGroup {
        gap: var(--spacing-2);
      }

      .choiceRow {
        align-items: center;
        gap: var(--spacing-3);
      }
    `}

  </Preview.CSS>
</Preview>
