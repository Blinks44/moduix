## Overview

`Skeleton` is a single loading placeholder block. It does not wrap a Base UI primitive.

Use it as a thin styled `div`, then compose loading layouts with `Stack` or your own markup. The
root is marked `aria-hidden` because it represents loading surfaces, not readable content.

## Composition

`Skeleton` stays intentionally small:

- use `width` and `height` for line and block placeholders
- use `size` with `shape="circle"` for avatar-style placeholders
- use `radius` when a surface should match a card or media container
- use `animated={false}` when motion is unnecessary
- use `className`, `style`, and `--skeleton-*` variables for styling overrides

### Defaults

| Prop       | Default | Values                             |
| ---------- | ------- | ---------------------------------- |
| `height`   | `1rem`  | CSS length (`number` = `px`)       |
| `width`    | -       | CSS length (`number` = `px`)       |
| `radius`   | -       | CSS length (`number` = `px`)       |
| `size`     | -       | Sets `width` and `height` together |
| `shape`    | `rect`  | `rect`, `circle`                   |
| `animated` | `true`  | `true`, `false`                    |

## Examples

### Card

Use a larger block plus shorter text lines to reserve card layout while content is loading.

<Preview>
  <SkeletonCardExample />

  <Preview.Code>
    {`

      import styles from "./skeleton-card-demo.module.css";

      export function SkeletonCardDemo() {
        return (
          <Stack gap={16} className={styles.card}>
            <Skeleton height={148} radius="var(--radius-lg)" />
            <Stack gap={12}>
              <Skeleton width="70%" height={20} />
              <Skeleton height={14} />
              <Skeleton width="82%" height={14} />
            </Stack>
          </Stack>
        );
      }
    `}

  </Preview.Code>

### Responsive Layout

Keep responsive layout logic in your wrapper markup while `Skeleton` stays a visual building block.

<Preview>
  <SkeletonCompositionExample />

  <Preview.Code>
