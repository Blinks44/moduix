## API Reference

`Skeleton` is a single loading placeholder block and does not wrap a Base UI primitive.
Use it as a thin styled `div`, then compose card, media, and list layouts with `Stack` or your own
markup.
The root is marked `aria-hidden` because it describes loading surfaces, not readable content.

## Anatomy

`Skeleton` is a single visual block. Layout stays in `Stack` or your own markup.

```text
Stack
├─ Skeleton
├─ Skeleton
└─ Skeleton
```

```tsx
<Stack gap={10}>
  <Skeleton height={18} />
  <Skeleton width="86%" height={18} />
  <Skeleton width="64%" height={18} />
</Stack>
```

| Part       | Role                                                                                                 |
| ---------- | ---------------------------------------------------------------------------------------------------- |
| `Skeleton` | Base placeholder block. Use size props for the common case and `style`/`className` for custom cases. |

## Composition

Use `Skeleton` directly for text lines and custom blocks. For most flex layouts, pair it with
`Stack`. Use `width`, `height`, `radius`, `size`, and `shape` for fast setup, then fall back to
`style` when you need custom CSS values or overrides.

`Skeleton` accepts standard `div` props plus `animated`. The component exposes `--skeleton-bg`,
`--skeleton-radius`, and `--skeleton-animation` for targeted styling.

### Defaults

| Prop       | Default | Values                             |
| ---------- | ------- | ---------------------------------- |
| `height`   | `1rem`  | CSS length (`number` = `px`)       |
| `width`    | -       | CSS length (`number` = `px`)       |
| `radius`   | -       | CSS length (`number` = `px`)       |
| `size`     | -       | Sets `width` and `height` together |
| `shape`    | `rect`  | `rect`, `circle`                   |
| `animated` | `true`  | `true`, `false`                    |

## Examples

### Card

Use a larger `Skeleton` block with smaller line skeletons to reserve the shape of card content.

<Preview>
  <SkeletonCardExample />

  <Preview.Code>

### Composition

Use `Stack` for the common flex path. For responsive layout rules or more custom structure, keep
that logic in your own wrapper markup while `Skeleton` stays a thin visual block.

<Preview>
  <SkeletonCompositionExample />

  <Preview.Code>
    {`
      import { Skeleton, Stack } from "moduix";
      import styles from "./skeleton-layout-demo.module.css";

      export function SkeletonCompositionDemo() {
        return (
          <Stack gap={12} className={styles.layoutExample}>
            <Stack gap={12} className={styles.listRow}>
              <Skeleton width={72} height={48} />
              <Stack gap={8} className={styles.growStack}>
                <Skeleton width="62%" height={14} />
                <Skeleton height={14} />
              </Stack>
            </Stack>
            <Stack gap={12} className={styles.listRow}>
              <Skeleton width={72} height={48} />
              <Stack gap={8} className={styles.growStack}>
                <Skeleton width="48%" height={14} />
                <Skeleton height={14} />
              </Stack>
            </Stack>
          </Stack>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .layoutExample {
        width: min(24rem, calc(100vw - var(--spacing-8)));
        padding-block: 4px;
      }

        align-items: center;
      }

      .growStack {
        flex: 1 1 0;
