[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Anatomy

`Skeleton` is a single visual part. Layout stays in the surrounding markup.

```text
Skeleton.Root
└─ children (optional, revealed when loading={false})
```

| Part                         | Role                                                                                        |
| ---------------------------- | ------------------------------------------------------------------------------------------- |
| `Skeleton` / `Skeleton.Root` | Placeholder surface. Renders an Ark factory element and accepts `asChild` for custom hosts. |

## Composition

### Recommended composition

Use `Skeleton` directly for text lines, cards, avatars, and media placeholders. It renders one
decorative block while `loading` is true and reveals children when `loading` is false.

`variant="pulse"` is the default. Use `variant="none"` when motion should stay quiet.
Reduced-motion users always get a static surface.

### When to go custom

Use `asChild` when another semantic element must own the rendered node. Keep the child single and
semantic, matching Ark's composition guidance.

Use `width`, `height`, `boxSize`, and `borderRadius` for common dimensions. For more specialized
surfaces, apply `className`, `style`, or the public CSS variables.

### Default props

| Prop           | Default                         | Notes                                                                                                       |
| -------------- | ------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `loading`      | `true`                          | Controls whether placeholder styles are active or children are shown normally.                              |
| `variant`      | `pulse`                         | Supports `pulse` and `none`.                                                                                |
| `height`       | `1rem` while loading            | Loading CSS sets the default placeholder height. Loaded content uses natural height unless a height is set. |
| `width`        | `100%`                          | CSS sets the default root width. Numeric prop values become `px`.                                           |
| `boxSize`      | -                               | Sets width and height together when that axis is not provided.                                              |
| `borderRadius` | `var(--skeleton-border-radius)` | Numeric prop values become `px`; style overrides still win.                                                 |

### Behavioral notes

- `Skeleton.Root` is the same component as `Skeleton`, provided for namespace consistency.
- `loading={true}` sets `aria-hidden`, `data-state="loading"`, and `data-loading`.
- `loading={false}` sets `data-state="loaded"` and lets children use their natural height unless `height`, `boxSize`, or `style.height` is set.
- `variant` is styling-only; no internal timers or state management are added.
- `asChild` comes from Ark factory composition and replaces legacy render-prop composition.

## Styling

`Skeleton` exposes Ark-style part and state hooks plus public `--skeleton-*` variables for color,
radius, and motion.

### Styling hooks

- `[data-scope="skeleton"][data-part="root"]` targets the exported root.
- `data-slot="skeleton-root"` is the stable moduix selector for the root.
- `data-state="loading"` and `data-state="loaded"` reflect the `loading` prop.
- `data-variant="pulse"` and `data-variant="none"` reflect the variant.
- `data-loading` is present only while placeholder styles are active.
