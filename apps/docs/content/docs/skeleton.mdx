`Skeleton` is a standalone moduix component and does not wrap a dedicated Ark UI primitive.

## Composition

### When to stay high-level

Stay with `Skeleton` for most loading placeholders. The component is just one decorative block with
size props and a small styling surface.

Use `width` and `height` for text lines and rectangular surfaces. Use `size` with `shape="circle"`
for avatar-style placeholders. Use `radius` when the placeholder should match a card, media frame,
or another rounded surface.

### When to go custom

There is no lower-level moduix composition tree for this component. Keep layout, responsive
behavior, and loading structure in the surrounding markup, and let each `Skeleton` instance stay a
single visual block.

### Default props

| Prop       | Default | Values                             |
| ---------- | ------- | ---------------------------------- |
| `height`   | `1rem`  | CSS length (`number` = `px`)       |
| `width`    | -       | CSS length (`number` = `px`)       |
| `radius`   | -       | CSS length (`number` = `px`)       |
| `size`     | -       | Sets `width` and `height` together |
| `shape`    | `rect`  | `rect`, `circle`                   |
| `animated` | `true`  | `true`, `false`                    |

### Behavioral notes

- Numeric `width`, `height`, `radius`, and `size` values are converted to `px`.
- `size` fills both dimensions, but explicit `width` or `height` still wins on its own axis.
- `shape="circle"` only sets `border-radius: 50%` when `radius` is not provided. It does not force
  equal width and height.
- `animated={false}` removes the `data-animated` hook and disables the pulse animation for that
  instance.
- Reduced-motion users still get a static skeleton even when `animated` stays `true`.

## Styling

`Skeleton` exposes one stable root selector, an optional animation hook, and three public
`--skeleton-*` variables for color, radius, and motion.

### CSS Properties

### Styling hooks

- `data-slot="skeleton-root"` is the stable selector for the exported root.
- `[data-animated]` is present only when animation is enabled for that instance.
- `className` and `style` are applied directly to the root skeleton element.
