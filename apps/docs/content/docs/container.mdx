## Overview

`Container` is the library's centered page-width primitive. It keeps a readable content column,
adds responsive inline gutters, and stays out of vertical layout decisions.

Use `size` to choose the content width and `gutter` to control the edge padding. Use `as` when the
wrapper should also carry page semantics such as `main` or `section`, and use `className` or
`--container-*` variables when a page needs local layout overrides.

## Anatomy

`Container` is a single root layout primitive. It owns width, max width, centering, and inline
gutters while leaving block spacing to parent sections or local CSS.

```text
Container
└─ children
```

```tsx
<Container size="lg" gutter="md">
  <main>Page content</main>
</Container>
```

| Part        | Role                                                     |
| ----------- | -------------------------------------------------------- |
| `Container` | Root layout wrapper. Controls content width and gutters. |

## Composition

Use `size` for content width and `gutter` for edge spacing.
Use `as` when the wrapper should render a semantic landmark or sectioning element.

### Defaults

| Prop     | Default | Values                               |
| -------- | ------- | ------------------------------------ |
| `size`   | `lg`    | `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `gutter` | `md`    | `none`, `sm`, `md`, `lg`             |
| `as`     | `div`   | Any React element type               |

`Container` does not manage vertical rhythm. Add section spacing outside it.
`size="full"` still keeps inline gutter padding unless `gutter="none"` is used.
Use `Bleed` inside `Container` when content must escape the constrained column.
Use `className` when you need custom alignment or local variable overrides.
The root exposes `data-slot="container-root"`, `data-size`, and `data-gutter` for styling and tests.

## Examples

### Sizes

Use `size` to choose the content column width. `full` fills the available inline space while preserving the selected gutter.

### With Bleed

Keep the text column constrained and let only the wider child opt out.

<Preview>
  <ContainerBleedExample />

            <Heading as="h3" size="lg">Article body</Heading>
            <Text tone="muted">
              Keep the reading width constrained, then use Bleed for content that should stretch wider.
            </Text>
            <Bleed inline="md">
              <div className={styles.bleedSurface}>Bleed content escapes the constrained column.</div>
            </Bleed>
          </Container>
        );
      }

        max-width: 42rem;
