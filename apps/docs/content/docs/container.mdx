## API Reference

`Container` is a moduix layout primitive and does not wrap a Base UI primitive.
Use it to constrain page content with consistent responsive gutters. The `size`
prop controls the useful content width; `gutter` adds inline padding outside that
content width so large-screen columns do not become narrower because of padding.

## Anatomy

`Container` is a single root layout primitive. It sets width, max content width,
inline gutters, and horizontal alignment while leaving block spacing to parent
sections or local CSS.

```text
Container
└─ children
```

```tsx
<Container size="lg" gutter="md">
  <main className={styles.content}>Page content</main>
</Container>
```

| Part        | Role                                                                 |
| ----------- | -------------------------------------------------------------------- |
| `Container` | Root layout wrapper. Controls content width, gutters, and alignment. |

## Composition

Use `size` for the target content width and `gutter` for viewport edge protection.
The default `size="lg"` and `gutter="md"` are intended for common page layouts.
Use `align` when a narrower container should be pinned to one side, and `as` for
semantic elements such as `main`, `section`, or `article`.

| Prop     | Default  | Values                               |
| -------- | -------- | ------------------------------------ |
| `size`   | `lg`     | `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `gutter` | `md`     | `none`, `sm`, `md`, `lg`             |
| `align`  | `center` | `start`, `center`, `end`             |
| `as`     | `div`    | Any valid React element or component |

`Container` does not own vertical rhythm. Put section spacing on the section
around it, and use `Bleed` inside a `Container` when media or panels need to
escape the constrained content column.

## Examples

### Sizes

Use `size` to choose the content column width.

  <ContainerSizesExample />
