## API Reference

`Container` is a simple layout wrapper for a centered content column with responsive inline gutters.  
Use `size` to set content width and `gutter` to keep responsive inline spacing around it.
`size="full"` expands to the available inline space while keeping the chosen gutter inside it.
Use `as` when the wrapper should render a different root element.

## Anatomy

`Container` is a single root layout primitive. It sets width, max content width,
inline gutters, and centering while leaving block spacing to parent
sections or local CSS.

```text
Container
└─ children
```

```tsx
<Container size="lg" gutter="md">
  <main>Page content</main>
</Container>
```

| Part        | Role                                                     |
| ----------- | -------------------------------------------------------- |
| `Container` | Root layout wrapper. Controls content width and gutters. |

## Composition

Use `size` for content width and `gutter` for edge spacing.
Use `as` when the wrapper should render a different element.

### Defaults

| Prop     | Default | Values                               |
| -------- | ------- | ------------------------------------ |
| `size`   | `lg`    | `xs`, `sm`, `md`, `lg`, `xl`, `full` |
| `gutter` | `md`    | `none`, `sm`, `md`, `lg`             |
| `as`     | `div`   | Any React element type               |

`Container` does not manage vertical rhythm. Add section spacing outside it.
`size="full"` still keeps inline gutter padding unless `gutter="none"` is used.
Use `Bleed` inside `Container` when content must escape the constrained column.
Use `className` when you need custom alignment or local variable overrides.

## Examples

### Sizes

Use `size` to choose the content column width. `full` fills the available inline space while preserving the selected gutter.

  <ContainerSizesExample />

  <Preview.Code>

            <Container size="md"><Text>md</Text></Container>
            <Container size="lg"><Text>lg</Text></Container>
