## Default Path

Use `Stepper` with `StepperList` and explicit `StepperItem step={...}` values when you need ordered
progress without a large wizard abstraction.

`StepperTrigger` defaults to a native button. Use `render={<div />}` for a display-only progress
stepper, or `render={<a />}` / router-link composition when each step should navigate.

| Part                 | Role                                                               |
| -------------------- | ------------------------------------------------------------------ |
| `Stepper`            | Navigation landmark. Owns `currentStep` and `orientation`.         |
| `StepperList`        | Ordered list of visible steps.                                     |
| `StepperItem`        | One numbered step. Resolves `inactive`, `current`, or `completed`. |
| `StepperTrigger`     | Optional interactive wrapper. Defaults to a native button.         |
| `StepperIndicator`   | Number or completed check icon.                                    |
| `StepperContent`     | Layout wrapper for step text.                                      |
| `StepperTitle`       | Main step label.                                                   |
| `StepperDescription` | Secondary helper text.                                             |

## Composition

`Stepper` stays small on purpose:

- `currentStep` derives status only
- panel rendering stays in consumer code
- validation, form state, and route state stay in consumer code
- step navigation stays explicit in JSX through `StepperTrigger`

Rules of thumb:

| Pattern              | Best fit                                                 |
| -------------------- | -------------------------------------------------------- |
| `render={<div />}`   | Read-only onboarding progress or checklist summaries     |
| default button path  | Client-side multi-step forms and local flow state        |
| `render={<a />}`     | Anchor or router-link navigation between steps           |
| `status="completed"` | Manual overrides when visual progress differs from order |

`StepperItem` requires `step`. The current implementation does not auto-number from child position.

## Examples

### Controlled State

Keep the current step in React state when each step should be clickable.

<Preview>
  <ControlledStepperExample />

  <Preview.Code>
    {`
      import {
        Stepper,
        StepperContent,
        StepperDescription,
        StepperIndicator,
        StepperItem,
        StepperList,
        StepperTitle,
        StepperTrigger,
      } from "moduix";
      import { useState } from "react";

      const steps = [
        { step: 1, title: "Account" },
        { step: 2, title: "Profile" },
        { step: 3, title: "Billing" },
      ];

      export function ControlledStepperDemo() {
        const [currentStep, setCurrentStep] = useState(2);

        return (
          <Stepper currentStep={currentStep}>
            <StepperList>
              {steps.map((item) => (
                <StepperItem key={item.step} step={item.step}>
                  <StepperTrigger onClick={() => setCurrentStep(item.step)}>
                    <StepperIndicator />
                    <StepperContent>
                      <StepperTitle>{item.title}</StepperTitle>
                    </StepperContent>
                  </StepperTrigger>
                </StepperItem>
              ))}
            </StepperList>
          </Stepper>
        );
      }
    `}

  </Preview.Code>
</Preview>
