## Choosing the right component

Use `Popover` for compact interactive content related to a trigger. Use `Tooltip` for
non-interactive hints and `Dialog` for larger or strongly interruptive workflows.

## Basic

Compose the Ark structure explicitly. Positioning options belong to `Popover` and custom button
hosts use `asChild`.

  <Preview.Code>

## Anatomy

```text
Popover.Root
├─ Popover.Anchor (optional)
├─ Popover.Trigger
│  └─ Popover.Indicator (optional)
└─ Portal
   └─ Popover.Positioner
      └─ Popover.Content
         ├─ Popover.Arrow
         │  └─ Popover.ArrowTip
         ├─ Popover.Header
         │  ├─ Popover.Title
         │  └─ Popover.Description
         ├─ Popover.Body
         ├─ Popover.Footer
         └─ Popover.CloseTrigger
```

`Popover.RootProvider`, `Popover.Context`, `usePopover`, and `usePopoverContext` expose Ark's
external-state and context surfaces. `Header`, `Body`, and `Footer` are layout-only Moduix additions.

## Composition

### Recommended composition

Keep `Portal`, `Popover.Positioner`, and `Popover.Content` explicit. `Popover` is equivalent to
`Popover.Root`. Configure placement, collision behavior, offsets, and same-width behavior through
`positioning`.

### When to go custom

Use `Anchor` when the positioning reference differs from the trigger. Use `asChild` with exactly one
semantic child for custom trigger, anchor, content, or close-trigger hosts. Omit `Portal` when a
popover inside another overlay should render inline; otherwise combine portal usage with
`lazyMount` and `unmountOnExit` to avoid stale overlay nodes.

### Default props

Moduix adds visual defaults only. Ark owns `defaultOpen`, `open`, `modal`, `autoFocus`,
`closeOnEscape`, `closeOnInteractOutside`, `initialFocusEl`, `ids`, `lazyMount`, `present`,
`unmountOnExit`, and `positioning`.

### Behavioral notes

- `onOpenChange` receives `{ open }`; callbacks are not converted to positional values.
- `modal` is boolean and enables focus trapping, scroll lock, outside interaction blocking, and
  screen-reader isolation.
- `RootProvider` receives the object returned by `usePopover`; do not wrap the same state in `Root`.
- `Popover.Arrow` renders the default `ArrowTip`; render `ArrowTip` explicitly only to customize it.
- Ark closes on Escape and outside interaction by default. Both behaviors can be disabled separately.
- Use `ids` when stable trigger, content, title, description, anchor, or close-trigger IDs are needed.
- CSS exit animations use Ark's closed presence state. Use `present` for JavaScript-driven lifecycles.

## Examples

### Controlled

Use Ark's details object when synchronizing open state with surrounding UI.

<Preview>
  <ControlledPopoverExample />
  <Preview.Code>
    {`
      import { Button, Popover, Portal } from "moduix";
      import { useState } from "react";

      export function ControlledPopoverDemo() {
        const [open, setOpen] = useState(false);

        return (
          <Popover open={open} onOpenChange={(details) => setOpen(details.open)}>
            <Popover.Trigger asChild><Button>Open</Button></Popover.Trigger>
            <Portal>
              <Popover.Positioner>
                <Popover.Content>
                  <Popover.Title>Controlled popover</Popover.Title>
                  <Popover.CloseTrigger>Close</Popover.CloseTrigger>
                </Popover.Content>
              </Popover.Positioner>
            </Portal>
          </Popover>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{`/* Uses the default moduix popover styles. */`}</Preview.CSS>
  <Preview.Data>{`const initialOpen = false;`}</Preview.Data>
</Preview>

### Root Provider

Use `usePopover` with `RootProvider` when controls outside the rendered popover tree need its state
or methods.

<Preview>
  <RootProviderPopoverExample />
  <Preview.Code>
    {`
      import { Button, Popover, usePopover } from "moduix";

      export function RootProviderDemo() {
        const popover = usePopover({
          positioning: { placement: "bottom-start", gutter: 8 },
        });

        return (
          <>
            <Button onClick={() => popover.setOpen(!popover.open)}>Toggle externally</Button>
            <Popover.RootProvider value={popover}>
              <Popover.Trigger asChild><Button>Open from trigger</Button></Popover.Trigger>
              <Portal>
                <Popover.Positioner>
                  <Popover.Content>
                    <Popover.Title>External state</Popover.Title>
                  </Popover.Content>
                </Popover.Positioner>
              </Portal>
            </Popover.RootProvider>
          </>
        );
      }
    `}

  </Preview.Code>
  <Preview.CSS>{`/* Uses the default moduix popover styles. */`}</Preview.CSS>
  <Preview.Data>{`const placement = "bottom-start";`}</Preview.Data>
</Preview>

### Modal

Modal popovers trap focus and isolate outside content. Use `initialFocusEl` for a deliberate initial
focus target.

<Preview>
  <ModalPopoverExample />
  <Preview.Code>
    {`
      import { useRef } from "react";

      export function ModalPopoverDemo() {
        const inputRef = useRef(null as HTMLInputElement | null);

        return (
          <Popover modal initialFocusEl={() => inputRef.current}>
            <Popover.Trigger asChild><Button>Invite teammates</Button></Popover.Trigger>
            <Portal>
              <Popover.Positioner>
                <Popover.Content>
                  <Popover.Title>Invite teammates</Popover.Title>
                  <input ref={inputRef} className="input" />
                  <Popover.CloseTrigger>Done</Popover.CloseTrigger>
                </Popover.Content>
              </Popover.Positioner>
            </Portal>
          </Popover>
        );
      }
    `}

  </Preview.Code>

### Styling hooks

Ark parts retain `data-scope="popover"` and their `data-part` values. Moduix adds
`data-slot="popover-*"` to rendered wrappers. Target `data-state="open|closed"`,
`data-placement`, `data-nested`, `data-has-nested`, and `data-expanded` for state-aware styling.

`Positioner` exposes `--reference-width`, `--reference-height`, `--available-width`,
`--available-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`. `Content` exposes
`--layer-index` and `--nested-layer-count`; `Arrow` exposes Ark's arrow measurement variables.
