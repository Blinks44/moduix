## Choosing the right component

Use `Popover` for compact interactive content related to a trigger. Use `Tooltip` for
non-interactive hints and `Dialog` for larger or strongly interruptive workflows.

## Basic

Compose the Ark structure explicitly. Positioning options belong to `Popover` and custom button
hosts use `asChild`.

## Anatomy

```text
Popover.Root
├─ Popover.Anchor (optional)
├─ Popover.Trigger
│  └─ Popover.Indicator (optional)
└─ Overlay subtree (automatically portalled)
   └─ Popover.Positioner
      └─ Popover.Content
         ├─ Popover.Arrow
         │  └─ Popover.ArrowTip
         ├─ Popover.Header
         │  ├─ Popover.Title
         │  └─ Popover.Description
         ├─ Popover.Body
         ├─ Popover.Footer
         └─ Popover.CloseTrigger
```

| Part                         | Role                                                             |
| ---------------------------- | ---------------------------------------------------------------- |
| `Popover` / `Popover.Root`   | Root state, popup lifecycle, dismissal, focus, and positioning.  |
| `Popover.RootProvider`       | Renders a popover tree from state returned by `usePopover`.      |
| `Popover.Anchor`             | Optional custom positioning reference.                           |
| `Popover.Trigger`            | Button that toggles the popover and may carry a trigger `value`. |
| `Popover.Indicator`          | Optional trigger state indicator.                                |
| `Popover.Positioner`         | Floating layer that exposes Ark positioning CSS variables.       |
| `Popover.Content`            | Interactive popup surface.                                       |
| `Popover.Arrow` / `ArrowTip` | Optional positioned arrow and its visible tip.                   |
| `Popover.Title`              | Accessible title wired to the content.                           |
| `Popover.Description`        | Accessible description wired to the content.                     |
| `Popover.CloseTrigger`       | Button that dismisses the popover.                               |
| `Popover.Header` / `Body`    | Moduix layout wrappers with stable slots.                        |
| `Popover.Footer`             | Moduix action row wrapper with stable slot.                      |
| `Popover.Context`            | Inline access to Ark popover state.                              |
| `usePopoverContext`          | Hook for reusable child parts inside a popover tree.             |

## Composition

The root portals `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; structural parts remain explicit and styleable.

### Recommended composition

Keep `Popover.Positioner` and `Popover.Content` explicit. `Popover` is equivalent to
`Popover.Root`. Configure placement, collision behavior, offsets, and same-width behavior through
`positioning`.

### When to go custom

Use `Anchor` when the positioning reference differs from the trigger. Use `asChild` with exactly one
semantic child for custom trigger, anchor, content, or close-trigger hosts. Set
`portalled={false}` when a popover inside another overlay should render inline; otherwise combine
the default portalled behavior with `lazyMount` and `unmountOnExit` to avoid stale overlay nodes.

### Default props

| Prop                     | Default | Notes                                                        |
| ------------------------ | ------- | ------------------------------------------------------------ |
| `autoFocus`              | `true`  | Focuses the first focusable element when the popover opens.  |
| `closeOnEscape`          | `true`  | Closes the popover on Escape.                                |
| `closeOnInteractOutside` | `true`  | Closes the popover on outside interaction.                   |
| `lazyMount`              | `false` | Defers popup content mounting until first open.              |
| `modal`                  | `false` | Enables focus trap, scroll lock, and outside isolation.      |
| `portalled`              | `true`  | Portals the positioner and preserves Ark tab behavior.       |
| `portalRef`              | -       | Targets a custom portal container.                           |
| `skipAnimationOnMount`   | `false` | Keeps the initial presence animation enabled.                |
| `unmountOnExit`          | `false` | Removes popup content after close when enabled.              |
| `defaultOpen`            | -       | Initial uncontrolled open state.                             |
| `open`                   | -       | Controlled open state.                                       |
| `ids`                    | -       | Stable IDs for trigger, content, title, description, anchor. |
| `initialFocusEl`         | -       | Function returning the element to focus on open.             |
| `positioning`            | -       | Ark floating positioning options.                            |
| `present`                | -       | Controlled presence for JavaScript animation lifecycles.     |

### Behavioral notes

- Refs forward to the underlying Ark DOM parts: `Trigger` and `CloseTrigger` target buttons, while
  `Anchor`, `Positioner`, `Content`, `Arrow`, `ArrowTip`, `Title`, `Description`, `Indicator`,
  `Header`, `Body`, and `Footer` target their rendered elements.
- `onOpenChange` receives `{ open }`; callbacks are not converted to positional values.
- `modal` is boolean and enables focus trapping, scroll lock, outside interaction blocking, and
  screen-reader isolation.
- `RootProvider` receives the object returned by `usePopover`; do not wrap the same state in `Root`.
- `Popover.Context`, `usePopover`, and `usePopoverContext` are exported for Ark state access.
- `Popover.Arrow` renders the default `ArrowTip`; render `ArrowTip` explicitly only to customize it.
- Ark closes on Escape and outside interaction by default. Both behaviors can be disabled separately.
- Use `ids` when stable trigger, content, title, description, anchor, or close-trigger IDs are needed.
- CSS exit animations use Ark's closed presence state. Use `present` for JavaScript-driven lifecycles.

## Examples

### Controlled

Use Ark's details object when synchronizing open state with surrounding UI.

<Preview>
  <ControlledPopoverExample />
  <Preview.Code>
    {`
      import { Button, Popover } from "@moduix/react";
      import { useState } from "react";

      export function ControlledPopoverDemo() {
        const [open, setOpen] = useState(false);

        return (
          <div className="stack">
            <span>Popover is {open ? "open" : "closed"}</span>
            <Popover open={open} onOpenChange={(details) => setOpen(details.open)}>
              <Popover.Trigger asChild><Button>Open controlled popover</Button></Popover.Trigger>
              <Popover.Positioner>
                <Popover.Content>
                  <Popover.Header>
                    <Popover.Title>Publish changes?</Popover.Title>
                    <Popover.Description>
                      This action will make your latest updates visible to all users.
                    </Popover.Description>
                  </Popover.Header>
                  <Popover.Footer>
                    <Popover.CloseTrigger>Close</Popover.CloseTrigger>
                  </Popover.Footer>
                </Popover.Content>
              </Popover.Positioner>
            </Popover>
          </div>
        );
      }
    `}

### Root Provider

Use `usePopover` with `RootProvider` when controls outside the rendered popover tree need its state
or methods.

<Preview>

        return (
          <div className="stack">
            <span>Popover is {popover.open ? "open" : "closed"}</span>

### Styling hooks

Ark parts retain `data-scope="popover"` and their `data-part` values. Every exported rendered part
accepts `className`; Moduix adds `data-slot="popover-*"` to rendered wrappers. Target
`data-state="open|closed"`,
`data-placement`, `data-nested`, `data-has-nested`, and `data-expanded` for state-aware styling.

`Positioner` exposes `--reference-width`, `--reference-height`, `--available-width`,
`--available-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`. `Content` exposes
`--layer-index` and `--nested-layer-count`; `Arrow` exposes Ark's arrow measurement variables.
