## Anatomy

`Popover` combines a trigger and a floating content layer. Keep `PopoverTrigger` and
`PopoverContent` under the same `Popover` root unless you intentionally connect them with
`createPopoverHandle`.

```text
Popover
├─ PopoverTrigger
│  └─ trigger content
└─ PopoverContent
   ├─ PopoverHeader (optional)
   │  ├─ PopoverTitle (optional)
   │  └─ PopoverDescription (optional)
   ├─ PopoverBody (optional)
   ├─ PopoverFooter (optional)
   │  └─ PopoverClose (optional)
   └─ arrow (optional)

PopoverContent service slots (optional)
├─ portal
├─ backdrop
├─ positioner
└─ viewport
```

```tsx
<Popover>
  <PopoverTrigger render={<Button />}>Open popover</PopoverTrigger>
  <PopoverContent withBackdrop withViewport>
    <PopoverHeader>
      <PopoverTitle>Popover title</PopoverTitle>
      <PopoverDescription>Short contextual details.</PopoverDescription>
    </PopoverHeader>
    <PopoverBody>Popup content</PopoverBody>
    <PopoverFooter>
      <PopoverClose>Close</PopoverClose>
    </PopoverFooter>
  </PopoverContent>
</Popover>
```

| Part                 | Role                                                                                                         |
| -------------------- | ------------------------------------------------------------------------------------------------------------ |
| `Popover`            | Root state machine. Controls `open`, `defaultOpen`, `onOpenChange`, `modal`, and shared behavior.            |
| `PopoverTrigger`     | Interactive anchor that opens and closes the popup. Supports click and optional hover opening.               |
| `PopoverContent`     | Floating container that renders popup structure, positioning, and optional arrow/backdrop/viewport plumbing. |
| `PopoverHeader`      | Semantic top area for title and description. Useful for consistent spacing in information popovers.          |
| `PopoverTitle`       | Main heading inside the popup. Helps users quickly identify popup purpose.                                   |
| `PopoverDescription` | Supporting text under the title. Use for concise context or status details.                                  |
| `PopoverBody`        | Optional free-form content region when header/footer wrappers are not enough.                                |
| `PopoverFooter`      | Optional action area, commonly for dismiss, confirm, or secondary controls.                                  |
| `PopoverClose`       | Built-in close action that dismisses the current popover without custom state wiring.                        |

Style the visible layers first: `PopoverTrigger`, `PopoverContent`, and header/body/footer
content. Reach for `classNames` only when you need to style internal service slots such as
`portal`, `backdrop`, `positioner`, `viewport`, or `arrow`.

## Composition

Use `Popover` for root state and behavior props such as `open`, `defaultOpen`, `onOpenChange`,
`modal`, and `handle`.

Pass placement props such as `side`, `align`, `sideOffset`, `alignOffset`, `collisionBoundary`,
and `collisionPadding` directly to `PopoverContent`. `className` styles the visible popup;
`classNames` styles internal service slots. Use `container` to choose the portal container,
`withBackdrop` to enable the optional backdrop, `withViewport` to wrap content in Base UI's
viewport, `withArrow={false}` to remove the default arrow, and `arrowContent` to replace the
default arrow graphic. Reach for `slotProps` only when you need Base UI escape-hatch props such as
`keepMounted` on the portal or custom viewport rendering.

When replacing the trigger element with `render`, keep `nativeButton={true}` for real button-like
elements and set `nativeButton={false}` for non-button elements such as `div` or `span`.

For focus-sensitive popovers, pass `initialFocus` and `finalFocus` to `PopoverContent`. If you use
`modal` or `modal="trap-focus"` on `Popover`, keep a `PopoverClose` action inside the popup so
keyboard users and touch screen readers always have a way to leave it.

### Defaults

| Prop           | Default | Values                                                                      |
| -------------- | ------- | --------------------------------------------------------------------------- |
| `withArrow`    | `true`  | `boolean`                                                                   |
| `withBackdrop` | `false` | `boolean`                                                                   |
| `withViewport` | `false` | `boolean`                                                                   |
| `sideOffset`   | `8`     | `number`                                                                    |
| `container`    | `null`  | `HTMLElement \| ShadowRoot \| React.RefObject<HTMLElement \| null> \| null` |

```tsx
<PopoverContent
  withBackdrop
  withViewport
  className={styles.popup}
  classNames={{
    portal: styles.portal,
    backdrop: styles.backdrop,
    positioner: styles.positioner,
    viewport: styles.viewport,
    arrow: styles.arrow,
  }}
  slotProps={{
    portal: { keepMounted: true },
    positioner: { collisionPadding: 12 },
  }}
/>
```

            </Popover>
          </div>
