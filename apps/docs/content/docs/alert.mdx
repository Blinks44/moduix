## API Reference

`Alert` is a standalone moduix component and does not wrap a Base UI primitive.
Use it for inline messages with a small composition-first API: one `variant` prop on the root,
semantic content parts, and regular children for everything else.

## Anatomy

`AlertIcon` is optional. `AlertContent` groups the text slots and keeps the layout predictable.

```text

   â”śâ”€ AlertTitle

```

```tsx
<Alert variant="info">
  <AlertIcon>
    <InfoIcon />
  </AlertIcon>
  <AlertContent>
    <AlertTitle>Workspace sync is active</AlertTitle>
    <AlertDescription>Changes are being synced across all connected devices.</AlertDescription>
  </AlertContent>
</Alert>
```

| Part               | Role                                                    |
| ------------------ | ------------------------------------------------------- |
| `Alert`            | Root slot. Applies variant styles and semantics.        |
| `AlertIcon`        | Optional icon slot. Sizes child SVG icons consistently. |
| `AlertContent`     | Text content wrapper for title and description.         |
| `AlertTitle`       | Alert heading slot. Renders `h3` by default.            |
| `AlertDescription` | Supporting message slot.                                |

## Composition

Use `variant` for intent: `default`, `info`, `success`, `warning`, or `destructive`.

Keep text, links, and extra actions inside `AlertContent`. The component intentionally avoids
special-purpose action props or slot bags; normal composition already covers those cases cleanly.

Use `className` on the root or parts for local styles. Override `--alert-*` CSS variables on the
root slot when the alert needs local theming.

`role` defaults to `status`, and switches to `alert` for `variant="destructive"`.
`AlertTitle` renders `h3` by default and accepts `as="h1"` through `as="h6"` when the surrounding
document outline needs a different heading level.

### Defaults

| Prop      | Default   | Values                                                    |
| --------- | --------- | --------------------------------------------------------- |
| `variant` | `default` | `default`, `info`, `success`, `warning`, `destructive`    |
| `role`    | Auto      | `status` by default, `alert` when `variant="destructive"` |

### Props

| Component          | Props                                                 |
| ------------------ | ----------------------------------------------------- |
| `Alert`            | native `div` props, `variant`, `className`, `role`    |
| `AlertIcon`        | native `span` props, defaults to `aria-hidden="true"` |
| `AlertContent`     | native `div` props                                    |
| `AlertTitle`       | heading props, `as="h1"` through `as="h6"`            |
| `AlertDescription` | native `div` props                                    |

## Custom Composition

Compose extra controls inside `AlertContent` when the message needs a recovery path or follow-up
action. Keep dismiss state in the consuming component when a local alert should be hidden.

<Preview>
  <CustomCompositionAlertExample />

  <Preview.Code>
    {`
      import {
        Alert,
        AlertContent,
        AlertDescription,
        AlertIcon,
        AlertTitle,
        Button,
        InfoIcon,
      } from "moduix";
      import { useState } from "react";
      import styles from "./alert.module.css";

      export function AlertCustomCompositionDemo() {
        const [visible, setVisible] = useState(true);

        if (!visible) return null;

        return (
          <Alert variant="warning" className={styles.customAlert}>
            <AlertIcon>
              <InfoIcon />
            </AlertIcon>
            <AlertContent>
              <AlertTitle>Storage is almost full</AlertTitle>
              <AlertDescription>
                You are using 92% of the available storage. Archive old uploads or upgrade the plan.
              </AlertDescription>
              <div className={styles.actions}>
                <Button size="sm">Review uploads</Button>
                <Button size="sm" variant="outline" onClick={() => setVisible(false)}>
                  Dismiss
                </Button>
              </div>
            </AlertContent>
          </Alert>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
