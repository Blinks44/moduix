## API Reference

Ark UI does not ship a dedicated `alert` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

Public parts are `Alert.Root`, `Alert.Indicator`, `Alert.Content`, `Alert.Title`, and
`Alert.Description`.

## Choosing the right component

Use `Alert` when the message should stay inline in the page layout and participate in the normal
document flow.

- Choose **`Alert`** for inline status, validation, and system messages inside forms, panels, or page content.
- Choose **`Toast`** for transient notifications that appear outside the document flow and dismiss on their own.

## Anatomy

```text
Alert.Root
├─ Alert.Indicator (optional)
└─ Alert.Content
   ├─ Alert.Title
   └─ Alert.Description
```

| Part                | `data-part`   | Role                                                         |
| ------------------- | ------------- | ------------------------------------------------------------ |
| `Alert.Root`        | `root`        | Root slot. Applies status styles and semantics.              |
| `Alert.Indicator`   | `indicator`   | Optional indicator slot. Sizes child SVG icons consistently. |
| `Alert.Content`     | `content`     | Text content wrapper for title and description.              |
| `Alert.Title`       | `title`       | Alert heading slot. Renders `h3` by default.                 |
| `Alert.Description` | `description` | Supporting message slot.                                     |

## Composition

### Recommended composition

Stay with `Alert.Root`, `Alert.Content`, `Alert.Title`, and `Alert.Description` for most inline
feedback messages.

Use `status` for intent: `neutral`, `info`, `success`, `warning`, or `error`.

Keep text, links, and extra actions inside `Alert.Content`. The component intentionally avoids
special-purpose action props or slot bags.

### When to go custom

Switch to custom composition only when the alert needs extra inline controls, local dismiss state,
or a different rendered element through Ark `asChild`.

Use `Alert.Title asChild` when the document outline needs a different heading element:

```tsx
<Alert.Title asChild>
  <h2>Billing issue</h2>
</Alert.Title>
```

### Default props

| Prop     | Default   | Values / Notes                                    |
| -------- | --------- | ------------------------------------------------- |
| `status` | `neutral` | `neutral`, `info`, `success`, `warning`, `error`  |
| `role`   | Auto      | `status` by default, `alert` for `status="error"` |

### Behavioral notes

- `Alert.Root` defaults to `role="status"` and switches to `role="alert"` for `status="error"`.
- `Alert.Title` renders `h3` by default and uses Ark `asChild` for heading polymorphism.
- `Alert.Indicator` defaults to `aria-hidden="true"`, so essential meaning should stay in the
  title or description.
- The component is intentionally presentational. It does not manage dismiss state, focus, or
  keyboard behavior for embedded actions.

## Styling

Use `className` on the root or parts for local styles. Override `--alert-*` CSS variables on the
root when the alert needs local theming.

Built-in statuses also read shared palette tokens:

| Status    | Shared token          | Default                      |
| --------- | --------------------- | ---------------------------- |
| `info`    | `--color-primary`     | theme primary                |
| `success` | `--color-success`     | `oklch(0.627 0.194 149.214)` |
| `warning` | `--color-warning`     | `oklch(0.795 0.184 86.047)`  |
| `error`   | `--color-destructive` | theme destructive            |

### Styling hooks

- Stable `data-slot` selectors: `alert-root`, `alert-indicator`, `alert-content`, `alert-title`,
  and `alert-description`
- Ark-style selectors: `data-scope="alert"` with `data-part="root"`, `indicator`, `content`,
  `title`, or `description`
- `Alert.Root` writes the selected status to `data-status`
