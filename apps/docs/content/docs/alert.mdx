## API Reference

`Alert` is a standalone moduix component and does not wrap a Base UI primitive.
Use it for inline messages with a small composition-first API: `variant` on the root, semantic content parts, and regular children for everything else.

## Anatomy

`AlertIcon` is optional. `AlertContent` groups the text slots and keeps the layout predictable.

```text
Alert
├─ AlertIcon (optional)
└─ AlertContent
   ├─ AlertTitle
   └─ AlertDescription
```

```tsx
<Alert variant="info">
  <AlertIcon>
    <InfoIcon />
  </AlertIcon>
  <AlertContent>
    <AlertTitle>Workspace sync is active</AlertTitle>
    <AlertDescription>Changes are being synced across all connected devices.</AlertDescription>
  </AlertContent>
</Alert>
```

| Part               | Role                                                    |
| ------------------ | ------------------------------------------------------- |
| `Alert`            | Root slot. Applies variant styles and semantics.        |
| `AlertIcon`        | Optional icon slot. Sizes child SVG icons consistently. |
| `AlertContent`     | Text content wrapper for title and description.         |
| `AlertTitle`       | Alert heading slot. Renders an `h3`.                    |
| `AlertDescription` | Supporting message slot.                                |

## Composition

Use `variant` for intent (`default`, `info`, `success`, `warning`, `destructive`).
Use `className` on the root or parts for local styles, and override CSS variables on the root slot when you need theme-level customization.

`role` defaults to `status`, and switches to `alert` for `variant="destructive"`.

### Defaults

| Prop      | Default   | Values                                                    |
| --------- | --------- | --------------------------------------------------------- |
| `variant` | `default` | `default`, `info`, `success`, `warning`, `destructive`    |
| `role`    | Auto      | `status` by default, `alert` when `variant="destructive"` |

### Variants

Use variants for neutral, informational, success, warning, and destructive states.

<Preview>
  <AlertVariantsExample />

  <Preview.Code>
    {`
      import { Alert, AlertContent, AlertDescription, AlertIcon, AlertTitle, CheckFilledIcon, InfoIcon } from "moduix";

      const variants = ["default", "info", "success", "warning", "destructive"] as const;

      export function AlertVariantsDemo() {
        return (
          <div>
            {variants.map((variant) => (
              <Alert key={variant} variant={variant}>
                <AlertIcon>
                  {variant === "success" ? <CheckFilledIcon /> : <InfoIcon />}
                </AlertIcon>
                <AlertContent>
                  <AlertTitle>{variant}</AlertTitle>
                  <AlertDescription>Use this alert for {variant} feedback.</AlertDescription>
                </AlertContent>
              </Alert>
            ))}
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>
