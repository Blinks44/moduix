## API Reference

`Alert` is a standalone moduix component and does not wrap a Base UI primitive.
Use it for inline messages with optional icon, dismiss control, and close animation.

## Anatomy

`Alert` is composed from semantic content slots. `AlertIcon` is optional. `AlertClose` can be
rendered manually, or automatically by setting `withCloseButton` on the root.

```text
Alert
├─ AlertIcon (optional)
├─ AlertContent
│  ├─ AlertTitle
│  └─ AlertDescription
└─ AlertClose (optional)
```

```tsx
<Alert variant="info" withCloseButton>
  <AlertIcon>
    <InfoIcon />
  </AlertIcon>
  <AlertContent>
    <AlertTitle>Workspace sync is active</AlertTitle>
    <AlertDescription>Changes are being synced across all connected devices.</AlertDescription>
  </AlertContent>
</Alert>
```

| Part               | Role                                                                 |
| ------------------ | -------------------------------------------------------------------- |
| `Alert`            | Root slot. Controls variant, size, visibility, and dismiss behavior. |
| `AlertIcon`        | Optional icon slot. Sizes child SVG icons consistently.              |
| `AlertContent`     | Text content wrapper for title and description.                      |
| `AlertTitle`       | Alert heading slot.                                                  |
| `AlertDescription` | Supporting message slot.                                             |
| `AlertClose`       | Close control wired to the nearest `Alert`.                          |

## Composition

Use `variant` for intent (`default`, `info`, `success`, `warning`, `destructive`).
Use `open` / `defaultOpen` / `onOpenChange` to control visibility.
Set `withCloseButton` to render the built-in top-right `CloseButton`, pass `closeButton` to override its content, and use `withDismissAnimation={false}` for instant close.
Use `className` on root or parts for local styles, override CSS variables on the root slot, and use `AlertTitle as` when the default `h3` does not fit the surrounding heading structure.
All exported parts forward refs to their underlying DOM elements.

### Defaults

| Prop                   | Default       | Values                                                               |
| ---------------------- | ------------- | -------------------------------------------------------------------- |
| `variant`              | `default`     | `default`, `info`, `success`, `warning`, `destructive`               |
| `size`                 | `md`          | `sm`, `md`, `lg`                                                     |
| `defaultOpen`          | `true`        | `true`, `false`                                                      |
| `withCloseButton`      | `false`       | `true`, `false`                                                      |
| `closeButton`          | `undefined`   | React node rendered inside the built-in top-right `AlertClose`       |
| `withDismissAnimation` | `true`        | `true`, `false`                                                      |
| `closeButtonLabel`     | `Close alert` | Any accessible text label for the built-in close button              |
| `role`                 | Auto          | `status` by default, `alert` when `variant="destructive"`, or custom |

### Variants

Use variants for neutral, informational, success, warning, and destructive states.

<Preview>
  <AlertVariantsExample />

  <Preview.Code>
    {`
      import { Alert, AlertContent, AlertDescription, AlertIcon, AlertTitle, InfoIcon } from "moduix";

      const variants = ["default", "info", "success", "warning", "destructive"] as const;

      export function AlertVariantsDemo() {
        return (
          <div>
            {variants.map((variant) => (
              <Alert key={variant} variant={variant}>
                <AlertIcon>
                  <InfoIcon />
                </AlertIcon>
                <AlertContent>
                  <AlertTitle>{variant}</AlertTitle>
                  <AlertDescription>Use this alert for {variant} feedback.</AlertDescription>
                </AlertContent>
              </Alert>
            ))}
          </div>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Dismissible

Set `withCloseButton` for the standard close control. Use `open` and `onOpenChange` for controlled mode.

<Preview>
  <AlertDismissibleExample />

  <Preview.Code>
    {`
      import { useState } from "react";
      import { Alert, AlertContent, AlertDescription, AlertIcon, AlertTitle, CheckFilledIcon } from "moduix";

      export function AlertDismissibleDemo() {
        const [open, setOpen] = useState(true);

        return (
          <Alert variant="success" open={open} onOpenChange={setOpen} withCloseButton>
            <AlertIcon>
              <CheckFilledIcon />
            </AlertIcon>
            <AlertContent>
              <AlertTitle>Saved</AlertTitle>
              <AlertDescription>The alert closes with the default exit animation.</AlertDescription>
            </AlertContent>
          </Alert>
        );
      }
    `}

  </Preview.Code>
</Preview>

### Custom Dismiss Layout

Render `AlertClose` manually when you need custom placement, button content, or direct access to the close button ref. Use `AlertTitle as` when the default heading level is not appropriate for the page structure.

<Preview>
  <AlertCustomDismissExample />

  <Preview.Code>
    {`
      import { Alert, AlertClose, AlertContent, AlertDescription, AlertIcon, AlertTitle, InfoIcon } from "moduix";

      export function AlertCustomDismissDemo() {
        return (
          <Alert variant="info">
            <AlertIcon>
              <InfoIcon />
            </AlertIcon>
            <AlertContent>
              <AlertTitle as="h2">Workspace sync is active</AlertTitle>
              <AlertDescription>
                Render AlertClose manually when you need custom placement or button content.
              </AlertDescription>
            </AlertContent>
            <AlertClose>Dismiss</AlertClose>
          </Alert>
        );
      }
    `}

  </Preview.Code>
</Preview>
