## API Reference

Ark UI does not ship a dedicated `alert` primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

Public parts are `Alert.Root`, `Alert.Indicator`, `Alert.Content`, `Alert.Title`,
`Alert.Description`, and `Alert.Actions`.

## Choosing the right component

Use `Alert` when the message should stay inline in the page layout and participate in the normal
document flow.

- Choose **`Alert`** for inline status, validation, and system messages inside forms, panels, or page content.
- Choose **`Toast`** for transient notifications that appear outside the document flow and dismiss on their own.

## Anatomy

```text
Alert / Alert.Root
├─ Alert.Indicator (optional)
├─ Alert.Title
├─ Alert.Description
└─ Alert.Content (optional)
   ├─ Alert.Title
   ├─ Alert.Description
   ├─ Alert.Actions (optional)
   └─ Extra custom blocks
```

| Part                   | Role                                                          | `data-part`   |
| ---------------------- | ------------------------------------------------------------- | ------------- |
| `Alert` / `Alert.Root` | Root slot. Applies status styles and semantics.               | `root`        |
| `Alert.Indicator`      | Optional indicator slot. Sizes child SVG icons consistently.  | `indicator`   |
| `Alert.Content`        | Optional content-column wrapper for grouped text and actions. | `content`     |
| `Alert.Title`          | Alert title slot. Renders `p` by default.                     | `title`       |
| `Alert.Description`    | Supporting message slot.                                      | `description` |
| `Alert.Actions`        | Optional wrapping row for action elements inside content.     | `actions`     |

## Composition

### Recommended composition

Use `Alert`, `Alert.Title`, and `Alert.Description` directly for the common inline message path.

Use `status` for intent: `neutral`, `info`, `success`, `warning`, or `error`.

Add `Alert.Indicator` when the alert needs an explicit visual cue. The component intentionally
avoids special-purpose action props or slot bags.

### When to go custom

Add `Alert.Content` when the alert text needs to stay in one column with extra inline controls,
local dismiss state, or additional custom blocks. Use `Alert.Actions` for a wrapped action row
inside that content column, and `Alert.Title asChild` when the document outline needs a different
heading element.

Use `Alert.Title asChild` when the document outline needs a different heading element:

```tsx
<Alert.Title asChild>
  <h2>Billing issue</h2>
</Alert.Title>
```

### Default props

| Prop     | Default   | Values / Notes                                    |
| -------- | --------- | ------------------------------------------------- |
| `status` | `neutral` | `neutral`, `info`, `success`, `warning`, `error`  |
| `role`   | Auto      | `status` by default, `alert` for `status="error"` |

### Behavioral notes

- `Alert.Root` defaults to `role="status"` and switches to `role="alert"` for `status="error"`.
- `Alert.Indicator` defaults to `aria-hidden="true"`.
- `Alert.Title` renders `p` by default and uses Ark `asChild` for heading polymorphism.
- Essential meaning should stay in the title or description, because the indicator stays
  `aria-hidden`.
- `Alert.Content` is optional. Use it when title, description, and extra controls should share one
  content column beside the indicator.
- `Alert.Actions` is optional. Use it when buttons or links should share the default wrapped action
  row styling.
- The component is intentionally presentational. It does not manage dismiss state, focus, or
  keyboard behavior for embedded actions.

## Styling

Use `className` on the root or parts for local styles. Override `--alert-*` CSS variables on the
root when the alert needs local theming.

Built-in statuses also read shared palette tokens:

| Status    | Shared token          | Default                      |
| --------- | --------------------- | ---------------------------- |
| `info`    | `--color-primary`     | theme primary                |
| `success` | `--color-success`     | `oklch(0.627 0.194 149.214)` |
| `warning` | `--color-warning`     | `oklch(0.795 0.184 86.047)`  |
| `error`   | `--color-destructive` | theme destructive            |

### Styling hooks

- Stable `data-slot` selectors: `alert-root`, `alert-indicator`, `alert-content`, `alert-title`,
  `alert-description`, and `alert-actions`
- Ark-style selectors: `data-scope="alert"` with `data-part="root"`, `indicator`, `content`,
  `title`, `description`, or `actions`
- `Alert.Root` writes the selected status to `data-status`
