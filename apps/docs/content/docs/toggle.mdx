## Choosing the right component

Use `Toggle` when the control is still a button that exposes a pressed state.

- Choose **`Toggle`** for favorites, saved states, formatting actions, and compact toolbar actions.
- Choose **`Switch`** for settings-like on/off controls that read as a preference row.
- Choose **`Checkbox`** for submitted list or form selections.
- Choose **`ToggleGroup`** when several toggles should share one selection model.

## Anatomy

```text
Toggle / Toggle.Root
└─ Toggle.Indicator (optional)
```

| Part                     | Role                                                                              |
| ------------------------ | --------------------------------------------------------------------------------- |
| `Toggle` / `Toggle.Root` | Two-state button root that handles pressed, focus, disabled, and `asChild` state. |
| `Toggle.Indicator`       | Renders pressed content and optional `fallback` content when the toggle is off.   |
| `Toggle.Context`         | Reads the current Ark toggle state through a render prop.                         |

## Composition

### Recommended composition

Use `Toggle` for the standard root. It forwards refs to the Ark button part and supports
`pressed`, `defaultPressed`, `onPressedChange(pressed)`, `disabled`, `asChild`, and regular button
attributes.

`Toggle.Indicator` is optional. Use it when the visible icon or label should change based on
whether the toggle is pressed.

### When to go custom

Use `Toggle.Context` or `useToggleContext()` when child content needs to read `pressed` without
creating a parallel React state layer.

Use `asChild` when another semantic button must own the rendered element. Keep the child a single
button-like element so Ark can merge the required props and behavior onto it.

Use `ToggleGroup` instead of manually coordinating several standalone toggles when multiple pressed
buttons belong to one shared choice set.

### Default props

| Prop      | Default   | Notes                                                   |
| --------- | --------- | ------------------------------------------------------- |
| `variant` | `default` | moduix visual treatment: `default`, `outline`, `ghost`. |
| `size`    | `md`      | moduix spacing or icon-only geometry.                   |

### Behavioral notes

- Icon-only toggles need an accessible name such as `aria-label="Favorite"`.
- `disabled` keeps the current visual state but prevents interaction.
- Ark writes `data-state="on" | "off"`, `data-pressed`, and `data-disabled` for styling.
- `useToggleContext()` must run below `Toggle`; `useToggle()` is the lower-level Ark state hook.
- Ark Toggle does not expose `RootProvider` or a hidden form input.
- legacy `render` and `nativeButton` are not part of the Ark-backed API; use `asChild`.

## Examples

### Controlled

Control `pressed` from React state when the toggle participates in app state or analytics.

<Preview>

## Styling

`Toggle` exposes a styled root, an optional indicator part, and the full `--toggle-*` token surface
for variants, sizes, focus ring, spacing, and icon sizing.

### Styling hooks

- `data-slot="toggle-root"` is the stable selector on the root.
- `data-slot="toggle-indicator"` is the stable selector on `Toggle.Indicator`.
- `data-scope="toggle"` and `data-part="root" | "indicator"` come from Ark UI.
- `data-state="on" | "off"` reflects pressed state on root and indicator.
- `data-pressed` is present when the toggle is pressed.
- `data-variant` reflects the moduix visual variant.
- `data-size` reflects the moduix size, including `icon-sm`, `icon-md`, and `icon-lg`.
- `data-disabled` drives disabled styles; Ark also preserves the native `disabled` attribute.
- Descendant `svg` elements are sized through `--toggle-icon-size`.
