## Choosing the right component

Use `Toggle` when the control is still a button that exposes a pressed state.

- Choose **`Toggle`** for favorites, saved states, formatting actions, and compact toolbar actions.
- Choose **`Switch`** for settings-like on/off controls that read as a preference row.
- Choose **`Checkbox`** for submitted list or form selections.
- Choose **`ToggleGroup`** when several toggles should share one selection model.

## Basic

Start with `Toggle` alone. This is the recommended path for standalone two-state actions that only
need child content plus pressed-state props from the primitive.

## Anatomy

`Toggle` is a single styled root.

```text
Toggle
└─ label and/or icon content
```

| Part     | Role                                                                    |
| -------- | ----------------------------------------------------------------------- |
| `Toggle` | Two-state button root that handles pressed, focus, and disabled states. |

The root exposes `data-slot="toggle-root"`, `data-variant`, `data-size`, and primitive state
attributes such as `data-pressed` and `data-disabled`.

## Composition

### When to stay high-level

Stay with `Toggle` for most saved states, toolbar actions, and formatting buttons.

`Toggle` keeps primitive pressed-state behavior intact. Use it for `pressed`, `defaultPressed`,
`onPressedChange`, `disabled`, `render`, `nativeButton`, and regular button attributes.

moduix adds two visual props on top of the primitive:

| Prop      | Default   | Values                                                  |
| --------- | --------- | ------------------------------------------------------- |
| `variant` | `default` | `default`, `outline`, `ghost`                           |
| `size`    | `md`      | `xs`, `sm`, `md`, `lg`, `icon-sm`, `icon-md`, `icon-lg` |

### When to go custom

Use `render` when the visible element or icon needs to depend on primitive state without changing
the interaction model.

Use `ToggleGroup` instead of manually coordinating several standalone toggles when multiple pressed
buttons belong to one shared choice set.

### Behavioral notes

- Use `defaultPressed` for uncontrolled mode and `pressed` with `onPressedChange` for controlled
  mode.
- Icon-only toggles need an accessible name such as `aria-label="Favorite"`.
- `disabled` keeps the current visual state but prevents interaction.
- `value` is only meaningful when the toggle participates in `ToggleGroup` selection.

### Custom Composition

Use `render` when the visible element or icon should depend on primitive state. Keep this as the
advanced path; most toggles should stay on normal children.

<Preview>
  <ToggleRenderCallbackExample />

  <Preview.Code>
    {`
      import { Toggle } from "moduix";
      import styles from "./toggle.module.css";

      export function ToggleRenderCallbackDemo() {
        return (

            size="icon-md"
            variant="outline"

                  <BookmarkIcon className={styles.customIcon} />

            )}
          />

      }

        return (
          <svg viewBox="0 0 16 16" fill="none" aria-hidden="true" focusable="false" {...props}>
            <path
              d="M4.5 2.75h7v10.5L8 11l-3.5 2.25V2.75Z"
              stroke="currentColor"
              strokeWidth="1.5"
              strokeLinejoin="round"
            />
          </svg>
        );
      }

      function BookmarkFilledIcon(props) {
        return (
          <svg viewBox="0 0 16 16" fill="currentColor" aria-hidden="true" focusable="false" {...props}>
            <path d="M4 2.75A.75.75 0 0 1 4.75 2h6.5a.75.75 0 0 1 .75.75v10.5a.75.75 0 0 1-1.16.63L8 12.03l-2.84 1.85A.75.75 0 0 1 4 13.25V2.75Z" />

## Styling

`Toggle` exposes one stable root plus the full `--toggle-*` token surface for variants, sizes,
focus ring, spacing, and icon sizing.

### Styling hooks

- `data-slot="toggle-root"` is the stable selector on a standalone toggle root.
- `data-variant` reflects the current visual variant.
- `data-size` reflects the current size, including `icon-sm`, `icon-md`, and `icon-lg`.
- `data-pressed` is present when the toggle is pressed.
- Primitive `data-disabled` and the native `[disabled]` attribute both drive the disabled styles.
- Direct child `svg` elements are sized through `--toggle-icon-size`.
