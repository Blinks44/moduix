## Choosing the right component

Use `Autocomplete` when the field accepts free-form text and suggestions help users complete
the value without restricting what they can type.

- Choose **`Autocomplete`** when users can enter free-form text and suggestions only help complete the value.
- Choose **`Combobox`** when users must choose predefined options, but typing helps search or filter the list.
- Choose **`Select`** when users must choose predefined options and typing in the trigger is not needed.

## Anatomy

`Autocomplete` keeps the popup infrastructure behind `AutocompleteContent`, so the default
composition stays short and focused on the field and list parts.

```text
Autocomplete
├─ AutocompleteField
│  ├─ AutocompleteFieldLabel
│  ├─ AutocompleteInputGroup
│  │  ├─ AutocompleteInput
│  │  └─ AutocompleteControlActions (optional)
│  │     ├─ AutocompleteClear (optional)
│  │     └─ AutocompleteTrigger (optional)
│  └─ AutocompleteFieldTrigger (alternate)
│     ├─ AutocompleteValue
│     └─ AutocompleteIcon (optional)
└─ AutocompleteContent
   ├─ AutocompleteStatus / AutocompleteEmpty
   └─ AutocompleteList
      ├─ AutocompleteItem
      ├─ AutocompleteGroup / AutocompleteCollection
      ├─ AutocompleteRow (for grid lists)
      └─ AutocompleteSeparator
```

| Part                         | Role                                                                                                                        |
| ---------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `Autocomplete`               | Root state machine. Controls the input value, filtering, highlighting, popup open state, and free-form typing.              |
| `AutocompleteField`          | Field wrapper for the visible control and label.                                                                            |
| `AutocompleteInput`          | Free-form text input for the common inline-field path.                                                                      |
| `AutocompleteFieldTrigger`   | Trigger-style field for the "input inside popup" composition.                                                               |
| `AutocompleteControlActions` | Optional slot for clear and open controls.                                                                                  |
| `AutocompleteContent`        | Default popup wrapper. Renders `AutocompletePortal`, `AutocompletePositioner`, and `AutocompletePopup` for the common case. |
| `AutocompleteList`           | Collection renderer for flat items, groups, or grid rows.                                                                   |
| `AutocompleteItem`           | One suggestion option.                                                                                                      |
| `AutocompleteStatus`         | Optional status region for loading, error, or result-count feedback.                                                        |
| `AutocompleteEmpty`          | Empty-state content when nothing matches.                                                                                   |

Most customizations should stay on visible parts such as `AutocompleteInput`, `AutocompleteItem`,
`AutocompleteItemText`, and `AutocompleteContent`.

## Composition

Use `Autocomplete` for root state and Base UI behavior props such as `items`, `value`,
`onValueChange`, `filter`, `filteredItems`, `limit`, `mode`, `autoHighlight`, `grid`,
`virtualized`, and `openOnInputClick`.

Use `itemToStringValue` for object items. Use `filter={null}` or `filteredItems` for external
search. Use `AutocompleteGroup`, `AutocompleteGroupLabel`, and `AutocompleteCollection` for grouped
data. Use `grid` with `AutocompleteRow` for grid navigation. `useAutocompleteFilter` provides Base
UI string-matching helpers, and `useAutocompleteFilteredItems` exposes the filtered result set for
custom renderers.

`AutocompleteContent` is the short path for the common case. It styles the popup surface and keeps
the default `Portal -> Positioner -> Popup` structure. It forwards popup props to the visible
surface and accepts the shared positioning props directly.

| `AutocompleteContent` prop                                                                   | Default         | Purpose                                               |
| -------------------------------------------------------------------------------------------- | --------------- | ----------------------------------------------------- |
| `sideOffset`                                                                                 | `5`             | Gap between the trigger and popup.                    |
| `showArrow`                                                                                  | `false`         | Enables the built-in arrow.                           |
| `side`, `align`                                                                              | Base UI default | Popup placement relative to the anchor.               |
| `alignOffset`, `arrowPadding`, `collisionAvoidance`, `collisionBoundary`, `collisionPadding` | Base UI default | Collision and arrow tuning for the common popup path. |

When you need a custom portal container, backdrop, custom arrow content, or a different popup
stack, switch to the lower-level parts instead of extending `AutocompleteContent`.

```tsx
<AutocompletePortal className={styles.portal}>
  <AutocompleteBackdrop className={styles.backdrop} />
  <AutocompletePositioner className={styles.positioner} sideOffset={8}>
    <AutocompletePopup className={styles.popup}>
      <AutocompleteArrow className={styles.arrow} />
      <AutocompleteEmpty>No tags found.</AutocompleteEmpty>
      <AutocompleteList>{/* items */}</AutocompleteList>
    </AutocompletePopup>
  </AutocompletePositioner>
</AutocompletePortal>
```

                {(movie) => (
                  <AutocompleteItem key={movie.id} value={movie}>

                      {movie.title} ({movie.year})
                    </AutocompleteItemText>
                  </AutocompleteItem>
                )}
              </AutocompleteList>
            </AutocompleteContent>
          </Autocomplete>
        );
      }
    `}

  </Preview.Code>

  <Preview.Data>
    {`
      const topMovies = [
        { id: "1", title: "The Shawshank Redemption", year: 1994 },
        { id: "2", title: "The Godfather", year: 1972 },
        { id: "3", title: "The Dark Knight", year: 2008 },
        { id: "4", title: "Pulp Fiction", year: 1994 },
      ];

      function isFuzzyMatch(value, query) {
        const normalizedValue = value.toLowerCase().trim();
        const normalizedQuery = query.toLowerCase().trim();

        if (normalizedQuery === "") {
          return true;
        }

        let queryIndex = 0;

        for (const character of normalizedValue) {
          if (character === normalizedQuery[queryIndex]) {
            queryIndex += 1;

            if (queryIndex === normalizedQuery.length) {
              return true;
            }
          }
        }

        return false;
      }
    `}

  </Preview.Data>

### Async Search

Control `value`, disable the built-in filter with `filter={null}`, and keep `AutocompleteStatus` mounted while its children change.

<Preview>
  <AsyncSearchAutocompleteExample />

  <Preview.Code>
    {`
      import {
        Autocomplete,
        AutocompleteField,
        AutocompleteFieldLabel,
        AutocompleteInputGroup,
        AutocompleteInput,
        AutocompleteControlActions,
        AutocompleteClear,
        AutocompleteTrigger,
        AutocompleteContent,
        AutocompleteStatus,
        AutocompleteEmpty,
        AutocompleteList,
        AutocompleteItem,
        AutocompleteItemText,
        useAutocompleteFilter,
      } from "moduix";
      import { useId, useRef, useState, useTransition } from "react";

      export function AsyncSearchAutocompleteDemo() {
        const id = useId();
        const { contains } = useAutocompleteFilter();
        const [value, setValue] = useState("");
        const [searchResults, setSearchResults] = useState([]);
        const [isPending, startTransition] = useTransition();
        const abortControllerRef = useRef(null);
        const trimmedValue = value.trim();

        const status = isPending
          ? "Searching..."
          : trimmedValue === ""
            ? "Start typing to search movies..."
          : trimmedValue !== "" && searchResults.length === 0
            ? \`No matches for "\${trimmedValue}".\`
            : null;

        return (
          <Autocomplete
            items={searchResults}
            value={value}
            filter={null}
            itemToStringValue={(item) => item.title}
            onValueChange={(nextValue) => {
              setValue(nextValue);

              const controller = new AbortController();
              abortControllerRef.current?.abort();
              abortControllerRef.current = controller;

              if (nextValue.trim() === "") {
                setSearchResults([]);
                return;
              }

              startTransition(async () => {
                await new Promise((resolve) => setTimeout(resolve, 250));

                if (controller.signal.aborted) {
                  return;
                }

                setSearchResults(
                  topMovies.filter(
                    (movie) =>
                      contains(movie.title, nextValue) ||
                      contains(movie.year.toString(), nextValue),
                  ),
                );
              });
            }}
          >
            <AutocompleteField>
              <AutocompleteFieldLabel htmlFor={id}>
                Search movies by name or year
              </AutocompleteFieldLabel>
              <AutocompleteInputGroup>
                <AutocompleteInput id={id} placeholder="e.g. Pulp Fiction or 1994" />
                <AutocompleteControlActions>
                  <AutocompleteClear aria-label="Clear value" />
                  <AutocompleteTrigger aria-label="Open suggestions" />
                </AutocompleteControlActions>
              </AutocompleteInputGroup>
            </AutocompleteField>

            <AutocompleteContent>
              <AutocompleteStatus>{status}</AutocompleteStatus>
              <AutocompleteEmpty>
                {trimmedValue !== "" && !isPending ? "Try a different query." : null}
              </AutocompleteEmpty>
              <AutocompleteList>
                {(movie) => (
                  <AutocompleteItem key={movie.id} value={movie}>
                    <AutocompleteItemText>
                      {movie.title} ({movie.year})
                    </AutocompleteItemText>

backdrop, positioner, or arrow need their own styling.
