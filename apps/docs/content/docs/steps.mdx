## Anatomy

```text
Steps.Root
├─ Steps.Progress (optional)
├─ Steps.List
│  └─ Steps.Item[index]
│     ├─ Steps.Trigger
│     │  └─ Steps.Indicator
│     └─ Steps.Separator
├─ Steps.Content[index]
├─ Steps.CompletedContent
├─ Steps.PrevTrigger
└─ Steps.NextTrigger

Steps.RootProvider
└─ same part tree connected to a Steps.useSteps() store
```

| Part                                | Role                                                                       |
| ----------------------------------- | -------------------------------------------------------------------------- |
| `Steps` / `Steps.Root`              | Controls step state, orientation, linear navigation, validation, and IDs.  |
| `Steps.RootProvider`                | Connects parts to state created with `Steps.useSteps()`.                   |
| `Steps.Progress`                    | Optional progressbar that reads Ark's `--percent` root variable.           |
| `Steps.List`                        | Tablist that owns step triggers.                                           |
| `Steps.Item`                        | One zero-based step item with a required `index`.                          |
| `Steps.Trigger`                     | Tab button for a step; supports `asChild` for custom semantic hosts.       |
| `Steps.Indicator`                   | Visual step marker. Defaults to `index + 1` or a check icon when complete. |
| `Steps.Separator`                   | Decorative connector between items.                                        |
| `Steps.Content`                     | Tabpanel for one step, matched by `index`.                                 |
| `Steps.CompletedContent`            | Tabpanel shown when the current step equals `count`.                       |
| `Steps.PrevTrigger` / `NextTrigger` | Buttons wired to Ark's previous and next actions.                          |

Use `Steps.useSteps()` with `Steps.RootProvider` for externally owned state. Other advanced Ark
state helpers remain available directly from `@ark-ui/react/steps` as escape hatches.

## Composition

### Recommended composition

`Steps.Root` owns the state machine. Use `count`, `defaultStep`, `step`, `onStepChange(details)`, `onStepComplete`, `linear`, `isStepValid`, `isStepSkippable`, `onStepInvalid(details)`, `orientation`, and `ids` directly on the root. Step items and panels use zero-based `index` values.

Use `Steps.RootProvider` with `Steps.useSteps()` when external controls need direct access to `value`,
`setStep`, `goToNextStep`, `goToPrevStep`, `resetStep`, `percent`, and item-state helpers. Do not
render `Steps.Root` and `Steps.RootProvider` for the same store.

### When to go custom

Use `asChild` on `Steps.Trigger`, `Steps.PrevTrigger`, or `Steps.NextTrigger` when the host element should be an anchor, router link, or design-system button. The child must remain a single semantic element that can carry Ark's ARIA, keyboard, and focus props.

`Steps.Indicator` has one Moduix addition: when children are omitted, it renders `index + 1` for incomplete/current steps and the shared check icon for completed steps. Pass children to fully control the marker.

### Default props

| Prop          | Default      | Notes                                                                                |
| ------------- | ------------ | ------------------------------------------------------------------------------------ |
| `orientation` | `horizontal` | Ark root orientation. Use `vertical` for stacked step navigation.                    |
| `linear`      | `false`      | Set `true` to prevent arbitrary trigger jumps and require next/previous progression. |
| `count`       | `1`          | Ark's total step count. Pass the visible step array length in real UIs.              |
| `defaultStep` | `0`          | Initial uncontrolled step. `count` means completed.                                  |

### Behavioral notes

- `step`, `defaultStep`, `details.step`, and `Steps.Item index` are zero-based Ark values.
- `Steps.Content` uses `role="tabpanel"` and is hidden when its step is not current.
- `Steps.List` uses `role="tablist"` with `aria-orientation`; triggers use `role="tab"` and `aria-controls`.
- `Steps.PrevTrigger` and `Steps.NextTrigger` become disabled when Ark reports no previous or next step.
- `ids` can provide stable root/list/trigger/content IDs when surrounding composition needs them.

## Styling

Style `Steps` through `className`, Ark `data-scope="steps"` / `data-part="..."`, Ark state attributes, Moduix `data-slot` hooks, and the public `--steps-*` CSS variables shown below.

### Styling hooks

The useful public hooks are:

- `className` on `Steps`, `Steps.RootProvider`, `Steps.List`, `Steps.Item`, `Steps.Trigger`, `Steps.Indicator`, `Steps.Separator`, `Steps.Content`, `Steps.CompletedContent`, `Steps.PrevTrigger`, `Steps.NextTrigger`, and `Steps.Progress`
- `data-slot="steps-root"`, `steps-root-provider`, `steps-list`, `steps-item`, `steps-trigger`, `steps-indicator`, `steps-separator`, `steps-content`, `steps-completed-content`, `steps-prev-trigger`, `steps-next-trigger`, and `steps-progress`
- Ark `data-scope="steps"` and `data-part` values for every primitive part
- `data-orientation="horizontal" | "vertical"` on root/list/item/trigger/content/separator parts
- `data-current`, `data-complete`, and `data-incomplete` on indicator, separator, and trigger state surfaces
- `data-state="open" | "closed"` on triggers and content panels
- `data-complete` on `Steps.Progress`
- `--percent` from Ark on the root, consumed by `Steps.Progress`
