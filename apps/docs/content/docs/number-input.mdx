## Anatomy

```text
NumberInput.Root
├─ NumberInput.Label
├─ NumberInput.Scrubber (optional)
├─ NumberInput.Control
│  ├─ NumberInput.DecrementTrigger
│  ├─ NumberInput.Input
│  └─ NumberInput.IncrementTrigger
├─ NumberInput.ValueText (optional)
└─ NumberInput.Context (optional render-prop access)

NumberInput.RootProvider
└─ same part tree connected to a useNumberInput() store
```

| Part                                           | Role                                                                   |
| ---------------------------------------------- | ---------------------------------------------------------------------- |
| `NumberInput` / `NumberInput.Root`             | Controls number input state, parsing, formatting, keyboard behavior.   |
| `NumberInput.RootProvider`                     | Connects parts to state created with `useNumberInput()`.               |
| `NumberInput.Label`                            | Accessible label associated with the managed input.                    |
| `NumberInput.Scrubber`                         | Optional drag target that changes the value through pointer movement.  |
| `NumberInput.Control`                          | Structural wrapper for input and increment/decrement triggers.         |
| `NumberInput.DecrementTrigger`                 | Button that decreases the current value. Defaults to the moduix minus. |
| `NumberInput.Input`                            | Editable numeric text input managed by Ark UI.                         |
| `NumberInput.IncrementTrigger`                 | Button that increases the current value. Defaults to the moduix plus.  |
| `NumberInput.ValueText`                        | Read-only text surface for the formatted value.                        |
| `NumberInput.Context`                          | Reads root state through an Ark render prop.                           |
| `useNumberInput()` / `useNumberInputContext()` | Ark state hooks exposed through moduix.                                |

## Composition

### Recommended composition

`NumberInput.Root` owns parsing, range checks, keyboard increment/decrement, spin-on-press, mouse
wheel changes, scrubber interaction, and formatting. Use `NumberInput` as the short root form and
compose the visible parts explicitly.

Use string values for `value` and `defaultValue`. Ark reports changes through
`onValueChange(details)`, where `details.value` is the string value and `details.valueAsNumber` is
the parsed number.

### Default props

| Prop                               | Default | Notes                                                        |
| ---------------------------------- | ------- | ------------------------------------------------------------ |
| `NumberInput` / `NumberInput.Root` | —       | No additional behavioral defaults; Ark defaults apply.       |
| `NumberInput.DecrementTrigger`     | —       | Renders the moduix minus icon when no children are provided. |
| `NumberInput.IncrementTrigger`     | —       | Renders the moduix plus icon when no children are provided.  |

### Behavioral notes

- Use `formatOptions`, not `format`, for localized number formatting.
- Use `translations.incrementLabel`, `translations.decrementLabel`, and `translations.valueText` for accessible localized labels.
- `min`, `max`, `step`, `allowOverflow`, `clampValueOnBlur`, `spinOnPress`, `inputMode`, and `allowMouseWheel` pass directly to Ark.
- `onValueInvalid(details)` reports `rangeUnderflow` or `rangeOverflow` when the value exceeds configured bounds.
- Use `ids` when external labels or descriptions need stable element IDs.
- All exported parts support `asChild` for custom host composition.

## Examples

### Controlled

Keep controlled state as a string and read `details.value` in `onValueChange`.

<Preview>
  <ControlledNumberInputExample />

  <Preview.Code>
    {`
      import { NumberInput } from "@moduix/react";
      import { useState } from "react";

      export function ControlledNumberInputDemo() {
        const [value, setValue] = useState("24");

        return (
          <div>
            <NumberInput value={value} onValueChange={(details) => setValue(details.value)}>
              <NumberInput.Label>Controlled value</NumberInput.Label>
              <NumberInput.Control>
                <NumberInput.DecrementTrigger />
                <NumberInput.Input />
                <NumberInput.IncrementTrigger />
              </NumberInput.Control>
            </NumberInput>
            <div className="number-input-state">Current value: {value || "empty"}</div>
          </div>
        );
      }
    `}

  </Preview.Code>

### Root Provider

Use `useNumberInput()` with `NumberInput.RootProvider` when state must be created outside the
rendered part tree.

<Preview>
  <RootProviderNumberInputExample />

  <Preview.Code>
    {`
      import { NumberInput, useNumberInput } from "@moduix/react";

      export function RootProviderNumberInputDemo() {
        const numberInput = useNumberInput({ defaultValue: "3", min: 1, max: 10 });

        return (
          <div>
            <NumberInput.RootProvider value={numberInput}>
              <NumberInput.Label>Guests</NumberInput.Label>
              <NumberInput.Control>
                <NumberInput.DecrementTrigger />
                <NumberInput.Input />
                <NumberInput.IncrementTrigger />
              </NumberInput.Control>
            </NumberInput.RootProvider>
            <div className="number-input-root-provider-actions">
              <button type="button" onClick={() => numberInput.setToMin()}>
                Min
              </button>
              <button type="button" onClick={() => numberInput.setToMax()}>
                Max
              </button>
            </div>
          </div>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>{numberInputExampleCss}</Preview.CSS>
