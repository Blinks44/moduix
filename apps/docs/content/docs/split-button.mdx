## API Reference

Ark UI does not ship a dedicated split-button primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Basic

Use `SplitButton` as the short root form, then keep the Ark-style structure visible with
`SplitButton.Action`, `SplitButton.Trigger`, `SplitButton.Positioner`, and

## Anatomy

```text
SplitButton.Root
├─ SplitButton.Action
├─ SplitButton.Trigger
└─ Overlay subtree (automatically portalled)
   └─ SplitButton.Positioner
      └─ SplitButton.Content
         ├─ Menu.Item
         ├─ Menu.Separator
         └─ other shared Menu parts
```

| Part                               | Role                                                                  |
| ---------------------------------- | --------------------------------------------------------------------- |
| `SplitButton` / `SplitButton.Root` | Owns Ark Menu root state and the grouped visual wrapper.              |
| `SplitButton.Action`               | Primary Button action. Inherits root `variant` and `size` by default. |
| `SplitButton.Trigger`              | Ark Menu trigger rendered as a moduix Button with a chevron fallback. |
| `SplitButton.Positioner`           | Ark Menu positioner with placement variables and layer z-index.       |
| `SplitButton.Content`              | Ark Menu content surface for shared `Menu.*` rows.                    |

## Composition

`SplitButton.Root` portals its menu `Positioner` automatically. Use `portalled={false}` for inline rendering and `portalRef` for a custom container; `Positioner` and `Content` remain explicit and styleable.

### Recommended composition

`SplitButton.Root` forwards Ark `Menu.Root` props such as `open`, `defaultOpen`,
`onOpenChange(details)`, `onSelect(details)`, `closeOnSelect`, `positioning`, `ids`, `lazyMount`,
`unmountOnExit`, and `typeahead`.

Use `SplitButton.Action` for the primary path. Put secondary actions in `SplitButton.Content` with
the shared `Menu.Item`, `Menu.Separator`, `Menu.ItemGroup`, `Menu.CheckboxItem`, and `Menu.RadioItem`
parts.

### When to go custom

Use `asChild` on `SplitButton.Action` when the primary action should render as a semantic link or
another compatible element.

Use `asChild` on `SplitButton.Trigger` only with a single semantic child that can receive button
props, Ark Menu trigger ARIA, and keyboard behavior.

Use `positioning` on `SplitButton.Root` for popup placement. Keep `SplitButton.Positioner` and
`SplitButton.Content` explicit so consumers can style or configure the popup structure without
rewriting the component.

When the menu itself needs advanced state access, compose directly with `Menu.RootProvider` and
`useMenu()` from the shared `Menu` API rather than adding split-button-specific state wrappers.

### Default props

| Surface               | Prop          | Default                                  | Notes                                                  |
| --------------------- | ------------- | ---------------------------------------- | ------------------------------------------------------ |
| `SplitButton.Root`    | `variant`     | `'default'`                              | Shared default for `Action` and `Trigger`.             |
| `SplitButton.Root`    | `size`        | `'md'`                                   | Shared default for `Action` and `Trigger`.             |
| `SplitButton.Root`    | `positioning` | `{ placement: 'bottom-end', gutter: 4 }` | Keeps the popup aligned to the trigger edge.           |
| `SplitButton.Trigger` | `aria-label`  | `'More actions'` when icon-only          | Override for a more specific accessible name.          |
| `SplitButton.Trigger` | `children`    | `ChevronDownIcon`                        | Replace when the trigger needs custom visible content. |

### Behavioral notes

- `SplitButton` has no legacy render contract and no flat part exports.
- `SplitButton.Action` follows the shared moduix `Button` `asChild` contract.
- `SplitButton.Trigger` renders a moduix `Button` as the Ark Menu trigger host. It keeps
  `data-slot="split-button-trigger"` and receives Ark trigger ARIA/state attributes such as
  `aria-expanded`, `aria-controls`, and `data-state`.
- `SplitButton.Positioner` receives Ark Menu CSS variables such as `--reference-width`,
  `--available-height`, `--x`, `--y`, `--z-index`, and `--transform-origin`.
- The root `variant` excludes the `link` button variant because grouped link-style split buttons do
  not form a coherent control.
- The root `size` excludes icon-only button sizes because the default path expects a visible label on
  the primary action.

## Styling

`SplitButton` uses moduix `Button` and `Menu` styles, plus split-button slots for the grouped
visual treatment.

### Styling hooks

- `data-scope="split-button"` and `data-part="root"` target the grouped wrapper.
- `data-slot="split-button-root"` targets the grouped wrapper.
- `data-slot="split-button-action"` targets the primary action side.
- `data-slot="split-button-trigger"` targets the dropdown trigger side.
- `data-slot="split-button-positioner"` targets the Ark Menu positioner wrapper.
- `data-slot="split-button-content"` targets the Ark Menu content surface.
- `SplitButton.Trigger` renders through the shared `Button` host and carries Ark Menu trigger ARIA
  and state attributes including `data-state="open" | "closed"`.
- `SplitButton.Content` carries Ark Menu content attributes including `data-state`, `data-side`,
  `data-placement`, `data-nested`, and `data-has-nested`.
