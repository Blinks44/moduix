## API Reference

`Tag` is a standalone moduix component for selected values, filters, assignees, and other compact
tokens that may need an inline remove action. Use `TagRemove` when the token should expose a clear,
explicit dismiss button.

## Choosing the right component

Use `Tag` when the UI is showing **the selected value itself** and the value may need to be removed,
reordered, or otherwise managed as a token.

- Choose **`Badge`** when the UI is showing short metadata such as status, category, version, or count.
- Choose **`Tag`** when the UI is showing selected values, filters, assignments, or labels that belong to the current state of the interface.

## Anatomy

The default consumer path keeps the token presentational and the remove action explicit:

```text
Tag
├─ svg icon (optional)
├─ TagLabel
└─ TagRemove (optional)
```

```tsx
<Tag>
  <TagLabel>TypeScript</TagLabel>
  <TagRemove aria-label="Remove TypeScript tag" />
</Tag>
```

| Part        | Role                                                                |
| ----------- | ------------------------------------------------------------------- |
| `Tag`       | Root slot. Applies token layout, border, size, and variant styling. |
| `TagLabel`  | Truncation wrapper for the visible token value.                     |
| `TagRemove` | Optional inline action for dismissing or removing the token.        |

## Composition

`Tag` stays a plain `span`, so the component does not own selection state, remove behavior, or list
management. Consumers decide whether to render `TagRemove` and what should happen on click.

Keep the token value in `TagLabel` when long text may truncate. Leading icons can stay as direct
children of `Tag`; they inherit `currentColor` and use `--tag-icon-size`. `TagRemove` is a styled
Base UI button that defaults to `type="button"` and renders a close icon when `children` are
omitted.

| Prop      | Default   | Values                                                    |
| --------- | --------- | --------------------------------------------------------- |
| `variant` | `default` | `default`, `secondary`, `outline`, `ghost`, `destructive` |
| `size`    | `md`      | `sm`, `md`                                                |

## Examples

Use the built-in variants for the common visual tones.

changing the public API.
