## API Reference

`Tag` is a standalone moduix component for selected values, filters, assignees, and other compact
state tokens. There is no dedicated Ark UI `Tag` primitive; `TagRemove` composes the shipped primitive
button when the token needs an explicit inline remove action.

## Choosing the right component

Use `Tag` when the UI is showing **the selected value itself** and that value may need to be
removed, reordered, or managed as part of the current interface state.

- Choose **`Badge`** when the UI is showing short metadata such as status, category, version, or count.
- Choose **`Tag`** when the UI is showing selected values, filters, assignments, or labels that belong to the current state of the interface.

## Basic

Start with `Tag` and `TagLabel`. Add `TagRemove` only when the surrounding UI actually supports
removing the token.

## Anatomy

The public structure stays small and keeps the remove action explicit.

```text
Tag
├─ svg icon (optional)
├─ TagLabel
└─ TagRemove (optional)
```

| Part        | Role                                                                |
| ----------- | ------------------------------------------------------------------- |
| `Tag`       | Root slot. Applies token layout, border, size, and variant styling. |
| `TagLabel`  | Truncation wrapper for the visible token value.                     |
| `TagRemove` | Optional inline action for dismissing or removing the token.        |

## Composition

### When to stay high-level

Stay with `Tag`, `TagLabel`, and `TagRemove` for selected values, filter chips, assignees, and
other token-like UI.

Use `variant` for the built-in tones: `default`, `secondary`, `outline`, `ghost`, and
`destructive`. Use `size="sm"` for denser rows of tokens.

Keep long visible values inside `TagLabel` so truncation happens on the label instead of clipping
icons or the remove button.

### When to go custom

Switch to `className` overrides and local CSS variables when the token needs a different tone,
spacing, or inline remove treatment. `Tag` stays a presentational `span`, so there is no separate
low-level composition path to learn.

### Default props

| Surface     | Prop         | Default      | Values / Notes                                                                |
| ----------- | ------------ | ------------ | ----------------------------------------------------------------------------- |
| `Tag`       | `variant`    | `default`    | `default`, `secondary`, `outline`, `ghost`, `destructive`                     |
| `Tag`       | `size`       | `md`         | `sm`, `md`                                                                    |
| `TagRemove` | `type`       | `button`     | Prevents accidental form submission                                           |
| `TagRemove` | `children`   | close icon   | Replaced when custom children are passed                                      |
| `TagRemove` | `aria-label` | `Remove tag` | Applied only when there are no custom children and no `aria-labelledby` value |

### Behavioral notes

- `Tag` is presentational and never becomes interactive on its own.
- `TagRemove` is explicit composition, not a root-level `onRemove` prop.
- `TagRemove` uses `Button` semantics, so give it a specific accessible name when several removable tags appear together.
- Direct child `svg` icons inherit `currentColor`, do not receive pointer events, and use `--tag-icon-size`.

### Styling hooks

- `data-slot="tag-root"`, `data-slot="tag-label"`, and `data-slot="tag-remove"` are stable selectors on the exported parts.
- `Tag` writes the selected size and tone to `data-size` and `data-variant`.
- `TagLabel` is the intended truncation wrapper for long values.
- `TagRemove` inherits the token color, uses `--tag-remove-icon-size` for its icon, and exposes the same `className` override path as other exported parts.
