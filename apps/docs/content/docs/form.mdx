## Basic

Start with `Form` as the native root, compose fields with `Field`, and use `onFormSubmit` when
object-shaped values are easier to work with than raw `FormData`.

<Preview>
  <FormExample />

  <Preview.Code>
    {`
      import {
        Button,
        Field,
        FieldError,
        FieldLabel,
        Form,
        Input,
        Spinner,
      } from "moduix";
      import { useState } from "react";

      async function validateHomepage(homepage: string) {
        await new Promise((resolve) => {
          setTimeout(resolve, 500);
        });

        try {
          const value = new URL(homepage);

          if (value.hostname.endsWith("example.com")) {
            return {
              homepage: "The example.com domain is not allowed.",
            };
          }

          return {};
        } catch {
          return {
            homepage: "Please enter a valid URL.",
          };
        }
      }

      export function FormDemo() {
        const [errors, setErrors] = useState({} as Record<string, string>);
        const [submitting, setSubmitting] = useState(false);

        return (
          <Form
            errors={errors}
            validationMode="onBlur"
            className="form"
            onFormSubmit={async (values) => {
              setSubmitting(true);
              const nextErrors = await validateHomepage(String(values.homepage ?? ""));
              setErrors(nextErrors);
              setSubmitting(false);
            }}
          >
            <Field name="homepage">
              <FieldLabel>Homepage</FieldLabel>
              <Input
                type="url"
                required
                defaultValue="https://example.com"
                placeholder="https://example.com"
                pattern="https?://.*"

              <FieldError match="patternMismatch">
                Please start with http:// or https://.
              </FieldError>
              <FieldError />
            </Field>
            <Button
              type="submit"
              disabled={submitting}
              aria-busy={submitting || undefined}
            >
              {submitting ? (
                <>
                  <Spinner decorative size="sm" />
                  Submitting
                </>
              ) : (
                "Submit"
              )}
            </Button>
          </Form>
        );
      }

## Anatomy

The high-level path is intentionally short:

```text
Form
├─ Field
│  ├─ FieldLabel
│  ├─ control (Input, FieldControl, Textarea, Select, ...)
│  └─ FieldDescription / FieldError
└─ submit actions
```

| Part   | Role                                                                      |
| ------ | ------------------------------------------------------------------------- |
| `Form` | Native form root that coordinates submit lifecycle and form-level errors. |

## Composition

### When to stay high-level

Stay with `Form`, `Field`, and the exported field controls for most forms. `Form` keeps the native
`<form>` element, sets shared validation timing through `validationMode`, and surfaces external
field errors through the `errors` prop.

Use `onFormSubmit` for the default submit path when object-shaped values are enough for the handler.

### When to go custom

Drop to native `onSubmit` when you need the real submit event, `event.currentTarget`, manual
`FormData`, or submitter-specific logic.

Use `action` with React `useActionState` or server actions when the submit flow should stay native
and action results need to feed back into `FieldError`.

Use `actionsRef` as the narrow imperative escape hatch when another control needs to validate one
field or the whole form before the next user step.

### Default props

| Prop             | Default       | Values                                            |
| ---------------- | ------------- | ------------------------------------------------- |
| `validationMode` | `onSubmit`    | `onSubmit`, `onBlur`, `onChange`                  |
| `render`         | native `form` | Any valid React element or component via `render` |

### Behavioral notes

- `errors` keys should match the nested `Field name` values so `FieldError` can render them.
- `onFormSubmit` is the convenience path for object-shaped values; Base UI prevents the native
  submit event default before this callback runs.
- Native `action` and native `onSubmit` still work when you need lower-level form orchestration.
- A nested `Field` can override the root `validationMode` when one control needs different timing.

### Action State

Use a form `action` with React `useActionState` when action results should feed back into field
errors.

<Preview>
  <FormActionStateExample />

  <Preview.Code>
    {`
      import {
        Button,
        Field,
        FieldError,
        FieldLabel,
        Form,
        Input,
        Spinner,
      } from "moduix";
      import { useActionState } from "react";

      interface ActionState {
        serverErrors?: Record<string, string>;
        message?: string;
      }

      async function submitUsername(
        _previousState: ActionState,
        formData: FormData,
      ) {
        await new Promise((resolve) => {
          setTimeout(resolve, 500);
        });

        const username = String(formData.get("username") ?? "");

        if (!username.trim()) {
          return {
            serverErrors: {
              username: "Username is required.",
            },
          };
        }

        if (username.toLowerCase() === "admin") {
          return {
            serverErrors: {
              username: "'admin' is reserved for system use.",
            },
          };
        }

        return {
          serverErrors: {},
          message: \`Saved as \${username}.\`,
        };
      }

      export function ActionStateForm() {
        const [state, formAction, loading] = useActionState(
          submitUsername,
          {} as ActionState,
        );

        return (
          <Form
            action={formAction}
            errors={state.serverErrors}
            validationMode="onBlur"
            className="form"
          >
            <Field name="username">
              <FieldLabel>Username</FieldLabel>
              <Input required defaultValue="admin" placeholder="e.g. alice132" />
              <FieldError />
            </Field>
            <Button
              type="submit"
              disabled={loading}
              aria-busy={loading || undefined}
            >
              {loading ? (
                <>
                  <Spinner decorative size="sm" />
                  Submitting
                </>
              ) : (
                "Submit"
              )}
            </Button>
            {state.message ? <p className="helper">{state.message}</p> : null}
          </Form>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .form {
        width: min(20rem, 100%);
      }

      .helper {
        margin: 0;
        color: var(--color-muted-foreground);
        font-size: var(--text-sm);
        line-height: var(--line-height-text-sm);
      }
    `}

  </Preview.CSS>
</Preview>

### Actions Ref

Pass `actionsRef` when another control needs to trigger validation for one field or the whole form.

<Preview>
  <FormActionsRefExample />

  <Preview.Code>
    {`
      import {
        Button,
        Field,
        FieldError,
        FieldLabel,
        Form,
        Input,
      } from "moduix";
      import { useRef, useState } from "react";

      export function ActionsRefForm() {
        const actionsRef = useRef<{
          validate: (fieldName?: string) => void;
        } | null>(null);
        const [errors, setErrors] = useState({} as Record<string, string>);

        return (
          <Form
            actionsRef={actionsRef}
            errors={errors}
            validationMode="onSubmit"
            className="form"
            onFormSubmit={(values) => {
              const nextErrors: Record<string, string> = {};
              const email = String(values.email ?? "");

              if (!email.trim()) {
                nextErrors.email = "Email is required.";
              } else if (!email.endsWith("@test.com")) {
                nextErrors.email = "Use a @test.com email.";
              }

              setErrors(nextErrors);
            }}
          >
            <Field name="email">
              <FieldLabel>Work Email</FieldLabel>
              <Input type="email" required placeholder="name@test.com" />
              <FieldError />
            </Field>
            <Button
              type="button"
              variant="outline"
              onClick={() => {
                actionsRef.current?.validate("email");
              }}
            >
              Validate Email
            </Button>
            <Button type="submit">Submit</Button>
          </Form>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
    {`
      .form {
        width: min(20rem, 100%);
      }
    `}

  </Preview.CSS>
</Preview>

### Styling hooks

- `data-slot="form-root"` is the stable selector on the exported root.
- `className` can be a string or Base UI callback class name; moduix merges it with the default
  root styles.
- `--form-gap`, `--form-width`, and `--form-max-width` are the full public root layout contract.
- Compose deeper visual changes through `Field`, `Input`, `Textarea`, `Select`, and submit controls
  instead of expecting extra form-specific slots.
