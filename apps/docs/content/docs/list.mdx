## API Reference

`List` is a moduix-native wrapper over semantic `ul`, `ol`, and `li` elements, so the shipped API
surface is documented on this page.

## Basic

Start with `List` and `ListItem`. This is the recommended path when you want native list semantics
with moduix spacing, typography, and marker tokens.

## Anatomy

The normal consumer path is short:

```text
List (ul | ol)
└─ li | ListItem
```

| Part       | Role                                                                |
| ---------- | ------------------------------------------------------------------- |
| `List`     | Semantic list root that controls marker, spacing, size, and tone.   |
| `ListItem` | Optional thin wrapper around a native `li` with a stable item slot. |

## Composition

### When to stay high-level

Stay with `List` and `ListItem` for most content blocks, settings summaries, and ordered step
lists.

Use `as` to choose list semantics, `marker` for the common marker presets, and `gap`, `size`, plus
`tone` for the design-system presentation layer.

### When to go custom

`List` does not expose a lower-level subtree or marker render API. When you need something more
custom, keep the semantic root and switch to plain `li` elements, wrapper layout outside the list,
or native CSS such as `li::marker`.

Keep items as direct `li` children. The built-in spacing and marker styling apply only to direct
list items.

### Default props

| Prop     | Default   | Values                                                 |
| -------- | --------- | ------------------------------------------------------ |
| `as`     | `ul`      | `ul`, `ol`                                             |
| `marker` | by `as`   | `disc`, `decimal`, `none`                              |
| `gap`    | `sm`      | `xs`, `sm`, `md`, `lg`, `xl`, `2xl`                    |
| `size`   | `md`      | `xs`, `sm`, `md`, `lg`, `xl`                           |
| `tone`   | `default` | `default`, `muted`, `subtle`, `primary`, `destructive` |

### Behavioral notes

- Ordered-list props such as `start`, `reversed`, and `type` continue to pass through when
  `as="ol"` is used.
- Leave `marker` on its default ordered path when you want browser-provided marker styles such as
  `type="A"` or `type="i"`.
- Use `marker="none"` to hide markers while keeping list semantics. `List` adds `role="list"` on
  that path unless you explicitly override `role`.
- `ListItem` is optional. Use plain `li` elements when the exported item slot adds no value.

### Ordered Start

Use `as="ol"` for ordered steps. Native ordered-list props still work, so `start={3}` continues
numbering from the underlying `ol`.

### Styling hooks

`List` exposes a small public styling surface:

- `data-slot="list-root"` on the semantic root
- `data-slot="list-item"` on `ListItem`
- `data-gap="xs" | "sm" | "md" | "lg" | "xl" | "2xl"` for spacing-specific styling
- `data-marker="disc" | "decimal" | "none"` for marker-specific styling
- `data-marker-mode="auto" | "explicit"` when marker behavior depends on the root element or an
  explicit override
- `data-size="xs" | "sm" | "md" | "lg" | "xl"` for typography size styling
- `data-tone="default" | "muted" | "subtle" | "primary" | "destructive"` for semantic color paths
- native `li::marker` targeting for marker-specific color or weight overrides
