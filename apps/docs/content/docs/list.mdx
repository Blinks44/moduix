## API Reference

Ark UI does not ship a dedicated list primitive. Moduix builds this component with the official
[Ark factory](https://ark-ui.com/docs/guides/composition#the-ark-factory), follows Ark's
[composition](https://ark-ui.com/docs/guides/composition), [styling](https://ark-ui.com/docs/guides/styling), and
[ref](https://ark-ui.com/docs/guides/ref) guidance.

## Anatomy

The normal consumer path is short:

```text
List / List.Root
└─ List.Item | li
```

| Part                 | Role                                                               |
| -------------------- | ------------------------------------------------------------------ |
| `List` / `List.Root` | Semantic list root that controls marker, spacing, size, and tone.  |
| `List.Item`          | Optional Ark-style item part for a stable `li` slot and `asChild`. |

## Composition

### Recommended composition

Stay with `<List>` for unordered content and use `as="ol"` when you need ordered semantics. Use `asChild` only when the caller needs to own a custom semantic host.

### When to go custom

`List` does not expose an internal state machine, nested structure helpers, or a marker render API. When you need a custom host element or per-item markup, use `asChild`, plain `li` children, and native CSS such as `li::marker`.

Keep items as direct `li` descendants. The built-in spacing and marker styling apply only to direct list items.

### Default props

| Prop      | Default         | Notes                                                                |
| --------- | --------------- | -------------------------------------------------------------------- |
| `gap`     | `sm`            | Spacing between direct list items.                                   |
| `size`    | `md`            | Typography size token.                                               |
| `tone`    | `default`       | Semantic color path for text and markers.                            |
| `marker`  | semantic `auto` | `disc` on `ul`, browser-provided ordered markers on `ol`, or `none`. |
| `as`      | `ul`            | Switches the native list host between `ul` and `ol`.                 |
| `asChild` | `false`         | Replaces the semantic host with a single caller-owned child element. |

### Behavioral notes

- Ordered-list props such as `start`, `reversed`, and `type` are available when `as="ol"` is used.
- Use `marker="none"` to hide markers while keeping list semantics. `List` adds `role="list"` on that path unless you explicitly override `role`.
- `List.Item` is optional. Use plain `li` elements when the exported item slot adds no value.

### Styling hooks

`List` exposes a small public styling surface:

- root: `data-scope="list"`, `data-part="root"`, `data-slot="list-root"`
- item: `data-scope="list"`, `data-part="item"`, `data-slot="list-item"`
- `data-gap="xs" | "sm" | "md" | "lg" | "xl" | "2xl"` for spacing-specific styling
- `data-marker="auto" | "disc" | "decimal" | "none"` for marker-specific styling
- `data-size="xs" | "sm" | "md" | "lg" | "xl"` for typography size styling
- `data-tone="default" | "muted" | "subtle" | "primary" | "destructive"` for semantic color paths
- native `li::marker` targeting for marker-specific color or weight overrides
