## Overview

`List` keeps native `ul` and `ol` semantics and adds library-level control over
markers, spacing, typography, and tone. `ListItem` is optional: use it for the
exported item slot, or render plain `li` elements directly.

When `as="ol"` is used, native ordered-list props like `start`, `reversed`, and `type`
continue to pass through to the underlying `ol`. Leave `marker` on its default ordered
path when you want browser-provided alpha or roman numbering from `type`.

## Anatomy

`List` renders a semantic list root with direct `li` children. The root owns marker
style, spacing, and typography tokens.

```text
List (ul | ol)
└─ li | ListItem
```

| Part       | Role                                                                |
| ---------- | ------------------------------------------------------------------- |
| `List`     | Semantic list root that controls marker, spacing, size, and tone.   |
| `ListItem` | Optional thin wrapper around a native `li` with a stable item slot. |

## Composition

Use `as` for list semantics, `marker` for marker style or visibility, and `size`,
`gap`, `tone`, plus `className` for visual customization.

The default marker follows the root element: `ul` uses `disc`, `ol` uses `decimal`.
Use `marker="none"` to keep list semantics without visible markers. For deeper marker
styling, rely on native `li::marker` selectors and root CSS variables instead of a
custom marker API.

Use `ListItem` when you want the stable item slot. Use plain `li` elements when a
wrapper component would add no value. Keep items as direct `li` children so the root
spacing and marker styles keep applying.

### Defaults

| Prop     | Default   | Values                                                 |
| -------- | --------- | ------------------------------------------------------ |
| `as`     | `ul`      | `ul`, `ol`                                             |
| `marker` | by `as`   | `disc`, `decimal`, `none`                              |
| `gap`    | `sm`      | `xs`, `sm`, `md`, `lg`, `xl`, `2xl`                    |
| `size`   | `md`      | `xs`, `sm`, `md`, `lg`, `xl`                           |
| `tone`   | `default` | `default`, `muted`, `subtle`, `primary`, `destructive` |

## Examples

### Ordered Start

Use `as="ol"` for ordered steps. Native ordered-list props still work, so `start={3}`
continues numbering from the underlying `ol`.

<Preview>
  <OrderedListExample />

  <Preview.Code>
    {`
      import { List, ListItem } from "moduix";

      export function OrderedStartListDemo() {
        return (
          <List as="ol" start={3} className={styles.list}>
            {items.map((item) => (
              <ListItem key={item}>{item}</ListItem>
            ))}
          </List>
        );
      }
    `}

  </Preview.Code>

  <Preview.CSS>
