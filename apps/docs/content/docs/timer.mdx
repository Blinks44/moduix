## Anatomy

```text
Timer.Root
├─ Timer.Area
│  ├─ Timer.Item
│  └─ Timer.Separator
└─ Timer.Control
   └─ Timer.ActionTrigger

Timer.RootProvider
└─ same part tree connected to Ark useTimer()
```

| Part                   | Role                                                                    |
| ---------------------- | ----------------------------------------------------------------------- |
| `Timer` / `Timer.Root` | Owns the Ark timer state, ids, interval, and callbacks.                 |
| `Timer.RootProvider`   | Connects parts to timer state created with Ark `useTimer()`.            |
| `Timer.Area`           | Groups the visible time parts.                                          |
| `Timer.Item`           | Renders a formatted time unit such as `hours`, `minutes`, or `seconds`. |
| `Timer.Separator`      | Renders separators between time units.                                  |
| `Timer.Control`        | Groups action triggers.                                                 |
| `Timer.ActionTrigger`  | Dispatches `start`, `pause`, `resume`, `reset`, or `restart`.           |

## Composition

### Recommended composition

`Timer` is the short root form. It is equivalent to `Timer.Root`, and the child parts stay
explicit so the rendered anatomy matches Ark UI.

Use `targetMs` for a stopwatch that stops at a target time. Use `startMs` when the initial elapsed
or remaining time should not be zero. Use `countdown` with `startMs` for a countdown timer.

### When to go custom

Use `Timer.RootProvider` with Ark `useTimer()` when the timer API must be created outside the
rendered subtree. Use Ark `Timer.Context` or `useTimerContext()` when nearby UI needs access to
`running`, `paused`, `time`, `formattedTime`, `progressPercent`, or the programmatic `start()`,
`pause()`, `resume()`, `reset()`, and `restart()` methods.

Use `interval` for sub-second displays. Keep the interval as coarse as the UI allows; the default
Ark interval is `1000`.

### Default props

| Prop              | Default   | Notes                                                               |
| ----------------- | --------- | ------------------------------------------------------------------- |
| `autoStart`       | `false`   | Starts the timer on mount.                                          |
| `countdown`       | `false`   | Counts down instead of up.                                          |
| `startMs`         | `0`       | Initial duration in milliseconds.                                   |
| `targetMs`        | —         | Target duration in milliseconds.                                    |
| `interval`        | `1000`    | Tick interval in milliseconds.                                      |
| `ids`             | generated | Overrides the root and area ids.                                    |
| `translations`    | built in  | Customizes the accessible area label through `areaLabel`.           |
| `onTick(details)` | —         | Receives the current value, numeric time, and formatted time parts. |
| `onComplete()`    | —         | Runs when the timer reaches its target.                             |

### Behavioral notes

`Timer.ActionTrigger` renders a button by default, so keyboard activation follows native button
behavior. Ark hides state-dependent start, pause, resume, and reset actions when they are not
available; `restart` remains available when rendered. Timer does not use roving focus or composite
keyboard navigation.

All exported parts support `asChild` through Ark polymorphic composition. Keep replacement
children semantic, especially for `Timer.ActionTrigger`.

`Timer.Item` preserves Ark `data-scope="timer"`, `data-part="item"`, `data-type`, and `--value`.
Unit labels are plain composition around items. moduix adds centered root layout, stable
`data-slot` hooks, SVG icon sizing inside action triggers, and CSS defaults without changing the
Ark timer machine.

## Styling

Use `className` on exported parts for structural overrides and CSS variables for visual defaults.
Ark attributes remain available for part-specific styling.

### Styling hooks

- `data-slot="timer-root"`, `timer-root-provider`, `timer-area`, `timer-item`,
  `timer-separator`, `timer-control`, and `timer-action-trigger`.
- Ark anatomy hooks: `data-scope="timer"` and
  `data-part="root|area|item|separator|control|action-trigger"`.
- `Timer.Item` exposes `data-type` and Ark's `--value` runtime variable.
- Ark applies `hidden` to action triggers that do not match the current timer state.
