## Choosing the right component

Choose `ProgressCircular` for compact inline status, cards, toolbar badges, and small dashboards.
Choose `ProgressLinear` when horizontal comparison against a full track matters.

## Composition

### Recommended composition

`ProgressCircular` is the short root form. It is equivalent to `ProgressCircular.Root`, and the SVG
circle parts stay explicit so the rendered anatomy matches Ark UI.

Use `defaultValue` for uncontrolled progress, `value` with `onValueChange(details)` for controlled
progress, and `defaultValue={null}` or `value={null}` for indeterminate progress.

### When to go custom

Use `ProgressCircular.RootProvider` with `useProgress()` from `@ark-ui/react/progress` when
progress state must be created outside the rendered subtree.

Use `translations.value(details)` for custom visible and accessible value text. Wrap the circle and
value text in your own layout element when the number should sit in the center.

Use `ids` when the label, circle, or surrounding layout need stable explicit IDs across a more
custom tree. Use `ProgressCircular.View` when supporting UI should switch with the Ark progress
state instead of duplicating state checks outside the component. If you need Ark context APIs or
hooks, import them directly from `@ark-ui/react/progress`.

### Default props

| Prop            | Default                | Notes                                   |
| --------------- | ---------------------- | --------------------------------------- |
| `defaultValue`  | `50`                   | Initial uncontrolled value.             |
| `min`           | `0`                    | Lower bound for percent and ARIA value. |
| `max`           | `100`                  | Upper bound for percent and ARIA value. |
| `formatOptions` | `{ style: 'percent' }` | Controls `ValueText` formatting.        |
| `locale`        | `'en-US'`              | Locale used by `Intl.NumberFormat`.     |

### Behavioral notes

`ProgressCircular.Circle` receives the progressbar role and ARIA value attributes from Ark. Ark
also owns the SVG variables for radius, circumference, percent, and stroke offset.

All exported parts support `asChild` through Ark polymorphic composition. Keep replacement children
semantic and preserve the progressbar semantics on the circle.

`ProgressCircular.ValueText` announces changes with `aria-live="polite"`. Keep the visible label in
the tree, or provide `aria-label`/`aria-labelledby` when composing without `ProgressCircular.Label`.

## Styling

Use `className` on exported parts for structural styling changes. For state styling, prefer Ark
data attributes, Ark SVG variables, and moduix `data-slot` hooks.

### Styling hooks

- `data-slot="progress-circular-root"`, `progress-circular-root-provider`,
  `progress-circular-label`, `progress-circular-circle`, `progress-circular-circle-track`,
  `progress-circular-circle-range`, `progress-circular-value-text`, and `progress-circular-view`.
- Ark hooks: `data-scope="progress"`, `data-part`, `data-state="indeterminate|loading|complete"`,
  `data-orientation`, `data-value`, and `data-max`.
- Ark circle variables: `--size`, `--thickness`, `--radius`, `--circumference`, `--percent`, and
  stroke offset values generated by Ark.
