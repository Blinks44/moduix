# Async List

## API Reference

<PrimitiveReference href="https://ark-ui.com/docs/collections/async-list" />

`useAsyncList` manages remote list data and the request lifecycle around it. It owns loaded items,
loading and sorting state, errors, filtering, pagination cursors, and request cancellation.

```tsx
import { useAsyncList } from '@ark-ui/react/collection';
```

## Basic loading

Set `autoReload: true` when the first request should run on mount. Pass the provided `signal` to
`fetch` so a superseded request can be cancelled.

```tsx
type User = {
  id: number;
  name: string;
};

function UserList() {
  const list = useAsyncList<User>({
    autoReload: true,
    async load({ signal }) {
      const response = await fetch('/api/users', { signal });

      if (!response.ok) {
        throw new Error('Could not load users');
      }

      return { items: await response.json() };
    },
  });

  if (list.loading && list.items.length === 0) return <p>Loading…</p>;
  if (list.error instanceof Error) return <p>{list.error.message}</p>;

  return (
    <>
      <ul>
        {list.items.map((user) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
      <button type="button" onClick={list.reload} disabled={list.loading}>
        Reload
      </button>
    </>
  );
}
```

`autoReload` defaults to `false`. Without it, call `list.reload()` when the application is ready to
load.

## Configuration

| Prop                    | Purpose                                                     | Default     |
| ----------------------- | ----------------------------------------------------------- | ----------- |
| `load`                  | Required async loader.                                      | —           |
| `sort`                  | Optional client-side async or synchronous sorting function. | —           |
| `autoReload`            | Run the initial load on mount.                              | `false`     |
| `initialItems`          | Items shown before the first successful load.               | `[]`        |
| `dependencies`          | Primitive values that trigger a reload when changed.        | `[]`        |
| `initialFilterText`     | Initial server-side filter text.                            | `''`        |
| `initialSortDescriptor` | Initial `{ column, direction }` descriptor.                 | `undefined` |
| `onSuccess`             | Called with the successfully loaded items.                  | —           |
| `onError`               | Called with the request error.                              | —           |

Dependencies support `string`, `number`, `boolean`, `null`, and `undefined`. Keep object and array
state out of the dependency list; derive stable primitive values instead.

## Load contract

Every `load` call receives:

| Value            | Purpose                                                  |
| ---------------- | -------------------------------------------------------- |
| `signal`         | Abort signal for the active request.                     |
| `filterText`     | Current text set through `setFilterText`.                |
| `cursor`         | Cursor returned by the previous load, when loading more. |
| `sortDescriptor` | Current sort column and direction.                       |

Return an object with `items` and an optional `cursor`:

```tsx
const list = useAsyncList<User, string>({
  autoReload: true,
  async load({ cursor, signal }) {
    const params = new URLSearchParams();

    if (cursor) params.set('cursor', cursor);

    const response = await fetch(`/api/users?${params}`, { signal });
    if (!response.ok) throw new Error('Could not load users');

    const data: { users: User[]; nextCursor?: string } = await response.json();
    return {
      items: data.users,
      cursor: data.nextCursor,
    };
  },
});
```

## State and actions

| State            | Meaning                                    |
| ---------------- | ------------------------------------------ |
| `items`          | Current loaded items.                      |
| `filterText`     | Current filter text.                       |
| `cursor`         | Cursor for the next page.                  |
| `sortDescriptor` | Current sort descriptor.                   |
| `loading`        | A load request is running.                 |
| `sorting`        | A sort operation is running.               |
| `empty`          | The current item array is empty.           |
| `hasMore`        | A cursor exists for another page.          |
| `error`          | Error from the most recent failed request. |

| Action                 | Purpose                                        |
| ---------------------- | ---------------------------------------------- |
| `reload()`             | Replace items with a fresh load.               |
| `loadMore()`           | Load the current cursor and append the result. |
| `abort()`              | Cancel the active request.                     |
| `setFilterText(value)` | Set filter text and reload.                    |
| `clearFilter()`        | Clear filter text.                             |
| `sort(descriptor)`     | Apply a sort descriptor.                       |

## Filtering

Filtering is server-driven through the `load` callback:

```tsx
function UserSearch() {
  const list = useAsyncList<User>({
    autoReload: true,
    async load({ filterText, signal }) {
      const params = new URLSearchParams({ query: filterText });
      const response = await fetch(`/api/users?${params}`, { signal });

      if (!response.ok) throw new Error('Search failed');
      return { items: await response.json() };
    },
  });

  return (
    <input
      type="search"
      value={list.filterText}
      onChange={(event) => list.setFilterText(event.currentTarget.value)}
    />
  );
}
```

`useAsyncList` cancels superseded loads. The application controls when or whether to debounce
`setFilterText`; the server controls matching rules and result limits.

## Pagination

Return a cursor while more data is available, then call `loadMore()`:

```tsx
{
  list.hasMore && (
    <button type="button" onClick={list.loadMore} disabled={list.loading}>
      {list.loading ? 'Loading…' : 'Load more'}
    </button>
  );
}
```

Do not compute `hasMore` separately; it reflects whether the last result returned a cursor.

## Sorting

For server-side sorting, read `sortDescriptor` in `load`:

```tsx
function useServerSortedUsers() {
  return useAsyncList<User>({
    autoReload: true,
    async load({ sortDescriptor, signal }) {
      const params = new URLSearchParams();

      if (sortDescriptor) {
        params.set('sort', String(sortDescriptor.column));
        params.set('direction', sortDescriptor.direction);
      }

      const response = await fetch(`/api/users?${params}`, { signal });
      if (!response.ok) throw new Error('Could not sort users');

      return { items: await response.json() };
    },
  });
}
```

For client-side sorting, provide `sort`:

```tsx
function useClientSortedUsers() {
  return useAsyncList<User>({
    autoReload: true,
    async load({ signal }) {
      const response = await fetch('/api/users', { signal });
      if (!response.ok) throw new Error('Could not load users');
      return { items: await response.json() };
    },
    sort({ items, descriptor }) {
      const direction = descriptor.direction === 'ascending' ? 1 : -1;

      return {
        items: items.toSorted(
          (a, b) =>
            String(a[descriptor.column]).localeCompare(String(b[descriptor.column])) * direction,
        ),
      };
    },
  });
}
```

The descriptor column is typed as a key of the item type. Prefer server sorting when pagination is
also server-owned; sorting only the currently loaded page is usually misleading.

## Dependencies

Use dependencies when an external filter should reload the list:

```tsx
function useDepartmentUsers(department: string) {
  return useAsyncList<User>({
    autoReload: true,
    dependencies: [department],
    async load({ signal }) {
      const params = new URLSearchParams({ department });
      const response = await fetch(`/api/users?${params}`, { signal });

      if (!response.ok) throw new Error('Could not load users');
      return { items: await response.json() };
    },
  });
}
```

## Error and cancellation behavior

- Check `response.ok`; `fetch` does not reject for HTTP error statuses.
- Pass `signal` to every cancellable request.
- Filtering and sorting keep the current items available while work is pending. `reload()` clears
  the current items before loading a fresh result.
- Use `reload()` for retry actions.
- Treat `error` as unknown application input and narrow it before reading `.message`.
- Use `onSuccess` and `onError` for external effects, not for duplicating the hook's item or error
  state.
