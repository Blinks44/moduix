## API Reference

`SnapCarousel` is a local moduix component. It keeps browser scrolling and CSS `scroll-snap` as the
interaction model, then adds a small composition layer with optional previous and next controls.

It does not manage active-slide state, indicators, autoplay, looping, slide announcements, or
edge-aware control disabling.

## Basic

Start with `SnapCarousel`, `SnapCarouselContent`, and the optional previous and next controls. This
is the recommended path when each direct child should become one slide.

## Anatomy

The normal consumer path is short:

```text
SnapCarousel
├─ SnapCarouselContent
├─ SnapCarouselPrevious
└─ SnapCarouselNext
```

The explicit composition path stays available:

```text
SnapCarousel
├─ SnapCarouselViewport
│  └─ SnapCarouselContent
│     └─ SnapCarouselItem
├─ SnapCarouselPrevious
└─ SnapCarouselNext
```

| Part                   | Role                                                                               |
| ---------------------- | ---------------------------------------------------------------------------------- |
| `SnapCarousel`         | Root `div` that writes `data-align` and `data-orientation`.                        |
| `SnapCarouselViewport` | Focusable native scroll container.                                                 |
| `SnapCarouselContent`  | Sugar wrapper in the default path, or the explicit flex track in full composition. |
| `SnapCarouselItem`     | One snap point.                                                                    |
| `SnapCarouselPrevious` | Optional button that scrolls backward by most of the current viewport.             |
| `SnapCarouselNext`     | Optional button that scrolls forward by most of the current viewport.              |

## Composition

### When to stay high-level

Stay with `SnapCarouselContent` directly under `SnapCarousel` when each direct child should become a
slide and you do not need to own the viewport markup.

- `SnapCarouselContent` automatically creates `SnapCarouselViewport` in that default path.
- `itemClassName` applies to the generated `SnapCarouselItem` wrappers.
- `aria-label`, `aria-labelledby`, `aria-describedby`, `tabIndex`, and `onKeyDown` are forwarded to
  the generated viewport, not the inner track.

### When to go custom

Switch to `SnapCarouselViewport`, `SnapCarouselContent`, and `SnapCarouselItem` when you need direct
viewport props, explicit item wrappers, or a different structure around each slide.

### Default props

| Part                   | Prop            | Default        | Notes                                                |
| ---------------------- | --------------- | -------------- | ---------------------------------------------------- |
| `SnapCarousel`         | `align`         | `'start'`      | Accepts `'start' \| 'center' \| 'end'`.              |
| `SnapCarousel`         | `orientation`   | `'horizontal'` | Accepts `'horizontal' \| 'vertical'`.                |
| `SnapCarouselViewport` | `tabIndex`      | `0`            | Keeps the native scroll viewport keyboard-focusable. |
| `SnapCarouselContent`  | `itemClassName` | unset          | Used only in the default sugar path.                 |
| Controls               | `type`          | `'button'`     | Applies to both previous and next controls.          |

### Behavioral notes

- Previous and next controls call native `scrollBy(...)` by about `85%` of the current viewport
  size.
- When `aria-label` or `aria-labelledby` is present on the viewport, it defaults to `role="region"`
  so the label is announced meaningfully.
- Controls get default accessible labels only when they stay icon-only.
- Native `disabled` works as expected on the controls. `aria-disabled="true"` only affects semantics
  and styling, so prevent the click action yourself when you use it.
- Vertical carousels need a bounded height from local CSS or `--snap-carousel-height`.

## Styling

Style `SnapCarousel` through `className`, the stable `data-slot` hooks, `data-align`,
`data-orientation`, and the public `--snap-carousel-*` variables from `theme.css`.

### Styling hooks

- `data-slot="snap-carousel-root"` owns the root layout and orientation-aware positioning.
- `data-slot="snap-carousel-viewport"` is the native scroll container and focus ring hook.
- `data-slot="snap-carousel-content"` is the flex track.
- `data-slot="snap-carousel-item"` is the snap item wrapper and the hook for width and snap-stop
  overrides.
- `data-slot="snap-carousel-previous"` and `data-slot="snap-carousel-next"` own the default control
  visuals.
- `SnapCarousel` writes `data-align="start" | "center" | "end"` and
  `data-orientation="horizontal" | "vertical"` on the root.
