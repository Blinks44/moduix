## API Reference

`SnapCarousel` is the small native carousel in moduix. It keeps browser scrolling and CSS
`scroll-snap` as the real interaction model, then adds a composition-first wrapper with optional
previous / next controls.

It does **not** include active-slide state, indicators, autoplay, looping, edge-aware button
disabling, or slide announcements. If you need those features, reach for a separate higher-level
carousel pattern instead of expecting this wrapper to own them.

## Default Path

Use `SnapCarousel > SnapCarouselContent` for the common case. `SnapCarouselContent` creates the
viewport automatically and wraps each direct child in a snap item.

## Anatomy

```text
SnapCarousel
├─ SnapCarouselContent
├─ SnapCarouselPrevious
└─ SnapCarouselNext

SnapCarousel
├─ SnapCarouselViewport
│  └─ SnapCarouselContent
│     └─ SnapCarouselItem
├─ SnapCarouselPrevious
└─ SnapCarouselNext
```

| Part                   | Role                                                                               |
| ---------------------- | ---------------------------------------------------------------------------------- |
| `SnapCarousel`         | Root `div` with `data-align` and `data-orientation`.                               |
| `SnapCarouselViewport` | Focusable native scroll container.                                                 |
| `SnapCarouselContent`  | Sugar wrapper in the default path, or the explicit flex track in full composition. |
| `SnapCarouselItem`     | One snap point.                                                                    |
| `SnapCarouselPrevious` | Optional button that scrolls backward by most of the viewport.                     |
| `SnapCarouselNext`     | Optional button that scrolls forward by most of the viewport.                      |

## Composition

Use the default path when each direct child should become a slide. Drop to
`SnapCarousel > SnapCarouselViewport > SnapCarouselContent > SnapCarouselItem` when you need
explicit item wrappers, direct viewport props, or different slide markup per item.

In the default path:

- `SnapCarouselContent` forwards `aria-label`, `aria-labelledby`, `aria-describedby`, `tabIndex`, and
  `onKeyDown` to the generated viewport.
- When a label is provided, the generated viewport becomes a named `region`, so the label is exposed
  meaningfully to assistive tech.
- `itemClassName` applies to the generated `SnapCarouselItem` wrappers.

## Styling API

All visual parts accept `className`. Stable styling hooks:

- `data-slot="snap-carousel-root"`
- `data-slot="snap-carousel-viewport"`
- `data-slot="snap-carousel-content"`
- `data-slot="snap-carousel-item"`
- `data-slot="snap-carousel-previous"`
- `data-slot="snap-carousel-next"`
- `data-align="start|center|end"` on the root
- `data-orientation="horizontal|vertical"` on the root

Use `--snap-carousel-item-size` to show adjacent slides and `--snap-carousel-item-gap` to control
spacing between them.

```css
.cardsCarousel {
  --snap-carousel-item-size: min(15rem, 72%);
  --snap-carousel-item-gap: var(--spacing-4);
}
```

## Accessibility and UX

- Put an accessible name on the viewport. In the default path, pass it to `SnapCarouselContent`; in
  explicit composition, pass it to `SnapCarouselViewport`.
- The viewport is focusable by default, so browser-native keyboard scrolling can work without extra
  carousel state.
- Previous / next buttons are intentionally stateless. They do not track the current slide or disable
  themselves at the edges.
- Native `disabled` works on the controls. `aria-disabled="true"` is only semantics and styling; when
  you use it, prevent the click action yourself in `onClick`.
- Slides are plain `div` wrappers. The component does not announce `"slide 2 of 5"` or add roving
  focus between slides.
