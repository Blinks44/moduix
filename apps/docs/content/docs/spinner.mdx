## Overview

`Spinner` is a compact loading primitive. Base UI does not provide a dedicated spinner primitive, so
moduix keeps this one intentionally small: one default ring indicator, size control, and standard
status semantics.

Use it for pending controls, inline loading feedback, and compact empty states. By default it
announces loading with `role="status"` and `aria-label="Loading"`. Set `decorative` when nearby
text already covers that, or pass `aria-label` / `aria-labelledby` for a more specific status name.

## Anatomy

`Spinner` exposes one public root and one internal visual slot.

```text
Spinner
└─ indicator
```

```tsx
<Spinner size="md" />
```

| Part                | Role                                                                    |
| ------------------- | ----------------------------------------------------------------------- |
| `Spinner`           | Root status element. Use `size`, `decorative`, and standard span props. |
| `spinner-indicator` | Internal visual slot that hosts the default ring or custom children.    |

## Composition

`Spinner` forwards standard `span` props. Use `size` for common density changes and `decorative`
when the spinner is paired with visible loading text. Override the announcement with `aria-label`
or `aria-labelledby` when you need a more specific status message. Pass `children` to swap the
indicator while keeping the wrapper sizing and spin behavior.

### Defaults

| Prop         | Default   | Values                       |
| ------------ | --------- | ---------------------------- |
| `size`       | `md`      | `xs`, `sm`, `md`, `lg`, `xl` |
| `decorative` | `false`   | `true`, `false`              |
| `aria-label` | `Loading` | String                       |

## Examples

### Inline Loading Text

Use `decorative` when the pending state already has visible text next to it.

<Preview>
  <SpinnerInlineExample />

## Custom Composition

Pass custom children when you need a different indicator while keeping Spinner sizing and status
semantics.
