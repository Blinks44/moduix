## API Reference

`Spinner` is a local moduix component. There is no dedicated Ark UI spinner primitive for this API.

## Basic

Start with `Spinner` for compact pending states inside buttons, inline status rows, and small empty
states. The default path announces loading automatically with `role="status"` and a fallback
`aria-label="Loading"`.

## Anatomy

The normal consumer path is short:

```text
Spinner
└─ spinner-indicator
   └─ spinner-ring (default indicator only)
```

| Part                | Role                                                                       |
| ------------------- | -------------------------------------------------------------------------- |
| `Spinner`           | Root status element. Supports `size`, `decorative`, and native span props. |
| `spinner-indicator` | Inner rotating wrapper that keeps sizing and spin behavior.                |
| `spinner-ring`      | Default ring indicator. Renders only when custom `children` are absent.    |

## Composition

### When to stay high-level

Stay with the default ring when a compact loading indicator is enough.

- Use `size` for density changes.
- Use `decorative` when adjacent visible text already communicates the loading state.
- Use `aria-label` or `aria-labelledby` when the status name should be more specific than
  `"Loading"`.

### When to go custom

`Spinner` does not expose a lower-level parts API. The custom path is `children`, which replaces the
default ring inside the same rotating wrapper.

### Default props

| Prop         | Default     | Notes                                                    |
| ------------ | ----------- | -------------------------------------------------------- |
| `size`       | `'md'`      | Accepts `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'`.          |
| `decorative` | `false`     | Switches the root from status semantics to presentation. |
| `aria-label` | `'Loading'` | Applied by default when the spinner is not decorative.   |

### Behavioral notes

- The root always renders as a `span` with `data-slot="spinner-root"` and `data-size`.
- Non-decorative spinners render with `role="status"`.
- `decorative={true}` switches the root to `role="presentation"` and `aria-hidden="true"`.
- The rotating wrapper is always `data-slot="spinner-indicator"`.
- Custom `children` replace only the default ring. The wrapper still owns sizing and rotation.
- Spinner motion is disabled automatically under `prefers-reduced-motion`.

## Styling

Style `Spinner` through `className`, the stable `data-slot` hooks, `data-size`, and the public
`--spinner-*` variables from `theme.css`.

### CSS Properties

### Styling hooks

- `data-slot="spinner-root"` is the stable selector for the exported root.
- `data-slot="spinner-indicator"` is the rotating inner wrapper.
- `data-slot="spinner-ring"` targets the default ring only.
- `Spinner` writes `data-size="xs" | "sm" | "md" | "lg" | "xl"` on the root.
- `className` applies only to the root. Target the indicator or ring with descendant selectors from
  that root when needed.
