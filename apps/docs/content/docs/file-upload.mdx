## Basic

Use `FileUpload` as the root, render a `Label`, add either a `Trigger` or `Dropzone`, list files
from `FileUpload.Context`, and keep `HiddenInput` in the tree for native form behavior.

<Preview>
  <FileUploadExample />

  <Preview.Code>
    {`
      import { FileUpload } from "@moduix/react";

      export function FileUploadDemo() {
        return (
          <FileUpload className="file-upload-demo file-upload-simple-demo" maxFiles={maxFiles}>
            <FileUpload.Label>Attachments</FileUpload.Label>
            <FileUpload.Trigger>Choose files</FileUpload.Trigger>
            <FileUpload.ItemGroup>
              <FileUpload.Context>
                {({ acceptedFiles }) =>
                  acceptedFiles.map((file) => (
                    <FileUpload.Item key={file.name} file={file}>

                      <FileUpload.ItemDeleteTrigger aria-label={\`Remove \${file.name}\`} />
                    </FileUpload.Item>
                  ))
                }
              </FileUpload.Context>
            </FileUpload.ItemGroup>
            <FileUpload.HiddenInput />
          </FileUpload>
        );
      }
    `}

## Anatomy

```text
FileUpload / FileUpload.Root
├─ FileUpload.Label
├─ FileUpload.Dropzone
│  ├─ FileUpload.DropzoneIcon
│  └─ FileUpload.Trigger
├─ FileUpload.ItemGroup[type]
│  └─ FileUpload.Context
│     └─ FileUpload.Item[file]
│        ├─ FileUpload.ItemPreview[type]
│        │  └─ FileUpload.ItemPreviewImage
│        ├─ FileUpload.ItemName
│        ├─ FileUpload.ItemSizeText
│        └─ FileUpload.ItemDeleteTrigger
├─ FileUpload.ClearTrigger
└─ FileUpload.HiddenInput
```

| Part                             | Role                                                                 |
| -------------------------------- | -------------------------------------------------------------------- |
| `FileUpload` / `FileUpload.Root` | Owns file state, validation, drag/drop, paste, and form integration. |
| `FileUpload.RootProvider`        | Renders a file upload tree from state returned by `useFileUpload`.   |
| `FileUpload.Label`               | Accessible label for the hidden input and upload controls.           |
| `FileUpload.Dropzone`            | Drag/drop target. Use `disableClick` with a nested trigger.          |
| `FileUpload.DropzoneIcon`        | Decorative upload icon helper for dropzone surfaces.                 |
| `FileUpload.Trigger`             | Button that opens the native file picker.                            |
| `FileUpload.HiddenInput`         | Native file input for forms, validation, reset, and submission.      |
| `FileUpload.ItemGroup`           | List wrapper for accepted or rejected file items.                    |
| `FileUpload.Item`                | File row. Pass the current `file` object.                            |
| `FileUpload.ItemPreview`         | Conditional preview slot matched by MIME type.                       |
| `FileUpload.ItemPreviewImage`    | Image preview rendered inside `ItemPreview type="image/*"`.          |
| `FileUpload.ItemName`            | File name text.                                                      |
| `FileUpload.ItemSizeText`        | Localized file size text.                                            |
| `FileUpload.ItemDeleteTrigger`   | Removes one file. Defaults to the moduix trash icon.                 |
| `FileUpload.ClearTrigger`        | Clears selected files. Defaults to the moduix close icon.            |
| `FileUpload.Context`             | Inline access to accepted files, rejected files, and machine APIs.   |
| `useFileUpload` / context hooks  | External state and custom child-part access.                         |

## Composition

### Recommended composition

Keep the Ark part tree visible. Use `Trigger` for compact upload controls and `Dropzone` for
drag-and-drop surfaces. If `Trigger` is nested inside `Dropzone`, pass `disableClick` to `Dropzone`
so the native file picker is not opened twice.

Render `HiddenInput` whenever the upload participates in a form. Read accepted files from
`FileUpload.Context.acceptedFiles` and rejected files from `FileUpload.Context.rejectedFiles`.

### When to go custom

Use the compact item shape for simple attachment lists. Add `ItemPreview type="image/*"` and a
fallback `ItemPreview type=".*"` when a dropzone or media flow benefits from richer rows. Use
`asChild` only when the child is a single semantic element that can preserve the required button,
label, or input behavior.

### Default props

| Prop                  | Default | Notes                                                        |
| --------------------- | ------- | ------------------------------------------------------------ |
| `allowDrop`           | `true`  | Enables drag/drop on `Dropzone`.                             |
| `maxFiles`            | `1`     | Increase or remove it for multi-file and directory uploads.  |
| `minFileSize`         | `0`     | Minimum accepted file size in bytes.                         |
| `maxFileSize`         | `∞`     | Maximum accepted file size in bytes.                         |
| `preventDocumentDrop` | `true`  | Prevents accidental navigation when dropping outside target. |
| `locale`              | `en-US` | Used for file size formatting.                               |

### Behavioral notes

`acceptedFiles`, `defaultAcceptedFiles`, and `onFileChange(details)` preserve Ark controlled and
uncontrolled state. `onFileAccept(details)` and `onFileReject(details)` keep Ark detail objects.
`Field.Root` and `Fieldset.Root` can provide disabled, invalid, required, and read-only state.
Use `ids` when separate labels or descriptions need stable accessibility links.

### Error Handling

Use `onFileReject(details)` for application feedback and keep validation constraints on the root.

<Preview>
  <FileUploadErrorHandlingExample />

  <Preview.Code>
    {`
      import { FileUpload } from "@moduix/react";
      import { useState } from "react";

      export function ErrorHandlingFileUploadDemo() {
        const [message, setMessage] = useState("");

        return (
          <div className="file-upload-stack">
            <FileUpload
              className="file-upload-demo file-upload-simple-demo"
              accept={accept}
              maxFiles={maxFiles}
              maxFileSize={maxFileSize}
              onFileReject={(details) => setMessage(\`\${details.files.length} file(s) rejected\`)}
              onFileAccept={() => setMessage("Files accepted")}
            >
              <FileUpload.Label>Images</FileUpload.Label>
              <FileUpload.Trigger>Select images</FileUpload.Trigger>
              <FileUpload.ItemGroup>
                <FileUpload.Context>
                  {({ acceptedFiles }) =>
                    acceptedFiles.map((file) => (
                      <FileUpload.Item key={file.name} file={file}>
                        <FileUpload.ItemName />
                        <FileUpload.ItemDeleteTrigger aria-label={\`Remove \${file.name}\`} />

                </FileUpload.Context>

              <FileUpload.ItemGroup type="rejected">
                <FileUpload.Context>

### Styling hooks

Target stable moduix slots such as `[data-slot="file-upload-dropzone"]`,
`[data-slot="file-upload-item"]`, and `[data-slot="file-upload-item-delete-trigger"]`.
Ark also exposes `data-scope="file-upload"`, part-specific `data-part`, and state attributes:
`data-disabled`, `data-readonly`, `data-invalid`, `data-required`, `data-dragging`, and `data-type`.
