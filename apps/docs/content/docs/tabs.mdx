`Tabs` keeps the default path small:

- `Tabs` owns selected state and orientation
- `TabsList` renders the default active indicator for you
- `TabsTab` and `TabsPanel` stay close to the Base UI primitive API

`Tabs` does not impose layout width. Let the surrounding layout or your own `className` decide how
wide the control should be.

## Anatomy

`Tabs` keeps the default path small: root, list, tabs, and panels.
`TabsList` renders the default indicator internally. Add `TabsIndicator` only when you need to
override indicator props or styles directly.


```tsx
<Tabs defaultValue="overview">
  <TabsList>
    <TabsTab value="overview">Overview</TabsTab>
    <TabsTab value="projects">Projects</TabsTab>
  </TabsList>
  <TabsPanel value="overview">Overview content</TabsPanel>
  <TabsPanel value="projects">Projects content</TabsPanel>
</Tabs>
```

| Part            | Role                                                                                            |
| --------------- | ----------------------------------------------------------------------------------------------- |
| `Tabs`          | Root state machine for selected value, orientation, variant, and controlled/uncontrolled state. |
| `TabsList`      | Groups tabs, owns keyboard list behavior, and renders the default active indicator.             |
| `TabsIndicator` | Optional low-level escape hatch for custom indicator props or styling.                          |
| `TabsTab`       | Interactive tab button. Its `value` must match a `TabsPanel` value.                             |
| `TabsPanel`     | Content region for the active tab. Use `keepMounted` when hidden DOM must stay mounted.         |

## Composition

Use `Tabs` for state props such as `defaultValue`, `value`, `onValueChange`, `orientation`, and
`variant`.

Base UI behavior stays on the matching visible parts:

- `TabsList`: `activateOnFocus`, `loopFocus`
- `TabsTab`: `disabled`, `render`, `nativeButton`
- `TabsPanel`: `keepMounted`

Visible parts accept `className`. For the common case, let `TabsList` render the built-in
indicator. When you need direct control over indicator props such as `renderBeforeHydration`, or a
custom indicator class, compose `TabsIndicator` inside `TabsList`. Adding an explicit
`TabsIndicator` replaces the built-in one for that list.

### Defaults

| Prop               | Default      | Values                   |
| ------------------ | ------------ | ------------------------ |
| `Tabs.variant`     | `default`    | `default`, `line`        |
| `Tabs.orientation` | `horizontal` | `horizontal`, `vertical` |

```tsx
import { Tabs, TabsIndicator, TabsList, TabsPanel, TabsTab } from 'moduix';
import styles from './custom-indicator-tabs-demo.module.css';

export function CustomIndicatorTabsDemo() {
  return (
    <Tabs defaultValue="profile" className={styles.root}>
      <TabsList className={styles.list}>
        <TabsTab value="profile" className={styles.tab}>
          Profile
        </TabsTab>
        <TabsTab value="security" className={styles.tab}>
          Security
        </TabsTab>
        <TabsIndicator className={styles.indicator} renderBeforeHydration />
      </TabsList>
      <TabsPanel value="profile" className={styles.panel}>
        Profile settings
      </TabsPanel>
      <TabsPanel value="security" className={styles.panel}>
        Security settings
      </TabsPanel>
    </Tabs>
  );
}
```
